지금부터 선생님 모드의 수업 일지 기능을 아래 구조로 확장하라.

이번 작업 목표는
1. 수업 탭 안에 “수업 일지 인덱스”를 만들고
2. 학생 기준으로 해당 학생에게 노출된 모든 일지 이력을 볼 수 있게 하고
3. 검색/요일/시간 필터를 넣고
4. 일지 작성 화면에서 “문장 불러오기” 팝업을 통해 문장을 미리 조합한 후 삽입하며
5. 바로 전 삽입 취소 / 전체 삭제 / 통합 검색까지 가능하게 만드는 것이다.

중요:
- 기존 수업 탭, 주간/일간/월간, 반 상세, 반 공통 일지/학생별 추가 일지 구조는 유지
- 기존 일지 작성 목적은 절대 훼손 금지
- 기능을 새로 덧붙이는 방식으로 구현
- 이번 작업은 실제 작동과 테스트 완료까지 포함한다

==================================================
[1. 수업 탭 상단 구조 수정]

현재 수업 탭에서
주간 / 일간 / 월간 구조를 유지하되
“수업 일지” 진입 요소를 추가하라.

규칙:
- 기존 선택 버튼은 반등록 아래로 내리는 현재 방향 유지 가능
- 상단/근처에 “수업 일지” 진입 버튼 또는 탭 추가
- 이 버튼은 “오늘 작성해야 하는 일지 작성 화면”이 아니라
  “작성된 일지 목록 인덱스 화면”으로 이동해야 함

즉:
수업 일지 = 일지 인덱스 페이지

==================================================
[2. 홈의 미작성 일지와 수업 일지의 역할 구분]

1. 홈 > 미작성 일지
- 오늘 스케줄 기준
- 아직 안 쓴 일지만 보여주는 빠른 작성 진입용

2. 수업 탭 > 수업 일지
- 작성된 일지 인덱스
- 조회/수정/검색/필터용

이 둘은 목적이 다르므로 분리 유지

==================================================
[3. 일지 인덱스 화면 - 핵심 규칙]

수업 일지 버튼을 누르면
일지 목록 인덱스 페이지가 열려야 한다.

이 인덱스의 핵심 규칙:
“학생에게 실제로 보여지는 모든 일지 내용이 다 보여야 한다”

즉:
한 학생 기준으로 보면 아래가 모두 이력에 잡혀야 한다.

1. 반 공통 일지
- 그 학생이 속한 반에서 작성된 공통 일지

2. 학생별 추가 일지
- 그 학생에게만 추가된 개별 일지

즉 일지 인덱스는
반 공통/학생별 추가를 따로 떼는 것이 아니라
“학생에게 노출된 전체 일지 이력”을 보여주는 구조여야 한다.

==================================================
[4. 일지 인덱스 표시 방식]

최신 작성 일지가 가장 위로 올라오게 정렬한다.

각 항목은 최소 아래 정보를 표시:
- 작성일
- 요일
- 시간
- 반 이름
- 학생 이름(관련 학생이 있으면)
- 일지 유형 배지
  - 반 공통
  - 학생 추가
  - 또는 학생에게 보여지는 통합 이력 안에서 구분 표시

예:
- 3월 19일 목요일 / 수 14:00반 / 반 공통 일지
- 3월 19일 목요일 / 수 14:00반 / 서태웅 추가 일지

항목 클릭 시:
- 해당 일지 상세/수정 화면으로 진입

==================================================
[5. 일지 인덱스 검색 기능]

일지 인덱스 화면 상단에 검색창을 추가하라.

검색 기준:
- 학생 이름 입력 시 해당 학생에게 노출된 일지만 필터링
- 즉 반 공통 일지라도 그 학생이 포함된 반이면 검색 결과에 보여야 함
- 학생별 추가 일지도 당연히 보여야 함

중요:
- 검색창은 상단 고정
- 입력 즉시 필터링
- 검색어 비우면 전체 목록 복원

==================================================
[6. 일지 인덱스 필터 기능]

검색창 옆 또는 아래에 선택형 필터를 추가하라.

필터 2개:
1. 요일
2. 시간

--------------------------------------
[6-1. 요일 필터]

요일 선택형 팝업 또는 드롭다운
선택 항목:
- 월
- 화
- 수
- 목
- 금
- 토

일요일은 현재 수업 없으면 제외 가능

--------------------------------------
[6-2. 시간 필터]

시간 선택형 팝업 또는 드롭다운
규칙:
- 수업이 실제 있는 시간만 표시
- 예: 오후 1시~9시 사이 활성 시간만
- 수업 없는 시간은 표시하지 않음

예:
- 13:00
- 14:00
- 15:00
- 16:00
- 19:00
- 20:00

==================================================
[7. 필터 동작 규칙]

- 검색 + 요일 + 시간 필터는 동시에 적용 가능
- 선택형 필터는 누르면 팝업/선택창으로 열림
- 선택 후 즉시 목록 반영
- 필터 해제 시 전체 목록 복원

==================================================
[8. 일지 작성 화면 - 문장 불러오기 버튼 추가]

현재 일지 작성 화면 구조:
- 반 공통 일지
- 학생별 추가 일지

이 구조는 유지하고
각 입력 영역에 “문장 불러오기” 기능을 추가하라.

적용 위치:
1. 반 공통 일지 영역
2. 학생별 추가 일지 영역

문장 불러오기 버튼을 누르면
현재 화면을 벗어나지 않는 하단 시트형 팝업(바텀시트)이 열려야 한다.

==================================================
[9. 문장 불러오기 팝업 구조]

팝업은 단순 리스트가 아니라
“미리 조합/미리보기 후 삽입” 구조여야 한다.

상단:
- 제목: 문장 불러오기
- 닫기 버튼
- 검색창

카테고리 탭:
- 초급
- 중급
- 상급
- 커스텀

각 카테고리:
- 최대 30개 문장 슬롯 표시 가능
- 현재는 기본 문장 세트 또는 샘플 문장으로 시작 가능

중요:
- 문장 클릭 시 곧바로 본문에 넣지 말고
- 팝업 내부의 “미리보기 영역”에 먼저 쌓이게 할 것

==================================================
[10. 문장 통합 검색 규칙 - 중요]

문장 불러오기 팝업 안 검색은
카테고리별 검색이 아니라 “전체 문장 통합 검색”으로 동작해야 한다.

규칙:
- 검색어를 입력하면
  초급/중급/상급/커스텀 구분 없이
  전체 문장 중 키워드가 포함된 문장을 리스트업
- 검색 결과에는 어느 카테고리 문장인지 작은 배지로 표시 가능
- 검색어가 없을 때만 카테고리 탭별 기본 리스트를 보여줌

즉:
검색 상태에서는 카테고리 경계를 무시하고 전체 문장을 섞어서 보여줄 것

예:
검색어 “호흡”
→ 초급/중급/상급/커스텀에 있는 “호흡” 관련 문장 전부 결과에 표시

==================================================
[11. 문장 불러오기 팝업 - 미리보기 방식]

팝업 내부에 “삽입 예정 미리보기 영역”을 둔다.

동작:
- 문장 하나 클릭 → 미리보기 영역에 1문장 추가
- 또 클릭 → 그 아래 줄에 추가
- 문장 단위로 줄바꿈되어 쌓임

규칙:
- 한 문장 = 한 번 클릭
- 문단 자동 띄움 없음
- 문장마다 다음 줄로 자연스럽게 이어짐
- 사용자가 엔터 치는 기능은 실제 일지 본문에서 직접 수행

==================================================
[12. 문장 삽입 완료 / 취소 / 삭제 기능]

팝업 안에 아래 버튼이 있어야 한다.

1. 완료
- 미리보기 영역의 문장들을 현재 선택된 입력 영역(반 공통 또는 학생별 추가 일지)에 삽입

2. 취소
- 이번 팝업에서 조합한 내용 버리고 닫기

3. 바로 전 삽입 취소
- 미리보기 영역에서 마지막으로 추가한 문장 1개만 제거

4. 전체 삭제
- 미리보기 영역에 조합한 문장 전체 비우기

중요:
- “바로 전 삽입 취소”와 “전체 삭제” 둘 다 반드시 있어야 함
- 미리보기 기반이어야 이 두 기능이 가능함

==================================================
[13. 삽입 완료 후 본문 반영 규칙]

완료를 누르면
현재 선택된 입력칸의 커서 위치에
미리보기 문장들이 삽입된다.

규칙:
- 문장 단위 줄바꿈 유지
- 자동 공백 문단 추가 금지
- 기존 본문 앞/중간/뒤 어디든 커서 위치 기준 삽입
- 삽입 후 팝업 닫기

즉:
사용자가 현재 커서를 둔 위치에
완성된 문장 묶음이 들어가야 한다

==================================================
[14. 학생별 추가 일지에서도 동일하게 적용]

학생별 추가 일지에서도
같은 문장 불러오기 팝업을 재사용한다.

차이:
- 삽입 대상만 다름
- 선택된 학생의 추가 일지 입력칸에 들어감

즉:
반 공통 / 학생별 추가 모두 같은 팝업 구조 사용

==================================================
[15. 문장 세팅은 이번 단계에서 준비만]

이번 작업은
문장 선택/삽입 기능을 작동시키는 게 우선이다.

우선 목표:
- 초급/중급/상급/커스텀 카테고리 구조 존재
- 각 카테고리당 30개 슬롯 구조 준비
- 현재는 기본 샘플 문장 또는 임시 기본값으로 동작 가능

문장 세팅 편집 UI는 다음 단계로 미뤄도 됨

==================================================
[16. 기존 기능 유지]

반드시 유지:
- 수업 탭 구조
- 주간 스케줄표
- 반 선택 기능
- 기존 일지 작성 화면
- 학생별 추가 일지 구조
- 홈의 미작성 일지 진입 기능

이번 작업은 위 기능을 깨지 않고
인덱스/검색/필터/문장 불러오기 팝업을 추가하는 것이다

==================================================
[17. 먼저 확인할 것]

작업 전에 아래를 먼저 파악하라.

1. 수업 탭 메인 파일
2. 현재 반 상세/일지 작성 화면 파일
3. 홈의 미작성 일지 진입 파일
4. 일지 저장 구조 파일
5. 학생별 추가 일지 파일
6. 현재 사용 중인 공통 팝업/바텀시트 컴포넌트
7. 커서 위치 저장/텍스트 삽입 가능한 입력 컴포넌트 구조
8. 학생별로 노출되는 일지 이력 계산 로직

==================================================
[18. 테스트 - 반드시 실제 수행]

테스트 1.
수업 탭에서 수업 일지 버튼/탭을 눌렀을 때 일지 인덱스 화면으로 이동하는지 확인

테스트 2.
일지 인덱스가 최신 작성 순으로 정렬되는지 확인

테스트 3.
학생 이름 검색 시 그 학생에게 노출된 반 공통 일지 + 학생별 추가 일지가 모두 보이는지 확인

테스트 4.
요일 필터 선택 시 월~토 중 선택한 요일만 보이는지 확인

테스트 5.
시간 필터 선택 시 실제 활성화된 수업 시간만 선택 목록에 보이는지 확인

테스트 6.
반 공통 일지에서 문장 불러오기 버튼 클릭 시 바텀시트 팝업이 열리는지 확인

테스트 7.
검색어 입력 시 카테고리 구분 없이 전체 문장에서 결과가 리스트업되는지 확인

테스트 8.
문장 클릭 시 본문에 바로 들어가지 않고 미리보기 영역에 쌓이는지 확인

테스트 9.
바로 전 삽입 취소 버튼이 마지막 문장 1개만 제거하는지 확인

테스트 10.
전체 삭제 버튼이 미리보기 전체를 비우는지 확인

테스트 11.
완료 버튼을 누르면 현재 커서 위치에 문장들이 삽입되는지 확인

테스트 12.
학생별 추가 일지에서도 같은 팝업이 정상 동작하는지 확인

테스트 13.
기존 주간 스케줄표/홈 미작성 일지 기능이 깨지지 않았는지 확인

작업 후 아래 순서로 보고하라.

5. 문장 불러오기 팝업 구조
6. 통합 검색 구현 방식
7. 미리보기/바로 전 삽입 취소/전체 삭제 처리 방식
8. 커서 위치 삽입 처리 방식
9. 테스트 1~13 결과
10. 아직 남은 다음 단계(문장 세팅 편집 UI 등)가 있으면 명시

[20. 절대 금지]

- 학생 검색 시 학생별 추가 일지만 보이고 반 공통 일지가 빠지는 것 금지
- 문장 검색을 카테고리별로 따로만 동작하게 만드는 것 금지
- 문장 클릭 시 바로 본문에 넣어버리는 방식 금지
- 미리보기 없이 취소/전체삭제 불가능한 상태 금지
- 검색창만 있고 실제 필터 동작 안 되는 상태 금지
- 수업 없는 시간까지 필터에 전부 노출하는 것 금지
- 기존 일지 작성 기능 깨뜨리기 금지
- 주간 스케줄표 동선 망가뜨리기 금지

이번 작업은
수업 일지 인덱스를 학생 기준 전체 이력 구조로 만들고,
학생 이름 검색 시 그 학생에게 노출된 모든 일지를 볼 수 있게 하며,
문장 불러오기 팝업에서는 카테고리와 무관한 통합 검색 + 미리보기 조합 + 바로 전 삽입 취소/전체 삭제까지 실제로 동작하게 만드는 작업이다.
