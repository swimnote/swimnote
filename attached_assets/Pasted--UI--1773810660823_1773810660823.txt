지금부터 앱 전체 메뉴 구조를 최종 구조로 재정리하라.
이번 작업은 단순 UI 수정이 아니라,
네 가지 모드(슈퍼관리자 / 관리자 / 선생님 / 학부모)의
탭 구조, 탭 내부 기능 배치, 중복 제거, 기능 연결 구조를 최종 정리하는 작업이다.

가장 중요:
- 기존 기능 로직, 데이터 구조, 저장 구조, 승인 구조, 회원가입 구조, 출결 구조, 정산 구조는 최대한 건드리지 말 것
- 기존 기능은 삭제하지 말고, “적절한 탭 안으로 이동/통합”할 것
- 같은 기능이 여러 탭/여러 메뉴에 중복되어 있으면 반드시 하나로 정리할 것
- 탭은 “주제별 작업 공간”이어야 하며, 그 탭 안에서 관련 작업이 끝나게 만들 것
- 다른 모드와 연결되는 기능은 끊기지 않게 반드시 확인할 것
- 작업 후 실제 테스트까지 수행하고 결과를 상세히 보고할 것

==================================================
[0. 작업 시작 전 필수 출력]

코드를 수정하기 전에 반드시 먼저 아래를 출력하라.

1. 전체 파일 구조 트리
2. 현재 각 모드 하단 탭을 구성하는 파일 경로
3. 현재 각 모드의 홈 화면 파일 경로
4. 현재 각 기능(출결, 일지, 회원관리, 승인, 정산, 메신저, 공지)의 화면 파일 경로
5. 중복 기능으로 보이는 화면 목록
   - 예: 선생님 출결탭 / 선생님 수업 안 출결 / 관리자 승인탭 / 관리자 사람 안 승인 등
6. 이동/통합이 필요한 주요 기능 목록

이 정보를 먼저 출력한 뒤에만 작업을 시작하라.

==================================================
[1. 이번 작업의 최종 목표]

네 가지 모드를 아래 최종 구조로 재정리한다.

핵심 원칙:
1. 탭 = 주제
2. 탭 안에서 끝낸다
3. 중복 탭 제거
4. 연결 버튼으로 이어준다
5. 같은 앱처럼 느껴지게 구조 통일

즉:
- “출결 따로, 일지 따로”처럼 쪼개지지 않게
- “회원관리 따로, 승인 따로”처럼 흩어지지 않게
- 한 탭 안에서 관련 작업을 연속으로 끝낼 수 있게 정리할 것

[2. 선생님 모드 최종 구조]

선생님 하단 탭 최종 구조는 아래로 고정한다.

- 홈
- 수업
- 메신저
- 정산
- 더보기

--------------------------------------
[2-1. 선생님 홈]

역할:
- 오늘 해야 할 일 요약
- 빠른 진입

포함 내용:
- 오늘 수업 리스트

- 보강 예정
- 미실시 수업
- 읽지 않은 메신저
- 빠른 버튼(오늘 수업 바로가기 등)

--------------------------------------
[2-2. 선생님 수업 탭]  ← 핵심 탭

여기서 아래 기능을 전부 끝낼 수 있게 한다.

포함 기능:
- 주간 시간표
- 반 선택
- 반별 출석 / 결석 / 보강 처리
- 반별 수업일지 작성/수정/조회
- 학생 상태 확인
- 임시 이동 처리
- 미등록 학생 교체
- 결근 처리
- 기타 수업 생성

중요:
선생님의 기존 “출결 탭”과 “일지 탭”은 별도 메인 탭으로 유지하지 말고
수업 탭 안으로 통합한다.

즉:
- 선생님 출결 단독 탭 제거/비노출
- 선생님 일지 단독 탭 제거/비노출
- 단, 기능은 삭제하지 말고 수업 탭 내부 진입 구조로 옮길 것

--------------------------------------
[2-3. 선생님 메신저 탭]

포함 기능:
- 전체 메시지
- 대상 선택 메시지
- 사진 전송
- 회원이전 요청
- 메신저 앨범

다른 탭에 흩어져 있는 메신저 관련 진입이 있으면 이 탭으로 모을 것

--------------------------------------
[2-4. 선생님 정산 탭]

포함 기능:
- 회원별 진행 횟수
- 체험 수업 카운트
- 임시 이동 수업 카운트
- 기타 수업 정산
- 미실시 / 보강 반영
- 소멸 처리
- 월 저장
- 다음 달 시작

즉:
돈 / 횟수 / 마감 / 다음 달 시작을 한 탭 안에서 끝내게 할 것

--------------------------------------
[2-5. 선생님 더보기]

포함 기능:
- 자주 쓰는 문장
- 내 정보
- 설정
- 드물게 쓰는 기능

중요:
출결관리, 반관리, 공지 관련 기능이 따로 흩어져 있으면
수업 탭 안으로 통합하라.

즉:
- 출결 단독 탭 제거/비노출
- 반관리 단독 탭 제거/비노출
- 공지 관련 진입도 가능하면 수업 탭 안으로 연결

포함 기능:

[6. 중복 제거 규칙]

아래 기준으로 중복을 제거하라.

1. 같은 기능이 단독 탭과 하위 메뉴에 동시에 있으면
   단독 탭을 제거/비노출하고 하위 메뉴로 통합한다

2. 출결, 일지, 반관리는 “수업” 주제 안으로 묶는다

3. 회원, 학부모, 선생님, 승인은 “사람” 주제 안으로 묶는다

4. 매출, 단가, 다음 달 생성, 보강 이월은 “정산” 주제 안으로 묶는다

5. 승인 기능은 관련 주제 탭 안으로 넣고 별도 메인 탭으로 두지 않는다

6. 기능 삭제는 금지
   단, 메인 탭/메뉴에서 중복 노출되는 진입만 제거/비노출하고
   실제 기능 로직은 관련 주제 탭 안으로 옮길 것

==================================================
[7. 연결 규칙 - 매우 중요]

기능 이동/통합 후 아래 연결이 반드시 살아 있어야 한다.

1. 관리자 승인 → 선생님 로그인 가능
2. 선생님 수업피드백 작성 → 학부모 화면 반영
3. 관리자 휴무일 설정 → 선생님 스케줄 반영
4. 선생님 결근 처리 → 보강/미실시/정산 반영
5. 관리자 단가표 → 선생님 정산 계산 반영
6. 슈퍼관리자 승인 → 관리자 계정 활성화
7. 회원 상태 변경 → 수업/정산/출결에 영향 반영

핵심:
탭 구조를 바꾸더라도 데이터 흐름은 절대 끊기면 안 된다.

==================================================
[8. 구현 방식]

반드시 아래 방식으로 구현하라.

1. 기존 기능 코드는 최대한 유지
2. 탭 구성과 진입 구조만 재배치
3. 기능을 새로 만들기보다 기존 화면을 적절한 탭 안에 이동/연결
4. 제거된 탭/메뉴는 실제 삭제보다 우선 비노출 처리해도 됨
5. 기존 버튼/기능 연결은 새 위치에서도 반드시 작동해야 함
6. 파일 구조가 너무 복잡하면 화면 라우트만 재배치하고 내부 컴포넌트는 재사용

==================================================
[9. 작업 순서]

반드시 아래 순서대로 진행하라.

1. 현재 중복 탭/중복 메뉴 목록 출력
2. 네 가지 모드별 새 탭 구조 적용
3. 중복 기능 진입 제거/비노출
4. 관련 기능을 해당 탭 안 하위 구조로 이동/연결
5. 각 모드 간 연결 기능 점검
6. 실제 테스트

중간 체크포인트 저장:
- 체크포인트 1: 현재 구조 분석 완료
- 체크포인트 2: 선생님 구조 정리 완료
- 체크포인트 3: 관리자 구조 정리 완료
- 체크포인트 4: 학부모/슈퍼관리자 구조 정리 완료
- 체크포인트 5: 연결 테스트 완료

==================================================
[10. 절대 금지]

- 기능 로직 수정 금지
- 데이터 구조 수정 금지
- 승인/회원가입/출결/일지/정산 로직 수정 금지
- 임의로 문구 변경 금지
- 버튼 기능 끊기 금지
- 테스트 로그인 삭제 금지
- 모드 진입 삭제 금지
- “기능이 없어지는 식의 단순 삭제” 금지

이번 작업은
기능 삭제가 아니라 “구조 재배치 + 중복 제거”다.

==================================================
[11. 테스트 지시 - 반드시 실제 수행]

수정 후 아래 테스트를 반드시 실제 수행하고 결과를 상세히 보고하라.

테스트 1.
선생님 모드에서 출결/일지가 수업 탭 안에서 처리 가능한지 확인

테스트 2.
선생님 모드에서 기존 출결 탭/일지 탭 중복 진입이 제거되었는지 확인

테스트 3.
관리자 모드에서 회원/학부모/선생님/승인이 사람 탭 안에서 모두 처리되는지 확인

테스트 4.
관리자 모드에서 출결/반관리/공지 관련 기능이 수업 탭 안에서 처리되는지 확인

테스트 5.
관리자 모드에서 단가/매출/다음 달 생성/보강 이월이 정산 탭 안에서 이어지는지 확인

테스트 6.
학부모 모드에서 중복 메뉴 없이 홈/수업피드백/앨범/출결/더보기 구조로 정리되었는지 확인

테스트 7.
슈퍼관리자 모드에서 승인 기능이 수영장 탭 안으로 정리되었는지 확인

테스트 8.
기능 이동 후 관리자 승인 → 선생님 로그인 / 선생님 피드백 → 학부모 반영 같은 연결이 깨지지 않았는지 확인

테스트 9.
기존 테스트 로그인 화면과 각 모드 진입이 그대로 유지되는지 확인

==================================================
[12. 보고 형식]

작업 후 아래 순서로 보고하라.

1. 현재 중복 구조를 어떻게 분석했는지
2. 모드별 최종 탭 구조
3. 제거/비노출한 중복 탭/메뉴 목록
4. 어떤 기능을 어떤 탭 안으로 이동/연결했는지
5. 모드 간 연결 확인 결과
6. 테스트 1~9 결과
7. 아직 남은 2차 작업 필요 항목
8. 다른 기능/화면에 영향 준 부분이 있으면 숨기지 말고 적기

==================================================
[13. 작업 범위 한정]

이번 작업은
“모드별 구조 정리 + 중복 제거 + 기능 이동/연결”
까지만 수행한다.

새 기능 개발이나 세부 버그 수정은 이번 작업 범위가 아니다.
