추가 확정 사항 반영해서 관리자 모드/데이터 정책 다시 수정한다.

==================================================
[1. 퇴원자 과금 정책 수정]
==================================================

현재 문구 중
"퇴원자는 과금에 포함됩니다" 문구가 있으면 전부 수정한다.

정확한 과금 기준:
- 과금 포함: 정상회원, 연기회원
- 과금 제외: 퇴원자, 탈퇴회원, 삭제회원, 미승인 학부모

퇴원자/아카이브 화면 문구 수정:
- 퇴원자는 과금에 포함되지 않습니다
- 최종 퇴원 처리 시 학부모 앱 접근 차단
- 복구 가능 기간 내에는 관리자가 복구 가능

회원 집계/과금 계산 로직도 문구와 동일하게 맞춘다.
UI 문구만 바꾸지 말고 실제 계산 조건도 수정한다.

==================================================
[2. 초대 안내 기록 정책 확정]
==================================================

초대 관련 기록은 관리자 전용으로 유지한다.
학부모/선생님이 보는 메뉴로 노출하지 않는다.

메뉴명:
- "초대 내역" -> "초대 안내 기록" 으로 수정

정책:
- 관리자가 학생/선생님 등록 후 문자 앱 호출
- 앱 다운로드/가입이 안 되었으면 같은 대상에게 다시 재안내 가능
- 플랫폼은 문자 전송 성공/실패를 추적하지 않음
- 기록에는 "문자 앱 호출 시각"만 저장

기록 항목:
- 대상 구분 (학부모 / 선생님)
- 대상 이름
- 대상 휴대폰 번호
- 수영장명
- 안내 생성 시각
- 마지막 재안내 시각
- 호출 횟수

추가 기능:
- 같은 번호로 "재안내" 버튼 제공
- 재안내 버튼 누르면 문자 앱 다시 열기
- 이전 문구 불러오기
- 최근 호출 횟수 표시

==================================================
[3. 스냅샷 이름 규칙 통일]
==================================================

모든 수영장 공통 스냅샷 이름 규칙 고정:

형식:
[수영장이름]_스냅샷_[YYYY-MM-DD]_[HH-mm]

예시:
토이키즈_스냅샷_2026-03-22_11-38

자동 스냅샷도 같은 규칙 사용:
- 토이키즈_스냅샷_2026-03-22_12-00
- 토이키즈_스냅샷_2026-03-22_13-00

필요 시 타입은 내부 필드로만 분리:
- snapshotType: auto / manual / before_restore / before_delete
화면 표시 이름은 통일 규칙 유지

즉:
- 사용자 화면에서는 이름 형식 통일
- 시스템 내부에서만 타입 구분

==================================================
[4. 백업/복구 화면 수정]
==================================================

현재 백업 없음 화면 보강:
- 마지막 자동 백업 시각 표시
- 자동 백업 주기 표시
- 첫 백업 없을 때 안내 문구 표시
- 지금 백업 버튼 유지

추가 표시:
- 자동 백업 주기: 1시간
- 최근 7일: 1시간 단위 스냅샷
- 최근 30일: 일 단위 대표 스냅샷
- 최근 90일: 주 단위 대표 스냅샷

스냅샷 카드 항목:
- 스냅샷 이름
- 생성 시각
- 생성 사유
- 스냅샷 유형(내부 표시 가능)
- 복구 버튼
- 삭제 버튼
- 비교 버튼(mock 가능)

==================================================
[5. 스냅샷 삭제 기능 보강]
==================================================

관리자가 백업 데이터를 삭제할 수 있게 유지
단, 경고문 강제 적용

삭제 경고문 정확히 표시:
"데이터 유지를 위하여 백업 데이터를 삭제할 경우 이전 데이터는 영원히 복구할 수 없습니다."

추가 경고:
- 삭제된 스냅샷은 되돌릴 수 없습니다
- 현재 운영 데이터에는 즉시 영향이 없지만 과거 복구 가능 범위가 줄어듭니다

삭제 절차:
1. 경고 팝업
2. 확인 체크
3. 삭제 버튼 활성화

로그 저장:
- snapshot_delete
- 삭제자
- 삭제 시각
- 삭제된 스냅샷 이름
- snapshotType

==================================================
[6. 이벤트 기록 버그 수정 유지 - 최우선]
==================================================

여전히 최우선 수정 대상:
- Invalid Date
- NaN:NaN
- undefined
- 잘못된 summary

필수값:
- createdAt
- summary
- actorName
- targetName

기본값 보정:
- createdAt 없으면 new Date().toISOString()
- actorName 없으면 "시스템"
- targetName 없으면 "대상 없음"
- summary 없으면 eventType 기준 기본 문구 생성

절대 화면에 노출 금지:
- Invalid Date
- NaN:NaN
- undefined

==================================================
[7. 초대 문구 정책]
==================================================

앱 링크는 수영장별이 아닌 공용 링크 사용

문자 본문에 포함:
- 스윔노트 설치 안내
- iPhone 공용 링크
- Android 공용 링크
- 수영장 검색 안내
- 학부모 승인 기준 안내

학부모 승인 기준 문구:
- 자녀 이름
- 생년월일
- 보호자 휴대폰 번호
를 수영장 등록 정보와 동일하게 입력해야 승인 가능

선생님 안내 문구:
- 등록된 휴대폰 번호로 가입
- 가입 후 수영장 검색
- 수영장 관리자 승인 또는 자동 연결 안내

==================================================
[8. 테스트 추가]
==================================================

반드시 다시 테스트:
1. 퇴원자 과금 제외 계산 반영 여부
2. 퇴원자 문구 수정 여부
3. 초대 안내 기록이 관리자만 보이는지
4. 재안내 버튼 정상 동작 여부
5. 문자 앱 열기 시 번호/본문 자동 입력 여부
6. 스냅샷 이름 규칙 통일 여부
7. 스냅샷 삭제 경고문 표시 여부
8. 백업/복구 화면에서 마지막 자동 백업 표시 여부
9. 이벤트 기록 Invalid Date / undefined 제거 여부
