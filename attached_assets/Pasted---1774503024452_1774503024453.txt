[스윔노트 — 공지사항 전체 구조 확정 + 공지함 + 다시보지않기 지시문]

이번 작업은 지원센터에서 공지를 등록하고,
등록된 공지를 관리자·선생님·학부모 모두에게 보여주며,
다시보지않기/닫기/공지함 구조까지 한 번에 정리하는 작업이다.

━━━━━━━━━━━━━━━━━━
1. 공지 대상 확정
━━━━━━━━━━━━━━━━━━

공지 대상은 아래 전체다.

- 관리자
- 선생님
- 학부모

즉,
특정 역할만 보는 공지가 아니라
기본 공지는 해당 수영장 소속 사용자 전체에게 노출한다.

원칙:
- 같은 수영장 안의 관리자/선생님/학부모 모두 공지 수신
- 다른 수영장에는 노출 금지

━━━━━━━━━━━━━━━━━━
2. 공지 노출 방식
━━━━━━━━━━━━━━━━━━

공지 노출은 "팝업/모달" 방식으로 한다.

규칙:
- 로그인 후 보여줘야 할 공지가 있으면 자동 노출
- 상세보기 페이지 따로 만들지 않는다
- 내용이 길어도 팝업 내부에서 스크롤로 끝까지 보게 한다

즉:
👉 공지는 열면 그 자리에서 다 읽는다
👉 별도 상세 페이지 없음

━━━━━━━━━━━━━━━━━━
3. 공지 팝업 구성
━━━━━━━━━━━━━━━━━━

공지 팝업에는 아래만 넣는다.

- 제목
- 작성일
- 내용 (길면 스크롤)
- 버튼 2개:
  - 닫기
  - 다시 보지 않기

[버튼 규칙]

1. 닫기
- 지금만 닫기
- 다음 로그인 때 다시 보일 수 있음

2. 다시 보지 않기
- 해당 공지 1건만 다시 안 보이게 처리
- 다른 공지에는 영향 없음

중요:
- "다시보지않기"는 전체 공지를 끄는 기능이 아님
- 해당 공지 단건 기준으로만 처리

━━━━━━━━━━━━━━━━━━
4. 다시보지않기 저장 방식
━━━━━━━━━━━━━━━━━━

공지별로 사용자 단위 숨김 기록을 저장한다.

예시 테이블 또는 기록 구조:
- notice_hidden_logs
  - id
  - notice_id
  - user_id
  - role
  - hidden_at

규칙:
- 같은 사용자가 같은 공지에 다시보지않기 누르면
  다음 로그인부터 해당 공지 자동 팝업 미노출
- 공지함에서는 볼 수 있게 유지
- 삭제가 아니라 숨김 처리

즉:
👉 메인 팝업에서는 숨김
👉 공지함에서는 다시 확인 가능

━━━━━━━━━━━━━━━━━━
5. 닫기 동작
━━━━━━━━━━━━━━━━━━

닫기 버튼은 저장형 숨김이 아니다.

규칙:
- 닫기만 누르면 읽음 처리 또는 세션 종료 처리만 가능
- 다음 로그인 시 다시 보여줄 수 있음
- 다시보지않기와 절대 같은 동작이면 안 됨

즉:
- 닫기 = 이번에만 닫기
- 다시보지않기 = 이 공지 다시 자동팝업 금지

━━━━━━━━━━━━━━━━━━
6. 공지함 구조
━━━━━━━━━━━━━━━━━━

공지함은 관리자/선생님/학부모 전부 제공한다.

위치:
- 개인정보
- 톱니바퀴(설정) 내부
- "공지사항" 또는 "공지함" 메뉴로 진입

규칙:
- 지난 공지는 공지함에서 확인
- 자동 팝업에서 숨긴 공지도 공지함에서는 확인 가능
- 공지함에서는 목록 → 탭 → 내용 보기 구조

중요:
- 공지 상세 별도 페이지는 만들지 않음
- 공지함에서도 항목 누르면 팝업 또는 같은 화면 확장/모달로 내용 표시

━━━━━━━━━━━━━━━━━━
7. 공지함 목록 구성
━━━━━━━━━━━━━━━━━━

공지함 목록에는 아래만 간단히 표시:

- 제목
- 작성일
- 작성 주체 또는 수영장명
- 새 공지 여부
- 숨김 여부(선택)

너무 많은 정보 금지

항목 누르면:
- 바로 내용 표시
- 길면 스크롤

━━━━━━━━━━━━━━━━━━
8. 지원센터 공지 등록 구조
━━━━━━━━━━━━━━━━━━

지원센터에서 공지 작성 가능하게 한다.

작성 항목:
- 수영장 선택
- 제목
- 내용
- 상태

등록 시:
1. notices 저장
2. 해당 수영장 소속 관리자/선생님/학부모 대상 푸시 발송
3. 다음 로그인 시 공지 팝업 노출 대상에 포함

━━━━━━━━━━━━━━━━━━
9. 푸시 발송 조건
━━━━━━━━━━━━━━━━━━

공지 등록 시 푸시는 해당 수영장 전체 사용자에게 발송:

- 관리자
- 선생님
- 학부모

조건:
- 동일 수영장만
- 비활성/탈퇴 제외
- push token 있는 계정만 발송

푸시 제목 예:
[토이키즈] 공지사항

푸시 내용 예:
4월 휴강 일정 안내

━━━━━━━━━━━━━━━━━━
10. 공지 읽음/노출 기준
━━━━━━━━━━━━━━━━━━

공지 노출 판단 기준 예시:

- 숨김 기록 없음
- 활성 공지 상태
- 대상 수영장 일치
- 로그인 사용자 역할이 관리자/선생님/학부모 중 하나
- 공지 노출 기간 내(기간 기능 있으면 적용)

권장:
- 최신 공지 우선 노출
- 여러 개면 순서대로 보이거나 목록형 팝업 검토 가능
- 처음에는 가장 최신 1건 우선 노출도 가능

━━━━━━━━━━━━━━━━━━
11. 관리자/선생님/학부모 공통 규칙
━━━━━━━━━━━━━━━━━━

세 역할 모두 동일 규칙:

- 로그인 후 공지 자동팝업 가능
- 닫기 가능
- 다시보지않기 가능
- 설정(개인정보/톱니바퀴) 안에서 공지함 진입 가능
- 공지함에서 지난 공지 확인 가능

즉:
👉 역할별로 공지 보는 방법이 다르면 안 됨
👉 진입 위치만 각 화면 설정 메뉴 안으로 맞춘다

━━━━━━━━━━━━━━━━━━
12. 금지 사항
━━━━━━━━━━━━━━━━━━

- 공지 상세 페이지 따로 만들기
- 다시보지않기를 전체 공지 차단처럼 만들기
- 닫기와 다시보지않기를 같은 동작으로 만들기
- 공지함 없이 팝업에서만 보이게 만들기
- 다른 수영장 공지까지 보이게 만들기
- 푸시만 보내고 앱 안에서 공지 저장 안 하기

━━━━━━━━━━━━━━━━━━
13. 실제 테스트
━━━━━━━━━━━━━━━━━━

반드시 실제로 확인:

1. 지원센터에서 공지 등록 가능
2. notices DB 저장 확인
3. 해당 수영장 관리자/선생님/학부모에 푸시 발송 확인
4. 로그인 시 공지 팝업 자동 노출 확인
5. 긴 내용 스크롤 확인
6. 닫기 누르면 다음 로그인에 다시 보이는지 확인
7. 다시보지않기 누르면 다음 로그인에 자동 팝업 안 뜨는지 확인
8. 공지함에서 해당 공지 다시 볼 수 있는지 확인
9. 관리자/선생님/학부모 모두 설정 안에서 공지함 진입 가능한지 확인
10. 다른 수영장 계정에는 안 보이는지 확인

━━━━━━━━━━━━━━━━━━
14. 보고 형식
━━━━━━━━━━━━━━━━━━

반드시 포함:

- 수정 파일 경로
- 지원센터 공지 등록 화면 구성
- 공지 팝업 UI 구성
- 닫기 / 다시보지않기 동작 차이
- 공지함 위치 및 화면 구성
- notices 저장 결과
- 숨김 로그 저장 결과
- 푸시 발송 결과
- 역할별(관리자/선생님/학부모) 노출 테스트 결과

“공지 추가 완료”만 쓰지 말고
실제 저장/노출/숨김/공지함 동작 결과 같이 제출할 것

━━━━━━━━━━━━━━━━━━
