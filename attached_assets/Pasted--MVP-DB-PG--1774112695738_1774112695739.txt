프로젝트: 스윔노트 전국 수영 레슨 운영 플랫폼 MVP
현재 작업 목표:
DB 연결 직전 / PG 연결 직전까지 실제 작동하는 구조로 코딩한다.
즉, 화면만 만드는 게 아니라 상태/정책/용량/구독/리스크/로그가 mock store 기준으로 실제로 작동해야 한다.

절대 금지:
- undefined 표시
- Invalid Date 표시
- 빈 화면 방치
- 버튼 눌렀는데 상태 안 바뀌는 구조
- 같은 데이터를 화면마다 따로 관리하는 구조
- 업로드 정책/플랜 정책/노출 정책이 섞이는 구조

export function safeText(value: unknown, fallback = '-') {
  if (value === null || value === undefined) return fallback
  const str = String(value).trim()
  return str.length ? str : fallback
}

export function formatDateSafe(value?: string | null, fallback = '-') {
  if (!value) return fallback
  const d = new Date(value)
  if (Number.isNaN(d.getTime())) return fallback
  return d.toLocaleString('ko-KR')
}

export function calcPercent(used: number, total: number) {
  if (!total || total <= 0) return 0
  return Math.min(999, Math.round((used / total) * 100))
}

- undefined 노출 금지
- Invalid Date 노출 금지

기술 구현:
- 파일 업로드 시 MediaAsset 생성
- 상태 흐름:
  uploading -> uploaded -> processing -> ready
- previewFreeSizeMb 와 previewPaidSizeMb 를 mock 계산해서 저장
- 무료 운영자/무료 범위는 previewFree 사용
- 유료 운영자는 previewPaid 사용

중요:
플랜 변경 시 재업로드 금지

환불 정책 기본 문구는 아래로 seed 저장

구독 변경은 즉시 적용됩니다.
상위 플랜 변경 시 남은 기간 기준 차액이 즉시 결제됩니다.
하위 플랜 변경 시 남은 기간 기준 차액은 환불되지 않고, 다음 결제 시 차감되는 크레딧으로 적립됩니다.
구독 해지 시 유료 기능은 즉시 제한되며, 서비스는 읽기전용 상태로 전환됩니다.
구독 해지 후 24시간이 경과하면 저장된 사진 및 영상 데이터는 자동 삭제되며 복구되지 않습니다.
이미 결제된 이용요금은 원칙적으로 환불되지 않습니다.
단, 다음 결제가 발생하지 않는 상태에서 남아 있는 크레딧은 환불될 수 있습니다.
사용자는 구독 해지 전 데이터 삭제 정책을 충분히 확인해야 하며, 삭제된 데이터는 복구되지 않습니다.

운영자는 읽기 전용만 가능
슈퍼관리자는 수정 가능
버전 기록 남기기
