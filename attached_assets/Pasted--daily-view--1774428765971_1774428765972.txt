지금 반영된 선생님 daily view 구현은 내가 지시한 구조와 다르다.
기능이 일부 적용된 것은 맞지만, 구현 해석이 잘못됐다.
이번에는 “실행 파일 확인 후 그 파일만 수정” 방식 그대로 유지하면서, 아래 오해된 부분을 정확히 다시 고쳐라.

중요:
- 선생님 모드만 수정
- 관리자 모드 절대 변경 금지
- 화면 이동 최소화
- 실무 작업은 현재 흐름 안에서 끝내기
- 버튼을 페이지 이동용으로 쓰지 말고, 현재 화면 즉시 처리용으로 바꿔라

━━━━━━━━━━━━━━━━━━
1. 현재 구현에서 잘못된 점
━━━━━━━━━━━━━━━━━━

1) [출석] / [결석] 버튼이 출결 화면으로 이동하도록 구현됨
→ 틀림
→ 이 버튼들은 페이지 이동 버튼이 아니라 “학생 행에서 즉시 상태 변경” 버튼이어야 한다

2) [반이동] 버튼이 학생 상세 화면으로 이동함
→ 틀림
→ 이 버튼은 “학생 상세 보기”가 아니라 “반이동 작업 시작” 버튼이어야 한다

3) 팝업/페이지 이동 규칙이 적용되지 않음
→ 스케줄러 실무 흐름은 최대한 현재 바텀시트/팝업 안에서 끝내야 한다
→ 불필요한 별도 페이지 이동 금지

━━━━━━━━━━━━━━━━━━
2. 선생님 daily view 최종 규칙 (다시 고정)
━━━━━━━━━━━━━━━━━━

학생 행 구성:
- 이름 / 주 N회
- [출석] [결석] [반이동]
- 우측 화살표(>) = 학생 상세

여기서 의미는 아래와 같다.

[출석]
- 페이지 이동 금지
- 현재 학생 행에서 즉시 출석 처리
- 이미 결석 상태였다면:
  attendance = present
  해당 makeup_session 삭제
  빨간 점 제거
  이름 줄은 유지

[결석]
- 페이지 이동 금지
- 현재 학생 행에서 즉시 결석 처리
- attendance = absent
- makeup_session 생성 (pending)
- 이름 앞 빨간 점 표시
- 결석 학생 상단 정렬

[반이동]
- 학생 상세 페이지 이동 금지
- 현재 흐름 안에서 “반이동 선택 팝업/시트” 열기
- 반이동은 작업 버튼이다

[우측 화살표 >]
- 이 버튼만 학생 상세 진입 용도
- 상세 정보 보기 전용

━━━━━━━━━━━━━━━━━━
3. 출결 규칙 (선생님 모드 전용)
━━━━━━━━━━━━━━━━━━

- 수업 시간이 지나면 자동으로 출석 완료 상태
- 이름 중앙 줄 표시
- 수업 전/중/후 언제든 결석 변경 가능
- 결석 시 보강 자동 생성
- 다시 출석으로 바꾸면 보강 삭제
- 지각 없음
- 사유 없음
- 저장 버튼 없음
- 즉시 저장

즉:
[출석] / [결석]은 토글형 상태 변경 버튼이다
절대 다른 화면으로 보내면 안 된다

━━━━━━━━━━━━━━━━━━
4. 팝업/페이지 이동 규칙 재적용
━━━━━━━━━━━━━━━━━━

이번 작업부터 아래 규칙을 정확히 지켜라.

A. 스케줄러에서 시작된 작업은
가능하면 스케줄러 흐름 안에서 끝낸다

B. 작업 버튼(출석/결석/반이동/반배정)은
우선 팝업/바텀시트/인라인 처리로 해결한다

C. 별도 페이지 이동은 아래만 허용
- 학생 상세 보기
- 정말 긴 입력이 필요한 화면
- 독립 관리 화면

D. 한 작업이 끝나면
직전 단계로 자연스럽게 복귀해야 한다

E. 반이동/반제외 같은 실무 작업은
가능하면 full page가 아니라 action sheet / bottom sheet로 처리해라

━━━━━━━━━━━━━━━━━━
5. 지금 고칠 대상
━━━━━━━━━━━━━━━━━━

선생님 daily view에서 아래를 다시 구현해라.

1. [출석] 버튼
- 즉시 present 처리
- 결석 상태였으면 makeup_session 삭제
- 페이지 이동 없음

2. [결석] 버튼
- 즉시 absent 처리
- makeup_session 생성
- 페이지 이동 없음

3. [반이동] 버튼
- 학생 상세 이동 금지
- 반이동 선택 팝업/시트 오픈

4. [>] 버튼
- 학생 상세 이동 유지

5. 결석 상태 표시
- 이름 앞 빨간 점
- 수업 완료면 이름 줄 유지

6. 정렬
- 결석 학생 상단

━━━━━━━━━━━━━━━━━━
6. 실행 파일 확인 방식 유지
━━━━━━━━━━━━━━━━━━

이번에도 수정 전에 실제 실행 파일 확인 절차를 유지해라.

- 수정하는 화면 최상단에 임시 debug 텍스트 삽입
- 웹 / Expo Go 둘 다 확인
- 실제 실행 파일이 맞는지 먼저 증명
- 그 파일에만 수정 적용
- 수정 후 debug 제거

반드시 제출:
- 실제 수정 파일 경로
- 실제 route 경로
- debug 문구 확인 결과

━━━━━━━━━━━━━━━━━━
7. 검증 기준
━━━━━━━━━━━━━━━━━━

반드시 실제 앱에서 아래를 확인해라.

1. [결석] 클릭
- 페이지 이동 없이 즉시 빨간 점 표시
- makeup_session 생성
- 학생 상단 정렬

2. [출석] 클릭
- 결석 상태가 출석으로 복구
- makeup_session 삭제
- 빨간 점 제거
- 줄 유지

3. [반이동] 클릭
- 학생 상세가 아니라 반이동 팝업/시트 오픈

4. [>] 클릭
- 학생 상세 화면 이동

5. 관리자 모드
- 아무 UI 변경 없음

━━━━━━━━━━━━━━━━━━
8. 보고 형식
━━━━━━━━━━━━━━━━━━

답변 마지막에는 반드시 아래를 적어라.

1. 잘못 구현된 부분 요약
2. 실제 수정 파일 경로
3. 실제 route 경로
4. [출석]/[결석]/[반이동]/[>] 각각 최종 동작
5. makeup_session 생성/삭제 검증 결과
6. 페이지 이동 제거 여부
7. 관리자 모드 영향 없음 여부
8. 내가 직접 확인할 위치

중요:
- “수정 완료” 금지
- “페이지 이동 안 함 / 즉시 처리”를 실제로 증명해라
