이번 작업은 관리자모드 수업/선생님관리 구조 개편이다.
목표는 관리자모드와 선생님모드의 레이아웃과 흐름을 통일해서, MVP 이후 디자인 일괄 변경이 가능하도록 만드는 것이다.

[최종 확정 구조]
- 관리자에서 선생님관리로 들어가면 탭은 일간 / 월간 두 가지만 사용한다
- 관리자에서 하단 수업 탭으로 들어가면 화면은 주간 고정으로 사용한다
- 관리자 수업 탭의 주간 화면에서는 반 개설 기능까지 함께 처리한다
- 선생님 선택 화면은 카드형이 아니라 리스트형으로 만든다
- 선생님 목록은 여러 명일 경우 가나다순 정렬로 보여준다
- 반이 1개뿐이어도 예외 없이 동일한 흐름을 유지한다
- 즉, 바로 반상세로 보내지 말고 항상 같은 단계로 이동하게 한다
- 사용자가 손에 익도록 모든 경로의 구조를 통일한다

[작업 목표]
관리자모드에서
- 수업 탭은 주간 스케줄 보드 중심 구조로 고정한다
- 선생님 관리 탭은 일간 / 월간 구조로 구성한다
- 관리자모드는 선생님 전체를 관리하므로 스케줄 화면 진입 후 반드시 선생님 선택 단계가 들어가야 한다
- 단, 수업 탭은 주간 고정이지만 반 개설 기능을 포함해야 한다

[꼭 지켜야 할 구조]

1. 관리자 수업 탭 구조 = 주간 고정
- 관리자 하단 수업 탭 진입
- 주간 시간표 보드 표시
- 시간 또는 셀 선택
- 선생님 선택 페이지 진입
- 선택한 시간대에 가능한 선생님 목록 표시
- 선생님 선택
- 해당 시간대의 반목록 표시
- 반 선택
- 반 상세 현황판 진입

추가
- 관리자 수업 탭 주간 화면에서는 반 개설 버튼이 있어야 한다
- 반 개설은 선택한 요일 / 시간 기준으로 자연스럽게 이어지게 한다
- 반이 1개뿐이어도 반목록 화면을 생략하지 않는다

2. 관리자 선생님관리 구조 = 일간 / 월간
A. 일간 구조
- 관리자 선생님관리 진입
- 일간 탭 선택
- 날짜 기준 시간 또는 일정 항목 선택
- 선생님 선택 페이지 진입
- 선생님 선택
- 해당 시간대 반목록 표시
- 반 선택
- 반 상세 현황판 진입

B. 월간 구조
- 관리자 선생님관리 진입
- 월간 탭 선택
- 날짜 선택
- 해당 날짜 수업 시간대 표시
- 시간 선택
- 선생님 선택 페이지 진입
- 선생님 선택
- 해당 시간대 반목록 표시
- 반 선택
- 반 상세 현황판 진입

[화면 구조 변경 요구사항]

A. 관리자모드 수업 탭 변경
현재 카드형 반 리스트 중심 화면을 주간 고정 구조로 변경한다.

상단 공통 영역
- 상단 타이틀 유지
- 우측 상단 버튼 영역 유지 가능
- 선택, 반 등록, 보강 등 기존 기능 버튼은 유지하되 레이아웃만 통일
- 관리자 수업 탭은 일간 / 월간 탭 없이 주간 화면만 사용한다
- 이 화면은 관리자의 반 개설과 주간 조회를 동시에 수행하는 메인 화면이다

주간 탭 상세
- 두 번째 사진에 있는 선생님모드 주간 시간표 테이블 디자인을 그대로 재사용
- 관리자모드에서는 이 보드를 그대로 쓰되, 셀이나 수업을 눌렀을 때 바로 반정보로 가지 말고 먼저 선생님 선택 페이지로 이동해야 한다
- 즉 관리자 수업 구조는
  주간시간표 보드 → 선생님 선택 → 반목록 → 반정보
  흐름으로 구성한다

주간 화면 추가 요구사항
- 반 개설 버튼 제공
- 선택한 요일 / 시간 컨텍스트를 유지한 상태에서 반 개설 가능
- 반 개설 후 동일한 주간 보드에 즉시 반영되게 처리
- 뒤로가기 시 주간 보드 위치와 선택 상태 유지

B. 관리자 선생님 관리 화면 변경
현재 선생님 관리 화면은 오늘 / 주간 두 개 탭인데 아래처럼 바꾼다.

변경 후 탭 구성
- 일간
- 월간

주간 탭은 제거하고 월간으로 교체한다.

1) 선생님 관리 - 일간 탭
- 기존 오늘 확인할 것 구조는 유지 가능
- 다만 아래 일정 영역은 일별 스케줄 탐색 구조로 연결되게 수정
- 시간 또는 수업 항목을 누르면 바로 반정보가 아니라
  선생님 선택 페이지 → 반목록 → 반정보
  순서로 이동하게 한다
- 관리자모드이므로 항상 선생님 선택 단계가 먼저 들어가야 한다
- 반이 1개뿐이어도 반목록을 반드시 보여줘야 한다

2) 선생님 관리 - 월간 탭
- 네 번째 사진의 선생님모드 월간 화면 디자인 재사용
- 날짜 클릭 시 해당 날짜의 수업 시간 목록 표시
- 시간 선택 후 선생님 선택
- 선생님 선택 후 해당 반목록
- 반 선택 후 반정보 진입
- 반이 1개뿐이어도 반드시 반목록을 보여줘야 한다

[선생님 선택 페이지 추가 요구사항]
관리자모드에는 반드시 공통 컴포넌트 형태의 선생님 선택 페이지를 새로 만든다.

목적
- 관리자 수업 주간 / 관리자 선생님관리 일간 / 관리자 선생님관리 월간 흐름에서 공통 재사용
- 선택된 날짜, 요일, 시간 정보를 상단에 표시
- 해당 시간에 배정된 선생님만 노출
- 선생님이 한 명이어도 자동 진입하지 말고 동일하게 선택 화면을 보여준다
- 사용자 동선 학습을 위해 항상 같은 구조를 유지한다

표시 방식
- 카드형 금지
- 리스트형으로 표시
- 여러 명일 경우 가나다순 정렬
- 빠르게 찾을 수 있도록 이름 중심 정렬
- 추후 필요하면 초성 검색 또는 검색창 추가 가능하도록 확장성 있게 만든다

표시 항목
- 선택한 날짜
- 선택한 요일
- 선택한 시간
- 선생님 이름
- 담당 반 수
- 출결 미확인 개수
- 일지 미작성 개수
- 버튼 또는 행 클릭: 해당 선생님 선택

공통 규칙
- 관리자 수업 주간 보드에서 클릭했을 때도 이 페이지로 온다
- 관리자 선생님관리 일간에서 클릭했을 때도 이 페이지로 온다
- 관리자 선생님관리 월간에서 클릭했을 때도 이 페이지로 온다
- 컴포넌트 중복 생성 금지
- 하나의 공용 route 또는 modal 또는 sheet 구조로 재사용

[반목록 화면 규칙]
반목록은 반드시 별도 단계로 둔다.

반목록 필수 규칙
- 반이 1개뿐이어도 생략 금지
- 항상 선생님 선택 다음 단계에서 반목록을 보여준다
- 사용자가 같은 손동작으로 빠르게 이동할 수 있게 모든 흐름 동일 유지
- 반목록에서 반명, 시간, 정원 또는 현재 인원, 상태 정도를 표시
- 반 선택 후 반 상세 현황판 진입

[반정보 진입 후 구조]
반정보는 이전에 말한 구조와 동일하게 유지한다.

반정보 상세 화면 기본 요소
- 반명
- 요일 / 시간
- 담당 선생님
- 학생 수
- 학생 목록
- 학생 클릭 시 학생정보 진입
- 학생 상태 표시: 정상 / 보강 / 결석 등
- 출결 상태 표시
- 일지 작성 여부 표시
- 결석 처리 가능
- 출석 명단 확인 가능

이동 흐름
- 관리자든 선생님이든 최종적으로 들어가는 반 상세 화면은 같은 레이아웃 사용
- 관리자에서 들어가도 같은 반상세
- 선생님에서 들어가도 같은 반상세
- 권한에 따라 버튼만 다르게 표시

[레이아웃 통일 규칙]
MVP 이후 디자인 일괄 수정이 가능해야 하므로 아래를 강제한다.

공통 디자인 규칙
- 상단 헤더 높이 통일
- 탭 버튼 스타일 통일
- 리스트 행 높이 통일
- 카드 radius 통일
- 버튼 radius 통일
- 여백 spacing 토큰 통일
- 시간표 보드 셀 높이 통일
- 월간 달력 셀 크기 통일
- 리스트 내부 아이콘 크기 통일
- 색상 토큰 분리
- 상태 배지 스타일 통일

컴포넌트 분리 필수
- PageHeader
- ModeTabs
- TeacherPickerList
- ScheduleWeekBoard
- ScheduleMonthBoard
- TimeSlotList
- ClassList

- ClassDetailPanel
- StatusBadge

- LoadingState

금지사항
- 관리자용 / 선생님용 화면을 각각 따로 복붙해서 만들지 말 것
- 같은 UI를 파일만 다르게 중복 생성하지 말 것
- 주간보드 / 월간보드 UI는 하나의 공통 컴포넌트로 props만 다르게 받을 것
- 선생님 한 명일 때만 예외 처리해서 화면을 건너뛰지 말 것
- 반 하나일 때만 예외 처리해서 화면을 건너뛰지 말 것

[데이터 / 상태 구조 요구사항]
공통 상태 설계
- selectedEntryMode: class_week / teacher_day / teacher_month
- selectedDate
- selectedWeekRange
- selectedWeekday
- selectedTimeSlot
- selectedTeacherId
- selectedClassId

추천 데이터 흐름
- 날짜 또는 시간 선택
- 선생님 선택
- 반 선택
- 반상세 진입

관리자 수업 탭 주간의 반 개설 흐름
- 주간 보드에서 요일 / 시간 선택
- 반 개설 버튼 또는 반 개설 액션 진입
- 선생님 선택 또는 선생님 지정
- 반 정보 입력
- 저장
- 저장 후 동일 위치 주간 보드로 복귀
- 새로 생성된 반 즉시 반영

필수 처리
- 선택값 뒤로가기 시 유지
- 새로고침 시 최소한 현재 화면과 선택 날짜 유지
- 데이터 없을 때 빈 상태 UI 표시
- 선생님 없음 / 반 없음 / 수업 없음 상태 각각 구분 표시
- 가나다순 정렬 항상 유지

[관리자 / 선생님 권한 차이]
관리자
- 전체 선생님 스케줄 접근 가능
- 선생님 선택 단계 필수
- 반 등록, 삭제, 수정 가능
- 전체 반 현황 접근 가능

선생님
- 본인 스케줄만 보이므로 선생님 선택 단계 없음
- 기존 선생님모드 주간 / 월간 디자인 유지
- 본인 반만 접근 가능

즉
- 선생님모드는 기존 구조 유지
- 관리자모드는 같은 UI를 쓰되 선생님 선택 단계와 반목록 단계를 항상 유지
이 원칙으로 구현한다

[네비게이션 규칙]

관리자 수업 탭
- 하단 수업 탭 진입 시 주간 화면 고정
- 주간 보드 클릭 → 선생님 선택 → 반목록 → 반상세
- 반 개설도 이 화면 안에서 처리

관리자 선생님관리
- 일간 진입 가능
- 월간 진입 가능
- 일간 또는 월간에서 선택 후 선생님 선택 → 반목록 → 반상세

뒤로가기 규칙
- 반상세 → 반목록
- 반목록 → 선생님선택
- 선생님선택 → 시간선택 또는 날짜선택 화면
- 시간선택 또는 날짜선택 → 해당 탭 메인
- 뒤로가기를 눌렀을 때 선택값과 스크롤 위치가 유지되어야 한다

[테스트 및 작동확인 의무]
이번 작업은 UI만 수정이 아니라 실제로 클릭 흐름이 전부 작동하는지 확인해야 한다.

- 시간 / 셀 선택 가능

- 선생님 목록 가나다순 표시 정상
- 선생님 선택 후 반목록 진입 가능
- 반이 1개여도 반목록 표시되는지 확인
- 반정보 진입 가능

- 반 개설 후 주간 보드에 즉시 반영되는지 확인

2. 관리자 선생님관리 일간 탭
- 일간 탭 클릭 가능

- 선생님 선택 리스트 진입 가능
- 선생님 선택 후 반목록 진입 가능
- 반이 1개여도 반목록 표시되는지 확인
- 반상세 진입 가능

3. 관리자 선생님관리 월간 탭
- 월간 탭 클릭 가능
- 달력 렌더링 정상
- 날짜 선택 가능
- 시간 목록 표시 가능
- 선생님 선택 리스트 진입 가능
- 가나다순 정렬 정상
- 반목록 진입 가능
- 반이 1개여도 반목록 표시되는지 확인
- 반상세 진입 가능

4. 선생님모드

- 기존 월간 디자인 유지
- 선생님 선택 단계 없이 본인 반만 정상 접근

[반드시 제출할 것]
작업 끝났다고만 하지 말고 아래를 같이 제출할 것

제출 항목
1. 수정한 페이지 목록
2. 추가한 컴포넌트 목록
3. 변경한 라우팅 목록
4. 권한 분기 처리 방식
5. 실제 클릭 테스트 결과
6. 실패한 항목이 있으면 실패 항목과 원인
7. 임시 데이터로 확인한 시나리오
8. 선생님 리스트 정렬 방식 설명
9. 반 1개일 때도 동일 구조를 유지한 테스트 결과

- 관리자 선생님관리 > 월간 > 3/16 선택 > 15:00 선택 > 선생님선택 리스트 > 김선생 선택 > 반목록 > 반상세 진입 성공

중요
- 테스트 안 하고 끝냈다고 말하지 말 것
- 실제 클릭한 경로별 성공 / 실패 결과를 체크리스트 형태로 제출할 것
- 동작 안 되는 버튼이 하나라도 있으면 완료라고 하지 말 것

[구현 우선순위]
1. 관리자 수업 탭 주간 고정 구조 변경
2. 관리자 수업 탭 반 개설 동선 연결
3. 선생님 선택 리스트 공용 컴포넌트 추가
4. 반목록 공용 단계 추가
5. 관리자 선생님관리 화면 일간 / 월간 구조 변경
6. 공통 반상세 연결
7. 뒤로가기 상태 유지
8. 전체 클릭 테스트
9. 레이아웃 정리 및 중복 제거

[개발 원칙]
- MVP 기준으로 동작 우선
- 디자인은 현재 톤 유지
- 나중에 전체 디자인 교체 가능하도록 공통 컴포넌트화
- 복붙 구조 금지
- 하드코딩 최소화
- 관리자 / 선생님 공통 로직 최대한 공유
- 클릭 dead-end 금지
- 데이터 없는 경우도 반드시 빈 화면 처리
- 예외 흐름 최소화
- 사용자 손동작이 익숙해지도록 모든 경로의 단계 수를 통일

[추가 구현 지시]
반 선택 이후 들어가는 반상세는 이전에 말한 구조를 유지해 연결한다.
아래 기능 버튼이 이후 단계에서 이어질 수 있도록 라우팅 자리도 같이 잡아둘 것
- 학생정보
- 출결
- 결석처리
- 일지작성 여부
- 보강 표시
- 수업 현황판

지금 단계에서는 최소한 화면 진입 및 데이터 전달까지는 작동해야 한다.

[추가 검증 지시]
- 각 버튼과 탭에 대해 실제 클릭 후 이동 결과를 텍스트로 보고해라
- 콘솔 에러가 있으면 함께 보고해라
- 동작 안 하는 버튼이 하나라도 있으면 완료로 표시하지 마라
- 가능하면 임시 테스트 데이터는 2명 이상 선생님 기준으로 검증해라
- 선생님 리스트는 반드시 가나다순으로 검증해라
- 반 1개인 경우와 반 2개 이상인 경우 둘 다 테스트해라
- 리플릿에서 실제 실행 후 작동확인까지 끝낸 다음 보고해라
