DATA POLICY for Members:
Do NOT blindly hard delete unless absolutely safe.
Member records are likely connected to:
- child/student profile
- class assignment
- attendance
- lesson feedback
- photos/albums
- parent connections
- historical records

Recommended policy:
Option A: Soft Delete
- member.status = "deleted"
- hidden from normal active lists
- historical records preserved

Option B: Archive
- move to archived/deleted members collection/section
- preserve relational integrity

Hard delete is NOT recommended unless all related data is safely handled.

Please make sure:
- deleting a member does not break related parent/teacher/admin views
- deleted members do not still appear in active lists
- storage and query logic both respect deleted state

==================================================
[EDGE CASES / EXCEPTIONS]
==================================================

Handle these properly:

- 0 selected items => bulk delete disabled
- repeated click while deleting => blocked
- already deleted item => safe guard
- network failure => error feedback
- partial failure in bulk delete => partial success/failure message
- empty list after delete => proper empty state
- select all then unselect some => state remains correct
- search/filter state should not break delete flow
- paginated lists should refresh correctly
- mobile layout should not break action bar
