[스윔노트 레벨 시스템 “사용안함” 기능 추가 + 전모드 표시 제어 통합 지시서]

지금부터 기존 레벨 시스템(1~10단계)에 “사용안함(비활성)” 기능을 추가한다.
이번 작업의 목적은 수영장마다 레벨 개수가 다를 수 있기 때문에
불필요한 레벨을 숨기고 실제 사용하는 레벨만 모든 모드에서 일관되게 보이게 만드는 것이다.

중요:
- UI만 숨기는 게 아니라 데이터/선택/노출/리스트 전부에서 완전히 제외해야 한다
- 관리자/선생님/학부모 모든 화면에서 동일하게 반영되어야 한다
- 기존 레벨 시스템과 충돌 없이 확장해야 한다
- 사용안함 레벨이 실수로 노출되거나 선택되지 않도록 완전히 차단해야 한다
- 기존 기능(출결/일지/학부모/보강/학생관리) 절대 깨지지 않게 구현할 것
- 작업 후 반드시 실제 테스트까지 완료할 것

==================================================
1. 구현 목표
==================================================

- 기본 레벨은 1~10까지 유지
- 수영장 관리자가 각 레벨을 “사용” 또는 “사용안함”으로 설정 가능
- 사용안함 레벨은 아래에서 완전히 제외:
  - 선생님 화면
  - 학부모 화면
  - 학생 레벨 선택 리스트
  - 일지/피드백 표시
  - 학생 프로필 표시
- 관리자 레벨 설정 화면에서는 항상 전체 1~10 보이되 상태만 표시

==================================================
2. 레벨 사용 여부 필드 추가
==================================================

레벨 설정 데이터에 아래 필드를 추가한다.

필드:
- is_active (boolean)

기본값:
- true

동작:
- true → 사용
- false → 사용안함

==================================================
3. 관리자 레벨 설정 화면 수정
==================================================

각 레벨(1~10) 항목마다 “사용안함” 토글 또는 버튼 추가

구성:
- 레벨명
- 설명
- 학습 내용
- 테스트 기준
- 뱃지 설정
- [사용 / 사용안함 토글]

UI 동작:
- ON → 사용
- OFF → 사용안함

표시 규칙:
- 사용안함 상태일 때는 해당 레벨 카드/행을 시각적으로 비활성 표시 (회색 등)
- 하지만 관리자 화면에서는 항상 보이게 유지 (숨기지 않음)

==================================================
4. 사용안함 레벨 전체 시스템 제외 규칙
==================================================

is_active = false 인 레벨은 아래에서 전부 제외해야 한다.

--------------------------------------------------
4-1. 선생님 모드
--------------------------------------------------
- 학생 레벨 변경 선택 리스트에서 제외
- 일지/피드백 화면에서 표시되지 않음
- 학생 카드/프로필에서 표시되지 않음

--------------------------------------------------
4-2. 학부모 모드
--------------------------------------------------
- 자녀 프로필 레벨 표시에서 제외
- 수영정보 레벨 설명 목록에서도 제외
- 사용안함 레벨 정보는 아예 접근 불가

--------------------------------------------------
4-3. 관리자 모드 (운영 화면)
--------------------------------------------------
- 학생 프로필 표시에서 제외
- 단, 레벨 설정 화면에서는 표시 유지

==================================================
5. 학생 레벨 데이터 정합성 처리
==================================================

중요: 이미 학생이 해당 레벨을 사용 중일 수 있음

아래 규칙을 적용한다.

CASE 1:
학생이 사용안함 처리된 레벨을 가지고 있을 경우

처리:
- 자동 변경 금지 (데이터 보존)
- 관리자/선생님 화면에서 경고 표시 가능
- “비활성 레벨” 표시 추가

예:
- 레벨: 7 (사용안함)
→ 표시: [비활성 레벨]

CASE 2:
선생님이 레벨 변경 시

- 사용안함 레벨은 선택 불가

==================================================
6. 레벨 선택 UI 필터링
==================================================

모든 레벨 선택 UI에서 반드시 아래 적용

조건:
WHERE is_active = true

적용 대상:
- 선생님 학생 레벨 변경
- 관리자 레벨 변경 (허용 시)
- 초기 학생 생성 시 레벨 선택 (있다면)

==================================================
7. 학부모 수영정보 레벨 설명 필터링
==================================================

학부모 화면에서 레벨별 설명을 보여줄 때

조건:
- is_active = true 인 레벨만 표시

즉:
- 사용안함 레벨은 학부모에게 절대 노출 금지

==================================================
8. 레벨 뱃지 표시 규칙
==================================================

- is_active = true → 정상 표시
- is_active = false → 표시 안 함 (또는 비활성 표시)

학생이 비활성 레벨을 가지고 있는 경우:
- 뱃지 대신 “비활성 레벨” 표시 가능

==================================================
9. 기본값 처리 (Fallback)
==================================================

수영장이 레벨 설정을 하나도 안 건드렸을 경우:

- 기본 1~10 모두 is_active = true
- 정상적으로 전부 표시

수영장이 일부만 활성화했을 경우:

- is_active = true만 사용
- 나머지는 완전히 숨김

==================================================
10. UI/UX 규칙
==================================================

- 관리자 화면: 전체 레벨 항상 보이되 ON/OFF로 상태 구분
- 선생님/학부모: 사용 레벨만 보임
- 사용안함 레벨이 UI에 노출되면 오류로 간주
- 레벨 수가 6개면 실제 화면에도 6개만 보여야 한다
- 탭/카드/리스트에서 빈 공간 생기지 않게 정리

==================================================
11. 수정 금지 영역
==================================================

다음 기능 절대 깨지면 안 됨:

- 기존 학생 레벨 변경 기능
- 학부모 자녀 프로필
- 일지/피드백 구조
- 보강/출결 시스템
- 관리자 권한 구조
- 다자녀 탭 시스템

==================================================
12. 테스트 및 검증 필수
==================================================

작업 후 반드시 아래 테스트 수행:

1. 관리자에서 레벨 10개 모두 보이는지
2. 특정 레벨 OFF 시 관리자 화면에서 비활성 표시 되는지
3. OFF된 레벨이 선생님 레벨 선택 리스트에서 안 보이는지
4. OFF된 레벨이 학부모 화면에서 안 보이는지
5. 학생 레벨 변경 시 OFF된 레벨 선택 불가능한지
6. 레벨 6개만 활성화 시 실제 화면도 6개만 보이는지
7. 기존 학생 데이터 깨지지 않는지
8. 비활성 레벨 가진 학생 처리 정상인지
9. 레벨 설명/뱃지/프로필 전부 정상 동작하는지

[작업 완료 보고 형식]
1. 적용된 레벨 필터링 방식
2. 수정된 파일 목록
3. 실제 테스트 결과
4. 비활성 레벨 처리 방식 설명
5. 남은 이슈 여부

==================================================
13. 최종 목적
==================================================

- 수영장마다 다른 레벨 개수를 유연하게 지원
- 불필요한 레벨은 완전히 숨김
- 관리자/선생님/학부모 화면 모두 일관성 유지
- 레벨 시스템을 실제 운영에 맞게 최적화

이 기준으로 꼼꼼하게 구현하라.
