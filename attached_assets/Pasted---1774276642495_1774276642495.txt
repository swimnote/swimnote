[운영 관리 팝업 상태 복원 + 홈으로 잘못 튀는 네비게이션 버그 수정 지시서]

현재 버그를 정확히 수정하라.

실제 원하는 동작은 다음과 같다.

==================================================
1. 문제 상황
==================================================

현재 흐름:
홈 화면
→ 운영 관리 아이콘 클릭
→ 아래에서 운영 관리 팝업(회원 명부, 학부모 계정, 선생님 관리 등 아이콘 모음) 열림
→ 회원 명부 진입
→ 학생 상세 진입
→ 뒤로가기

현재 잘못된 동작:
- 뒤로가기를 누르면 홈 화면으로 가버림

원하는 올바른 동작:
- 학생 상세에서 뒤로가기
→ 회원 명부로 복귀
- 회원 명부에서 뒤로가기
→ 운영 관리 팝업이 열린 상태로 복귀
- 운영 관리 팝업을 닫거나 홈을 눌렀을 때만 홈 화면으로 복귀

즉,
운영 관리 팝업은 사라지면 안 되고
그 상태가 유지된 채 복귀해야 한다.

==================================================
2. 같은 규칙이 적용되어야 하는 곳
==================================================

운영 관리 팝업 안의 모든 메뉴에 공통 적용:

- 회원 명부
- 학부모 계정
- 선생님 관리
- 승인 관리
- 초대 안내 기록
- 결제·구독
- 추가 용량
- 매출 관리

예시:
운영 관리 팝업
→ 학부모 계정
→ 부모님 상세
→ 학생 연결 팝업
→ 닫기/뒤로가기

올바른 결과:
- 부모님 상세 또는 학부모 계정 화면으로 복귀
- 다시 뒤로가기 시 운영 관리 팝업 열린 상태 복귀
- 홈 화면으로 바로 가면 안 됨

==================================================
3. 수정 목표
==================================================

운영 관리 팝업을 단순 임시 UI가 아니라
“상위 네비게이션 컨텍스트”로 유지하라.

하위 화면에서 뒤로가기 시:
- 먼저 직전 하위 화면 복귀
- 그다음 운영 관리 팝업 복귀
- 마지막에만 홈 복귀

절대 금지:
- 하위 화면에서 바로 홈 화면으로 점프
- popup state 소실
- selected submenu state 소실
- replace로 홈 화면 강제 이동

==================================================
4. 반드시 점검할 원인
==================================================

아래를 우선적으로 확인하라.

1. 운영 관리 팝업 open state가 route 전환 시 사라지는지
2. 하위 화면 진입 시 router.replace를 사용해서 상위 상태가 날아가는지
3. 뒤로가기 버튼이 router.back이 아니라 홈 route push/replace로 연결되어 있는지
4. popup 내부 메뉴 상태(selectedMenu)가 전역/상위 state로 유지되지 않는지
5. 하위 페이지 진입 후 popup reopen 복원 로직이 없는지

==================================================
5. 수정 방식
==================================================

반드시 아래 원칙으로 수정하라.

1. 운영 관리 팝업의 열림 상태(open state)를 유지
2. 운영 관리 팝업에서 선택한 메뉴 상태(selected section)를 유지
3. 하위 상세 페이지에서 back 시
   - 직전 목록 화면 복귀
   - 목록 화면 back 시 popup 열린 상태 복귀
4. popup을 닫는 행위와 뒤로가기를 구분
5. 홈 복귀는 오직 아래 경우만 허용
   - 홈 버튼 직접 클릭
   - popup의 X 버튼/바깥 터치로 닫기
   - 명시적 홈 이동

==================================================
6. 구체적 테스트 시나리오
==================================================

반드시 아래를 실제 클릭 테스트하라.

시나리오 A:
홈
→ 운영 관리 아이콘
→ 회원 명부
→ 서태윤 프로필
→ 뒤로가기
예상 결과:
→ 회원 명부

→ 다시 뒤로가기
예상 결과:
→ 운영 관리 팝업 열린 상태

→ popup 닫기 또는 홈 클릭
예상 결과:
→ 홈 화면

시나리오 B:
홈
→ 운영 관리 아이콘
→ 학부모 계정
→ 서태윤 부모님 상세
→ 학생 연결 팝업
→ 닫기/뒤로가기
예상 결과:
→ 학부모 계정 또는 부모님 상세의 직전 맥락 복귀

→ 다시 뒤로가기
예상 결과:
→ 운영 관리 팝업 열린 상태

시나리오 C:
운영 관리 팝업
→ 선생님 관리
→ 상세
→ 수정
→ 저장
예상 결과:
→ 상세 또는 목록 복귀
→ 다시 뒤로가면 운영 관리 팝업 열린 상태

==================================================
7. 적용 범위 확대
==================================================

이 규칙은 운영 관리만이 아니라
“홈 화면에서 아이콘 눌러 바텀시트/팝업/아이콘 모음이 열리고,
그 안에서 하위 메뉴로 들어가는 구조” 전체에 공통 적용 가능한지 점검하라.

즉,
운영 관리뿐 아니라 같은 패턴의 다른 아이콘 메뉴도
동일한 방식으로 스택 유지되게 맞춰라.

==================================================
8. 수정 금지 영역
==================================================

다음은 깨지면 안 된다.
- 운영 관리 메뉴 내용 자체
- CRUD 저장 로직
- 회원 상세 데이터
- 학부모 연결 기능
- 승인/결제/매출 기능
- 홈 버튼 자체 기능

이번 작업은 네비게이션 스택과 popup 상태 복원 수정이다.

==================================================
9. 완료 보고 형식
==================================================

반드시 아래 형식으로 보고하라.

1. 홈으로 튀던 직접 원인
2. 운영 관리 팝업 상태를 어떻게 유지하도록 수정했는지
3. selected menu 상태를 어떻게 복원했는지
4. 수정한 파일 목록
5. 회원 명부 → 상세 → 뒤로가기 테스트 결과
6. 학부모 계정 → 팝업 → 뒤로가기 테스트 결과
7. 다른 운영 관리 메뉴 회귀 테스트 결과
8. 아직 홈으로 잘못 튀는 화면이 남아 있는지 여부

중요:
반드시 실제 클릭 테스트 결과를 적어라.
“논리상 될 것 같음” 금지.
