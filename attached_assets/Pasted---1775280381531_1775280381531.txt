학부모 연결은 이제 바로 잘 돼 이제 자녀가 두명이상일경우 어떻게 사용할거냐가 관건인데 어떤방식으로하면 충돌없이 자녀 두명을 학부모가 받아서 이용할수있을까

충돌 없이 가려면
“학부모 1계정 = 여러 자녀 연결”로 하고,
로그인 주체는 학부모, 데이터 조회 주체는 선택된 자녀로 완전히 분리해야 한다.

지금 제일 안전한 방식은 이거다.

결론
추천 방식

학부모 계정 1개 + 자녀별 개별 연결 + 첫 화면에서 자녀 선택/전환

즉:

학부모는 한 번만 로그인
자녀는 여러 명 연결 가능
홈에서는 “전체 요약” 먼저 보여줌
세부 화면은 선택된 자녀 기준으로 보여줌

이게 제일 안 꼬인다.

왜 이 방식이 맞냐

형제자매가 생기면 보통 꼬이는 이유가 3개다.

1. 전화번호가 동일함

형제 둘 다 학부모 전화번호는 같을 가능성이 높다.

그래서 예전처럼
전화번호만으로 연결하면 무조건 꼬인다.

2. 홈에서 누구 정보인지 섞임
출석은 첫째
피드백은 둘째
사진은 둘 다

이렇게 섞이면 학부모가 헷갈린다.

3. 자동 연결 시 중복 매칭 위험

같은 수영장 + 같은 전화번호로 학생 2명이 등록돼 있으면
학부모가 한 명만 입력했는데 두 명이 다 붙을 수도 있다.

그래서 “연결”과 “조회”를 분리해야 한다.

가장 안전한 구조
1. 연결 단위는 무조건 학생별 1건

테이블은 이런 개념이면 된다.

parent_accounts : 학부모 계정
students : 학생
parent_students : 학부모-학생 연결 테이블

즉 학부모 1명이 학생 여러 명과 연결될 수 있어야 한다.

예:

parent A ↔ student 1
parent A ↔ student 2

이렇게 각각 별도 row로 저장

핵심

학생 2명을 한 덩어리로 저장하지 말고, 1학생 = 1연결 row
그래야 충돌이 안 난다.

2. 자동 연결은 “첫 자녀 1명”만 즉시 처리

지금 네 시스템 기준으로는
회원가입 온보딩 때:

수영장 선택
학생 이름 입력
전화번호 인증

이걸로 첫 번째 자녀를 자동 연결하면 된다.

즉 가입 직후는:

첫 자녀만 연결
나머지 형제자매는 나중에 “우리 아이 추가”로 붙임

이게 중요한 이유는
처음부터 같은 전화번호로 형제 둘을 한 번에 자동 매칭하면
동명이인, 오기입, 반쯤 등록 상태에서 꼬일 수 있기 때문이다.

내가 이걸 꼭 포함한 이유는
형제자매를 한 번에 자동연결하는 것보다, 첫 자녀 자동연결 + 추가 연결 방식이 훨씬 안전하기 때문이다.

3. “우리 아이 추가” 기능을 따로 둬라

학부모 홈에 버튼 하나 있으면 된다.

우리 아이 추가
형제/자매 연결 추가

눌렀을 때:

같은 수영장 기준
학생 이름 입력
필요하면 생년월일 뒷자리 또는 반명 같은 추가 확인값 1개
일치 시 연결

이러면 기존 계정에 두 번째 자녀를 안전하게 붙일 수 있다.

자동 연결 기준을 어떻게 해야 충돌이 없냐
첫 자녀 자동 연결

가입 시에는 지금 방식 유지 가능

pool_id
학부모 전화번호
학생 이름

이 3개 일치 시 연결

둘째부터는 추가 확인값 1개 넣는 걸 추천

형제자매 추가 연결 때는
이 3개만으로도 가능은 한데, 더 안전하게 하려면 하나를 더 넣는 게 좋다.

추천 우선순위:

반 이름
생년월일 4자리
학생 등록번호
보호자 성명 일부 확인

가장 무난한 건
학생 이름 + 학부모 전화번호 + 같은 수영장 기본,
필요 시 생년월일 4자리 추가다.

왜냐면 형제 둘은 전화번호가 같으니까
이름 오타나 동명이인만 생겨도 위험해진다.

홈 화면은 어떻게 가야 하냐

너가 예전에 말한 선호 구조 그대로 가면 된다.

첫 화면

학부모 홈 첫 화면은 전체 현황 요약

예:

첫째 출석
둘째 출석
최근 피드백 2건
새 사진 알림
미확인 공지

즉 자녀별 카드 요약으로 보여줘야 한다.

상세 화면

상세 들어가면 선택된 자녀 기준

상단에:

자녀 전환 탭 또는 드롭다운

예:

김민준
김서윤

선택한 자녀 기준으로

출석
피드백
사진
진도

보이게 하면 된다.

핵심

조회 컨텍스트를 항상 자녀 1명으로 고정
이게 충돌 방지 핵심이다.

전체 요약은 여러 명 가능하지만
세부 화면은 항상 한 명 기준이어야 한다.

충돌 없이 쓰려면 꼭 넣어야 하는 규칙
규칙 1. 활성 자녀 개념

앱 안에 selectedStudentId 개념이 있어야 한다.

홈 요약은 전체
상세는 selectedStudentId 기준

이게 없으면 화면마다 자녀가 섞인다.

규칙 2. 연결 테이블에 unique

중복 연결 방지 필수

예:

parent_id + student_id unique

이거 없으면 둘째 추가하다가 첫째가 또 붙을 수 있다.

규칙 3. 같은 전화번호 학생 여러 명 허용

형제 구조에서는 당연히 필요하다.

즉 DB에서
“전화번호 하나당 학생 1명만” 같은 제약 있으면 안 된다.

대신

학생별 row는 독립
연결도 학생별 독립
규칙 4. 자동 연결과 추가 연결을 분리
가입 시: 첫 자녀 자동 연결
가입 후: 우리 아이 추가로 둘째 연결

이렇게 분리해야 UI도 깔끔하고 디버깅도 쉽다.
