지금부터 회원가입 시스템 중 “수영장 관리자 가입 구조”와 관련 승인 조건을 다시 수정하라.

중요:
- 기존 테스트 로그인 화면 유지
- 기존 각 모드 진입 유지
- 다른 기능/화면/모드 구조는 건드리지 말 것
- 이번 작업은 수영장 관리자 회원가입 구조 수정과 관련 승인 조건 수정만 수행
- 기존 로그인/회원가입 구현분 중 관련 부분만 수정
- 수정 후 실제 흐름 테스트까지 완료하고 결과를 상세히 보고할 것

==================================================
[1. 이번 수정 목표]

수정할 핵심 항목 4개:

1) 수영장 관리자 가입을 최대한 간편하게 만든다
2) 사업자등록번호 / 사업자등록증 업로드를 관리자 가입 단계에서 제거한다
3) 계좌인증 / 계좌인증 단계 / 계좌인증 필수 조건을 제거한다
4) 슈퍼관리자 승인 조건에서도 사업자인증/계좌인증 요구를 제거한다

==================================================
[2. 최종 가입 원칙]

수영장 관리자 가입은 아래 원칙으로 단순화한다.

최종 구조:
수영장 관리자 선택
→ 계정 정보 입력
→ 수영장 기본 정보 입력
→ 가입 완료/신청 완료

즉,
- 사업자 인증 없음
- 계좌 인증 없음
- 기본 수영장 운영 정보만 입력
- 가입 절차를 최대한 짧고 가볍게 만든다

==================================================
[3. 수영장 관리자 가입 화면 수정]

--------------------------------------
[3-1. 계정 생성 화면]

유지/수정할 항목:
- 이름
- 아이디
- 연락처
- 비밀번호
- 비밀번호 확인

주의:
- 로그인 식별자는 이메일이 아니라 아이디다
- 이메일을 로그인 아이디처럼 쓰는 구조가 남아 있으면 제거 또는 선택 정보로만 둔다
- placeholder, 라벨, 안내 문구 모두 아이디 기준으로 통일

예:
- "관리자 이메일(로그인 아이디)" → "아이디"
- 예시값도 이메일 형식 제거

--------------------------------------
[3-2. 수영장 정보 입력 화면]

이번 가입 단계에서 받는 정보는 “기본 수영장 정보”만 남긴다.

유지 가능한 항목:
- 수영장 이름
- 영문표시명(파일명용) 필요하면 유지 가능
- 주소
- 대표 전화
- 대표자 이름
- 관리자 이름
- 관리자 연락처
- 아이디

제거할 항목:
- 사업자등록번호
- 사업자등록증 이미지 업로드
- 사업자등록증 관련 안내 문구
- 계좌인증
- 예금주명
- 은행명
- 계좌번호
- 계좌 인증 버튼
- 계좌 인증 완료 상태 표시
- 계좌 인증 필수 관련 문구

==================================================
[4. 슈퍼관리자 승인 조건 수정]

기존에 슈퍼관리자 승인 조건/안내/문구/로직 어딘가에 아래 내용이 있으면 제거하라.

제거 대상:
- 사업자등록증이 있어야 승인 가능
- 사업자등록번호 확인 후 승인
- 계좌인증 완료 후 승인
- 계좌인증 정보 검토 후 승인

수정 후 슈퍼관리자 승인 기준:
- 수영장 기본 정보 확인
- 관리자 계정 정보 확인
- 필요 시 운영자 수동 승인

즉,
슈퍼관리자 승인 화면/문구/조건에서
사업자 인증 / 계좌 인증 요구가 보이지 않게 수정할 것

==================================================
[5. 문구 수정 지시]

수영장 관리자 가입 / 수영장 등록 신청 / 슈퍼관리자 승인 관련 화면의 문구를 아래 기준으로 수정하라.

수정 전 예시:
- 사업자등록증 업로드
- 사업자등록번호
- 계좌인증 완료 후 신청 가능
- 승인 시 사업자 정보 확인
- 승인 시 계좌 정보 확인

수정 후 예시:
- 수영장 기본 정보를 입력해 주세요
- 신청 내용은 플랫폼 운영자가 검토 후 승인합니다
- 승인 후 입력한 아이디로 로그인할 수 있습니다

핵심:
사업자인증 / 계좌인증을 연상시키는 문구 전부 제거

==================================================
[6. 데이터 구조 처리]

주의:
기존 다른 기능이 사업자 관련 필드나 계좌 관련 필드를 참조하고 있을 수 있으므로,
무작정 삭제하지 말고 이번 가입 단계에서는 사용하지 않도록만 처리하라.

즉:
- 기존 필드가 남아 있어도 됨
- 단, 관리자 가입 UI/흐름/검증에서는 사용 금지
- 다른 기능 깨지지 않게 유지

이번 가입 단계에서 실제 사용하는 핵심 필드:
- username 또는 loginId
- password
- managerName
- managerPhone
- poolName
- poolEnglishName (있으면)
- poolAddress
- representativeName
- representativePhone

비활성 처리 대상:
- businessRegistrationNumber
- businessRegistrationImage
- bankName
- bankAccount
- accountHolder
- accountVerificationStatus

==================================================
[7. 회원가입 경로 전체 반영]

최종 경로를 아래처럼 맞춰라.

--------------------------------------
[수영장 관리자]
계정 생성
→ 수영장 기본 정보 입력
→ 가입 신청 완료
→ 필요 시 슈퍼관리자 승인 대기

--------------------------------------
[선생님]
변경 없음
- 수영장 검색(2글자 이상)
- 정보 입력
- 가입 승인 요청
- 관리자 승인 대기

--------------------------------------
[학부모]
변경 없음
- 수영장 찾기
- 또는 수영장 코드 입력
- 코드 유효시간 24시간 유지

==================================================
[8. 연결 안정성 절대 원칙]

이번 수정은 구조를 단순화하는 작업이지만,
기존 기능을 깨뜨리면 안 된다.

반드시 아래 원칙을 지켜라.

[기능 이동/수정 시 공통 체크]
1. 기존 기능 로직 최대한 건드리지 말 것
2. 가입 화면 위치/필드/문구만 수정할 것
3. 다른 모드 화면은 건드리지 말 것
4. 수정 후 버튼 이동 정상 여부 확인
5. 데이터 전달 정상 여부 확인
6. 테스트 로그인과 기존 모드 진입이 유지되는지 확인
7. 선생님/학부모 가입 흐름이 깨지지 않았는지 확인

핵심:
기능은 최대한 건드리지 않은 상태에서 수정하고,
서로 연결된 부분은 반드시 확인하면서 진행할 것

==================================================
[9. 테스트 지시 - 반드시 실제 수행]

수정 후 아래 테스트를 전부 수행하고 결과를 상세히 보고하라.

테스트 1.
수영장 관리자 가입 화면에서 이메일 로그인 구조가 제거되고 아이디 입력으로 보이는지 확인

테스트 2.
수영장 관리자 가입 흐름에서 사업자등록번호 / 사업자등록증 업로드 UI가 제거되었는지 확인

테스트 3.
수영장 관리자 가입 흐름에서 계좌인증 관련 UI/단계/문구가 제거되었는지 확인

테스트 4.
수영장 관리자 가입이 기본 수영장 정보만으로 완료/신청 가능한지 확인

테스트 5.
슈퍼관리자 승인 관련 문구/조건에서 사업자 인증 / 계좌 인증 요구가 제거되었는지 확인

테스트 6.
수정 후 관리자 계정이 아이디 기반으로 로그인 가능한지 확인

테스트 7.
기존 테스트 로그인 화면과 각 모드 테스트 진입이 그대로 유지되는지 확인

테스트 8.
선생님 회원가입 흐름이 깨지지 않았는지 확인

테스트 9.
학부모 회원가입 흐름(수영장 찾기 / 코드 입력)이 깨지지 않았는지 확인

==================================================
[10. 보고 형식]

작업 후 아래 순서로 보고하라.

1. 수영장 관리자 가입 구조를 어떻게 단순화했는지
2. 제거한 항목 목록
3. 슈퍼관리자 승인 조건/문구에서 제거한 항목
4. 로그인 아이디 구조를 어떻게 반영했는지
5. 다른 기능에 영향 없이 처리한 방식
6. 테스트 1~9 결과
7. 아직 남은 임시 처리 부분
8. 다른 화면/기능에 영향 준 부분이 있으면 숨기지 말고 적기

==================================================
[11. 절대 금지]

- 테스트 로그인 삭제 금지
- 기존 모드 진입 삭제 금지
- 선생님/학부모 가입 흐름 깨뜨리기 금지
- 다른 주요 탭 화면 수정 금지
- 사업자/계좌 관련 필드를 무작정 삭제해서 다른 로직 깨뜨리기 금지
- 이번 작업 범위를 넘어가서 다른 UI 재배치 금지

이번 작업은
“수영장 관리자 가입 간소화 + 사업자/계좌 인증 제거 + 슈퍼관리자 승인 조건 수정”
까지만 수행한다.
다른 기능은 건드리지 말 것.
