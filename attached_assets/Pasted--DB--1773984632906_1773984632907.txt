선생님 모드의 회원관리 > 대기자 명단 구조를 아래 규칙으로 수정한다.
기존 기능 삭제 금지, 관리자 모드와 동일 DB 및 동일 상태값 사용, 화면만 분리한다.

[핵심 수정]
현재 대기자 명단에서 전체 탭에 퇴원생만 보이는 문제를 수정한다.
전체 탭에는 내가 관리하는 전체 회원이 보여야 한다.
정상 회원, 미배정 회원, 연기예정 회원, 퇴원예정 회원을 모두 포함한다.

[선생님 모드 대기자 명단 탭 변경]
기존 탭:
전체 / 대기 / 연기 / 퇴원

변경 탭:
전체 / 미배정 / 연기예정 / 퇴원예정

[탭 의미]
전체:
내가 관리하는 전체 회원 표시
정상 포함, 미배정 포함, 연기예정 포함, 퇴원예정 포함

미배정:
신규 등록 또는 연결 완료 후 아직 반배정/선생님배정이 안 된 회원만 표시

연기예정:
연기 처리됐지만 이번 달 말일까지 수업 유지 중인 회원 표시

퇴원예정:
퇴원 처리됐지만 이번 달 말일까지 수업 유지 중인 회원 표시

[상태 처리 규칙]
선생님이 회원을 연기 또는 퇴원 처리하면 기본값은 즉시 확정이 아니라 예정 상태로 저장한다.

연기 처리 시:
- memberStatus는 현재 상태 유지
- pendingStatusChange = 연기
- 대기자 명단의 연기예정 탭에 표시
- 카드에 [연기예정] 배지 표시

퇴원 처리 시:
- memberStatus는 현재 상태 유지
- pendingStatusChange = 퇴원
- 대기자 명단의 퇴원예정 탭에 표시
- 카드에 [퇴원예정] 배지 표시

[예정 상태 카드 동작]
대기자 명단의 연기예정/퇴원예정 회원 카드를 누르면 액션 팝업을 띄운다.

연기예정 팝업 버튼:
- 이달 말 연기 유지
- 오늘 즉시 연기
- 취소

퇴원예정 팝업 버튼:
- 이달 말 퇴원 유지
- 오늘 즉시 퇴원
- 취소

[오늘 즉시 연기]
- memberStatus를 즉시 휴원/연기 상태로 전환
- 정상/주1/주2/주3 리스트에서 즉시 제거
- 연기 리스트 또는 휴원 리스트 규칙에 맞게 이동
- 관리자 모드에도 즉시 반영

[오늘 즉시 퇴원]
- memberStatus를 즉시 퇴원 상태로 전환
- 정상/주1/주2/주3 리스트에서 즉시 제거
- 퇴원 리스트로 즉시 이동
- 관리자 모드에도 즉시 반영

[이달 말 유지]
- pendingStatusChange 유지
- 현재 달 수업 유지
- 말일 처리 또는 다음 달 수업 생성 시 실제 상태 반영

[말일 반영]
월말 처리 또는 다음 달 수업 생성 시 pendingStatusChange를 실제 상태로 반영한다.
연기예정은 실제 연기/휴원 상태로 전환한다.
퇴원예정은 실제 퇴원 상태로 전환한다.
반영 후 pendingStatusChange를 초기화한다.

[공통 카드]
대기자 명단도 다른 모든 회원 리스트와 동일한 UnifiedMemberCard를 사용한다.
검색 결과도 동일 카드 사용.
카드에는 이름, 대표상태 배지 1개, 주횟수 배지 1개, 예정 배지(있을 때만), 반정보, 선생님정보, 보호자정보를 동일 위치에 표시한다.

[관리자 모드 연동]
선생님 모드에서 미배정/연기예정/퇴원예정/즉시퇴원/즉시연기 중 어떤 상태 변경을 해도
관리자 모드의 동일 회원 상태 게시판과 리스트에 즉시 반영한다.
화면별로 별도 데이터 쓰지 말고 공통 상태 저장소와 공통 계산 함수를 사용한다.

[보강관리와 분리]
수업관리 > 수강생관리 > 보강대기리스트는 결석/보강 처리용으로 유지한다.
회원관리 > 대기자 명단은 미배정/연기예정/퇴원예정 관리용으로 분리한다.
두 리스트를 서로 섞지 않는다.

[필수 테스트]
1. 전체 탭에 정상 회원 포함 전체 회원이 보이는지
2. 미배정 탭에 신규 미배정 회원이 보이는지
3. 연기 처리 시 연기예정 탭으로 이동하는지
4. 퇴원 처리 시 퇴원예정 탭으로 이동하는지
5. 퇴원예정 회원에서 오늘 즉시 퇴원 버튼이 동작하는지
6. 이달 말 유지 시 현재 수업을 계속 유지하는지
7. 월말 또는 다음 달 수업 생성 시 실제 상태가 확정되는지
8. 관리자 모드 동일 게시판에 즉시 반영되는지
9. 검색 결과에서도 같은 회원카드 디자인이 유지되는지
