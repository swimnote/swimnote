지금부터 관리자 모드의 하단 탭 “사람”을 “인원관리”로 재구성하라.

이번 작업은 단순 레이아웃 수정이 아니라
관리자 모드에서 사람 탭의 구조 자체를
“작은 상단 칩 필터 방식”에서
“관리 허브형 인원관리 구조”로 바꾸는 작업이다.

중요:
- 기존 상단 작은 칩 구조(회원, 학부모, 선생님, 승인, 미등록)를 첫 화면에서 제거
- 하단 탭 이름도 “사람”에서 “인원관리”로 변경
- 첫 화면은 사람 목록이 아니라 관리 허브가 되어야 함
- 회원/학부모/선생님/승인·미배정으로 크게 나눈 뒤, 각각 전용 목록 화면으로 들어가게 할 것
- 재탭 시 인원관리 첫 화면으로 복귀하도록 할 것
- 다른 모드/다른 탭의 구조는 건드리지 말 것

==================================================
[1. 하단 탭 이름 변경]

관리자 모드 하단 탭의 “사람” 이름을 아래처럼 변경

기존:
- 사람

변경:
- 인원관리

중요:
- 관리자 모드에서만 적용
- 선생님/학부모 등 다른 모드에서 사람 탭 구조를 공유 중이면 관리자용 분기 처리
- 아이콘은 기존 사람 아이콘 유지 가능

==================================================
[2. 인원관리 첫 화면 구조]

인원관리 첫 화면은 “목록 화면”이 아니라 “관리 허브”로 만든다.

즉:
처음 들어가면 학생 목록이 바로 길게 나오는 구조를 버리고,
관리자가 어떤 대상을 관리할지 먼저 고르게 한다.

첫 화면 구성:
1. 상단 제목
2. 요약 카드 4개
3. 빠른 작업 버튼 영역

==================================================
[3. 상단 제목]

제목:
- 인원관리

부제 또는 설명 문구 선택 가능:
- 회원, 학부모, 선생님, 승인 상태를 관리합니다

==================================================
[4. 상단 요약 카드 4개]

인원관리 첫 화면에 아래 4개 카드 배치

1. 회원관리
2. 학부모관리
3. 선생님관리
4. 승인·미배정

형태:
- 2열 그리드 또는 세로 카드 구조
- 작은 칩 대신 충분히 큰 카드형 터치 영역
- 숫자 요약이 보이면 좋음

예시 표시:
회원관리
- 정상 / 연기 / 탈퇴 수

학부모관리
- 연결 완료 / 미연결 수

선생님관리
- 승인 / 미승인 / 부관리자 수

승인·미배정
- 승인대기 수 / 미배정회원 수

중요:
- 이 첫 화면에서는 더 이상 작은 상단 칩을 쓰지 말 것
- 카드가 잘리지 않고 한눈에 들어와야 함

==================================================
[5. 빠른 작업 영역]

요약 카드 아래에 빠른 작업 버튼 영역 추가

추천 버튼:
- 회원 명단 업로드
- 학부모 초대 발송
- 미배정회원 보기
- 승인 처리

중요:
- 실제 자주 쓰는 작업만 짧게 배치
- 버튼 몇 개만 두고 과도하게 늘리지 말 것

==================================================
[6. 회원관리 전용 화면]

회원관리 카드 클릭 시
회원 전용 목록 화면으로 이동

이 화면에서는 학생 회원만 관리한다.

상단 필터는 단순화:
- 정상
- 연기
- 탈퇴

검색창:
- 학생 이름 / 보호자 검색

목록:
- 학생 카드 리스트

중요:
- 회원관리 화면 안에는 학부모/선생님/승인/미배정 섞지 말 것

==================================================
[7. 학부모관리 전용 화면]

학부모관리 카드 클릭 시
학부모 전용 목록 화면으로 이동

상단 필터 예:
- 연결 완료
- 초대 완료
- 미연결

검색창:
- 학부모 이름 / 전화번호

목록:
- 학부모 카드 리스트

==================================================
[8. 선생님관리 전용 화면]

선생님관리 카드 클릭 시
선생님 전용 목록 화면으로 이동

상단 필터 예:
- 승인됨
- 미승인
- 부관리자

검색창:
- 이름 / 전화번호 / 아이디

목록:
- 선생님 카드 리스트

==================================================
[9. 승인·미배정 전용 화면]

승인·미배정 카드 클릭 시
별도 전용 화면으로 이동

이 화면 안에서만 상단 탭 2개 사용

탭:
- 승인대기
- 미배정회원

즉:
승인과 미배정은 같은 성격(처리 업무)이므로 여기서 묶는다

중요:
이제 첫 화면에서 승인/미등록을 작은 칩으로 직접 제어하지 말고
전용 화면 안에서 2탭으로 나눈다

==================================================
[10. 기존 사람 탭의 작은 칩 구조 제거]

관리자 모드 인원관리 첫 화면에서는
기존 상단 작은 칩 구조를 제거하라.

제거 대상:
- 회원
- 학부모
- 선생님
- 승인
- 미등록
작은 pill/chip 한 줄 구조

이 구조는 첫 화면에서 더 이상 사용하지 않는다

이유:
- 잘림 문제 반복
- 관리 목적과 상태 필터가 섞여 복잡함
- 관리자 입장에서 한눈에 안 들어옴

==================================================
[11. 재탭 초기화 규칙 적용]

관리자 모드에서 하단 탭 “인원관리”를 다시 누르면
항상 인원관리 첫 화면(허브 화면)으로 복귀하게 하라.

예:
- 회원관리 목록 안에 있음
- 학부모관리 상세 안에 있음
- 승인·미배정 화면 안에 있음

이 상태에서 하단 “인원관리” 탭을 다시 누르면
무조건 인원관리 첫 화면으로 이동

추가 규칙:
- 스크롤도 초기화
- 하위 화면 상태도 정리

중요:
기존에 요청한 탭 재탭 초기화 규칙을
인원관리 구조에도 제대로 적용할 것

==================================================
[12. 뒤로가기 규칙]

인원관리 하위 화면 안에서는
왼쪽 위 뒤로가기 버튼 누르면 직전 화면으로 돌아가고

인원관리 탭 자체를 다시 누르면
항상 인원관리 첫 화면으로 가야 한다

즉:
- 뒤로가기 = 이전 페이지
- 탭 재탭 = 인원관리 홈

==================================================
[13. 기존 데이터/기능 유지]

반드시 유지:
- 기존 학생/학부모/선생님 데이터
- 기존 검색 로직 (해당 전용 화면 안에서 사용 가능)
- 승인 기능
- 미등록/미배정 회원 기능
- 명단 업로드/초대 발송 기능
- 기존 상세 페이지 진입

이번 작업은
레이아웃과 진입 구조를 재구성하는 것이지
데이터 구조를 깨는 작업이 아니다

인원관리 첫 화면에 학생 목록이 바로 나오지 않고 요약 카드 4개가 보이는지 확인

[17. 절대 금지]

- 첫 화면에 기존 작은 칩 구조를 다시 억지로 유지하는 것 금지
- 학생/학부모/선생님/승인/미등록을 다시 한 줄 칩에 섞는 것 금지
- 다른 탭/다른 모드 손대기 금지
- 데이터 구조 임의 변경 금지
- 재탭 초기화 빠뜨리기 금지

이번 작업은
관리자 모드의 사람 탭을 인원관리 허브형 구조로 재설계해
처음 들어가면 회원관리/학부모관리/선생님관리/승인·미배정으로 크게 나눠 보여주고,
각 전용 화면 안에서만 세부 목록을 관리하게 만들며,
탭 재탭 시 항상 인원관리 첫 화면으로 복귀하게 만드는 작업이다.
