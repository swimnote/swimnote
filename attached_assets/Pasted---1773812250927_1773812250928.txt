관리자 모드 메뉴 구조를 아래 기준으로 수정하라.

[목표]
현재 관리자 하단 탭에 있는 “정산”은 실제로 선생님 수업 매출 정산 탭이 아니라
수영장의 앱 구독/사용량/플랜 관리 성격이다.
따라서 하단 탭 구조를 재정리한다.

==================================================
[1. 관리자 하단 탭 변경]

기존:
- 홈
- 사람
- 수업
- 메신저
- 정산
- 더보기

변경 후:
- 홈
- 사람
- 수업
- 메신저
- 매출관리
- 더보기

즉,
기존 하단 탭 “정산”은 제거하고
새 하단 탭 “매출관리”를 추가한다.

==================================================
[2. 기존 정산 탭 처리]

현재 하단 탭 “정산”으로 연결되어 있는 화면은
하단 탭에서 제거하고 더보기 안으로 이동시킨다.

이름 변경:
- 기존 명칭 “정산” → “구독관리”

위치는:
- 관리자 모드 > 더보기 안

기존 기능은 삭제하지 말고 그대로 유지하되,
메인 하단 탭이 아니라 더보기 내부 메뉴로 이동할 것.

==================================================
[3. 구독관리의 역할]

더보기 안의 “구독관리”는 아래 기능만 담당한다.

- 현재 구독 상태
- 무료 / 유료 플랜
- 결제 카드
- 저장 용량 / 사용량
- 플랜 변경
- 앱 이용 관련 구독 정보

즉:
이 화면은 “선생님 매출 정산”이 아니라
“수영장의 앱 사용 요금/플랜/용량 관리” 화면으로 명확히 분리한다.

==================================================
[4. 새 하단 탭 매출관리 추가]

관리자 하단 탭에 새로 “매출관리”를 추가한다.

이 탭은 “선생님 수업 정산 및 다음 달 발생 관리” 전용 탭이다.

==================================================
[5. 매출관리 탭에 넣을 기능]

아래 기능을 모두 매출관리 탭 안으로 정리하라.

- 선생님별 매출 결과
- 회원별 수업 횟수
- 체험 수업 카운트
- 임시 이동 정산
- 기타 수업 정산
- 소멸 / 미실시 / 보강 반영
- 월 저장
- 다음 달 수업 발생
- 다음 달 시작
- 보강 이월 정리

핵심:
선생님들 매출관리와 관련된 기능은 전부 이 탭으로 모은다.

==================================================
[6. 기능 이동 원칙]

1. 기능 삭제 금지
2. 기존 데이터 구조 수정 금지
3. 기존 정산 로직은 최대한 유지
4. 단지 적절한 위치로 이동/정리할 것
5. 기존 버튼 연결이 끊기지 않게 할 것
6. 화면명/탭명만 바꾸고 끝내지 말고, 실제 진입 위치도 바꿀 것

==================================================
[7. 관리자 더보기 정리]

더보기 안에는 아래와 같은 구조가 되게 정리한다.

예시:
- 구독관리
- 승인 관리
- 선생님 관리
- 학부모 계정
- 수업 관리
- 삭제/복구 센터
- 단가표
- 휴무일 관리
- 기타 설정

단,
기존에 이미 하단 탭에 있는 핵심 기능과 중복되지 않게 조정할 것.

==================================================
[8. 테스트 지시]

수정 후 반드시 실제 테스트하라.

테스트 1.
관리자 하단 탭에 “매출관리”가 새로 보이는지 확인

테스트 2.
기존 하단 탭 “정산”이 사라졌는지 확인

테스트 3.
더보기 안에 “구독관리”가 보이는지 확인

테스트 4.
구독관리 화면에 기존 구독/플랜/카드/용량 정보가 그대로 유지되는지 확인

테스트 5.
매출관리 탭에서 선생님 매출 관련 기능이 모여 있는지 확인

테스트 6.
다음 달 수업 발생 기능이 매출관리 탭 안으로 정리되었는지 확인

테스트 7.
기존 정산 관련 로직/데이터 연결이 깨지지 않았는지 확인

==================================================
