지금부터 관리자 모드 > 더보기 > 데이터 관리 내부 네비게이션과 원본 데이터 삭제 화면 스크롤 문제를 수정하라.

이번 작업은 기능 추가가 아니라
“공통 뒤로가기 규칙 적용” + “원본 데이터 삭제 화면 스크롤 수정”
2가지만 정확히 고치는 작업이다.

중요:
- 다른 기능 건드리지 말 것
- UI 내용 변경 최소화
- 데이터 관리 내부 화면들만 수정
- 실제 테스트까지 수행할 것

==================================================
[1. 수정 대상]

관리자 모드 > 더보기 > 데이터 관리 안의 상세 페이지들

대상 화면 예:
- 저장공간 현황
- 계정별 사용량
- 카테고리별 사용량
- 원본 데이터 삭제
- 이벤트 기록

==================================================
[2. 뒤로가기 공통 규칙 적용]

현재 데이터 관리 내부 상세 페이지의 뒤로가기 동작이
우리가 정한 공통 규칙과 다르면 수정하라.

최종 규칙:

1. 데이터 관리 상세 화면에서 뒤로가기 1회
→ 데이터 관리 첫 화면으로 이동

예:
더보기 > 데이터 관리 > 카테고리별 사용량
여기서 뒤로가기
→ 데이터 관리 첫 화면

2. 데이터 관리 첫 화면에서 뒤로가기
→ 더보기 화면으로 이동

즉:
데이터 관리 내부는
자기 내부 첫 화면까지 한 단계씩 빠지고,
계속 뒤로가면 최종적으로 더보기 화면까지 가야 한다.

중요:
- 뒤로가기 누를 때 바로 홈이나 다른 탭으로 가지 말 것
- 반드시 현재 흐름 기준으로 한 단계씩만 나가야 함
- 현재 우리가 정한 공통 규칙과 맞게 수정할 것

==================================================
[3. 데이터 관리 내부 상세 페이지 공통 규칙]

모든 데이터 관리 상세 페이지에서 아래 규칙 적용:

- 뒤로가기 버튼 누르면 데이터 관리 첫 화면으로 복귀
- 데이터 관리 첫 화면에서 뒤로가기 누르면 더보기로 복귀
- 하단 더보기 탭을 다시 누르면 더보기 첫 화면으로 유지
- 다른 하단 탭 이동 시 현재 상태 유지 규칙은 기존 공통 규칙대로 유지

==================================================
[4. 원본 데이터 삭제 페이지 스크롤 수정]

현재 “원본 데이터 삭제” 페이지가 스크롤되지 않는 문제를 수정하라.

이 페이지는 내용이 길어질 수 있으므로
반드시 전체 화면 세로 스크롤 가능해야 한다.

필수:
- 페이지 전체가 세로 스크롤 가능
- 하단 버튼/설명/경고문/비밀번호 입력 구간까지 전부 내려서 볼 수 있어야 함
- 작은 아이폰 화면에서도 잘려서 안 보이는 구간 없어야 함

중요:
- 단순 일부 카드만 보이고 아래가 잘리는 상태 금지
- 스크롤 컨테이너가 잘못 잡혀 있으면 수정
- safe area와 하단 탭, 브라우저 바에 가리지 않게 할 것

==================================================
[5. 원본 데이터 삭제 페이지에서 확인할 요소]

스크롤 가능한 상태에서 아래 요소들이 끝까지 보여야 한다.

- 경고 문구
- 삭제 데이터 종류 선택
- 기간 선택
- 삭제 대상 미리보기
- 영구 삭제 실행 버튼
- 비밀번호 인증 단계가 있으면 그 영역까지
- 하단 안내 문구

즉 페이지 전체가 자연스럽게 위아래로 움직여야 한다.

==================================================
[6. 레이아웃 주의]

원본 데이터 삭제 페이지는
하단 탭과 브라우저/기기 하단 영역 때문에 잘리기 쉬우므로 아래를 지켜라.

- 하단 여백 충분히 확보
- 마지막 CTA 버튼이 탭 위에 가려지지 않게 할 것
- 스크롤 마지막에서 버튼이 완전히 보여야 함
- iPhone Safari/웹뷰 환경 기준으로도 확인할 것

[9. 테스트]

반드시 실제 테스트하라.

테스트 1.
더보기 > 데이터 관리 > 저장공간 현황
→ 뒤로가기
→ 데이터 관리 첫 화면으로 가는지 확인

테스트 2.
데이터 관리 첫 화면
→ 뒤로가기
→ 더보기 화면으로 가는지 확인

테스트 3.
더보기 > 데이터 관리 > 계정별 사용량
→ 뒤로가기
→ 데이터 관리 첫 화면으로 가는지 확인

테스트 4.
더보기 > 데이터 관리 > 카테고리별 사용량
→ 뒤로가기
→ 데이터 관리 첫 화면으로 가는지 확인

테스트 5.
더보기 > 데이터 관리 > 원본 데이터 삭제
→ 페이지 전체 세로 스크롤 가능한지 확인

테스트 6.
원본 데이터 삭제 페이지 맨 아래까지 내려서
영구 삭제 실행 관련 버튼/영역이 다 보이는지 확인

테스트 7.
아이폰 작은 화면 기준에서도
하단 탭이나 브라우저 하단 영역에 마지막 버튼이 가려지지 않는지 확인

[11. 절대 금지]

- 뒤로가기 누르면 홈으로 바로 보내기 금지
- 데이터 관리 상세에서 더보기를 건너뛰고 다른 탭으로 보내기 금지
- 원본 데이터 삭제 화면에서 일부만 스크롤되게 하는 임시처리 금지
- 작은 화면에서 버튼이 가려진 상태로 완료 보고 금지

이번 작업은
데이터 관리 내부 상세 페이지가 공통 뒤로가기 규칙을 따르게 하고,
원본 데이터 삭제 페이지를 아이폰 환경에서도 끝까지 스크롤 가능하게 만드는 수정 작업이다.
