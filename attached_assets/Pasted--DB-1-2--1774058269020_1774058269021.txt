지금부터는 DB 연결이나 인증보다 먼저, 스윔노트의 본질 기능을 강화하는 작업을 우선해줘.
핵심은 두 가지다.

1. 스케줄러를 선생님 업무의 중심 허브로 전면 개편
2. 홈 화면에서 스케줄러/일지/출결/보강으로 이어지는 편의성 강화

이번 작업은 저장/서버보다 UI 구조, 화면 흐름, 작업 복귀, 조작 편의성에 집중한다.
즉, 지금 단계는 “실사용 감각”을 먼저 완성하는 단계다.

==================================================
1. 전체 방향
==================================================

- 스케줄러는 단순 일정 보기 화면이 아니라
  출결 / 일지 / 보강 / 날짜 메모의 중심 허브로 만든다.
- 선생님모드 기본 진입 스케줄러는 무조건 월간 스케줄
- 홈 화면에서도 오늘 해야 할 일과 오늘 수업을 스케줄러와 연결해서 보여준다
- 작업 후에는 홈으로 튕기지 말고 항상 작업하던 흐름으로 복귀시킨다

==================================================
2. 선생님모드 홈 화면 개편
==================================================

선생님 홈은 지금보다 더 한눈에 들어오게 정리해줘.
중요한 건 “오늘 바로 해야 할 것”이 보이는 구조다.

홈 화면 상단에는 아래를 배치:
- 선생님 이름 / 역할 표시
- (전환 가능한 계정이면) `관리자로 전환` 또는 `선생님으로 전환` 버튼
- 오늘 날짜
- 오늘 수업 요약

홈 화면 본문은 아래 순서로 정리:
1. 오늘 할 일
2. 오늘 수업
3. 미작성 일지
4. 보강 확인
5. 스케줄러 바로가기

[오늘 할 일]
- 오늘 처리 안 된 출결
- 오늘 미작성 일지
- 오늘 보강 예정
- 날짜 메모가 있으면 그것도 표시

[오늘 수업]
- 시간순으로 리스트업
- 각 수업 카드에는
  - 시간
  - 반명
  - 학생 수
  - 현재 상태 요약
- 카드 클릭 시 해당 날짜 스케줄러 상세 팝업 또는 반 상세로 연결

[미작성 일지]
- 오늘 기준 미작성 일지 우선 노출
- 클릭 시 바로 일지 작성 페이지 또는 해당 날짜 상세 팝업으로 연결

[보강 확인]
- 오늘 보강 예정/미처리 보강 표시
- 클릭 시 보강 관리 페이지로 이동

[스케줄러 바로가기]
- 홈에 스케줄러 메인 버튼 추가
- 누르면 월간 스케줄러로 이동

중요:
홈에서 각 카드/버튼을 누르면
사용자가 “무슨 기능을 눌러야 하지?” 고민하지 않게
바로 다음 업무 화면으로 가게 해줘.

==================================================
3. 스케줄러 메인 구조 전면 개편
==================================================

선생님모드 스케줄러는 월간 중심으로 전면 개편한다.

[기본 규칙]
- 기본 진입 화면은 무조건 월간 스케줄
- 상단 보기 탭은 `월 / 주 / 일`
- 기본값은 `월`
- 오늘 날짜 강조 표시

[월간 달력 날짜 칸 표시]
- 반 이름 표시하지 말 것
- 그 날짜에 수업이 있는 `시간`만 표시
예:
15시
16시
17시

- 수업이 여러 개면 시간순으로 표시
- 기본은 시간 표시 우선

==================================================
4. 날짜 클릭 동작
==================================================

월간 달력에서 날짜 칸 클릭 시
해당 날짜 상세 팝업(또는 바텀시트)이 열린다.

[토글 규칙]
- 날짜 클릭 → 상세 팝업 열림
- 같은 날짜 다시 클릭 → 상세 팝업 닫힘
- 팝업 바깥 빈 화면 클릭 → 팝업 닫힘

즉:
- 날짜는 토글 방식으로 동작
- 닫히면 다시 월간 달력 원화면만 보이게

==================================================
5. 날짜 상세 팝업 구조
==================================================

날짜 상세 팝업에는 아래 정보를 넣어줘.

- 선택한 날짜
- 그 날짜 수업 리스트
- 날짜 메모
- 작업 버튼

각 수업 카드에는 아래 정보 표시
- 시간
- 반명
- 인원 수
- 출석 상태 요약
- 일지 작성 여부

예:
15:00 초급반 / 5명
16:00 중급반 / 4명
17:00 개인레슨 / 1명

[수업 리스트 정렬]
- 항상 시간순
- 위에서부터 빠른 시간 → 늦은 시간
- 완료/미완료 그룹 분리 금지

[완료 표시 방식]
- 완료된 수업은 배지보다 글자 전체에 취소선(가로줄) 표시
- 예:
15:00 초급반 → 수업이 끝났으면 전체 텍스트 취소선
16:00 중급반 → 아직 안 끝났으면 일반 표시

즉:
- 시간순 정렬은 유지
- 끝난 일정만 가로줄로 표시

==================================================
6. 반 카드 클릭 시 동작
==================================================

날짜 상세 팝업 안에서 반 카드를 누르면
학생 리스트를 팝업 안에 직접 길게 펼치지 말고
반 상세 페이지로 이동시켜줘.

[반 상세 페이지 기능]
- 학생 리스트 확인
- 출석 체크
- 결석 처리
- 반 조정
- 학생 상태 확인
- 적용 버튼

[복귀 규칙]
- 반 상세에서 `적용` 또는 `저장` 누르면
  절대 홈으로 가지 말 것
- 반드시 방금 작업하던 `같은 날짜 상세 팝업 상태`로 복귀

예:
26일 팝업 열림
→ 15:00반 클릭
→ 반 상세 이동
→ 적용
→ 다시 26일 상세 팝업 복귀

==================================================
7. 보강 기능 연결
==================================================

날짜 상세 팝업 안에 `보강` 버튼 또는 탭 추가

동작:
- 보강 버튼 클릭
- 기존 보강 관리 리스트 페이지로 이동
- 보강 작업 후 `적용 완료`
- 다시 같은 날짜 상세 팝업으로 복귀

중요:
- 홈으로 보내지 말 것
- 월간 스케줄러 첫 화면만 띄우지 말 것
- 반드시 방금 열어놓은 날짜 팝업 상태 복귀

==================================================
8. 일지 작성 기능 연결
==================================================

날짜 상세 팝업 안에 `일지 작성` 버튼 추가

동작:
- 일지 작성 버튼 클릭
- 기존 일지 작성 페이지로 이동
- 일지 작성 완료
- 다시 같은 날짜 상세 팝업으로 복귀

복귀 후:
- 해당 수업 카드 상태가 즉시 `작성 완료`로 갱신되게 해줘
- 일지 작성은 팝업 안에서 긴 폼을 직접 입력하지 말고 별도 페이지 이동 유지

==================================================
9. 날짜 메모 기능
==================================================

메모는 공용 메모/반별 메모로 나누지 말고
`날짜 메모` 하나만 사용한다.

[메모 규칙]
- 날짜 단위 메모
- 날짜 상세 팝업 안에서 확인/수정 가능
- 달력 날짜 칸에는 메모 유무만 표시 가능

예:
- 학부모 요청
- 행사
- 특이사항
- 준비물
- 오늘 꼭 확인할 것

심플하게 유지할 것.

==================================================
10. + 버튼 기능
==================================================

스케줄러 우측 하단 + 버튼은 아래 기능으로 구성

- 수업 추가
- 보강 추가
- 개인레슨 추가
- 메모 추가

==================================================
11. 필터 기능
==================================================

스케줄러에 필터 기능 강화

필터 항목:
- 내 반만 보기
- 특정 반 선택
- 특정 시간대
- 보강만 보기
- 미작성 일지만 보기

필터는 월간/주간/일간 보기 전체에 적용 가능한 구조로 열어둘 것

==================================================
12. 일정 없음 화면
==================================================

날짜에 수업이 없으면 아래처럼 보여줘.

- `이 날은 수업이 없습니다`
- 보강 추가 버튼
- 메모 추가 버튼

빈 화면으로 끝나지 않게 해줘.

==================================================
13. 가장 중요한 복귀 규칙
==================================================

스케줄러에서 이동한 모든 작업은
무조건 같은 날짜 상세 팝업으로 복귀시켜라.

적용 대상:
- 반 상세
- 보강 관리
- 일지 작성

금지:
- 홈으로 튕기기
- 스케줄러 첫 화면만 띄우기
- 날짜 선택 상태 사라지기

정답 흐름:
월간 달력
→ 날짜 클릭
→ 날짜 상세 팝업
→ 반 상세 / 보강 / 일지 작성
→ 적용/완료
→ 다시 같은 날짜 상세 팝업 복귀

==================================================
14. 홈 화면과 스케줄러 연결 강화
==================================================

선생님 홈에서 스케줄러와 연결을 더 촘촘하게 해줘.

[오늘 할 일 카드]
- 클릭 시 오늘 날짜 상세 팝업 열기 또는 오늘 반 상세 바로 이동

[오늘 수업 카드]
- 클릭 시 해당 날짜 상세 팝업 열기

[미작성 일지]
- 클릭 시 해당 날짜/해당 수업 일지 작성으로 이동

[보강 확인]
- 클릭 시 보강 관리 리스트 이동

[스케줄러 버튼]
- 클릭 시 월간 스케줄러로 이동

즉 홈에서 스케줄러로 이어지는 동선이 매우 짧아야 한다.

==================================================
15. 가독성 개선
==================================================

현재 홈과 스케줄러가 아이콘화는 됐지만 한눈에 안 들어오는 느낌이 있으니
아래 기준으로 가독성 정리

- 홈에서는 오늘 해야 할 것 위주로 단순화
- 스케줄러에서는 날짜 클릭 전후 흐름이 직관적이어야 함
- 한 화면에 너무 많은 버튼을 늘어놓지 말고
  “다음 액션”이 뭔지 바로 알 수 있게 배치
- 팝업 안 수업 리스트는 시간순 유지
- 완료된 것은 취소선으로 직관적 구분

==================================================
16. 이번 단계에서 우선 구현할 것
==================================================

이번 단계는 DB/인증 붙이기 전에 아래만 먼저 완성

1. 선생님 홈 가독성 및 오늘 할 일 중심 구조
2. 월간 기본 스케줄러
3. 날짜 팝업
4. 반 상세 이동/복귀
5. 보강 이동/복귀
6. 일지 이동/복귀
7. 날짜 메모
8. 홈과 스케줄러 연결

==================================================
17. 작업 후 직접 확인할 체크리스트
==================================================

반드시 실제로 눌러서 확인해줘.

[홈]
1. 오늘 할 일 카드가 보이는지
2. 오늘 수업 카드가 시간순으로 보이는지
3. 미작성 일지/보강/스케줄러 버튼이 직관적으로 보이는지

[스케줄러]
4. 기본 진입이 월간인지
5. 날짜 칸에 반 이름이 아니라 수업 시간이 표시되는지
6. 날짜 클릭 시 팝업이 열리는지
7. 같은 날짜 다시 클릭 시 팝업이 닫히는지
8. 빈 화면 클릭 시 팝업이 닫히는지
9. 팝업 안 수업 리스트가 시간순인지
10. 완료된 수업이 취소선으로 표시되는지

[복귀]
11. 반 상세에서 적용 후 같은 날짜 팝업으로 복귀하는지
12. 보강 페이지에서 적용 완료 후 같은 날짜 팝업으로 복귀하는지
13. 일지 작성 완료 후 같은 날짜 팝업으로 복귀하는지
14. 날짜 메모 입력/수정이 되는지
15. 홈에서 눌렀던 카드가 스케줄러와 자연스럽게 연결되는지

==================================================
18. 보고 방식
==================================================

작업 후에는
- 어떤 화면을 바꿨는지
- 어떤 경로를 연결했는지
- 어떤 항목이 아직 미완성인지
솔직하게 정리해서 보고해줘.
작업했다고만 하지 말고 실제로 눌러서 확인한 결과를 같이 보여줘.
