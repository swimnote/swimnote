지금부터 선생님 모드 > 더보기에 “피드백커스텀” 메뉴를 추가하고,
일지 자동완성에 쓰이는 문장 목록을 선생님이 직접 수정할 수 있게 구현하라.

이번 작업 목표는
선생님이 더보기에서 자신의 피드백 문장 세트를 관리할 수 있게 만드는 것이다.

중요:
- 기존 일지 작성 기능은 절대 깨지면 안 된다
- 기존 문장 불러오기 팝업 구조는 유지
- 이번 작업은 “문장 세팅 편집 UI”를 추가하는 단계다
- 순서 변경 기능은 이번 단계에서 제외
- 샘플 문장 내용은 지금 하드코딩 최소 또는 placeholder 가능
- 기능 작동과 테스트까지 완료해야 한다

==================================================
[1. 메뉴 추가 위치]

선생님 모드 > 더보기 안에
새 메뉴를 추가한다.

메뉴명:
- 피드백커스텀

위치 추천:
- 설정 메뉴 또는 개인 설정 섹션 안
- 일지/수업 관련 설정과 가까운 위치

중요:
- 선생님 모드에만 우선 추가
- 관리자/학부모/슈퍼관리자에는 이번 단계에서 넣지 않음

==================================================
[2. 피드백커스텀 화면 역할]

이 화면은
선생님 개인의 일지 자동완성 문장 목록을 관리하는 페이지다.

기능:
1. 카테고리별 문장 보기
2. 문장 추가
3. 문장 수정
4. 문장 삭제
5. 카테고리 이름 수정
6. 카테고리 초기화
7. 전체 초기화

==================================================
[3. 카테고리 구조]

기본 카테고리 4개 유지

내부 key는 고정:
- beginner
- intermediate
- advanced
- custom

기본 라벨(표시명):
- 초급
- 중급
- 상급
- 커스텀

중요:
- 내부 key는 바뀌지 않음
- 라벨만 변경 가능
- 일지 작성 팝업에서도 이 라벨이 그대로 반영되어야 함

==================================================
[4. 초기 상태 규칙]

초기 상태는 아래처럼 구성

- 초급: 기본 샘플 문장 세트 존재
- 중급: 기본 샘플 문장 세트 존재
- 상급: 기본 샘플 문장 세트 존재
- 커스텀: 빈 상태로 시작

샘플 문장은 지금은 임시값 또는 placeholder로 두어도 됨
중요한 것은 구조와 수정 기능 구현이다

==================================================
[5. 문장 개수 규칙]

각 카테고리당 최대 문장 수:
- 100개

표시 방식:
- 빈 슬롯을 100개 다 보여주지 말 것
- 실제 등록된 문장만 리스트로 표시
- 상단에 현재 개수 표시

예:
- 현재 18 / 100

중요:
100개 초과 추가 금지

==================================================
[6. 화면 구성]

피드백커스텀 화면 구조:

상단:
- 제목: 피드백커스텀
- 설명 문구
  예: 일지 작성 시 불러올 문장을 직접 수정하고 관리할 수 있습니다.

그 아래:
- 카테고리 탭
  - 초급
  - 중급
  - 상급
  - 커스텀

카테고리별 본문:
- 현재 개수 표시 (예: 18 / 100)
- 문장 리스트
- 문장 추가 버튼
- 카테고리 이름 수정 버튼
- 현재 카테고리 초기화 버튼

화면 하단 또는 상단 우측:
- 전체 초기화 버튼

==================================================
[7. 문장 리스트 UI]

각 문장은 리스트 또는 카드 한 줄 구조로 표시한다.

한 줄에 포함:
- 문장 내용
- 수정 버튼
- 삭제 버튼

중요:
- 너무 복잡한 카드 말고 수정이 쉬운 단순 리스트형
- 모바일에서 한 손으로 누르기 쉽게 터치 영역 확보

==================================================
[8. 문장 추가 기능]

문장 추가 버튼을 누르면
현재 카테고리에 새 문장을 넣을 수 있어야 한다.

방식:
- 팝업 또는 바텀시트 입력창
- 제목: 문장 추가
- 입력 필드 1개
- 저장 / 취소

규칙:
- 최대 100자
- 빈 문장 저장 금지
- 저장 시 현재 카테고리 리스트에 즉시 반영

==================================================
[9. 문장 수정 기능]

문장 수정 버튼을 누르면
해당 문장을 수정할 수 있어야 한다.

방식:
- 팝업 또는 바텀시트
- 기존 문장 값 미리 채워놓기
- 저장 / 취소

규칙:
- 최대 100자
- 저장 시 리스트에 즉시 반영
- 수정 후 일지 작성 팝업에도 즉시 반영

==================================================
[10. 문장 삭제 기능]

문장 삭제 버튼을 누르면
바로 삭제하지 말고 확인 팝업을 띄운다.

문구 예:
- 이 문장을 삭제하시겠습니까?

버튼:
- 취소
- 삭제

삭제 후:
- 현재 카테고리 리스트에서 제거
- 일지 작성 팝업에서도 즉시 제거 반영

==================================================
[11. 카테고리 이름 수정 기능]

현재 선택한 카테고리의 표시명을 수정할 수 있어야 한다.

예:
- 초급 → 물적응
- 중급 → 자유형기초

방식:
- 팝업 입력
- 저장 / 취소

중요:
- 내부 key는 변경 금지
- label만 변경
- 변경 후 일지 작성 팝업의 탭 이름에도 즉시 반영

==================================================
[12. 카테고리 초기화 기능]

현재 선택한 카테고리만
기본 상태로 되돌릴 수 있어야 한다.

규칙:
- 초급/중급/상급: 기본 샘플 문장으로 복구
- 커스텀: 빈 상태로 복구
- 현재 수정 내용은 사라짐

확인 팝업 필수:
- 현재 카테고리 문장을 기본값으로 되돌리시겠습니까?

==================================================
[13. 전체 초기화 기능]

전체 초기화 버튼을 누르면
4개 카테고리 전체를 초기 상태로 복구한다.

복구 기준:
- 초급/중급/상급: 기본 샘플 문장
- 커스텀: 빈 상태

강한 경고 문구 필수:
- 현재 수정한 모든 피드백 문장이 삭제되고 기본값으로 복구됩니다.

버튼:
- 취소
- 전체 초기화

==================================================
[14. 저장 반영 규칙]

피드백커스텀에서 수정한 내용은
일지 작성 화면의 문장 불러오기 팝업에 즉시 반영되어야 한다.

반드시 반영될 것:
- 카테고리 이름
- 문장 추가
- 문장 수정
- 문장 삭제
- 카테고리 초기화
- 전체 초기화

즉:
앱 재실행 없이 바로 반영

==================================================
[15. 저장 범위]

이 설정은 선생님 개인 설정이다.

규칙:
- A선생님이 수정한 내용은 A선생님에게만 적용
- B선생님은 자기 설정 별도 유지
- 관리자 공통 설정과 섞지 말 것

==================================================
[16. 이번 단계에서 제외할 것]

이번 작업에서는 아래 기능 제외

- 문장 순서 변경
- 드래그 정렬
- 샘플 문장 고도화
- 관리자 공통 배포 기능
- 카테고리 추가/삭제 기능

즉:
4개 카테고리 고정 + 문장 CRUD + 초기화만 구현

==================================================
[17. 먼저 확인할 것]

수정 전에 아래를 먼저 파악하라.

1. 선생님 더보기 화면 파일
2. 현재 일지 작성 화면 파일
3. 문장 불러오기 팝업 파일
4. 자동완성 문장 데이터 저장 구조
5. 선생님 계정별 설정 저장 가능한 구조
6. 공통 팝업/바텀시트 컴포넌트

==================================================
[18. 테스트 - 반드시 실제 수행]

테스트 1.
선생님 더보기에서 피드백커스텀 메뉴가 보이는지 확인

테스트 2.
피드백커스텀 진입 시 4개 카테고리 탭이 정상 표시되는지 확인

테스트 3.
문장 추가 후 현재 카테고리 리스트에 즉시 보이는지 확인

테스트 4.
문장 수정 후 리스트에 즉시 반영되는지 확인

테스트 5.
문장 삭제 시 확인 팝업 후 실제 삭제되는지 확인

테스트 6.
카테고리 이름 수정 후 피드백커스텀과 일지 작성 팝업 양쪽에 즉시 반영되는지 확인

테스트 7.
카테고리 초기화 시 현재 카테고리만 기본 상태로 복구되는지 확인

테스트 8.
전체 초기화 시 4개 카테고리 전부 초기 상태로 돌아가는지 확인

테스트 9.
커스텀 카테고리 초기화 시 빈 상태로 돌아가는지 확인

테스트 10.
한 카테고리에서 100개 초과 추가가 막히는지 확인

테스트 11.
피드백커스텀에서 저장한 문장이 실제 일지 작성 > 문장 불러오기 팝업에서 바로 보이는지 확인

테스트 12.
A선생님과 B선생님 설정이 서로 섞이지 않는지 확인

[20. 절대 금지]

- 피드백커스텀에서 바꿨는데 일지 작성 팝업에 반영 안 되는 상태 금지
- 문장 100개 제한 없이 추가되는 상태 금지
- 카테고리 이름 변경 시 내부 key까지 바꾸는 것 금지
- 커스텀 초기화가 샘플 문장으로 채워지는 것 금지
- 기존 일지 자동완성 기능 깨뜨리기 금지

이번 작업은
선생님 더보기 안에 피드백커스텀 메뉴를 만들고,
선생님이 자신의 일지 자동완성 문장 세트를 카테고리별로 직접 추가/수정/삭제/초기화할 수 있게 하며,
변경 내용이 일지 작성 화면에 즉시 반영되도록 만드는 작업이다.
