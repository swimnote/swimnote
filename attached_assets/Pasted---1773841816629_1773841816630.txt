지금부터 앱 전체의 뒤로가기 규칙을 공통 규칙으로 통일하라.

이번 작업 목표는
각 탭/모드 안에서 하위 메뉴를 들어갔다가 뒤로가기를 누르면
항상 그 탭(모드)의 첫 화면까지 단계적으로 복귀하도록 만드는 것이다.

가장 중요:
- 뒤로가기는 절대 홈이나 다른 탭으로 바로 튀면 안 된다
- 마지막 복귀점은 항상 현재 탭/모드의 첫 화면이어야 한다
- 특히 더보기 안의 모든 메뉴는 마지막에 더보기 첫 화면으로 돌아와야 한다
- 이 규칙은 더보기뿐 아니라 다른 모드/탭도 전부 동일하게 적용해야 한다

==================================================
[1. 공통 최종 규칙]

모든 탭/모드에 아래 규칙을 적용한다.

1. 하위 화면에서 뒤로가기
→ 바로 이전 단계로 이동

2. 뒤로가기를 계속 누르면
→ 최종적으로 현재 탭/모드의 첫 화면으로 이동

3. 현재 탭/모드의 첫 화면에서는
→ 내부 뒤로가기는 더 이상 내려가지 않음
(홈이나 다른 탭으로 자동 이동 금지)

4. 다른 하단 탭을 누르면
→ 그 탭으로 이동

5. 같은 하단 탭을 다시 누르면
→ 그 탭의 첫 화면으로 복귀
→ 스크롤도 초기화

==================================================
[2. 더보기 탭 규칙 - 가장 중요]

더보기 안의 모든 메뉴/상세/하위 페이지는
뒤로가기를 계속 누르면
마지막에는 반드시 “더보기 첫 화면”으로 돌아와야 한다.

예시:
더보기 > 데이터 관리 > 계정별 사용량 > 계정 상세
뒤로가기 1회 → 계정별 사용량
뒤로가기 2회 → 데이터 관리 첫 화면
뒤로가기 3회 → 더보기 첫 화면

중요:
- 더보기 내부에서 뒤로가기 마지막이 홈으로 가면 안 된다
- 다른 탭으로 넘어가면 안 된다
- 반드시 더보기 첫 화면이어야 한다

==================================================
[3. 이 규칙을 적용할 범위]

전체 앱 공통 적용

대상:
- 관리자 모드
- 선생님 모드
- 학부모 모드
- 슈퍼관리자 모드(현재 있으면 포함, 없으면 앞으로 적용 가능 구조로)

각 모드 안의 모든 하단 탭에도 적용:
예:
- 홈
- 사람
- 수업
- 메신저
- 매출관리
- 더보기
등

즉:
각 탭의 내부 메뉴는 전부 자기 탭 첫 화면까지 단계적으로 복귀해야 한다.

==================================================
[4. 예시로 정리]

--------------------------------------
예시 A. 더보기 탭
더보기 > 데이터 관리 > 원본 데이터 삭제
뒤로가기 → 데이터 관리 첫 화면
뒤로가기 → 더보기 첫 화면

--------------------------------------
예시 B. 사람 탭
사람 > 선생님 관리 > 선생님 상세
뒤로가기 → 선생님 관리
뒤로가기 → 사람 첫 화면

--------------------------------------
예시 C. 수업 탭
수업 > 특정 반 상세 > 출결관리
뒤로가기 → 특정 반 상세
뒤로가기 → 수업 첫 화면

--------------------------------------
예시 D. 매출관리 탭
매출관리 > 휴무일 지정 > 상세/설정
뒤로가기 → 휴무일 지정
뒤로가기 → 매출관리 첫 화면

==================================================
[5. 절대 금지할 동작]

다음 동작이 나오면 안 된다.

1. 뒤로가기를 눌렀는데 바로 홈으로 튀는 것
2. 뒤로가기를 눌렀는데 다른 탭으로 이동하는 것
3. 현재 탭의 첫 화면으로 가야 하는데 중간 단계가 무시되는 것
4. 더보기 내부에서 마지막 복귀점이 더보기가 아닌 것
5. 어떤 탭은 되고 어떤 탭은 안 되는 불일치

==================================================
[6. 구현 방식]

현재 앱의 각 탭/모드 내 라우팅 구조를 확인하고
각 탭마다 “첫 화면(route root)” 개념을 명확히 잡아라.

필수 구현 개념:
- 각 탭마다 root screen 정의
- 내부 stack/history는 탭별로 유지
- back action은 현재 탭 stack 내에서만 pop
- stack이 root까지 도달하면 거기서 멈춤
- 같은 탭 재탭 시 해당 탭 root로 reset + scroll top

즉:
전역 뒤로가기 금지
탭 내부 스택 뒤로가기만 허용

==================================================
[7. 더보기 탭 특별 확인]

더보기는 메뉴가 많아서 가장 헷갈리기 쉬우므로
반드시 모든 하위 메뉴에서 아래가 맞는지 확인하라.

예상 구조:
더보기 첫 화면
 ├ 승인 관리
 ├ 선생님 관리
 ├ 학부모 계정
 ├ 데이터 관리
 │   ├ 저장공간 현황
 │   ├ 계정별 사용량
 │   ├ 카테고리별 사용량
 │   ├ 원본 데이터 삭제
 │   └ 이벤트 기록
 ├ 구독관리
 ├ 수영장 설정
 └ 기타 설정

이 모든 경로에서
뒤로가기를 반복하면 마지막은 더보기 첫 화면이어야 한다.

==================================================
[8. 하단 탭 규칙 재확인]

하단 탭 동작 규칙도 같이 맞춘다.

1. 다른 탭 누름
→ 해당 탭으로 이동
→ 기존 상태는 유지 가능

2. 현재 탭을 다시 누름
→ 그 탭 첫 화면으로 복귀
→ 내부 stack reset
→ 스크롤 초기화

중요:
팝업은 이 규칙에 포함하지 않는다
팝업은 기존 팝업 닫기 규칙 유지

==================================================
[9. 먼저 확인할 것]

수정 전에 아래를 먼저 확인하라.

1. 각 모드의 하단 탭 네비게이션 구조
2. 각 탭의 root screen 정의 방식
3. back button 공통 헤더/공통 네비게이션 처리 파일
4. 더보기 내부 하위 페이지 라우팅 구조
5. 데이터 관리 내부 라우팅 구조
6. 사람/수업/매출관리/메신저 내부 라우팅 구조

==================================================
[10. 테스트]

반드시 실제 테스트하라.

테스트 1.
더보기 > 데이터 관리 > 저장공간 현황
→ 뒤로가기
→ 데이터 관리 첫 화면
→ 뒤로가기
→ 더보기 첫 화면

테스트 2.
더보기 > 데이터 관리 > 계정별 사용량 > 특정 계정 상세
→ 뒤로가기 반복
→ 마지막이 더보기 첫 화면인지 확인

테스트 3.
사람 > 선생님 관리 > 선생님 상세
→ 뒤로가기 반복
→ 마지막이 사람 첫 화면인지 확인

테스트 4.
수업 > 특정 반 > 하위 관리 화면
→ 뒤로가기 반복
→ 마지막이 수업 첫 화면인지 확인

테스트 5.
매출관리 > 휴무일 지정 > 하위 설정
→ 뒤로가기 반복
→ 마지막이 매출관리 첫 화면인지 확인

테스트 6.
현재 탭(예: 더보기)을 다시 한 번 누르면
→ 더보기 첫 화면으로 복귀하는지 확인

테스트 7.
현재 탭 재탭 시 스크롤도 초기화되는지 확인

[12. 절대 금지]

- 뒤로가기 누르면 홈으로 보내기 금지
- 더보기 안에서 마지막 복귀점이 더보기 첫 화면이 아니게 만드는 것 금지
- 탭마다 다른 규칙 쓰기 금지
- 현재 탭 재탭 시 첫 화면 복귀 안 되는 상태 금지
- 일부 화면만 수정하고 완료 보고 금지

이번 작업은
모든 모드/탭에서 뒤로가기를 누르면 반드시 현재 탭의 첫 화면까지 단계적으로 복귀하고,
특히 더보기 내부 메뉴는 마지막에 반드시 더보기 첫 화면으로 돌아오게 만드는 공통 네비게이션 수정 작업이다.
