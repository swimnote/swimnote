[스윔노트 초대방식 메뉴 수정 + 기본 문자앱 연결 + 자동완성 템플릿 시스템 구현 지시서]

지금부터 운영설정 > 초대방식 기능을 실제 운영 기준으로 다시 구현한다.
이번 작업의 핵심은 SMS 과금/카운팅 구조를 전부 제거하고,
기기의 기본 문자메시지 앱을 호출하는 방식으로
선생님/학부모 초대 메시지를 자동완성해서 보낼 수 있게 만드는 것이다.

중요:
- 우리는 SMS 서비스를 판매하거나 과금하지 않는다
- 문자 크레딧, 문자 건수 과금, SMS 발송량 정산 구조 전부 제거
- 앱은 “기본 문자앱 열기 + 수신번호/본문 자동완성”만 담당
- 문자 전송 자체는 사용자의 기기 문자앱에서 처리
- 문자앱 호출 로그와 초대 시도 로그는 남길 것
- iOS/Android 모두 동작하게 만들 것
- 기능은 실제 작동 기준으로 구현하고, 버튼만 보이게 두지 말 것

==================================================
1. 운영설정 > 초대방식 기능 목적 재정의
==================================================

현재 초대방식 메뉴는 SMS 과금 구조가 아니라
“기본 문자앱을 이용한 초대 기능”으로 동작해야 한다.

초대 대상:
- 선생님
- 학부모

핵심 동작:
1. 초대 버튼 누름
2. 프로필에 저장된 전화번호 자동 반영
3. 문자 내용 자동완성
4. 기기의 기본 문자앱 열기
5. 사용자가 문자앱에서 직접 발송

==================================================
2. 제거할 기능
==================================================

아래 기능은 전부 제거하거나 비활성 처리:
- SMS 과금 구조
- 문자 건수 카운팅
- 문자 크레딧
- 관리자/선생님/학부모 SMS 사용량 과금
- 문자 발송 잔여 횟수
- 문자 서비스 비용 정산 UI

단,
완전히 제거하지 말고 로그 목적의 아래는 유지:
- 초대 시도 로그
- 문자앱 호출 로그
- 누가 누구에게 초대를 시도했는지 기록

==================================================
3. 초대 메시지 템플릿 구조
==================================================

초대 메시지는 선생님용 / 학부모용을 분리한다.

--------------------------------------------------
3-1. 선생님 초대 문구
--------------------------------------------------

선생님 초대 문구는 수정 불가 고정 템플릿으로 구현한다.

고정 기본 문구:
“{수영장이름} 수영장에서 선생님으로 초대했습니다. 링크를 확인해주세요.
{iOS링크}
{Android링크}”

규칙:
- {수영장이름} = 관리자가 등록한 현재 수영장 이름으로 자동 치환
- 다운로드 링크는 앱스토어/플레이스토어 링크 자동 포함
- 문구 수정 기능 없음
- 관리자는 미리보기만 가능

선생님용 문구를 고정하는 이유:
- 운영상 문구 일관성 유지
- 실수 방지
- 관리 복잡도 감소

--------------------------------------------------
3-2. 학부모 초대 문구
--------------------------------------------------

학부모 초대 문구는 관리자 수정 가능하게 구현한다.
다만 완전 자유 텍스트만 두지 말고,
변수 치환 템플릿 방식으로 만든다.

기본 문구:
“{수영장이름}에서 스윔노트를 이용할 수 있도록 {학생이름} 학부모님을 초대합니다.
앱을 설치한 뒤 링크를 확인해주세요.
{iOS링크}
{Android링크}”

수정 가능 범위:
- 학부모 초대 문구 본문 수정 가능
- 단, 아래 변수는 유지 가능해야 함:
  - {수영장이름}
  - {학생이름}
  - {iOS링크}
  - {Android링크}

권장 방식:
- 기본 템플릿 제공
- 관리자는 템플릿 문장만 수정 가능
- 변수는 버튼으로 삽입 가능
- 미리보기 제공

==================================================
4. 자동 치환 변수
==================================================

문자 작성 시 아래 변수를 자동으로 치환한다.

필수 변수:
- {수영장이름} = 현재 로그인한 운영자의 수영장명
- {학생이름} = 학부모 초대 시 선택한 학생 프로필 이름
- {iOS링크} = 앱스토어 다운로드 링크
- {Android링크} = 플레이스토어 다운로드 링크

중요:
- 선생님 초대에는 학생이름 변수 불필요
- 학부모 초대에는 학생이름 자동 입력 필수

==================================================
5. 전화번호 자동 연결
==================================================

프로필에 전화번호가 저장되어 있으면
초대 버튼 클릭 시 문자앱을 열 때 수신번호도 같이 채워서 열어야 한다.

적용 대상:
- 선생님 프로필
- 학부모 프로필

동작:
1. 프로필에서 초대 버튼 클릭
2. 저장된 전화번호 읽기
3. 문자앱 열기
4. 수신번호 자동 입력
5. 본문 자동완성

중요:
전화번호가 없으면 문자앱은 열리되,
전화번호 입력이 필요하다는 안내 표시

==================================================
6. 플랫폼별 문자앱 호출 방식
==================================================

반드시 iOS / Android 모두 대응한다.

목표:
- 각 기기의 기본 문자앱을 여는 것
- 수신번호와 메시지 본문을 함께 넘기는 것

iOS:
- 문자 작성 가능 여부 확인 후 메시지 작성 화면 열기
- 수신자 / 본문 사전 입력

Android:
- 기본 SMS 인텐트로 문자앱 열기
- 수신번호 / 본문 사전 입력

중요:
- 특정 서드파티 메시지 앱 강제 금지
- 기기 기본 메시지 처리 방식 사용
- 발송은 사용자가 최종 수행

==================================================
7. 초대 방식 UI 구성
==================================================

운영설정 > 초대방식 화면에 아래 구성

A. 초대 방식 안내
- “스윔노트는 문자 발송 서비스를 판매하지 않습니다”
- “기기의 기본 문자앱을 열어 초대 메시지를 보냅니다”

B. 선생님 초대 문구
- 고정 문구 미리보기
- 복사 버튼(optional)
- 테스트 열기 버튼

C. 학부모 초대 문구
- 기본 템플릿 표시
- 수정 입력창
- 변수 삽입 버튼
- 미리보기
- 저장 버튼
- 초기화 버튼

D. 다운로드 링크 설정
- 앱스토어 링크
- 플레이스토어 링크
- 복사 버튼
- 현재 적용 링크 표시

==================================================
8. 프로필 화면 연동
==================================================

선생님/학부모 프로필 또는 목록에서
초대 버튼을 누르면 자동 동작해야 한다.

--------------------------------------------------
8-1. 선생님 초대
--------------------------------------------------
- 선생님 프로필에서 초대 버튼
- 전화번호 자동 반영
- 고정 선생님 문구 자동완성
- 기본 문자앱 열기

--------------------------------------------------
8-2. 학부모 초대
--------------------------------------------------
- 학부모 프로필 또는 학생 프로필 기준 초대 버튼
- 전화번호 자동 반영
- 선택 학생 이름 자동 반영
- 저장된 학부모 템플릿 자동완성
- 기본 문자앱 열기

중요:
학부모가 자녀 여러 명인 경우,
초대 발송 시 현재 선택된 학생 이름이 자동 반영되게 할 것

==================================================
9. 로그 기록
==================================================

문자 자체 과금/발송 성공 카운트는 하지 않지만
초대 시도와 문자앱 호출 기록은 남긴다.

기록 예시:
- createdAt
- actor
- target_type (teacher / parent)
- target_name
- target_phone
- pool_id
- student_name(optional)
- invite_type
- message_template_type
- open_sms_app_success
- summary

중요:
이 로그는 복구/감사/운영 추적 목적이다.

==================================================
10. 수정 정책 결정
==================================================

최종 정책은 아래로 구현한다.

- 선생님 초대 문구: 수정 불가
- 학부모 초대 문구: 수정 가능
- 단, 변수 치환 방식 유지
- 수영장 이름 자동 치환
- 학생 이름 자동 치환
- 전화번호 자동 반영
- 앱스토어/플레이스토어 링크 자동 포함

==================================================
11. 예외 처리
==================================================

아래 예외를 반드시 처리

1. 전화번호 없음
- 문자앱 열기 전 안내
- 전화번호 등록 유도

2. 학생 이름 없음
- 학부모 초대 버튼 비활성 또는 오류 안내

3. 링크 미설정
- 기본 공용 링크 사용 또는 경고 표시

4. 문자앱 사용 불가 기기
- 안내 메시지 표시
- “문자 발송이 가능한 기기에서 이용해주세요”

12. 수정 금지 영역
==================================================

이번 작업에서 깨지면 안 되는 것:

- 기존 운영설정 전체 레이아웃
- 기존 수영장 이름 관리 구조

14. 최종 목적
==================================================

이번 작업의 목적은
SMS 과금 서비스 없이도
운영자가 선생님/학부모를 빠르게 초대할 수 있게
기본 문자앱 자동완성 초대 시스템을 완성하는 것이다.

핵심:
- SMS 과금 제거
- 기본 문자앱 연동
- 수신번호 자동입력
- 수영장 이름 자동치환
- 학생 이름 자동치환
- 선생님 문구 고정
- 학부모 문구 수정 가능
- iOS/Android 동시 대응
- 로그 기록 유지

이 기준으로 꼼꼼하게 구현하라.
