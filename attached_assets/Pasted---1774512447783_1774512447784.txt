[스윔노트 — 실사용 전 최종 보강 지시문 (수영장 검색 / 내정보 / 승인 더미 제거 / 학부모 초대 / 브랜드설정 / 선생님 설정 / 정산 기타 반영)]

이번 작업은 테스트용 구조를 걷어내고, 실제 운영자가 바로 쓸 수 있게 만드는 최종 보강 작업이다.
기능을 새로 복잡하게 만드는 게 아니라, 막혀 있는 흐름을 실제 사용 가능 상태로 정리하는 것이 목표다.

━━━━━━━━━━━━━━━━━━
1. 학부모 수영장 검색 정상화
━━━━━━━━━━━━━━━━━━

현재 문제:
- 수영장 관리자로 토이키즈 스윔 클럽을 등록했는데
- 학부모가 수영장 검색 시 결과에 안 나옴

수정 목표:
- 등록된 수영장 목록이 학부모 수영장 검색에 보여야 함
- 검색어 입력 시 해당 수영장이 정확히 검색되어야 함

필수 규칙:
- 검색 대상은 superAdminDb의 swimming_pools 기준
- approval_status = approved 인 수영장만 노출
- hidden / deleted / 비활성 수영장 제외
- 부분 검색 가능하게 할 것
- 예: “토이”, “스윔”, “클럽” 모두 검색 가능

검증:
- 토이키즈 스윔 클럽 검색 시 노출 확인
- 빈 검색 시 등록된 수영장 목록 기본 노출 확인
- 다른 수영장도 검색 가능 확인

━━━━━━━━━━━━━━━━━━
2. 관리자 / 선생님 / 학부모 내정보 구조 정리
━━━━━━━━━━━━━━━━━━

현재 문제:
- 관리자 모드에는 내정보가 없음
- 비밀번호 변경도 없음
- 관리자 사용자가 선생님 모드로 가서만 일부 편집 가능

수정 목표:
- 관리자 / 선생님 / 학부모 모두 “내 정보” 진입 가능
- 비밀번호 변경 기능을 내 정보 안에 추가
- 관리자도 자기 정보 확인/수정 가능해야 함

필수 규칙:
- 내 정보 항목에 아래 포함
  - 이름
  - 연락처
  - 이메일(있으면)
  - 비밀번호 변경
- 비밀번호 변경은 현재 비밀번호 확인 + 새 비밀번호 입력 구조
- 저장 시 superAdminDb auth/user 기준으로 실제 반영

관리자:
- 관리자 모드에서도 내 정보 메뉴 제공
- 더보기 / 설정 안에서 진입 가능

선생님:
- 기존 이름 카드 옆 편집 구조 유지 가능
- 단, 비밀번호 변경 항목 반드시 추가

학부모:
- 향후 동일 구조 사용 가능하게 맞춰둘 것

검증:
- 관리자 모드에서 내 정보 진입 가능 확인
- 비밀번호 변경 성공 확인
- 변경 후 새 비밀번호 로그인 가능 확인

━━━━━━━━━━━━━━━━━━
3. 승인관리 더미 데이터 완전 제거
━━━━━━━━━━━━━━━━━━

현재 문제:
- 승인 관리에 박수진, 최민호, 강지연
- 승인 탭에 김영희, 이철수
- 거절됨에 정미경 등 테스트용 더미 이름이 남아 있음

수정 목표:
- 승인관리 화면의 테스트 더미 데이터 완전 제거
- 실제 가입 신청/승인 데이터만 보이게 할 것

필수 규칙:
- UI 숨김 금지
- 실제 DB 기준 삭제 또는 초기화
- 하드코딩 배열/샘플 상태값/목데이터 전부 제거

검증:
- 승인관리 목록에 더미 이름 0건
- 실제 가입 요청이 있을 때만 표시
- 새로고침 후 다시 안 나타남

━━━━━━━━━━━━━━━━━━
4. 학부모 직접 추가 구조 개편
━━━━━━━━━━━━━━━━━━

현재 문제:
- 이름 / 전화번호 / PIN 번호 구조가 실제 운영상 의미 약함
- 학부모와의 연결 흐름이 약함

수정 목표:
- PIN 번호 제거
- 이름 + 전화번호만 입력해서 학부모 직접 추가
- 추가 시 학부모에게 앱 다운로드 안내 문자 발송 흐름 연결

입력 항목:
- 학부모 이름
- 학부모 전화번호

제거:
- PIN 번호 필드 제거

버튼 동작:
- 추가 버튼 클릭 시 팝업 표시

팝업 문구 예:
“해당 학부모에게 애플리케이션 다운로드 안내 메시지가 발송됩니다.”

동작 방식:
- 실제 SMS API 연동 전까지는 휴대폰 기본 메시지 앱 열기 방식 사용
- 전화번호 자동 입력
- 안내 문구 자동 입력
- 사용자는 전송 버튼만 누르면 되게 구성

문자 앱 열기 규칙:
- sms: 또는 기기 메시지 앱 호출
- 받는 번호 자동 입력
- 메시지 본문 자동 입력
- 메시지 앱 발송 후 앱 복귀 시 현재 화면 유지 또는 자연 복귀

안내 문구 예시:
“안녕하세요. 스윔노트 학부모 앱 설치 안내입니다. 아래 경로에서 앱을 다운로드 후 회원가입을 진행해주세요.”

중요:
- 메시지 자동 전송 금지
- 사용자가 최종 전송 버튼만 누르는 구조
- 실제 앱 다운로드 링크 자리 포함 가능

검증:
- 이름/전화번호만으로 추가 가능
- 메시지 앱 열림
- 번호 자동 입력 확인
- 메시지 본문 자동 입력 확인
- 앱 복귀 후 현재 화면 정상 유지

━━━━━━━━━━━━━━━━━━
5. 학부모 승인 요청 연동
━━━━━━━━━━━━━━━━━━

추가 규칙:
- 학부모가 앱에서 가입/승인 요청을 하면
- 관리자 승인 관리 또는 학부모 연결 대기 목록으로 올라와야 함

즉 두 경로 모두 존재:
1. 관리자가 직접 추가 → 안내 메시지 발송
2. 학부모가 직접 가입 요청 → 승인 목록에 표시

검증:
- 직접 추가와 가입 요청 흐름이 서로 충돌하지 않는지 확인

━━━━━━━━━━━━━━━━━━
6. 수영장 관리자 공지 = 메시지처럼 표시
━━━━━━━━━━━━━━━━━━

정책:
- 수영장 관리자가 보내는 공지는 별도 복잡한 구조보다
- 메시지처럼 자연스럽게 보이게 처리

표시 방향:
- “공지사항입니다” 성격의 메시지형 공지
- 앱 내 공지 팝업 / 공지함에서 읽기 쉽게 단순 노출
- 이미 만든 공지 시스템 안에서 표현만 더 단순하게 유지

━━━━━━━━━━━━━━━━━━
7. 화이트라벨 / 브랜드 설정 정리
━━━━━━━━━━━━━━━━━━

화이트라벨 정책:
- 기본은 비활성
- 유료 사용자만 활성화 가능

표시 문구:
“화이트 라벨 옵션에 구독하실 경우 스윔노트 이름 대신 등록된 수영장 이름 또는 관리자 이름만 표시됩니다.”

구현 규칙:
- 무료/미구독 상태: 비활성, 잠금 또는 안내 문구 표시
- 유료 상태: 활성 가능

브랜드 설정:
- 색상/명칭 등 커스텀은 유지 가능
- 세부 구성은 자유롭게 두되 과도하게 복잡하게 만들지 말 것

━━━━━━━━━━━━━━━━━━
8. 수영장 관리 메뉴 제거
━━━━━━━━━━━━━━━━━━

현재 판단:
- “수영장 관리” 메뉴에서 새 수영장을 추가하는 구조는 지금 단계에서 불필요
- 프랜차이즈/체인점 복사 구조는 나중 단계로 미룸

수정 목표:
- 현재 앱에서 “수영장 관리” 탭/아이콘 제거
- 지금은 각 수영장이 개별 가입하는 방식으로 운영

검증:
- 수영장 관리 메뉴가 UI에서 사라졌는지 확인
- 관련 라우트 진입 버튼 제거 확인

━━━━━━━━━━━━━━━━━━
9. 수영장 설정의 사업자등록번호 입력 가능화
━━━━━━━━━━━━━━━━━━

현재 문제:
- 수영장 설정 안의 사업자등록번호가 입력 불가

수정 목표:
- 사업자등록번호 입력 가능하게 변경
- 저장 가능하게 할 것

규칙:
- 숫자 및 하이픈 허용
- 저장 시 superAdminDb의 수영장 설정/정보 테이블에 반영
- 읽기 전용 금지

검증:
- 입력 가능 확인
- 저장 후 재진입 시 값 유지 확인

━━━━━━━━━━━━━━━━━━
10. 선생님 설정 화면 정리
━━━━━━━━━━━━━━━━━━

현재 문제:
- 선생님이 앱 설정(테마 변경)을 못 바꾸는데 메뉴는 있음
- 불필요한 설정이 남아 있음

수정 목표:
- 선생님이 실제 못 쓰는 앱 설정 항목 제거
- 대신 선생님에게 필요한 알림 설정만 남김

수정 규칙:
- 선생님 설정에서 “앱 설정” 제거
- 알림 설정은 유지
- 선생님에게 해당하는 알림 종류는 모두 개별 ON/OFF 가능하게 연결

예:
- 공지 알림
- 출결 관련 알림
- 보강 관련 알림
- 메시지/업무 알림

검증:
- 선생님 설정에서 앱 설정 메뉴 제거 확인
- 알림 토글 저장 확인

━━━━━━━━━━━━━━━━━━
11. 선생님 내 정보 비밀번호 변경
━━━━━━━━━━━━━━━━━━

현재 문제:
- 선생님 내 정보에 비밀번호 변경이 없음

수정 목표:
- 선생님 내 정보에서 비밀번호 변경 가능

규칙:
- 이름 카드 옆 편집 또는 내 정보 화면 안에 비밀번호 변경 항목 추가
- 실제 auth 반영
- 변경 후 재로그인 검증

━━━━━━━━━━━━━━━━━━
12. 정산의 기타 항목 보강
━━━━━━━━━━━━━━━━━━

현재 문제:
- 다른 선생님이 나에게 보낸 보충수업이나
- 소멸된 수업이 정산에서 기타로 안 보임

수정 목표:
- 정산에 기타 항목 표시
- 기타 안에서 최소 아래 2종 구분

표시 방식 예:
- 보충 이동
- 소멸 수업

또는
- 내가 받은 수업
- 소멸 수업

권장:
- 기타 아래 세부 구분 표시
- 현재 월 정산에 수량 반영

내부 기준:
- 다른 선생님에게서 넘겨받은 보강 = handover_to_teacher
- 소멸 처리 = self_extinguish 또는 expired 계열

검증:
- 다른 선생님이 나에게 넘긴 건이 기타에 보이는지
- 소멸 처리 건이 기타에 보이는지
- 중복 계산 없는지 확인

━━━━━━━━━━━━━━━━━━
13. 환불 처리 기준 확인
━━━━━━━━━━━━━━━━━━

정책 확인:
- 수업 중간 환불 시
- 환불 전까지 진행한 수업만 출석/정산 집계
- 이후 수업은 집계 제외

이번 단계에서는:
- 환불 이후 미래 수업 집계 제외 로직 점검
- 별도 새 UI보다 기존 정산/출결 계산이 맞는지 우선 확인

━━━━━━━━━━━━━━━━━━
14. 실제 검증 항목
━━━━━━━━━━━━━━━━━━

반드시 실제로 확인:

1. 학부모 수영장 검색 시 등록 수영장 노출
2. 관리자 내 정보 진입 가능
3. 관리자 비밀번호 변경 가능
4. 선생님 내 정보 비밀번호 변경 가능
5. 승인관리 더미 데이터 제거
6. 학부모 직접 추가 시 PIN 없이 진행
7. 메시지 앱 열기 + 번호/본문 자동 입력
8. 학부모 승인 요청 목록 정상 표시
9. 수영장 관리 메뉴 제거
10. 사업자등록번호 입력/저장 가능
11. 선생님 설정에서 앱 설정 제거
12. 선생님 알림 설정 토글 가능
13. 정산 기타 항목 표시
14. 환불 이후 미래 수업 집계 제외 확인

- 수정 파일 경로
