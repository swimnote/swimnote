지금부터 관리자 모드와 선생님 모드에 내부 업무 메신저 기능을 추가하라.
이번 작업은 단순 UI 추가가 아니라, 실제로 메시지가 오가고, 사진이 저장되고, 회원 이전 카드가 작동하는 수준까지 구현하고 테스트해야 한다.

중요:
- 이 메신저는 카톡 대체제가 아니라 수영장 내부 업무 전달 메신저다.
- 학부모는 사용하지 않는다.
- 관리자와 선생님만 사용한다.
- 슈퍼관리자는 이 메신저에 진입하지 않는다.
- 기능만 보이게 하지 말고 실제 데이터 저장/반영/전달까지 구현하라.
- 수정 후 반드시 충분히 테스트하고, 관리자↔선생님 / 선생님↔선생님 간 메시지 전달이 실제 되는지 검증 보고하라.

--------------------------------------------------
[1. 메신저 기본 구조]

메신저는 관리자 모드와 선생님 모드의 하단 탭에 직접 추가한다.
더보기 안에 넣지 말고, 메인 기능 탭으로 바로 접근 가능하게 한다.

메신저는 한 수영장 내부의 관리자/선생님만 사용하는 업무용 메신저다.

핵심 성격:
- 전체 공용 메신저
- 특정 대상 지정 메시지 가능
- 특정 대상 메시지도 전체 메신저 안에 공개적으로 보임
- 사진 첨부 가능
- 영상 첨부 불가
- 회원 이전 카드 전송 가능

--------------------------------------------------
[2. 메시지 표시 방식]

2-1. 전체 메시지
대상 지정 없이 보내면 전체 메시지로 처리한다.
표시 방식:
- 보낸 사람 아이콘
- 보낸 사람 이름
- 메시지 내용
- 시간

예시 느낌:
김선생: 오늘 5시반 보강 자리 있습니다

2-2. 특정 대상 메시지
특정 선생님 또는 관리자를 선택해서 보내면
메시지창에 "보낸사람 → 받는사람" 구조로 표시한다.

표시 방식:
- 보낸 사람 아이콘 + 이름
- 연결 표시(→)
- 받는 사람 아이콘 + 이름
- 메시지 내용
- 시간

예시 느낌:
김선생 → 이선생 : 서태웅 회원 전반 요청합니다

중요:
- 특정 대상 메시지도 전체 메신저 안에 남는다.
- 비공개 1:1 채팅방 구조가 아니다.
- 업무 전달 흐름이 모두 보이게 만든다.

--------------------------------------------------
[3. 메신저 화면 구조]

3-1. 상단
- 대상 선택 영역
- 현재 대상 표시
  - 아무도 선택 안 하면: 전체
  - 특정 사람 선택 시: OOO 선생님에게 보내기
- 필터 탭 제공

3-2. 상단 필터
필터는 아래 3개를 먼저 구현
- 전체
- 사진
- 회원이전

필터 동작:
- 전체: 모든 메시지 표시
- 사진: 사진 첨부 메시지만 표시
- 회원이전: 회원 보내기/승인/완료 관련 카드 메시지만 표시

3-3. 중간
- 메시지 리스트
- 시간순 정렬
- 최신 메시지가 아래에 쌓이는 일반 메신저 방식
- 스크롤 가능

3-4. 하단
- 입력창
- 사진 첨부 버튼
- 보내기 버튼

--------------------------------------------------
[4. 대상 선택 기능]

- 관리자와 선생님 목록을 선택할 수 있게 구현
- 아무도 선택 안 하면 전체 메시지
- 특정 사람 선택하면 그 대상에게 보내는 메시지

단,
특정 사람을 선택해도 그 메시지는 전체 메신저 안에서 모두에게 보이게 구현한다.
즉 숨김 메시지가 아니라 "대상 지정 공개 메시지" 구조다.

대상 선택 해제 기능도 있어야 한다.
선택 해제하면 다시 전체 메시지로 전환된다.

--------------------------------------------------
[5. 사진 첨부 기능]

- 메신저에서는 사진만 첨부 가능
- 영상 첨부 금지
- 사진을 전송하면 메신저 대화에도 보이고, 메신저 앨범에도 자동 저장되게 구현
- 메신저 앨범에서 사진 다시 확인 가능
- 다운로드 가능

메신저 앨범은 관리자/선생님이 공용으로 보는 업무용 사진 저장소다.

--------------------------------------------------
[6. 회원 이전 기능]

이 메신저 안에서 회원 이전 기능을 구현한다.
이건 일반 텍스트가 아니라 카드형 메시지로 보여야 한다.

6-1. 보내는 쪽 로직
기존 담당 선생님이 회원 정보에서 "회원 보내기"를 누르면,
그 시점에서 보내는 선생님은 권한 이전을 승인한 것으로 본다.

즉 보내는 선생님은 별도 추가 승인 없음.

6-2. 받는 쪽 로직
받는 선생님이 메신저에서 해당 회원 이전 카드 보고 "승인"을 누르면
그 시점에 회원 이전 합의 완료로 처리한다.

즉 구조:
- 보내는 선생님: 회원 보내기 = 승인 완료
- 받는 선생님: 승인 = 최종 이전 완료

관리자 최종 승인 단계는 일단 없음.

--------------------------------------------------
[7. 회원 이전 시 실제 이동 데이터]

중요:
회원 이전 시 과거 데이터는 절대 넘어가면 안 된다.

넘어가지 않는 것:
- 기존 출결 기록
- 기존 수업피드백/일지
- 기존 사진/기록
- 과거 담당 이력

위 데이터는 기존 선생님 쪽에 그대로 남아 있어야 한다.

넘어가는 것:
- 앞으로 수업할 권한
- 주 몇 회 수업 정보
- 남은 보강 횟수

즉 현재 운영에 필요한 최소 권한만 이동시킨다.

회원 이전 승인 완료 시:
- 기존 선생님 시간표에서 해당 회원 제거
- 새 선생님은 자동 반배정 금지
- 새 선생님이 직접 새 반배정 하게 만들 것
- 이후부터의 출결/수업/보강만 새 선생님이 담당

--------------------------------------------------
[8. 회원 이전 카드 구조]

메신저 안에서 회원 이전 메시지는 카드형으로 표시한다.

카드에 표시할 정보:
- 학생 이름
- 보내는 선생님
- 받는 선생님
- 주 몇 회
- 남은 보강
- 상태
- 승인 버튼

상태 단계:
- 대기
- 완료
- 거절 (거절 기능은 구조만 열어두고 버튼 구현 여부는 선택적으로 보고)

회원 이전 카드도 전체 메신저 안에 기록으로 남아야 한다.

--------------------------------------------------
[9. 관리자 권한]

수영장 관리자는 이 메신저를 사용할 수 있다.
관리자는:
- 전체 메시지 보기
- 전체 메시지 보내기
- 특정 대상 메시지 보내기
- 사진 보기/다운로드
- 회원 이전 메시지 보기
- 회원 이전 흐름 확인

슈퍼관리자는 이 메신저에 들어가지 않는다.
진입 경로도 만들지 말 것.

--------------------------------------------------
[10. 저장/연동 원칙]

반드시 실제로 저장되고 서로 연동되어야 한다.

필수 연동:
- 선생님이 보낸 메시지 → 다른 선생님/관리자 메신저에 즉시 반영
- 관리자가 보낸 메시지 → 선생님 메신저에 즉시 반영
- 특정 대상 메시지 → 전체 메신저 안에 표시되되 대상 정보 포함
- 사진 전송 → 메신저 대화 + 메신저 앨범에 동시에 저장
- 회원 보내기 → 회원 이전 카드 생성
- 받는 선생님 승인 → 회원 권한 이동 반영
- 권한 이동 후 기존 시간표에서 제거
- 새 선생님 쪽에서 반배정 필요 상태로 표시

가짜 UI 금지.
버튼 눌렀을 때 실제 상태 변화가 저장되어야 한다.

--------------------------------------------------
[11. UI/UX 원칙]

- 모바일에서 카카오톡처럼 익숙한 배치 참고
- 하지만 기능은 업무용으로 단순하게 유지
- 읽음 표시, 답글 스레드, 이모티콘, 음성메시지, 영상 첨부는 넣지 말 것
- 메시지 입력/전송이 빠르게 되게 구현
- 긴 메시지 스크롤 자연스럽게 처리
- 상단 필터 눌렀을 때 즉시 목록 반영
- 대상 선택 시 현재 대상이 명확히 보이게
- 대상 해제도 쉽게 되게

--------------------------------------------------
[12. 로그 기록]

회원 이전은 별도 로그로도 남겨라.

최소 기록 항목:
- 누가 보냈는지
- 누구에게 보냈는지
- 어떤 회원인지
- 언제 보냈는지
- 언제 승인됐는지
- 상태(대기/완료/거절)

이건 관리자 기록 확인용으로 쓸 수 있게 저장할 것.

--------------------------------------------------
[13. 테스트 - 반드시 실제로 수행]

수정 후 아래 테스트를 반드시 직접 수행하고 결과를 상세히 보고해라.

테스트 1. 선생님 → 전체 메시지
- 선생님 계정 로그인
- 전체 메시지 전송
- 다른 선생님/관리자 계정에서 동일 메시지가 보이는지 확인

테스트 2. 선생님 → 특정 선생님 메시지
- 선생님 A가 선생님 B 선택 후 메시지 전송
- 메신저 안에서 "보낸사람 → 받는사람" 구조로 표시되는지 확인
- 관리자/다른 선생님 화면에서도 해당 메시지가 전체 메신저에 보이는지 확인

테스트 3. 관리자 → 전체 메시지
- 관리자 계정에서 전체 메시지 전송
- 선생님 계정들에서 보이는지 확인

테스트 4. 사진 첨부
- 선생님이 사진 첨부 메시지 전송
- 메신저에 사진 표시되는지 확인
- 메신저 앨범에도 저장되는지 확인
- 다운로드 되는지 확인

테스트 5. 회원 이전
- 선생님 A가 회원 보내기 실행
- 메신저에 회원 이전 카드 생성되는지 확인
- 선생님 B가 승인
- 주 몇 회/남은 보강만 이동하는지 확인
- 기존 출결/일지는 이동하지 않는지 확인
- 기존 시간표에서 제거되는지 확인
- 새 선생님은 반배정이 필요한 상태가 되는지 확인

테스트 6. 필터
- 전체 / 사진 / 회원이전 필터 각각 클릭
- 목록이 정상적으로 필터링되는지 확인

테스트 7. 반복 테스트
- 관리자↔선생님, 선생님↔선생님 메시지를 여러 번 주고받아
- 상태 꼬임 없이 계속 저장/표시되는지 확인

--------------------------------------------------

3. 특정 대상 메시지 표시 방식

5. 회원 이전 카드 동작 방식
6. 승인 후 실제 이동 데이터 범위
7. 테스트 1~7 결과
8. 남은 문제/불안정한 부분
9. 임시 처리한 부분 있으면 숨기지 말고 적기

[실패 기준]

아래 중 하나라도 해당하면 실패로 보고해라.

- 메시지가 계정 간 실제 전달되지 않음
- 특정 대상 메시지가 전체 메신저에 안 보임
- 사진이 메신저 앨범에 저장되지 않음
- 회원 이전 승인 후 과거 출결/일지가 같이 넘어감
- 기존 시간표 제거가 안 됨
- 새 선생님 반배정 필요 상태가 안 생김
- 필터가 가짜로만 보이고 실제 동작 안 함

이 작업은 반드시 실제 동작까지 구현하고 테스트 보고까지 완료해라.
