[학부모 회원가입 + 수영장 연결 + 학생 자동연결 시스템 — 기존 로직 폐기 예정, V2 구조로 전면 재구축]

현재 상태:
학부모 모드에서 관리자 모드의 학생 정보와 연결되는 핵심 기능이 완전히 깨져 있으며,
기존 승인/대기/수동 승인 구조를 단순화하는 과정에서
학부모 연결 로직 전체가 꼬인 상태다.

이제부터는 기존 로직에 덧패치하지 말고,
학부모 회원가입부터 자동연결까지를 V2 구조로 새로 재구축한다.

중요:
- 앱 전체를 새로 만드는 것이 아니다
- DB 전체를 갈아엎는 것도 아니다
- “학부모 온보딩 + 수영장 연결 + 학생 자동연결” 경로만 새로 만든다
- 기존 로직은 바로 삭제하지 말고 비활성 상태로 유지 후, V2 검증 완료 후 제거

━━━━━━━━━━━━━━━━━━
1. 핵심 목표 구조
━━━━━━━━━━━━━━━━━━

학부모 흐름:

1) 회원가입
2) 수영장 검색 및 선택 (1개만 소속)
3) 전화번호 입력/인증
4) 학생 이름 입력
5) 즉시 자동연결 시도

결과:

A. 관리자에 학생 정보 있음
→ 학생 이름 + 전화번호 일치
→ 즉시 자동 연결 완료
→ 학부모 홈 활성화

B. 관리자에 학생 정보 없음
→ 수영장 연결만 완료
→ 학생 연결 “대기 상태”
→ “학생 정보 등록되면 자동 연결” 안내 표시

관리자 흐름:

6) 학생 등록 (이름 + 전화번호)
7) 기존 대기 중 학부모 데이터와 비교
→ 일치 시 자동 연결

핵심:
“학부모 먼저 가입 가능”
“관리자 나중 등록 가능”
“조건 맞는 순간 자동 연결”

━━━━━━━━━━━━━━━━━━
2. 절대 원칙
━━━━━━━━━━━━━━━━━━

- 기존 로직 위에 덧패치 금지
- 승인 버튼 / pending / request / approved 흐름 제거 (V2에서는 사용 금지)
- UI 수정으로 해결 금지
- 반드시 서버 중심 자동연결 구조로 구현
- 기존 코드 삭제 금지 (V2 완성 전까지 유지)
- 성공/실패를 로그로 명확히 구분

━━━━━━━━━━━━━━━━━━
3. 용어 통일 (중요)
━━━━━━━━━━━━━━━━━━

혼용 금지:

- 내부(DB/API/코드): student로 통일
- 학부모 화면: “우리 아이”
- 관리자 화면: “학생”

child, kid, parentChild 등 혼용된 용어 전수 조사 후 정리

━━━━━━━━━━━━━━━━━━
4. 상태 구조 (핵심)
━━━━━━━━━━━━━━━━━━

화면은 한 번에 입력해도,
DB 상태는 반드시 아래 3단계로 분리:

1) 수영장 연결 완료
2) 학생 연결 대기
3) 학생 연결 완료

이 분리 없으면 다시 같은 버그 발생함

━━━━━━━━━━━━━━━━━━
5. 정규화 규칙 (무조건 적용)
━━━━━━━━━━━━━━━━━━

전화번호:
- 숫자만 유지
- 하이픈 제거
- 공백 제거

이름:
- trim
- 공백 정리

관리자 등록 / 학부모 입력 모두 동일 적용

━━━━━━━━━━━━━━━━━━
6. V2 자동연결 로직 (새로 구현)
━━━━━━━━━━━━━━━━━━

새 함수로 반드시 분리:

- normalizeStudentName()
- normalizeParentPhone()
- createParentOnboardingData()
- tryMatchStudent()
- linkParentToStudentV2()
- getParentStudentsV2()

기존 함수 수정 금지, 새 경로 생성

━━━━━━━━━━━━━━━━━━
7. 자동연결 트리거 (3중 구조)
━━━━━━━━━━━━━━━━━━

반드시 아래 3개 모두 구현:

1) 학부모 가입 직후 → 즉시 매칭
2) 관리자 학생 등록 직후 → 즉시 매칭
3) 학부모 앱 진입/홈 진입 시 → 재매칭

추가:
- “다시 확인” 버튼 제공

중요:
학부모가 계속 입력하는 구조 금지
→ 시스템이 계속 확인해야 한다

━━━━━━━━━━━━━━━━━━
8. 학부모 화면 상태
━━━━━━━━━━━━━━━━━━

상태 1: 수영장 미연결
상태 2: 학생 입력 전
상태 3: 연결 대기
→ “아직 수영장에 학생 정보가 등록되지 않았습니다”
→ “등록되면 자동 연결됩니다”
→ “다시 확인” 버튼

상태 4: 연결 완료
→ 정상 홈

중요:
가짜 성공 상태 절대 금지

━━━━━━━━━━━━━━━━━━
9. 관리자 등록 시 동작
━━━━━━━━━━━━━━━━━━

학생 등록 시:

- 이름/전화번호 정규화
- 같은 수영장 대기 데이터 조회
- 일치 시 자동 연결

관리자는 승인 버튼 없음
→ 입력만 하면 됨

━━━━━━━━━━━━━━━━━━
10. 기존 로직 처리 방식
━━━━━━━━━━━━━━━━━━

- 기존 연결 로직은 삭제 금지
- V2 경로에서 절대 타지 않게 차단
- V2 완성 후 제거 예정

━━━━━━━━━━━━━━━━━━
11. 필수 테스트
━━━━━━━━━━━━━━━━━━

A. 학부모 먼저 → 관리자 나중
→ 자동 연결 성공

B. 관리자 먼저 → 학부모 나중
→ 즉시 연결 성공

C. 이름 불일치 → 실패

D. 전화번호 불일치 → 실패

E. 중복 연결 → 방지

━━━━━━━━━━━━━━━━━━
12. 필수 로그
━━━━━━━━━━━━━━━━━━

- 입력값 (raw + normalized)
- pool_id
- 조회 결과 수
- 매칭된 student id
- 연결 성공/실패
- 홈 조회 결과
- 에러 원문

“됐다” 금지
