지금부터는 기능 추가가 아니라 앱 전체 안정성 점검/수정 작업으로 진행해줘.
특히 iPhone에서 선생님모드 진입 시 뒤로가기가 안 되고 앱이 멈추는 문제를 최우선으로 해결해줘.

이번 작업 목표:
1. 모든 화면에서 뒤로가기 / 홈버튼이 정상 작동
2. 모든 팝업이 빈 화면 터치 / X 버튼으로 정상 닫힘
3. 쪽지 / 메신저 / 입력창에서 키보드가 올라와도 입력영역을 가리지 않음
4. iPhone에서 화면 멈춤 / 뒤로가기 불가 / 터치 막힘 같은 현상 제거
5. 단순 수정했다고 말하지 말고 실제로 전수 점검 후 검증 결과까지 보고

==================================================
[1. 최우선 버그: 선생님모드 iPhone 뒤로가기 먹통]
==================================================

문제:
- 선생님모드에서 iPhone으로 실행하면 뒤로가기가 안 됨
- 특정 화면 진입 후 앱이 멈춘 것처럼 보임
- 이전 화면으로 복귀가 안 되거나 터치가 막힘

해결 요구:
1. 선생님모드 전체 라우트/내비게이션 구조를 점검
2. iPhone Safari / Expo Go 환경에서 뒤로가기 이벤트가 막히는 구간 확인
3. modal, overlay, absolute layer, pointerEvents, zIndex, gesture, scroll lock 때문에 터치/뒤로가기가 막히는 곳 찾기
4. 뒤로가기 호출 시 실제 이전 스택이 없으면 안전하게 선생님 홈으로 보내는 fallback 추가
5. 화면 전환 후 로딩/오버레이가 남아서 앱이 멈춘 것처럼 보이는 현상 제거

필수 fallback 규칙:
- canGoBack() 가능하면 뒤로가기
- 불가능하면 해당 역할의 홈으로 이동
- 선생님모드에서는 fallback 홈 = 선생님 홈

==================================================
[2. 전체 화면 전수 점검 항목]
==================================================

앱 전체 모든 화면에 대해 아래 항목을 점검하고 수정해줘.

점검 대상:
- 학부모모드 전 화면
- 선생님모드 전 화면
- 관리자모드 전 화면
- 슈퍼관리자모드 전 화면(현재 구현된 범위만)

점검 항목:
1. 뒤로가기 버튼 작동 여부
2. 홈 버튼 작동 여부
3. 버튼 눌렀을 때 올바른 역할 홈으로 가는지
4. 경로 꼬임 없이 이전 화면 또는 홈으로 이동하는지
5. 누른 뒤 반응 없거나 멈춤 현상 없는지

==================================================
[3. 뒤로가기 / 홈버튼 공통 규칙 통일]
==================================================

모든 화면에 아래 규칙을 공통 적용해줘.

[뒤로가기]
1. 이전 화면이 있으면 이전 화면으로 이동
2. 이전 화면이 없으면 역할별 홈으로 fallback 이동
- 학부모 -> 학부모 홈
- 선생님 -> 선생님 홈
- 관리자 -> 관리자 홈
- 슈퍼관리자 -> 슈퍼관리자 홈

[홈버튼]
1. 항상 현재 역할의 홈으로 이동
2. 다른 역할 홈으로 섞여 들어가면 안 됨
3. 관리자 홈 버튼이 선생님 홈이나 학부모 홈으로 가면 안 됨
4. 학부모 홈 버튼이 관리자 페이지로 가면 안 됨

==================================================
[4. 팝업 공통 닫기 규칙]
==================================================

앱 전체의 모든 팝업 / 모달 / 바텀시트 / 오버레이에 대해 아래 규칙 적용해줘.

1. 팝업 바깥 빈 화면 클릭 시 닫힘
2. X 버튼 클릭 시 닫힘
3. 닫힌 뒤 배경 화면 터치가 정상 복구되어야 함
4. 팝업 닫은 후 invisible overlay가 남아서 화면이 안 눌리는 현상 없어야 함
5. 팝업 여러 개 열릴 경우 최상단 팝업만 닫히도록 처리
6. 팝업 닫을 때 상태값이 완전히 false로 정리되게 해줘
7. modal backdrop가 남지 않게 정리

특히 점검할 것:
- 스케쥴러 날짜 팝업
- 일지 상세 팝업
- 삭제 확인 팝업
- 이미지 상세 보기
- 선택 삭제 팝업
- 계정 전환 팝업
- 설정 팝업
- 알림/쪽지 관련 팝업

==================================================
[5. 키보드 올라올 때 입력창 가림 문제 전수 수정]
==================================================

문제:
- 쪽지 / 메신저 / 채팅 / 입력창 화면에서 키보드가 올라오면
  입력하는 글자 영역 또는 입력창 자체를 가림
- iPhone에서 특히 심함

해결 요구:
1. 키보드 올라오면 입력창이 키보드 위로 자동 고정되게 해줘
2. 입력 중인 텍스트 영역도 가능한 범위에서 같이 보이게 스크롤 조정
3. 화면마다 KeyboardAvoidingView / keyboard-aware scroll / safe area 대응 적용
4. iPhone과 Android 모두 대응
5. 입력창이 absolute/fixed면 키보드 높이만큼 위로 올려줘
6. 키보드 닫히면 원위치 복귀
7. send 버튼 / 저장 버튼이 키보드에 가려지지 않게 해줘

반드시 점검할 입력 화면:
- 학부모 쪽지 작성
- 학부모 문의 상세
- 선생님 메신저
- 관리자 메신저
- 일지 작성 입력칸
- 날짜 메모 입력칸
- 설정/프로필 텍스트 입력칸
- 로그인/회원가입 입력칸
- 비밀번호 찾기 입력칸

==================================================
[6. 구현 방식 요구]
==================================================

단순히 한두 화면만 고치지 말고 공통 컴포넌트/패턴으로 정리해줘.

가능하면 아래처럼 정리:
1. 공통 Header 컴포넌트
- 뒤로가기
- 홈버튼
- 역할별 fallback home 처리

2. 공통 Modal/Popup 컴포넌트
- backdrop click 닫기
- X 버튼 닫기
- overlay cleanup

3. 공통 Keyboard-safe 입력 레이아웃
- iOS/Android 분기 포함
- safe area + keyboard offset 처리

4. 라우트 fallback 유틸
- goBackSafe(role)
- goHome(role)

[7. 강력한 검증 요구]
==================================================

이번 작업은 절대 "수정했습니다"로 끝내지 말 것.
실제로 직접 눌러서 검증하고, 검증 결과를 화면/기능별로 보고해줘.

검증 방식:
1. 각 역할 홈에서 주요 하위 페이지 진입
2. 뒤로가기 눌러 복귀 확인
3. 홈버튼 눌러 역할 홈 이동 확인
4. 팝업 열기
5. 빈 화면 클릭 닫기 확인
6. X 버튼 닫기 확인
7. 팝업 닫은 뒤 배경 터치 정상 확인
8. 쪽지/메신저 열기
9. 키보드 올리기
10. 입력창이 키보드 위로 올라오는지 확인
11. 텍스트 입력 가능한지 확인
12. 전송/저장 버튼 눌리는지 확인

특히 iPhone 시뮬레이터/Preview 기준으로 검증 강하게 해줘.

[10. 최종 목표]
==================================================

이 앱은 MVP 완성 직전 단계다.
지금은 기능 추가보다 안정성이 우선이다.

이번 작업 최종 목표는 아래 4개다.
- iPhone에서도 선생님모드 뒤로가기 정상
- 전체 화면 뒤로가기/홈버튼 정상
- 모든 팝업 닫기 정상
- 모든 입력 화면 키보드 대응 정상

강하게 전수 점검하고, 실제 작동 확인까지 끝낸 뒤 보고해줘.
