[스윔노트 전체 네비게이션 규칙 통일 + 뒤로가기/팝업 닫기/저장 후 이동 로직 정리 지시서]

지금부터 앱 전체 화면 전환 규칙을 통일한다.
이번 작업의 목적은 디자인 수정이 아니라
“뒤로가기 / 닫기 / 저장 후 이동 / 홈 이동” 규칙을 전 페이지에서 일관되게 맞추는 것이다.

중요:
- 현재 일부 화면에서 팝업을 닫거나 저장 후 홈 화면 또는 아이콘 첫 화면으로 튀는 문제가 있다
- 이건 전부 수정해야 한다
- 홈 이동은 오직 사용자가 명시적으로 홈을 눌렀을 때만 발생해야 한다
- 그 외 모든 닫기/뒤로가기/저장하기는 기본적으로 직전 화면 복귀로 통일한다
- 이번 작업은 전모드(슈퍼관리자/수영장관리자/선생님/학부모) 전체에 적용한다
- 기능 로직은 건드리지 말고 네비게이션/전환 규칙만 바로잡아라
- 수정 후 반드시 각 흐름을 실제로 눌러서 테스트하고 보고하라

==================================================
1. 핵심 네비게이션 원칙
==================================================

앱 전체에서 아래 규칙을 강제한다.

1. 홈으로 이동
- 홈 버튼을 명시적으로 눌렀을 때만 가능
- 홈 화면 강제 점프 금지
- 저장하기/닫기/뒤로가기로 홈 이동 금지

2. 뒤로가기
- 무조건 직전 화면으로 이동
- 아이콘 첫 화면으로 강제 이동 금지
- 홈으로 강제 이동 금지

3. 팝업 닫기
- 팝업을 열기 직전 화면으로 복귀
- 아이콘 첫 화면 이동 금지
- 홈 이동 금지

4. 저장 후 이동
- 저장 성공 시 기본값은 “직전 화면 복귀”
- 저장 후 홈 이동 금지
- 저장 후 아이콘 첫 화면 이동 금지
- 저장 결과는 직전 화면에 반영

==================================================
2. 적용 범위
==================================================

다음 전 화면/전 모드에 동일 적용:

- 슈퍼관리자
- 수영장 관리자
- 선생님
- 학부모

적용 대상:
- 페이지 이동
- 팝업 열기/닫기
- 모달
- 상세 화면
- 수정 화면
- 저장/확인 후 이동
- 뒤로가기 버튼
- 헤더 뒤로가기
- 하드웨어/제스처 뒤로가기와 연결된 동작(가능 범위)

==================================================
3. 전환 규칙 상세
==================================================

--------------------------------------------------
3-1. 아이콘 메인 화면 규칙
--------------------------------------------------

아이콘이 모여 있는 화면(각 모드의 메인 허브 화면)은
“해당 모드의 홈” 역할이다.

규칙:
- 아이콘 화면 안으로 들어간 뒤에는 그 내부 이동 스택을 유지해야 한다
- 내부 화면에서 뒤로가기를 누르면 직전 화면으로 차례대로 나와야 한다
- 아이콘 화면 안의 첫 페이지로 강제 리셋 금지

예:
아이콘 메인 > 시간표 > 시간상세 > 반배정 팝업
- 팝업 닫기 → 시간상세
- 뒤로가기 → 시간표
- 뒤로가기 → 아이콘 메인

--------------------------------------------------
3-2. 팝업/모달 규칙
--------------------------------------------------

팝업은 독립 홈이 아니다.
팝업은 직전 화면 위에 잠깐 올라온 레이어다.

규칙:
- 팝업 열기 전의 화면 상태를 유지
- 팝업 닫기 = 직전 화면 복귀
- 팝업 저장 = 저장 후 직전 화면 복귀
- 팝업 확인 = 팝업만 닫기
- 팝업 닫기 후 홈 이동 금지
- 팝업 닫기 후 아이콘 첫 화면 이동 금지

--------------------------------------------------
3-3. 상세/수정 화면 규칙
--------------------------------------------------

상세 화면에서 수정 화면으로 들어가면
수정 저장 후 기본적으로 상세 화면으로 돌아와야 한다.

예:
회원목록 > 학생상세 > 학생정보수정
- 저장 후 → 학생상세
- 뒤로가기 → 회원목록

예:
시간표 > 수업상세 > 일지작성 팝업
- 저장 후 → 수업상세 또는 시간표의 직전 맥락 유지
- 홈 이동 금지

--------------------------------------------------
3-4. 저장 버튼 규칙
--------------------------------------------------

저장 버튼은 네비게이션 초기화 버튼이 아니다.

규칙:
- 저장 성공 → 현재 편집 종료 + 직전 화면 복귀
- 홈 이동 금지
- 아이콘 첫 화면 이동 금지
- 필요 시 저장 완료 토스트/알림만 표시
- 저장 결과는 직전 화면에 즉시 반영

--------------------------------------------------
3-5. 확인 버튼 규칙
--------------------------------------------------

확인 버튼도 저장과 같은 원칙을 따른다.

규칙:
- 확인 = 현재 팝업/현재 단계 종료
- 직전 화면 유지
- 홈 이동 금지

==================================================
4. 홈 이동 허용 예외
==================================================

아래 경우만 홈 이동 허용:

- 사용자가 홈 버튼을 직접 클릭
- 로그아웃 후 로그인 첫 화면 복귀
- 수영장 전환 후 새 수영장 진입
- 앱 구조상 강제 재진입이 필요한 특수 상황

중요:
이 예외 외에는 어떤 저장/닫기/확인/뒤로가기에서도 홈 이동하면 안 된다.

==================================================
5. 수정 방법
==================================================

다음 항목을 전수 점검하고 수정:

1. router.push / router.replace / router.back 사용 위치 전수 점검
2. 팝업/모달 닫기 로직 점검
3. 저장 성공 후 navigation 처리 점검
4. 아이콘 내부 첫 화면으로 리셋시키는 잘못된 replace/navigation 초기화 제거
5. 직전 화면 상태를 유지하도록 수정
6. 가능한 경우 stack navigation 철학에 맞게 back 사용 우선
7. 정말 홈 복귀가 필요한 경우만 명시적으로 처리

중요:
편의상 replace로 메인 화면 보내는 코드가 있으면 찾아서 수정하라.

==================================================
6. 특히 우선 점검할 흐름
==================================================

우선 아래 흐름부터 집중 점검:

- 시간표/스케줄 관련 팝업
- 반배정 관련 팝업
- 일지 작성/수정 팝업
- 학생 상세/수정 화면
- 보강 정책/레벨 설정/운영설정 화면
- 공지/푸시 설정 화면
- 학부모 자녀 상세/탭 전환 관련 화면
- 관리자/선생님/학부모 프로필 설정 화면

==================================================
7. 테스트 방식
==================================================

수정 후 반드시 실제 클릭 테스트를 한다.

각 모드별로 아래를 반복:

1. 아이콘 메인 화면 진입
2. 하위 화면 진입
3. 상세 화면 진입
4. 팝업 열기
5. 팝업 닫기
6. 저장 후 복귀
7. 뒤로가기 반복
8. 홈 버튼 클릭 시에만 홈 이동 확인

테스트 시 반드시 확인:
- 직전 화면으로 복귀하는지
- 첫 화면으로 튀지 않는지
- 홈으로 튀지 않는지
- 저장 결과가 사라지지 않는지
- 내부 스택이 자연스럽게 유지되는지

==================================================
8. 수정 금지 영역
==================================================

이번 작업에서 절대 깨지면 안 되는 것:
- 실제 저장 로직
- 실제 CRUD 기능
- 권한 로직
- 데이터 구조
- 푸시/로그/결제/보강/레벨 기능 자체

이번 작업은 네비게이션 규칙 통일이지
기능 자체를 다시 만드는 작업이 아니다.

==================================================
9. 완료 보고 형식
==================================================

반드시 아래 형식으로 상세 보고하라.

1. 잘못된 이동 규칙이 있던 화면 목록
2. 홈으로 잘못 튀던 원인
3. 직전 화면 복귀로 수정한 방식
4. 수정한 파일 목록
5. 각 모드별 테스트 결과
6. 팝업 닫기/저장 후 복귀 테스트 결과
7. 아직 남은 예외 화면이 있는지 여부

중요:
“전체 수정 완료” 한 줄 보고 금지
반드시 실제 눌러본 흐름 기준으로 적어라

==================================================
10. 최종 목적
==================================================

이번 작업의 최종 목적은
앱 전체에서
- 뒤로가기
- 팝업 닫기
- 저장 후 이동
- 홈 이동
규칙을 완전히 통일하는 것이다.

핵심:
- 홈은 홈 버튼으로만 간다
- 나머지는 전부 직전 화면 복귀
- 팝업은 레이어일 뿐, 화면 스택을 초기화하지 않는다
- 저장은 홈 이동이 아니라 편집 종료다

이 기준으로 전 페이지를 꼼꼼하게 정리하라.
