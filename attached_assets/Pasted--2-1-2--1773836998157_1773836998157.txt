지금부터 관리자 모드에 아래 2가지 기능을 함께 추가하라.

1. 저장공간 관리 화면 아래 “킬 스위치(영구 삭제 도구)”
2. 관리자 더보기 안 “이벤트 기록” 타임라인

이번 작업 목표는
관리자가 저장공간을 직접 관리할 수 있게 하고,
주요 운영 행위를 이벤트 타임라인 로그로 남길 수 있게 만드는 것이다.

가장 중요:
- 원본 데이터는 삭제 시 실제로 영구 삭제
- 복구 기능 없음
- 대신 로그만 남김
- 기능 연결을 실제로 확인하면서 작업할 것
- 작업 후 반드시 실제 테스트까지 수행하고 상세 보고할 것

==================================================
[1. 이번 작업 최종 목표]

추가할 기능:

A. 관리자 > 더보기 > 저장공간 관리
- 저장공간 총합/선생님별 사용량 아래에
  “영구 삭제 도구(킬 스위치)” 추가

B. 관리자 > 더보기
- “이벤트 기록” 메뉴 추가
- 이벤트 기록 화면은 타임라인 형식으로 표시

==================================================
[2. 저장공간 관리 화면 내 킬 스위치]

위치:
관리자 > 더보기 > 저장공간 관리 화면 안에서
현재 보이는 데이터 표시 영역 아래쪽에 새 섹션으로 추가

섹션명:
- 영구 삭제 도구
또는
- 킬 스위치

설명 문구:
선택한 기간보다 오래된 데이터를 영구 삭제합니다.
삭제 후 복구할 수 없습니다.

==================================================
[3. 킬 스위치 조건 선택 UI]

--------------------------------------
[3-1. 기간 선택]

둘 다 제공:
A. 빠른 선택 버튼
- 3개월 이상
- 6개월 이상
- 1년 이상

B. 직접 입력
- 직접 개월 수 입력칸
- 예: 9개월 이상

규칙:
- 빠른 선택 또는 직접 입력 둘 다 가능
- 관리자가 원하는 방식으로 선택 가능

--------------------------------------
[3-2. 데이터 종류 선택]

체크박스/토글로 독립 선택 가능하게 한다.

선택 항목:
- 사진
- 영상
- 수업 기록

설명:
- 사진 = 앨범/사진 데이터
- 영상 = 영상 데이터
- 수업 기록 = 수영일지/수업기록류 데이터 묶음

중요:
- 사진만 삭제 가능
- 영상만 삭제 가능
- 수업기록만 삭제 가능
- 사진+영상
- 전체 선택
모두 가능해야 함

==================================================
[4. 삭제 대상 미리보기]

관리자가 조건을 선택하면
삭제 전에 아래 정보를 미리 계산해서 보여줘야 한다.

표시 항목:
- 삭제 대상 개수
- 예상 절감 용량

예:
- 사진 182개
- 영상 14개
- 수업기록 326개
- 예상 절감 2.8GB

중요:
삭제 실행 전에 예상 결과를 먼저 보여줘야 함

==================================================
[5. 1차 경고 팝업]

삭제 실행 버튼 클릭 시
바로 삭제하지 말고 1차 경고 팝업 표시

팝업 문구 예시:
선택한 조건의 데이터는 영구 삭제되며 복구할 수 없습니다.
계속 진행하시겠습니까?

버튼:
- 취소
- 계속

중요:
이 단계에서는 아직 삭제 실행 금지

==================================================
[6. 2차 인증 - 관리자 비밀번호 재입력]

1차 경고 통과 후
반드시 관리자 비밀번호 재입력 단계 추가

구성:
- 제목: 관리자 인증
- 설명: 영구 삭제를 진행하려면 비밀번호를 입력하세요
- 비밀번호 입력창
- 버튼: 취소 / 영구 삭제 실행

로직:
1. 입력한 비밀번호와 현재 로그인한 관리자 계정 비밀번호 비교
2. 일치할 때만 삭제 실행 가능
3. 불일치 시:
   - “비밀번호가 일치하지 않습니다” 오류 표시
   - 삭제 실행 차단

중요:
- 비밀번호 인증 없이 영구 삭제 실행 금지
- 입력 비밀번호 저장 금지
- 검증 후 즉시 폐기

==================================================
[7. 삭제 실행 규칙]

비밀번호 인증 성공 시
선택한 조건에 맞는 데이터를 즉시 영구 삭제한다.

원칙:
- 휴지통 없음
- 복구 기능 없음
- 되돌리기 없음

즉:
킬 스위치 실행 시 원본 데이터는 완전히 지워진다

==================================================
[8. 킬 스위치 실행 후 남길 로그]

원본 데이터는 지우되,
반드시 이벤트 로그만 남긴다.

삭제 로그에 남길 항목:
- 실행 관리자 이름/아이디
- 실행 시간
- 기간 조건
- 삭제 데이터 종류
- 삭제 개수
- 절감된 용량

예:
2026-03-18 14:22
관리자 김원장
6개월 이상
사진 + 영상
214개 삭제
3.4GB 절감

==================================================
[9. 이벤트 기록 메뉴 추가]

관리자 > 더보기 안에
새 메뉴 “이벤트 기록” 추가

이벤트 기록은 타임라인 형식으로 표시한다.

구성:
- 상단 제목: 이벤트 기록
- 필터/카테고리 선택 가능
- 최신순 타임라인 리스트

==================================================
[10. 이벤트 기록 카테고리]

아래 카테고리를 로그로 남기고 타임라인에서 볼 수 있게 한다.

필수 카테고리:
1. 삭제 기록
2. 결제 기록
3. 구독 기록
4. 해지 기록
5. 권한 기록
6. 선생님 기록

추천 추가 카테고리:
7. 저장공간 기록
8. 휴무일 기록

==================================================
[11. 각 카테고리 로그 상세]

--------------------------------------
[11-1. 삭제 기록]
킬 스위치 실행 로그

항목:
- 실행 관리자
- 시간
- 기간 조건
- 데이터 종류
- 삭제 개수
- 절감 용량

--------------------------------------
[11-2. 결제 기록]

항목:
- 결제 일시
- 결제 관리자
- 결제 항목
- 결제 금액
- 결제 상태

예:
- 구독 연장
- 저장공간 추가
- 영상 기능 추가

--------------------------------------
[11-3. 구독 기록]

항목:
- 구독 시작
- 플랜 변경

- 변경 관리자

--------------------------------------
[11-4. 해지 기록]

항목:
- 구독 해지

--------------------------------------
[11-5. 권한 기록]

항목:

예:

[11-6. 선생님 기록]

항목:

- 부관리자 지정/해제
- 수업 인수
- 담당 변경

중요:
수업 인수 기록은 반드시 남겨야 함

예:
- 박선생 담당 회원 24명, 김선생에게 인수
- 반 3개 이전 완료

[11-7. 저장공간 기록] (추천)

항목:
- 용량 초과 경고

- 저장공간 증가/변경

--------------------------------------
[11-8. 휴무일 기록] (추천)

항목:
- 휴무일 지정
- 휴무일 해제
- 대상 월/날짜
- 실행 관리자

==================================================
[12. 이벤트 기록 화면 UI]

타임라인 카드 또는 리스트 1건당 표시할 것:
- 날짜/시간
- 이벤트 종류
- 대상
- 실행자
- 한 줄 설명

예:
2026.03.18 14:22
삭제 기록
관리자 김원장
6개월 이상 사진/영상 214개 삭제, 3.4GB 절감

필터 예시:
- 전체
- 삭제
- 결제
- 구독
- 권한
- 선생님
- 저장공간
- 휴무일

==================================================
[13. 로그 보존 규칙]

중요:
- 원본 데이터는 삭제해도 로그는 남김
- 로그는 이벤트 요약만 남기고 원본 콘텐츠는 남기지 않음
- 회원 세부 수정 이력은 이번 단계에서 굳이 남기지 않아도 됨
- 선생님 승인/해제/인수 기록은 반드시 남김

==================================================
[14. 기존 기능 보호]

중요:
- 저장공간 총합/게이지 기능 유지
- 선생님별 사용량 표시 유지
- 기존 결제/구독 구조 유지
- 기존 승인/선생님 관리 기능 유지
- 기존 로그가 있으면 통합 가능하되 기능 깨지지 않게 할 것
- 원본 데이터 삭제와 로그 저장은 분리 처리할 것

==================================================
[15. 기능 연결 체크 필수]

반드시 아래를 실제로 확인하면서 작업하라.

체크 항목:
1. 킬 스위치 조건 선택 후 삭제 대상 개수/절감 용량 계산되는지
2. 1차 경고 팝업이 뜨는지
3. 관리자 비밀번호 재입력 없이는 삭제가 안 되는지
4. 비밀번호 맞으면 실제 영구 삭제되는지
5. 삭제 후 저장공간 총합과 게이지가 줄어드는지
6. 삭제 후 이벤트 기록에 삭제 로그가 남는지
7. 부관리자 지정/해제 시 권한 기록이 남는지
8. 선생님 승인/해제/수업 인수 시 선생님 기록이 남는지
9. 결제/구독/해지 이벤트가 기록되는지
10. 이벤트 기록 타임라인에서 필터별로 보이는지

==================================================
[16. 테스트 시나리오]

반드시 실제 테스트하라.

시나리오 A.
관리자 > 저장공간 관리
→ 6개월 이상 + 사진 선택
→ 삭제 대상 미리보기 확인
→ 1차 경고
→ 비밀번호 입력
→ 영구 삭제 실행
→ 총 사용량 감소 확인
→ 이벤트 기록에 삭제 로그 확인

시나리오 B.
선생님 승인
→ 이벤트 기록에 선생님 승인 로그 남는지 확인

시나리오 C.
부관리자 지정
→ 이벤트 기록에 권한 기록 남는지 확인

시나리오 D.
수업 인수 실행
→ 이벤트 기록에 선생님 기록 남는지 확인

시나리오 E.
결제/구독/해지 관련 이벤트 발생
→ 이벤트 기록에 각각 남는지 확인

==================================================
[17. 작업 전 먼저 확인]

수정 전에 아래를 먼저 파악하라.

1. 관리자 저장공간 관리 화면 파일
2. 현재 총 사용량 계산 파일
3. 사진/영상/수업기록 저장 구조 파일
4. 관리자 비밀번호 검증 파일
5. 결제/구독/해지 관련 파일
6. 승인/권한 변경 관련 파일
7. 수업 인수 관련 파일
8. 더보기 화면 파일
9. 로그 저장 가능한 파일/구조

==================================================
[18. 작업 후 보고 형식]

작업 후 아래 순서로 보고하라.

1. 수정 파일 목록
2. 킬 스위치 배치 위치
3. 기간/데이터 종류 선택 구현 방식
4. 삭제 대상 미리보기 계산 방식
5. 관리자 비밀번호 인증 방식
6. 영구 삭제 실행 방식
7. 이벤트 기록 화면 구성 방식
8. 카테고리별 로그 저장 방식
9. 테스트 시나리오 A~E 결과
10. 아직 남은 후속 작업이 있으면 명시

==================================================
[19. 절대 금지]

- 경고 없이 바로 삭제 실행 금지
- 비밀번호 인증 없이 삭제 실행 금지
- 복구 가능한 삭제 구조로 변경 금지
- 원본 데이터까지 로그에 남기는 것 금지
- 화면만 만들고 실제 로그 저장 안 되는 상태 금지
- 삭제 후 저장공간 수치가 안 줄어드는 상태 금지

이번 작업은
관리자 저장공간 관리 아래에 복구 불가 킬 스위치를 추가하고,
관리자 더보기 안에 주요 운영 행위를 남기는 이벤트 기록 타임라인을 실제로 동작하게 만드는 작업이다.
