이번 작업은 선생님 모드 전체 구조 개편이다.
목표는 선생님이 실제 수업 운영에 필요한 기능을 한 화면 흐름 안에서 빠르게 처리할 수 있게 만드는 것이다.
단순 조회 화면이 아니라 선생님이 직접 수업, 출결, 보강, 일지, 사진/영상, 회원상태, 본인정보까지 관리할 수 있는 구조로 확장해야 한다.

[최종 방향]
기존 선생님 모드의 내반 / 오늘 스케줄 / 출결 같은 분리된 구조를 더 실무형으로 정리한다.

핵심 방향
- 첫 화면은 오늘 스케줄 중심 메인 화면으로 만든다
- 오늘 스케줄 밑에 빠른 이동 탭을 둔다
- 빠른 이동 탭에서 선생님이 자주 쓰는 메뉴를 즉시 이동 가능하게 한다
- 화면은 모두 실제 데이터와 연결되어야 한다
- 출결, 보강, 일지, 사진/영상, 회원상태, 관리설정이 서로 연동되어야 한다
- 관리자가 보는 데이터와 선생님이 보는 데이터는 원본을 공유하고 권한만 다르게 한다

────────────────────
[1. 선생님 모드 메인 구조 개편]
첫 화면은 오늘 스케줄 중심으로 재구성한다.

첫 페이지 메인 구성
- 상단: 오늘 스케줄
- 중단: 오늘 수업 리스트
- 하단: 빠른 이동 탭

빠른 이동 탭 메뉴
- 수업 관리
- 출결 관리
- 수영일지
- 사진영상
- 관리설정

기존의 내반 / 오늘 스케줄 / 출결 같은 메뉴를 분산시키지 말고
오늘 스케줄을 메인으로 두고
하단 빠른 이동 탭으로 각 기능으로 가게 한다

즉 개념상
- 오늘 스케줄 = 메인 대시보드
- 수업 관리 = 내반/주간/월간/회원 보기
- 출결 관리 = 출결 + 보강
- 수영일지 = 일지 작성/수정/조회
- 사진영상 = 업로드 + 사용량 확인
- 관리설정 = 선생님 정보/권한/회원현황/용량/퇴사요청 등

────────────────────
[2. 오늘 스케줄 화면]
오늘 스케줄은 선생님이 오늘 해야 할 일 중심으로 보여준다.

표시 항목
- 오늘 날짜
- 오늘 총 수업 수
- 오늘 예정 반 리스트
- 각 반의 시간
- 반 이름
- 학생 수
- 출결 상태 요약
- 일지 작성 여부
- 사진/영상 전송 여부
- 보강 학생 포함 여부

오늘 수업 각 행에서 바로 이동 가능한 액션
- 출결
- 일지
- 반정보 보기
- 사진/영상 보내기

중요
- 오늘 수업에서 출결 누르면 해당 시간 출석 체크 페이지로 바로 이동
- 오늘 수업에서 일지 누르면 해당 시간 일지 작성 페이지로 바로 이동
- 중간 목록 화면 없이 바로 실무 화면 진입

────────────────────
[3. 빠른 이동 탭 구조]
빠른 이동 탭은 아래 메뉴로 구성한다.

- 수업 관리
- 출결 관리
- 수영일지
- 사진영상
- 관리설정

모든 탭은 실제 이동 가능해야 하고 연결된 데이터가 동일해야 한다.

────────────────────
[4. 수업 관리 탭]
기존 내반/주간 구조를 확장해서 선생님의 수업 운영 전체를 관리하는 탭으로 만든다.

수업 관리 탭 안에 들어갈 수 있는 세부 메뉴 예시
- 오늘 스케줄
- 주간 스케줄
- 월간 스케줄
- 반 목록
- 회원 목록

필수 기능
- 내 반 주간 페이지
- 요일/시간별 반 확인
- 반별 현재 인원 / 정원 확인
- 반 클릭 시 반상세 이동
- 반상세에서 학생 목록 확인
- 반상세에서 정원 수정 가능
- 정원 수정은 승인 없이 즉시 저장
- 관리자와 동일 원본 데이터 반영

주간 페이지 표시 규칙
- 정원이 다 찬 반은 빨간색 표시
- 정원이 남은 반은 다른 색으로 표시
- 색상만 보이는 게 아니라 남은 자리 수도 같이 표시
- 예: 5/5, 4/5, 2/5 등

중요
- 이 주간 페이지는 보강 배정에서도 재사용해야 한다
- 즉 수업 관리에서 보는 주간 스케줄과 보강 배정용 주간 스케줄은 같은 원본 반 데이터를 사용해야 한다

────────────────────
[5. 출결 관리 탭]
출결 관리 탭은 아래 2개 하위 탭으로 구성한다.

- 출결 체크
- 보강 관리

현재 출결 체크만 있는 구조를 보강 관리까지 포함하는 구조로 확장한다.

────────────────────
[6. 출결 체크 탭]
출결 체크 탭은 선생님이 본인 반 출석/결석을 처리하는 곳이다.

기능
- 날짜별 출결 확인
- 오늘 수업 출결
- 학생별 출석 / 결석 처리
- 결석 처리 시 자동으로 보강 대기 생성
- 이미 처리된 출결 상태 표시
- 관리자가 처리한 데이터도 동일하게 반영

오늘 스케줄에서 출결 눌렀을 때도 이 화면의 해당 반/해당 시간 페이지로 바로 들어가야 한다.

즉 동선
- 오늘 스케줄 > 출결 클릭 > 해당 시간 출결 페이지 직행
- 출결 관리 > 출결 체크 > 반 선택 > 해당 시간 출결 페이지

둘 다 같은 출결 처리 화면을 사용해야 한다

────────────────────
[7. 보강 관리 탭]
보강 관리 탭은 관리자 모드와 동일한 원리로 동작해야 한다.
즉 결석 처리된 학생 리스트가 보이고, 보강 지정이 가능해야 한다.

핵심 규칙
- 관리자 모드와 같은 원본 보강 데이터를 사용
- 선생님 본인 반과 관련 있는 보강 건 위주로 표시
- 한쪽에서 보강 지정하면 다른 쪽에도 즉시 반영
- 선생님이 지정한 보강도 관리자 화면에 즉시 반영

보강 관리 리스트 표시 항목
- 학생 이름
- 원래 반명
- 결석 일자
- 결석 시간
- 현재 상태
- 보강 지정 버튼

추가 기능
- 리스트에서 학생을 누르면 내 반 주간 페이지 또는 보강 배정용 주간 페이지로 이동
- 여기서 정원이 다 차지 않은 반들만 선택 가능하게 보여준다
- 정원이 다 찬 반은 빨간색 표시
- 정원이 남은 반은 다른 색 표시
- 남은 자리가 있는 반만 선택 가능하게 하거나, 최소한 우선적으로 보이게 한다

핵심 동선
- 출결 관리 > 보강 관리 > 학생 선택
- 보강 배정용 주간 스케줄 페이지 이동
- 빈자리 있는 반 확인
- 해당 반 선택
- 보강 수업 지정 완료
- 지정되면 보강 대기 리스트에서 즉시 제거
- 관리자 모드에도 즉시 반영

보강 지정용 주간 페이지 표시 정보
- 반명
- 요일/시간
- 정원
- 현재 인원
- 남은 자리 수
- 색상 상태 표시

예시 색상 규칙
- 정원 가득 참 = 빨간색
- 정원 여유 있음 = 다른 색
- 추후 색상 규칙은 토큰화해서 변경 가능하게 설계

중요
- 선생님이 알아서 넣을 수 있게 비어 있는 반 목록 중심으로 보여주면 된다
- 불필요하게 복잡한 추천 알고리즘은 지금 단계에서 넣지 않는다

────────────────────
[8. 수영일지 탭]
수영일지 탭은 기존 구조 유지하되 실제 연결을 강화한다.

기능
- 오늘 수업에서 일지 클릭 시 해당 시간 일지 작성 페이지로 바로 이동
- 기존 작성 일지 조회
- 작성된 일지 수정 가능
- 개별 학생별 추가 메모 가능
- 관리자와 학부모 화면에 연결되는 원본 데이터 사용

동선
- 오늘 스케줄 > 일지 클릭 > 해당 반 일지 작성 직행
- 수영일지 탭 > 반 선택 > 일지 작성/수정

중요
- 수업 일지는 이번 작업에서 구조를 크게 바꾸지 않는다
- 다만 오늘 스케줄과의 직행 연결은 반드시 구현한다

────────────────────
[9. 사진영상 탭]
사진영상 탭은 선생님이 회원들에게 보낸 사진/영상 관리와 사용량 확인 기능까지 포함해야 한다.

필수 기능
- 사진 업로드
- 영상 업로드
- 반 전체 전송
- 개별 회원 전송
- 내가 보낸 사진/영상 목록 확인
- 내가 사용한 총 용량 표시

중요
- 선생님별 용량 사용량이 표시되어야 한다
- 추후 용량 과금에 사용될 수 있으므로 정확한 데이터가 필요하다
- 현재 사용량이 몇 GB인지 또는 MB인지 표시
- 가능하면 총 사용량 + 이번 달 사용량 구분 가능하게 설계

표시 예시
- 총 사용 용량
- 이번 달 업로드 용량
- 반별 업로드 수
- 최근 업로드 목록

최소 필수
- 내가 회원들에게 보낸 사진/영상의 총 용량 표시

────────────────────
[10. 관리설정 탭 추가]
내반 스케줄, 출결, 수영일지, 사진영상 옆에 관리설정 탭을 새로 추가한다.

이 탭은 선생님이 본인 운영 현황과 설정을 보는 관리용 탭이다.

관리설정 탭에 들어갈 수 있는 항목 예시
- 선생님 정보 수정
- 관리자 모드 전환
- 선생님 권한 탈퇴 요청
- 용량 관리
- 회원 현황
- 내반 통계

즉 이 탭은 단순 설정이 아니라
선생님용 운영 관리 센터처럼 만들어야 한다

────────────────────
[11. 관리설정 탭 상세 구성]
A. 선생님 정보 수정
- 이름
- 연락처
- 프로필 정보
- 간단한 소개 또는 메모
- 비밀번호 변경 등 확장 가능하게

B. 관리자 모드 전환
- 관리자 권한이 있는 계정이면 관리자 모드로 전환 가능
- 권한 없는 계정은 표시하지 않거나 비활성 처리

C. 선생님 권한 탈퇴 요청
- 퇴사 또는 권한 종료 요청 기능
- 요청 사유 입력 가능
- 관리자에게 전달되는 구조로 설계
- 실제 즉시 삭제가 아니라 요청 접수 형태

D. 용량 관리
- 내가 사용한 총 업로드 용량
- 이번 달 사용량
- 사진/영상 개수
- 반별 사용량 확장 가능하게 설계

E. 내반 통계
관리설정 안에 내반통계 영역을 만든다.

표시 항목
- 월요일 회원 수
- 화요일 회원 수
- 수요일 회원 수
- 목요일 회원 수
- 금요일 회원 수
- 토요일 회원 수
- 일요일 회원 수 필요 시 확장 가능하게 설계

핵심 규칙
- 요일별 회원 수만 간단하게 보여주면 된다
- 현재 내가 담당 중인 활성 회원 기준으로 집계한다
- 화면은 카드형 또는 줄형으로 간단하고 한눈에 보이게 만든다

F. 회원 현황
관리설정 안에 회원현황 메뉴를 만든다.

구조
- 회원현황 메뉴를 누르면 세로 3줄 리스트 형태로 보여준다
- 각 줄에는 상태명 + 인원수 표시가 함께 있어야 한다

표시 구조
- 정상회원 (인원수)
- 연기회원 (인원수)
- 탈퇴회원 (인원수)

즉 예시
- 정상회원 18명
- 연기회원 3명
- 탈퇴회원 6명

핵심 규칙
- 세로 3줄로 단순하고 바로 보이게 만든다
- 상태별 인원수를 같이 표시한다
- 각 줄을 누르면 해당 상태 회원 리스트로 이동 가능하게 만든다

회원 상태 정의
- 정상회원: 현재 내 수업에 실제로 소속되어 있는 활성 회원
- 연기회원: 잠시 쉬고 있는 회원
- 탈퇴회원: 수영을 완전히 그만둔 회원

담당 선생님 기준 규칙
- 회원이 나한테 배우다가 옆반 선생님에게 이동하면 그 회원은 이동된 선생님 쪽으로 소속이 넘어간다
- 이동 후에는 현재 담당 선생님 기준으로 회원 목록이 다시 계산되어야 한다
- 이전 선생님 쪽 정상회원 목록에 계속 남아 있으면 안 된다

기록 남김 규칙
- 연기 또는 탈퇴 기록은 마지막으로 그 회원을 수업 관리하던 선생님 쪽에 남긴다
- 이동 이력 자체는 지금 단계에서 별도로 만들지 않는다
- 즉 회원현황에서는 현재 담당 또는 마지막 담당 기준으로만 처리하면 된다

회원현황 리스트 표시 항목 예시
- 학생 이름
- 현재 상태
- 현재 또는 마지막 소속 반
- 상태 변경일
- 메모

정상회원 리스트
- 현재 내가 담당하는 활성 회원만 표시

연기회원 리스트
- 현재 내가 마지막으로 관리하던 상태에서 연기로 전환된 회원 표시
- 복귀 예정일이 있으면 함께 표시 가능

탈퇴회원 리스트
- 현재 내가 마지막으로 관리하던 상태에서 탈퇴 처리된 회원 표시
- 탈퇴일 표시 가능

────────────────────
[12. 선생님도 수업 탭이 많아야 한다는 방향 반영]
선생님은 실제 수업 운영을 직접 하는 역할이므로
관리할 수 있는 기능이 충분히 많아야 한다.

따라서 탭을 단순 조회용으로 만들지 말고
실제 관리 가능한 구조로 확장한다.

즉
- 수업 관리
- 출결 관리
- 수영일지
- 사진영상
- 관리설정
이 각각 독립적인 실무 탭이 되게 만든다

────────────────────
[13. 명칭 변경 제안 반영]
기존 내반 / 오늘 스케줄 / 출결 같은 명칭은 보다 실무형으로 바꾼다.

메인 화면
- 오늘 스케줄

빠른 이동 탭
- 수업 관리
- 출결 관리
- 수영일지
- 사진영상
- 관리설정

즉 첫 화면에서 오늘 스케줄을 보여주고
밑의 빠른 이동 탭으로 각 관리 기능에 들어가는 구조로 정리한다

────────────────────
[14. 데이터 연동 원칙]
이번 작업은 화면만 만드는 것이 아니라 실제 실시간 반영 구조여야 한다.

반드시 지킬 것
- 출결 처리 시 보강 대기 즉시 생성
- 보강 지정 시 관리자/선생님 모두 즉시 반영
- 정원 수정 시 수업관리/출결/보강/일지 인원표시 모두 즉시 반영
- 사진영상 용량 사용량 즉시 누적
- 회원 상태(정상/연기/탈퇴) 변경 시 관리설정 회원현황 즉시 반영
- 회원이 다른 선생님 반으로 이동하면 현재 소속/담당 기준이 즉시 변경되어야 함
- 연기/탈퇴 이력은 마지막 담당 선생님 기준으로 남아야 함

────────────────────
[15. 공통 컴포넌트 및 구조 유지]
중복 화면 생성 금지
같은 기능은 같은 컴포넌트를 재사용한다.

예시
- 출결 처리 화면 공통 재사용
- 보강 배정용 주간 스케줄 공통 재사용
- 반상세 공통 재사용
- 정원 표시 배지 공통화
- 용량 표시 카드 공통화
- 회원상태 리스트 공통화

금지사항
- 선생님용이라고 같은 기능을 새로 복붙해 만들지 말 것
- 관리자와 선생님의 원본 데이터 모델을 분리하지 말 것
- 상태값을 화면마다 다르게 쓰지 말 것

────────────────────
[16. 필수 테스트]
완료 전에 실제 작동 확인 필수

A. 오늘 스케줄 테스트
- 오늘 스케줄에서 출결 클릭 시 해당 시간 출결 화면 직행 확인
- 오늘 스케줄에서 일지 클릭 시 해당 시간 일지 작성 화면 직행 확인

B. 출결/보강 테스트
- 선생님 결석 처리 시 보강 관리 리스트 생성 확인
- 관리자 결석 처리 건이 선생님 보강 관리에도 보이는지 확인
- 보강 관리에서 학생 클릭 시 보강 배정용 주간 페이지 이동 확인
- 빈자리 있는 반 선택 후 보강 지정 저장 확인
- 저장 후 리스트에서 즉시 제거 확인

C. 정원 표시 테스트
- 주간 페이지에서 정원 가득 찬 반 빨간색 표시 확인
- 정원 여유 반 다른 색 표시 확인
- 현재 인원/정원 표시 확인

D. 사진영상 테스트
- 사진 업로드 후 용량 누적 확인
- 영상 업로드 후 총 사용량 반영 확인
- 사용량 카드 표시 확인

E. 관리설정 테스트
- 선생님 정보 수정 저장 확인
- 관리자 모드 전환 버튼 조건부 표시 확인
- 탈퇴 요청 기능 진입 확인
- 내반통계 > 요일별 회원 수 표시 확인
- 회원현황 > 정상회원(인원수) 줄 표시 확인
- 회원현황 > 연기회원(인원수) 줄 표시 확인
- 회원현황 > 탈퇴회원(인원수) 줄 표시 확인
- 회원현황 각 줄 클릭 시 해당 회원 리스트 이동 확인
- 회원이 옆반 선생님으로 이동했을 때 현재 담당 선생님 기준 목록 반영 확인
- 연기/탈퇴 처리 시 마지막 담당 선생님 기준으로 기록 남는지 확인

────────────────────
[17. 제출해야 할 결과]
작업 후 아래를 같이 제출할 것

1. 수정한 선생님 모드 페이지 목록
2. 추가한 탭 목록
3. 추가한 세부 메뉴 목록
4. 보강 배정 동선 설명
5. 사진/영상 용량 계산 방식 설명
6. 관리설정 회원현황 집계 방식 설명
7. 내반통계 요일별 집계 방식 설명
8. 마지막 담당 선생님 기준 기록 처리 방식 설명
9. 실제 클릭 테스트 결과
10. 실패한 항목과 원인
11. 리플릿 실행 및 확인 결과

테스트 결과 예시 형식
- 선생님 메인 > 오늘 스케줄 > 15:00반 출결 클릭 > 출석 체크 페이지 직행 성공
- 선생님 메인 > 오늘 스케줄 > 15:00반 일지 클릭 > 일지 작성 페이지 직행 성공
- 출결 관리 > 보강 관리 > 학생A 클릭 > 보강 배정용 주간 스케줄 이동 성공
- 보강 배정용 주간 페이지 > 빈자리 있는 반 선택 > 보강 지정 저장 성공
- 사진영상 > 사진 업로드 > 총 사용량 반영 성공
- 관리설정 > 내반통계 > 요일별 회원 수 표시 성공
- 관리설정 > 회원현황 > 정상회원 18명 / 연기회원 3명 / 탈퇴회원 6명 표시 성공
- 학생B를 옆반 선생님으로 이동 > 기존 선생님 정상회원 목록 제거 성공
- 학생C 연기 처리 > 마지막 담당 선생님 연기회원 리스트 기록 성공
- 학생D 탈퇴 처리 > 마지막 담당 선생님 탈퇴회원 리스트 기록 성공

중요
- UI만 만들고 끝내지 말 것
- 실제 저장/이동/반영 확인 후 완료로 보고할 것
- 리플릿에서 실제 실행 후 결과를 제출할 것
