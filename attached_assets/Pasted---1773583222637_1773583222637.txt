로그인 화면을 처음부터 다시 정리해줘.
기존 기능은 최대한 유지하되 레이아웃과 진입 구조를 더 명확하고 깔끔하게 리팩토링해줘.
기존 코드와 겹치거나 충돌하지 않게 정리하고, 필요하면 공통 컴포넌트로 분리해줘.

목표:
1. 앱 첫 화면인 로그인 화면을 더 정돈된 서비스형 구조로 개선
2. 로그인 / 테스트 계정 빠른 진입 / 회원가입 진입 경로를 명확하게 분리
3. 사용자가 처음 봐도 어디로 들어가야 하는지 헷갈리지 않게 구성
4. 추후 운영용 실제 앱처럼 보여야 함
5. 기존 인증 로직과 충돌하지 않게 구현

반드시 반영할 조건:
1. 상단에는 앱 로고 + 앱 이름 + 부제 유지
   - 앱 이름: 스윔노트
   - 부제: 수영장 통합 관리 플랫폼
2. 메인 로그인 박스는 깔끔한 카드형 구조로 유지
3. 입력창은
   - 아이디
   - 비밀번호
   두 개만 유지
4. 로그인 버튼은 크게 유지하고 모바일에서 한눈에 보이게 해줘
5. 현재 있는 테스트 계정 빠른 로그인 영역은 유지하되 더 정돈된 레이아웃으로 바꿔줘
6. 테스트 계정은 역할별로 명확히 보여줘
   - 플랫폼 운영자
   - 수영장 관리자
   - 선생님
   - 학부모
7. 빠른 로그인 카드는 같은 크기/간격/디자인 규칙으로 통일
8. 현재 선택 또는 클릭 시 레이아웃이 흔들리거나 카드 크기가 바뀌지 않게 해줘

하단 진입 링크 구조:
1. 수영장 사업자인가요? → 가입 신청하기
2. 선생님으로 초대받으셨나요? → 초대 코드로 가입
3. 학부모이신가요? → 학부모 가입 / 로그인 안내
4. 각 링크는 서로 의미가 겹치지 않게 문구를 분리해줘
5. 링크 영역은 로그인 화면 하단에 너무 복잡하지 않게 2~3줄로 정리

사용자 경험 요구:
1. 로그인 화면은 “누가 이 앱을 쓰는지”가 바로 보여야 함
2. 한 화면 안에서
   - 로그인
   - 테스트 진입
   - 가입 경로 안내
   가 동시에 가능해야 함
3. 하지만 정보가 너무 많아 보이지 않게 시각적으로 구역 분리
4. 로그인 카드 / 테스트 계정 영역 / 가입 안내 영역 사이 간격을 일정하게 유지

디자인 요구:
1. 전체 배경은 지금 앱 톤과 맞는 밝은 회색/화이트 계열 유지
2. 메인 포인트 컬러는 현재 앱의 파란색 계열 유지
3. 로그인 카드, 테스트 계정 카드, 하단 링크 텍스트의 스타일을 전부 통일감 있게 정리
4. 모서리 radius, 그림자, padding, margin 규칙 통일
5. font size hierarchy를 정리해서 제목/본문/보조문구가 확실히 구분되게 해줘
6. 입력창 placeholder와 아이콘 간격도 정리
7. 작은 화면에서도 잘리지 않게 반응형으로 맞춰줘

기능 요구:
1. 기존 로그인 기능은 그대로 동작해야 함
2. 기존 테스트 계정 빠른 로그인도 그대로 동작해야 함
3. 테스트 카드 클릭 시 해당 계정으로 바로 로그인되게 유지
4. 가입 신청하기 / 초대 코드 가입 / 학부모 가입은 기존 라우트가 있으면 연결하고
5. 아직 없으면 임시 라우트 또는 TODO 주석으로 명확히 연결 포인트를 남겨줘

구조 요구:
1. 로그인 화면 코드를 너무 길게 한 파일에 몰아넣지 말고 분리해줘
2. 예:
   - components/auth/LoginForm.tsx
   - components/auth/QuickLoginCard.tsx
   - components/auth/AuthEntryLinks.tsx
3. 로그인 화면은 조립형으로 정리
4. 추후 로고/문구/테스트 계정만 쉽게 수정할 수 있게 데이터 배열 기반으로 작성

테스트 계정 섹션 구현 방식:
- 데이터 배열 기반으로 역할, 아이디, 비밀번호, 아이콘, 색상 톤을 관리
- 화면에는
  역할명
  ID / PW
  를 보기 쉽게 표시
- 클릭 영역 넓게
- 카드 높이 고정
- 선택해도 크기 변화 없음

로그인 화면 문구 정리:
- 상단 제목 아래 짧은 설명 문구를 추가해줘
  예: 운영자, 선생님, 학부모가 하나의 앱에서 연결되는 수영장 관리 서비스
- 단, 너무 길지 않게 한 줄 또는 두 줄 이내로 정리

주의사항:
1. 지금까지 만든 역할 구조(플랫폼 운영자 / 수영장 관리자 / 선생님 / 학부모)를 절대 흐리지 말 것
2. 관리자 전용 / 선생님 전용 / 학부모 전용 회원가입 진입점을 로그인 화면에서 명확히 구분할 것
3. 나중에 실제 서비스 배포용으로 써도 어색하지 않게 정리할 것
4. 현재 다른 화면들과 디자인 톤이 맞게 유지할 것
5. 기존 기능을 깨지 않도록 라우팅/로그인 로직은 최대한 보존할 것

추가로 해줘:
1. 키보드가 올라와도 로그인 버튼이 가려지지 않게 처리
2. 스크롤이 필요하면 자연스럽게 되게 처리
3. iPhone Safari / Expo Web / 모바일 앱 환경에서 모두 보기 좋게 맞춰줘
4. safe area 고려
5. 로그인 성공 후 기존 역할별 분기 로직이 그대로 작동하는지 확인해줘

최종 결과:
- 로그인 화면이 현재보다 훨씬 정돈된 서비스형 첫 화면처럼 보이게
- 역할별 빠른 진입과 실제 로그인, 회원가입 경로가 한 번에 이해되게
- 기존 내용과 겹치지 않고 공통 컴포넌트 기반으로 깔끔하게 정리
