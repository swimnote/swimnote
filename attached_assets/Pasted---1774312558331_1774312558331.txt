[스윔노트 출시 직전 전체 통합 시뮬레이션 + 버그 수정 + 회귀 테스트 지시서]

지금부터 스윔노트 앱 전체를 출시 직전 기준으로 처음부터 끝까지 실제 서비스처럼 시뮬레이션하며 점검한다.
이번 작업의 목적은 “기능이 있는지” 확인하는 수준이 아니라,
회원가입부터 회원등록, 승인, 반배정, 출결, 보강, 일지, 사진, 푸시, 구독, 결제, 정산, 읽기모드, 삭제정책까지
전 흐름을 실제 운영 시나리오로 돌려보면서 문제를 찾고 즉시 수정하는 것이다.

중요:
- 화면만 확인하고 완료 보고 금지
- 실제 데이터 생성/변경/상태 전환까지 확인
- 하나 고치면 관련 기능 회귀 테스트까지 반드시 수행
- iPhone Expo Go 기준 우선 확인
- 웹에서만 되고 모바일에서 멈추는 기능은 실패로 간주
- 확인 안 한 기능은 반드시 “미확인”이라고 적을 것
- 거짓 완료 보고 절대 금지

==================================================
0. 테스트 목표
==================================================

최종 목표:
1. 앱 전체 핵심 흐름이 실제로 끊김 없이 작동하는지 확인
2. 관리자 / 선생님 / 학부모(학부모 모드) 모두에서 권한과 화면 흐름이 맞는지 확인
3. 구독/결제/정산 구조가 실제 데이터 기준으로 맞는지 확인
4. 버그를 찾으면 즉시 수정하고, 수정 후 재검증
5. 모든 핵심 흐름이 안정화되면 그 다음 단계로 앱스토어 등록 준비로 넘어간다

==================================================
1. 테스트 원칙
==================================================

- 테스트는 실제 사용자처럼 순서대로 진행한다
- 가능한 한 seed/mock 데이터만 보지 말고 실제 생성/수정/저장/상태 변경을 해본다
- 한 기능을 고친 뒤에는 반드시 관련 화면 전체를 다시 확인한다
- 뒤로가기/홈/팝업 닫기 규칙이 무너지면 버그로 기록한다
- 모바일에서 멈춤, 터치 안 먹음, 겹침, 바텀시트 잔상, 잘못된 route 이동은 모두 버그다

==================================================
2. 테스트 데이터 기본 시나리오
==================================================

기본 테스트 구조를 아래처럼 준비한다.

테스트 수영장:
- 토이키즈 1개를 메인 테스트 수영장으로 사용
- 필요하면 추가 수영장 1~2개 더 생성

기본 계정:
- 슈퍼관리자 1
- 수영장 관리자 1
- 선생님 1~2
- 학부모 1~2

학생/회원 테스트 데이터:
- 반배정된 회원
- 미배정 회원
- 학부모 연결 완료 회원
- 학부모 미연결 회원
- 승인대기 회원/학생
- 결석/보강 대상 회원
- 레벨이 있는 회원
- 사진/일지가 있는 회원

==================================================
3. 전체 시뮬레이션 순서
==================================================

아래 순서대로 실제로 돌린다.

[1단계] 회원가입 및 계정 흐름
1. 신규 학부모 가입
2. 관리자 승인/자동승인 흐름 확인
3. 선생님 계정 승인 확인
4. 로그인/로그아웃/재로그인
5. 비정상 진입 시 로그인 화면 복귀 확인

[2단계] 회원 등록/승인/연결
1. 관리자 홈 퀵버튼 회원추가 → 어린이 직접 등록
2. 선생님 홈 퀵버튼 회원추가 → 등록요청
3. 관리자 승인대기 리스트 반영
4. 승인 후 active 전환
5. 학부모 연결 / 미연결 상태 확인
6. 미배정 / 학부모 미연결 분리 집계 확인

[3단계] 반배정/반해제/미배정 흐름
1. 미배정 회원이 관리자/선생님 공통 목록에 보이는지
2. 선생님 스케줄러 → 날짜 → 시간 → 반배정 변경 진입
3. 바텀시트 잔상 없는지
4. 반배정 완료 시 미배정 목록에서 제거되는지
5. 반해제 시 다시 미배정으로 복귀하는지
6. 관리자/선생님 카운트가 실시간 반영되는지

[4단계] 출결/보강/일지
1. 선생님이 출석 처리
2. 결석 처리
3. 보강 리스트 이동
4. 보강 정책에 맞는 기한/횟수 반영
5. 보강 완료 처리
6. 일지 작성
7. 학부모 모드에서 일지 확인
8. 수정/삭제 이력 표시 확인

[5단계] 사진/앨범/용량
1. 사진 업로드
2. 앨범 반영
3. 학부모 열람
4. 압축본 다운로드 정책 확인
5. 저장공간 사용량 반영
6. 80%/90%/100% 정책 시뮬레이션
7. 100%에서 업로드 차단과 팝업 확인
8. 사진 정리하기 버튼 동작 확인

[6단계] 레벨/피드백/설정
1. 레벨 설정
2. 레벨 비활성/사용안함 처리
3. 선생님 레벨 변경
4. 학부모에서 레벨 뱃지와 설명 확인
5. 수업설정 > 피드백 기본설정 연결 확인
6. 운영설정/수업설정 중복 제거 확인

[7단계] 푸시/공지
1. 공지 등록
2. 공지 즉시 푸시 조건 확인
3. 수업 전날 알림 시간 설정 확인
4. 수업 당일 n시간 전 알림 설정 확인
5. 일지 등록 푸시
6. 사진만 업로드 시 사진 푸시
7. 학부모/선생님/관리자별 푸시 수신 규칙 확인

[8단계] 구독/결제/정산
1. 무료 플랜 확인 (5명/100MB)
2. 스타터/베이직/스탠다드/어드밴스/프로/맥스/엔터프라이즈 2000/3000 표시 확인
3. 슈퍼관리자 구독 상품 설정과 수영장 관리자 구독관리 값이 같은지 확인
4. 플랜 선택/업그레이드 시 subscription 저장
5. revenue/settlement 로그 생성
6. 매출정산관리 반영
7. 환불 상태 반영
8. 업그레이드/다운그레이드 상태 반영
9. 과거 플랜명 잔재 없는지 확인

[9단계] 제한 정책/읽기모드
1. 회원 수 초과 시 회원추가 잠금
2. 용량 100%에서 업로드 차단
3. 결제 실패 즉시 읽기모드 전환
4. 관리자 메시지 / 선생님·학부모 메시지 분리
5. 7일 경고 / 14일 삭제 상태 전환 로직 확인
6. 읽기모드에서도 결제/구독 화면 접근 가능한지 확인

==================================================
4. 화면 흐름/UX 공통 점검
==================================================

모든 모드에서 아래를 공통으로 점검한다.

- 뒤로가기 누르면 직전 화면으로 가는지
- 팝업 닫기(X) 하면 팝업만 닫히는지
- 홈 버튼 누르면 홈으로 가는지
- 저장 후 갑자기 홈으로 튀지 않는지
- 바텀시트가 다음 화면 뒤에 남지 않는지
- 카드/칩/버튼/배지 겹침 없는지
- 아이폰에서 터치 안 먹는 구간 없는지
- 월간/주간/일간 스케줄러 멈춤 없는지

==================================================
5. 발견 즉시 수정해야 하는 버그 유형
==================================================

아래는 발견 즉시 수정하고 같은 세션에서 재테스트한다.

- 잘못된 route 연결
- 팝업 잔상/바텀시트 중첩
- 목록 누락/실시간 카운트 미반영
- 관리자/선생님/학부모 권한 충돌
- 읽기모드 차단 누락
- 과거 플랜/과거 가격 잔재
- 터치 멈춤 / 모바일 렌더 오류
- 홈/뒤로가기 규칙 위반
- 동일 기능이 두 군데서 서로 다르게 동작하는 문제

==================================================
6. 회귀 테스트 원칙
==================================================

버그 하나 수정하면 아래를 반드시 다시 확인한다.

예:
- 반배정 관련 버그 수정
  → 반배정 / 반해제 / 미배정 카운트 / 관리자 목록 / 선생님 목록 / 학부모 표시까지 재검증

- 결제 플랜 수정
  → 슈퍼관리자 상품설정 / 수영장 구독관리 / 매출정산 / 제한 로직까지 재검증

- 설정 메뉴 연결 수정
  → 수업설정 / 운영설정 / 해당 상세 화면 전체 재검증

==================================================
7. 우선순위
==================================================

버그 수정 우선순위는 아래 순서로 한다.

1. 데이터/상태 오류
2. 결제/정산/제한 정책 오류
3. 기능 동작 오류
4. 네비게이션/팝업 흐름 오류
5. UI 겹침/가독성 문제

==================================================
8. 완료 기준
==================================================

다음 조건을 만족해야 “출시 직전 안정화 완료”로 본다.

- 핵심 시나리오가 처음부터 끝까지 끊김 없이 작동
- 관리자/선생님/학부모 권한이 명확하고 충돌 없음
- 미배정/미연결/승인대기/정상 상태 분리 정확
- 구독/결제/정산 로그가 같은 기준으로 연결
- 읽기모드/삭제정책 정상 작동
- 모바일(iPhone Expo Go)에서 주요 화면 멈춤 없음
- 뒤로가기/팝업/홈 흐름 규칙 통일 완료
- 과거 플랜/과거 가격/과거 용어 잔재 제거 완료

==================================================

반드시 아래 형식으로 자세히 보고하라.

1. 테스트한 전체 시나리오 목록
2. 발견한 버그 목록

4. 수정한 파일 목록

중요:
- “전체 점검 완료” 한 줄 보고 금지
- 실제로 어떤 기능을 눌렀고 어떤 결과가 나왔는지 적어라
