[긴급 수정 지시서 - 선생님 모드 반배정 해제 버튼 무동작 수정 + “반 제외-미배정 이동” 전면 통일]

지금부터 선생님 모드 스케줄러에서
반배정 관련 해제/대기 버튼이 눌려도 실제로 작동하지 않는 문제를 긴급 수정한다.

이번 작업 목표:
1. 기존의 “해제”, “대기” 개념을 제거하거나 정리하고
2. 버튼명을 명확하게 “반 제외-미배정 이동”으로 통일하고
3. 클릭 시 실제로 회원이 해당 반에서 빠지고 미배정 상태로 이동되게 만들며
4. 이 동작이 월간/주간/일간 및 관련 반배정 화면 전체에 동일하게 적용되도록 한다.

중요:
- 이번 작업은 단순 버튼 텍스트 변경이 아니다
- 실제 데이터 상태 변경, 목록 반영, 카운트 반영, UI 반영까지 끝나야 한다
- 한 화면만 고치고 다른 스케줄러 화면에 남아 있으면 실패
- iPhone Expo Go 기준으로 실제 클릭 테스트 필수
- 거짓 완료 보고 절대 금지

==================================================
1. 현재 문제 정의
==================================================

현재 문제:
- 선생님 모드 스케줄에서 반배정 화면 진입
- 기존 “해제” 또는 “대기” 버튼을 눌러도
  실제로 회원 상태가 바뀌지 않거나 반영이 안 됨
- UI만 있고 기능 연결이 끊겨 있을 가능성이 큼

최종 목표:
- 해제/대기 버튼 개념을 정리
- “반 제외-미배정 이동” 버튼으로 명확화
- 누르면 즉시 해당 반 소속에서 제거되고 미배정 상태로 이동

==================================================
2. 버튼명/개념 정리
==================================================

기존:
- 해제
- 대기

변경:
- 반 제외-미배정 이동

원칙:
- “대기”라는 애매한 표현 제거
- “해제”도 단독 표현 대신 목적이 보이게 변경
- 사용자가 눌렀을 때 어떤 결과가 나는지 바로 이해되게 한다

중요:
- 반에서 빠진다는 것
- 미배정으로 이동한다는 것
두 개가 버튼명 또는 확인 문구에 명확히 드러나야 한다

==================================================
3. 실제 동작 정의
==================================================

“반 제외-미배정 이동” 클릭 시 반드시 일어나는 일:

1. 해당 회원의 현재 반/스케줄 연결 제거
2. assignment_status = unassigned (미배정)
3. 해당 반 현재 소속 회원 목록에서 즉시 제거
4. 미배정 목록에 즉시 추가
5. 관리자/선생님 양쪽 카운트 즉시 반영
6. 학부모 미연결 상태와는 별개로 유지
7. 과거 기록(출결/일지/보강)은 유지

중요:
- 회원 삭제 아님
- 승인대기 아님
- 단순히 반 소속만 제거되고 “미배정”으로 이동하는 것

==================================================
4. 적용 범위 (전 화면 강제)
==================================================

반드시 아래 모든 화면에 동일 적용:

A. 선생님 모드
- 월간 스케줄러
- 주간 스케줄러
- 일간 스케줄러
- 반배정 변경 화면
- 반 상세/현재 소속 회원 리스트
- 스케줄 시간 클릭 후 나오는 바텀시트/팝업

B. 관리자 모드
- 동일한 반배정/반제외 기능이 있다면 동일 문구/동작으로 정리
- 미배정 카운트 및 목록 즉시 반영 확인

중요:
- 화면마다 버튼명/동작이 다르면 안 된다
- 하나는 해제, 하나는 대기, 하나는 미배정 이동이면 실패
- 전부 같은 개념으로 통일

==================================================
5. UI/문구 정리
==================================================

버튼명:
- 반 제외-미배정 이동

확인 팝업 문구 예시:
“이 회원을 현재 반에서 제외하고 미배정 상태로 이동하시겠습니까?
기존 수업 기록은 유지됩니다.”

버튼:
- [취소]
- [이동]

중요:
- 실수 클릭 방지 위해 확인 팝업 추가
- 바로 삭제되는 것처럼 보이면 안 됨
- 기록 유지 안내도 같이 보여줄 것

==================================================
6. 데이터 처리 기준
==================================================

반드시 확인:
- assigned_class_id / class_group_id / current_schedule_id 등
  현재 반 소속을 의미하는 필드를 정확히 해제
- 미배정 판단 공통 기준과 일치하게 처리
- pending 승인대기 상태로 바꾸지 말 것
- 학부모 연결 상태(parent_connection_status)는 건드리지 말 것

즉:
- 수업 상태만 변경
- 계정 연결 상태는 유지

==================================================
7. 실시간 반영 강제
==================================================

버튼 클릭 후 저장 성공 시 즉시 반영되어야 한다.

즉시 변해야 하는 것:
- 현재 반 회원 수
- 반배정 변경 화면의 현재 소속 목록
- 미배정 회원 목록
- 관리자 미배정 카운트
- 선생님 미배정 카운트
- 관련 검색 결과

중요:
- 새로고침해야만 보이는 구조 금지
- invalidate/refetch 또는 공통 상태 갱신으로 즉시 반영

==================================================
8. 회귀 테스트 필수
==================================================

시나리오 A
- 선생님 월간 스케줄러 진입
- 시간 선택
- 반배정 화면 진입
- 현재 소속 회원에 대해 “반 제외-미배정 이동” 클릭
기대 결과:
- 반 목록에서 즉시 제거
- 미배정 목록에 즉시 추가

시나리오 B
- 주간 스케줄러에서 동일 동작
기대 결과:
- 동일하게 작동

시나리오 C
- 일간 스케줄러에서 동일 동작
기대 결과:
- 동일하게 작동

시나리오 D
- 관리자 화면에서 해당 회원 확인
기대 결과:
- 미배정 상태 반영
- 카운트 반영

시나리오 E
- 학부모 연결된 회원에 대해 반 제외 실행
기대 결과:
- 반만 제외
- 학부모 연결 상태는 유지
- 학부모 미연결 카운트는 변하지 않음

시나리오 F
- 과거 출결/일지 조회
기대 결과:
- 기존 기록 유지

9. 의심 항목 전수 점검
==================================================

조금이라도 의심되면 바로 확인:
- 버튼 onPress가 비어 있는지
- 잘못된 handler 재사용 중인지
- “해제”와 “대기”가 다른 상태값을 쓰는지
- 반에서 빠지지만 미배정 목록에는 안 들어가는지
- 관리자와 선생님 화면 기준이 다른지
- 월간에서는 되고 주간/일간에서는 안 되는지
- Expo Go에서 클릭이 안 먹는 overlay가 있는지
