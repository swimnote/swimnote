지금부터 관리자 모드와 선생님 모드에 “미등록회원 일괄등록 + 미등록회원 공유 명단 + 선생님 반배정 전환” 기능을 추가하라.

이번 작업 목표는
1. 관리자가 학생 이름/학부모 전화번호 엑셀을 업로드해 미등록회원 명단을 만들고
2. 관리자와 선생님이 같은 미등록회원 명단을 공유하며
3. 선생님이 미등록회원을 반배정하면 정상회원으로 전환되게 만드는 것이다.

가장 중요:
- 기존 사람탭/회원/반배정/일지/학부모 연결 구조를 깨지 말 것
- 미등록회원 명단은 관리자와 선생님이 연동된 같은 데이터여야 한다
- 반배정 전에도 학생 데이터는 존재
- 반배정 후 일지 작성 가능
- 학부모 가입 전에는 일지를 볼 수 없음
- 추후 학부모 가입 시 기존 학생/일지/수업 데이터가 자동 연결되어야 한다

==================================================
[1. 최종 상태 흐름]

회원 상태 흐름을 아래처럼 고정한다.

- 초대 전
- 초대 완료
- 가입 완료
- 미등록회원
- 반배정 완료 → 정상회원

중요:
학부모 가입 완료와 반배정 완료는 별개다

즉:
- 학부모 가입 완료했어도 반배정 안 되면 여전히 미등록회원 가능
- 반배정 먼저 되어도 학부모 가입 전이면 학부모는 열람 불가
- 나중에 학부모 가입되면 학생 데이터와 연결

==================================================
[2. 관리자 역할]

관리자는 아래까지만 담당한다.

1. 템플릿 다운로드
2. 엑셀 업로드
3. 데이터 검증
4. 미등록회원 명단 생성
5. 학부모 초대 문자 발송

중요:
관리자는 반배정하지 않는다
반배정은 선생님이 한다

==================================================
[3. 엑셀 템플릿 구조]

엑셀 템플릿 컬럼은 딱 2개만 사용한다.

- 학생 이름
- 학부모 전화번호

예시:
학생 이름 | 학부모 전화번호
서태웅   | 01012345678

규칙:
- 컬럼 순서 고정
- 샘플 한 줄 포함 가능
- 칸 위치 변경 금지 안내 포함

==================================================
[4. 엑셀 업로드 기능]

관리자 > 사람 탭 안에 아래 기능 추가

- 템플릿 다운로드
- 회원 명단 업로드

업로드 후 순서:
1. 파일 파싱
2. 검증
3. 검증 결과 화면
4. 정상 항목 등록

==================================================
[5. 검증 규칙]

엑셀 각 행 검증:

필수:
- 학생 이름 비어 있으면 오류
- 학부모 전화번호 비어 있으면 오류
- 전화번호 형식 이상하면 오류
- 같은 파일 안 학생+전화번호 중복이면 중복 표시
- 기존 미등록회원 명단에 같은 학생+전화번호 있으면 중복 표시

결과 상태:
- 정상
- 중복
- 오류

규칙:
- 검증 없이 바로 등록 금지
- 정상 데이터만 등록 가능

==================================================
[6. 미등록회원 저장 구조]

정상 검증된 항목은 “미등록회원”으로 저장한다.

미등록회원 저장 필드 최소:
- student_id
- 학생 이름
- 학부모 전화번호
- invite_status
  - 초대 전
  - 초대 완료
  - 가입 완료
- registration_status
  - 미등록회원
  - 정상회원
- created_at
- assigned_class_id (없으면 null)
- assigned_teacher_id (없으면 null)
- parent_account_id (없으면 null)

중요:
미등록회원과 정상회원을 같은 사람 데이터 축 안에서 관리하되
상태로 구분해도 되고,
목록은 분리해서 보여줘야 한다

==================================================
[7. 관리자 사람 탭 안 미등록회원 목록]

관리자 > 사람 탭 안에 “미등록회원” 목록을 추가하라.

표시 항목:
- 학생 이름
- 학부모 전화번호
- 초대 상태
- 반배정 상태
  예:
  - 미등록회원
  - 정상회원 전환 완료
- 등록일

기능:
- 검색창
- 체크박스 선택
- 전체 선택
- 학부모 초대 버튼

중요:
미등록회원은 사람 탭 안에 있어야 한다
더보기로 빼지 말 것

==================================================
[8. 관리자 미등록회원 검색 기능]

미등록회원 목록 상단에 검색창 추가

검색 기준:
- 학생 이름
- 학부모 전화번호 일부

입력 즉시 필터링
검색어 비우면 전체 목록 복원

==================================================
[9. 학부모 초대 발송]

미등록회원 목록에서
관리자가 여러 명 선택 후 학부모 초대 문자를 발송할 수 있어야 한다.

규칙:
- 선택된 인원 수 표시
- 발송 전 확인 팝업 필수

확인 팝업 예:
- 선택한 12명의 학부모에게 초대 문자를 발송하시겠습니까?

발송 후:
- 초대 상태: 초대 전 → 초대 완료

중요:
선택 없이 발송 금지

==================================================
[10. 선생님 모드 - 미등록회원 연동]

선생님 모드에서도
관리자와 동일한 “미등록회원 명단 데이터”를 볼 수 있어야 한다.

즉:
관리자에서 업로드해 만든 미등록회원 명단을
선생님도 실시간/동일 데이터로 봐야 한다.

중요:
관리자용 따로, 선생님용 따로 데이터 복사 금지
반드시 연동된 같은 명단을 사용

==================================================
[11. 선생님 모드에서 미등록회원 보기]

선생님 모드의 반배정/수업관리 흐름 안에
“미등록회원 가져오기” 기능을 추가하라.

추천 위치:
- 내 반 관리 화면
- 반배정 화면
- 또는 수업 관련 관리 안

버튼명:
- 미등록회원 가져오기

클릭 시:
- 미등록회원 리스트 팝업 또는 페이지 표시
- 검색창 포함
- 학생 이름 기준 검색 가능

목록 표시:
- 학생 이름
- 학부모 전화번호
- 초대 상태
- 가입 상태

==================================================
[12. 선생님 반배정 기능]

선생님이 미등록회원 리스트에서 학생을 선택하고
현재 반 또는 선택한 반에 배정할 수 있어야 한다.

동작:
1. 선생님이 미등록회원 선택
2. 반 선택 또는 현재 반에 넣기
3. 확인
4. 상태 변경

상태 변화:
- registration_status: 미등록회원 → 정상회원
- assigned_class_id 저장
- assigned_teacher_id 저장

중요:
이 시점부터 학생은 정상회원으로 간주
반에 들어가고 수업/출결/일지 대상이 됨

==================================================
[13. 학부모 가입 전 반배정 허용]

중요한 규칙:
학부모가 아직 가입하지 않았어도
선생님은 미등록회원을 반배정할 수 있다.

즉:
- 학부모 가입 전에도 반배정 가능
- 반배정 후 수업/출결/일지 작성 가능

하지만:
- 학부모가 가입하기 전에는
  학부모 화면에서 아무것도 볼 수 없음

==================================================
[14. 학부모 나중 가입 시 자동 연결]

추후 학부모가 가입 완료되면
기존 학생 데이터와 자동 연결되어야 한다.

연결 기준:
- 학생 이름 + 학부모 전화번호
또는
- 미리 발급된 초대 코드/연결 키

연결 후:
- 기존 학생 데이터 유지
- 기존 반배정 유지
- 기존 출결/일지/수업 데이터 유지
- 학부모는 가입 이후부터 자기 학생 데이터 열람 가능

즉:
선생님이 먼저 운영한 데이터가
나중에 학부모에게 한 번에 연결되는 구조

==================================================
[15. 일지/열람 규칙]

반배정 후:
- 선생님은 일지 작성 가능
- 수업/출결/이동 처리 가능

학부모 가입 전:
- 학부모 열람 불가

학부모 가입 후:
- 기존 누적 데이터까지 연결해서 볼 수 있게 구조 준비

이번 단계에서는 최소한:
- 가입 전 열람 차단
- 가입 후 연결 가능 구조
를 확보할 것

==================================================
[16. 기존 기능 보호]

반드시 유지:
- 사람 탭 기존 구조
- 회원/학부모/선생님 관리 구조
- 반배정 구조
- 수업/출결/일지 구조
- 역할/권한 구조

이번 작업은
미등록회원 업로드와 반배정 흐름을 추가하는 것이지
기존 구조를 갈아엎는 작업이 아니다

==================================================
[17. 먼저 확인할 것]

수정 전에 아래를 먼저 파악하라.

1. 관리자 사람 탭 파일
2. 회원 데이터 저장 구조
3. 미등록회원 저장 구조 가능 여부
4. 엑셀 업로드/파싱 파일
5. 문자 발송 연결 구조
6. 선생님 반배정 화면 파일
7. 현재 반배정 로직 파일
8. 학부모 가입 연결 로직 파일
9. 검색 가능한 리스트 컴포넌트

[20. 절대 금지]

- 관리자와 선생님 미등록회원 명단을 따로 따로 유지하는 것 금지
- 학부모 가입 전 반배정 막는 것 금지
- 반배정 후에도 계속 미등록회원 상태로 남는 것 금지
- 미등록회원과 정상회원 목록이 뒤섞여 보이는 것 금지
- 초대 발송 전에 인원 확인 없이 바로 발송하는 것 금지
- 기존 반배정/일지 구조 깨뜨리기 금지

이번 작업은
관리자가 학생 이름/학부모 전화번호 엑셀을 업로드해 미등록회원 명단을 만들고,
선생님이 그 같은 명단에서 학생을 가져와 반배정하면 정상회원으로 전환되며,
학부모는 나중에 가입해도 기존 수업/일지 데이터와 자동 연결될 수 있게 만드는 작업이다.
