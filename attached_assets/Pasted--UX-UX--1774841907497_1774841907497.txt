[스윔노트 학부모모드 UX 개선 — 상세 코딩 작업 지시문 (누락 없이 한 번에 적용용)]

지금부터 스윔노트 학부모모드 UX를 전면 개선한다.
이번 작업은 **기능 추가보다 UX 재배치와 콘텐츠 표현 강화가 핵심**이다.
현재 학부모모드는 버튼은 있으나 정보 밀도가 낮고, 실제로 앱을 열었을 때 “지금 확인해야 할 것”이 바로 보이지 않아 비어 보이는 문제가 있다.
따라서 아래 지시대로 **사진/피드백 1순위, 성장 2순위, 출석 3순위** 기준으로 구조를 재설계하라.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
0. 절대 규칙
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

* 기존 코드 절대 통째로 덮어쓰지 말고, 신규 섹션 컴포넌트 분리 후 교체 방식으로 작업
* 기존 학부모 기능 삭제 금지
* 기존 라우트 유지
* 기존 API/DB 구조와 충돌 나지 않게 점검
* 더미 텍스트 남발 금지
* “정보 없음” 문구를 메인 화면에 반복 노출하지 말 것
* 읽음 처리 로직은 반드시 사용자 기준으로 동작하게 구현
* 작업 후 보고서 작성 필수

━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 작업 목표
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━

학부모모드를 “버튼 모음 화면”에서 “상태 + 콘텐츠 + 행동 유도 화면”으로 바꾼다.

핵심 UX 목표:

* 앱을 열자마자 자녀의 최신 상태가 보여야 함
* 사진/피드백이 최우선 노출되어야 함
* 한 번 확인한 알림은 사라져야 함
* 빈 화면처럼 보이지 않게 해야 함
* 스타벅스식으로 “계속 확인할 이유가 있는 홈”으로 바꿔야 함

우선순위:

1. 사진 / 피드백
2. 성장
3. 출석

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
2. 적용 대상 화면
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

최소 아래 화면들을 점검하고 수정한다.

* 학부모 홈
* 자녀 프로필
* 수영정보
* 설정
* 앨범
* 수업일지
* 공지
* 쪽지

우선순위는 학부모 홈 > 자녀 프로필 > 수영정보 순서로 진행한다.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
3. 학부모 홈 화면 전면 개편
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

현재 홈 화면은 상단 자녀 카드 + 아이콘 버튼 + 최신소식 빈 카드 구조인데, 이것을 아래 구조로 재배치한다.

최종 홈 구조 순서:

1. 상단 수영장명 + 알림/설정 버튼
2. 자녀 메인 상태 카드
3. 빠른 바로가기 버튼 그리드
4. 오늘 확인할 것 카드
5. 최신 피드백 카드
6. 최근 사진 카드
7. 최근 공지 카드
8. 성장 카드
9. 출석 카드

화면에 데이터가 없을 경우도 빈 영역처럼 보이지 않도록 처리한다.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
4. 자녀 메인 상태 카드 개편
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

현재 상단 민트 카드에는 자녀 이름과 반 배정 전 정도만 보이는데, 이것을 아래처럼 바꾼다.

표시 항목:

* 자녀 이름
* 현재 반 또는 반 배정 전
* 새 사진 개수
* 새 피드백 개수
* 오늘 수업 여부
* 오늘 수업 시간이 있으면 시간 표시
* 최근 레벨 또는 현재 레벨
* 우측 화살표 유지 (자녀 프로필 진입)

예시 구조:

* 서태웅
* 초급반 / 반 배정 전
* 새 사진 12장
* 새 피드백 1건
* 오늘 16:00 수업 예정
* 현재 레벨 10

표시 규칙:

* 새 사진/새 피드백이 없으면 해당 배지 숨김
* 오늘 수업이 없으면 “오늘 수업 없음” 또는 해당 줄 숨김
* 레벨이 없으면 “레벨 기록 없음” 대신 너무 거슬리는 문구 쓰지 말고 조용한 보조 텍스트 처리

스타일:

* 민트 메인 톤 유지
* 카드 한 장만으로도 “지금 확인할 가치”가 느껴져야 함
* 너무 많은 줄바꿈 금지, 2~4줄 안에서 정리

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
5. 빠른 바로가기 버튼 그리드 개편
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

현재 버튼은 기능명만 보이는데, 반드시 “기능 + 상태 숫자” 구조로 바꾼다.

버튼 유지 후보:

* 수업일지
* 앨범
* 출결
* 쪽지
* 수영정보
* 공지

각 버튼 표기 규칙:

* 수업일지: 새 피드백 n건
* 앨범: 새 사진 n장
* 출결: 이번달 x / y회
* 쪽지: 읽지 않음 n개
* 수영정보: 업데이트 시 badge, 없으면 기본 표시
* 공지: 새 공지 n건

핵심:

* 버튼은 죽어 있는 메뉴가 아니라 상태를 알려주는 카드여야 함
* 숫자 또는 상태 문구가 없는 버튼은 최소화
* 학부모가 자주 보는 순서대로 상단 배치
* 사진/피드백 관련 버튼을 가장 먼저 보이게 배치

권장 배치:
1행: 수업일지 / 앨범 / 출결
2행: 공지 / 쪽지 / 수영정보

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
6. “오늘 확인할 것” 카드 추가
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

학부모 홈에 신규 섹션 추가:
제목: 오늘 확인할 것

표시 대상:

* 읽지 않은 피드백
* 새 사진
* 새 공지
* 읽지 않은 쪽지

출력 예시:

* 수업일지 1건
* 새 사진 12장
* 공지 2건
* 읽지 않은 쪽지 1개

규칙:

* 읽지 않은 항목이 하나도 없으면 카드 자체 숨김
* 있으면 홈 상단부에 보여서 행동 유도
* 각 항목 클릭 시 해당 화면으로 이동 가능하게 구성해도 좋음
* 단순 텍스트 나열보다 짧은 리스트 카드 형태 권장

이 섹션은 꼭 구현한다.
이 카드가 있어야 홈이 살아난다.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
7. 최신 피드백 카드 추가
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

학부모 홈 메인 콘텐츠 1순위다.

표시 내용:

* 가장 최근 수업일지 1건 또는 2건
* 작성일
* 간단 요약
* 선생님 이름 가능하면 표시
* 새 일지 여부 표시

구성 예시:

* 오늘 수업 피드백
* 자유형 호흡 안정됨
* 킥은 조금 더 보완 필요
* 2026-03-30 / 김선생

규칙:

* 너무 긴 일지는 2~3줄 요약 후 더보기
* 피드백이 없으면 카드 전체 숨김 또는 “최근 피드백이 없습니다”를 메인 상단에 두지 말고 조용히 최소 표시
* 새 피드백이 있으면 배지 표시
* 해당 카드 누르면 수업일지 상세로 이동

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
8. 최근 사진 카드 추가
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

학부모 홈 메인 콘텐츠 1순위다. 피드백과 같은 중요도로 다룬다.

표시 내용:

* 최근 사진 썸네일 1~4장
* 새 사진 총 개수
* 최근 업로드 날짜
* 앨범으로 이동 버튼 또는 카드 전체 클릭 이동

규칙:

* 새 사진 있으면 “새 사진 n장” 표시
* 없으면 최근 사진 몇 장이라도 보여서 빈 화면 느낌 줄이기
* 썸네일이 전혀 없으면 앨범 버튼만 덩그러니 두지 말고 카드 숨김 또는 최소 안내형 구성
* 카드 높이 너무 크지 않게

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
9. 최근 공지 카드 추가
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

기존 최신소식 빈 카드 구조를 없애고 공지 중심 카드로 바꾼다.

표시 내용:

* 최근 공지 1~2건 제목
* 등록일
* 새 공지 배지
* “공지 전체보기” 링크

규칙:

* 공지가 없으면 큰 빈 박스 금지
* 공지 없음 상태는 작고 덜 거슬리게 처리
* 공지와 수업일지를 같은 “최신소식”으로 뭉개지 말고 분리해서 보여준다

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
10. 성장 카드 추가
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

우선순위 2순위다.
학부모가 “아이의 변화”를 느끼게 만드는 카드다.

표시 내용:

* 현재 레벨
* 이전 레벨 대비 변화
* 최근 1개월 변화 또는 최근 테스트 결과
* 가능하면 짧은 성장 코멘트

예시:

* 현재 레벨 10
* 최근 1개월 +1 상승
* 자유형 안정도 향상

규칙:

* 그래프가 있으면 좋지만 MVP에서는 숫자형/텍스트형도 가능
* 레벨 데이터 없으면 카드 축소 또는 숨김
* “기록 없음”을 크게 강조하지 말 것

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
11. 출석 카드 추가
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

우선순위 3순위로 하단 배치한다.

표시 내용:

* 이번달 출석 횟수 / 예정 횟수
* 출석률 %
* 최근 출결 상태

예시:

* 이번달 6 / 8회
* 출석률 75%
* 최근 수업 출석 완료

규칙:

* 출석은 정보 제공용이지 메인 감성 카드가 아님
* 너무 위로 올리지 말 것
* 숫자만 나열하지 말고 짧은 상태 문구 함께 제공

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
12. 알림/읽음 처리 로직 구현
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

이 부분은 필수다.
학부모 UX 핵심 규칙은 “한 번 보면 사라지는 구조”다.

원칙:

* 모든 알림은 1회성 소비 구조
* 확인하면 사라짐
* 새 데이터가 들어오면 다시 생성

항목별 규칙:

1. 사진

* 새 사진 업로드 시 unread 상태 생성
* 앨범 진입 시 해당 학생 기준 새 사진 전체 읽음 처리
* 읽음 후 홈의 “새 사진 n장” 제거

2. 피드백(수업일지)

* 새 일지 작성 시 unread 상태 생성
* 해당 일지 또는 수업일지 목록 진입 시 읽음 처리 정책 정리
* 홈의 “새 피드백 n건” 제거

3. 공지

* 새 공지 등록 시 unread 상태 생성
* 공지 상세 진입 시 읽음 처리
* 전체보기 진입 시 전체 읽음 처리 옵션 허용 가능

4. 쪽지

* 새 메시지 수신 시 unread_count 증가
* 대화방 진입 시 읽음 처리
* 홈 배지 제거

5. 출석

* 알림이 아니라 상태 정보
* 읽음 처리 개념 없음

UI 규칙:

* 읽지 않은 항목이 0이면 숫자 배지 숨김
* 상단 카드, 버튼, 오늘 확인할 것 카드 모두 동기화
* 같은 데이터를 여러 군데 표시하더라도 읽음 처리 후 모두 동시에 사라져야 함

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
13. 데이터 구조 / 저장 방식 점검
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

가능하면 아래 방식으로 설계 점검한다.

읽음 처리 테이블 예시:

* photo_reads
* diary_reads
* notice_reads
* message_reads 또는 thread read state

최소 필드:

* user_id
* student_id 또는 content_id
* read_at

핵심:

* 학부모 계정 기준 읽음 처리
* 자녀 여러 명 연결 구조라면 child별 구분 가능해야 함
* 한 자녀만 읽은 상태가 다른 자녀에 섞이지 않게 처리

학부모 계정이 자녀 여러 명 연결되는 구조를 고려해,
모든 unread 계산은 “현재 선택된 자녀 기준”으로 먼저 동작하게 구현한다.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
14. 자녀 프로필 화면 개편
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

현재 자녀 프로필은 기본 정보 / 수영장 정보 / 바로가기 구조인데, 너무 정적이다.
아래 항목을 보강한다.

유지:

* 이름
* 출생연도
* 소속 수영장
* 반 정보
* 바로가기 버튼

추가/개편:

* 현재 레벨 카드 강조
* 최근 피드백 요약 1건 추가
* 최근 출석 상태 1줄 추가
* 최근 사진 개수 1줄 추가
* 반 배정 전 상태는 너무 크게 보이지 않게 정리
* 바로가기 버튼도 상태형 배지 일부 반영

자녀 프로필은 “상세 상태 화면”처럼 보여야 한다.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
15. 수영정보 화면 개편
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

현재 수영정보 화면은 카드가 많지만 대부분 비어 있어서 오히려 허전하다.

대상 섹션:

* 수영장 소개
* 수업료 안내
* 레벨 테스트 안내
* 이벤트/소식
* 수영 용품 소개

수정 규칙:

* 실제 데이터 없는 섹션은 숨김 또는 축소
* “정보가 등록되면 여기에 표시됩니다” 같은 문구를 여러 카드에 반복 노출 금지
* 기본 정보가 있으면 그 카드만 노출
* 수영장 소개, 수업료, 레벨 테스트 정도는 기본 텍스트라도 채워서 최소 정보성 확보
* 이벤트/용품은 없으면 과감히 숨김

즉, 없는 내용을 빈 카드로 여러 개 보여주지 말 것.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
16. 설정 화면 보완
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

현재 설정 화면은 기본 구조가 괜찮지만 아래를 보완한다.

보완 항목:

* 상단에 현재 선택 자녀 표시
* 자녀 여러 명이면 자녀 전환 진입 강조
* 알림 설정 상태 요약 표시 가능하면 추가
* 앱 버전, 로그아웃 영역은 하단 유지
* 너무 많은 메뉴를 늘리지 말 것

설정은 복잡하게 만들지 말고, 학부모가 계정/자녀/공지/알림을 다루는 곳으로 명확히 유지한다.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
17. UI 스타일 가이드
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

스타일은 지금 쓰는 민트/네이비/포인트 컬러 체계를 유지하되,
스타벅스식 UX 느낌을 위해 다음을 반영한다.

* 카드마다 역할이 분명해야 함
* 같은 흰 카드 반복으로 지루하지 않게 정보 밀도 차등 배치
* 상단 핵심 카드 > 중간 액션 버튼 > 하단 콘텐츠 카드 순으로 리듬감 만들기
* 숫자/배지를 적극 사용
* “없음” 메시지는 작고 덜 거슬리게
* 그림만 큰 빈 박스 금지
* 아이콘 색과 의미를 맞추기
* 메인 홈에서는 사진/피드백 카드가 시각적으로 가장 눈에 띄게

주의:

* 디자인만 화려하게 하지 말고, 실제 데이터가 들어왔을 때 예쁘게 보이게 구성
* 컨텐츠 없는 상태도 너무 휑하지 않게 최소 안내 구성

━━━━━━━━━━━━━━━━━━━━━━━━━━━━

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
19. 상태별 예외 처리
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

반드시 점검:

* 자녀 0명 연결 상태
* 자녀 1명 연결 상태
* 자녀 여러 명 연결 상태
* 반 배정 전
* 레벨 데이터 없음
* 사진 없음
* 일지 없음
* 공지 없음
* 쪽지 없음
* 오늘 수업 없음

각 상태에서 홈이 깨지거나 너무 비어 보이지 않아야 한다.

특히 자녀 0명일 때는 연결 요청/등록 유도 구조가 필요하고,
자녀 여러 명일 때는 현재 선택 자녀 기준으로 홈 내용이 바뀌어야 한다.

23. 최종 목표
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

* 학부모 홈이 더 이상 비어 보이지 않게 만들 것
* 사진/피드백이 가장 먼저 보이게 할 것
* 성장과 출석이 보조 정보로 자연스럽게 따라오게 할 것
* 읽음 처리 후 배지가 사라지는 구조를 완성할 것
* 자녀 여러 명 연결 구조에서도 문제없이 동작하게 할 것
* 현재 MVP 수준에서 실사용 가능한 학부모 UX로 끌어올릴 것

작업 시작 전 현재 학부모모드 관련 파일 구조와 데이터 흐름 먼저 분석하고,
작업 후 반드시 전체 동작 테스트와 보고서까지 마무리하라.
