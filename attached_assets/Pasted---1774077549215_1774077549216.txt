지금부터 앱 전체의 뒤로가기/팝업/페이지 복귀 방식을 전면 재정의해서 수정해줘.
특히 선생님모드 스케쥴러에서 반정보를 보고 다시 돌아왔을 때 터치가 멈추는 버그를 최우선으로 해결해줘.

이번 작업은 단순 버그 수정이 아니라
`페이지 + 팝업 + 아이콘 진입 경로를 전부 기억하는 뒤로가기 시스템`을 만드는 작업이다.

==================================================
[1. 최우선 버그 수정]
==================================================

문제:
- 선생님모드에서 스케쥴러 진입
- 날짜 눌러 상세 팝업 열기
- 반 정보 항목을 눌러 반 정보 페이지 이동
- 다시 스케쥴러로 돌아오면 터치가 멈추고 앱이 작동 안 함

해결 요구:
1. 반 정보 페이지에서 뒤로가기/복귀 후 overlay, pointerEvents, modal backdrop, absolute layer가 남지 않게 해줘
2. 팝업 상태로 복귀할 때 invisible layer가 화면을 막지 않게 정리
3. 스케쥴러 복귀 후 터치/스크롤/버튼이 정상 작동해야 함
4. 반 정보 카드 오른쪽 화살표도 정상 클릭되게 수정
5. 현재는 화살표가 안 눌리는데, 카드 전체 클릭 또는 화살표 클릭 모두 반 정보 페이지로 이동하게 해줘

==================================================
[2. 스케쥴러 -> 반정보 -> 뒤로가기 규칙]
==================================================

반 정보를 어디서 눌렀는지에 따라 복귀 위치가 달라져야 한다.

[경우 A]
홈화면에서 스케쥴러 관련 반 목록(홈 위쪽 카드/리스트)에서 반 정보를 본 경우

흐름:
홈화면
→ 홈 위 반 목록 화면/상태
→ 반 정보 페이지

이 경우 뒤로가기:
- 반 정보 페이지 뒤로가기
→ 홈화면 위에 있는 반 목록 상태로 복귀
- 거기서 사용자가 직접 닫거나 빠져나오면
→ 홈 기본 상태로 복귀

즉 홈에서 들어간 반 정보는 뒤로가기를 했을 때 홈 반 목록 상태로 먼저 돌아와야 한다.

[경우 B]
스케쥴러 월간 달력에서 날짜를 눌러 팝업 열고 그 안에서 반 정보를 본 경우

흐름:
스케쥴러 월간 화면
→ 날짜 팝업 열림
→ 반 정보 페이지

이 경우 뒤로가기:
- 반 정보 페이지 뒤로가기
→ 무조건 바로 월간 스케쥴러로 가지 말고
→ 방금 열어놨던 `같은 날짜 팝업 상태`로 먼저 복귀

그 다음:
- 사용자가 날짜 팝업을 닫으면
→ 월간 스케쥴러 화면이 보이게 해줘

즉 팝업에서 들어간 페이지는 뒤로가기로 나오면 팝업으로 돌아와야 한다.

==================================================
[3. 뒤로가기 시스템 공통 규칙 재정의]
==================================================

앱 전체에 아래 규칙을 통일 적용해줘.

핵심 원칙:
`사용자가 지나온 경로를 뒤로가기 스택으로 기억하고, 팝업도 그 경로에 포함한다`

규칙:
1. 아이콘을 눌러 진입한 경우
- 그 아이콘 진입 화면도 경로에 포함
- 뒤로가기를 계속 하면 해당 아이콘 선택 진입 상태까지 돌아가야 함
- 사용자가 그 아이콘 선택 상태를 닫거나 나와야 비로소 그 이전 경로로 감

2. 팝업에서 페이지로 넘어간 경우
- 페이지에서 뒤로가기를 하면 무조건 팝업으로 복귀
- 팝업을 직접 닫아야 그 다음 뒤로가기 경로로 넘어감

3. 팝업은 전부 `뒤로가기 경로에 포함`
- 단순 오버레이가 아니라 navigation history 일부로 취급

4. 사용자가 홈버튼을 누른 경우
- 홈버튼을 누르는 순간 기존 뒤로가기 경로를 초기화
- 그 뒤 새로 이동한 경로부터 다시 기록 시작

5. 뒤로가기 버튼이 없는 최상위 화면까지 도달한 경우
- 그 시점에서 뒤로가기 경로 초기화
- 이후 새 이동부터 다시 기록

즉:
- 경로 기억은 계속 유지
- 홈버튼/최상위 진입점 도달 시 초기화

==================================================
[4. 팝업 포함 네비게이션 규칙]
==================================================

팝업은 단순 시각 요소가 아니라 navigation stack처럼 관리해줘.

예시 1:
월간 스케쥴러
→ 21일 팝업 열기
→ 반 정보 페이지
→ 뒤로가기
= 21일 팝업 복귀
→ 팝업 닫기
= 월간 스케쥴러

예시 2:
홈
→ 수업관리 아이콘
→ 반 목록 팝업/중간 화면
→ 반 정보 페이지
→ 뒤로가기
= 반 목록 상태 복귀
→ 다시 뒤로가기/닫기
= 홈

즉:
- page push 뿐 아니라
- modal open도 stack entry로 취급

==================================================
[5. 구현 방식 요구]
==================================================

단순히 goBack()만 쓰지 말고 경로 기억 시스템을 만들어줘.

가능하면 아래 개념으로 정리:
- route stack
- modal stack
- source context (어디서 들어왔는지)
- goBackSafe()
- closeModalAndStay()
- goHomeAndResetHistory()

필요한 저장 정보 예시:
- currentRole
- currentRoute
- previousRoute
- openModalType
- modalSourceRoute
- iconEntrySource
- schedulerSelectedDate

특히 스케쥴러에서는
- 어떤 날짜 팝업이 열려 있었는지
- 그 상태에서 어떤 반 정보를 눌렀는지
를 기억해줘야 함

==================================================
[6. 화살표 클릭 버그 수정]
==================================================

현재 날짜 팝업 안 수업 카드 오른쪽 화살표가 안 눌리는 문제 수정

수정 규칙:
1. 카드 전체 클릭 가능
2. 오른쪽 화살표 클릭도 가능
3. 둘 다 동일하게 반 정보 페이지로 이동
4. 클릭 영역이 겹치거나 overlay에 막히지 않게 수정

==================================================
[7. 홈/아이콘 진입 규칙]
==================================================

홈화면이나 아이콘을 눌러서 들어간 경우,
그 진입 단계도 뒤로가기 경로에 포함시켜줘.

예:
홈
→ 수업관리 아이콘 클릭
→ 반 목록/중간 화면
→ 반 정보 페이지

뒤로가기:
반 정보 페이지
→ 반 목록/중간 화면
→ 홈

즉 아이콘 눌러서 들어간 첫 진입 상태도 무조건 스택에 포함

==================================================
[8. 전체 모드 통일]
==================================================

이 뒤로가기/팝업 경로 기억 방식은 선생님모드만이 아니라 전체 모드에 통일 적용해줘.

적용 대상:
- 학부모
- 선생님
- 관리자
- 슈퍼관리자(현재 구현 범위)

동일 규칙:
- 페이지 이동은 경로에 포함
- 팝업 열기/닫기도 경로에 포함
- 홈 버튼 누르면 히스토리 초기화
- 최상위 화면 도달 시 히스토리 초기화

==================================================
[9. 강력 검증 요구]
==================================================

이번 작업은 반드시 실제로 눌러서 검증해줘.
수정했다고만 하지 말고 아래 항목 하나씩 확인하고 결과를 보고해줘.

검증 항목:
1. 선생님모드 스케쥴러에서 날짜 팝업 열기
2. 반 정보 카드 클릭
3. 반 정보 페이지 이동
4. 뒤로가기
5. 같은 날짜 팝업으로 복귀하는지 확인
6. 팝업 상태에서 터치가 멈추지 않는지 확인
7. 팝업 닫기
8. 월간 스케쥴러 정상 동작 확인

9. 홈화면 반 목록에서 반 정보 진입
10. 뒤로가기 시 홈 위 반 목록 상태로 복귀하는지 확인
11. 다시 닫으면 홈으로 복귀하는지 확인

12. 카드 오른쪽 화살표 클릭 가능한지 확인
13. 카드 전체 클릭도 가능한지 확인

14. 홈버튼 누른 뒤 히스토리 초기화되는지 확인
15. 최상위 화면까지 갔을 때 히스토리 초기화되는지 확인

==================================================
[10. 보고 형식]
==================================================

작업 후 아래처럼 보고해줘.

- 스케쥴러 반 정보 복귀 후 터치 멈춤: 해결/미해결
- 날짜 팝업 -> 반 정보 -> 뒤로가기 -> 날짜 팝업 복귀: 통과/문제
- 홈 반 목록 -> 반 정보 -> 뒤로가기 -> 홈 반 목록 복귀: 통과/문제
- 카드 화살표 클릭: 통과/문제
- 카드 전체 클릭: 통과/문제
- 홈버튼 히스토리 초기화: 통과/문제
- 최상위 화면 히스토리 초기화: 통과/문제

문제가 남아 있으면 어느 화면, 어느 경로에서 남는지 정확히 적어줘.

==================================================
[11. 최종 목표]
==================================================

이 앱은 페이지 이동과 팝업 이동이 섞여 있기 때문에,
뒤로가기 시스템을 단순 페이지 기준이 아니라
`사용자가 지나온 실제 경로 전체`를 기준으로 재구성해야 한다.

최종 목표:
- 팝업도 뒤로가기 경로에 포함
- 페이지도 뒤로가기 경로에 포함
- 홈/아이콘 진입점도 경로에 포함
- 홈버튼/최상위 도달 시만 히스토리 초기화
- 어디서 들어갔든 뒤로가기를 누르면 사용자가 방금 있던 구조로 자연스럽게 돌아가게 해줘
