지금부터 선생님 모드 > 수업 > 내 반 > 반 선택 팝업 안의 “반배정” 기능을 수정하라.

현재 문제:
- 미등록회원 50명이 있어도 반배정 버튼을 누르면 학생이 보이지 않음
- 선생님이 실제로 학생을 반에 넣을 수 없는 상태임

이번 작업 목표는
반배정 버튼을 누르면
토이키즈 계정의 배정 가능한 학생들이 모두 리스트에 뜨게 하고,
학생별 주횟수(주 1회/2회/3회)에 따라 남은 배정 가능 횟수만큼 계속 반배정 리스트에 남아 있게 만드는 것이다.

가장 중요:
- 선생님은 관리자와 연동된 같은 학생 명단을 봐야 한다
- 반배정 대상은 “완전 미배정 학생만”이 아니라
  “추가 반 배정이 가능한 학생 전체”여야 한다
- 주 2회/3회 학생은 일부 반을 배정해도 남은 횟수만큼 계속 리스트에 남아야 한다
- 같은 반 중복 배정 금지
- 기존 수업/출결/일지 구조 깨지지 말 것

==================================================
[1. 반배정 대상 정의 수정]

반배정 버튼을 누르면 아래 학생들이 리스트에 보여야 한다.

1. 아직 반이 하나도 없는 학생
2. 이미 반이 일부 배정되었지만, 주횟수만큼 다 배정되지 않은 학생

즉:
배정 대상 조건은 아래로 한다.

조건:
assigned_class_count < weekly_target_count

또는
weekly_target_count가 아직 설정되지 않았다면
초기 반배정 대상에 포함

중요:
- 미등록회원도 포함
- 정상회원이지만 배정 미완료인 학생도 포함
- 완전히 다 배정된 학생만 목록에서 제외

==================================================
[2. 관리자/선생님 학생 명단 연동]

선생님 반배정 리스트는
관리자와 연동된 같은 토이키즈 학생 명단을 사용해야 한다.

즉:
- 관리자에서 만든 테스트 회원 50명이 선생님 반배정 리스트에 보여야 한다
- 별도 선생님 전용 가짜 리스트 금지
- 토이키즈 계정 기준 학생 데이터를 그대로 공유

==================================================
[3. 반배정 팝업에서 보여줄 학생 정보]

학생 리스트 카드에는 최소 아래 정보 표시

- 학생 이름
- 학부모 전화번호 또는 상태
- 현재 배정 수 / 주횟수

예:
서태웅
010-1000-0001
0 / 2 배정

아리
010-1000-0002
1 / 3 배정

weekly_target_count가 아직 없으면
예:
- 주횟수 미설정
으로 표시 가능

==================================================
[4. 첫 반배정 시 주횟수 설정]

학생을 처음 반에 넣을 때
아직 주횟수(weekly_target_count)가 설정되지 않았다면
반배정 전에 주횟수를 먼저 선택하게 하라.

선택 팝업:
- 주 1회
- 주 2회
- 주 3회

선택 후:
- weekly_target_count 저장
- 현재 반에 학생 추가
- assigned_class_count = 1로 계산

중요:
- 최초 1회만 설정
- 이미 주횟수 설정된 학생은 다시 묻지 않음

==================================================
[5. 배정 후 리스트 잔류 규칙]

배정 후 아래처럼 처리하라.

1. 학생을 현재 반에 추가
2. 현재 학생의 배정 반 수 재계산
3. 배정 반 수 < 주횟수 이면
   → 반배정 리스트에 계속 남김
4. 배정 반 수 == 주횟수 이면
   → 반배정 리스트에서 제거

예:
주 2회 학생
- 1개 반 배정 후: 계속 남음
- 2개 반 배정 후: 제거

주 3회 학생
- 1개 배정 후 남음
- 2개 배정 후 남음
- 3개 배정 후 제거

==================================================
[6. 중복 배정 방지]

반드시 막아야 할 것:
- 같은 학생이 같은 반에 두 번 배정되는 것

규칙:
현재 반에 이미 배정된 학생은
반배정 리스트에서는 보이더라도
현재 반에 추가 버튼 비활성화하거나
“이미 이 반에 배정됨” 처리할 것

또는
리스트 자체에서 현재 반 배정 학생은 제외 가능

중요:
같은 반 중복 배정 절대 금지

==================================================
[7. 반배정 리스트 검색 기능]

반배정 팝업 상단에 검색창 추가

검색 기준:
- 학생 이름
- 학부모 전화번호 일부

입력 즉시 필터링
검색어 비우면 전체 복원

==================================================
[8. 반배정 UI 동작]

현재 반 예: 월 13:00반

반배정 버튼 클릭
→ 반배정 가능 학생 리스트 표시

학생 선택 시:
- 주횟수 미설정이면 주횟수 선택 팝업
- 주횟수 설정돼 있으면 바로 배정 확인 팝업

배정 확인 팝업 예:
- 이 학생을 현재 반에 배정하시겠습니까?

배정 완료 후:
- 현재 반 학생 목록 즉시 반영
- 반배정 리스트도 즉시 갱신
- 남은 배정 가능 횟수 있으면 리스트에 계속 남음

==================================================
[9. 데이터 구조 반영]

반드시 관리할 값:

- student_id
- weekly_target_count (주 1회/2회/3회)
- assigned_class_ids 또는 assigned schedule slots
- assigned_class_count
- registration_status
- parent_phone
- parent_link_status

중요:
assigned_class_count는 실제 배정된 반 개수 기준으로 계산

==================================================
[10. 미등록회원과 반배정 관계]

미등록회원이어도 선생님이 반배정 가능해야 한다.

즉:
- 학부모 가입 전이어도 반배정 가능
- 반배정 후 출결/일지 작성 가능
- 학부모는 가입 전까지 열람 불가
- 나중에 가입 시 기존 데이터 연결

반배정 완료 후:
- 학생은 정상회원 또는 배정완료 상태로 전환 가능
- 단 부모 가입 여부와는 별개

==================================================
[11. 기존 기능 유지]

반드시 유지:
- 출결 버튼 기능
- 수업일지 버튼 기능
- 미배정 버튼 기능
- 반 이동 기능
- 주간 스케줄표
- 현재 반 학생 목록 표시
- 기존 메신저 자동 기록 구조

이번 작업은
반배정 대상과 배정 횟수 로직을 제대로 잡는 것이다

==================================================
[12. 먼저 확인할 것]

수정 전에 아래를 먼저 파악하라.

1. 선생님 수업 탭 파일
2. 반 선택 팝업 파일
3. 반배정 팝업 파일
4. 관리자/선생님 공용 학생 데이터 구조
5. 학생 반 배정 로직
6. 주횟수 저장 구조
7. assigned_class_count 계산 로직
8. 검색 가능한 학생 리스트 컴포넌트

==================================================
[13. 테스트 - 반드시 실제 수행]

테스트 1.
토이키즈 테스트 회원 50명이 반배정 리스트에 보이는지 확인

테스트 2.
반이 하나도 없는 학생이 리스트에 보이는지 확인

테스트 3.
주횟수 미설정 학생 선택 시 주 1/2/3회 선택 팝업이 뜨는지 확인

테스트 4.
주 2회 학생을 1개 반에 배정한 후 리스트에 계속 남는지 확인

테스트 5.
주 2회 학생을 2개 반에 배정하면 리스트에서 사라지는지 확인

테스트 6.
주 3회 학생을 1개/2개 반에 배정한 뒤 계속 남는지 확인

테스트 7.
주 3회 학생을 3개 반에 배정하면 사라지는지 확인

테스트 8.
같은 학생이 같은 반에 중복 배정되지 않는지 확인

테스트 9.
반배정 후 현재 반 학생 목록에 즉시 반영되는지 확인

테스트 10.
검색창으로 학생 이름 검색이 정상 동작하는지 확인

테스트 11.
기존 출결/수업일지/반 이동/미배정 기능이 깨지지 않았는지 확인

==================================================

작업 후 아래 순서로 보고하라.

6. 중복 배정 방지 방식
7. 테스트 1~11 결과
8. 아직 남은 예외 상황이 있으면 명시

[15. 절대 금지]

- 미등록회원이 있는데 반배정 리스트가 비어 있는 상태 금지
- 주 2회/3회 학생이 일부 배정 후 리스트에서 사라지는 것 금지
- 같은 반 중복 배정 허용 금지
- 주횟수 미설정 상태를 처리하지 않고 배정 막히는 것 금지
- 기존 출결/일지/반 이동 구조 깨뜨리기 금지

이번 작업은
반배정 버튼을 눌렀을 때 배정 가능한 학생이 제대로 보이게 하고,
학생별 주횟수에 따라 남은 횟수만큼 계속 반배정 리스트에 남아 있어
선생님이 여러 반에 차례대로 배정할 수 있게 만드는 작업이다.
