[스윔노트 출시 직전 무한 점검/의심 항목 전수 검증/즉시 수정 지시서]

지금부터 스윔노트 전체 프로젝트에 대해
“조금이라도 의심되면 바로 확인하고, 문제면 즉시 수정하고, 수정 후 다시 같은 흐름을 반복 테스트하는 방식”으로
무한 점검 모드에 들어간다.

이번 작업은 단순 기능 추가가 아니다.
출시 직전 기준으로
- mock 잔재 제거
- 플랜/결제/정산 실데이터 정합성 확인
- 네비게이션/팝업/뒤로가기 흐름 확인
- 권한/상태/카운트/집계 정확성 확인
- iPhone Expo Go 기준 UI/터치/멈춤 확인
을 전부 반복 점검하는 작업이다.

중요:
- “조금이라도 의심되면 확인”이 원칙이다
- 애매하면 넘어가지 말고 반드시 확인
- 화면이 맞아 보여도 실제 데이터가 mock이면 실패
- 한 군데 고치면 관련된 다른 화면까지 무조건 회귀 테스트
- 확인 안 한 것은 완료라고 하지 마라
- 실제 클릭/저장/상태변경/카운트변경/로그생성까지 검증해야 완료로 인정
- 거짓 완료 보고 절대 금지

==================================================
0. 이번 점검의 최우선 목표
==================================================

1. 슈퍼관리자 화면에 남아 있는 mock/목데이터를 실데이터로 교체
2. 300명 플랜의 growth 잔재를 전부 제거하고 ADVANCE로 완전 통일
3. 플랜/구독/결제/정산/환불 흐름을 하나의 기준으로 맞추기
4. 의심되는 모든 연결/카운트/상태/화면흐름을 전부 재검증
5. 그 후 전체 운영 시뮬레이션으로 넘어갈 준비 완료

==================================================
1. 1차 집중 점검 대상 (무조건 먼저)
==================================================

A. super/subscriptions.tsx
- 현재 Zustand/목데이터 기반이면 실 API 연동으로 교체
- 슈퍼관리자가 실제 수영장별 구독 상태를 볼 수 있어야 함
- 실제 구독 테이블/구독 API/플랜 정의와 100% 일치해야 함

B. super/more.tsx
- 결제이슈 / 저장위험 / 각종 경고 배지가 mock이면 실데이터로 교체
- 운영에 바로 쓰는 화면이므로 가짜 뱃지 금지

C. 플랜명 growth 잔재
- growth, GROWTH, 성장형 등 과거 300명 플랜 이름 잔재 전수 검색
- 화면/정산/로그/필터/팝업/리포트 전부 ADVANCE로 통일

==================================================
2. 플랜/결제/정산 전수 검증
==================================================

다음 항목은 하나라도 의심되면 전부 재확인:

- 슈퍼관리자 구독 상품 설정 화면
- 수영장 관리자 구독관리 화면
- revenue_logs / settlement_logs / subscriptions 데이터 구조
- 플랜 카드 표시값
- 무료 플랜 값
- 회원수 제한 값
- 저장공간 값
- 최초 50% 할인 표시값
- 환불 상태 반영
- 업그레이드/다운그레이드 기록
- 스토어 수수료 계산

반드시 확인할 최종 기준:
- FREE 5명 / 100MB / 0원
- 스타터 30명 / 600MB / 2900원
- 베이직 50명 / 1GB / 3900원
- 스탠다드 100명 / 5GB / 9900원
- 어드밴스 300명 / 20GB / 29000원
- 프로 500명 / 40GB / 59000원
- 맥스 1000명 / 100GB / 99000원
- 엔터프라이즈 2000 / 250GB / 179000원
- 엔터프라이즈 3000 / 400GB / 249000원

의심 체크 포인트:
- 스타터가 614MB처럼 잘못 보이는지
- 무료 플랜이 10명 / 512MB 같은 과거값인지
- 유료100 / 유료300 같은 과거 이름이 남아 있는지
- growth가 실제로 노출되거나 내부 표시명으로 남아 있는지
- 엔터프라이즈가 무료/무제한 등 과거 mock 잔재가 있는지

==================================================
4. 슈퍼관리자 화면 실데이터 검증
==================================================

슈퍼관리자 화면은 아래를 실제 데이터로 보여야 한다:

- 수영장 목록
- 현재 구독 플랜
- 결제 상태
- 결제 실패 수
- 저장공간 위험 수
- 매출 합계
- 플랜별 결제 건수
- 환불 건수
- 실수익 추정값

점검:
- 수영장 관리자에서 결제/플랜 변경하면 슈퍼관리자 화면에 즉시 또는 재조회 시 반영되는지
- 수영장 이름/plan_id/plan_name 누락 없는지
- 결제는 됐는데 슈퍼에서 안 보이는 케이스 없는지

==================================================
5. 상태/카운트/집계 실시간 반영 점검
==================================================

다음은 전부 “즉시 또는 재조회 시 정확히 반영”되어야 함:

- 미배정 카운트
- 학부모 미연결 카운트
- 승인대기 카운트
- 회원수 사용량
- 저장공간 사용량
- 결제 상태
- 읽기모드 상태
- 삭제 예약 상태
- 환불 건수
- 플랜별 구독 수

의심되면 확인:
- 관리자와 선생님 화면이 같은 기준으로 계산하는지
- 카운트가 캐시 때문에 늦게 바뀌는지
- 저장/승인/반배정/반해제 후 즉시 반영되는지

==================================================
6. 네비게이션/팝업/뒤로가기 무한 점검
==================================================

다음 원칙 위반이 있는지 의심되면 바로 확인:

- 상세에서 뒤로가면 홈으로 튀는지
- 팝업 닫으면 직전 화면이 아니라 홈으로 가는지
- 바텀시트가 다음 화면 뒤에 남는지
- 저장 후 첫 화면으로 리셋되는지
- 수업설정/운영설정 버튼이 잘못 연결되는지
- 반배정 진입 시 이전 팝업 잔상이 남는지
- 아이폰에서 터치가 막히는 invisible overlay가 있는지

특히:
- 관리자 모드
- 선생님 모드
- 스케줄러
- 회원 관리
- 수업설정
- 운영설정
- 구독/결제
를 집중 점검

7. iPhone Expo Go 우선 점검
==================================================

웹에서만 잘 되는 건 의미 없다.
반드시 iPhone Expo Go 기준으로 아래를 점검:

- 월간/주간/일간 스케줄러 멈춤 여부
- 터치 안 먹는 화면 여부
- 팝업/바텀시트 겹침 여부
- 리스트 스크롤 이상 여부
- 홈 버튼/뒤로가기 흐름
- 카드/배지/칩 겹침
- safe area 침범
- 긴 텍스트 잘림

조금이라도 느리거나 멈추면:
- 원인 추적
- 상태 루프/useEffect 반복 여부 확인
- overlay/pointerEvents 확인
- 리스트 최적화 확인

==================================================
8. 권한/상태 분리 검증
==================================================

절대 섞이면 안 되는 것:
- 미배정 vs 학부모 미연결
- pending 승인대기 vs active 미배정
- 관리자 직접 등록 vs 선생님 등록요청
- 관리자 결제 메시지 vs 선생님/학부모 제한 메시지
- 읽기모드 vs 정상모드

반드시 확인:
- 미배정인데 미연결일 수 있음
- 반배정 되었는데 미연결일 수 있음
- pending은 미배정 카운트에 섞이면 안 됨
- 선생님 등록요청은 active 즉시 생성되면 안 됨

9. 전체 운영 시뮬레이션 준비 점검
==================================================

전체 시뮬레이션에 들어가기 전 아래가 준비되어 있어야 함:

- 관리자 직접등록 가능
- 선생님 등록요청 가능
- 승인대기 처리 가능
- 반배정/반해제 정상
- 미배정/미연결 분리 정상
- 일지/사진/보강 정상
- 구독/결제/정산 화면 실데이터 기반
- 읽기모드 정책 정상

이 중 하나라도 불안하면 먼저 수정하고 넘어간다.

  → 관리자/선생님/홈 카드/목록/필터 전부 재확인
