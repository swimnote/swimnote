수업 일지 화면을 조회 전용 인덱스 화면이 아니라 “미작성 일지 작업 화면”으로 변경한다.
기존 일지 조회 기능은 유지하되, 미작성 일지를 바로 찾아서 작성할 수 있는 구조를 추가한다.

=====================================
[핵심 목표]

1. 수업 일지 화면 상단에 “일지 작성” 버튼 추가
2. 기존 요일별/시간별 인덱스 중심 구조를 미작성 일지 리스트 중심으로 보강
3. 미작성 일지는 날짜 순서로 정렬
4. 가장 오래된 미작성부터 최신 미작성까지 순차 리스트업
5. 리스트에서 선택하면 바로 일지 작성 페이지로 이동
6. 기존 일지 조회 기능은 유지하되, 작성이 우선인 구조로 변경

=====================================
[상단 구조 수정]

현재 수업 일지 화면 상단 구조:
- 검색창
- 요일 필터
- 시간 필터
- 건수 표시

이 구조를 아래처럼 수정한다.

상단 헤더 아래:
- 우측 또는 검색창 아래에 “일지 작성” 버튼 추가

버튼명:
- 일지 작성

규칙:
- 일지 작성 버튼은 항상 보여야 한다
- 눌렀을 때 “미작성 일지 리스트 화면” 또는 같은 화면의 작성 모드로 진입
- 기존 일지 조회 리스트와 분리된 작업 흐름 제공

=====================================
[일지 작성 버튼 동작]

일지 작성 버튼 클릭 시
미작성 일지를 날짜 기준으로 정렬한 리스트를 보여준다.

이 리스트는 기존의
- 요일별 분류
- 시간별 분류
방식이 아니라

반드시 아래 기준으로 정렬한다:
- 지난 날짜 미작성 먼저
- 최신 미작성은 아래쪽
- 즉, 오래된 미작성 → 최근 미작성 순

정렬 기준:
오름차순
YYYY-MM-DD 오래된 날짜부터 정렬

=====================================
[미작성 일지 리스트 규칙]

리스트에는 “아직 일지를 쓰지 않은 수업”만 보여준다.

표시 항목:
- 수업 날짜
- 요일
- 수업 시간
- 반 이름
- 선생님 이름(필요 시)
- 출결 상태 또는 수강생 수
- 작성 상태 배지: 미작성

예시:
- 2026-03-12 (목) / 화13반 / 13:00 / 미작성
- 2026-03-13 (금) / 금18반 / 18:00 / 미작성
- 2026-03-14 (토) / 토10반 / 10:00 / 미작성

리스트는 카드형 또는 행형으로 보이되
눌렀을 때 바로 해당 수업 일지 작성 페이지로 이동해야 한다.

=====================================
[작성 진입 규칙]

미작성 일지 리스트에서 항목 클릭 시
바로 해당 날짜 + 반 + 시간 기준의 일지 작성 화면으로 이동한다.

기준 키:
- classId
- lessonDate
- lessonTime or scheduleSlotId

즉,
반 + 날짜 + 시간
기준으로 해당 일지 작성 페이지를 정확히 열어야 한다.

=====================================
[기존 조회 기능 유지]

기존 일지 조회 기능은 삭제하지 않는다.
다만 화면의 우선순위를 아래처럼 바꾼다.

1. 일지 작성 버튼
2. 미작성 일지 리스트
3. 기존 조회/검색 기능

즉, 지금 화면은 “일지 없는 빈 화면”이 아니라
“작성할 일지가 있으면 바로 쓸 수 있는 작업 화면”이 되어야 한다.

=====================================
[필터 구조 변경]

미작성 일지 작성 화면에서는
요일별 분류를 우선 사용하지 않는다.

요일 필터 / 시간 필터는 조회용으로만 유지 가능하지만
미작성 작성 화면에서는 기본 정렬 기준을 아래로 고정한다.

- 날짜 오름차순
- 오래된 미작성 우선
- 같은 날짜 안에서는 시간순 정렬

정렬 예:
2026-03-10 10:00
2026-03-10 13:00
2026-03-11 15:00
2026-03-12 18:00

=====================================
[빈 상태 문구 변경]

현재처럼 “일지가 없습니다 / 아직 작성된 수업 일지가 없습니다”만 보여주지 말고
상황에 따라 빈 상태 문구를 나눈다.

1. 작성된 일지도 없고 미작성도 없을 때
- 표시:
  작성할 일지가 없습니다.

2. 조회 결과가 없을 때
- 표시:
  조건에 맞는 일지가 없습니다.

3. 미작성 일지가 있을 때
- 빈 화면 대신 리스트를 보여준다

=====================================
[추가 UI 제안]

미작성 일지 리스트 상단에 작은 요약 바 추가:
- 미작성 N건

예:
- 미작성 7건

그리고 정렬 문구 표시:
- 오래된 순

=====================================
[공통 데이터 규칙]

미작성 일지 여부 판단 기준:
- 수업은 존재함
- 출결 또는 수업 슬롯은 존재함
- 해당 슬롯에 연결된 일지 데이터가 없음

즉 아래 조건이면 미작성으로 본다.
- scheduled class exists
- lesson slot exists
- journal record is null or empty

=====================================
[구현 구조]

기존 수업 일지 화면에 아래 추가:

1. 상단 버튼
- WriteJournalButton

2. 미작성 리스트 데이터 selector
- getUnwrittenLessonJournals()
- return: 반 + 날짜 + 시간 기준 배열

3. 정렬 함수
- sort by lessonDate asc
- same date sort by lessonTime asc

4. 리스트 UI
- UnwrittenJournalList
- UnwrittenJournalCard

5. 클릭 시 이동
- navigate('JournalWritePage', { classId, lessonDate, lessonTime })

=====================================
[관리자/선생님 공통 원칙]

선생님 모드에서는 특히 미작성 일지 작업이 빨라야 하므로
이 화면을 조회보다 작성 중심으로 바꾼다.

관리자 모드에도 유사 구조가 있다면 데이터 기준은 동일하게 쓰되
선생님 모드 우선으로 UX를 맞춘다.

=====================================
[필수 테스트]

1. 상단에 일지 작성 버튼이 보이는지
2. 버튼 클릭 시 미작성 일지 리스트가 뜨는지
3. 리스트가 오래된 날짜부터 정렬되는지
4. 같은 날짜 내에서 시간순 정렬되는지
5. 항목 클릭 시 해당 수업 일지 작성 페이지로 바로 이동하는지
6. 작성 완료 후 해당 항목이 미작성 리스트에서 사라지는지
7. 전체 건수 표시가 즉시 갱신되는지
8. 기존 조회/검색 기능이 유지되는지
9. 미작성이 없을 때 빈 상태 문구가 자연스럽게 나오는지
