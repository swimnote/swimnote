[스윔노트 푸시 시스템 전체 구현 + 전모드 연동 + 테스트 강제 포함 최종 지시서]

이 작업은 MVP 핵심 기능이다.
단순 UI 구현이 아니라 실제 발송 로직, 예약 로직, 권한 구조, ON/OFF 제어까지
모든 기능이 “실제로 작동”해야 완료로 인정한다.

작업 후 반드시:
- 모든 푸시 트리거 직접 테스트
- 예약 푸시 시간 테스트
- ON/OFF 정상 작동 확인
- 중복 발송 없는지 확인
- 각 모드별 정상 수신 확인
→ 완료 보고 시 테스트 결과 필수 제출

거짓 완료 보고 금지.

==================================================
1. 전체 구조 개요
==================================================

푸시는 3개 대상만 먼저 구현한다.

- 학부모
- 선생님
- 수영장 관리자

슈퍼관리자는 이번 작업에서 제외.

==================================================
2. 푸시 발송 항목 (확정)
==================================================

[학부모]
- 공지사항 (즉시)
- 수업 전날 알림 (기본: 20시)
- 수업 당일 알림 (기본: 1시간 전)
- 일지 등록 (즉시)
- 사진 업로드 (일지 없을 경우만 즉시)

[선생님]
- 오늘 수업 일정 알림
- 학부모 쪽지 도착
- 메신저 메시지 도착

[수영장 관리자]
- 메신저 메시지
- 구독 알림 (무조건 발송)
- 결제 알림 (무조건 발송)
- 운영 경고 알림 (통합 ON/OFF)

==================================================
3. 중복 방지 로직 (필수)
==================================================

- 일지 + 사진 함께 업로드 → “일지 푸시 1개만 발송”
- 사진만 업로드 → 사진 푸시 발송
- 동일 이벤트 중복 발송 금지

==================================================
4. 트리거 구조
==================================================

[즉시 발송]
- 공지 등록
- 일지 저장 완료
- 사진 업로드 완료
- 메시지 수신

[예약 발송]
- 전날 수업 알림 → 전날 설정 시간
- 당일 수업 알림 → 수업 X시간 전

==================================================
5. 관리자 푸시 설정 (운영설정에 추가)
==================================================

위치:
관리자 모드 > 운영설정 > “푸시 메시지 설정” 메뉴 신규 생성

설정 항목:

1. 전날 수업 알림 시간
- 기본값: 20:00

2. 당일 수업 알림 기준
- 기본값: 1시간 전
- 선택 가능: 1시간 / 2시간 / 3시간

3. 학부모 푸시 문구
- 항목별 템플릿 수정 가능
- 초기화 버튼 제공 (슈퍼관리자 기본값으로 복구)

==================================================
6. 학부모 푸시 설정 (프로필 설정)
==================================================

위치:
학부모 모드 > 프로필(톱니바퀴) > 푸시 알림 설정

항목:
- 공지사항 알림 ON/OFF
- 수업 일정 알림 ON/OFF
- 일지 알림 ON/OFF
- 사진 알림 ON/OFF

==================================================
7. 선생님 푸시 설정 (프로필 설정)
==================================================

위치:
선생님 모드 > 프로필(톱니바퀴) > 푸시 알림 설정

항목:
- 수업 일정 알림 ON/OFF
- 쪽지 알림 ON/OFF
- 메신저 알림 ON/OFF

==================================================
8. 수영장 관리자 푸시 설정 (프로필 설정)
==================================================

위치:
관리자 모드 > 프로필(톱니바퀴) > 푸시 알림 설정

항목:
- 메신저 알림 ON/OFF
- 운영 경고 알림 ON/OFF

중요:
- 구독/결제 알림은 ON/OFF 없음 (항상 발송)

==================================================
9. 푸시 문구 정책
==================================================

- 기본 문구: 슈퍼관리자 기준
- 수영장 관리자: 학부모용만 수정 가능
- 초기화 버튼으로 기본값 복구 가능
- 선생님/관리자 시스템 알림 문구 수정 금지

==================================================
10. 예약 발송 로직
==================================================

필수 구현:

[전날 알림]
- 다음날 수업 존재하는 학생 대상
- 매일 설정된 시간에 일괄 발송

[당일 알림]
- 수업 시간 기준 X시간 전 자동 발송

==================================================
11. 데이터 구조
==================================================

푸시 설정 필드:

- user_id
- role (parent/teacher/admin)
- notification_type
- is_enabled

관리자 설정:

- pool_id
- prev_day_push_time
- same_day_push_offset

푸시 로그:

- createdAt
- actor
- target_user_id
- role
- type
- status
- message
- triggered_by

==================================================
12. UI 위치 규칙 (중요)
==================================================

- “받는 설정” → 각 모드 프로필 톱니바퀴
- “보내는 설정” → 관리자 운영설정

절대 섞지 말 것

==================================================
13. 테스트 (필수 수행)
==================================================

반드시 직접 테스트:

1. 공지 등록 → 즉시 푸시 확인
2. 일지 작성 → 푸시 확인
3. 사진 단독 업로드 → 푸시 확인
4. 일지+사진 → 1개만 발송 확인
5. 전날 알림 → 시간 맞춰 발송 확인
6. 당일 알림 → 정확한 시간 발송 확인
7. 학부모 ON/OFF 작동 확인
8. 선생님 ON/OFF 작동 확인
9. 관리자 ON/OFF 작동 확인
10. 구독/결제 푸시 항상 오는지 확인
11. 여러 사용자 동시 테스트
12. UI 깨짐 여부 확인

==================================================
14. 완료 보고 형식 (반드시 제출)
==================================================

1. 구현된 기능 목록
2. 테스트 결과
3. 예약 푸시 동작 방식
4. ON/OFF 동작 결과
5. 수정 파일 목록
6. 남은 문제

==================================================
15. 최종 목적
==================================================

이 시스템은 단순 알림이 아니라
“운영 흐름을 끊기지 않게 만드는 핵심 기능”이다.

- 과하지 않게
- 빠짐없이
- 정확하게
- 안정적으로

모든 조건 충족해야 완료다.
