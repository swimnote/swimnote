[긴급 구조 단순화 지시서 - 기타수업 기능 완전 제거 + 선생님 반 삭제 기능 추가]

지금부터 기타수업 기능을 전체 시스템에서 제거하고,
스케줄러 구조를 “정규 반 시간표 + 회원 배정/해제” 중심으로 단순화한다.

이번 결정은 기능 축소가 아니라
운영 충돌과 예외 분기를 줄여서 앱 안정성을 높이기 위한 구조 정리다.

중요:
- 기타수업 관련 UI/저장/API/팝업/버튼/문구를 전부 제거
- 남은 스케줄러는 정규 반 시간표만 기준으로 동작
- 선생님도 반 삭제 가능하게 권한 추가
- 반 삭제는 “즉시 과거 기록 삭제”가 아니라
  “다음 주부터 반 비활성/미노출” 방식으로 처리
- 반 삭제 시 소속 회원은 즉시 미배정으로 이동
- 기존 출결/일지/기록은 유지
- 테스트 없이 완료 보고 금지

==================================================
1. 기타수업 기능 전체 제거
==================================================

제거 대상:
- 선생님 스케줄러 하단 팝업의 `기타수업` 버튼
- `기타 수업 추가` 팝업/모달/바텀시트
- 기타수업 등록 버튼
- 기타수업 저장 API 호출
- 기타수업용 상태값 / payload / validation
- 기타수업 관련 문구/토스트/로그
- 기타수업 관련 DB 저장 구조가 있으면 미사용 처리 또는 안전하게 비활성화
- 기타수업 관련 테스트 코드 / mock 데이터 / seed 데이터

중요:
- 화면에서만 숨기고 내부 로직 남겨두지 말 것
- 메뉴, 버튼, onPress, route, modal state까지 정리
- 스케줄러에서 기타수업 흔적이 0개가 되어야 함

==================================================
2. 스케줄러 운영 원칙 단순화
==================================================

최종 구조:
- 정규 반 시간표만 운영
- 기존 반 시간표 생성/조회
- 회원 배정
- 회원 해제
- 반 삭제

금지:
- 정규 반 외의 예외 수업 추가 기능
- 기타수업 직접 생성
- 별도 임시수업 저장 기능

==================================================
3. 선생님 반 삭제 기능 추가
==================================================

선생님 모드에서도 반 삭제 버튼을 제공한다.

위치:
- 스케줄러에서 반 상세/반배정/해당 반 관리 영역
- 관리자와 동일한 맥락에서 찾을 수 있는 위치
- 버튼명은 명확하게 `반 삭제`

동작:
- 해당 반 삭제 버튼 클릭
- 확인 팝업 노출
- 확인 시 “다음 주부터 반 비활성/미노출” 처리

중요:
- 선생님도 직접 삭제 가능
- 단, 물리 삭제가 아니라 일정 기준 삭제여야 함

==================================================
4. 반 삭제 정책
==================================================

반 삭제 기준은 아래처럼 처리한다.

사용자가 “이번 주 특정 요일부터 반 삭제”를 실행하면:
- 그 주차까지의 기존 수업 기록은 유지
- 해당 반은 다음 주부터 스케줄러에서 보이지 않음
- 즉, 미래 반복 일정만 끊긴다

예:
- 수요일 반 삭제 실행
- 이번 주 수요일 이전/해당 주차 기록은 그대로 남음
- 다음 주 수요일부터는 해당 반이 생성/노출되지 않음

중요:
- 과거 일지/출결/사진/기록 절대 삭제 금지
- 미래 반복 스케줄만 정지/비활성화

==================================================
5. 반 삭제 시 회원 처리
==================================================

반 삭제가 확정되면
현재 그 반에 소속된 회원은 전원 즉시 `미배정`으로 이동시킨다.

규칙:
- assignment_status = unassigned (미배정)
- 반과의 연결 제거
- 관리자/선생님 미배정 목록에 즉시 반영
- 학부모 미연결 상태와 혼동하지 않음

중요:
- 반 삭제 = 회원 삭제 아님
- 회원은 그대로 유지
- 반만 사라지고 회원은 미배정으로 전환

==================================================
6. 기록 유지 정책
==================================================

반 삭제 후에도 아래는 그대로 남아 있어야 한다.

유지 대상:
- 기존 일지
- 기존 출결
- 기존 보강 기록
- 기존 사진/앨범
- 기존 레벨 정보
- 기존 수정/삭제 로그

즉,
반이 미래 일정에서 사라져도
과거 운영 이력은 조회 가능해야 한다.

중요:
- 기록 조회 화면에서 과거 반명/시간 정보가 필요하면 그대로 표시
- 과거 데이터 참조 무결성 깨지지 않게 할 것

==================================================
7. 스케줄 생성 구조 수정
==================================================

현재 반복 반/정규 반 생성 구조에서
삭제된 반은 다음 주부터 생성되지 않게 처리한다.

가능한 구현 예:
- class_groups/status = inactive
또는
- recurrence_end_date 설정
또는
- deleted_from_date 설정

원칙:
- 과거 일정은 유지
- 미래 일정만 중단

중요:
- 다음 주 기준 계산 정확히
- 주간/월간/일간 스케줄러 모두 같은 기준 사용

==================================================
8. UI/문구 수정
==================================================

다음에서 기타수업 흔적 제거:
- 선생님 스케줄러 하단 액션 버튼
- 기타수업 저장 팝업
- 기타수업 안내 문구
- 테스트용 로그/문구

반 삭제 확인 팝업 문구 예시:
“이 반을 삭제하면 다음 주부터 시간표에서 사라집니다.
현재 소속 회원은 미배정으로 이동하며,
기존 수업 기록과 일지는 유지됩니다.”

버튼:
- [취소]
- [반 삭제]

==================================================
9. 권한 및 연결 확인
==================================================

반드시 확인:
- 선생님이 반 삭제 가능
- 관리자도 반 삭제 가능 (기존 유지)
- 반 삭제 후 관리자/선생님 양쪽 미배정 카운트 즉시 반영
- 학부모 화면은 기존 기록만 유지되고 미래 수업은 사라지는지
- 미배정과 미연결 카운트가 섞이지 않는지

10. 회귀 테스트 필수
==================================================

시나리오 A
- 선생님 스케줄러 진입
기대 결과:
- 기타수업 버튼 없음
- 관련 팝업 없음

시나리오 B
- 선생님이 반 삭제 실행
기대 결과:
- 확인 팝업 노출
- 삭제 확정 가능

시나리오 C

기대 결과:

시나리오 D

기대 결과:
- 전원 미배정 이동
- 관리자/선생님 목록 즉시 반영

시나리오 E
- 과거 일지/출결 조회
기대 결과:
- 기존 기록 유지
- 조회 가능

시나리오 F
- 관리자도 동일 반 확인
기대 결과:
- 미래 반 없음
- 과거 기록 유지
- 미배정 카운트 반영
