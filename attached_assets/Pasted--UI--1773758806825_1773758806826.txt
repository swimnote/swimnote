지금부터 회원가입/승인 로직 버그 수정만 진행하라.

중요:
- 기존 테스트 로그인 화면 유지
- 기존 각 모드 진입 유지
- 다른 기능/탭/UI 구조는 건드리지 말 것
- 이번 작업은 승인 상태 전환, 선생님 가입 승인 흐름, 선생님 아이디 계정 생성 로직만 수정
- 수정 후 반드시 실제 테스트까지 수행하고 결과를 상세 보고할 것
- 기능은 최대한 건드리지 않은 상태에서 수정하고, 연결은 반드시 확인하면서 진행할 것

==================================================
[1. 현재 확인된 문제]

문제 1.
슈퍼관리자 > 수영장 관리 화면에서
미승인 수영장 카드의 [승인] 버튼을 눌러도
실제 상태가 승인으로 변경되지 않는다.

문제 2.
선생님 회원가입은
수영장 검색 → 정보 입력 → 가입 요청 보내기
이후 관리자 승인 대기여야 하는데,
현재는 바로 가입되어 로그인 가능한 상태가 된다.

문제 3.
선생님 회원가입 화면에
이메일만 있고 “실제로 로그인에 사용할 아이디” 입력칸이 없다.

==================================================
[2. 이번 작업 목표]

이번 수정 목표는 아래 4개다.

1) 슈퍼관리자 수영장 승인 버튼이 실제 상태 변경되게 수정
2) 선생님 가입 요청은 즉시 가입이 아니라 "승인 대기" 상태로 저장되게 수정
3) 관리자 화면에서 선생님 가입 요청을 확인하고 [승인] / [거절] 할 수 있게 수정
4) 선생님 회원가입 화면에 “사용할 아이디” 입력칸을 추가하고 실제 로그인에 적용

==================================================
[3. 수영장 승인 버튼 버그 수정]

화면:
슈퍼관리자 > 수영장 관리

현재 문제:
- 미승인 카드에서 [승인] 눌러도 상태가 안 바뀜

수정 사항:
1. [승인] 버튼 클릭 시 해당 수영장 account/status 값을 실제로 변경할 것
2. 변경 후 아래가 즉시 반영되어야 함
   - 카드 상태: 미승인 → 무료 이용(또는 승인 상태)
   - 미승인 목록 개수 감소
   - 승인된 목록/전체 목록 개수 반영
3. 새로고침/재진입 후에도 상태 유지되게 저장 처리
4. [반려] 버튼도 동일하게 상태 저장 처리
   - 미승인 → 반려
   - 반려 탭/필터에 반영

상태 예시:
- pending
- approved_free
- rejected

주의:
- UI만 바뀌는 척하지 말고 실제 데이터 상태를 변경해야 함
- 상태 변경 후 검색/필터/개수 카운트도 다시 계산되어야 함

==================================================
[4. 선생님 회원가입 흐름 수정]

현재 잘못된 흐름:
선생님이 가입 요청을 보내면 바로 계정 생성 + 로그인 가능 상태가 됨

반드시 아래처럼 수정할 것.

정상 흐름:
선생님 가입 신청
→ 관리자 승인 대기
→ 관리자가 승인
→ 그때부터 로그인 가능

즉,
가입 요청 단계에서는 절대 ACTIVE 계정을 만들지 말 것

==================================================
[5. 선생님 가입 상태 구조]

선생님 가입 요청 시 상태값은 아래처럼 저장

- pending_approval = 관리자 승인 대기
- approved = 승인 완료
- rejected = 거절

가입 요청 직후:
- 로그인 불가
- 수영장 정보 접근 불가
- 완료 화면에는 “관리자 승인 후 로그인 가능” 안내

==================================================
[6. 선생님 회원가입 화면 수정]

화면:
선생님 회원가입 > 정보 입력 화면

현재 항목:
- 이름
- 이메일
- 비밀번호
- 전화번호(선택)

수정 후 항목:
- 이름
- 이메일
- 사용할 아이디
- 비밀번호
- 전화번호(선택)

배치:
이메일 아래에 “사용할 아이디” 입력칸 추가

예시 라벨:
- 사용할 아이디
- 또는 아이디

placeholder 예시:
- teacher_kim
- toykids_coach1

중요:
- 실제 계정 생성/로그인은 이메일이 아니라 이 아이디 기준으로 처리
- 이메일은 연락용 정보로만 보관 가능
- 로그인 화면에서 선생님은 아이디/비밀번호로 로그인

==================================================
[7. 선생님 가입 요청 저장 방식 수정]

가입 요청 보내기 버튼 클릭 시 아래처럼 동작해야 함.

정상 동작:
1. 선택한 수영장 ID 연결
2. 이름 / 이메일 / 아이디 / 비밀번호 / 연락처 저장
3. teacherRequests 또는 승인 대기 목록에 저장
4. 상태 = pending_approval
5. 일반 선생님 계정 목록에는 아직 ACTIVE로 넣지 말 것
6. 관리자 승인 전까지 로그인 금지

즉,
“가입 요청 데이터”와 “실제 활성 계정 데이터”를 분리하거나,
같은 데이터라도 상태값 기준으로 로그인 차단할 것

==================================================
[8. 관리자 승인 화면 생성/수정]

관리자 모드에서
선생님 가입 요청을 확인하고 승인/거절할 수 있어야 한다.

필수 기능:
1. 관리자 > 사람 > 승인
또는
관리자 > 승인 관리 > 선생님 승인
에 가입 요청 목록 표시

요청 카드 표시 항목:
- 이름
- 이메일
- 아이디
- 요청 수영장
- 요청일

버튼:
- 승인
- 거절

--------------------------------------
[8-1. 승인 버튼 동작]

승인 시:
- 상태 pending_approval → approved
- 선생님 로그인 가능
- 선생님 목록에 반영
- 해당 수영장 소속 선생님으로 연결

--------------------------------------
[8-2. 거절 버튼 동작]

거절 시:
- 상태 pending_approval → rejected
- 로그인 불가 유지
- 필요 시 거절 목록/상태 반영

==================================================
[9. 선생님 로그인 제한]

로그인 로직 수정:
선생님 계정 로그인 시

1. 아이디 없음 → 가입 유도
2. 아이디 존재 + 비밀번호 일치
   → 상태 확인

상태가
- approved 이면 로그인 허용
- pending_approval 이면 로그인 차단 + 안내
- rejected 이면 로그인 차단 + 안내

안내 문구 예시:
- 승인 대기 중입니다. 관리자 승인 후 로그인할 수 있습니다.
- 가입 요청이 거절되었습니다. 관리자에게 문의해 주세요.

즉,
비밀번호만 맞는다고 로그인시키지 말 것.
반드시 승인 상태까지 확인할 것.

==================================================
[10. 완료 화면 수정]

선생님 가입 요청 완료 화면 문구는 아래 취지로 맞출 것.

예시:
가입 요청 완료!
[수영장명] 관리자가 요청을 확인 후 승인하면 로그인할 수 있습니다.

버튼:
- 로그인 화면으로

주의:
완료 화면 이후 바로 로그인되면 안 됨
로그인 시도해도 승인 전까지 막혀야 함

==================================================
[11. 기존 연결 유지 조건]

아래는 절대 유지할 것:
- 테스트 로그인 화면
- 각 모드 테스트 진입
- 다른 회원가입 흐름(학부모, 관리자)
- 다른 탭 화면
- 기존 수업/출결/메신저/정산 기능

이번 작업은
수영장 승인 상태 변경 + 선생님 승인 가입 로직
이 두 가지 중심 버그 수정만 한다.

==================================================
[12. 연결 안정성 체크 - 매 작업마다 포함]

반드시 아래 체크를 하면서 수정할 것.

[공통 체크]
1. 수정한 버튼이 실제 데이터 상태를 바꾸는지 확인
2. 새로고침 후 상태 유지되는지 확인
3. 다른 화면의 카운트/목록이 맞게 반영되는지 확인
4. 승인 전/후 로그인 제한이 올바른지 확인
5. 기존 테스트 로그인과 모드 진입이 유지되는지 확인
6. 다른 회원가입 흐름이 깨지지 않았는지 확인

핵심:
기능은 최대한 건드리지 않은 상태에서 수정하고,
서로 연결된 부분은 반드시 확인하면서 진행할 것

==================================================
[13. 테스트 지시 - 반드시 실제 수행]

수정 후 아래 테스트를 전부 수행하고 결과를 상세히 보고하라.

테스트 1.
슈퍼관리자 > 수영장 관리에서 미승인 수영장 [승인] 버튼 클릭
→ 상태가 실제로 승인으로 바뀌는지 확인

테스트 2.
승인 후 미승인 개수 / 전체 개수 / 필터 목록이 즉시 반영되는지 확인

테스트 3.
슈퍼관리자 > 수영장 관리에서 [반려] 버튼 클릭
→ 반려 상태 저장 및 필터 반영 확인

테스트 4.
선생님 회원가입 화면에서 이메일 아래 “사용할 아이디” 입력칸이 추가되었는지 확인

테스트 5.
선생님 가입 요청 후 즉시 로그인되지 않고
“승인 대기” 상태가 되는지 확인

테스트 6.
관리자 화면에서 선생님 가입 요청 목록이 보이는지 확인

테스트 7.
관리자가 [승인] 누르면 해당 선생님이 실제로 로그인 가능한지 확인

테스트 8.
관리자가 [거절] 누르면 해당 선생님이 로그인 불가능한지 확인

테스트 9.
선생님 로그인 시 아이디/비밀번호 기준으로 로그인되는지 확인
(이메일 로그인 아님)

테스트 10.
기존 테스트 로그인 화면과 각 모드 진입이 그대로 유지되는지 확인

테스트 11.
학부모/관리자 회원가입 흐름이 이번 수정으로 깨지지 않았는지 확인

==================================================
[14. 보고 형식]

작업 후 아래 순서로 보고하라.

1. 수영장 승인 버튼 버그 원인과 수정 방식
2. 상태값을 어떻게 바꿨는지
3. 선생님 가입 요청 저장 구조를 어떻게 바꿨는지
4. 관리자 승인/거절 UI를 어디에 어떻게 연결했는지
5. 선생님 로그인 제한을 어떻게 걸었는지
6. 사용할 아이디 필드를 어떻게 추가했는지
7. 테스트 1~11 결과
8. 아직 남은 임시 처리 부분
9. 다른 기능/화면에 영향 준 부분이 있으면 숨기지 말고 적기

==================================================
[15. 절대 금지]

- 테스트 로그인 삭제 금지
- 기존 모드 진입 삭제 금지
- 선생님 승인 없이 즉시 로그인되게 두기 금지
- 승인 버튼 눌러도 UI만 바뀌고 실제 상태 저장 안 되는 방식 금지
- 다른 주요 탭 화면 수정 금지
- 이번 작업 범위를 넘어가는 UI 재배치 금지

이번 작업은
“수영장 승인 상태 저장 버그 수정 + 선생님 가입 승인 흐름 정상화 + 선생님 아이디 로그인 적용”
까지만 수행한다.
다른 기능은 건드리지 말 것.
