지금부터 선생님 모드 전체 UI를 관리자 모드와 같은 운영 규칙으로 재배치하라.

이번 작업 목표:
- 선생님 모드를 홈 중심 운영 구조로 재배치
- 기존 기능은 절대 깨뜨리지 않기
- 관리자 모드와 연동되는 데이터 흐름 절대 끊기지 않게 유지
- 마지막에 실제 작동 테스트까지 완료하기
- 버그 없이 안정적으로 동작하게 만들기

가장 중요:
- 이번 작업은 “기능 추가”가 아니라 “구조 재배치 + 진입 규칙 통일”
- 관리자 모드와 연동되는 출결 / 보강 / 회원 / 정산 / 저장공간 / 메신저 데이터는 그대로 유지
- 선생님 모드만의 홈/팝업/페이지 구조를 관리자 모드와 같은 철학으로 맞춘다
- 규칙은 철저하게 적용한다
- 중간에 대충 연결만 하고 끝내지 말고, 마지막에 실제 동작 테스트까지 반드시 수행한다

==================================================
[0. 공통 규칙 - 관리자/선생님 공통 운영 규칙 철저 적용]

이번 작업 전에 아래 규칙을 선생님 모드에도 그대로 적용하라.

--------------------------------------
[공통 규칙 1. 전체 구조]

전체 구조는 아래 3단계까지만 허용

1단계: 홈
2단계: 아이콘 팝업
3단계: 실제 페이지

즉:
홈 → 팝업 → 페이지

절대 4단계 이상 금지

--------------------------------------
[공통 규칙 2. 페이지 규칙]

모든 실제 페이지는 공통 헤더 사용

헤더 구성:
- 왼쪽: 뒤로가기
- 오른쪽: 홈
- 수정 가능/입력 가능 페이지는 저장 버튼 추가

규칙:
- 뒤로가기 = 바로 이전 단계
- 홈 = 선생님 홈 첫 화면
- 저장 가능한 페이지는 저장 버튼 반드시 존재

--------------------------------------
[공통 규칙 3. 팝업 규칙]

- 아이콘 클릭 시 뜨는 팝업은 디자인 통일
- 바깥 터치 닫기
- X 닫기
- 드래그 닫기
- 아이콘 3열 그리드
- 팝업 안에서는 긴 실데이터를 과도하게 보여주지 않는다
- 팝업은 중간 분류 역할

--------------------------------------
[공통 규칙 4. 이동 규칙]

선생님 모드의 핵심 업무 이동은 “스케줄 중심”으로 정리한다.

즉:
- 출결하다가 수업일지로 직접 점프하지 않음
- 수업일지에서 회원관리로 직접 점프하지 않음
- 다시 스케줄/상위 페이지로 돌아와서 이동

예:
스케줄 → 출결
뒤로
스케줄 → 수업일지
뒤로
스케줄 → 반관리

즉:
모든 업무는 스케줄을 중심 허브로 보고 정리

--------------------------------------
[공통 규칙 5. 리스트 규칙]

- 리스트/카드 구조는 컴팩트하게
- 검색창 상단
- 선택/삭제 기능 기존대로 유지
- 카드/버튼/간격/상태 표시 통일
- 버그 없이 빠르게 확인 가능하게

==================================================
[1. 선생님 홈 화면 재배치]

선생님 모드 첫 화면을
관리자 모드와 같은 “홈 중심 운영 화면”으로 재배치한다.

현재 하단 탭바 기반 진입 구조는 제거한다.
단, 기능은 사라지면 안 된다.

즉:
- 탭바는 없애되
- 기존 기능들은 홈 아이콘/팝업/페이지 구조로 다 들어가게 해야 함

==================================================
[2. 선생님 홈 상단 배너 구조]

선생님 홈 상단은 아래 3개 영역으로 구성

1. 첫 줄 배너 (오늘 기준)
2. 둘째 줄 배너 (누적 업무)
3. 얇은 줄 배너 (스케줄 메모)

--------------------------------------
[2-1. 첫 줄 배너 - 오늘 기준]

항목 4개:
- 오늘 수업
- 출결 완료
- 일지 작성
- 주간 스케줄

설명:
- 오늘 수업: 오늘 잡힌 수업 개수
- 출결 완료: 오늘 수업 기준 완료 개수
- 일지 작성: 오늘 수업 기준 완료 개수
- 주간 스케줄: 빠른 실행 버튼

중요:
- “주간 스케줄”은 빠른 실행
- 수업관리 아이콘을 거치지 않고 바로 주간 스케줄 페이지로 이동 가능해야 함

--------------------------------------
[2-2. 둘째 줄 배너 - 누적 업무]

항목 4개:
- 미작성 일지 [숫자]
- 보강 대기 [숫자]
- 출석 미체크 [숫자]
- 안 읽은 쪽지 [숫자]

중요:
- 이 숫자는 오늘치만이 아니라 누적 포함
- 밀린 일이 많을수록 숫자가 커짐
- 선생님이 홈에 들어오면 “밀린 일”을 먼저 보게 하는 구조

--------------------------------------
[2-3. 얇은 줄 배너 - 스케줄 메모]

얇은 진입형 줄로 표시

동작:
- 스케줄 메모 클릭
- 주간 스케줄표로 이동
- 날짜 선택
- 해당 날짜 메모 페이지 열림

즉:
스케줄 메모는 독립 기능이 아니라
주간 스케줄과 연결된 메모 기능으로 고정

==================================================
[3. 선생님 홈 메인 아이콘 8개]

홈 배너 아래 메인 아이콘 8개 배치

최종 8개:
1. 수업관리
2. 회원관리
3. 보강관리
4. 쪽지
5. 메신저
6. 정산
7. 내정보
8. 설정

배치:
- 관리자 모드와 동일한 톤/규칙 유지
- 4 x 2 그리드
- 한 화면에 8개 다 보이게
- 아이콘 카드 스타일 통일
- 숫자 배지 필요 시 유지

==================================================
[4. 수업관리 구조]

수업관리는 선생님 모드 핵심 허브다.

수업관리 아이콘 클릭 시 팝업 또는 중간 분류에서 아래로 이동

세부 분류:
- 스케줄
- 출결
- 수업일지
- 반관리

--------------------------------------
[4-1. 스케줄]

스케줄 클릭 시
기존에 만들어둔 일간/주간/월간이 함께 있는 스케줄 페이지로 이동

중요:
- 주간이 중심
- 일간/월간은 탭 전환
- 기존 스케줄 기능 유지
- 주간 스케줄이 중심 허브 역할

--------------------------------------
[4-2. 출결]

출결 클릭 시 페이지:
- 오늘 출결
- 출결 기록

중요:
- 기존 출결 페이지 유지
- 관리자 모드와 연동 끊기지 말 것
- 출결 처리 후 상태가 홈/보강/정산에 반영되는 흐름 유지

--------------------------------------
[4-3. 수업일지]

수업일지 클릭 시
바로 “일지 작성 페이지”로 이동

중요:
- 홈에서 수업일지 누르면 작성 쪽으로 바로 가는 구조
- 페이지 내부 상단 또는 내부 분류에서
  - 일지 목록
  - 작성
  을 처리할 수 있게 유지

즉:
진입은 작성 중심
상세 기능은 페이지 안에서 처리

--------------------------------------
[4-4. 반관리]

반관리 클릭 시 페이지:
- 반 생성 가능
- 반 정보
- 학생 목록
- 정원 상태

중요:
- 기존에 만들어둔 반 관련 기능 유지
- 반 생성도 여기 포함
- 관리자와 연결된 반 정보 구조 유지

==================================================
[5. 회원관리 구조]

회원관리 아이콘 클릭 시 중간 분류 또는 페이지

세부 분류:
- 내 반 회원
- 반배정
- 반이동
- 미배정 회원

설명:
- 내 반 회원 → 내가 담당 중인 학생 목록
- 반배정 → 기존 반배정 페이지
- 반이동 → 기존 반이동 페이지
- 미배정 회원 → 배정 안 된 학생 보기

중요:
- 쪽지 기능은 여기서 제거
- 회원관리와 쪽지는 완전히 분리
- 관리자 모드의 회원 데이터와 연동 끊기지 말 것

==================================================
[6. 보강관리 구조]

보강관리 아이콘 클릭 시 중간 분류 또는 페이지

세부 분류:
- 보강 대기
- 보강 현황

--------------------------------------
[6-1. 보강 대기]

보강 대기 페이지:
- 대기자 리스트 확인
- 리스트에서 바로 보강 배정 가능
- 보강 배정 누르면 주간 스케줄표로 이동

즉:
보강 배정은 별도 메뉴 남발하지 말고
보강 대기 페이지 안에서 바로 처리

--------------------------------------
[6-2. 보강 현황]

보강 현황은 월별 확인 페이지

표시 구조:
- 현재 월 기준
- 전월 이월 보강
- 이번 달 보강

예:
3월 화면
- 전월 이월 보강 몇 명
- 이번 달 보강 몇 명

중요:
- 결석 체크 즉시 보강대기로 넘어가는 흐름 유지
- 보강 현황은 리스트 페이지가 아니라
  월별 통계/잔량 확인용 성격

==================================================
[7. 쪽지 구조]

쪽지는 독립 아이콘으로 둔다.

쪽지 아이콘 클릭 시
“간편 팝업” 먼저 오픈

팝업 내용:
- 최근 쪽지
- 안 읽은 쪽지

중요:
- 홈에서 쪽지는 최대한 간편하게
- 팝업은 빠른 확인용
- 팝업에서 항목 누르면 쪽지 페이지로 이동
- 쪽지 페이지에서 전체/상세/답장 등 자세한 처리

즉:
홈 = 빠른 확인
페이지 = 자세한 처리

==================================================
[8. 메신저 구조]

메신저는 현재처럼 별도 유지

동작:
- 메신저 아이콘 클릭
- 바로 메신저 페이지 진입

중요:
- 메신저는 관리자/선생님 내부 업무용 대화
- 쪽지와 완전히 분리
- 기존 메신저 기능 유지
- 관리자 모드와 연동 유지

==================================================
[9. 정산 구조]

정산은 독립 아이콘 유지

내용:
- 월 정산
- 수업 통계
- 개인 실적

중요:
- 기존 선생님 정산 기능 유지
- 관리자 정산 확인 구조와 연동 유지
- 정산 데이터 흐름 끊기지 말 것

==================================================
[10. 내정보 구조]

내정보에는 선생님 본인 관련 기능만 넣는다.

구성:
- 내 프로필 수정
- 연락처/개인정보 수정
- 권한 관련 정보
- 관리자 모드 전환 버튼 (해당자만 하단 노출)

중요:
- 관리자 모드 전환은 상단 공통 버튼으로 두지 않음
- 해당되는 사람만 내정보 하단 쪽에서 보이게
- 공통 사용자에게 보이지 않게 분기 처리

==================================================
[11. 설정 구조]

설정은 개인 설정 중심

구성:
- 저장공간
- 알림 설정
- 앱 설정
- 피드백 기본 설정
- 기타 설정

중요:
- 저장공간은 설정으로 이동
- 기존 더보기 내부 개인 설정 기능은 여기로 정리 가능
- 기능은 유지, 진입 구조만 정리

==================================================
[12. 관리자 모드와 연동되는 부분 절대 유지]

아래 연결은 절대 끊기면 안 된다.

1. 스케줄/반정보
2. 반배정/반이동
3. 미배정 회원
4. 출결
5. 결석 → 보강대기
6. 보강 배정
7. 정산 데이터
8. 메신저
9. 저장공간/데이터량

즉:
데이터는 관리자와 공유
선생님 모드는 진입 구조만 선생님용으로 재배치

==================================================
[13. 하단 탭바 제거]

현재 선생님 모드 하단 탭바는 제거하라.

단:
- 기능은 하나도 사라지면 안 된다
- 홈 아이콘/팝업/페이지 구조로 전부 대체
- 탭바 제거로 기능 진입이 끊기지 않게 연결 유지

==================================================
[14. 기존 기능 절대 보호]

아래 기능은 절대 깨지면 안 된다.

1. 일간 / 주간 / 월간 스케줄
2. 출결 / 출결 기록
3. 수업일지 작성 / 일지 목록 / 피드백커스텀
4. 반 생성 / 반 정보 / 학생 목록 / 정원 상태
5. 내 반 회원 / 반배정 / 반이동 / 미배정 회원
6. 보강 대기 / 보강 배정 / 보강 현황
7. 쪽지 / 메신저
8. 정산
9. 내정보 / 저장공간 / 설정
10. 관리자 모드와의 데이터 연동

기능 로직 변경 금지
진입 구조/배치만 재구성

==================================================
[15. 먼저 확인할 것]

작업 전에 아래를 먼저 파악하라.

1. 선생님 홈 화면 파일
2. 기존 하단 탭바 구성 파일
3. 기존 수업 / 정산 / 더보기 / 메신저 / 쪽지 진입 구조
4. 공통 팝업 컴포넌트
5. 공통 헤더 컴포넌트
6. 스케줄 페이지 파일
7. 출결 / 수업일지 / 반관리 / 회원관리 / 보강관리 / 정산 관련 파일
8. 관리자와 공유하는 데이터 구조
9. 모드 전환 버튼 분기 로직

==================================================
[16. 작업 순서]

반드시 아래 순서로 진행하라.

1. 공통 규칙 선생님 모드에 적용
2. 하단 탭바 제거
3. 선생님 홈 배너 3개 영역 구성
4. 메인 아이콘 8개 배치
5. 수업관리 / 회원관리 / 보강관리 / 쪽지 / 메신저 / 정산 / 내정보 / 설정 연결
6. 관리자 연동 포인트 점검
7. 공통 헤더/홈/뒤로가기/저장 버튼 점검

[17. 테스트 - 반드시 실제 수행]

테스트 1.
선생님 홈에 첫 줄 배너(오늘 수업 / 출결 완료 / 일지 작성 / 주간 스케줄)가 보이는지 확인

테스트 2.
둘째 줄 배너(미작성 일지 / 보강 대기 / 출석 미체크 / 안 읽은 쪽지)가 누적 숫자로 보이는지 확인

테스트 3.
스케줄 메모 클릭 시 주간 스케줄 → 날짜 선택 → 메모 페이지 흐름이 되는지 확인

테스트 4.
메인 아이콘 8개가 정상 보이는지 확인

테스트 5.
수업관리 클릭 후 스케줄/출결/수업일지/반관리로 정상 이동하는지 확인

테스트 6.
스케줄 클릭 시 기존 일간/주간/월간 페이지가 정상 나오는지 확인

테스트 7.
출결 클릭 시 오늘 출결/출결 기록 흐름이 정상인지 확인

테스트 8.
수업일지 클릭 시 바로 작성 페이지로 진입하는지 확인

테스트 9.
반관리 클릭 시 반 생성/반 정보/학생 목록/정원 상태가 유지되는지 확인

테스트 10.
회원관리 클릭 시 내 반 회원/반배정/반이동/미배정 회원이 정상인지 확인

테스트 11.
보강관리 클릭 시 보강 대기/보강 현황이 정상인지 확인

테스트 12.
보강 대기 페이지에서 바로 보강 배정 후 주간 스케줄표로 이동되는지 확인

테스트 13.
쪽지 클릭 시 간편 팝업(최근/안 읽은)이 뜨는지 확인

테스트 14.
쪽지 팝업에서 항목 클릭 시 쪽지 상세 페이지로 이동하는지 확인

테스트 15.
메신저 아이콘 클릭 시 바로 메신저 페이지로 가는지 확인

테스트 16.
정산/내정보/설정이 정상 이동하는지 확인

테스트 17.
관리자 모드와 연동되는 회원/보강/정산/저장공간 데이터가 끊기지 않았는지 확인

테스트 18.
모든 페이지에서 뒤로가기/홈/저장 버튼이 규칙대로 동작하는지 확인

테스트 19.
탭바 제거 후에도 기능 진입이 하나도 안 사라졌는지 확인

테스트 20.
전체적으로 버그 없이 동작하는지 최종 확인

[19. 절대 금지]

- 기능 로직 변경 금지
- 관리자 연동 끊기기 금지
- 탭바 제거하면서 기능 사라지게 만들기 금지
- 수업관리/회원관리/보강관리 중복 진입으로 다시 꼬이게 만들기 금지
- 쪽지와 메신저를 섞기 금지
- 모드 전환 버튼을 모든 선생님에게 공통 노출 금지
- 테스트 없이 완료 보고 금지

선생님 모드를 관리자 모드와 같은 운영 규칙으로 재배치하되,
홈 배너 / 8개 아이콘 / 수업 중심 구조 / 쪽지와 메신저 분리 / 관리자 데이터 연동 유지까지 모두 반영하고,
마지막에 실제 작동 테스트까지 완료하는 작업이다.
