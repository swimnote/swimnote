[MAIN GOAL]
==================================================

I want a clean first login flow with these rules:

1. First screen must show ONLY:
   - ID input
   - Swimming Pool Search section

2. Password must NOT appear on the first screen.
   Password should only appear on the next screen after ID is entered and verified.

3. ID detection logic:
   - If entered ID already exists -> go to password screen
   - If entered ID does not exist -> treat it as desired signup ID and go to signup screen automatically

4. After successful login:
   - show a mode/role selection screen
   - roles: Admin / Teacher / Parent
   - user enters app by choosing a role

5. If user belongs to multiple swimming pools / organizations:
   - add organization selector at the top-left of the role selection screen
   - user must be able to switch organization there

6. If app is reopened and login session is still valid:
   - do NOT show login again
   - directly show organization + role selection screen

7. Inside app:
   - user can move to role/mode switch page without logging out
   - user can also switch organization there if they belong to multiple pools
