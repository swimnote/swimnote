[스윔노트 — 회원 상태 흐름 최종 완성 (연기 / 퇴원 / 월 전환 / 유료회원 기준)]

이번 작업은 회원 상태 관리 기능을
“운영 기준에 맞게 완전히 고정”하는 작업이다.

UI + 상태 + 정산 기준 + 월 전환까지 모두 포함한다.

━━━━━━━━━━━━━━━━━━
1. 절대 규칙
━━━━━━━━━━━━━━━━━━

- DB 구조 변경 금지
- API 구조 변경 금지
- 기존 상태 정의 유지

작업 범위:
- 상태 반영
- 카운트 기준
- 월 전환 로직
- 검증

━━━━━━━━━━━━━━━━━━
2. 유료회원 카운트 기준
━━━━━━━━━━━━━━━━━━

카운트 대상:

- status = active → 포함
- status = suspended → 포함
- status = withdrawn → 제외

추가 조건:

- pending 상태는 무시
- status 기준으로만 계산

━━━━━━━━━━━━━━━━━━
3. 탭별 데이터 기준
━━━━━━━━━━━━━━━━━━

대기자:
- status = active
- assigned_class_ids 없음

연기예정:
- pending_status_change = suspended

퇴원예정:
- pending_status_change = withdrawn

연기(확정):
- status = suspended

퇴원:
- status = withdrawn

━━━━━━━━━━━━━━━━━━
4. 월 전환 로직 (핵심)
━━━━━━━━━━━━━━━━━━

월초(1일 00:00 기준):

모든 학생에 대해:

IF pending_status_change 존재:

- status = pending_status_change
- pending_status_change = null
- pending_effective_month = null

즉:

pending → 실제 상태로 승격

━━━━━━━━━━━━━━━━━━
5. 월 전환 구현 방식
━━━━━━━━━━━━━━━━━━

옵션 A (권장):
- 서버 cron 또는 배치 작업

옵션 B:
- 첫 로그인 시 체크

조건:
- 해당 월 아직 처리 안 됐을 경우만 실행

━━━━━━━━━━━━━━━━━━
6. UI 반영 규칙
━━━━━━━━━━━━━━━━━━

연기예정:
- "연기예정" 배지 표시

퇴원예정:
- "퇴원예정" 배지 표시

월 전환 이후:

- 배지 제거
- status 기준으로 이동

━━━━━━━━━━━━━━━━━━
7. 검증 (필수 — 미제출 시 실패)
━━━━━━━━━━━━━━━━━━

1. 유료회원 카운트 검증

케이스:

- active 3명
- suspended 2명
- withdrawn 2명

결과:
→ 카운트 = 5명

2. 연기(next_month)

- pending 생성 확인
- 배지 표시 확인

3. 퇴원(next_month)

- pending 생성 확인
- 배지 표시 확인

4. 월 전환 시뮬레이션

- pending → status 변경 확인
- pending 필드 초기화 확인

5. UI 반영

- 연기 → 연기탭 이동
- 퇴원 → 퇴원탭 이동

6. load() 이후 정상 반영

━━━━━━━━━━━━━━━━━━
8. 로그 확인
━━━━━━━━━━━━━━━━━━

월 전환 시 로그 출력:

[MONTHLY_STATUS_UPDATE]
student: <id>
from: pending → status 적용 완료

━━━━━━━━━━━━━━━━━━
9. 제출 (필수)
━━━━━━━━━━━━━━━━━━

1. 수정 파일 목록
2. 유료회원 카운트 로직 코드
3. 월 전환 처리 코드
4. 테스트 결과 (6개 항목)
5. 실제 로그 출력

━━━━━━━━━━━━━━━━━━
핵심 한 줄
