[선생님 앱 화면 구조 개편 및 관리자-선생님 연동 최적화 개발 지시]

목표:
선생님이 실제 수업 현장에서 빠르게 사용할 수 있도록 "내반 / 출결 / 수영일지 / 사진 / 영상" 흐름을 단순화한다.
현재 카드 크기가 너무 크고 정렬이 불안정한 문제를 수정하고,
관리자가 회원등록 및 반배정을 하면 선생님 화면에 즉시 반영되도록 데이터 연동 구조를 안정화한다.

핵심 방향:
1. 선생님 화면은 큰 카드 나열형이 아니라 "주간 시간표 중심 구조"로 변경
2. 모든 기능은 동일한 시간표 기준으로 진입하게 통일
3. 관리자에서 회원등록 + 반배정 완료 시 선생님 화면에 즉시 반영
4. 선생님이 한 손으로 빠르게 체크할 수 있도록 클릭 수 최소화
5. 사진과 영상은 별도 업로드 구조로 분리하되 화면 구조는 동일한 패턴 유지

--------------------------------------------------
1. 공통 데이터 연동 규칙
--------------------------------------------------

관리자가 어린이 회원등록 후 반배정을 완료하면 아래 데이터가 즉시 선생님 계정에 반영되어야 한다.

필수 동기화 데이터:
- 회원 기본정보
- 소속 반
- 담당 선생님
- 수업 요일
- 수업 시간
- 출결 대상 여부
- 사진/영상 업로드 대상 여부
- 수영일지 기록 대상 여부

데이터 반영 규칙:
- 관리자에서 저장 완료 즉시 teacherClassAssignments를 갱신
- 선생님 앱은 별도 새로고침 없이 반영되도록 실시간 상태 동기화 또는 저장 후 즉시 재조회
- 최소한 각 탭 진입 시 최신 데이터 다시 불러오기
- 반배정이 없는 학생은 선생님 화면에 표시하지 않음
- 비활성 반(학생 0명)은 회색 비활성 처리
- 활성 반(학생 1명 이상)은 클릭 가능 표시

정렬 기준:
- 1순위: 요일 순서 (월 화 수 목 금 토 일)
- 2순위: 시간 오름차순
- 3순위: 반 이름 가나다순
- 4순위: 학생 이름 가나다순

--------------------------------------------------
2. 선생님 홈 구조 기본 원칙
--------------------------------------------------

선생님 메뉴 구성:
- 내반
- 출결
- 수영일지
- 사진
- 영상

중요:
기존처럼 큰 카드 여러 개를 세로로 길게 나열하지 말고,
상단은 간결한 탭 메뉴 또는 세그먼트 메뉴로 구성한다.

카드 UI 규칙:
- 카드 높이 축소
- 한 화면에 여러 시간대가 보이도록 밀도 높게 구성
- 큰 썸네일형 카드 금지
- 텍스트 중심 + 상태 뱃지 방식 사용
- 좌우 여백과 정렬 통일
- 반 이름 / 시간 / 학생수 / 활성상태를 한 줄 또는 두 줄 안에 정리

--------------------------------------------------
3. 내반 탭 구조
--------------------------------------------------

내반 탭은 "주간 시간표 기반"으로 다시 설계한다.

화면 구조:
- 상단: 요일 선택 바 (월 화 수 목 금 토 일)
- 중단: 선택한 요일의 시간표 리스트
- 각 시간 슬롯은 13:00 ~ 21:00 범위에서 관리자 설정값 기준으로 표시
- 반이 개설된 시간은 활성화
- 개설되지 않은 시간은 비활성 처리

시간표 슬롯 표시 항목:
- 시간
- 반 이름
- 학생 수
- 활성/비활성 상태
- 담당 선생님 본인 반만 노출

동작:
- 개설된 반 클릭 시 학생목록 화면 이동
- 학생목록 클릭 시 학생 상세정보 화면 이동

학생목록 화면:
- 이름
- 출생년도
- 수업반
- 등록상태
- 최근 출결 요약
- 최근 일지 작성 여부
- 사진/영상 업로드 이력 간단 표시 가능하면 추가

학생 상세정보 화면:
- 이름
- 출생년도
- 보호자 연결 여부
- 반 정보
- 요일/시간
- 최근 출결 기록
- 최근 수영일지 기록
- 개인 사진/영상 업로드 내역

추천 개선:
내반 탭 첫 화면에 "오늘 수업만 보기" 토글도 추가
- 기본은 오늘 요일 자동 선택
- 필요 시 다른 요일로 이동 가능

--------------------------------------------------
4. 출결 탭 구조
--------------------------------------------------

출결 탭도 내반과 동일하게 "주간 시간표 기반" 구조로 통일한다.

출결 화면 구조:
- 상단: 요일 선택 바
- 중단: 시간표 슬롯 리스트
- 해당 시간 반 클릭 시 학생목록 표시

학생목록 출결 UI:
각 학생 행에 아래 버튼 2개를 표시
- 출석
- 결석

상단 고정 액션 버튼:
- 모두출석

하단 고정 액션 버튼:
- 완료

출결 동작 규칙:
- 모두출석 클릭 시 현재 목록 학생 전체를 출석 상태로 일괄 적용
- 각 학생별로 출석/결석 수동 수정 가능
- 완료 버튼 클릭 시 저장 후 창 닫기 또는 이전 화면으로 복귀
- 완료 전 이탈 시 저장 여부 확인 모달 표시 가능
- 이미 체크된 학생은 상태 색상 또는 뱃지로 구분
- 미체크 학생 존재 시 완료 버튼 누를 때 경고 표시 가능
- 그래도 저장은 가능하게 할지 여부는 옵션화하되 기본은 "미체크 있으면 경고 후 진행"

출결 최적화 추천:
- 학생 이름 왼쪽에 작은 프로필 원형 아바타 또는 이니셜 표시 가능
- 출석/결석 버튼은 토글형으로 크게 만들지 말고 가로 2버튼 압축형으로 구성
- 모두출석 버튼은 상단 우측 고정
- 완료 버튼은 하단 고정으로 항상 보이게

--------------------------------------------------
5. 수영일지 탭 구조
--------------------------------------------------

수영일지 역시 같은 시간표 구조를 사용한다.

화면 구조:
- 상단: 요일 선택 바
- 중단: 시간표 슬롯 리스트
- 시간표 클릭 시 바로 일지작성 페이지 진입

일지작성 페이지 기능:
- 오늘 수업 일지 작성 입력폼
- 저장 버튼
- 취소 버튼
- 지난 일지 보기 버튼

요구사항 반영:
같은 수업시간의 지난번 수업일지를 볼 수 있게 해야 한다.

구체 동작:
1. 시간표 클릭
2. 일지작성 페이지 진입
3. 상단에 이전 동일 반/동일 시간대의 최근 일지 요약 노출
4. 선생님은 새 일지 작성 가능

추가 버튼 구성:
- 저장
- 지난일지보기
- 일지취소하고나가기

중요:
사용자가 아무것도 입력하지 않았을 때도
"지난일지보기" 또는 "일지취소하고나가기"를 선택할 수 있어야 한다.

추천 UX 개선:
일지 진입 시 아래 2가지 방식 중 더 나은 방식으로 구현
A안:
- 상단 탭: 새 일지 작성 / 지난 일지 보기
B안:
- 기본은 새 일지 작성 화면
- 상단 우측에 "지난 일지" 버튼 배치

권장안:
B안 사용
이유:
현장에서는 새 일지 작성이 주기능이라서 바로 입력창이 보여야 빠르다.

일지 항목 예시:
- 수업 날짜
- 반 이름
- 출석 인원
- 오늘 훈련 내용
- 진도
- 특이사항
- 다음 수업 메모

이전 일지 불러오기 기능 추천:
- 직전 일지를 템플릿처럼 불러와 수정 저장 가능하게 하면 선생님 사용성이 매우 좋아짐

--------------------------------------------------
6. 사진 탭 / 영상 탭 구조
--------------------------------------------------

사진과 영상은 분리하되 구조는 같게 만든다.

현재 요구사항에 따라 최종 카드 구조는 총 4개 버튼이 필요하다.

사진 영역:
- 반전체앨범
- 개인앨범

영상 영역:
- 반전체앨범
- 개인앨범

즉 화면에는 아래 4개 카드 버튼 구성:
1. 사진 - 반전체앨범
2. 사진 - 개인앨범
3. 영상 - 반전체앨범
4. 영상 - 개인앨범

카드 UI 원칙:
- 2 x 2 그리드 형태
- 동일 크기
- 아이콘 + 텍스트
- 과도한 높이 금지
- 사진과 영상 구분이 명확해야 함

7. 사진 업로드 흐름
--------------------------------------------------

[사진 - 반전체앨범]

사진 업로드 기능:

- 삭제 후 재업로드 가능
- 업로드 완료 버튼 필요

--------------------------------------------------

영상 업로드는 사진과 별도 카드로 구성한다.

[영상 - 반전체앨범]

- 시간표 화면 표시

9. 시간표 공통 컴포넌트로 통일
--------------------------------------------------

아래 5개 화면은 모두 같은 시간표 컴포넌트를 재사용한다.
- 내반
- 출결
- 수영일지
- 사진 업로드 경로
- 영상 업로드 경로

공통 시간표 컴포넌트 규칙:
- 요일 탭 공통
- 시간 슬롯 공통
- 활성/비활성 상태 공통
- 학생 수 뱃지 공통
- 클릭 시 각 기능 페이지로 라우팅만 다르게 처리

예시:
- 내반에서 시간 클릭 -> 학생목록
- 출결에서 시간 클릭 -> 출결목록
- 수영일지에서 시간 클릭 -> 일지작성
- 사진에서 시간 클릭 -> 사진 업로드 대상 선택
- 영상에서 시간 클릭 -> 영상 업로드 대상 선택

이렇게 하나의 공통 구조를 쓰면 정렬 오류, 연동 오류, UI 불일치가 크게 줄어든다.

--------------------------------------------------
11. 성능/연동 최적화
--------------------------------------------------

반드시 수정할 것:
- 관리자 저장 직후 teacher/class/student 매핑 데이터 재계산
- 선생님 탭 진입 시 최신 데이터 fetch
- local state만 믿지 말고 저장 후 source of truth 재조회
- 학생 이동/반변경/시간변경 시 이전 반에서도 즉시 제거
- 삭제된 학생은 시간표 학생수에서도 즉시 제외

권장:
- optimistic update + 저장 성공 후 서버 재동기화
- 공통 query key 사용
- 반/학생/출결/일지 데이터 캐시 분리
- UI에서 정렬은 항상 서버 응답 후 한 번 더 정렬 보정

--------------------------------------------------
12. UI 개선 권장안
--------------------------------------------------

내가 추천하는 더 나은 방식:

기존 방식:
- 카드가 크고 메뉴별 성격이 제각각이라 현장에서 느림

권장 방식:
- "시간표 우선 구조 + 기능별 진입만 다름"
이 구조가 선생님 사용성 훨씬 좋음

추가 추천:
1. 오늘 수업 우선 표시
2. 가장 최근 사용한 반 상단 고정
3. 학생 수 표기
4. 미작성 일지 뱃지 표시
5. 출결 미완료 반 빨간 점 표시
6. 사진/영상 업로드 완료 여부 표시
7. 선생님이 자주 보는 반 즐겨찾기 가능하면 추가

가장 중요한 원칙:
선생님은 예쁜 화면보다
"지금 수업반 빨리 찾기"
"출결 빨리 끝내기"
"일지 바로 쓰기"
이 3개가 훨씬 중요하다.
그래서 큰 카드형보다 압축형 실무 UI가 맞다.

--------------------------------------------------
13. 개발 우선순위
--------------------------------------------------

1차 수정:
- 내반 시간표 구조
- 관리자-선생님 반배정 즉시 연동
- 카드 크기 및 정렬 수정

2차 수정:
- 출결 시간표 구조
- 모두출석 / 완료 버튼
- 저장 후 닫기 동작

3차 수정:
- 수영일지 작성 + 지난일지보기 + 취소하고나가기

4차 수정:
- 사진 2개 카드
- 영상 2개 카드
- 반전체/개인 업로드 분기

--------------------------------------------------
14. 꼭 반영할 세부 문구 및 동작
--------------------------------------------------

필수 버튼:
- 모두출석
- 완료
- 지난일지보기
- 일지취소하고나가기

필수 화면 분기:
- 개설된 반만 활성화
- 시간표 클릭 -> 학생목록 또는 작성화면 이동
- 학생목록 클릭 -> 학생정보
- 사진/영상은 반전체 / 개인 선택 분리

최종 목표:
선생님이 앱을 열었을 때
"오늘 내 수업 찾기 -> 학생 확인 -> 출결 체크 -> 일지 작성 -> 사진/영상 업로드"
이 흐름이 1~2분 안에 끝나도록 설계한다.

위 요구사항 기준으로 기존 선생님 화면 UI와 데이터 연결 구조를 전면 수정하고,
중복 컴포넌트는 공통화해서 유지보수 가능하게 리팩토링해줘.
기존 기능을 깨지 않도록 하면서 단계적으로 적용하고,
반드시 관리자 회원등록/반배정 결과가 선생님 화면에 즉시 반영되게 수정해줘.
