지금부터 앱 전체 UX/UI 재구성 작업을 시작하라.

이번 작업 목표는
지금까지 만든 기능과 목록, 탭, 연결 로직은 절대 깨지지 않게 유지하면서
전체 UX/UI를 우리가 확정한 공통 규칙에 맞게 다시 구성하는 것이다.

가장 중요:
- 기존 기능은 한 개도 망가지면 안 된다
- 기존 탭과 메뉴 목록도 임의 삭제 금지
- 기능 연결, 버튼 연결, 저장 구조, 승인 구조, 정산 구조, 수업 구조, 회원 구조 모두 유지
- 디자인/배치/진입 구조는 재구성 가능
- 기능 목적은 유지해야 함
- 예: 주간 스케줄표는 디자인이 바뀌어도 “주간 스케줄을 보고 선택해서 수업/출결/일지/이동 처리”라는 목적은 그대로 살아 있어야 한다
- 실수 줄이기 위해 작업 전 구조 분석 먼저 하고, 체크포인트마다 저장하며 진행할 것

==================================================
[0. 작업 시작 전 - 질문 대신 먼저 분석할 것]

작업 전에 사용자에게 다시 물어보지 말고
아래를 먼저 분석해서 출력하라.

1. 전체 파일 구조 트리
2. 모드별 탭 구조
   - 관리자
   - 선생님
   - 학부모
   - 슈퍼관리자
3. 공통 레이아웃/헤더/탭/팝업 관련 파일
4. 기능별 핵심 화면 파일
   - 사람/회원/승인
   - 수업/주간 스케줄/출결/일지
   - 메신저
   - 매출관리/정산
   - 저장공간/데이터관리
   - 더보기
5. 현재 UX 규칙과 충돌하는 화면 목록
6. 반드시 보존해야 하는 기능 목록

이 분석 결과를 먼저 출력한 뒤 작업 시작

==================================================
[1. 이번 작업의 절대 원칙]

1. 기능 보존 100%
2. 탭/목록 보존 100%
3. 기능 목적 보존 100%
4. UX 동작 규칙만 통일
5. 디자인/배치 재정리 가능
6. 기능 삭제 금지
7. 기능 연결 끊기 금지
8. 데이터 구조 변경 최소화
9. 모든 모드 같은 앱처럼 보이게 통일
10. 체크포인트마다 저장

==================================================
[2. 적용할 공통 UX/UI 규칙]

이번 재구성은 아래 규칙을 전부 반영해야 한다.

--------------------------------------
[2-1. 하단 탭 규칙]

- 하단 탭은 항상 고정
- 현재 탭 재탭 → 해당 탭 첫 화면 + 스크롤 초기화
- 다른 탭 이동 → 기존 탭 상태 유지
- 어떤 상세 화면 안에서도 탭 버튼은 항상 활성화
- 하단 탭 디자인/간격/아이콘/라벨 스타일 통일

--------------------------------------
[2-2. 뒤로가기 규칙]

- 뒤로가기는 현재 탭 내부에서 한 단계씩만 이동
- 절대 홈 탭이나 다른 탭으로 바로 튀지 않음
- 반복 뒤로가기의 마지막은 항상 “현재 탭의 첫 화면”
- 특히 더보기 내부 메뉴는 마지막 복귀점이 반드시 더보기 첫 화면

--------------------------------------
[2-3. 팝업 vs 페이지 기준]

팝업:
- 현재 흐름을 유지해야 할 때
- 선택/확인/짧은 입력
- 스크롤 많아도 가능
- 예: 문장 선택, 옵션 선택, 기간 선택

페이지:
- 독립된 흐름
- 다음 단계 존재
- 저장/완료 필요
- 주제가 확장되는 경우

기준 한 줄:
“이 화면에서 나가면 안 되면 팝업, 흐름이 이어지면 페이지”

--------------------------------------
[2-4. 상단 버튼 규칙]

- 탭 첫 화면 제외하고 페이지에는
  좌측 상단 뒤로가기(←)
  우측 상단 X
  둘 다 둘 수 있다
- 둘 다 결과가 같아도 됨
- 바깥 터치 종료는 팝업에만 허용
- 페이지는 바깥 터치 종료 금지

--------------------------------------
[2-5. 홈 화면 규칙]

모든 모드 홈은 같은 구조:
1. 업데이트/요약 카드
2. 필수 바로가기
3. 최근 작업/최근 변화

즉:
홈 = “지금 뭐 해야 하는지 + 자주 쓰는 것”

--------------------------------------
[2-6. 리스트/상세 규칙]

리스트:
- 가볍게 탐색
- 검색창 상단 고정
- 필터/칩 상단

상세:
- 액자형 구조
- 상단 요약 카드
- 상태 표시
- 액션 버튼
- 세부 정보

즉:
리스트 → 상세(액자형) → 기능 실행

--------------------------------------
[2-7. 옵션 선택 카드 규칙]

- 옵션은 카드형 선택 우선
- 터치 영역 넓게
- 선택 상태 명확하게
- 토글보다 카드/체크 구조 우선

--------------------------------------
[2-8. 검색창 위치 규칙]

- 리스트 화면: 상단 고정
- 홈: 기본적으로 없음
- 더보기: 내부 진입 후 필요한 화면에만
- 팝업 검색도 가능하나 제목 아래 배치

==================================================
[3. 모드별 재구성 목표]

--------------------------------------
[3-1. 관리자 모드]

탭 구조 유지:
- 홈
- 사람
- 메신저
- 매출관리
- 더보기
(현재 실제 구조 분석 후 유지)

목표:
- 홈은 요약/업데이트/바로가기 중심
- 사람은 회원/학부모/선생님/승인 액자 구조
- 매출관리는 월/휴무일/선생님별 매출내역 구조
- 더보기는 설정/운영/데이터관리 허브
- 데이터관리는 목록 → 상세 진입 구조
- 이벤트기록/킬스위치/저장공간은 데이터관리 안에서 구조화

--------------------------------------
[3-2. 선생님 모드]

탭 구조 유지:
- 홈
- 수업
- 메신저
- 매출관리
- 더보기
(현재 실제 구조 유지)

목표:
- 홈은 오늘 할 일
- 수업은 주간 스케줄표 중심
- 스케줄표 기능 목적 절대 훼손 금지
- 반 진입 후 출결/일지/학생상태/이동처리 유지
- 매출관리 구조는 관리자와 항목 일치
- 더보기는 설정/역할전환/개인도구 중심

--------------------------------------
[3-3. 학부모 모드]

탭 구조 유지:
- 홈
- 수업피드백
- 앨범
- 출결
- 더보기
(현재 구조 분석 후 유지)

목표:
- 홈은 자녀 요약 + 최근 소식 + 바로가기
- 수업피드백/앨범/출결은 카드/액자형 일관성
- 더보기는 공지/레벨/설정/고객센터 구조화

--------------------------------------
[3-4. 슈퍼관리자 모드]

현재 구조 분석 후 유지하되
관리자/선생님/학부모와 같은 UX 규칙 적용

목표:
- 홈은 플랫폼 요약
- 리스트는 액자형
- 설정/운영/구독은 더보기 또는 주제별 페이지로 정리
- 같은 앱처럼 보이게 통일

==================================================
[4. 기능 절대 보존 항목]

아래 기능들은 목적과 연결이 절대 망가지면 안 된다.

1. 주간 스케줄표
2. 반 선택 후 출결/결석/보강 처리
3. 일지 작성/수정/조회
4. 학생 상태 보기
5. 임시 이동 처리
6. 미등록 교체
7. 결근 처리
8. 메신저 전체/대상/사진/회원이전
9. 관리자 승인/부관리자 지정/해제/수업인수
10. 관리자/선생님 매출관리 연결
11. 휴무일 지정
12. 저장공간 계산
13. 킬 스위치
14. 이벤트 기록
15. 역할 전환
16. 뒤로가기/탭 상태 유지 규칙

==================================================
[5. 재구성 방식]

반드시 아래 방식으로 작업하라.

1. 기존 기능 코드는 최대한 재사용
2. 공통 레이아웃/헤더/카드 구조로 감싸는 방식 우선
3. 기능을 새로 만들기보다 진입 구조와 화면 배열을 바꾼다
4. 중복 노출은 줄이되 기능 삭제 금지
5. 리스트 → 상세 → 기능 실행 흐름으로 재배치
6. 더보기는 허브형 구조로 정리
7. 데이터관리/운영/설정처럼 묶을 수 있는 것은 묶는다

==================================================
[6. 체크포인트 저장 규칙]

체크포인트를 반드시 남기고 진행하라.

체크포인트 1:
전체 구조 분석 완료

체크포인트 2:
공통 UX 규칙 적용용 레이아웃 정리 완료

체크포인트 3:
관리자 모드 재구성 완료

체크포인트 4:
선생님 모드 재구성 완료

체크포인트 5:
학부모 모드 재구성 완료

체크포인트 6:
슈퍼관리자 모드 재구성 완료

체크포인트 7:
전체 기능 연결 테스트 완료

==================================================
[7. 테스트 - 반드시 실제 수행]

이번 작업 후 반드시 아래를 실제 테스트하라.

--------------------------------------
테스트 1. 탭 동작
- 현재 탭 재탭 시 첫 화면 복귀 + 스크롤 초기화
- 다른 탭 이동 시 상태 유지

--------------------------------------
테스트 2. 뒤로가기
- 각 탭 내부에서 한 단계씩 뒤로 이동
- 마지막은 현재 탭 첫 화면
- 더보기 내부는 마지막이 반드시 더보기 첫 화면

--------------------------------------
테스트 3. 팝업/페이지 구분
- 문장 선택/기간 선택/옵션 선택은 팝업 유지
- 독립 흐름 화면은 페이지로 구성
- 바깥 터치 종료는 팝업만

--------------------------------------
테스트 4. 관리자 기능 보존
- 사람/승인/부관리자 지정/수업인수
- 매출관리/휴무일/데이터관리
- 킬 스위치/이벤트 기록

--------------------------------------
테스트 5. 선생님 기능 보존
- 주간 스케줄표
- 반 진입 후 출결/일지/학생 상태/이동 처리
- 매출관리
- 메신저

--------------------------------------
테스트 6. 학부모 기능 보존
- 홈
- 수업피드백
- 앨범
- 출결
- 더보기

--------------------------------------
테스트 7. 저장공간/로그 보존
- 저장공간 수치 정상
- 킬 스위치 실행 후 로그 남음
- 이벤트 기록 표시 정상

--------------------------------------
테스트 8. 역할 전환
- 권한 있는 계정만 전환 버튼 노출
- 전환 후 해당 모드 첫 화면 진입

작업 후 아래 순서로 보고하라.

2. 수정한 파일 목록
3. 공통 UX 규칙을 어떻게 반영했는지
4. 관리자 모드 재구성 내용
5. 선생님 모드 재구성 내용
6. 학부모 모드 재구성 내용
7. 슈퍼관리자 모드 재구성 내용
8. 보존한 기능 목록
9. 테스트 1~8 결과
10. 아직 남은 UX 불일치/후속 작업 항목

==================================================
[9. 절대 금지]

- 기능 삭제 금지
- 탭 삭제 금지
- 목록 삭제 금지
- 기능 목적 훼손 금지
- 주간 스케줄표 기능 축소 금지
- 승인/정산/수업/저장공간/로그/역할전환 로직 훼손 금지
- 임시로 되는 것처럼만 보이게 하는 가짜 수정 금지

이번 작업은
지금까지 만든 기능을 하나도 망가뜨리지 않고
전체 UX/UI 규칙을 전 모드에 적용해
같은 앱처럼 느껴지게 재구성하는 작업이다.
