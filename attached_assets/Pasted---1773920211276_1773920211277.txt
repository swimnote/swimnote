아래 내용을 하나의 작업으로 이어서 구현해주세요.
우선 기능을 먼저 연결하고, 세부 버그는 이후 단계에서 잡아가겠습니다.
중요한 것은 지금 단계에서 구조와 흐름이 실제로 작동하게 만드는 것입니다.

[1. 관리자모드 수업 화면 구조 통일]

관리자모드의 수업 화면을 현재 선생님모드와 최대한 동일한 UI 구조로 맞춰주세요.

공통으로 동일하게 유지할 요소:
- 상단 헤더 영역
- 공지 / 보강 / 수강생관리 / 반등록 버튼 영역
- 일간 / 주간 / 월간 탭 구조
- 주간 시간표 그리드
- 반 카드 디자인
- 하단시트(바텀시트) 스타일
- 전체 여백과 카드 톤
- 하단 탭바 구조

즉, 관리자와 선생님은 같은 수업 UI를 사용하고 권한과 버튼 기능만 다르게 적용해주세요.

--------------------------------------------------
[2. 관리자모드 반 카드 클릭 시 기능]

관리자모드에서 주간/일간/월간 수업표의 반 카드를 누르면
선생님모드처럼 하단시트가 열리게 해주세요.

관리자모드 하단시트 상단 표시 항목:
- 반 이름
- 요일 / 시간
- 담당 선생님명
- 현재 인원 / 정원
- 학생 목록

관리자모드 하단시트에서는 아래 버튼만 노출해주세요:
- 반배정
- 미등록
- 반이동

아래 버튼은 관리자모드에서는 제거해주세요:
- 출결
- 수업일지
- 결석

즉 관리자모드는 수업운영 기록용이 아니라
반편성 / 배정 / 이동 / 담당지정 중심으로만 사용하게 해주세요.

--------------------------------------------------
[3. 관리자모드 반배정 기능]

관리자모드에서도 반배정 기능은 유지해주세요.

동작 방식:
- 관리자모드 하단시트에서 `반배정` 클릭
- 선생님모드의 반배정 화면과 같은 방식으로 이동
- 학생 리스트를 보여주고 선택해서 해당 반에 넣을 수 있게 해주세요
- 추가 버튼 방식, 리스트 표시 방식, 저장 후 즉시 반영 방식도 선생님모드와 동일하게 맞춰주세요

표시 항목 예시:
- 학생 이름
- 연락처 일부
- 현재 배정 수
- 주당 횟수
- 추가 버튼

배정 후 처리:
- 해당 반 인원 수 즉시 갱신
- 반 상세 학생 목록 즉시 갱신
- 주간/일간/월간에서도 즉시 반영
- 새로고침 없이 반영

--------------------------------------------------
[4. 관리자모드 미등록 기능]

관리자모드 하단시트에서 `미등록` 버튼을 누르면
미등록 또는 미배정 상태 회원 리스트를 띄워주세요.

기능 목적:
- 다음달 수업 편성을 위해 미리 회원을 반에 넣는 용도
- 아직 정식 배정되지 않은 회원을 반에 연결하는 용도

동작 방식:
- 선생님모드의 리스트업 구조와 비슷하게 구성
- 회원 선택 후 해당 반에 넣을 수 있게 처리
- 저장 후 즉시 반영

빈 상태 문구:
- `미등록 회원이 없습니다.`

--------------------------------------------------
[5. 관리자모드 반이동 기능]

관리자모드 하단시트에서 `반이동` 버튼을 누르면
기존 다른 반에 속한 학생을 현재 반으로 이동할 수 있게 해주세요.

동작 방식:
- 이동 가능한 학생 리스트 표시
- 선택해서 현재 반으로 이동
- 이동 후 기존 반에서는 제거
- 새 반에는 즉시 반영
- 관련 인원 수 즉시 갱신
- 새로고침 없이 반영

빈 상태 문구:
- `이동 가능한 학생이 없습니다.`

--------------------------------------------------
[6. 담당 선생님 지정 / 변경 기능 추가]

관리자모드에서는 반의 담당 선생님을 지정하거나 변경할 수 있게 해주세요.
이 기능은 관리자에게만 허용합니다.
선생님모드에서는 담당 선생님 변경 불가입니다.

기능 목적:
- 신규 입사 예정 선생님이 있는 경우
- 퇴사 예정 선생님이 있는 경우
- 선생님 공석 상태에서 다음달 반편성을 미리 잡아야 하는 경우
- 미지정 반도 먼저 개설하고 학생 배정 구조를 만들어야 하는 경우

규칙:
- 담당 선생님이 없어도 반은 존재할 수 있음
- 담당 선생님이 없는 반은 `미지정`으로 표시
- 미지정 반도 반배정 / 미등록 / 반이동 가능해야 함

기능 방식:
- 관리자모드 하단시트 또는 반 상세 상단에서 담당선생님 영역 클릭 가능하게 처리하거나
- `선생님 지정` 또는 `담당 변경` 버튼을 추가해주세요
- 누르면 선생님 리스트가 열리고 선택 가능
- 선택 후 저장하면 반 카드 / 상세 / 주간표에 즉시 반영
- 새로고침 없이 반영

선생님 리스트 기준:
- 현재 활성 상태 선생님 기준으로 우선 구현
- 이후 재직예정/퇴사예정 세부 조건은 나중에 추가 가능하게 구조 열어두기

--------------------------------------------------
[7. 수강생관리 버튼 유지]

상단의 `수강생관리` 버튼은 관리자모드에도 선생님모드와 동일한 위치에 유지해주세요.

동작 방식:
- 버튼 클릭 시 바텀시트 오픈
- 탭 2개 유지
  1. 미배정회원 리스트
  2. 보강대기리스트

기본 진입 탭:
- 미배정회원 리스트

관리자모드에서는 이 리스트를 조회 + 배정 연결용으로 사용 가능하게 해주세요.
이미 반배정 기능을 유지하기로 했으므로, 관리자에서도 필요한 경우 이 리스트를 통해 이동 가능하게 해주세요.

--------------------------------------------------
[8. 미배정회원 리스트 규칙]

표시 대상:
- 현재 수영장에 등록된 회원 중 반 배정이 아직 완료되지 않은 회원

배정 완료 기준:
- 주1회 회원: 1개 반 배정 완료 시 제거
- 주2회 회원: 2개 반 배정 완료 시 제거
- 주3회 회원: 3개 반 배정 완료 시 제거

중요:
- 일부만 배정된 상태면 계속 리스트에 남아야 함
- 남은 배정 횟수가 0이 될 때만 제거

각 항목 표시:
- 학생이름
- 보호자명 또는 연락처 일부
- 주당 횟수
- 현재 상태 `미배정`
- 남은 배정 횟수

액션:
- `반배정` 버튼 클릭 시 바로 반 선택 또는 해당 반 연결 화면으로 이동
- 저장 완료 후 주간 기본 화면으로 복귀
- 리스트/수업표 데이터 즉시 갱신

빈 상태 문구:
- `현재 미배정 회원이 없습니다.`

--------------------------------------------------
[9. 보강대기리스트 규칙]

표시 대상:
- 결석 처리되어 보강 가능 상태인 회원
- 아직 보강 반 배정이 완료되지 않은 회원

각 항목 표시:
- 학생이름
- 원래 소속 반
- 결석일
- 보강 가능 여부
- 현재 상태 `보강대기`

정렬:
- 결석일 오름차순
- 가장 오래 기다린 회원이 위로 오게 정렬

액션:
- `보강배정` 클릭 시 바로 보강 가능한 반 선택 화면으로 이동
- 정원이 찬 반은 선택 불가 또는 빨간색 표시
- 정원 여유 반만 선택 가능
- 저장 완료 후 주간 기본 화면으로 복귀
- 리스트/수업표 데이터 즉시 갱신

빈 상태 문구:
- `현재 보강 대기 회원이 없습니다.`

--------------------------------------------------
[10. 저장 후 복귀 규칙]

반배정 / 보강배정 / 반이동 / 담당선생님 변경 저장 후에는
항상 수업 화면의 기본 상태로 복귀하게 해주세요.

복귀 기준:
- 주간 화면 기본 수업표로 복귀
- 선택 상태 해제
- 팝업 닫힘
- 필터 초기화
- 변경 결과는 즉시 반영

즉 저장 후 중간 상태가 남지 않고 깔끔한 기본 수업표 상태로 돌아오게 해주세요.

--------------------------------------------------
[11. 다음달 스케줄 생성 기준]

반편성 기준은 지금 당장 월별 고정 규칙으로 박지 말고
`정산 실행 시점`을 다음달 스케줄 생성의 기준점으로 처리해주세요.

기본 원칙:
- 관리자가 정산을 누르면 다음달 수업 스케줄 생성
- 정산 전에도 관리자는 다음달 반 / 수업 / 배정을 미리 일부 만들어둘 수 있음
- 담당 선생님 미지정 반도 미리 생성 가능
- 학생 배정도 일부 선반영 가능

즉, 다음달 데이터는
1) 정산 전에 수동으로 미리 만든 데이터
2) 정산 시 전달 기준으로 자동 생성되는 데이터
이 둘이 병합되어 완성되는 구조로 만들어주세요.

--------------------------------------------------
[12. 정산 시 다음달 스케줄 병합 규칙]

정산 실행 시 전달의 반복 수업을 다음달로 자동 생성해주세요.

단, 이미 정산 전에 다음달 스케줄을 일부 미리 만들어둔 경우
기존 데이터를 덮어쓰지 말고 병합 방식으로 처리해주세요.

핵심 규칙:
- 전달에서 넘어오는 수업과
- 정산 전에 미리 생성한 다음달 수업이
- 중복되지 않는 경우에만 자동 추가 생성

중복되는 경우:
- 새로 만들지 않음
- 기존 수동 생성 데이터 유지
- 중복 생성 금지

즉,
자동 생성은 수동 생성 데이터를 보존하면서
충돌 없는 수업만 추가하는 방식으로 구현해주세요.

--------------------------------------------------
[13. 중복 판단 기준]

다음 항목을 기준으로 중복 여부를 판단해주세요.

우선 기준:
- 연월
- 수영장 ID
- 반 ID 또는 반 템플릿 ID
- 요일
- 시간
- 반 이름

담당 선생님은 변경 가능성이 있기 때문에
중복 판단의 절대 기준이 아니라 보조 비교값으로 처리해주세요.

이유:
같은 금 19:00반인데 담당 선생님만 바뀌는 경우가 생길 수 있으므로
담당 선생님을 절대 중복키로 쓰면 같은 반이 중복 생성될 수 있습니다.

--------------------------------------------------
[14. 정산 전 사전 편성 허용]

정산 전에 다음달 반편성을 미리 만들어둔 경우 아래가 가능해야 합니다.

- 빈 반 생성
- 미지정 선생님 반 생성
- 다음달 학생 일부 반배정
- 다음달 반이동 구조 준비
- 담당 선생님 나중 지정

즉, 아직 다음달이 확정 전이어도 관리자가 스케줄 틀을 먼저 만들 수 있어야 합니다.

--------------------------------------------------
[15. 정산 시 결과 처리]

정산 버튼 실행 후 결과는 아래처럼 처리해주세요.

- 기존에 미리 만든 다음달 수업 유지
- 전달에서 자동 이월된 수업 중 중복되지 않는 것만 추가
- 결과적으로 하나의 다음달 수업표로 합쳐짐
- 인원, 반 목록, 담당선생님 정보 즉시 반영
- 관리자모드와 선생님모드에서 동일 데이터 참조

--------------------------------------------------
[16. 권한 및 공통 처리]

공통:
- 새로고침 없이 데이터 반영
- 주간/일간/월간에서 동일 데이터 참조
- 관리자모드에서 수정한 내용은 선생님모드에도 바로 보이게 처리

관리자모드:
- 반배정 가능
- 미등록 가능
- 반이동 가능
- 담당선생님 변경 가능
- 출결 / 수업일지 / 결석 기능 없음

선생님모드:
- 기존 구조 유지
- 출결 / 수업일지 / 결석 / 반배정 / 미등록 / 반이동 기능 유지
- 담당선생님 변경 불가

--------------------------------------------------
[17. 예외 처리]

- 담당 선생님이 없는 반은 `미지정` 표시
- 학생이 없는 반도 열람 가능
- 빈 반도 반배정 가능
- 배정 가능한 학생이 없으면 `배정 가능한 회원이 없습니다.`
- 미등록 가능한 학생이 없으면 `미등록 회원이 없습니다.`
- 반이동 가능한 학생이 없으면 `이동 가능한 학생이 없습니다.`
- 미배정회원 리스트가 비면 `현재 미배정 회원이 없습니다.`
- 보강대기리스트가 비면 `현재 보강 대기 회원이 없습니다.`

--------------------------------------------------
[18. 구현 우선순위]

이번 단계는 완벽한 고도화보다 실제 작동 우선입니다.
우선 아래 순서로 구현해주세요.

1순위:
- 관리자모드 하단시트 버튼 구조 변경
- 반배정 / 미등록 / 반이동 연결
- 담당선생님 지정/변경 연결
- 저장 후 즉시 반영

2순위:
- 수강생관리 리스트 연동 정리
- 미배정회원 / 보강대기리스트 정렬 및 빈 상태 처리

3순위:
- 정산 시 다음달 스케줄 병합 생성 로직

중요:
- 버튼을 눌렀을 때 실제로 이동/저장/반영이 되어야 함
- 화면끼리 연결이 끊기지 않아야 함
- 더미처럼 보이지 않고 실제 데이터 흐름이 이어져야 함
- 한 기능 저장 시 관련 리스트/반 카드/인원 수가 동시에 갱신되어야 함

[20. 작업 후 확인 항목]

작업 후 아래를 꼭 직접 확인해주세요.

- 관리자모드에서 반 카드 클릭 시 하단시트가 정상 열리는지
- 하단시트에 반배정 / 미등록 / 반이동만 보이는지
- 관리자모드에서 담당선생님 지정/변경이 되는지
- 저장 후 주간표에 바로 반영되는지
- 미배정회원 리스트가 규칙대로 줄어드는지
- 보강대기리스트가 결석일순으로 정렬되는지
- 정산 전 다음달 수업을 미리 만들어도 유지되는지
- 정산 시 전달 수업이 중복 없이 다음달에 병합 생성되는지
- 관리자에서 만든 변경사항이 선생님모드에도 보이는지

체크하지 않은 상태로 완료 처리하지 말고,
실제로 눌러보고 데이터 반영까지 확인한 뒤 진행해주세요.
