학부모 가입/승인/로그아웃/뒤로가기 네비게이션 버그를 한 번에 수정해라.

현재 확인된 버그:
1. 학부모 회원가입은 완료되지만 승인 전 로그인 시 "가입한 계정 없음"처럼 처리되거나 회원가입 화면으로 이동함
2. 관리자 승인 후 학부모 홈 진입은 되지만 로그아웃 또는 뒤로가기 시 로그인 화면으로 정상 복귀하지 않음
3. 학부모 화면에서 뒤로가기를 누르면 로그인 화면 대신 빈 회색 화면이 나옴
4. 즉, 인증 상태 분기와 네비게이션 스택 관리가 같이 꼬여 있음

수정 목표:
- 승인 전/승인 후/미가입 상태를 명확히 분리
- 로그인/로그아웃/뒤로가기 시 무조건 올바른 화면으로 이동
- 빈 화면이 절대 뜨지 않게 공통 fallback 처리
- 인증 상태와 네비게이션 스택을 일관되게 정리

반드시 수정할 항목:

1. 계정 상태 분리
- not_found: 계정 없음
- pending: 가입 완료, 승인 대기
- approved: 승인 완료
- rejected: 거절됨(있으면)

2. 로그인 분기 수정
- DB에 계정이 있으면 절대 not_found로 보내지 말 것
- pending이면 승인대기 화면 또는 제한 화면으로 이동
- approved이면 학부모 홈으로 이동
- not_found일 때만 회원가입 화면으로 이동

3. 로그아웃 처리 수정
- accessToken, refreshToken, session, currentUser, role, parentId, selectedChildId, selectedPoolId 등 전부 삭제
- AsyncStorage / SecureStore / localStorage / zustand / redux / context 전부 초기화
- auth listener 남아있으면 정리
- logout 후 navigate/back 사용 금지
- 반드시 인증 스택의 첫 화면(로그인 화면)으로 reset 처리

4. 뒤로가기 처리 수정
- 학부모 홈에서 뒤로가기 시 빈 화면으로 가지 않게 할 것
- 인증 이후 진입한 홈 화면은 뒤로가기로 이전 비정상 스택이 남지 않게 replace 또는 reset 구조로 바꿀 것
- 로그인 화면에서만 뒤로가기 허용 또는 앱 종료 처리
- 홈 화면 진입 전에 불완전한 중간 페이지가 스택에 남지 않게 정리할 것

5. 빈 화면 방지
- 인증 가드, role guard, parent guard, child 연결 guard 에서 null만 반환하지 말 것
- 로딩 중이면 로딩 화면
- 미인증이면 로그인 화면
- 승인 대기면 승인대기 화면
- child 미연결이면 자녀 연결 안내 화면
- 어떤 경우에도 회색 빈 화면만 보이는 상태 금지

6. 라우팅 규칙 통일
- 회원가입 완료 -> 승인대기 안내
- 승인 후 로그인 성공 -> 학부모 홈 reset
- 로그아웃 -> 로그인 화면 reset
- 뒤로가기 -> 유효한 이전 화면 없으면 빈 화면이 아니라 로그인 화면 또는 앱 종료 처리

7. 실제 점검 항목
- 새 학부모 계정 회원가입
- 승인 전 로그인 시도
- "가입한 계정 없음"이 아닌 "승인 대기" 또는 제한 화면 표시
- 관리자 승인
- 같은 계정 로그인
- 학부모 홈 진입
- 뒤로가기 클릭
- 로그인 화면 또는 정의된 화면으로 정상 이동
- 빈 회색 화면이 나오면 실패
- 로그아웃 클릭
- 로그인 화면으로 이동
- 앱 재실행 시 자동 로그인 여부 확인
- 뒤로가기 눌러도 홈 재진입 안 되는지 확인

추가로 코드 점검:
- navigation.navigate, router.back, router.push 만으로 처리한 곳 전부 확인
- 로그인/로그아웃/권한 변경 시 반드시 replace 또는 reset 이 필요한지 재검토
- conditional rendering 에서 return null 하는 구간 찾아서 전부 명시적 fallback 화면으로 교체
- auth 초기 로딩 상태와 비로그인 상태를 혼동하지 말 것
