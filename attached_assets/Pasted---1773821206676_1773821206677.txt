다음 작업은 관리자 모드 매출관리와 선생님 모드 매출관리의 항목/데이터/상태를 일치시키고 서로 연결하는 것이다.

[목표]
- 선생님 모드에서 월별 정산 데이터를 저장/제출할 수 있게 한다
- 관리자 모드에서는 같은 월 기준으로 선생님별 제출 결과를 확인할 수 있게 한다
- 관리자/선생님 화면의 집계 항목과 계산 기준을 동일하게 맞춘다

==================================================
[1. 공통 항목 통일]

관리자 모드 매출관리와 선생님 모드 매출관리에서
아래 항목을 동일하게 사용하라.

공통 항목:
- 해당 월 매출결산액
- 수업시간
- 수업인원
- 보강
- 체험수업
- 이동
- 연기
- 탈퇴
- 정산 상태

관리자:
- 모든 선생님 목록 표시

선생님:
- 자기 데이터만 표시

==================================================
[2. 선생님 모드 매출관리]

선생님 모드 매출관리 화면에 아래 기능을 구성하라.

- 월 선택
- 월별 정산 요약
- 본인 정산 항목 카드
- 회원별 정산 내역
- 기타 수기 정산
- 저장 버튼
- 이번 달 정산 제출 버튼

정산 상태:
- 미정산
- 저장됨
- 제출완료

최소 구현:
- 미정산
- 제출완료
우선 가능

==================================================
[3. 관리자 모드 매출관리]

관리자 모드 매출관리 화면에서는
현재 만든 “선생님별 매출내역” 목록을 유지하되
선생님 모드 제출 데이터와 연결하라.

각 선생님 카드에는 아래를 표시:
- 선생님 이름
- 해당 월 매출결산액
- 수업시간
- 수업인원
- 보강
- 체험수업
- 이동
- 연기
- 탈퇴
- 정산 상태

정산 상태는 선생님 제출 여부와 연결할 것

예:
- 아직 제출 안 함 → 미정산
- 제출 완료 → 제출완료

==================================================
[4. 월별 저장]

정산 데이터는 월별로 저장되게 한다.

요구사항:
- 2026년 03월 저장
- 2026년 04월 별도 저장
- 다시 03월로 돌아오면 03월 데이터 유지
- 관리자와 선생님 모두 같은 월을 보면 같은 기준 데이터를 확인 가능

==================================================
[5. 제출 연결]

선생님이 해당 월 정산을 제출하면:

1. 선생님 월 정산 상태가 제출완료로 바뀜
2. 관리자 모드 같은 월 화면에서 해당 선생님 카드가 제출완료로 반영됨
3. 관리자는 월별로 누가 제출했고 누가 미제출인지 확인 가능해야 함

==================================================
[6. 계산 기준 통일]

관리자와 선생님의 숫자가 다르게 보이면 안 된다.
아래 항목은 반드시 같은 계산 기준을 써라.

- 수업시간
- 수업인원
- 보강
- 체험수업
- 이동
- 연기
- 탈퇴
- 매출결산액

작업 후 각 항목을 어떤 기준으로 계산했는지 보고하라.

==================================================
[7. 기존 화면 유지]

- 관리자 매출관리 상단의 “이번 달 총 매출” 카드 유지
- 휴무일 지정 버튼 유지
- 월 이동 UI 유지
- 기존 데이터 구조는 최대한 유지
- 기존 정산 기능 삭제 금지

==================================================
[8. 테스트]

반드시 실제 테스트하라.

테스트 1.
선생님 모드에서 월별 정산 항목이 보이는지 확인

테스트 2.
선생님 모드에서 저장 후 같은 월 다시 들어왔을 때 유지되는지 확인

테스트 3.
선생님 모드에서 제출 후 상태가 제출완료로 바뀌는지 확인

테스트 4.
관리자 모드 같은 월 화면에서 해당 선생님 카드가 제출완료로 반영되는지 확인

테스트 5.
다른 월로 이동 후 다시 돌아와도 월별 저장 데이터가 유지되는지 확인

테스트 6.
관리자와 선생님 화면의 항목 값이 같은 기준으로 보이는지 확인
