[스윔노트 회원가입 역할 선택 화면 — UX 수정 지시문]

목표:
회원가입 1단계(역할 선택 화면)의 가독성, 이해도, 가입 전환율 개선
기능 변경 없이 UI/UX만 개선

━━━━━━━━━━━━━━━━━━━━━━━

1. 역할 카드 문구 수정 (필수)
   ━━━━━━━━━━━━━━━━━━━━━━━

기존 문구 전부 제거 후 아래로 교체

[수영장 대표]
수영장을 직접 운영하는 원장/관리자
또는 1인 레슨 팀을 운영하는 선생님

[선생님]
스윔노트에 가입된 수영장에서 근무 중인 선생님
(수영장 대표의 초대 후 가입 가능)

[학부모]
스윔노트에 가입된 수영장에 자녀가 등록된 학부모
(회원 등록 완료 후 이용 가능)

조건 문구(괄호)는 반드시:

* 글자 크기 작게 (12~13px)
* 색상 회색 (#888 또는 #999)
* 줄바꿈해서 하단에 배치

━━━━━━━━━━━━━━━━━━━━━━━
2. 선택 상태 UI 추가 (핵심)
━━━━━━━━━━━━━━━━━━━━━━━

현재 문제:
선택 상태가 시각적으로 구분되지 않음

수정:

선택된 카드:

* 배경색: 민트 (앱 메인 컬러)
* 테두리: 민트 진한색 2px
* 아이콘: 흰색
* 텍스트: 흰색
* 카드 우측 상단 체크 아이콘 표시

선택 안 된 카드:

* 배경: 흰색
* 테두리: 연회색 (#E5E5E5)
* 아이콘: 기존 색 유지

━━━━━━━━━━━━━━━━━━━━━━━
3. 다음 버튼 상태 제어 (필수)
━━━━━━━━━━━━━━━━━━━━━━━

초기 상태:

* 비활성
* 색상: 회색 (#CCC)
* 클릭 불가

역할 선택 시:

* 활성화
* 색상: 민트
* 클릭 가능

※ 선택 없이 다음 넘어가는 동작 절대 불가

━━━━━━━━━━━━━━━━━━━━━━━
4. 카드 레이아웃 개선
━━━━━━━━━━━━━━━━━━━━━━━

* 카드 간 간격: 16~20px 확보
* 카드 내부 패딩: 16px 이상
* 아이콘과 텍스트 간 간격: 8~12px

텍스트 스타일:

* 제목: font-weight 600~700 (굵게)
* 설명: 14px
* 조건(괄호): 12~13px (작게)

━━━━━━━━━━━━━━━━━━━━━━━
5. 텍스트 구조 규칙 (중요)
━━━━━━━━━━━━━━━━━━━━━━━

각 카드 구성:

[아이콘]
[역할명 (굵게)]
[설명 1줄]
[조건 (작은 글씨)]

절대 금지:

* 문장 2줄 이상 길게 쓰기
* 조건을 본문에 섞기

━━━━━━━━━━━━━━━━━━━━━━━
6. 예외 처리 (필수)
━━━━━━━━━━━━━━━━━━━━━━━

선생님 / 학부모 선택 후 가입 진행 시:

해당 수영장이 스윔노트 미가입일 경우
→ 안내 화면 또는 토스트 표시:

"현재 근무 중인 수영장이 스윔노트에 가입되어 있어야 이용할 수 있습니다"

또는

"자녀가 등록된 수영장이 스윔노트에 가입되어 있어야 합니다"

━━━━━━━━━━━━━━━━━━━━━━━
7. 개발 방식 (중요)
━━━━━━━━━━━━━━━━━━━━━━━

* 기존 코드 수정 금지
* 새로운 컴포넌트로 UI 구성
* 기존 기능과 연결만 수행

작업 순서:

1. 현재 구조 분석
2. UI 컴포넌트 새로 생성
3. 선택 상태 로직 추가
4. 버튼 활성 조건 연결
5. 기존 흐름과 연결
6. 실제 터치/흐름 테스트
7. 문제 없을 시 기존 UI 제거

━━━━━━━━━━━━━━━━━━━━━━━
8. 최종 검증 체크리스트
━━━━━━━━━━━━━━━━━━━━━━━

* 역할 선택 시 시각적으로 바로 구분되는가
* 선택 안 하면 다음 버튼이 눌리지 않는가
* 문구가 2초 안에 이해되는가
* 조건 문구가 눈에 거슬리지 않는가
* 실제 가입 흐름에서 막히는 부분 없는가

회원가입 1단계에서 이탈률 감소 + 이해도 개선 + 브랜드 인지 강화
