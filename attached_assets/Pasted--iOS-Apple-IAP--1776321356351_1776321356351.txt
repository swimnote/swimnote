[스윔노트 iOS 리젝 — Apple 로그인 + IAP “확정 검증 지시문”]

지금부터 할 작업은 “수정”이 아니라
👉 **모든 원인을 확정적으로 검증해서 애플 심사에서 재리젝이 절대 안 나오게 만드는 것**이다.

중요:
- 원인 추측 금지
- “아마 이거 같습니다” 금지
- 반드시 “검증 완료된 사실 + 스크린샷 + 로그”로 보고

1. Apple 로그인 — 원인 확정 검증
━━━━━━━━━━━━━━━━━━

현재 상태:
- Apple 시트는 뜨지만 “Sign-Up Not Completed”
- 서버 도달 여부 불명확
- capability 문제로 추정되지만 확정 아님

해야 할 것:

[1] App ID 설정 검증 (스크린샷 필수)
- Apple Developer → Identifiers → com.swimnote.app
- Sign in with Apple 체크 여부 캡처
- Configure 화면 캡처
- Primary App ID 설정 여부 캡처

[2] Entitlement 검증 (이거 매우 중요)
- 빌드된 앱의 entitlements 확인
- “com.apple.developer.applesignin” 포함 여부 확인
- 포함 안 되어 있으면 → 빌드 설정 문제 확정

[3] Provisioning Profile 재생성
- 기존 프로파일 삭제
- 새로 Generate
- EAS 빌드에 반영 확인

[4] 실제 Apple 로그인 단계 로그 추적
다음 4단계 각각 로그 찍어서 제출:

① Apple signInAsync() 호출 시작
② Apple credential 반환 여부
③ 서버 요청 전 payload 로그
④ 서버 요청 도달 여부

👉 여기서 분기:
- credential 자체가 안 오면 → Apple 설정 문제
- credential 오는데 서버 안 가면 → 클라이언트 문제
- 서버까지 가는데 실패 → 서버 로직 문제

이걸로 “실패 위치 100% 특정”

[5] 테스트 (스크린샷 필수)
- iPhone 신규 설치
- iPad 신규 설치
- 업데이트 설치
- 재로그인

👉 각 단계 성공 화면 캡처 제출

2. IAP — 원인 확정 검증
━━━━━━━━━━━━━━━━━━

현재 상태:
- “구독 상품 로드 실패”
- RevenueCat offerings 실패

해야 할 것:

[1] App Store Connect 상태 검증 (스크린샷 필수)
- Paid Apps Agreement 상태 캡처 (Active)
- 각 구독 상품 상태 캡처 (Ready to Submit or Submitted)
- Subscription Group 설정 캡처
- App Review Screenshot 등록 여부 캡처

[2] Product ID 일치 검증
- 코드에서 사용하는 ID
- RevenueCat 등록 ID
- App Store Connect ID

👉 3개 완전 동일한지 표로 정리해서 제출

[3] RevenueCat 설정 검증
- Products 탭 캡처
- Offerings 탭 캡처
- entitlement 연결 캡처

[4] getOfferings() 실패 원인 로그
- 네트워크 에러인지
- 상품 없음인지
- 권한 문제인지

👉 에러 코드 그대로 제출

[5] 실제 sandbox 구매 테스트 (스크린샷 필수)
- 상품 로드 성공 화면
- 결제 시트
- 결제 성공 화면
- 구독 반영 화면
