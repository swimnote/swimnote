지금부터 관리자 모드와 선생님 모드의 “매출관리” 기능을 월별 선생님 정산 보고 구조로 개편하라.

가장 중요:
- 현재 화면 상단의 “이번 달 총 매출” 카드 영역은 절대 건드리지 말 것
- 현재 월 선택 UI(예: 2026년 03월)와 휴무일 지정 버튼도 유지할 것
- 수정 범위는 그 아래 영역부터다
- 기존 기능을 함부로 삭제하지 말고, 필요한 구조로 확장/연결할 것
- 관리자 모드와 선생님 모드가 서로 연결되어야 한다
- 반드시 실제 테스트까지 수행하고 보고할 것

==================================================
[1. 목표]

현재 관리자 모드 > 매출관리 화면에서
“회원별 수업 횟수” 영역을 아래 구조로 변경한다.

새 제목:
- 선생님별 매출내역

이 영역에서는 해당 월 기준으로
등록된 선생님들의 월 정산 결과를 카드형 또는 리스트형으로 보여준다.

또한 선생님 모드에도 동일한 매출관리 항목 구조를 만들어서
선생님이 자기 월 정산을 마치면
관리자에게 해당 월 정산내역을 보고하는 방식으로 연결한다.

==================================================
[2. 관리자 모드 > 매출관리 수정]

현재 구조:
- 이번 달 총 매출 (유지)
- 회원별 수업 횟수 (변경 대상)
- 기타 수기 정산 (가능하면 유지)

변경 후 구조:
1. 이번 달 총 매출  ← 유지, 수정 금지
2. 선생님별 매출내역  ← 새로 구성
3. 기타 수기 정산  ← 유지 가능

==================================================
[3. 관리자 모드 > 선생님별 매출내역 항목]

해당 월 기준으로 등록된 선생님별로 아래 항목을 표시하라.

각 선생님별 표시 항목:
- 선생님 이름
- 해당 월 매출결산액
- 수업시간
- 수업인원
- 보강 개수
- 체험 수업 개수
- 이동 개수
- 연기 개수
- 탈퇴 개수
- 정산 상태
  (예: 미정산 / 제출완료 / 관리자확인)

설명:
- 수업시간 = 해당 월 실제 진행 수업 시간/횟수 기준 집계
- 수업인원 = 해당 월 실제 수업에 참여한 인원 기준 집계
- 이동 = 임시 이동/이동 처리 카운트
- 연기 = 연기 회원 관련 카운트
- 탈퇴 = 탈퇴 처리된 회원 관련 카운트

==================================================
[4. 선생님 모드 > 매출관리 기능 추가/연결]

선생님 모드에도 매출관리 화면 구조를 동일한 개념으로 만든다.

선생님 모드 매출관리에서 보여줄 항목:
- 해당 월 매출결산액
- 수업시간
- 수업인원
- 보강 개수
- 체험 수업 개수
- 이동 개수
- 연기 개수
- 탈퇴 개수

그리고 하단 또는 명확한 위치에 아래 버튼 추가:
- 이번 달 정산 제출
- 저장
(필요 시 두 버튼 또는 저장 후 제출 구조 가능)

핵심:
선생님은 자기 데이터만 보고 정산을 마무리한 뒤
“제출”을 누르면 관리자 모드에 해당 월 정산내역이 보고되어야 한다.

==================================================
[5. 제출 흐름]

선생님 모드에서 해당 월 정산을 제출하면 아래처럼 처리한다.

1. 선생님이 해당 월 데이터 확인
2. 저장 가능
3. “이번 달 정산 제출” 실행
4. 해당 월 정산 상태가 제출완료로 변경
5. 관리자 모드 > 매출관리 > 선생님별 매출내역에 해당 월 제출 결과 반영
6. 관리자는 제출 여부를 월별로 확인 가능

정산 상태 예시:
- 미정산
- 저장됨
- 제출완료
- 관리자확인

이번 단계에서는 최소한
- 미정산
- 제출완료
정도만 있어도 됨

==================================================
[6. 월별 저장]

이 기능은 월별로 계속 저장되어야 한다.

요구사항:
1. 2026년 03월 정산 데이터 저장
2. 2026년 04월로 이동해도 별도 데이터 유지
3. 다시 2026년 03월로 오면 이전 정산 데이터 그대로 보임
4. 관리자/선생님 모두 월 선택 시 그 월 기준 데이터 표시
5. 월별 기록 누적 저장

즉:
정산 화면은 단순 계산 화면이 아니라
“월별 보관되는 정산 기록 화면”으로 동작해야 한다.

==================================================
[7. 관리자-선생님 연결]

연결 규칙:

1. 선생님 모드에서 제출한 월 정산 데이터는
   관리자 모드에서 같은 월 기준으로 확인 가능해야 한다

2. 관리자 모드에서 보이는 선생님별 매출내역은
   선생님 제출 데이터를 기반으로 연결되어야 한다

3. 선생님이 아직 제출하지 않은 경우
   관리자 화면에는 해당 선생님 상태를
   “미정산” 또는 “미제출”로 표시할 것

4. 제출 후에는
   관리자 화면에서 해당 선생님의 상태가 “제출완료”로 바뀌어야 한다

==================================================
[8. UI 요구사항]

관리자 모드 > 매출관리:
- “이번 달 총 매출” 아래에 “선생님별 매출내역” 섹션 추가
- 선생님별 카드/리스트로 보기 쉽게 표시
- 항목이 많으므로 카드 안 정보 정렬을 깔끔하게 구성
- 월별 이동 시 리스트 내용도 바뀌게 할 것

선생님 모드 > 매출관리:
- 관리자와 같은 항목 구조 사용
- 단, 자기 데이터만 표시
- 저장 / 제출 버튼 명확히 배치

==================================================
[9. 기존 항목 처리]

현재 “회원별 수업 횟수” 영역은
관리자 모드 매출관리에서 제거하거나 비노출 처리하고,
그 자리를 “선생님별 매출내역”으로 대체한다.

단,
기존 계산 로직이 재사용 가능하면 내부적으로 활용해도 된다.
표시만 회원 단위에서 선생님 단위로 재구성할 것.

==================================================
[10. 집계 기준]

가능한 범위 내에서 아래 기준으로 집계하라.

- 매출결산액: 해당 월 정산 결과 금액
- 수업시간: 월간 실제 진행 수업 시간/횟수
- 수업인원: 월간 실제 집계 인원
- 보강 개수: 보강 지정/진행 수 기준
- 체험 수업: 체험 처리 수 기준
- 이동: 임시 이동/이동 처리 수 기준
- 연기: 연기 회원 관련 수 기준
- 탈퇴: 탈퇴 회원 관련 수 기준

중요:
정확한 기준은 현재 앱의 기존 데이터 구조에 맞춰 최대한 일관되게 잡고,
보고서에 어떤 기준으로 계산했는지 적을 것

==================================================
[11. 절대 금지]

- 상단 “이번 달 총 매출” 카드 수정 금지
- 월 이동 UI 삭제 금지
- 휴무일 지정 버튼 삭제 금지
- 기존 정산 관련 데이터 구조 함부로 변경 금지
- 기존 선생님/관리자 연결 로직 훼손 금지
- 기능만 만들어놓고 저장/반영 안 되게 하는 것 금지

==================================================
[12. 테스트 지시]

수정 후 반드시 실제 테스트하라.

테스트 1.
관리자 모드 > 매출관리에서 “회원별 수업 횟수” 대신
“선생님별 매출내역”이 보이는지 확인

테스트 2.
등록된 선생님 목록이 월별로 보이는지 확인

테스트 3.
각 선생님 카드/리스트에 아래 항목이 보이는지 확인
- 매출결산액
- 수업시간
- 수업인원
- 보강
- 체험
- 이동
- 연기
- 탈퇴
- 상태

테스트 4.
선생님 모드 > 매출관리에서 동일 항목 구조가 보이는지 확인

테스트 5.
선생님 모드에서 저장 후 같은 월 다시 들어왔을 때 데이터 유지되는지 확인

테스트 6.
선생님 모드에서 “이번 달 정산 제출” 후
관리자 모드 같은 월 화면에 제출완료 상태가 반영되는지 확인

테스트 7.
다른 월로 바꿨다가 다시 돌아와도 월별 저장 데이터가 유지되는지 확인

테스트 8.
상단 “이번 달 총 매출” 카드가 그대로 유지되는지 확인

테스트 9.
휴무일 지정 버튼과 월 이동 기능이 그대로 살아 있는지 확인

==================================================
