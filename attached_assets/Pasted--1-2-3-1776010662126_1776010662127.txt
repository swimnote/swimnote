환불 정책 기능을 아래 기준으로 한 단계 강화하세요.

목표:
1. 정책 동의 시 “어떤 버전의 정책에 동의했는지” 저장
2. 정책 미동의 상태에서는 유료 결제 진입/실행 차단
3. 슈퍼관리자(수영장 관리자) 가입 후 첫 진입 온보딩에서 정책 동의를 반드시 받도록 변경

중요:
이번 작업은 단순 UI 추가가 아니라 운영/결제 정책 강제 구조입니다.
기존 환불 정책 화면과 settings 연동은 유지하고, 그 위에 버전/강제동의/결제차단을 추가하세요.

━━━━━━━━━━━━━━
1. 정책 버전 저장 구조 추가
━━━━━━━━━━━━━━

현재 policy_versions / policy_consents 구조를 아래 기준으로 정리하세요.

A. policy_versions
필수 필드:
- id
- policy_type (예: refund_policy)
- version (문자열 또는 숫자 버전)
- content
- is_active
- created_at

원칙:
- 현재 활성 정책 1개를 명확히 식별 가능해야 함
- /admin/refund-policy 조회 시 현재 활성 버전도 함께 반환

B. policy_consents
필수 필드:
- user_id
- policy_type
- policy_version
- agreed_at

중요:
동의 기록은 “현재 정책 버전”과 함께 저장해야 합니다.
단순 agreed_at만 저장하지 말고, 어떤 버전에 동의했는지 반드시 남기세요.

정책 변경 시:
- 새 버전 생성
- 기존 동의는 새 버전에 대해 미동의 상태로 간주

━━━━━━━━━━━━━━
2. API 응답 구조 수정
━━━━━━━━━━━━━━

GET /admin/refund-policy 응답에 아래를 포함하세요.

예시:
{
  "policy_type": "refund_policy",
  "version": "v1.1",
  "content": "...",
  "agreed": true,
  "agreed_at": "2026-04-13T10:20:00Z",
  "agreed_version": "v1.1",
  "needs_reagree": false
}

정책 버전이 바뀌어 기존 동의가 무효인 경우:
- agreed = false 또는
- needs_reagree = true
로 명확히 표현하세요.

POST /admin/refund-policy/agree 동작:
- 현재 활성 policy_version 기준으로 동의 저장
- ON CONFLICT 시 agreed_at, policy_version 갱신

━━━━━━━━━━━━━━
3. 슈퍼관리자/수영장관리자 첫 진입 온보딩에 정책 동의 추가
━━━━━━━━━━━━━━

대상:
- pool_admin 가입 직후 첫 진입
- 또는 로그인 후 현재 활성 환불정책 미동의 상태인 pool_admin

동작 원칙:
- 환불 정책 미동의 상태면 일반 홈/결제/설정으로 바로 진입시키지 말고
  “정책 확인 온보딩 화면”으로 먼저 보냄
- 이 화면에서 정책 내용을 읽고 동의해야 다음 단계로 진행 가능

온보딩 화면 요구사항:
- 제목: 환불 정책 확인
- 현재 활성 정책 버전 표시
- 정책 핵심 내용 목록 표시
- 전체 내용 보기 또는 스크롤 가능
- 체크박스 또는 확인 UI
- 하단 버튼: “환불 정책에 동의하고 계속하기”
- 버튼 누를 때 이중 확인 Alert 유지 가능

완료 후:
- 동의 저장
- 정상 홈 또는 다음 온보딩 단계로 이동

중요:
이 온보딩은 pool_admin에게만 강제
학부모/선생님은 제외

━━━━━━━━━━━━━━
4. 유료 결제 차단 로직 추가
━━━━━━━━━━━━━━

목표:
환불 정책 미동의 상태에서는 유료 결제 진입/실행이 불가능해야 함

적용 범위:
- 구독 화면 진입
- 결제 버튼 클릭
- 서버에서 실제 결제 처리 직전 검증

권장 방식:
A. 프론트 차단
- 환불정책 미동의면 결제 버튼 비활성 또는 결제 화면에서 차단 메시지 표시
- 메시지 예시:
  "유료 결제를 진행하려면 환불 정책 동의가 필요합니다"
  [환불 정책 확인하러 가기]

B. 서버 차단
- 결제 관련 API 또는 결제 상태 변경 직전
- 현재 활성 환불 정책 버전에 동의했는지 확인
- 미동의면 403 또는 명확한 비즈니스 에러 반환

예시:
{
  "success": false,
  "code": "REFUND_POLICY_AGREEMENT_REQUIRED",
  "message": "유료 결제를 진행하려면 환불 정책 동의가 필요합니다."
}

중요:
프론트만 막지 말고 서버에서도 반드시 차단하세요.

━━━━━━━━━━━━━━
5. settings 및 슈퍼관리자 연동 유지/보완
━━━━━━━━━━━━━━

A. pool_admin settings > 운영 설정 > 환불 정책 확인
- 기존 기능 유지
- 현재 버전 표시
- 동의 완료 시 동의 버전과 날짜 표시
- 새 버전 등장 시 “재동의 필요” 상태로 보이게 수정

B. 슈퍼관리자 대시보드
- 정책 미확인 카운트는 “현재 활성 정책 버전 기준 미동의 수영장” 기준으로 계산
- 과거 버전에만 동의한 수영장은 미확인으로 다시 잡히도록 수정

━━━━━━━━━━━━━━
6. UX 규칙
━━━━━━━━━━━━━━

- 정책 미동의는 단순 경고가 아니라 실제 진행 차단
- 결제 차단 사유를 명확히 설명

- "현재 버전: v1.1"
