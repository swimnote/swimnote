현재 슈퍼관리자 분류는 좋아졌지만, SMS 발송의 주체와 과금 소유권이 잘못 잡혀 있다.
기능을 삭제하지 말고 역할 기준으로 소유권을 바로잡아라.

==================================================
[1. SMS 발송 주체 수정]
==================================================

중요:
슈퍼관리자가 직접 초대 SMS를 보내는 구조가 아니라,
수영장 관리자(운영자)가 선생님/학부모에게 초대 SMS를 보내는 구조로 바꾼다.

정리:
- 발송 주체: 운영자
- 과금 대상: 운영자
- 슈퍼관리자 역할: 정책 설정 + 사용량/과금 내역 조회 + 차단 기준 관리

슈퍼관리자 화면에서는
"초대/SMS 발송 관리"를
"SMS 템플릿/발송 로그/인증 로그 관리" 중심으로 바꾼다.

==================================================
[2. 운영 관리 그룹 수정]
==================================================

운영 관리 하위 메뉴에서
"SMS 과금·사용량"은 유지하되 기능을 아래로 확장:

- 운영자별 월 발송 건수
- 운영자별 무료 제공 건수
- 운영자별 초과 과금 건수
- 운영자별 예상 과금액
- 운영자별 미납 상태
- 운영자별 SMS 차단 여부
- 운영자별 초과 허용 여부
- 운영자별 SMS 유형별 발송 수
  (초대 / 인증 / 안내 / 경고)

==================================================
[3. 지원 센터 그룹 수정]
==================================================

지원 센터의 "초대/SMS 발송 관리"는
슈퍼관리자가 직접 보내는 화면이 아니라
아래 관리 화면으로 수정:

- SMS 템플릿 관리
- 운영자 발송 로그 보기
- 발송 실패 로그 보기
- 인증번호 발송 기록
- 초대 링크 만료 관리
- 재발송 이력 확인

==================================================
[4. 운영자 모드에 새 메뉴 추가]
==================================================

운영자 모드에 새 메뉴 추가:
"초대·문자 관리"

이 메뉴에서 운영자가 직접 할 수 있어야 하는 것:
- 선생님 초대 SMS 발송
- 학부모 연결 요청 SMS 발송
- 초대 재발송
- 만료 초대 확인
- 링크 복사
- 이번 달 무료 제공 잔여 건수 보기
- 초과 과금 예상액 보기
- 발송 실패 내역 보기
- 현재 SMS 차단 여부 보기

==================================================
[5. 회원가입 SMS 인증 추가]
==================================================

공통 회원가입 흐름에 휴대폰 인증을 반드시 추가:

- 휴대폰 번호 입력
- 인증번호 발송
- 인증번호 확인
- 가입 완료 후 역할 선택

정책은 플랫폼 공통으로 사용:
- 가입 시 SMS 인증 필수
- 인증번호 유효시간
- 재발송 대기시간
- 일일 발송 제한
- 동일 번호 중복 처리 규칙

이 설정은 슈퍼관리자 "보안·설정" 내 플랫폼 인증 정책에서 관리한다.

==================================================
[6. 2차 인증 범위 수정]
==================================================

2차 인증은 슈퍼관리자만 적용한다.
운영자/선생님/학부모에는 넣지 않는다.

슈퍼관리자 보안설정에서:
- OTP 앱 인증
- SMS 인증
- 이메일 인증
- OTP + SMS 백업
- 로그인 시 항상 2차 인증
- 킬스위치/삭제/권한 변경 시만 추가 인증

구현 유지

추가:
- 2차 인증 강제 여부
- 복구 코드/백업 코드 영역
- 2차 인증 실패 로그
- OTP 재등록 플로우 mock

==================================================
[7. 감사·리스크에 SMS 항목 추가]
==================================================

리스크 센터 / 감사 로그에 아래 항목 반영:
- SMS 발송 실패 급증
- 인증 SMS 실패율 경고
- SMS Provider 장애
- SMS 미납 운영자
- SMS 차단 운영자

==================================================
[8. 테스트 필수]
==================================================

반드시 테스트:
1. 슈퍼관리자 홈 진입
2. 운영 관리 > SMS 과금·사용량 진입
3. 지원 센터 > 템플릿/발송 로그/인증 로그 진입
4. 운영자 모드 > 초대·문자 관리 진입
5. 운영자가 선생님 초대 SMS 발송 mock
6. 운영자가 학부모 연결 SMS 발송 mock
7. 운영자 SMS 과금 예상액 반영
8. 회원가입 휴대폰 인증 흐름 mock
9. 슈퍼관리자 OTP 설정/변경
10. 슈퍼관리자 외부 서비스 상태 화면 진입
11. 뒤로가기/스크롤/팝업/재진입 점검

완료라고 할 때는
"화면 열림"만 말하지 말고,
상태 변화와 역할 분리까지 확인해서 보고할 것.

==================================================
[9. 작업 완료 보고 형식]
==================================================

1. 슈퍼관리자와 운영자의 SMS 역할 분리를 어떻게 바꿨는지
2. 운영자 모드에 추가한 초대·문자 관리 기능 목록
3. 슈퍼관리자 SMS 과금·사용량 관리에 추가한 항목
4. 회원가입 SMS 인증 구현 상태
5. 슈퍼관리자 2차 인증 구현 상태
6. 아직 mock 인 부분
7. 테스트 완료 항목
