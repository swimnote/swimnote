0. 작업 전 필수 규칙 (절대 위반 금지)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

- 기존 코드 절대 덮어쓰기 금지
- 모든 수정은 신규 컴포넌트 생성 후 교체 방식으로 진행
- 관리자 / 선생님 / 학부모 모드 데이터 연결 절대 깨지면 안 됨
- superAdminDb 기준 로직 유지
- poolDb 접근 금지 (백업 전용)
- 작업 후 전체 플로우 테스트 필수
- 작업 완료 후 보고서 작성 필수

※ 관리자모드 작업과 충돌 절대 금지
※ 관리자에서 수정된 데이터 구조 그대로 사용

---

- 수업 처리 속도 최적화
- 중복 제거
- 구조 단순화
- 뒤로가기 정상화

※ 기능 추가 금지 / 구조 정리만 수행

2. 선생님 홈 구조 개편
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[상단 고정 유지]

- 오늘 수업 수
- 출석 미체크
- 미작성 일지
- 보강 대기

※ 기존 데이터 그대로 사용

---

[스케줄러 바로가기 유지]

- 위치 유지
- 클릭 시 수업관리 이동

---

[오늘 수업 영역 — 핵심 변경]

아이콘형 제거 → 리스트형 변경

형식:

시간순 정렬

예:
14:00 초급반 (5명)
김철수, 이영희, 박민수, 최민호, 정하늘

16:00 중급반 (7명)
서태웅, 김민준, 박서준, 이유진, 최하늘 외 2명

---

[표시 규칙]

- 최대 5명까지 이름 표시
- 초과 시 “외 n명”
- 한 줄 표시 (줄바꿈 금지)
- 인원 수 반드시 표시
- 리스트 스크롤 허용

---

[수업 클릭 동작]

- 페이지 이동 금지
- 하단 팝업(sheet) 오픈

---

[홈] [수업관리] [회원관리] [메신저] [설정]

---

[제거/이동]

- 정산 → 제거 (설정 또는 내부로 이동)
- 내정보 → 설정으로 이동
- 보강관리 → 독립 탭 제거 (수업 흐름 안으로 흡수)
- 쪽지 → 메신저 보조 구조

4. 수업관리 구조 개편 (핵심)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[기본 원칙]

수업관리 = 스케줄 중심

---

[월/주/일 역할]

월 → 전체 확인  
주 → 운영 확인  
일 → 실제 처리

---

[핵심 흐름]

수업관리 진입  
→ 스케줄 표시  
→ 날짜 클릭  
→ 큰 하단 팝업  
→ 해당 날짜 반 리스트  
→ 반 클릭  
→ 반 관리 화면

---

- 뒤로가기 시 홈으로 튐

---

수정:

정상 흐름:

수업관리  
→ 날짜 팝업  
→ 반 화면  
→ 회원 정보  
→ 뒤로가기  
→ 반 화면  
→ 뒤로가기  
→ 날짜 팝업  
→ 뒤로가기  
→ 수업관리

---

금지:

- 홈으로 점프
- 루트로 이동
