지금부터 업무 메신저에 아래 기능을 추가하라.

이번 작업은 기존 대화/공지 구조를 유지한 상태에서
대화 탭의 업무 전달 기능을 확장하는 작업이다.

중요:
- 관리자 모드와 선생님 모드 둘 다 동일하게 적용
- 기존 대화 탭, 공지 탭, 입력창, 키보드 대응은 유지
- 이미 정상 동작하는 메신저 구조를 깨지 말 것
- 이번에는 메신저의 “업무 전달 도구”를 추가하는 작업이다

==================================================
[1. 이번 작업에서 추가할 기능]

대화 탭에서 아래 기능 추가:

1. + 버튼 클릭 시 첨부 메뉴 팝업
2. 파일 첨부 기능
3. 내 회원정보 올리기 기능
4. 상단 참가인원 표시 + 멤버 팝업
5. 특정 멤버에게만 보내는 메시지 기능

==================================================
[2. + 버튼 동작]

현재 대화 탭 입력창 왼쪽에 있는 + 버튼을 누르면
하단 팝업 또는 하단 시트 형태로 첨부 메뉴를 보여준다.

메뉴 항목:
- 파일 첨부
- 내 회원정보 올리기

팝업 규칙:
- 현재 페이지를 벗어나지 않는 선택 기능이므로 팝업으로 처리
- 바깥 터치 또는 X 버튼으로 닫기 가능
- 팝업 스크롤 가능
- 기존 팝업 규칙 준수

==================================================
[3. 파일 첨부 기능]

첨부 메뉴에서 “파일 첨부”를 누르면
기기 또는 현재 허용된 파일 선택 방식으로 파일을 선택할 수 있게 하라.

MVP 기준:
- 이미지/문서 파일 첨부 가능
- 첨부 후 채팅창에 파일 메시지 카드 형태로 표시
- 누르면 파일명/미리보기 또는 다운로드 동작 가능 구조 준비

메시지 타입:
- attachment_file

표시 예:
- 파일 아이콘
- 파일명
- 전송 시간

중요:
- 파일 첨부 후 실제 메시지로 저장되어야 함
- 관리자와 선생님 모두 사용 가능

==================================================
[4. 내 회원정보 올리기 기능]

첨부 메뉴에서 “내 회원정보 올리기”를 누르면
현재 로그인한 사용자가 담당 중인 어린이 회원 목록을 불러와서
선택할 수 있는 팝업을 띄운다.

규칙:
- 담당 선생님인 경우에만 회원 목록이 보여야 함
- 담당 선생님이 아니면 클릭/선택 불가
- 관리자도 자신의 구조상 담당 회원이 있으면 가능, 없으면 비활성 또는 빈 상태 처리

==================================================
[5. 회원 선택 팝업]

회원 선택 팝업 안에서는 아래가 가능해야 한다.

표시:
- 회원 이름
- 반 정보
- 주 몇 회
- 수업 시간

동작:
- 한 명 선택 후 “보내기”

보내면:
- 채팅창에 회원정보 카드 메시지로 올라감

메시지 타입:
- member_profile_card

==================================================
[6. 회원정보 카드 표시 형식]

채팅창에 올라가는 회원정보 카드는 아래 정보만 기본 노출한다.

기본 노출 정보:
- 회원 이름
- 주 몇 회
- 수업 시간
- 부모님 연락처

예:
서태웅
주 2회 / 화, 목 16:00
부모님 연락처 010-1234-5678

중요:
- 과도한 개인정보 노출 금지
- 이 카드가 채팅창 안에 보기 좋은 카드 형태로 들어가야 함

==================================================
[7. 회원정보 카드 클릭 동작]

회원정보 카드 클릭 시 규칙:

1. 담당 선생님인 경우
- 클릭 가능
- 추가 정보 상세 보기 가능
- 상세는 팝업 또는 별도 상세 팝업으로 처리

2. 담당 선생님이 아닌 경우
- 클릭 차단
- 또는 “담당 선생님만 볼 수 있습니다” 처리

즉:
기본 카드 정보는 공유 가능
추가 상세는 담당 선생님만 열람 가능

==================================================
[8. 채팅창 상단 참가인원 표시]

대화 탭 상단 헤더 쪽에
현재 메신저 참가인원 수를 표시하라.

예:
- 업무 메신저 7명
또는
- 참가 7

이 인원 표시를 누르면
메신저 멤버 팝업이 열려야 한다.

팝업 규칙:
- 이건 현재 화면을 벗어나지 않는 멤버 확인/선택 기능이므로 팝업 처리
- 팝업 안에서 멤버 리스트 표시

==================================================
[9. 멤버 팝업 기능]

멤버 팝업에서 보여줄 것:
- 멤버 이름
- 역할(관리자/선생님)
- 선택 가능 상태

기능:
- 멤버를 선택할 수 있어야 함
- 한 명만 선택하는 MVP 구조로 시작
- 선택 후 대화 탭 입력줄 위에 “누구에게만 보냄” 카드 표시

==================================================
[10. 특정 멤버에게만 보내기 기능]

멤버 팝업에서 특정 멤버를 선택하면
대화 탭 입력창 위쪽 또는 입력줄 시작 부분에
선택 카드가 표시되게 하라.

표시 예:
- 김선생에게만 보냄 [X]

이 상태에서 메시지를 보내면:
- 일반 전체 채팅이 아니라
- 특정 대상 지정 메시지로 전송

메시지 타입:
- directed_message

중요:
- UI상으로는 채널 안에서 보이되,
- “누구에게만 보냄” 표시가 명확해야 한다

==================================================
[11. 특정 대상 메시지 해제]

다시 일반 채팅으로 바꾸고 싶으면
입력줄 위/앞에 붙어 있는
“누구에게만 보냄” 카드의 X 버튼을 누르면 된다.

동작:
- 선택 대상 해제
- 일반 전체 채팅 상태로 복귀
- 입력 중인 텍스트는 유지 가능하면 유지

==================================================
[12. 관리자/선생님 공통 적용]

이번 기능은
- 관리자 모드 메신저
- 선생님 모드 메신저
둘 다 동일하게 적용한다.

즉:
- + 버튼 메뉴
- 파일 첨부
- 회원정보 카드
- 참가인원 팝업
- 특정 멤버에게만 보내기
모두 두 모드 동일

==================================================
[13. 기존 메신저 구조 유지]

반드시 유지:
- 대화 탭 / 공지 탭 구조
- 대화 탭 기본 진입
- 공지 탭 노란색 스타일
- 공지 탭 관리자만 작성
- 이동/보강 자동 메시지
- 자동 메시지 수정 불가
- 키보드 간섭 없는 입력창 구조

이번 작업은 확장이지 기존 기능 재구성이 아니다

==================================================
[14. 데이터 구조 추가]

기존 메시지 구조에 아래 타입을 추가하라.

message_type 추가:
- attachment_file
- member_profile_card
- directed_message

directed_message 추가 필드 예:
- target_member_id 또는 target_user_id
- visible_target_label

member_profile_card 필드 예:
- member_id
- member_name
- schedule_summary
- parent_phone

==================================================
[15. 먼저 확인할 것]

수정 전에 아래를 먼저 파악하라.

1. 현재 대화 탭 입력창 컴포넌트
2. + 버튼 처리 파일
3. 파일 첨부 허용 구조
4. 담당 회원 데이터를 불러오는 파일
5. 회원 상세 권한 판별 로직
6. 메신저 상단 헤더 파일
7. 멤버 리스트를 계산할 수 있는 데이터 구조
8. directed message 저장/표시 가능한 구조

==================================================
[16. 테스트 - 반드시 실제 수행]

테스트 1.
대화 탭에서 + 버튼 누르면 첨부 메뉴 팝업이 뜨는지 확인

테스트 2.
첨부 메뉴에서 파일 첨부 선택 후 실제 파일 메시지가 전송되는지 확인

테스트 3.
첨부 메뉴에서 내 회원정보 올리기 선택 시 담당 회원 목록이 뜨는지 확인

테스트 4.
회원 한 명 선택 후 보내면 채팅창에 회원정보 카드가 올라가는지 확인

테스트 5.
회원정보 카드에 기본 정보(주 몇 회, 수업 시간, 부모 연락처)가 보이는지 확인

테스트 6.
담당 선생님이 아닌 경우 카드 클릭 시 추가 상세가 막히는지 확인

테스트 7.
채팅창 상단 참가인원 수가 보이는지 확인

테스트 8.
참가인원 수 클릭 시 멤버 팝업이 뜨는지 확인

테스트 9.
멤버 한 명 선택하면 입력창 위/앞에 “누구에게만 보냄” 카드가 표시되는지 확인

테스트 10.
그 상태에서 메시지 전송 시 directed message로 저장/표시되는지 확인

테스트 11.
“누구에게만 보냄” 카드의 X를 누르면 일반 채팅으로 복귀하는지 확인

테스트 12.
위 모든 기능이 관리자 모드와 선생님 모드 둘 다 동일하게 동작하는지 확인

==================================================
[17. 작업 후 보고 형식]

작업 후 아래 순서로 보고하라.

1. 수정 파일 목록
2. + 버튼 팝업 구현 방식
3. 파일 첨부 구현 방식
4. 내 회원정보 올리기 구현 방식
5. 회원정보 카드 표시/권한 처리 방식
6. 참가인원 표시 및 멤버 팝업 구현 방식
7. directed message 구현 방식
8. 테스트 1~12 결과
9. 아직 남은 문제 있으면 명시

==================================================
[18. 절대 금지]

- + 버튼 눌렀는데 아무것도 안 뜨는 가짜 UI 금지
- 회원정보 카드가 담당 선생님 권한 없이 상세 열리는 것 금지
- 참가인원 표시만 있고 멤버 팝업 안 되는 상태 금지
- 특정 멤버 선택 UI만 있고 실제 directed message 처리 안 되는 상태 금지
- 관리자/선생님 중 한쪽만 되는 상태 금지
- 기존 메신저 입력창/키보드 구조 깨뜨리기 금지

이번 작업은
업무 메신저 대화 탭에 파일 첨부, 담당 회원 정보 카드 공유, 참가인원 팝업, 특정 멤버에게만 보내기 기능을 추가해서 실제 업무 전달 툴로 확장하는 작업이다.
