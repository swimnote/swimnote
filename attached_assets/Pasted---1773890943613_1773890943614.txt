지금부터 메신저의 “특정 멤버에게만 보내기” 상태 유지 규칙을 수정하라.

이번 작업은 메신저 마지막 마무리 작업이다.

현재 문제:
- 특정 멤버를 선택해서 메시지를 한 번 보내면
  선택 상태가 자동으로 꺼진다
- 이건 사용자가 원하는 동작이 아니다

최종 규칙:
선택한 멤버에게만 보내기 상태는
한 번 전송했다고 자동으로 해제되면 안 된다.
여러 번 연속으로 보내도 유지되어야 한다.

==================================================
[1. 유지 규칙]

특정 멤버 선택 후 표시되는
“누구에게만 보냄” 카드 상태는 유지되어야 한다.

즉:
- 멤버 선택
- 메시지 1회 전송
- 메시지 2회 전송
- 메시지 3회 전송
이렇게 여러 번 보내도 계속 유지

전송 후 자동 해제 금지

==================================================
[2. 상태가 꺼지는 경우 - 딱 2개만 허용]

이 상태는 아래 두 경우에만 해제되게 하라.

1. 사용자가 직접 X 버튼 눌러 해제
2. 사용자가 다른 하단 탭으로 이동했다가 다시 메신저 탭으로 들어온 경우

즉:
- 메신저 내부에서는 유지
- 메신저를 벗어나면 초기화
- 다시 메신저 들어오면 일반 채팅 상태로 시작

==================================================
[3. 해제되면 안 되는 경우]

아래 경우에는 절대 꺼지면 안 된다.

1. 메시지 전송 후
2. 메신저 내부에서 스크롤 후
3. 대화 탭 ↔ 공지 탭 전환 후 다시 대화 탭으로 돌아온 경우
4. 키보드 열고 닫은 경우
5. 입력창 포커스 해제 후 다시 입력한 경우

즉:
메신저 내부 활동만으로는 선택 상태가 사라지면 안 된다.

==================================================
[4. 메신저 내부 탭 전환 규칙]

메신저 안에서
- 대화 탭
- 공지 탭
을 오갈 때

특정 멤버에게만 보내기 상태는 유지한다.

다시 대화 탭으로 돌아오면
이전 선택 대상이 그대로 보이게 한다.

즉:
메신저 내부 탭 전환은 상태 초기화 조건이 아니다.

==================================================
[5. 다른 하단 탭 이동 시 초기화]

아래 경우에는 초기화한다.

예:
- 메신저 → 홈
- 메신저 → 수업
- 메신저 → 사람
- 메신저 → 매출관리
- 메신저 → 더보기

이후 다시 메신저 탭으로 들어오면
특정 대상 선택 상태는 비워져 있어야 한다.

즉:
메신저 탭 이탈 = 선택 상태 초기화

==================================================
[6. UI 동작]

선택 상태가 유지되는 동안
입력창 위/앞의 카드
“OO선생님에게만 보냄 [X]”
은 계속 보인다.

메시지 여러 번 전송해도 이 카드 유지

X 버튼 누르면:
- 선택 해제
- 일반 채팅 상태로 복귀

==================================================
[7. 데이터/상태 처리 방식]

권장 방식:
- 메신저 탭 내부 상태로 selectedTarget 유지
- 메시지 전송 시 selectedTarget을 초기화하지 말 것
- 하단 탭 change event에서만 초기화
- X 버튼 클릭 시 초기화

즉:
전송 함수 안에서 selectedTarget = null 처리 금지

==================================================
[8. 테스트]

반드시 실제 테스트하라.

테스트 1.
멤버 1명 선택
→ “OO에게만 보냄” 카드 표시 확인

테스트 2.
메시지 1회 전송 후
→ 카드가 그대로 남아 있는지 확인

테스트 3.
메시지 2회, 3회 연속 전송 후
→ 카드가 계속 유지되는지 확인

테스트 4.
공지 탭 갔다가 다시 대화 탭 복귀
→ 카드가 유지되는지 확인

테스트 5.
X 버튼 클릭
→ 카드가 사라지고 일반 채팅으로 복귀하는지 확인

테스트 6.
메신저에서 다른 하단 탭으로 이동
→ 다시 메신저 탭 들어오면 카드가 초기화되는지 확인

테스트 7.
관리자 모드 / 선생님 모드 둘 다 동일하게 동작하는지 확인

==================================================
[9. 작업 후 보고]

1. 수정 파일 목록
2. selected target 상태 유지 방식
3. 자동 해제되던 원인
4. 메신저 내부 탭 전환 시 유지 처리 방식
5. 다른 하단 탭 이동 시 초기화 처리 방식
6. 테스트 1~7 결과

==================================================
[10. 절대 금지]

- 메시지 1회 전송 후 자동 초기화 금지
- 공지 탭 갔다가 대화 탭 오면 초기화되는 것 금지
- X 버튼 없이 자동 해제되는 것 금지
- 다른 탭 갔다 와도 초기화 안 되는 것 금지

이번 작업은
특정 멤버에게만 보내기 상태를 메신저 내부에서는 계속 유지하고,
사용자가 직접 끄거나 메신저 탭을 벗어난 경우에만 해제되게 만드는 마무리 작업이다.
