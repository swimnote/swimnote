[결제플랜 전체 연동 지시서 - 슈퍼관리자 상품설정 → 수영장 결제 → 매출정산 완전 연결]

목표:
현재 만들어둔 결제 플랜을
1) 슈퍼관리자 구독상품설정에 정확히 등록하고
2) 수영장 관리자 결제 기능과 연결하고
3) 결제 발생 시 슈퍼관리자 매출정산관리까지 자동 반영되게 한다.

중요:
- 이 작업은 “결제 → 데이터 → 정산”까지 끝까지 연결하는 작업이다
- 중간에 한 단계라도 빠지면 실패
- 실제 결제 시뮬레이션까지 테스트해야 완료로 인정
- 금액/회원수/용량/상태 전부 정확히 반영되어야 한다

==================================================
1. 슈퍼관리자 구독상품설정 구성
==================================================

현재 우리가 확정한 플랜을 정확히 등록한다.

[무료]
- 회원 5명
- 100MB
- 가격 0원

[개인/소형]
- 30명 / 600MB / 2900원
- 50명 / 1GB / 3900원

[중형/수영장]
- 100명 / 5GB / 9900원
- 300명 / 20GB / 29000원 (어드밴스)
- 500명 / 40GB / 59000원
- 1000명 / 100GB / 99000원

[대형]
- 2000명
- 3000명

필드 반드시 포함:
- plan_id
- plan_name
- max_members
- storage_limit
- price
- is_active
- created_at

중요:
- 기존 잘못된 price 컬럼 있으면 price_per_month로 통일
- 누락 필드 절대 금지

==================================================
2. 수영장 관리자 결제 연결
==================================================

수영장 관리자가 플랜 선택 → 결제 시

반드시 처리되어야 하는 것:

1. 현재 플랜 상태 저장
2. 결제 상태 저장 (paid / failed)
3. 시작일 / 만료일 생성
4. 회원수 제한 적용
5. 저장공간 제한 적용

DB 구조 예시:

subscriptions
- id
- pool_id
- plan_id
- status (active / expired / failed)
- start_date
- end_date
- payment_status
- next_billing_date

==================================================
3. 결제 성공 시 처리
==================================================

결제 완료 시:

- 해당 pool_id에 플랜 적용
- 회원수 제한 업데이트
- 스토리지 제한 업데이트
- 상태 active 변경
- 매출 기록 생성

==================================================
4. 결제 실패 시 처리 (중요)
==================================================

정책 그대로 적용:

- 즉시 읽기모드 전환
- 모든 하위 계정 (선생님/학부모)도 제한

메시지:
관리자:
"결제 실패로 이용이 제한되었습니다. 재결제를 진행해주세요"

선생님/학부모:
"기능 이용이 제한되었습니다. 관리자에게 문의해주세요"

==================================================
5. 슈퍼관리자 매출정산 연동
==================================================

결제 발생 시 반드시 아래 데이터 생성:

revenue_logs

- id
- pool_id
- plan_id
- amount
- payment_method
- store_fee
- net_revenue
- created_at

계산:

store_fee = amount * 0.3
net_revenue = amount - store_fee

중요:
- 앱스토어/플레이스토어 수수료 반영
- 누락 없이 기록

==================================================
6. 매출정산관리 화면 연동
==================================================

슈퍼관리자에서 확인 가능해야 한다:

- 전체 매출 합계
- 플랜별 매출
- 수영장별 매출
- 결제 내역 리스트

필수 항목:
- 수영장 이름
- 플랜 이름
- 결제 금액
- 수수료
- 실수익
- 결제일

==================================================
7. 데이터 누락 방지
==================================================

반드시 체크:

- 결제했는데 revenue_logs 안 찍히는 경우 없어야 함
- plan_id 누락 금지
- pool_id 누락 금지
- 금액 0으로 들어가는 버그 금지

==================================================
8. 전체 흐름 테스트 (필수)
==================================================

시뮬레이션 반드시 수행:

조건:
- 수영장 10개
- 각 120명 이상

테스트:

1. 무료 → 유료 전환
2. 플랜 업그레이드
3. 플랜 다운그레이드
4. 결제 실패
5. 재결제 성공

확인:

- 회원수 제한 정상 적용
- 업로드 제한 정상 적용
- 매출정산 반영 정상
- UI 깨짐 없음

==================================================
9. 절대 깨지면 안 되는 것
==================================================

- 회원 제한 로직
- 스토리지 제한 로직
- 읽기모드 전환
- 기존 회원 데이터
- 사진 데이터

==================================================
10. 완료 보고 형식
==================================================

1. 등록된 플랜 목록
2. 결제 → 구독 반영 테스트 결과
3. 매출정산 기록 생성 확인
4. 수수료 계산 결과
5. 결제 실패 처리 결과
6. 10개 수영장 시뮬레이션 결과
7. 누락 데이터 여부
8. 남은 버그

중요:
- 반드시 실제 테스트 기반으로 작성
- “연결 완료” 같은 말 금지
