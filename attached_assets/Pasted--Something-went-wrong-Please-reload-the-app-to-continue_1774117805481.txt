현재 앱이 "Something went wrong / Please reload the app to continue" 에러 화면으로 죽고 있다.
이건 UI 수정이 아니라 런타임 크래시다.

지금부터 기능 추가 작업 잠시 멈추고, 앱이 다시 정상 실행되도록 크래시 원인을 먼저 잡아라.

우선순위:
1. 앱 시작 직후/슈퍼관리자 진입 직후 발생하는 fatal runtime error 찾기
2. 에러 boundary에 잡히는 실제 exception message 확인
3. import 경로, route 등록, store 초기화, seed data shape mismatch, undefined access, map 대상 null/undefined, date parsing, enum mismatch 전수 점검
4. 앱이 절대 죽지 않도록 안전 fallback 추가

반드시 할 것:

[1] 에러 로그 먼저 확인
- console error
- metro bundler error
- runtime stack trace
- 어떤 screen/component/store에서 죽는지 정확히 찾기
- "고쳤다" 하지 말고 원인 파일/원인 라인 요약해서 보여주기

[2] 가장 의심할 곳부터 점검
- SuperAdminHome
- OperatorListScreen
- SubscriptionManagementScreen
- StorageManagementScreen
- PolicyComplianceScreen
- AuditLogScreen
- SupportCenterScreen
- RiskCenterScreen
- FeatureFlagScreen
- ReadonlyControlScreen
- 새로 추가한 store 전체
- seed data와 type shape mismatch

[3] 특히 아래 에러 패턴 찾기
- undefined.map
- undefined.filter
- property access of undefined/null
- route name mismatch
- component export default / named export mismatch
- circular import
- invalid icon/component import
- invalid date formatting
- enum value mismatch
- missing provider wrapper
- Zustand store selector returning undefined
- list key/id missing causing render crash
- modal/portal provider 누락
- SafeArea / NavigationContainer 중복 또는 누락

[4] 앱 안 죽게 안전장치 추가
- 모든 list data에 Array.isArray 체크
- 모든 date는 formatDateSafe 사용
- 모든 text는 safeText 사용
- store 기본값 빈 배열/기본 객체로 채우기
- screen 진입 전에 필수 데이터 없으면 fallback UI 보여주기
- route param 없으면 fallback 처리
- 위험 화면은 loading/empty/error 3상태 분리
- ErrorBoundary에서 실제 에러 메시지 console로 남기기

[5] 이번 작업에서 가장 중요한 수정
- 앱 부팅 시 한 번에 모든 슈퍼관리자 screen/store/seed를 eager import 하지 말고,
  필요한 화면만 안전하게 import 되게 점검
- 새 seed data가 타입과 다르면 화면에서 직접 참조하지 말고 normalize 함수 거쳐서 사용
- audit log title/operatorName/date에 undefined 들어가도 화면 안 죽게 처리

[6] 수정 후 검증
- 앱 첫 진입
- 로그인
- 슈퍼관리자 홈 진입
- 운영자 관리 진입
- 구독/결제 진입
- 저장공간 진입
- 정책 진입
- 운영 로그 진입
- 고객센터 진입
- 리스크 센터 진입
- 기능 플래그 진입
