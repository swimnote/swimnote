선생님 모드의 회원 프로필 화면을 관리자 모드와 동일한 구조로 통일한다.
핵심은 선생님이 어느 경로에서 회원 프로필에 진입하든 항상 동일한 회원 프로필 UI와 동일한 상태 변경 기능을 사용하게 만드는 것이다.

=====================================
[최우선 목표]

1. 선생님 모드 회원 프로필에 관리자 모드와 동일한 "상태 변경" 버튼 추가
2. 상태 변경 버튼 클릭 시 관리자 모드와 동일한 상태 변경 팝업 사용
3. 선생님이 어느 경로에서 회원 프로필에 들어가도 동일한 프로필 화면으로 열리게 통합
4. 상태 변경 결과는 회원관리 / 반배정 / 대기자 명단 / 관리자 모드까지 즉시 연동
5. 기존 기능 삭제 금지, 기존 DB/API 경로 유지

=====================================
[프로필 진입 경로 통일]

선생님 모드에서 아래 어떤 경로로 회원 프로필에 진입하더라도
반드시 동일한 공통 프로필 페이지 TeacherMemberProfileScreen 을 연다.

적용 경로:
- 회원관리 > 전체 회원 리스트 > 회원 카드 클릭
- 회원관리 > 미배정 리스트 > 회원 카드 클릭
- 회원관리 > 연기예정 리스트 > 회원 카드 클릭
- 회원관리 > 퇴원예정 리스트 > 회원 카드 클릭
- 회원관리 > 검색 결과 > 회원 카드 클릭
- 반배정 화면 > 회원 카드 클릭
- 반관리 화면 > 회원 카드 클릭
- 수업관리 > 출결 명단 > 회원 클릭
- 수업관리 > 수강생관리 > 회원 클릭
- 보강관리 > 보강대기 리스트 > 회원 클릭
- 보강관리 > 보강현황 리스트 > 회원 클릭

즉, 선생님 모드에서 회원 프로필은 화면을 따로 만들지 말고 무조건 하나의 공통 프로필 화면만 사용한다.

=====================================
[공통 프로필 화면 구성]

선생님 모드 회원 프로필 화면은 관리자 모드 프로필 화면 레이아웃을 최대한 동일하게 사용한다.

구성:
- 상단 헤더
  - 좌측: 뒤로가기
  - 중앙: 회원 이름
  - 우측: 홈

- 상단 상태 표시
  - 회원 이름
  - 현재 대표 상태 배지 1개
  - 예정 상태 배지 1개(있을 때만)
  - 예: 정상 / 휴원 / 퇴원 / 정상+연기예정 / 정상+퇴원예정

- 탭 또는 섹션
  - 기본정보
  - 수업정보
  - 보강
  - 레벨/평가
  - 학부모공유
  - 필요 시 기존 탭 유지

- 저장 가능한 수정 화면이면 저장 버튼 유지

- 하단 또는 상태 관리 카드 영역에
  관리자 모드와 동일한 "상태 변경" 버튼 추가

=====================================
[상태 변경 버튼]

선생님 모드 회원 프로필 화면에 관리자 모드와 동일한 스타일의 버튼을 추가한다.

버튼명:
- 상태 변경

버튼 위치:
- 관리자 모드와 동일한 상태 관리 영역
- 현재 상태 뱃지 옆 또는 바로 아래
- 등록일 / 등록경로 카드와 같은 그룹 안에서 보이게 정리

규칙:
- 선생님 모드에서는 이 버튼이 항상 보여야 한다
- 단, 권한상 선생님이 변경 가능한 상태만 노출한다
- 관리자 모드와 동일한 버튼 디자인 사용
- 별도 다른 버튼명 사용 금지

=====================================
[상태 변경 팝업 공통 사용]

상태 변경 버튼을 누르면
관리자 모드에서 사용하는 상태 변경 팝업 컴포넌트를 그대로 재사용한다.
선생님 모드 전용으로 새 팝업을 만들지 말고 공통 컴포넌트를 사용한다.

공통 팝업명 예시:
MemberStatusChangeModal

팝업에서 노출할 상태 선택:
- 정상
- 미배정
- 연기(최종 표시는 휴원 또는 연기 규칙에 맞게)
- 퇴원

주의:
선생님 모드에서는 "아카이브"와 "영구 삭제"는 노출하지 않는다.
아카이브 / 영구 삭제는 관리자 전용 유지.

=====================================
[선생님 상태 변경 규칙]

선생님이 상태 변경 팝업에서 연기 또는 퇴원을 선택하면
즉시 확정하지 말고 관리자 모드와 동일하게 선택 흐름을 사용한다.

-------------------------------------
1. 연기 선택 시

팝업 1:
- 연기 처리하시겠습니까?

버튼:
- 즉시 연기
- 다음 달부터 연기
- 취소

즉시 연기:
- memberStatus 즉시 휴원/연기 상태로 변경
- 정상/주1/주2/주3 리스트에서 즉시 제거
- 휴원 리스트로 즉시 이동
- 관리자 모드에도 즉시 반영

다음 달부터 연기:
- 현재 상태 유지
- pendingStatusChange = 연기 저장
- 카드에 [연기예정] 배지 표시
- 대기자 명단 > 연기예정 탭에 표시
- 이번 달 수업 유지
- 말일 또는 다음 달 수업 생성 시 실제 연기 상태 반영

-------------------------------------
2. 퇴원 선택 시

팝업 1:
- 퇴원 처리하시겠습니까?

버튼:
- 즉시 퇴원
- 다음 달부터 퇴원
- 취소

즉시 퇴원:
- memberStatus 즉시 퇴원
- 정상/주1/주2/주3 리스트에서 즉시 제거
- 퇴원 리스트로 즉시 이동
- 관리자 모드에도 즉시 반영
- 해당 수영장 접근 차단은 관리자 최종 퇴원 처리 규칙 유지

다음 달부터 퇴원:
- 현재 상태 유지
- pendingStatusChange = 퇴원 저장
- 카드에 [퇴원예정] 배지 표시
- 대기자 명단 > 퇴원예정 탭에 표시
- 이번 달 수업 유지
- 말일 또는 다음 달 수업 생성 시 실제 퇴원 상태 반영

-------------------------------------
3. 정상 선택 시

- 상태 예약값 제거 가능
- 복귀 가능한 조건이면 정상으로 복귀
- 필요한 반배정이 완료되지 않았으면 정상이 아니라 미배정 상태로 유지
- 주횟수 태그는 assignedClassCount 기준으로 재계산

-------------------------------------
4. 미배정 선택 시

- 현재 수업 반에서 제외
- 대표 상태를 미배정으로 변경
- 미배정 리스트로 이동
- 관리자 모드와 선생님 모드 양쪽에 즉시 반영

=====================================
[대기자 명단과 연결]

선생님 모드 회원관리 > 대기자 명단은 아래 구조를 유지한다.

탭:
- 전체
- 미배정
- 연기예정
- 퇴원예정

상태 변경 버튼으로 변경한 결과는 여기에도 바로 반영한다.

규칙:
- 전체에는 정상 포함 전체 회원 표시
- 미배정은 신규 등록 후 아직 반/선생님 배정이 안 된 회원
- 연기예정은 연기 예약 회원
- 퇴원예정은 퇴원 예약 회원

즉시 연기/즉시 퇴원은 예약 탭이 아니라 실제 상태 리스트로 이동시킨다.

=====================================
[회원카드와 프로필 상태 표시 통일]

회원카드와 프로필 상단 상태 영역은 같은 상태 계산 함수를 사용한다.

공통 계산:
- getMemberPrimaryStatus(member)
- getMemberFrequencyTag(member)
- getMemberPendingBadge(member)

프로필에서도 표시:
- 대표 상태 배지 1개
- 주횟수 배지 1개
- 예약 상태 배지 1개(있을 때만)

예:
- [정상] [주2회]
- [정상] [주2회] [연기예정]
- [정상] [주1회] [퇴원예정]
- [휴원] [주3회]
- [퇴원]

=====================================
[권한 구분]

선생님 모드:
- 상태 변경 가능
- 연기 / 퇴원 / 미배정 / 정상 복귀 가능
- 아카이브 불가
- 영구 삭제 불가

관리자 모드:
- 상태 변경 가능
- 연기 / 퇴원 / 미배정 / 정상 가능
- 최종 퇴원 처리 가능
- 아카이브 가능
- 영구 삭제 가능

학부모 모드:
- 상태 변경 버튼 노출 금지

=====================================
[상태 변경 결과 연동 범위]

선생님 모드 프로필에서 상태를 바꾸면 아래 화면이 모두 즉시 반영돼야 한다.

선생님 모드:
- 회원관리 전체
- 미배정 리스트
- 대기자 명단
- 연기예정 탭
- 퇴원예정 탭
- 반배정 화면
- 출결 명단
- 수강생관리
- 보강관리

관리자 모드:
- 회원관리 전체
- 상태별 게시판
- 퇴원 관련 게시판
- 연기/휴원 관련 게시판
- 미배정 게시판

절대 화면별 독립 상태로 두지 말고 공통 상태 저장소를 사용한다.

=====================================
[컴포넌트 구조]

공통으로 재사용할 것:
- UnifiedMemberCard
- TeacherMemberProfileScreen
- MemberStatusChangeModal
- getMemberPrimaryStatus
- getMemberFrequencyTag
- getMemberPendingBadge

중요:
선생님 모드 전용 상태 변경 팝업을 따로 만들지 말고
관리자 모드의 상태 변경 팝업과 같은 공통 컴포넌트를 재사용한다.

=====================================
[렌더링/상태관리 규칙]

- 화면 진입 시 항상 최신 데이터 fetch
- 상태 변경 저장 후 즉시 재조회
- 낙관적 업데이트 금지
- 로딩/에러 상태 분리
- profile screen, member list, search result 모두 동일 데이터 기준 사용
- 리스트 key 안정화
- 불필요한 re-render 방지
- 관리자/선생님 데이터 동기화 유지

=====================================
[필수 테스트]

1. 선생님 모드에서 회원관리 경로로 프로필 진입 시 상태 변경 버튼이 보이는지
2. 반배정 경로로 프로필 진입 시 같은 상태 변경 버튼이 보이는지
3. 출결 경로로 프로필 진입 시 같은 프로필 화면이 열리는지
4. 보강관리 경로로 프로필 진입 시 같은 프로필 화면이 열리는지
5. 상태 변경 버튼 클릭 시 관리자 모드와 동일한 팝업이 뜨는지
6. 연기 선택 시 즉시 연기 / 다음 달 연기 선택이 되는지
7. 퇴원 선택 시 즉시 퇴원 / 다음 달 퇴원 선택이 되는지
8. 다음 달 연기/퇴원 선택 시 연기예정/퇴원예정 배지가 붙는지
9. 즉시 연기/즉시 퇴원 시 리스트가 즉시 이동하는지
10. 상태 변경 결과가 관리자 모드에 즉시 반영되는지
11. 선생님 모드 어느 경로에서든 회원 프로필 UI가 동일한지
12. 검색 결과에서 프로필로 들어가도 동일 화면이 열리는지
