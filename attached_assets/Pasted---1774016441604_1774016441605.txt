지금부터 앱의 로그인/회원가입/역할진입 구조를 처음부터 정리해서 구현해줘.
이번 작업은 부분 수정이 아니라 “앱 첫 진입부터 전체 라우트 구조를 안정화”하는 작업이다.
중요한 건 예쁜 UI보다 경로/권한/상태/자동진입이 꼬이지 않게 만드는 것이다.

[최우선 원칙]
1. 로그인은 공통 진입이다.
2. 로그인창에서는 수영장 선택을 하지 않는다.
3. 역할 선택은 로그인 이후에 한다.
4. 한 계정은 역할을 여러 개 가질 수 있다.
5. 마지막 사용 역할과 마지막 선택 대상을 기억해서 자동 진입한다.
6. 권한이 사라지면 자동 진입하지 말고 역할 선택 또는 안내 화면으로 보낸다.
7. 작업 후 실제로 모든 버튼과 경로를 눌러서 검증하고, 어떤 경로를 확인했는지 보고해라.

==================================================
1. 앱 첫 실행 구조
==================================================

앱 실행 시 아래 순서대로 검사해줘.

- 로그인 세션 존재 여부
- 계정 상태(active/suspended/withdrawn)
- 마지막 사용 역할(last_used_role)
- 마지막 사용 역할의 현재 권한 유효 여부
- 마지막 선택 수영장(last_used_tenant)
- 학부모인 경우 마지막 선택 자녀(last_selected_student)

분기 규칙:
1) 세션 없음
- 로그인 화면으로 이동

2) 세션 있음 + 마지막 역할 권한 유효
- 해당 역할 홈으로 바로 이동

3) 세션 있음 + 마지막 역할 권한 상실
- 역할 선택 화면으로 이동
- 사용 가능한 역할이 없으면 “현재 이용 가능한 권한이 없습니다” 안내 화면 표시

예:
- 어제 선생님 모드 사용
- 오늘 관리자에게 승인 해제됨
- 앱 재실행 시 선생님 홈으로 자동 진입하면 안 됨
- 역할 선택 또는 권한 없음 화면으로 보내야 함

==================================================
2. 로그인 화면
==================================================

첫 화면은 공통 로그인 화면으로 만들어줘.

구성:
- 아이디 입력
- 비밀번호 입력
- 로그인 버튼
- 회원가입 버튼
- 비밀번호 찾기 버튼

중요:
- 로그인창에서 수영장 선택 금지
- 로그인창에서 역할 선택 금지

로그인 성공 시:
- 계정 조회
- 계정 상태 확인
- 승인된 역할 목록 조회
- 역할 수/마지막 역할에 따라 다음 화면 분기

로그인 실패 시:
- “아이디 또는 비밀번호가 올바르지 않습니다”

정지 계정:
- “이용이 제한된 계정입니다”

==================================================
3. 회원가입 화면
==================================================

회원가입은 최소 정보만 받게 해줘.

공통 회원가입 필수 항목:
- 아이디
- 비밀번호
- 비밀번호 확인
- 성별
- 휴대폰 인증

주소, 이메일, 상세 개인정보는 받지 않는다.

검증:
- 아이디 중복 확인
- 비밀번호 일치 여부
- 휴대폰 인증 완료 여부
- 성별 선택 여부

회원가입 완료 후 바로 홈으로 보내지 말고 무조건 역할 선택 화면으로 보내라.

==================================================
4. 역할 선택 화면
==================================================

회원가입 후 또는 로그인 후 역할 선택이 필요한 경우 아래 화면을 띄워줘.

노출:
- 학부모
- 선생님
- 관리자

이 화면은 “첫 진입 역할 선택” 개념이다.
계정을 영구 고정하는 화면이 아니다.

규칙:
- 역할이 하나뿐이면 이 화면을 건너뛰고 바로 해당 홈으로 보낸다.
- 역할이 여러 개면 마지막 사용 역할이 있으면 자동 진입
- 마지막 사용 역할이 없으면 역할 선택 화면 표시

==================================================
5. 학부모 진입 구조
==================================================

학부모는 로그인 후 바로 홈으로 보내지 말고 아래 순서로 진입하게 해줘.

학부모 선택
→ 수영장 검색/선택
→ 자녀 연결 단계
→ 자동 승인 또는 승인 요청 판별
→ 학부모 홈 진입

[학부모 수영장 선택]
- 수영장 코드 방식 아님
- 수영장 이름 검색 방식
- 지역 포함 검색 가능하게 구조 열어두기
- 검색 결과 리스트에서 수영장 선택

[자동 승인 규칙]
학생 등록 시 보호자 전화번호를 최대 2개 저장할 수 있게 해줘.

예:
- 보호자 전화번호1
- 보호자 전화번호2

학부모 가입 시 휴대폰 인증 번호와
학생에 등록된 보호자 번호1/2 중 하나가 일치하면 자동 승인.

자동 승인 조건:
- 선택한 수영장 안에 학생 정보 존재
- 해당 학생의 보호자 번호1 또는 2와 가입 번호 일치

번호 일치 시:
- 자동으로 자녀 연결
- 바로 학부모 홈 진입

번호 불일치 시:
- 관리자 승인 요청으로 전환
- 승인 대기 상태 표시

[학부모 표시 이름]
학부모의 기본 표시는 실명 말고 호칭으로 사용한다.
학부모가 역할 진입 시 자녀와의 관계 호칭을 직접 입력하게 해줘.

예:
- 서태웅 엄마
- 서태웅 아빠
- 서태웅 할머니

중요:
- 실명은 기본 화면에서 노출하지 않는다.
- 실명은 학부모 프로필 상세에서만 보이게 한다.
- 관리자 화면/선생님 화면/학부모 화면 모두 호칭으로 통일한다.

호칭은 학부모가 직접 수정 가능하게 해줘.

[다자녀 구조]
학부모 계정 하나에 자녀 여러 명 연결 가능하게 해줘.
자녀가 2명 이상이면 상단 탭으로 전환한다.

예:
- 서태웅 | 서태민 | 서태훈

규칙:
- 현재 선택된 자녀 기준으로만 화면 전체가 바뀐다
- 공지 제외
- 수업일지, 앨범, 출결, 프로필, 레벨기록 등은 현재 탭 기준으로 렌더링

==================================================
6. 선생님 진입 구조
==================================================

선생님 선택
→ 수영장 선택
→ 승인 요청 또는 승인 상태 확인
→ 승인 완료면 선생님 홈
→ 승인 대기면 승인 대기 화면

승인 대기 화면에는 아래 포함:
- 요청한 수영장 이름
- 승인 대기 문구
- 다시 선택 버튼
- 로그아웃 버튼

승인 해제된 경우:
- 로그인은 가능
- 선생님 모드 진입은 불가
- “권한 승인된 수영장이 없습니다” 문구 표시

==================================================
7. 관리자 진입 구조
==================================================

관리자 선택
→ 관리 가능한 수영장 목록
→ 수영장 선택
→ 관리자 홈 진입

관리자는 복수 수영장을 운영할 수 있으므로
로그인창에서 수영장을 고르지 않고
관리자 역할 진입 후 수영장 선택을 하게 해줘.

마지막 선택 수영장은 저장하고
다음 진입 시 그 수영장으로 자동 진입할 수 있게 해줘.
단 권한이 살아있을 때만.

==================================================
8. 역할 복수 구조
==================================================

계정은 하나, 역할은 여러 개 가능하게 구조를 짜라.

예:
- 학부모만
- 선생님만
- 관리자만
- 관리자 + 선생님

역할 상태값 예시:
- parent
- teacher
- admin
- sub_admin

승인 상태값 예시:
- pending
- approved
- revoked

last_used_role
last_used_tenant
last_selected_student
이 상태를 저장해라.

==================================================
9. 계정 전환 구조
==================================================

관리자/부관리자만 계정 전환 가능하게 해줘.

전환 버튼 위치:
- 관리자/선생님 화면에서 사용자 이름 옆 작은 전환 버튼 또는 아이콘

노출 규칙:
- 관리자: 전환 버튼 활성
- 부관리자: 전환 버튼 활성
- 일반 선생님: 전환 버튼 숨김
- 학부모: 전환 버튼 없음

전환 버튼 누르면 가능한 역할만 보여줘.
예:
- 관리자 모드로 전환
- 선생님 모드로 전환

로그아웃 없이 즉시 전환.

중요:
버튼만 숨기지 말고 서버 권한도 같이 막아라.
일반 선생님이 관리자 API나 관리자 페이지 접근하면 차단되어야 한다.

==================================================
10. 학부모 홈 구조
==================================================

학부모 홈은 하단 탭 없이 아이콘형 구조로 만들어라.

상단 정보:
- 현재 선택 자녀 이름
- 수영장 이름
- 반/수업시간
- 쪽지 버튼
- 설정 톱니바퀴

메인 아이콘 6개:
- 공지사항
- 수업일지
- 앨범
- 수업일정표
- 교육프로그램
- 수영정보

설정은 상단 톱니바퀴에서만 들어가게 해줘.
6번째 아이콘은 설정이 아니라 수영정보다.

==================================================
11. 학부모 홈 첫 화면 배치
==================================================

홈은 카드형 미리보기로 짧게 보여준다.
최신순 2개만 노출.

순서 고정:
1. 공지
2. 일지
3. 출결
4. 앨범

그 아래 공간은 광고 영역으로 남겨라.

[공지]
- 수영장 전체 공지
- 자녀별 분기 없음
- 최신 2개만

[최신소식]
- 공지 + 쪽지 답변을 섞어서 최신순 2개 노출
- 쪽지 답변이 오면 “선생님 답변이 도착했어요” 같은 형태로 보이게
- 클릭 시 해당 쪽지 상세 이동

[일지]
- 최신 2개 미리보기
- 내부적으로는 반 공통으로 작성해도
  학부모에게는 절대 공통/반전체 느낌이 나지 않게
  “우리 아이에게 전달된 일지”처럼 보이게 렌더링

[출결]
- 최근 기록 미리보기

[앨범]
- 최신 사진 2개 미리보기
- 단체 업로드여도 학부모에게는 개별처럼 전달된 느낌으로 렌더링

==================================================
12. 학부모 수업일지 상세
==================================================

구조:
- 상단: 날짜 + 수업시간
- 바로 아래: 일지 내용
- 그 아래: 첨부 사진 세로 리스트

규칙:
- 제목 없음
- 사진은 세로로 줄줄이 보이게
- 확대는 가능
- 사진 다운로드는 금지
- 다운로드는 앨범에서만 가능

하단 버튼 3개:
- 좋아요
- 감사합니다
- 쪽지

[좋아요/감사합니다]
- 개인 반응 버튼
- 선생님에게 알림 없음
- 카운트 없음
- 다른 사용자에게 공유 없음
- 눌렀을 때 본인 UI만 활성화되면 됨
- 다시 누르면 토글 해제 가능하게 구조 열어두기

[쪽지]
- 일지에서 보낸 쪽지는 선생님이 받을 때 어떤 일지에서 온 건지 간단히 표시
예:
- 2026.03.20 17:00 수업 일지 문의

==================================================
13. 학부모 쪽지 구조
==================================================

쪽지는 채팅형이 아니라 문의형이다.
그리고 쪽지마다 독립 건이다.
한 쪽지는 한 건으로 끝난다.
스레드형으로 길게 이어지지 않는다.

규칙:
- 제목 없음
- 내용만 작성
- 선생님에게만 보냄
- 목록형 저장
- 자녀별로 목록 분리하지 않음
- 계정 기준으로 통합 목록

목록 표시:
- 내용 첫 줄 일부
- 답변대기 / 답변완료
- 시간

답변 상태:
- 마지막 작성자가 학부모면 답변대기
- 마지막 작성자가 선생님이면 답변완료

알림:
- 푸시 알림 보냄
- 홈 최신소식 2줄에도 반영

==================================================
14. 학부모 앨범 구조
==================================================

앨범은 최신순 + 무한스크롤 구조로 만들어라.

상단 우측 버튼:
- 선택
- 다운로드

동작:
- 선택 누르면 다중 선택 모드
- 다운로드 누르면 선택된 사진 또는 현재 사진 다운로드
- 저장 위치는 휴대폰 사진첩

중요:
- 사진은 업로드 시 3MB로 압축 업로드하는 구조를 가정
- 다운로드는 제한 없이 편하게 허용
- 일지에서는 다운로드 불가
- 앨범에서만 다운로드 가능

화면 느낌:
- 단체/개별 구분 문구 없이
- 학부모에게는 항상 개인 사진첩처럼 보여야 함

==================================================
15. 학부모 출결 구조
==================================================

출결은 달력형 금지.
리스트형으로 누적되게 만들어라.

월별 요약 통계 없음.
월 구분선만 넣는다.

예:
3월 시작
──────────
3월 20일 출석
3월 18일 결석
3월 15일 보강

──────────
2월 시작
──────────
2월 28일 출석

위로 갈수록 최신
아래로 갈수록 과거

무한 누적 리스트 형태로 구현

==================================================
16. 학부모 설정 구조
==================================================

설정은 상단 톱니바퀴에서 진입.

항목:
- 부모 프로필
- 자녀 관리
- 이용약관
- 개인정보처리방침
- 로그아웃

부모 프로필:
- 기본 표시는 호칭
- 실명은 프로필 상세에서만 보이게
- 호칭 수정 가능

자녀 관리:
- 연결된 자녀 목록
- 자녀 카드 클릭 시 자녀 프로필
- 새 자녀 연결 가능 구조 열어두기

==================================================
17. 선생님 화면에서 학부모 표기
==================================================

선생님 화면에서도 학부모는 실명 대신 호칭으로 보여줘.

예:
- 서태웅 엄마
- 서태웅 아빠

실명은 기본 리스트에서 보여주지 말고,
필요하면 상세 프로필에서만 보이게 해라.

==================================================
18. 관리자 백업 구조 (화면 뼈대만)
==================================================

관리자 설정 안에 데이터 백업 메뉴 자리 만들어라.

화면에 최소한 아래 정보 표시:
- 마지막 백업 시각
- 백업 상태
- 최근 변경 로그 적재 상태

설계 방향은:
- 실시간 전체 백업 아님
- 변경 로그 서버 저장
- 새벽 수집
- 일별 스냅샷
- 주간 보관본

지금은 화면 뼈대와 상태 표시만 먼저 넣어도 된다.

==================================================
19. 반드시 확인할 예외 처리
==================================================

아래는 실제로 눌러서 확인해라.

[로그인]
- 아이디/비번 오류
- 정지 계정
- 세션 유지
- 세션 만료
- 비밀번호 찾기 동작

[역할 진입]
- 역할 하나일 때 자동 진입
- 역할 여러 개일 때 마지막 역할 자동 진입
- 마지막 역할 권한 사라졌을 때 예외 처리

[학부모]
- 수영장 검색 결과 없음
- 보호자 번호 일치 자동 승인
- 번호 불일치 승인 요청
- 자녀 2명 이상 탭 전환
- 호칭 수정 즉시 반영
- 공지/일지/출결/앨범 미리보기 2개 표시
- 최신소식에 공지/쪽지 답변 혼합 표시
- 일지 사진 다운로드 금지
- 앨범 다운로드 가능
- 출결 월 구분선 누적형 표시

[선생님]
- 승인 대기
- 승인 해제
- 일반 선생님 전환 버튼 비노출
- 일지 기반 문의 표시

[관리자]
- 복수 수영장 선택
- 마지막 수영장 기억
- 계정 전환
- 백업 화면 뼈대

==================================================
20. 작업 방식
==================================================

이번 작업은 반드시 아래 순서로 해라.

1. 로그인/회원가입/역할 선택/자동진입 구조 먼저 고정
2. 학부모 진입 구조와 자동 승인 연결
3. 학부모 홈/일지/출결/앨범/쪽지 구현
4. 선생님/관리자 진입 분기 연결
5. 계정 전환/권한 차단 연결
6. 모든 버튼/뒤로가기/홈 동작 직접 확인
7. 마지막에 무엇을 구현했고 무엇이 아직 미완성인지 솔직하게 보고

중요:
- 작업했다고 하지 말고 실제로 화면 전환과 버튼 동작을 확인해라
- 관리자/선생님/학부모 경로가 섞이면 안 된다
- 하드코딩 임시 연결로 끝내지 말고 상태값 기반으로 정리해라
