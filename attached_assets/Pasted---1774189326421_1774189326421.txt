프로젝트: 스윔노트 토이키즈 관리자 모드 구조 재정리
목표: 메뉴 중복 제거, 기능 경로 단일화, 관리자/선생님 충돌 제거, 데이터 보호/복구 구조 강화

이번 작업은 기능 추가보다 우선해서
1. 메뉴 구조를 재분류하고
2. 각 기능의 진입 경로를 1개로 고정하고
3. 데이터 삭제/복구/보존을 한곳으로 묶고
4. 구독/추가용량/결제를 한곳으로 묶고
5. 더보기 메뉴를 최소화하는 작업이다.

절대 원칙:
- 한 기능당 진입 경로 1개만 남긴다
- 관리자 모드는 운영/명부/승인/구독/복구 중심
- 선생님 모드는 수업/출결/일지/스케줄 중심
- 데이터 보호와 복구가 최우선
- 메뉴 이름만 보고 어디 들어가야 하는지 바로 알 수 있어야 한다
- 더보기는 임시 창고처럼 쓰지 말고 최대한 없앤다
- 버튼 안 눌림, 칸 밀림, 팝업 잔상, 뒤로가기 꼬임 재발 금지

[1. 상위 카테고리 최종 확정]
==================================================

상위 구조는 아래 5개로 고정:
1. 홈

==================================================
[2. 운영 관리에 포함할 항목 최종 확정]
==================================================

운영 관리는 사람/운영업무/결제 관련을 묶는 카테고리다.

운영 관리 하위 메뉴:
- 승인 관리
- 선생님 관리
- 학부모 계정
- 학부모 초대 내역
- 회원 명부
- 반 개설/삭제
- 구독 관리
- 추가 용량 구매
- 결제/구독 상태
- 플랜 변경 이력

중요:
구독 관리와 추가 용량 구매는 반드시 같이 둔다.
결제 관련은 모두 운영 관리 안에서 같이 볼 수 있게 묶는다.

즉 아래 항목은 운영 관리 안으로 통합:
- 구독 관리
- 추가 용량 구매
- 결제 상태
- 플랜 상태
- 용량 구매 상태
- 구독 변경 내역

==================================================

기존 "운영 설정" 이름은 "수업 설정"으로 변경한다.

수업 설정 하위 메뉴:

- 보강 정책 설정

- 알림 설정
- 레벨/테스트 설정
- 피드백 기본 설정

- 출결 기준
- 일지 기본 설정

규칙:

- 실제 출결 조작, 일지 작성, 스케줄 수정은 여기 두지 않는다
- 수업 실제 실행은 선생님 모드에서만 한다

==================================================
[4. 데이터 관리에 포함할 항목 최종 확정]
==================================================

데이터 관리는 슈퍼관리자와 공유/연동되는 축이다.
데이터 보호/로그/삭제/복구/보존/사용량 관련을 전부 여기에 넣는다.

데이터 관리 하위 메뉴:
- 백업·복구
- 이벤트 기록
- 저장공간 현황
- 계정별 사용량
- 카테고리별 사용량
- 삭제·보존 정책
- 스냅샷 목록
- 복구 이력
- 슈퍼관리자 공유 로그 상태

중요:
기존의 아래 기능을 전부 하나로 통합:
- 데이터 삭제
- 데이터 복구
- 보존 정책
- 원본 데이터 삭제

즉 "원본 데이터 삭제"는 독립 메뉴로 두지 말고
데이터 관리 > 삭제·보존 정책 안으로 넣는다.

[5. 삭제·보존 정책 화면 통합]
==================================================

기존 분산된 삭제 관련 화면을 하나로 통합한다.

새 메뉴명:
- 삭제·보존 정책

하위 섹션:
A. 복구 가능 데이터
- 회원 정보
- 승인 상태
- 반/수업 설정
- 출결
- 일지 텍스트
- 설정값

B. 복구 불가 원본 데이터
- 사진 원본
- 영상 원본

C. 보존 기간 정책
- 사진
- 영상
- 기록/일지

D. 원본 삭제 실행
- 기간 선택
- 항목 선택
- 미리보기
- 삭제 실행

주의 문구:
- 삭제된 원본 파일은 복구할 수 없습니다
- 이벤트 로그와 삭제 실행 기록은 보존됩니다

[9. 연결 서비스 메뉴 해체/정리]
==================================================

현재 연결 서비스 안에 운영 메뉴가 섞여 있다면 해체

재배치:
- 선생님 관리 -> 운영 관리
- 학부모 계정 -> 운영 관리
- 학부모 초대 내역 -> 운영 관리
- 구독/추가용량/결제 -> 운영 관리
- 이벤트 기록/사용량 -> 데이터 관리
- 보강정책/레벨설정 -> 수업 설정

연결 서비스에는 진짜 외부 연동만 남긴다:
- 문자앱 호출 방식
- 앱스토어/플레이스토어 링크
- 외부 연동 상태 표시

[10. 더보기 메뉴 축소]
==================================================

더보기는 최대한 없앤다.

작업 원칙:
- 더보기 안에 있는 메뉴를 위 5개 상위 카테고리로 분산
- 중복 링크 제거
- 더보기는 정말 남길 수밖에 없는 보조 메뉴만 유지
- 가능하면 홈 또는 상위 카테고리 진입만으로 모든 기능 접근 가능하게
