지금부터 메신저 화면을 관리자/선생님 모드 포함 전부 공통 구조로 다시 구축하라.
이번 작업은 단순 버그 수정이 아니라, 메신저 레이아웃을 카카오톡과 유사한 사용감으로 재구성하는 작업이다.

중요:
- 기존 메신저 데이터/메시지 로직/전송 로직/탭 구조는 최대한 유지
- 다른 기능 화면은 건드리지 말 것
- 관리자/선생님 모드 메신저가 서로 다르게 동작하면 안 된다
- 메신저 화면은 반드시 공통 컴포넌트 기반으로 다시 구성할 것
- iPhone 기준으로 키보드, safe area, 하단 입력창, 메시지 스크롤이 모두 정상 작동해야 함
- 수정 후 관리자/선생님 모드 둘 다 실제 테스트하고 상세 보고할 것

==================================================
[1. 현재 버그 명확히 수정]

현재 확인된 버그:

1) 관리자 모드 메신저
- 입력창 터치해도 키보드가 안 뜨는 경우가 있음
- 즉, 입력 포커스/키보드 호출 구조가 깨져 있음

2) 선생님 모드 메신저
- 키보드가 올라오면 채팅 입력창을 가림
- 입력칸/전송 버튼이 키보드 뒤에 숨음

3) 관리자/선생님 모드 메신저 구조가 서로 다르게 깨져 있음
- 공통 UI 구조가 아님

이번 작업은 위 버그를 모두 수정하고,
메신저 화면을 공통 카톡형 레이아웃으로 다시 만든다.

==================================================
[2. 메신저 최종 구조 - 카카오톡형으로 재구축]

메신저 화면은 아래 4영역으로 재구성한다.

[상단 고정 헤더]
- 화면 제목
- 필요한 액션 버튼(기존 기능 유지)

[상단 탭/필터 영역]
- 전체 / 사진 / 회원이전 등 기존 탭 유지

[메시지 리스트 영역]
- 스크롤 가능
- 새 메시지 전송 시 자동 하단 정렬

[하단 입력 바]
- @ 버튼
- 사진 버튼
- 새로고침/전환 버튼(기존 기능 있으면 유지)
- 텍스트 입력창
- 전송 버튼

핵심:
입력 바는 카카오톡처럼 항상 하단에 고정되고,
키보드가 올라오면 키보드 바로 위로 함께 올라와야 한다.

==================================================
[3. 반드시 공통 컴포넌트로 만들 것]

관리자 모드 / 선생님 모드 / 향후 다른 메신저 화면이
모두 같은 방식으로 동작하도록 공통 컴포넌트로 다시 만들 것.

예시 구조:
- MessengerScreenLayout
- MessengerMessageList
- MessengerInputBar

금지:
- 관리자용 따로, 선생님용 따로 임시 처리
- 화면마다 다른 keyboard 대응
- 화면마다 다른 absolute 배치

==================================================
[4. 키보드 대응 - 최우선]

iPhone 기준으로 아래를 반드시 만족해야 한다.

1. 입력창 터치 시 키보드가 정상 호출될 것
2. 키보드가 올라오면 입력 바 전체가 키보드 위로 올라올 것
3. 입력창, 전송 버튼, 보조 버튼이 키보드에 가려지면 안 됨
4. 메시지 리스트 영역만 남은 공간에 맞게 줄어들고 스크롤될 것
5. 키보드 내려가면 입력 바는 다시 하단 safe area 위 원위치
6. 관리자/선생님 모드 모두 동일하게 작동할 것

구현 지시:
- iOS keyboard avoiding 구조를 공통 적용
- safe area inset 고려
- 입력 바는 keyboard height에 맞춰 이동
- 메시지 리스트는 flex로 남은 영역 차지
- absolute 고정만으로 처리하지 말 것

==================================================
[5. 입력 바 구조 - 카톡형 고정]

하단 입력 바는 아래처럼 구성한다.

좌측:
- @ 버튼
- 사진 버튼
- 회전/전환 버튼(기존 있으면 유지)

중앙:
- 둥근 입력창
- 여러 줄 입력 가능 여부는 현재 기능 유지하되, 최소 1줄 높이 보장

우측:
- 전송 버튼

동작 규칙:
- 입력창 누르면 키보드 뜸
- 글자 입력되면 전송 버튼 활성화
- 전송 후 입력창 비움
- 전송 후 메시지 리스트 최하단 유지

==================================================
[6. 메시지 리스트 규칙]

1. 메시지 리스트는 헤더/탭/입력바를 제외한 나머지 영역을 모두 사용
2. 스크롤 가능
3. 새 메시지 전송 시 하단으로 자동 이동
4. 키보드가 올라와도 최근 메시지가 계속 보이게
5. 공백이 과도하게 남지 않도록 레이아웃 재조정

현재 캡처처럼
상단이 비고 메시지들이 아래에 몰리는 비정상 레이아웃 수정할 것

즉:
- 리스트가 전체 높이를 자연스럽게 채우도록 수정
- 필요 이상 상단 빈 공간 생기지 않게 할 것

==================================================
[7. safe area / iPhone 대응]

iPhone 기준으로 반드시 아래 반영

1. 상단 헤더는 safe area 아래에서 시작
2. 하단 입력 바는 home indicator 영역을 침범하지 말 것
3. 키보드 + 하단 safe area + 입력 바가 서로 겹치지 않게 할 것
4. 화면 잘림/겹침 없이 보이게 할 것

==================================================
[8. 모드별 동작 동일화]

아래 두 모드에서 메신저가 완전히 같은 방식으로 동작해야 함

- 관리자 모드 메신저
- 선생님 모드 메신저

검증 기준:
- 입력창 포커스
- 키보드 호출
- 입력 바 위치
- 메시지 스크롤
- 전송 버튼 작동

어느 한쪽만 되고 한쪽 안 되면 실패

==================================================
[9. 기존 기능 유지]

아래 기능은 유지할 것

- 전체 / 사진 / 회원이전 탭
- 특정 대상 선택 전송 구조
- 사진 탭 / 앨범 진입 구조
- 회원이전 메시지 구조
- 기존 메시지 목록 / 표시 방식
- 기존 메신저 데이터 로직

이번 작업은 메신저 레이아웃 재구축 + 입력/키보드 버그 수정이다.
기능을 삭제하거나 다른 화면을 수정하지 말 것.

==================================================
[10. 절대 금지]

- 관리자/선생님 메신저를 따로따로 임시 수정 금지
- 입력창이 키보드에 가려지는 상태 유지 금지
- 입력창 눌러도 키보드 안 뜨는 상태 유지 금지
- 메시지 리스트 상단에 과도한 빈 공간 남기기 금지
- 기존 메시지/전송 기능 삭제 금지
- 다른 탭 화면 수정 금지

==================================================
[11. 테스트 지시 - 반드시 실제 수행]

수정 후 아래 테스트를 관리자 모드와 선생님 모드 둘 다 수행하라.

테스트 1.
입력창 터치 시 키보드가 즉시 뜨는지 확인

테스트 2.
키보드가 올라왔을 때 입력 바 전체가 키보드 위로 올라오는지 확인

테스트 3.
입력창 / 전송 버튼 / 보조 버튼이 키보드에 가려지지 않는지 확인

테스트 4.
메시지 리스트가 상단부터 자연스럽게 배치되고 과도한 빈 공간이 없는지 확인

테스트 5.
새 메시지 전송 시 메시지가 추가되고 자동으로 하단 정렬되는지 확인

테스트 6.
키보드가 내려가면 입력 바가 하단 safe area 위로 정상 복귀하는지 확인

테스트 7.
전체 / 사진 / 회원이전 탭 전환 후에도 입력 바/키보드 구조가 깨지지 않는지 확인

테스트 8.
관리자 모드 메신저와 선생님 모드 메신저가 동일하게 동작하는지 확인

테스트 9.
기존 메신저 기능(대상 선택, 사진, 회원이전 등)이 깨지지 않았는지 확인

==================================================
[12. 보고 형식]

작업 후 아래 순서로 보고하라.

1. 메신저를 어떤 공통 컴포넌트 구조로 다시 만들었는지
2. 관리자/선생님 모드 공통화 방식
3. 키보드 대응을 어떻게 처리했는지
4. 하단 입력 바 고정 구조를 어떻게 만들었는지
5. 메시지 리스트 빈 공간 문제를 어떻게 해결했는지
6. 테스트 1~9 결과
7. 아직 남은 임시 처리 부분
8. 다른 기능/화면에 영향 준 부분이 있으면 숨기지 말고 적기

==================================================
[13. 작업 범위 제한]

이번 작업은
메신저 레이아웃 재구축 + 키보드 버그 수정
까지만 수행한다.

다른 화면 통일 작업은 하지 말 것.
