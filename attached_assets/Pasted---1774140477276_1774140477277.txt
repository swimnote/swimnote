현재 슈퍼관리자 홈 메뉴 분류가 잘못되었고, 아이콘이 너무 많아 첫 화면에서 혼란을 준다.
기능을 줄이는 것이 아니라, 대분류 중심으로 다시 묶어서 재구성한다.

==================================================
[1. 홈 화면 재구성]
==================================================

홈은 운영 콘솔형 유지:
- KPI 6칸
- 오늘 처리할 일
- 리스크 요약
- 최근 감사 로그

그 아래 아이콘은 기존처럼 전부 나열하지 말고 5개만 둔다.

새 홈 대분류 아이콘 5개:
1. 운영 관리
2. 지원 센터
3. 보호·통제
4. 보안·설정
5. 감사·리스크

기존 세부 메뉴들은 각 대분류 하위 화면으로 이동시킨다.

==================================================
[2. 운영 관리 그룹]
==================================================

운영 관리 하위 메뉴:
- 운영자 관리
- 구독·결제 관리
- 구독 상품 설정
- 저장공간 관리
- 저장공간 정책
- SMS 과금/사용량 관리

SMS 과금/사용량 관리 화면 추가:
- 운영자별 월 SMS 발송 수
- 무료 제공 수
- 초과 발송 수
- 예상 과금액
- 미결제 SMS 사용량
- 발송 차단 여부
- 발송 실패 수

==================================================
[3. 지원 센터 그룹]
==================================================

지원 센터 하위 메뉴:
- 고객센터
- 정책·컴플라이언스
- 초대/SMS 발송 관리
- 인증번호 발송 기록

초대/SMS 발송 관리에서:
- 운영자 초대
- 선생님 초대
- 학부모 연결 요청
- 인증번호 발송
- 재발송
- 링크 만료 관리
- 발송 성공/실패 로그
를 볼 수 있게 한다.

==================================================
[4. 보호·통제 그룹]
==================================================

보호·통제 하위 메뉴:
- 데이터·킬스위치
- 백업/복구/스냅샷
- 기능 플래그
- 읽기전용 제어

==================================================
[5. 보안·설정 그룹]
==================================================

새 메뉴 추가: 슈퍼관리자 보안·설정

이 화면에 아래 기능을 한데 모은다.

A. 슈퍼관리자 계정 관리
- 계정 목록
- 권한 등급
- 활성 여부

B. 1차 인증
- 비밀번호 변경
- 최근 변경일
- 비밀번호 정책 표시

C. 2차 인증
- 활성 여부
- 기본 방식 선택
- 민감 작업 추가 인증 여부

2차 인증 옵션:
- 비활성
- OTP 앱 인증
- SMS 인증
- 이메일 인증
- OTP + SMS 백업
- 로그인 시 항상 2차 인증
- 킬스위치/삭제/권한변경 시만 추가 인증

D. 외부 서비스 연결 상태
- Supabase DB
- Cloudflare R2
- PortOne PG
- SMS Provider
- APNs
- Email Provider

각 항목 표시:
- 연결 상태
- 마지막 확인 시간
- 경고/정상/끊김

E. 세션/접속 관리
- 최근 로그인 시간
- 최근 로그인 IP
- 접속 기기
- 현재 활성 세션
- 다른 기기 로그아웃
- 로그인 실패 횟수

F. 보안 정책
- 로그인 실패 제한
- 계정 잠금 시간
- 재인증 필요 작업 목록

==================================================
[6. 감사·리스크 그룹]
==================================================

감사·리스크 하위 메뉴:
- 운영 로그·감사
- 장애·리스크 센터
- 보안 이벤트 로그
- 민감 작업 로그

==================================================
[7. 자동삭제/과금 흐름 유지 원칙]
==================================================

데이터 보호 우선 원칙 유지:
- 결제 실패만으로 자동삭제 금지
- 저장공간 초과만으로 자동삭제 금지
- 먼저 과금 유도
- 그 다음 경고
- 마지막 업로드 차단
- 해지 확정 후에만 삭제 예약 가능

==================================================
[8. 구현 원칙]
==================================================

- 홈 아이콘 수 줄이기
- 기능 삭제 금지, 위치 재분류
- 새 메뉴 추가 시 기존 기능 깨지지 않게 route 점검
- 보안/설정/연동은 한 화면에서 확인 가능하게
- derived state는 useMemo로만 계산
- useEffect 내 setState 무한루프 금지

==================================================
[9. 테스트 필수]
==================================================

반드시 실제로 테스트:
1. 홈에서 5개 대분류 진입 가능 여부
2. 운영 관리 하위 메뉴 진입
3. 지원 센터 하위 메뉴 진입
4. 보호·통제 하위 메뉴 진입
5. 보안·설정 하위 메뉴 진입
6. 감사·리스크 하위 메뉴 진입
7. 비밀번호 변경 화면 열림
8. 2차 인증 옵션 변경 동작
9. 외부 서비스 상태 표시
10. SMS 과금/사용량 화면 표시
11. 앱 재실행 후 무한루프 없음
12. 아이폰 스크롤/뒤로가기/팝업 정상 작동

테스트 없이 완료라고 하지 말고,
진입/상태변경/뒤로가기까지 확인 후 보고할 것.
