[스윔노트 — 학부모 가입 승인 후 로그인 + 자녀 연결 검증 지시문]

현재 학부모 가입 신청 → 관리자 커뮤니티 탭 조회 → 승인/거절 흐름은 정상 연결된 상태다.
이번 작업은 승인 이후 실제 사용 흐름이 끊기지 않는지 검증하는 작업이다.

핵심 목표:
- 승인된 학부모가 실제 로그인 가능한지
- 승인 후 학부모 관리 목록에 정상 반영되는지
- 자녀 연결을 나중에 했을 때 학부모 화면에 즉시 반영되는지

━━━━━━━━━━━━━━━━━━
1. 승인 후 로그인 가능 여부 검증
━━━━━━━━━━━━━━━━━━

반드시 실제 확인:

1. 학부모가 가입 신청
2. 관리자가 승인
3. 승인된 학부모 계정으로 로그인 시도
4. 로그인 성공 확인
5. 학부모 홈 또는 첫 화면 정상 진입 확인

추가 확인:
- rejected 상태 학부모는 로그인/연결 진행 차단되는지
- pending 상태 학부모는 정책에 맞는 안내가 나오는지

중요:
- 승인됐는데 로그인 안 되면 실패
- 승인 전/거절 상태가 승인처럼 보이면 실패

━━━━━━━━━━━━━━━━━━
2. 승인 후 학부모 관리 목록 반영
━━━━━━━━━━━━━━━━━━

반드시 확인:

- 승인된 학부모가 학부모 관리 목록에 보이는지
- 아직 자녀 미연결 상태라면 목록에서 구분 가능한지
- 학부모 상세 진입 가능 여부
- 학부모 전화번호/이름 정상 노출 여부

권장:
- 자녀 미연결 상태 표시가 있으면 더 좋음
- 단, 지금 단계에서는 “보인다/연결 가능하다”가 우선

━━━━━━━━━━━━━━━━━━
3. 자녀 연결 검증
━━━━━━━━━━━━━━━━━━

실제 흐름으로 검증:

1. 승인된 학부모 선택
2. 학부모 관리 화면에서 자녀 연결
3. 저장
4. DB 반영 확인
5. 학부모 앱 재진입 또는 재조회
6. 연결된 자녀 정보 노출 확인

반드시 확인:
- 연결 전에는 자녀 없음 상태
- 연결 후에는 자녀 카드/자녀 정보 표시
- 잘못된 학생 연결이 안 되는지
- 다른 수영장 학생 연결 금지

━━━━━━━━━━━━━━━━━━
4. 승인과 자녀 연결 분리 유지
━━━━━━━━━━━━━━━━━━

정책 유지:
- 가입 승인 = 계정 사용 허용
- 자녀 연결 = 별도 작업

즉:
- 승인 단계에서 자녀 연결 강제 금지
- 다시 승인/연결을 하나의 단계로 합치지 말 것

━━━━━━━━━━━━━━━━━━
5. 실시간 또는 즉시 반영 검증
━━━━━━━━━━━━━━━━━━

반드시 확인:

- 승인 후 학부모 관리 목록에 즉시 반영되는지
- 자녀 연결 후 학부모 앱에 즉시 반영되거나 재조회 시 반영되는지
- 캐시 때문에 예전 상태가 계속 보이지 않는지

허용:
- mutation 후 invalidate / refetch
- 재진입 시 최신값 보장

금지:
- 승인됐는데 화면엔 계속 pending
- 자녀 연결됐는데 앱엔 계속 미연결

━━━━━━━━━━━━━━━━━━
6. 점검 대상
━━━━━━━━━━━━━━━━━━

반드시 확인:

- 학부모 로그인 API/로직
- 승인 처리 API
- 학부모 관리 목록 API
- 학부모 상세 조회 로직
- 자녀 연결 저장 API
- 학부모 홈/첫 화면 조회 로직
- 캐시 무효화 또는 재조회 로직

━━━━━━━━━━━━━━━━━━
7. 실제 테스트 시나리오
━━━━━━━━━━━━━━━━━━

반드시 아래 순서로 검증:

1. 학부모 가입 신청
2. 관리자 커뮤니티 탭에서 요청 확인
3. 승인
4. 학부모 로그인 성공
5. 학부모 관리 목록에서 해당 계정 확인
6. 자녀 연결 전 상태 확인
7. 관리자 화면에서 자녀 연결
8. 학부모 앱에서 자녀 표시 확인
9. 거절 계정/대기 계정도 각각 테스트

━━━━━━━━━━━━━━━━━━
8. 실패 기준
━━━━━━━━━━━━━━━━━━

아래 중 하나라도 해당하면 실패:

- 승인됐는데 로그인 안 됨
- 승인된 학부모가 관리 목록에 안 보임
- 자녀 연결 저장 안 됨
- 학부모 앱에 자녀가 안 뜸
- 다른 수영장 학생이 연결됨
- 캐시 때문에 상태가 오래 갱신 안 됨

━━━━━━━━━━━━━━━━━━
9. 보고 형식
━━━━━━━━━━━━━━━━━━

반드시 포함:

- 수정 파일 경로
- 승인 후 로그인 결과
- 학부모 관리 목록 반영 결과
- 자녀 연결 전/후 DB 결과
- 학부모 앱 노출 결과
- pending / approved / rejected 상태별 테스트 결과
- 실제 클릭 테스트 결과

“연결됨”만 쓰지 말고
승인 → 로그인 → 자녀 연결 → 학부모 화면 반영까지 실제 결과를 같이 제출할 것

━━━━━━━━━━━━━━━━━━
