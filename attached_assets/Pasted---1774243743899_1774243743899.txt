[스윔노트 운영설정 구조 정리 + 브랜드설정/화이트라벨 분리 + 지점관리(멀티 수영장) 시스템 구현 통합 지시서]

이번 작업은 “설정 구조 정리 + 멀티 수영장 운영 기능” 구현이다.
기존에 겹쳐있는 메뉴를 정리하고,
수영장 운영자가 여러 수영장을 운영할 수 있는 구조를 완성한다.

중요:
- 단순 UI 정리가 아니라 실제 기능 연결까지 정확히 분리해야 한다
- 브랜드설정과 화이트라벨은 역할이 다르므로 절대 섞지 말 것
- 지점관리 기능은 로그인 흐름까지 영향을 주는 핵심 구조이므로 안정성 최우선
- 기존 데이터/권한/결제/학생/레벨 구조 절대 깨지면 안 됨
- 모든 기능은 실제 작동 기준으로 구현하고 테스트까지 완료할 것

==================================================
1. 브랜드설정 vs 화이트라벨 기능 분리
==================================================

현재 문제:
- 브랜드설정과 화이트라벨이 기능적으로 겹쳐 있음

해결:
기능을 명확히 분리하고 각각의 역할에 맞게 연결

--------------------------------------------------
1-1. 브랜드설정 (수영장 개별 브랜딩)
--------------------------------------------------

목적:
각 수영장이 “자기 브랜드”를 꾸미는 영역

포함 기능:
- 수영장 이름
- 로고 업로드
- 대표 색상 (primary color)
- 보조 색상
- 기본 텍스트 스타일
- 학부모 화면에 노출되는 이름
- 공지/문구 스타일

적용 범위:
- 학부모 모드
- 선생님 모드 일부
- 수영장 내부 UI

중요:
브랜드설정은 “디자인/표현 영역”

--------------------------------------------------
1-2. 화이트라벨 (플랫폼 숨김/제거)
--------------------------------------------------

목적:
플랫폼 흔적 제거

포함 기능:
- “Powered by 스윔노트” 제거
- 앱 내 스윔노트 명칭 숨김
- 수영장 이름만 노출
- 커스텀 앱 느낌 강화

적용 범위:
- 전체 앱
- 로그인 이후 모든 화면

중요:
화이트라벨은 “플랫폼 노출 제어”
브랜드설정과 절대 섞지 말 것

==================================================
2. 지점관리 (멀티 수영장 운영 기능)
==================================================

목적:
하나의 수영장 운영자가 여러 수영장을 운영할 수 있게 하는 기능

--------------------------------------------------
2-1. 지점관리 메뉴 생성
--------------------------------------------------

위치:
관리자 모드 > 운영설정 또는 별도 메뉴

메뉴명:
지점관리

--------------------------------------------------
2-2. 수영장 등록 기능
--------------------------------------------------

등록 조건:
- 전화번호 인증된 수영장 운영자만 가능

기능:
- 새로운 수영장 추가 등록
- 수영장 이름 입력
- 기본 정보 입력

등록 시:
- 새로운 pool_id 생성
- 기존 수영장과 완전히 분리된 데이터 공간 생성

중요:
- 학생/수업/결제 데이터 절대 공유 금지 (복사 제외)

--------------------------------------------------
2-3. 수영장 목록 화면
--------------------------------------------------

지점관리 화면에서
운영자가 관리하는 모든 수영장 목록 표시

표시 항목:
- 수영장 이름
- 현재 활성 수영장 표시
- 전환 버튼

--------------------------------------------------
2-4. 수영장 전환 기능
--------------------------------------------------

동작:
- 수영장 선택 시 해당 수영장으로 진입

중요:
- “로그인 직후 진입”과 동일하게 동작해야 함
- 상태/데이터/세션 전부 새로 로드
- 이전 수영장 상태 유지 금지

즉:
수영장 전환 = 재진입

==================================================
3. 수영장 초기 설정 (신규 등록 시)
==================================================

새로운 수영장 등록 시 기본적으로:

- 완전 초기 상태
- 설정 없음
- 레벨 없음(기본 fallback 사용)
- 가격 없음
- 결제 없음
- 피드백 없음

==================================================
4. 프랜차이즈/정보 복사 기능
==================================================

목적:
여러 수영장을 운영할 때 일부 설정만 복사 가능하게

--------------------------------------------------
4-1. 복사 가능한 항목 (제한)
--------------------------------------------------

아래 4개만 복사 가능:

1. 가격 설정
2. 레벨 설정
3. 결제 정보
4. 피드백 문구 세팅

그 외:
- 학생 데이터 ❌
- 출결 ❌
- 일지 ❌
- 사진 ❌
- 로그 ❌

--------------------------------------------------
4-2. 선택 복사 방식 (중요)
--------------------------------------------------

신규 수영장 등록 시 아래 UI 제공:

[ ] 가격 가져오기  
[ ] 레벨 가져오기  
[ ] 결제정보 가져오기  
[ ] 피드백 문구 가져오기  

→ 체크된 항목만 복사

즉:
전체 복사 금지
부분 선택 복사만 허용

--------------------------------------------------
4-3. 복사 시점
--------------------------------------------------

- 수영장 생성 직후
- 초기 세팅 단계에서만 가능

이후에는:
- 수동 수정만 가능

==================================================
5. 데이터 구조 및 분리
==================================================

필수 구조:

- user_id (운영자)
- pool_id (수영장)

관계:
- 한 user_id는 여러 pool_id를 가질 수 있음

모든 데이터는 반드시 pool_id 기준으로 분리:

- 학생
- 수업
- 출결
- 보강
- 레벨
- 결제
- 매출
- 로그

절대 금지:
- pool_id 없는 데이터
- 수영장 간 데이터 혼합

==================================================
6. 로그인/진입 흐름 수정
==================================================

조건:
운영자가 수영장 2개 이상 보유 시

로그인 후:
→ 수영장 선택 화면 먼저 표시

구성:
- 수영장 리스트
- 선택 버튼

선택 후:
→ 해당 수영장으로 진입

단일 수영장일 경우:
→ 바로 진입

==================================================
7. UI/UX 규칙
==================================================

- 브랜드설정과 화이트라벨 메뉴 명확히 분리
- 지점관리 메뉴는 눈에 잘 보이게 배치
- 수영장 전환 시 로딩 처리 명확히
- 수영장 이름 항상 상단에 표시
- 현재 활성 수영장 명확히 표시
- 복사 선택 UI는 체크박스 형태
- 잘못된 복사 방지 위해 설명 문구 추가

==================================================
8. 수정 금지 영역
==================================================

다음 기능 절대 깨지면 안 됨:

- 기존 로그인 시스템
- 기존 학생/출결/보강 구조
- 기존 레벨 시스템
- 기존 결제/매출 구조
- 기존 권한 체계
- 기존 학부모 연결 구조

==================================================
9. 테스트 및 검증 필수
==================================================

반드시 아래 테스트 수행:

1. 브랜드설정과 화이트라벨 기능이 완전히 분리되어 동작하는지
2. 화이트라벨 ON 시 플랫폼 이름 완전히 제거되는지
3. 수영장 2개 이상 등록 가능한지
4. 수영장 목록 정상 표시되는지
5. 수영장 전환 시 데이터 완전히 분리되는지
6. 전환 후 이전 수영장 데이터가 안 보이는지
7. 신규 수영장 생성 시 초기 상태 유지되는지
8. 선택 복사 기능 정상 작동하는지
9. 체크한 항목만 복사되는지
10. 레벨/가격/결제/피드백만 복사되는지
11. 학생/출결/일지 데이터 복사 안 되는지
12. pool_id 기준 데이터 완전 분리 확인
13. 로그인 시 수영장 선택 화면 정상 표시되는지

10. 최종 목적
==================================================

- 설정 구조 명확히 분리
- 수영장 운영자가 여러 지점 운영 가능
- 데이터 완전 분리
- 필요한 설정만 선택 복사
- 플랫폼 확장성 확보

이 기준으로 정확하게 구현하라.
