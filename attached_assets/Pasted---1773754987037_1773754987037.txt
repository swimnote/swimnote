지금부터 로그인/회원가입 시스템만 새로 구현하라.
중요:
- 기존 테스트 로그인 화면은 삭제하지 말고 유지
- 기존 각 모드(슈퍼관리자 / 관리자 / 선생님 / 학부모) 진입 구조 유지
- 다른 기능 화면, 탭 구조, 데이터 구조, 기존 모드 화면은 건드리지 말 것
- 이번 작업은 오직 로그인/회원가입 흐름만 추가/교체
- 로그인 UI를 현재 앱 첫 진입 화면에 적용하되, 테스트 로그인 진입 경로는 별도로 유지
- 작업 후 각 회원가입 경로를 전부 실제 테스트하고 결과를 상세 보고할 것

==================================================
[1. 작업 목표]

현재 앱의 첫 진입 로그인 화면을 아래 구조로 교체한다.

기존 유지:
- 테스트 로그인 화면 유지
- 테스트 모드별 진입 유지
- 기존 화면/기능/탭/데이터 구조 유지

새로 만들 것:
- 아이디/비밀번호 기반 로그인 화면
- 로그인 실패 분기
- 회원가입 진입
- 회원가입 역할 선택
- 수영장 관리자 가입
- 선생님 가입
- 학부모 가입 (수영장 찾기 / 코드 입력 2가지)
- 승인 대기 / 즉시 승인 흐름

==================================================
[2. 기존 화면 유지 조건]

1. 현재 앱에 있는 테스트 로그인 화면은 삭제 금지
2. 테스트 로그인 화면으로 들어가는 기존 버튼/경로 유지
3. 각 모드 테스트 진입(슈퍼관리자/관리자/선생님/학부모) 유지
4. 로그인/회원가입 시스템은 앱 기본 진입 화면에 새로 붙이고,
   테스트 로그인은 별도 보조 진입 경로로 둔다
5. 다른 모드 UI, 기능, 탭, 메신저, 출결, 수업, 정산 등은 절대 수정하지 말 것

==================================================
[3. 로그인 화면 구조]

로그인 첫 화면은 단순하게 만든다.

구성:
- 아이디 입력창
- 비밀번호 입력창
- 로그인 버튼
- 하단 작은 안내 문구: 처음 이용하시나요?

처음부터 크게 노출하지 않는 것:
- 회원가입 버튼 대형 노출 금지
- 비밀번호 찾기 처음부터 노출 금지

==================================================
[4. 로그인 판별 로직]

로그인 버튼 클릭 시 아래 순서대로 판별

--------------------------------------
[경우 A] 아이디 존재 + 비밀번호 일치
→ 해당 계정 역할에 맞는 홈/진입 화면으로 이동

--------------------------------------
[경우 B] 아이디 없음
→ 선택형 팝업 표시

팝업 문구:
가입된 계정이 없습니다

버튼:
- 다시 입력
- 회원가입

동작:
- 다시 입력: 팝업 닫고 로그인 화면 유지
- 회원가입: 회원가입 시작 화면 이동

--------------------------------------
[경우 C] 아이디 존재 + 비밀번호 틀림

1회 실패:
- 비밀번호가 일치하지 않습니다 문구만 표시
- 다시 입력 유도

2회 실패:
- 비밀번호가 일치하지 않습니다 문구 유지
- 비밀번호 찾기 버튼 노출

비밀번호 찾기 클릭 시:
- 한 번 더 비밀번호를 다시 입력하도록 재확인 팝업 표시

재확인 팝업:
제목: 비밀번호 확인
내용: 비밀번호를 다시 한 번 입력해 주세요
입력창: 비밀번호
버튼: 확인

여기서 다시 틀리면
→ 3회 실패로 간주
→ 실제 비밀번호 찾기 프로세스로 이동

==================================================
[5. 비밀번호 찾기]

비밀번호 찾기 기본 흐름:
- 아이디 확인
- 인증 수단 확인 (임시 구조 가능)
- 새 비밀번호 설정

이번 작업에서는 MVP 기준으로 아래까지 구현:
- 비밀번호 찾기 화면 진입
- 아이디 확인
- 새 비밀번호 재설정 화면
- 완료 후 로그인 화면 복귀

실제 문자/이메일 인증은 임시 구조로 두되,
화면 흐름은 완성형처럼 보이게 구현해도 됨

==================================================
[6. 회원가입 시작 화면]

회원가입 시작 시 첫 화면은 역할 선택

화면 제목:
회원 유형 선택

선택 카드 3개:
- 수영장 관리자
- 선생님
- 학부모

슈퍼관리자는 회원가입 대상에 넣지 않는다

==================================================
[7. 수영장 관리자 회원가입]

흐름:
수영장 관리자 선택
→ 수영장 운영 정보 입력
→ 관리자 정보 입력
→ 가입 완료

입력 항목:
- 수영장 이름
- 관리자 이름
- 아이디
- 비밀번호
- 연락처
- 수영장 운영 기본 정보

중요:
- 이미 어느 정도 만들어진 관리자 가입 입력 구조가 있으면 그걸 재사용
- 이번 작업은 기존 관리자 가입용 입력 폼을 로그인 시스템과 연결하는 방식으로 구현

처리 결과:
- 가입 완료
- 관리자 계정 생성
- 바로 사용 가능 또는 기존 테스트 정책에 맞게 활성화

==================================================
[8. 선생님 회원가입]

흐름:
선생님 선택
→ 수영장 선택
→ 선생님 정보 입력
→ 가입 승인 요청
→ 승인 대기 화면

--------------------------------------
[8-1. 수영장 선택]
- 지도/추천 없음
- 검색 방식만 사용
- 수영장 이름 두 글자 이상 입력 시 검색 시작
- 입력한 텍스트가 포함된 수영장 목록 표시
- 목록 선택 후 다음

예:
토이
토이키즈
입력 시 토이키즈스윔클럽 등 노출

--------------------------------------
[8-2. 선생님 정보 입력]
입력 항목:
- 이름
- 아이디
- 비밀번호
- 연락처
- 최소 필요 정보

버튼:
- 가입 승인 요청

--------------------------------------
[8-3. 결과]
- 상태: 승인 대기
- 관리자 승인 전까지 사용 불가
- 승인 대기 안내 화면 표시

==================================================
[9. 학부모 회원가입]

학부모는 두 가지 방식으로 가입

학부모 회원가입 첫 화면:
가입 방식을 선택해 주세요

버튼:
- 수영장 찾기
- 수영장 코드 입력

--------------------------------------
[9-1. 수영장 찾기 방식]

흐름:
수영장 찾기
→ 수영장 검색/선택
→ 학부모 정보 입력
→ 가입 요청
→ 관리자 승인 대기

수영장 검색 규칙:
- 두 글자 이상 입력 시 검색
- 입력값 포함 수영장 목록 표시
- 지도/추천 없음

학부모 정보 입력 항목:
- 학부모 이름
- 아이디
- 비밀번호
- 연락처
- 자녀 이름
- 필요한 기본 정보

결과:
- 가입 요청 완료
- 관리자 승인 대기 화면 표시

--------------------------------------
[9-2. 수영장 코드 입력 방식]

이 방식은 선생님/관리자가 먼저 학부모를 등록하고
코드를 발송한 경우에 사용

흐름:
수영장 코드 입력
→ 코드 확인
→ 미리 등록된 정보 자동 불러오기
→ 확인
→ 즉시 승인
→ 사용 가능

코드 규칙:
- 유효시간 24시간
- 만료 시 코드 만료 팝업 표시

코드 입력 화면 기능:
- 코드 입력
- 붙여넣기 버튼
- 가능하면 딥링크 구조 고려
- 문자 링크로 들어오면 자동 입력 가능한 구조 여지 마련

자동 불러오는 정보:
- 학부모 이름
- 자녀 정보
- 수영장 정보
- 미리 등록된 기본 정보

결과:
- 즉시 승인
- 바로 사용 가능

==================================================
[10. 상태값 구조]

회원가입 후 상태는 아래처럼 관리

- 수영장 관리자: ACTIVE
- 선생님: PENDING
- 학부모(수영장 찾기): PENDING
- 학부모(코드 입력): ACTIVE

==================================================
[11. 팝업 규칙 적용]

이번 로그인/회원가입에도 공통 팝업 규칙 적용

고정 규칙:
- 팝업 높이 75%
- 제목 가운데 정렬
- 뒤로가기 없음
- X 버튼으로 닫기
- 바깥 터치로 닫기
- 아래로 스와이프로 닫기
- 내용 많으면 팝업 내부 스크롤
- 스크롤바 표시
- 팝업 크기/위치 통일

팝업 종류:
- 선택형 팝업
- 입력형 팝업

==================================================
[12. 첫 진입 구조]

앱 첫 진입:
- 새 로그인 화면 표시

단, 기존 테스트 로그인 진입도 유지해야 하므로
아래 중 하나로 구현:
- 로그인 화면 하단에 작게 테스트 로그인 진입
또는
- 개발용 진입 버튼 별도 유지

중요:
- 테스트 로그인은 보조 경로
- 기본 진입은 새 로그인 화면

==================================================
[13. 절대 금지]

- 기존 테스트 로그인 기능 삭제 금지
- 기존 모드 화면 수정 금지
- 기존 탭 구조 수정 금지
- 기존 데이터 구조 대규모 변경 금지
- 로그인 구현하면서 다른 기능 건드리지 말 것

==================================================
[14. 테스트 지시 - 반드시 실제 수행]

수정 후 아래 테스트를 반드시 실제로 수행하고 결과를 상세히 보고할 것.

테스트 1. 로그인 성공
- 기존에 있는 계정으로 로그인
- 역할별로 정상 진입되는지 확인

테스트 2. 아이디 없음
- 존재하지 않는 아이디 입력
- 가입된 계정이 없습니다 팝업 표시 확인
- 다시 입력 / 회원가입 버튼 각각 동작 확인

테스트 3. 비밀번호 오류
- 1회 오류 시 오류문구만 표시되는지 확인
- 2회 오류 시 비밀번호 찾기 버튼 노출 확인
- 비밀번호 찾기 클릭 후 재확인 팝업 확인
- 3회 실패 후 실제 비밀번호 찾기 화면 진입 확인

테스트 4. 수영장 관리자 회원가입
- 입력 후 가입 완료되는지 확인
- 계정 생성되는지 확인

테스트 5. 선생님 회원가입
- 수영장 검색이 두 글자 이상부터 되는지 확인
- 수영장 선택 후 가입 요청 가능한지 확인
- 상태가 승인 대기로 저장되는지 확인

테스트 6. 학부모 수영장 찾기 회원가입
- 수영장 검색/선택 가능 여부
- 정보 입력 후 가입 요청
- 승인 대기 상태 저장 확인

테스트 7. 학부모 코드 회원가입
- 유효한 코드 입력 시 자동 정보 불러오기 확인
- 즉시 승인 확인
- 바로 사용 가능한지 확인

테스트 8. 코드 만료
- 만료된 코드 입력 시 만료 팝업 표시 확인

테스트 9. 테스트 로그인 유지 확인
- 기존 테스트 로그인 화면/경로가 그대로 남아 있는지 확인
- 각 모드 테스트 진입이 여전히 가능한지 확인

==================================================
[15. 보고 형식]

작업 후 아래 형식으로 보고해라.

1. 로그인/회원가입 화면을 어디에 어떻게 추가했는지
2. 기존 테스트 로그인 화면을 어떻게 유지했는지
3. 로그인 판별 로직 구현 방식
4. 회원가입 역할 선택 및 각 흐름 구현 방식
5. 코드 가입 방식 구현 방식
6. 상태값 처리 방식
7. 테스트 1~9 결과
8. 아직 남은 미완성/임시 처리 부분
9. 다른 기능을 건드린 부분이 있으면 숨기지 말고 적기

==================================================
[16. 실패 기준]

아래 중 하나라도 해당하면 실패로 보고해라.

- 테스트 로그인 화면이 사라짐
- 기존 모드 테스트 진입이 깨짐
- 아이디 없음 분기가 작동 안 함
- 비밀번호 2회 실패 흐름이 안 맞음
- 선생님/학부모 가입 분기가 헷갈리거나 안 이어짐
- 수영장 검색이 2글자 이상 조건대로 안 됨
- 코드 가입 즉시 승인이 안 됨
- 로그인 구현하면서 다른 주요 화면이 깨짐

이 작업은 로그인/회원가입만 만드는 작업이다.
다른 기능은 절대 건드리지 말고, 로그인 시스템만 교체/추가하고 테스트까지 완료해라.
