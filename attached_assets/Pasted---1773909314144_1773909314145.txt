지금부터 선생님 모드 > 수업 > 내 반 > 반 선택 팝업 안의 “반 이동” 기능을 최종 규칙대로 수정하라.

이번 작업 목표는
현재 선생님이 담당 중인 학생을 현재 반으로 이동시킬 때,
선택한 기존 반에서만 해당 학생을 제거하고 현재 반에 추가하며,
이동 기록을 메신저 공지 탭에 자동으로 남기게 만드는 것이다.

중요:
- 기존 수업 스케줄표, 반 선택 팝업, 출결/수업일지 기능은 깨지면 안 된다
- 반 이동은 현재 반으로 학생을 데려오는 기능이다
- 현재 반은 제거 대상 선택지에 나오면 안 된다
- 주 2회/3회 학생은 선택한 기존 반 하나에서만 제거해야 한다
- 이동 완료 후 메신저 공지 탭에 자동 메시지가 남아야 한다
- 학생 카드에는 주 몇회 표시가 반드시 들어가야 한다

==================================================
[1. 반 이동 기능 정의]

반 이동 버튼은 유지한다.

의미:
- 현재 선생님이 담당 중인 학생을
- 현재 선택한 반으로 이동시키는 기능

즉:
- 현재 반으로 학생을 데려오는 기능
- 단, 기존 반 전체 제거가 아니라
- 선택한 특정 반에서만 해당 학생을 제거하고 현재 반에 추가하는 구조

==================================================
[2. 반 이동 대상 제한]

반 이동 버튼 클릭 시
목록에는 “현재 선생님이 담당 중인 학생만” 보여야 한다.

표시 제외:
- 다른 선생님만 담당 중인 학생
- 현재 반과 무관한 전체 수영장 학생
- 미배정 학생 (이건 별도 미배정 버튼에서 처리)

==================================================
[3. 학생 카드 표시 정보]

반 이동 학생 목록의 각 카드에는 최소 아래 정보를 표시하라.

- 학생 이름
- 주 몇회
- 현재 속한 반 목록

예:
서태웅
주 2회
현재 반: 월 15:00반 / 수 14:00반

중요:
선생님이 이동 전에 현재 반 구성을 바로 이해할 수 있어야 한다

==================================================
[4. 현재 반 선택지 제외]

주 2회/3회 학생 이동 시
“어떤 반에서 제거할까요?” 팝업이 뜰 때

현재 선택한 반은 제거 대상 선택지에서 반드시 제외하라.

예:
현재 반 = 목 19:00반

학생 현재 반:
- 월 15:00반
- 수 14:00반
- 목 19:00반

제거 선택지에는 아래만 보여야 한다:
- 월 15:00반
- 수 14:00반

즉:
현재 반은 선택지에 나오면 안 된다

==================================================
[5. 주 1회 학생 이동 규칙]

주 1회 학생은 현재 반이 아닌 기존 반이 1개뿐이다.

동작:
1. 확인 팝업 표시
2. 이동 확인 시
   - 기존 반에서 해당 학생 제거
   - 현재 반에 학생 추가

확인 문구 예:
이 학생을 현재 반으로 이동하시겠습니까?
기존 반에서는 해당 학생만 제거됩니다.

==================================================
[6. 주 2회 / 주 3회 학생 이동 규칙]

주 2회 이상 학생은
기존 반 중 어떤 반에서 뺄지 먼저 선택해야 한다.

1단계 팝업 제목:
- 어떤 반에서 제거할까요?

선택지:
- 현재 반 제외한 기존 반들만 표시

2단계 확인 팝업:
- 선택한 반에서 해당 학생을 제거하고 현재 반으로 이동하시겠습니까?

중요:
- 선택한 기존 반 하나에서만 제거
- 다른 기존 반들은 그대로 유지
- 전체 반을 날리거나 나머지 반까지 건드리면 안 됨

==================================================
[7. 데이터 처리 순서]

반 이동 완료 시 아래 순서대로 처리하라.

1. 학생의 현재 반 목록 조회
2. 현재 반에 이미 포함돼 있는지 확인
   - 이미 있으면 중복 이동 금지
3. 주횟수 확인
4. 제거 대상 반 결정
   - 주1회: 자동
   - 주2회/3회: 사용자 선택
5. 선택된 기존 반에서 해당 학생만 제거
6. 현재 반에 학생 추가
7. 학생의 전체 반 수가 주횟수와 일치하는지 확인
8. 저장
9. 현재 반 학생 목록 즉시 갱신
10. 기존 반 목록도 반영

==================================================
[8. 절대 금지 로직]

아래는 절대 발생하면 안 된다.

- 현재 반이 제거 선택지에 보이는 것 금지
- 기존 반 전체 제거 금지
- 선택 안 하고 자동으로 반 제거 금지
- 다른 기존 반까지 함께 제거 금지
- 같은 학생이 같은 반에 중복 배정되는 것 금지
- 주횟수보다 많은 반에 배정되는 것 금지

==================================================
[9. 메신저 공지 자동 기록]

반 이동 완료 후
메신저 공지 탭에 자동 메시지를 반드시 남겨라.

메시지 타입:
- system_move

내용 예시:
- 서태웅 회원이 월 15:00반에서 목 19:00반으로 이동되었습니다.
- 아리 회원이 수 14:00반에서 목 19:00반으로 이동되었습니다.

규칙:
- 자동 메시지는 수정 불가
- 공지 탭 노란색 구조 유지
- 관리자/선생님이 모두 볼 수 있게 기존 공지 탭 규칙 그대로 적용

==================================================
[10. UI 문구 수정]

반 이동 팝업 안내 문구는 아래 의미가 분명하게 보이도록 수정하라.

좋은 예:
- 선택한 학생을 현재 반으로 이동합니다.
- 이동 시 선택한 기존 반에서만 해당 학생이 제거됩니다.

나쁜 예:
- 제거 후 완전 미배정
- 기존 반에서 제거됩니다
(너무 포괄적이고 오해 소지 있음)

[15. 절대 금지]

- 현재 반이 제거 대상 선택지에 나타나는 것 금지
- 선택한 반 하나만 제거해야 하는데 여러 반이 같이 제거되는 것 금지
- 이동 후 메신저 공지 기록이 남지 않는 것 금지
- 학생 카드에 주횟수 정보 빠뜨리는 것 금지
- 기존 출결/일지/미배정 기능 깨뜨리기 금지

이번 작업은
반 이동을 “선택한 기존 반 하나에서만 해당 학생을 제거하고 현재 반에 추가하는 구조”로 정확히 구현하고,
학생 카드에 주 몇회 표시를 넣으며,
이동 완료 시 메신저 공지 탭에 자동 기록까지 남기게 만드는 작업이다.
