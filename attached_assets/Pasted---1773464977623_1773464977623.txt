이번 작업은 비용 절감을 위해 한 번에 정리해서 진행해줘.
중복 수정 없이 전체 구조를 먼저 분석한 뒤 필요한 변경만 한 번에 반영해줘.

이번 작업 범위:
"수영장 승인 권한에 필수 서류 검증 조건 추가 + 승인 가능 조건 제한 + 승인 화면 보완"까지만 진행해줘.
그 외 UI 리디자인, 불필요한 리팩토링, 성능 최적화는 하지 마.

목표:
일반 플랫폼 관리자도 수영장 승인 권한(canApprovePools)을 가질 수 있지만,
수영장 승인은 필수 서류가 모두 업로드된 상태에서만 가능하게 해줘.

필수 승인 조건:
1. 사업자등록증 업로드 완료
2. 자동이체 계좌인증 업로드 또는 인증 완료

이 두 가지가 모두 충족되지 않으면 승인할 수 없게 해줘.

필수 요구사항:

1. 수영장 신청 데이터에 서류 상태 추가
- businessLicenseStatus
- bankAccountVerificationStatus

상태값 예시:
- notUploaded
- uploaded
- verified

2. 수영장 신청/상세 화면에 필수 서류 항목 추가
- 사업자등록증 업로드
- 자동이체 계좌인증 업로드 또는 인증 정보
- 현재 서류 상태를 볼 수 있게 해줘

3. 승인 가능 조건
- 두 서류가 모두 최소 uploaded 이상이어야 승인 버튼이 활성화되게 해줘
- 둘 중 하나라도 notUploaded 이면 승인 버튼 비활성화
- 사유 메시지 표시:
  "사업자등록증과 자동이체 계좌인증이 모두 업로드되어야 승인할 수 있습니다"

4. 백엔드/API 권한 체크
- 프론트엔드에서 버튼만 숨기지 말고
  승인 API에서도 두 서류 상태를 검사해줘
- 서류가 부족하면 승인 요청이 들어와도 실패하게 해줘
- 예:
  { "success": false, "message": "required documents missing" }

5. 권한 구조 유지
- superPlatformAdmin 은 승인 가능
- canApprovePools=true 인 일반 platformAdmin 도 승인 가능
- 단, 두 경우 모두 필수 서류 업로드 조건을 만족해야만 승인 가능

6. 상태 구조 정리
수영장 신청/승인 상태를 아래처럼 정리해줘:
- approvalPending
- documentsIncomplete
- documentsReady
- approvedFree
- approvedPaid
- rejected
- suspended

7. documentsReady 조건
- 사업자등록증과 자동이체 계좌인증이 모두 uploaded 이상이면
  documentsReady 로 처리 가능하게 해줘
- 이 상태일 때만 승인 가능하게 해줘

8. 플랫폼 관리자 승인 화면 보완
- 수영장 신청 목록/상세에서 아래가 보이게 해줘
  - 사업자등록증 업로드 여부
  - 자동이체 계좌인증 업로드 여부
  - 현재 승인 가능 여부
- 서류 누락이면 빨간색 또는 경고 텍스트로 표시
- 승인 버튼은 비활성화 상태로 보여줘도 됨

9. 테스트 데이터
- 서류 2개 다 없는 신청 1개
- 사업자등록증만 있는 신청 1개
- 두 서류 다 있는 신청 1개
- 이 3가지 테스트 케이스를 만들어
  승인 가능/불가가 정확히 나뉘는지 확인 가능하게 해줘

10. JSON 응답 유지
- 모든 API 응답은 JSON 형식 유지
- 에러 발생 시에도 JSON 반환
- 예:
  { "success": true, "data": ... }
  { "success": false, "message": "required documents missing" }

작업 방식:
1. 관련 신청 데이터 모델, 승인 API, 플랫폼 승인 화면을 먼저 분석
2. 같은 파일은 가능하면 한 번만 수정
3. UI 전체 변경, 스타일 정리, 성능 최적화는 하지 마
4. 이번 턴에서는 오직 승인 조건과 필수 서류 체크만 진행

마지막 결과 보고 형식:
1. 수정한 파일 목록
2. 추가한 상태값 목록
3. 추가한 서류 필드 목록
4. 승인 가능 조건 설명
5. 승인 불가 조건 설명
6. 테스트 케이스 목록
7. 남은 후속 작업 목록
