[스윔노트 — SMS 인증 실제 연결 작업 지시문]

현재 상태:
- 테스트 데이터 초기화 완료
- 슈퍼관리자 계정 로그인 정상
- 실제 가입 검증 가능한 초기 상태 확보
- mock SMS 코드(123456) 제거 완료

이번 작업은 가짜 인증 없이 실제 휴대폰 인증이 동작하도록 SMS 인증을 연결하는 작업이다.

1. 작업 목표
━━━━━━━━━━━━━━━━━━

구현 범위:

- 휴대폰번호로 인증번호 발송
- 인증번호 검증
- 인증 만료 처리
- 재사용 방지
- 재발송 제한
- 가입 흐름 실제 연결

핵심:
- mock 성공 처리 금지
- console.log만 찍고 통과 금지

2. 우선 구현 구조
━━━━━━━━━━━━━━━━━━

이번 단계는 아래 4개를 반드시 만든다.

1. DB 테이블
- phone_verifications

2. 서버 API
- POST /auth/send-sms-code
- POST /auth/verify-sms-code

3. SMS provider 연동 모듈
- provider별 발송 함수 분리
- sendSms() 공통 함수로 감싸기

4. 앱 화면 연결
- 관리자 가입
- 학부모 가입
- 이후 휴대폰 인증 필요한 화면 공통 사용 가능 구조

━━━━━━━━━━━━━━━━━━
3. DB 테이블 생성
━━━━━━━━━━━━━━━━━━

phone_verifications 테이블 생성

필수 컬럼:
- id
- phone
- code_hash
- purpose
- expires_at
- is_used
- created_at
- verified_at
- request_ip
- attempt_count

권장 규칙:
- code는 평문 저장 금지, 해시 저장
- 만료 시간 기본 3분
- 성공 검증 후 is_used = true
- verified_at 기록
- 같은 인증코드 재사용 금지

인덱스 권장:
- (phone, purpose, created_at desc)
- (phone, purpose, is_used)

purpose 예:
- pool_admin_signup
- parent_signup
- password_reset

━━━━━━━━━━━━━━━━━━
4. 인증번호 생성 규칙
━━━━━━━━━━━━━━━━━━

- 6자리 숫자
- 매 요청마다 새 코드 생성
- 너무 쉬운 고정값 금지
- 로그에 원문 코드 남기지 말 것

예:
- 638241
- 204917

━━━━━━━━━━━━━━━━━━
5. SMS 발송 API
━━━━━━━━━━━━━━━━━━

POST /auth/send-sms-code

입력:
- phone
- purpose

동작 순서:
1. 전화번호 형식 검사
2. 재발송 제한 검사
3. 6자리 코드 생성
4. code_hash 저장
5. expires_at 저장
6. SMS provider로 실제 발송
7. 성공/실패 응답 반환

성공 응답 예:
- success: true
- expires_in: 180

실패 예:
- invalid_phone
- cooldown_active
- provider_not_configured
- sms_send_failed

중요:
- provider 미설정이면 조용히 실패 금지
- 명확한 오류코드 반환

━━━━━━━━━━━━━━━━━━
6. SMS 검증 API
━━━━━━━━━━━━━━━━━━

POST /auth/verify-sms-code

입력:
- phone
- code
- purpose

동작 순서:
1. 최신 미사용 인증 레코드 조회
2. 만료 여부 확인
3. code_hash 비교
4. attempt_count 증가 처리
5. 일치하면 is_used = true, verified_at 저장
6. 불일치하면 실패 반환

성공 응답:
- success: true
- verified: true

실패 예:
- code_not_found
- code_expired
- invalid_code
- already_used
- too_many_attempts

━━━━━━━━━━━━━━━━━━
7. 제한/보안 규칙
━━━━━━━━━━━━━━━━━━

반드시 적용:

- 재발송 쿨다운: 60초
- 인증 만료: 3분
- 최대 입력 실패: 5회
- 5회 초과 시 해당 인증코드 무효 처리 또는 일정 시간 차단
- 이미 사용한 코드는 재검증 금지
- 같은 번호에 너무 많은 발송 요청 차단

추가 권장:
- request_ip 저장
- 번호별 일일 발송 제한 검토

━━━━━━━━━━━━━━━━━━
8. SMS provider 구조
━━━━━━━━━━━━━━━━━━

구조는 분리해서 만든다.

예:
- services/sms/sendSms.ts
- services/sms/providers/<provider>.ts

공통 함수:
- sendSms({ phone, message })

원칙:
- provider 교체 가능 구조
- 가입 로직 안에 provider 코드 직접 박아넣지 말 것

환경변수 예:
- SMS_PROVIDER
- SMS_API_KEY
- SMS_API_SECRET
- SMS_SENDER_PHONE
- SMS_SERVICE_ID (필요시)

중요:
- env 누락 시 명확히 실패
- fake success 금지
