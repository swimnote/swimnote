[수업설정 > 피드백 기본설정 버튼 연결 오류 수정 지시서]

현재 버그:
수영장 관리자 모드의
수업설정 팝업 안에 있는 `피드백 기본설정` 버튼이
원래 연결되어야 하는 “피드백 설정 전용 페이지”가 아니라
잘못해서 “수영장 설정 페이지”로 연결되고 있다.

이번 작업 목표:
- `피드백 기본설정` 버튼을 올바른 전용 페이지로 연결
- 잘못 연결된 route/action 제거
- 실제 피드백 설정 기능이 열리는지 검증
- 뒤로가기/팝업 복귀 규칙도 같이 확인

중요:
- 단순 텍스트 수정이 아니라 실제 네비게이션 연결 수정이다
- `수영장 설정 페이지`로 가는 잘못된 연결은 완전히 제거
- 네가 이미 만들어 놓은 “피드백 관리자가 설정하는 페이지”를 정확히 찾아서 그쪽으로 연결
- 테스트 없이 완료 보고 금지

==================================================
1. 버그 정의
==================================================

현재 경로:
수영장 관리자 홈
→ 수업설정 팝업
→ 피드백 기본설정 클릭
→ 잘못해서 수영장 설정 페이지로 이동

원하는 경로:
수영장 관리자 홈
→ 수업설정 팝업
→ 피드백 기본설정 클릭
→ 피드백 설정 전용 페이지 이동

==================================================
2. 반드시 먼저 확인할 것
==================================================

아래를 먼저 찾고 확인하라.

1. `피드백 기본설정` 버튼의 onPress / route / handler
2. 현재 연결된 잘못된 페이지 route
3. 실제로 연결되어야 하는 피드백 설정 전용 페이지 파일/route
4. 수업설정 팝업 안의 다른 버튼들과 연결 방식 비교
   - 수업운영설정
   - 보강정책설정
   - 권한설정
   - 알림설정
   - 레벨/테스트설정
   - 피드백 기본설정

중요:
`피드백 기본설정`만 오연결된 것인지,
아니면 route key/shared handler가 잘못 재사용되고 있는지도 같이 점검

==================================================
3. 수정 원칙
==================================================

- `피드백 기본설정` 버튼은 오직 피드백 설정 전용 페이지로만 연결
- 수영장 설정 페이지로 가는 연결 완전 제거
- 하드코딩된 잘못된 route name 제거
- 공통 메뉴 배열/아이템 매핑을 쓰고 있다면 label과 route mapping이 뒤바뀌지 않았는지 확인
- 아이콘/텍스트/UI는 유지하고 연결만 정확히 수정

==================================================
4. 피드백 기본설정이 열어야 하는 페이지
==================================================

다음 기능이 있는 전용 페이지여야 한다.

- 피드백 기본 문구 설정
- 수업 일지 기본 문구
- 칭찬/개선/메모 기본 구조
- 관리자 기본 템플릿 또는 기본값 설정

즉,
“수영장 운영 전체 설정” 페이지가 아니라
“피드백 설정 전용 페이지”여야 한다.

이미 만들어 놓은 페이지가 있으면 반드시 그 페이지를 연결하고,
없다면 기존 피드백 설정 기능이 있는 컴포넌트/스크린을 찾아 정확히 route 연결해라.

==================================================
5. 네비게이션 규칙도 같이 점검
==================================================

`피드백 기본설정` 클릭 후 전용 페이지로 이동했을 때:

- 뒤로가기 → 수업설정 팝업(또는 직전 수업설정 맥락)으로 복귀
- 홈으로 바로 튀면 안 됨
- 팝업/상세/페이지 전환 규칙은 지금까지 맞춘 공통 규칙 유지

중요:
연결만 고치고 뒤로가기 흐름 깨지면 안 된다.

==================================================
6. 회귀 테스트
==================================================

아래도 같이 확인:
- 수업운영설정 버튼 정상 여부
- 보강정책설정 버튼 정상 여부
- 권한설정 버튼 정상 여부
- 알림설정 버튼 정상 여부
- 레벨/테스트설정 버튼 정상 여부

즉:
수업설정 팝업 안의 메뉴 매핑 전체가 꼬여 있지 않은지 확인

==================================================
7. 테스트 필수 시나리오
==================================================

반드시 실제 클릭 테스트:

시나리오 A
- 관리자 홈 진입
- 수업설정 팝업 열기
- 피드백 기본설정 클릭
기대 결과:
- 수영장 설정 페이지가 아니라 피드백 설정 전용 페이지 열림

시나리오 B
- 피드백 설정 페이지에서 뒤로가기
기대 결과:
- 직전 수업설정 맥락으로 자연 복귀
- 홈으로 튀지 않음

시나리오 C
- 수업설정 팝업의 다른 버튼들도 각각 눌러보기
기대 결과:
- 각 버튼이 자기 페이지로 정확히 연결
- 서로 뒤바뀐 연결 없음

==================================================
8. 원인 후보
==================================================

아래 가능성 우선 점검:
- 메뉴 배열에서 label은 피드백인데 route가 settings로 잘못 매핑
- onPress 핸들러 copy/paste 실수
- route name 오타
- default export / named export 잘못 불러와서 fallback route로 감
- 피드백 설정 페이지가 누락되어 임시로 설정 페이지로 연결되어 있었음

==================================================
9. 수정 금지
==================================================

절대 깨지면 안 되는 것:
- 수업설정 팝업 UI 자체
- 다른 설정 메뉴 버튼 연결
- 홈/뒤로가기 공통 규칙
- 피드백 설정 페이지 내용 자체

이번 작업은
`피드백 기본설정` 버튼의 오연결 수정 작업이다.

==================================================
10. 완료 보고 형식
==================================================

반드시 아래 형식으로 보고하라.

1. 잘못 연결된 직접 원인
2. 수정한 route/action 내용
3. 수정한 파일 목록
4. 피드백 기본설정 클릭 테스트 결과
5. 뒤로가기 복귀 테스트 결과
6. 수업설정 팝업 내 다른 버튼 회귀 테스트 결과
7. 남아 있는 오연결 여부

중요:
- “연결 수정 완료” 한 줄 보고 금지
- 실제 클릭해서 열린 페이지 기준으로 적어라
- 미확인 버튼은 미확인이라고 적어라
