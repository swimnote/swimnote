[선생님 모드 네비게이션 규칙 미적용 부분 전면 수정 지시서]

현재 관리자 모드에는 뒤로가기/팝업 복귀 규칙이 어느 정도 적용되었지만,
선생님 모드에는 아직 동일 규칙이 적용되지 않았다.

이번 작업 목표는
관리자 모드에서 맞춘 네비게이션 규칙을
선생님 모드 전체에도 동일하게 적용하는 것이다.

중요:
- 선생님 모드에서 하위 화면/상세/팝업에서 뒤로가기 시 홈 화면으로 튀면 안 된다
- 팝업을 닫으면 반드시 직전 화면으로 돌아와야 한다
- 홈 화면으로 가는 것은 홈 버튼을 눌렀을 때만 허용
- 이번 작업은 선생님 모드 전체의 공통 네비게이션 규칙 수정이다
- 기능 로직은 건드리지 말고 네비게이션/스택/팝업 상태 복원만 수정하라
- 수정 후 반드시 실제 클릭 테스트를 하고 결과를 보고하라

==================================================
1. 선생님 모드 공통 규칙
==================================================

반드시 아래 규칙을 적용한다.

1. 상세 화면에서 뒤로가기
- 무조건 직전 목록/직전 화면 복귀
- 홈 화면 점프 금지

2. 팝업/모달 닫기
- 팝업을 연 직전 화면 복귀
- 홈 화면 점프 금지
- 아이콘 첫 화면 강제 이동 금지

3. 선생님 모드 아이콘/메뉴 안에서 진입한 하위 화면
- 한 단계씩만 뒤로 빠져나와야 함
- 스택 유지 필수

4. 저장/확인 후 이동
- 기본은 직전 화면 복귀
- 홈 이동 금지
- 선생님 모드 첫 화면 강제 이동 금지

==================================================
2. 반드시 우선 점검할 선생님 모드 화면
==================================================

아래 흐름을 전부 우선 점검하라.

- 선생님 홈/아이콘 화면
- 오늘 수업 일정
- 주간 수업
- 월간 수업
- 반 상세
- 학생 상세
- 출결 상세/팝업
- 일지 작성/수정 팝업
- 보강 관리/보강 상세
- 메시지/쪽지/메신저
- 프로필 설정
- 푸시 알림 설정

특히 아래를 집중 점검:
- 스케줄 > 일간/주간/월간 > 상세 진입 > 뒤로가기
- 시간 슬롯 > 반배정/상세 > 뒤로가기
- 학생 상세 > 수정/일지/출결 팝업 > 닫기/뒤로가기
- 메신저/쪽지 상세 > 뒤로가기

==================================================
3. 실제 버그 유형으로 간주할 것
==================================================

아래는 전부 버그로 간주하고 수정하라.

- 상세 화면에서 뒤로가기를 눌렀는데 홈으로 감
- 팝업 닫기 후 선생님 첫 화면으로 감
- 저장 후 선생님 아이콘 첫 화면으로 감
- 직전 목록이 아니라 상위 메인으로 튐
- 모달/바텀시트 상태가 사라짐
- selected tab / selected day / selected class 상태가 사라짐

==================================================
4. 원인 점검 항목
==================================================

아래를 전수 점검하라.

1. 선생님 모드에서 router.replace 남용 여부
2. 뒤로가기 버튼이 실제 back이 아니라 홈 route로 연결되어 있는지
3. 팝업 닫기 시 상위 state 초기화되는지
4. 스케줄 화면의 selectedDay / selectedClass / selectedSlot 상태가 유지되지 않는지
5. 목록 → 상세 진입 시 push 대신 replace를 사용하고 있는지
6. 저장 성공 후 공통 success handler가 홈으로 보내는지

==================================================
5. 수정 방식
==================================================

반드시 아래 원칙으로 수정하라.

- 목록 → 상세는 stack에 쌓이게 한다
- 팝업은 overlay/레이어로 처리하고 닫으면 직전 화면 복귀
- 홈 이동 코드는 홈 버튼/로그아웃/명시적 홈 이동 외에는 제거
- 선생님 모드 안에서의 이동은 한 단계씩만 back
- 스케줄 화면은 selected 상태를 유지한 채 복귀
- 공통 네비게이션 유틸이 있다면 선생님 모드에도 동일 적용

==================================================
6. 선생님 모드 테스트 시나리오
==================================================

반드시 실제 클릭 테스트:

시나리오 A:
선생님 홈
→ 오늘 수업 일정
→ 반 상세
→ 학생 상세
→ 뒤로가기
예상:
→ 반 상세
→ 다시 뒤로가기
→ 오늘 수업 일정
→ 다시 뒤로가기
→ 선생님 홈

시나리오 B:
선생님 홈
→ 스케줄
→ 월간
→ 날짜 선택
→ 시간 슬롯 선택
→ 상세/반배정
→ 뒤로가기
예상:
→ 월간 화면의 직전 상태 유지 복귀
(선택한 날짜/시간 맥락 유지)

시나리오 C:
선생님 홈
→ 학생 상세
→ 일지 작성 팝업
→ 저장
예상:
→ 학생 상세 또는 직전 화면 복귀
→ 홈으로 튀면 안 됨

시나리오 D:
선생님 홈
→ 메신저
→ 대화방
→ 뒤로가기
예상:
→ 메신저 목록
→ 다시 뒤로가기
→ 선생님 홈

시나리오 E:
선생님 홈
→ 프로필 설정
→ 푸시 알림 설정
→ 뒤로가기
예상:
→ 프로필 설정
→ 다시 뒤로가기
→ 선생님 홈

==================================================
7. 회귀 테스트
==================================================

수정 후 아래도 같이 확인:
- 관리자 모드 네비게이션 다시 안 깨졌는지
- 학부모 모드 네비게이션 안 깨졌는지
- 스케줄 일간/주간/월간 이동 안 깨졌는지
- 저장 로직, 출결 로직, 일지 로직은 정상인지

==================================================
8. 완료 보고 형식
==================================================

반드시 아래 형식으로 보고하라.

1. 선생님 모드에서 홈으로 잘못 튀던 화면 목록
2. 직접 원인
3. 수정한 공통 네비게이션 규칙
4. 수정한 파일 목록
5. 오늘 수업 일정 흐름 테스트 결과
6. 스케줄 월간/주간/일간 흐름 테스트 결과
7. 학생 상세/일지/출결 팝업 테스트 결과
8. 메신저/프로필 설정 테스트 결과
9. 아직 남은 예외 화면 여부

중요:
- “선생님 모드도 적용 완료” 한 줄 보고 금지
- 반드시 실제 클릭 흐름 기준으로 적어라
- 각 시나리오별 결과를 적어라

==================================================
9. 최종 목적
==================================================

이번 작업의 목적은
관리자 모드에 맞춘 뒤로가기/팝업 복귀 규칙을
선생님 모드 전체에도 동일하게 적용하는 것이다.

핵심:
- 홈은 홈 버튼으로만 이동
- 나머지는 직전 화면 복귀
- 팝업은 닫으면 직전 화면
- 스케줄/상세/메신저/설정 전부 동일 규칙 적용

이 기준으로 전면 점검하고 수정하라.
