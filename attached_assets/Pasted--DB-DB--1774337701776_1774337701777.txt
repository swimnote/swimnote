[DB 이원화 구조 보완 지시서 - 실제 운영 수준 완성]

현재 DB 이원화 구조는 기본 뼈대는 완료되었으나,
아직 실제 운영 가능한 상태가 아니다.

지금부터 아래 항목을 반드시 보완한다.

==================================================
1. 물리적 DB 분리 강제
==================================================

현재 상태:
POOL_DATABASE_URL 미설정 시 동일 DB 사용

이 구조는 허용하지 않는다.

수정:

- POOL_DATABASE_URL 미설정 시 서버 시작 실패하도록 변경
- fallback 제거

즉:
"pool DB와 super DB는 반드시 다른 DB여야 한다"

==================================================
2. pool DB에도 change log 생성
==================================================

현재:
- 로그는 super DB에만 있음

수정:

- pool DB에도 최소 로그 테이블 생성

예:
pool_change_logs

필수:
- 반배정/출결/일지/보강/회원변경 기록 저장

이유:
- super 로그 실패 대비
- 수영장 내부 감사 가능

==================================================
3. 이벤트 훅 범위 확장
==================================================

현재 이벤트 부족

반드시 추가:

- class_assign (반배정)
- class_unassign (반 제외)
- journal.create / update / delete
- makeup.create / complete
- schedule.change
- subscription.change
- read_only_mode.on/off

모든 핵심 운영 이벤트는 로그로 남겨야 한다

==================================================
4. 로그 복제 안정성 강화
==================================================

현재 retry queue 있음 → 유지

추가:

- retry_count 초과 시 dead-letter queue 생성
- 관리자 UI에서 재전송 버튼 추가
- 실패 로그 별도 표시

==================================================
5. DB 용량 측정 정확화
==================================================

현재 snapshot 방식 → 불완전

수정:

- pg_total_relation_size 기반 실제 용량 계산
- 테이블별 size 정확 계산

추가:

- DB 용량
- 파일 스토리지 용량(R2)
완전히 분리해서 표시

==================================================
6. 외부서비스 연결상태 2계층 UI 구현
==================================================

반드시 구현:

A. 슈퍼관리자용 연결상태
- super DB
- 결제 시스템
- OTP
- 로그 서버

B. 유저서비스용 연결상태
- pool DB
- 휴대폰 인증
- 푸시
- 파일 저장소

중요:
- 현재 UI에 없다면 반드시 새로 추가

==================================================
7. 이벤트 복제 검증 강화
==================================================

테스트 추가:

- 로그 누락 여부 검증
- 동일 이벤트 pool vs super 비교
- 랜덤 이벤트 샘플링 검사

==================================================
8. 현재 구조 진단 보고 추가
==================================================

반드시 보고:

- 현재 물리 분리 여부
- pool DB 실제 연결 상태
- super DB 실제 연결 상태
- fallback 사용 여부
- 로그 누락 여부

==================================================
9. 완료 기준 (업데이트)
==================================================

- 물리 DB 2개 분리 완료
- fallback 완전 제거
- pool / super 로그 모두 존재
- 모든 핵심 이벤트 로그화 완료
- DB / 파일 용량 분리 표시 완료
- 외부서비스 연결상태 UI 2계층 구현 완료
