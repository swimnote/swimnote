지금부터 기능 수정 자체보다 먼저, 내가 지시한 “선생님 모드 출결/보강/반배정 UI 변경”이 왜 적용되지 않는지 근본 원인을 추적하고, 그 원인을 해결한 뒤에만 실제 수정 작업을 다시 반영해라.

중요:
- 이번 작업은 "다시 수정"이 아니라 "왜 수정이 안 먹는지 확인 + 실제 실행되는 파일에 정확히 반영"이다
- 선생님 모드만 수정한다
- 관리자 모드 / 슈퍼관리자 모드는 절대 바꾸지 않는다
- mock 금지
- “수정 완료” 한 줄 금지
- 실제 화면 증거 / 실제 API 증거 / 실제 DB 증거를 제출해라

1. 내가 지시한 원래 작업 내용 (반드시 이 기준으로 다시 적용)
━━━━━━━━━━━━━━━━━━

[적용 범위]
- teacher role UI/로직만 수정
- pool_admin / super_admin UI는 절대 변경 금지

[선생님 모드 출결/보강/반배정 최종 규칙]

1) 수업관리 구조
- 수업관리 → 스케줄러 → 날짜 선택 → 반 선택 → 반 상세
- 모든 실무 작업은 여기서 논스톱으로 이어질 것

2) 날짜 화면
- 수업 추가만 유지
- 보강 버튼 제거
- 메모/음성메모는 아이콘 2개만 표시
- 메모 존재 시 빨간 점 표시

3) 반 리스트
- 시간 중복 표시 제거
- 예: "수요일 13:30"
- 밑에 현재 인원/정원 표시

4) 반 상세 상단
- 버튼 2개만 유지: [반배정] [수업일지]
- 반삭제는 상단 시간 영역에
- 미등록 버튼 제거
- 상단 출결 버튼 제거
- 상단 반이동 버튼 제거

5) 학생 리스트
- 이름 / 주 N회
- 개별 버튼: [출석] [결석] [반이동]
- 우측 화살표: 학생 상세

6) 반배정
- 미등록 버튼 없애고 반배정 안에서 미배정 학생을 바로 보여줄 것
- 반배정 팝업은 키보드 간섭 없이 상단 높은 위치
- KeyboardAvoidingView 적용
- 검색창/결과가 항상 키보드 위에 보이게

7) 출결 규칙 (선생님 모드 전용)
- 수업 시간 지나면 자동으로 출석 처리
- 이름 중앙에 줄 표시
- 수업 전/중/후 언제든 결석으로 변경 가능
- 결석 시:
  attendance = absent
  makeup_session 생성 (pending)
  이름 앞 빨간 점 표시
- 결석 → 출석 되돌리기 시:
  attendance = present
  makeup_session 삭제
  빨간 점 제거
  줄은 유지
- 결석 학생 상단 정렬
- 지각 없음
- 사유 없음
- 저장 버튼 없음, 즉시 저장

8) 보강 규칙
- 결석은 보강 대기 생성의 시작점
- 결석 후 즉시 다른 반으로 이동/배정 가능
- 보강 대기 리스트에는 결석 날짜 표시
- 결석 알림은 보내지 않음
- 보강 알림만:
  a. 보강 배정 즉시
  b. 보강 일정 변경 즉시
  c. 보강 당일

2. 먼저 할 일: 왜 변경이 안 먹는지 강제 추적
━━━━━━━━━━━━━━━━━━

지금까지 수정했는데 웹/Expo Go에 반영이 안 되었으므로,
아래를 먼저 수행해라.

[2-1. 화면 디버그 문구 삽입]
선생님 출결 화면 최상단에 아래 문구를 아주 크게 넣어라.

DEBUG_TEACHER_ATTENDANCE_0325_V1

조건:
- 빨간색
- 크게
- absolute top 또는 아주 눈에 띄는 위치
- 웹과 Expo Go에서 둘 다 확인 가능해야 함

추가로, 선생님 스케줄 화면에도 아래 문구를 넣어라.
DEBUG_TEACHER_SCHEDULE_0325_V1

반드시 제출:
- 삽입한 파일 경로
- 웹에서 보이는지
- Expo Go에서 보이는지
- 화면 캡처 또는 명확한 증거

[2-2. 실제 실행 파일 추적]
반드시 찾아라:
- 현재 선생님 출결 화면의 실제 route 경로
- 실제 렌더링 파일
- import chain
- 같은 이름의 attendance/schedule 파일이 몇 개인지

특히 아래 확인:
- artifacts/swim-app/app/(teacher)/...
- artifacts/swim-app/app/...
- 공용 route / wrapper / layout
- teacher route인지 admin route인지

반드시 제출:
- 실제 route
- 실제 렌더 파일 경로
- import chain
- 중복 파일 목록

[2-3. 현재 로그인 role 표시]
선생님 출결 화면 상단에 아래 debug를 같이 출력해라.
ROLE: teacher 또는 ROLE: pool_admin

이유:
- 관리자 계정으로 선생님 전환했는데 실제로 teacher role 화면이 아니라 admin 화면을 보고 있을 가능성을 확인하기 위함

[2-4. 캐시/번들 초기화]
반드시 아래를 실행하고 결과를 제출해라.
1. 서버 완전 종료
2. API 서버 완전 재기동
3. 앱 번들 캐시 초기화 모드로 시작
4. 웹 preview 새로 열기
5. 시크릿창에서도 확인
6. Expo Go 완전 종료 후 새 QR로 재접속

반드시 제출:
- 실제 재기동 절차
- 캐시 초기화 여부
- 웹/Expo Go 각각 debug 문구 표시 여부

[2-5. API 응답 디버그]
선생님 출결/스케줄 관련 API 응답에 임시 필드 추가:
"debug_version": "TEACHER_UI_0325_V1"

반드시 제출:
- 호출 API 경로
- 응답 JSON 일부
- debug_version 보이는지 여부

3. 변경 안 먹는 원인 특정 후에만 실제 수정 반영

위 2단계를 통해 실제 실행 파일이 특정되면,
그 파일에만 내가 지시한 선생님 모드 구조를 다시 정확히 반영해라.

중요:
- 잘못된 파일 여러 군데 동시에 수정 금지
- 실제 렌더 파일 하나를 기준으로 적용
- 관리자 모드에 영향 가면 실패
- 선생님 모드에만 적용할 것

4. 반영 후 실제 검증
━━━━━━━━━━━━━━━━━━

아래를 반드시 실제 앱 기준으로 확인해라.

[4-1. 선생님 출결]
- 수업 시간 지나면 자동으로 줄 표시
- 결석 클릭 시 빨간 점 표시
- 결석 클릭 시 makeup_session 생성
- 다시 출석으로 변경 시 makeup_session 삭제
- 결석 학생 상단 정렬

[4-2. 반배정 팝업]
- 키보드가 떠도 검색창이 보임
- 검색 결과가 키보드에 안 가림
- 학생 선택 가능
- 저장 가능
- 배정 후 미배정 목록에서 제거

[4-3. 수업관리 구조]
- 날짜 화면에 보강 버튼이 제거되었는지
- 반 상세 상단에 [반배정] [수업일지]만 있는지
- 학생 행에 [출석] [결석] [반이동]만 있는지
- 미등록 버튼이 사라졌는지

[4-4. 관리자 모드 영향 없음]
- 관리자 모드 화면이 바뀌지 않았는지
- 관리자 출결/보강 UI가 그대로인지
