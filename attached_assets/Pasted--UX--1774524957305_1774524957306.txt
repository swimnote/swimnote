[스윔노트 — 반 색상 변경 UX 수정 및 동기화 작업 지시]

현재 반 색상 기능에서 사용자 경험이 잘못되어 있다.

문제:

* 색상을 눌러도 카드 색이 즉시 변하지 않음
* 저장 버튼이 없고 저장 동작도 불명확함
* 주간 스케줄과 월간 스케줄 색상 반영이 서로 동기화되지 않으면 안 됨

이번 작업 목표는
“색상 클릭 즉시 눈에 보이게 바뀌고, 팝업을 닫으면 자동 저장되며, 주간/월간 어디서 바꿔도 동일 반 색상이 함께 반영되는 구조”로 수정하는 것이다.

━━━━━━━━━━━━━━━━━━

1. 최종 UX 기준
   ━━━━━━━━━━━━━━━━━━

반 색상 변경은 아래 흐름으로 동작해야 한다.

1. 색상 팝업 열기
2. 색상 하나 누르기
3. 누르는 즉시 현재 화면 카드 색이 바로 바뀜
4. 저장 버튼 없음
5. 팝업 바깥 빈 공간 클릭 시 팝업 닫힘
6. 팝업이 닫히는 시점에 현재 선택된 색으로 자동 저장
7. 저장 완료 후 주간/월간 모든 동일 반 카드에 같은 색 반영

즉,
“선택 즉시 프리뷰 + 팝업 닫을 때 자동 저장” 구조로 수정한다.

━━━━━━━━━━━━━━━━━━
2. 반드시 수정할 문제
━━━━━━━━━━━━━━━━━━

현재 가장 먼저 해결할 것:

* 색상 클릭 시 카드 색 즉시 변경 안 되는 문제
* 색상 선택 후 저장이 안 되는 문제

우선순위:

1. 즉시 프리뷰가 보이게 수정
2. 자동 저장 동작 수정
3. 주간/월간 동기화 연결

━━━━━━━━━━━━━━━━━━
3. 저장 방식 규칙
━━━━━━━━━━━━━━━━━━

저장 버튼 만들지 말 것.

저장 방식은 아래처럼 한다.

* 색상 클릭 = 로컬 상태 즉시 변경
* 팝업 내부에서는 선택된 색이 즉시 반영된 상태로 보임
* 팝업 바깥 클릭 또는 닫기 동작 발생 시
  마지막 선택값을 PATCH 저장
* 저장 성공 후 팝업 종료
* 저장 실패 시 원래 색으로 롤백 또는 에러 표시

중요:

* 색상 누를 때마다 즉시 PATCH 여러 번 날리지 말 것
* 실제 저장은 팝업 닫힐 때 1회만 실행 권장
* 단, 화면 프리뷰는 즉시 보여야 함

즉:
UI 프리뷰는 즉시
DB 저장은 닫을 때 1회

━━━━━━━━━━━━━━━━━━
4. 주간 / 월간 색상 동기화 규칙
━━━━━━━━━━━━━━━━━━

같은 class_group id를 쓰는 반이면
주간과 월간은 항상 같은 color 값을 보여야 한다.

규칙:

* 주간에서 색 변경 → 월간에서도 같은 반 색 즉시 반영
* 월간에서 색 변경 → 주간에서도 같은 반 색 즉시 반영
* 둘 다 같은 DB color 값을 읽어야 함
* 화면별 별도 색상 상태를 따로 두지 말 것

필수 확인:

* 주간 스케줄 데이터 source
* 월간 스케줄 데이터 source
* 둘 다 같은 class_groups.color 기준으로 렌더링하는지 확인

━━━━━━━━━━━━━━━━━━
5. 구현 방식 권장
━━━━━━━━━━━━━━━━━━

반 상세 팝업 또는 색상 피커 팝업 안에서

상태 예시:

* originalColor
* draftColor

동작 예시:

* 팝업 열릴 때 draftColor = originalColor
* 색상 클릭 시 draftColor 즉시 변경
* 카드 미리보기/현재 카드 UI는 draftColor 반영
* 바깥 클릭으로 팝업 닫힐 때
  draftColor !== originalColor 이면 PATCH 저장
* 저장 성공 시 전역 상태 또는 상위 state 갱신
* 저장 실패 시 originalColor로 복구

중요:

* 색상 클릭만 하고 닫지 않았을 때는 프리뷰만
* 닫는 순간 최종 선택값 저장

━━━━━━━━━━━━━━━━━━
6. 필수 수정 대상
━━━━━━━━━━━━━━━━━━

아래 파일 전부 확인:

* AdminClassDetailSheet.tsx
* ClassDetailSheet.tsx
* PastelColorPicker.tsx
* WeeklySchedule.tsx
* WeeklyTimetable.tsx
* classes.tsx
* 월간 스케줄 관련 파일 전체

확인할 것:

* 색상 선택 이벤트가 상위 카드 상태까지 연결되는지
* 팝업 외부 클릭 감지(onClose / onDismiss)에서 저장 호출되는지
* 저장 후 주간/월간 공통 state 또는 refetch로 동기화되는지

━━━━━━━━━━━━━━━━━━
7. 금지 사항
━━━━━━━━━━━━━━━━━━

* 저장 버튼 새로 추가 금지
* 색상 누를 때마다 PATCH 연속 호출 금지
* 주간/월간 각각 다른 색상 상태 유지 금지
* 한 화면만 바뀌고 다른 화면은 재접속해야 바뀌는 구조 금지

실패 기준:

* 색상 클릭해도 눈에 바로 안 변함
* 팝업 닫아도 저장 안 됨
* 주간에서 바꿨는데 월간은 그대로임
* 월간에서 바꿨는데 주간은 그대로임

━━━━━━━━━━━━━━━━━━
8. 반드시 테스트할 항목
━━━━━━━━━━━━━━━━━━

[즉시 프리뷰]

* 색상 클릭 즉시 카드 색 변경 확인

[자동 저장]

* 팝업 바깥 클릭 시 닫히면서 저장 확인
* PATCH 1회만 호출되는지 확인

[동기화]

* 주간에서 변경 후 월간 반영 확인
* 월간에서 변경 후 주간 반영 확인

[실패 처리]

* 저장 실패 시 롤백 또는 에러 표시 확인

[재조회]

* 팝업 닫고 다시 들어가도 같은 색 유지 확인
* 재로그인 후에도 유지 확인

━━━━━━━━━━━━━━━━━━
