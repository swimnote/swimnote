[스윔노트 — 공지사항 전체 발송 우선 구조로 수정 지시문]

현재 공지사항 관리 화면은 "수영장 선택" 기반으로만 만들어져 있다.
하지만 지금 우선 필요한 공지는 각 수영장별 공지가 아니라,
플랫폼 전체 사용자에게 보내는 "전체 공지"다.

이번 작업은 공지사항 관리 구조를
“전체 공지 우선 + 필요 시 수영장별 공지” 구조로 수정하는 작업이다.

━━━━━━━━━━━━━━━━━━
1. 핵심 원칙
━━━━━━━━━━━━━━━━━━

공지 우선순위는 아래로 한다.

1. 전체 공지
- 모든 수영장
- 모든 관리자
- 모든 선생님
- 모든 학부모

2. 수영장별 공지
- 특정 수영장만

즉,
공지 작성 화면의 기본값은 “전체 공지”여야 한다.
수영장 선택은 기본이 아니라 선택 옵션이다.

━━━━━━━━━━━━━━━━━━
2. 공지 대상 구조 수정
━━━━━━━━━━━━━━━━━━

현재:
- 수영장 선택만 존재

수정 후:
- 공지 대상 선택 먼저 배치

대상 선택 옵션:
- 전체 공지
- 수영장별 공지

기본 선택:
- 전체 공지

동작:
- 전체 공지 선택 시 수영장 선택 UI 숨김
- 수영장별 공지 선택 시에만 수영장 선택 UI 표시

즉,
전체 공지일 때는 수영장 선택을 요구하면 안 된다

━━━━━━━━━━━━━━━━━━
3. UI 수정
━━━━━━━━━━━━━━━━━━

공지사항 관리 화면 상단 수정:

현재:
- 수영장 선택
- 공지 등록

수정 후:
- 공지 대상 선택 (전체 / 수영장별)
- 수영장 선택 (수영장별일 때만 표시)
- 공지 등록 버튼

예시:

[전체 공지] [수영장별 공지]

전체 공지 선택 시:
- 수영장 선택 숨김

수영장별 공지 선택 시:
- 수영장 선택 노출

중요:
- 지금처럼 "수영장을 선택해주세요"가 기본 화면이면 안 됨
- 기본 화면에서 바로 전체 공지 등록 가능해야 함

━━━━━━━━━━━━━━━━━━
4. notices 구조 확장
━━━━━━━━━━━━━━━━━━

notices 저장 시 공지 범위를 구분할 수 있게 한다.

필드 예시 추가:
- audience_scope

값 예:
- global
- pool

규칙:
- global = 전체 공지
- pool = 수영장별 공지

pool 공지일 때만:
- swimming_pool_id 필요

global 공지일 때는:
- swimming_pool_id nullable 허용 또는 별도 처리

━━━━━━━━━━━━━━━━━━
5. 푸시 발송 규칙 수정
━━━━━━━━━━━━━━━━━━

[전체 공지]
대상:
- 모든 수영장 관리자
- 모든 선생님
- 모든 학부모

즉:
- 플랫폼 전체 사용자 대상
- 활성 계정만
- 유효한 push token만

[수영장별 공지]
대상:
- 선택한 수영장 관리자
- 선택한 수영장 선생님
- 선택한 수영장 학부모

중요:
- 전체 공지는 수영장 분리 없이 전체 발송
- 수영장별 공지는 기존 규칙 유지

━━━━━━━━━━━━━━━━━━
6. 푸시 메시지 형식
━━━━━━━━━━━━━━━━━━

[전체 공지]
제목:
[스윔노트] 공지사항

내용:
공지 제목

예:
[스윔노트] 시스템 점검 안내

[수영장별 공지]
제목:
[수영장명] 공지사항

내용:
공지 제목

━━━━━━━━━━━━━━━━━━
7. 공지 목록 표시 수정
━━━━━━━━━━━━━━━━━━

공지 목록에서 아래 구분이 바로 보이게 한다.

표시 항목 추가:
- 전체 공지
- 수영장별 공지

예:
- 전체
- 토이키즈
- 송파수영아카데미

즉,
목록만 봐도 전체 공지인지 특정 수영장 공지인지 구분돼야 한다.

━━━━━━━━━━━━━━━━━━
8. 공지 노출 규칙
━━━━━━━━━━━━━━━━━━

로그인 후 공지 팝업 노출 시:

- 전체 공지면 모든 사용자에게 노출
- 수영장별 공지면 해당 수영장 사용자에게만 노출

공지함에서도 동일:
- 전체 공지는 모두 보임
- 수영장별 공지는 자기 수영장 것만 보임

━━━━━━━━━━━━━━━━━━
9. 수정 대상
━━━━━━━━━━━━━━━━━━

반드시 수정:

- (super)/pool-notices.tsx
- notices API
- notices schema
- 공지 저장 로직
- 공지 목록 로직
- 푸시 발송 분기
- 공지 노출 로직
- 공지함 로딩 로직

중요:
- “수영장 선택 필수” 구조 제거
- “전체 공지 기본” 구조로 변경

━━━━━━━━━━━━━━━━━━
10. 테스트
━━━━━━━━━━━━━━━━━━

반드시 실제 확인:

1. 기본 화면에서 전체 공지 선택 상태인지
2. 전체 공지일 때 수영장 선택 UI 숨김인지
3. 전체 공지 등록 가능 여부
4. 전체 공지 저장 시 notices DB 저장 확인
5. 전체 공지 푸시가 관리자/선생님/학부모 전체에 가는지
6. 수영장별 공지 선택 시 기존처럼 수영장 선택 가능한지
7. 수영장별 공지는 해당 수영장에만 가는지
8. 공지 목록에서 전체/수영장별 구분 보이는지
9. 로그인 후 전체 공지가 모든 역할에 노출되는지

━━━━━━━━━━━━━━━━━━
11. 보고 형식
━━━━━━━━━━━━━━━━━━

반드시 포함:

- 수정 파일 경로
- UI 변경 전/후
- audience_scope 저장 결과
- 전체 공지 등록 테스트 결과
- 전체 푸시 발송 대상 결과
- 수영장별 공지 유지 결과
- 공지 목록 구분 표시 결과

“전체 공지 추가함”만 쓰지 말고
실제 저장/발송/노출 결과 같이 제출할 것
