현재 관리자 > 선생님 관리 화면 구조를 아래 기준으로 수정해줘.

중요:
기존 화면을 완전히 버리는 게 아니라,
상단의 "오늘 확인할 것" 영역은 현황판처럼 그대로 유지하고,
그 아래 메인 탐색 구조를 일정표 기반으로 바꿔줘.

핵심 목적:
관리자가 선생님을 먼저 보는 구조가 아니라,
수업 시간표를 보고
→ 시간 선택
→ 해당 시간에 반 개설한 선생님 선택
→ 해당 선생님이 개설한 반 선택
→ 마지막에 그 반의 현황판을 보는 구조로 변경

======================================
[1. 화면 전체 구조]
======================================

현재 선생님 관리 화면 상단의 "오늘 확인할 것" 요약 영역은 유지해줘.

예:
- 오늘 수업 수
- 결석 확인 필요
- 메시지 미발송
같은 현황판 성격은 그대로 유지

단, 그 아래 메인 영역은 일정표 기반 구조로 교체

상단 탭 유지:
- 오늘
- 주간

기본 진입은 "오늘"

======================================
[2. 오늘 탭 구조]
======================================

오늘 탭은 아래 순서로 동작해야 함.

1. 오늘 확인할 것 요약 영역 유지
2. 오늘 수업 일정표를 시간별로 표시
3. 시간 클릭 시 해당 시간에 반 개설한 선생님 목록 표시
4. 선생님 클릭 시 해당 선생님이 그 시간에 개설한 반 목록 표시
5. 반 클릭 시 해당 반 현황판 표시

--------------------------------------
[2-1. 오늘 수업 일정표]
--------------------------------------

오늘 날짜 기준 수업 시간표를 시간별 리스트 또는 타임라인 형태로 보여줘.

예:
- 15:00
- 16:00
- 17:00
- 18:00

각 시간 항목은 클릭 가능해야 함

표시 정보는 단순하게:
- 시간
- 그 시간대 개설 반 수 또는 수업 있음 표시

예:
15:00 / 2개 반
16:00 / 1개 반

--------------------------------------
[2-2. 시간 클릭 후 선생님 목록]
--------------------------------------

특정 시간 클릭 시,
그 시간에 반을 개설한 선생님 목록 표시

정렬:
- 선생님 이름 가나다순

표시 정보:
- 선생님 이름
- 해당 시간 개설 반 수

선생님 항목 클릭 가능

--------------------------------------
[2-3. 선생님 클릭 후 반 목록]
--------------------------------------

선생님을 클릭하면
그 선생님이 해당 시간에 개설한 반 목록 표시

표시 정보:
- 반 이름
- 반 인원
- 진행 상태 간단 표시 가능

예:
- 월 15:00 초급반 / 6명
- 월 15:00 중급반 / 4명

반 항목 클릭 가능

--------------------------------------
[2-4. 반 클릭 후 반 현황판]
--------------------------------------

반 클릭 시 마지막 단계로 해당 반 현황판 표시

이 반 현황판에는 아래 항목이 보여야 함

상단 요약:
- 반 이름
- 선생님 이름
- 수업 시간
- 총 학생 수
- 출석 수
- 결석 수
- 일지 작성 여부

그 아래 액션 또는 탭:
- 학생
- 출결
- 일지
- 결석

======================================
[3. 반 현황판 상세 내용]
======================================

--------------------------------------
[3-1. 학생]
--------------------------------------

학생 누르면 학생 명단 표시

표시 정보:
- 학생 이름
- 상태

상태 예:
- 정상
- 보강

중요:
보강 학생은 이름 옆에 보강 표시 배지 또는 아이콘 붙이기

--------------------------------------
[3-2. 출결]
--------------------------------------

출결 누르면 출석 명단 표시

표시 정보:
- 학생 이름
- 출석 상태

보강 학생이면 여기에도 이름 옆에 보강 표시 유지

--------------------------------------
[3-3. 일지]
--------------------------------------

일지 누르면 해당 반의 일지 작성 내용 표시

표시 정보:
- 반 전체 일지 내용
- 작성 여부
- 작성 시간
- 작성 선생님

간단 버전부터 먼저 구현해도 됨

--------------------------------------
[3-4. 결석]
--------------------------------------

결석 누르면 결석자 표시

표시 정보:
- 학생 이름
- 결석 표시

간단 버전부터 먼저 구현

======================================
[4. 주간 탭 구조]
======================================

주간 탭도 오늘 탭과 같은 탐색 흐름을 사용하되,
기준만 주간 스케줄로 바꿔줘.

흐름:
1. 주간 스케줄 표시
2. 요일 + 시간 클릭
3. 해당 시간대 선생님 목록 표시
4. 선생님 클릭
5. 해당 선생님 반 목록 표시
6. 반 클릭
7. 오늘 탭에서 사용하는 것과 동일한 반 현황판 표시

--------------------------------------
[4-1. 주간 스케줄]
--------------------------------------

주간 스케줄은 요일 + 시간 구조로 보여줘.

예:
월 / 15:00
월 / 16:00
화 / 15:00
수 / 18:00

표현 방식은 현재 프로젝트 UI에 맞춰
리스트형 또는 간단한 주간 보드형으로 구현 가능

중요:
요일과 시간이 함께 구분되어야 함
각 항목 클릭 가능해야 함

--------------------------------------
[4-2. 요일+시간 클릭 후]
--------------------------------------

해당 요일/시간에 반을 개설한 선생님 목록 표시
정렬은 가나다순

선생님 클릭 후
해당 시간의 반 목록 표시

반 클릭 후
오늘 탭과 동일한 반 현황판 표시

======================================
[5. 공통 원칙]
======================================

오늘 탭과 주간 탭의 마지막 반 상세 화면은
같은 컴포넌트를 재사용해줘.

즉 아래는 공통 반 현황판 컴포넌트로 구현:
- 학생
- 출결
- 일지
- 결석

======================================
[6. 데이터 연결]
======================================

이 구조는 선생님 모드 데이터와 연결되어야 함

반드시 같은 데이터 소스 사용:
- 시간표
- 반 개설 정보
- 선생님 정보
- 학생 명단
- 출결 정보
- 일지 작성 여부
- 결석 정보
- 보강 학생 여부

관리자용 별도 mock 구조 만들지 말고
현재 프로젝트 데이터와 연결 가능한 형태로 구현

======================================
[7. 구현 우선순위]
======================================

이번 작업은 아래 우선순위로 진행해줘.

1. 오늘 확인할 것 유지
2. 오늘 탭 시간표 기반 탐색 구조 구현
3. 시간 클릭 → 선생님 목록 → 반 목록 → 반 현황판 연결
4. 주간 탭도 같은 방식으로 연결
5. 반 현황판의 학생/출결/일지/결석 기본 표시 구현

중요:
학생 / 출결 / 일지 / 결석은 복잡한 편집기능보다
일단 "보이는 구조"부터 정확히 구현

======================================
[8. 확인 작업]
======================================

작업 후 아래를 직접 확인해줘.

1. 오늘 확인할 것 영역이 유지되는지
2. 오늘 탭에서 시간표가 시간별로 보이는지
3. 시간 클릭 시 선생님 목록이 뜨는지
4. 선생님 목록이 가나다순인지
5. 선생님 클릭 시 반 목록이 뜨는지
6. 반 클릭 시 반 현황판이 뜨는지
7. 학생 누르면 학생명단이 보이는지
8. 보강 학생은 학생/출결에서 보강 표시가 보이는지
9. 일지 누르면 일지 작성 내용 또는 작성 여부가 보이는지
10. 결석 누르면 결석자가 보이는지
11. 주간 탭도 같은 흐름으로 동작하는지
12. 오늘 탭과 주간 탭의 마지막 반 현황판이 같은 구조인지

======================================
[9. 완료 후 보고]
======================================

작업 완료 후 아래 형식으로 보고해줘.

- 오늘 탭 변경 내용
- 주간 탭 변경 내용
- 추가한 컴포넌트
- 재사용한 공통 반 현황판 컴포넌트
- 연결한 데이터 소스
- 직접 확인한 테스트 결과
- 아직 mock인 부분과 실제 연결된 부분 구분
