[STAGE 7 — 실제 운영 시나리오 통합 시뮬레이션 + 치명 버그 탐지 지시서]

지금부터 스윔노트 전체 시스템을 실제 운영 환경처럼 끝까지 시뮬레이션한다.

이 단계는 단순 테스트가 아니다.
“회원가입 → 운영 → 결제 → 실패 → 복구 → 정산 → 삭제”까지
전체 라이프사이클을 실제 데이터로 검증하는 단계다.

중요:
- 한 단계라도 막히면 즉시 중단하고 수정
- mock/가짜 데이터 발견 시 즉시 제거
- 시뮬레이션 중 발생하는 모든 오류는 치명 버그로 간주
- Expo Go 기준에서 멈추면 무조건 수정
- 완료 보고는 실제 결과 기반으로만 작성

==================================================
시나리오 A — 정상 가입 → 운영 → 결제
==================================================

1. 수영장 생성
2. 관리자 생성
3. 선생님 생성
4. 회원 등록 (직접등록 + 요청등록)
5. 승인 처리
6. 반배정
7. 출결 처리
8. 일지 작성
9. 사진 업로드
10. 무료 → 유료 플랜 전환

확인:
- subscription 생성 여부
- revenue_logs 생성 여부
- super/subscriptions 반영 여부
- super/more 위험도 변화 여부

==================================================
시나리오 B — 결제 실패 → 즉시 제한
==================================================

1. 결제 실패 상태 강제
2. 즉시 읽기모드 전환 확인

확인:
- 관리자: 결제 안내 메시지 표시
- 선생님/학부모: 기능 제한 메시지 표시
- 업로드/등록/수정 차단 여부

==================================================
시나리오 C — 저장공간 95% 초과
==================================================

1. 저장공간 임계값 강제 설정 (95% 이상)
2. 업로드 시도

확인:
- 경고 표시
- 업로드 차단
- 관리자 안내 문구
- 슈퍼관리자 위험 감지 반영

==================================================
시나리오 D — 100% 초과
==================================================

확인:
- 완전 차단
- 팝업 노출
- 업그레이드 버튼 정상 동작

==================================================
시나리오 E — 결제 복구
==================================================

1. 읽기모드 상태에서 재결제
2. 정상 상태 복귀

확인:
- 기능 정상 복구
- subscription 상태 변경
- revenue_logs 추가 기록

==================================================
시나리오 F — 자동 삭제 흐름
==================================================

1. subscription_end_at 설정
2. 7일 경고 상태 확인
3. 14일 삭제 조건 강제

확인:
- 삭제 대상 표시
- 슈퍼관리자 risk 반영
- cancel-deletion API 정상 작동

==================================================
공통 필수 체크
==================================================

- 모든 카운트 즉시 반영 여부
- 관리자/선생님/학부모 권한 분리
- 미배정 vs 미연결 분리
- 플랜/정산 데이터 일관성
- Expo Go에서 멈춤 없음
- 팝업/뒤로가기 정상

==================================================
치명 버그 기준
==================================================

다음 발생 시 즉시 수정:

- 앱 멈춤 / 터치 불가
- 데이터 저장 안 됨
- 정산 누락
- 플랜 불일치
- 권한 오류
- 카운트 틀림
- mock 데이터 노출

==================================================
완료 기준
==================================================

- 모든 시나리오 A~F 성공
- Expo Go 멈춤 없음
- super 화면과 실제 데이터 100% 일치
- 플랜/정산/구독 흐름 완전 연결

==================================================
보고 형식
==================================================

1. 각 시나리오 실행 결과
2. 발견 버그 목록
3. 수정 내용
4. 재테스트 결과
5. 남은 위험 요소
