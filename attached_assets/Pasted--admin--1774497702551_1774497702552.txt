[스윔노트 — 슈퍼관리자 라우팅 누수 전수점검 + 홈버튼 강제 수정 지시문]

현재 심각한 문제:
슈퍼관리자 백업/복구 페이지에서 홈 버튼을 누르면 수영장관리자(admin) 페이지로 이동한다.
심지어 admin 계정으로 로그인하지 않았는데 admin 화면으로 진입하는 것처럼 보인다.

이 문제는 단순 홈 버튼 오작동이 아니라
역할(role), 세션(session), activeRole, 저장된 last_used_role, 공통 헤더/탭 라우팅이 꼬여 있는 권한 누수 문제다.

이번 작업은 슈퍼관리자 영역 전체의 버튼/탭/이동 경로를 전수점검하고,
슈퍼관리자는 어떤 상황에서도 오직 슈퍼관리자 홈으로만 이동하게 강제하는 작업이다.

━━━━━━━━━━━━━━━━━━

1. 핵심 원칙
   ━━━━━━━━━━━━━━━━━━

* 슈퍼관리자는 절대 admin 화면으로 자동 이동하면 안 된다
* admin 로그인 안 된 상태에서 admin 화면처럼 보이면 안 된다
* 슈퍼 영역의 홈 버튼은 무조건 /(super)/dashboard 로만 가야 한다
* super_admin / platform_admin / super_manager 는 activeRole, last_used_role, 이전 세션값보다 우선한다

핵심:
👉 super 계열은 저장된 role보다 실제 계정 role이 항상 우선
👉 super 계열은 admin/teacher/parent 라우트로 새지 않게 막아야 함

━━━━━━━━━━━━━━━━━━
2. 즉시 수정 대상
━━━━━━━━━━━━━━━━━━

다음 경로/버튼/컴포넌트 전부 전수점검:

* 슈퍼관리자 백업/복구 페이지 홈 버튼
* 슈퍼관리자 공통 헤더의 홈 버튼
* 슈퍼관리자 탭 버튼
* 슈퍼관리자 사이드/퀵메뉴
* 뒤로가기 버튼
* 공통 Header/Nav 컴포넌트
* role 기반 redirect 훅
* _layout.tsx
* session/role context
* localStorage / AsyncStorage에 저장된 activeRole, last_used_role 사용 지점

중요:

* "백업 페이지 홈 버튼만 수정"으로 끝내지 말고
* 슈퍼 영역 전체 경로를 전수 확인할 것

━━━━━━━━━━━━━━━━━━
3. 강제 라우팅 규칙
━━━━━━━━━━━━━━━━━━

super 계열 역할:

* super_admin
* platform_admin
* super_manager

위 3개는 아래 규칙 강제 적용:

1. 홈 버튼
   → 무조건 /(super)/dashboard

2. 슈퍼 화면 내부 이동
   → 슈퍼 라우트 안에서만 이동

3. 저장된 activeRole / last_used_role
   → 슈퍼 계열에서는 무시하거나 즉시 초기화

4. 공통 헤더/공통 탭
   → super 여부를 먼저 검사 후 슈퍼 전용 목적지로 라우팅

절대 금지:

* super가 admin dashboard로 이동
* super가 teacher 홈으로 이동
* admin 로그인 안 했는데 admin UI 렌더링

━━━━━━━━━━━━━━━━━━
4. 세션/권한 누수 차단
━━━━━━━━━━━━━━━━━━

[문제 의심 지점]

* 이전 로그인 사용자 role 값이 남아 있음
* activeRole이 재사용됨
* 공통 홈 버튼이 현재 화면이 아니라 last_used_role 기준으로 이동함
* 공통 레이아웃이 role 우선순위를 잘못 처리함

[수정]

* 로그인한 사용자 id가 바뀌면 role 관련 저장값 초기화
* super 계열 로그인 시 admin/teacher 관련 저장값 폐기
* 공통 홈 버튼 계산 전에 실제 adminUser.role 먼저 확인
* super 계열이면 다른 분기 모두 건너뛰고 즉시 슈퍼 홈으로 이동

━━━━━━━━━━━━━━━━━━
5. 버튼/탭 전수점검 방식
━━━━━━━━━━━━━━━━━━

슈퍼관리자 화면에서 아래를 실제로 다 눌러서 확인:

* 홈
* 뒤로가기
* 상단 아이콘
* 하단 탭
* 카드형 진입 버튼
* 설정 버튼
* 백업/복구 버튼
* 운영자관리 버튼
* 로그인 후 첫 진입 경로
* 새로고침/앱 재실행 후 첫 진입 경로

검증 기준:

* 어떤 버튼을 눌러도 admin 화면으로 새지 않음
* 어떤 버튼을 눌러도 teacher/parent 화면으로 새지 않음
* 무조건 super 흐름 안에서만 움직임

━━━━━━━━━━━━━━━━━━
6. admin 화면 무단 진입 차단
━━━━━━━━━━━━━━━━━━

admin 계정으로 로그인하지 않았는데 admin 화면이 보이는 것 자체가 문제다.

반드시 점검:

* admin 레이아웃 진입 시 실제 role = pool_admin 인지 확인
* super 계정이 admin route URL로 들어가도 차단 또는 super 홈으로 재이동
* 화면만 보이고 API는 막히는 반쪽 차단 구조 금지
* 라우트 진입 자체를 막을 것

즉:

* 화면 렌더링 전 role 검사
* 권한 없으면 즉시 redirect
* admin route는 pool_admin만

━━━━━━━━━━━━━━━━━━
7. 수정 범위 확장
━━━━━━━━━━━━━━━━━━

아래도 같이 확인:

* 운영자관리 화면 진입 버튼
* 저장소 상태/인프라 상태 페이지 홈 버튼
* 슈퍼관리자 내 상세페이지들의 홈 버튼
* 공통 Modal / Sheet 닫기 후 이동 로직
* deeplink / router.replace / router.push 사용 지점

목표:
👉 슈퍼관리자 영역에서 admin으로 새는 경로 0개

8. 실제 테스트
━━━━━━━━━━━━━━━━━━

반드시 실제로 아래 테스트 수행:

1. 슈퍼관리자 로그인

2. 슈퍼 대시보드 진입 확인

3. 백업/복구 페이지 이동

4. 홈 버튼 클릭
   → 반드시 /(super)/dashboard 로 이동

5. 운영자관리 페이지 이동

6. 홈 버튼 클릭
   → 반드시 /(super)/dashboard 로 이동

7. 슈퍼 페이지 각종 상단/하단 버튼 전부 클릭
   → admin 이동 없음 확인

8. localStorage/AsyncStorage에 last_used_role = pool_admin 강제로 넣고 재실행
   → 슈퍼 로그인 시 admin으로 안 감

9. super 계정으로 admin route 직접 접근 시도
   → 차단 또는 슈퍼 홈 리다이렉트 확인

10. admin 로그인 없이 admin UI가 보이지 않는지 확인
