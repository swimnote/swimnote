[스윔노트 — 공지 노출 규칙 최종 수정 지시문 (자동로그인 + 앱 재실행 기준)]

현재 공지 시스템은 global / pool 이중 구조까지 만들어져 있다.
이번 작업은 공지 노출 규칙을 실제 사용 방식에 맞게 다시 정리하는 작업이다.

핵심 변경:
- 앱은 로그아웃하지 않으면 자동로그인 유지
- 공지는 “로그인할 때”가 아니라 “앱을 완전히 다시 켰을 때” 노출
- 공지 노출 순서는 전체 공지 먼저, 그다음 수영장 공지
- 다시보지않기는 다음 앱 실행부터 자동팝업 미노출
- 수영장 관리자 공지도 같은 규칙으로 적용

━━━━━━━━━━━━━━━━━━
1. 자동로그인 기준
━━━━━━━━━━━━━━━━━━

앱은 로그아웃하지 않으면 계속 로그인 상태를 유지한다.

즉:
- 매번 로그인 화면을 거치지 않아도 됨
- 공지 노출 기준을 “로그인 성공 시점”으로 잡지 말 것
- 공지 노출 기준은 “앱 재실행 후 첫 진입”으로 바꿀 것

━━━━━━━━━━━━━━━━━━
2. 공지 노출 트리거 기준
━━━━━━━━━━━━━━━━━━

공지 자동 팝업은 아래 경우에만 실행:

- 앱이 완전히 종료된 뒤 다시 실행되었을 때
- 즉, 콜드 런치(cold launch) 후 첫 진입 시

중요:
- 단순 백그라운드 복귀 시에는 매번 다시 띄우지 말 것
- “몇 시간 지나면 종료로 간주” 같은 고정 시간 규칙 사용 금지
- OS 종료/메모리 정리는 기기와 상황에 따라 달라지므로 시간 기준으로 판단하지 말 것
- 앱 프로세스가 새로 시작된 경우에만 “앱을 새로 켰다”로 판단

정리:
- 백그라운드 → 다시 복귀: 공지 자동팝업 X
- 앱 완전 재실행(프로세스 재시작): 공지 자동팝업 O

━━━━━━━━━━━━━━━━━━
3. 공지 노출 순서
━━━━━━━━━━━━━━━━━━

앱 재실행 후 공지 자동 노출 순서는 아래로 고정:

1. 슈퍼관리자 전체 공지 (global)
2. 수영장 관리자 공지 (pool)

즉:
- 전체 공지가 있으면 먼저 보여줌
- 그 다음 수영장 공지를 보여줌
- 순서 반대로 하지 말 것

추가 규칙:
- 숨김된 공지는 건너뜀
- deleted / hidden 상태 공지는 제외
- status = published만 자동 노출

━━━━━━━━━━━━━━━━━━
4. 전체 공지(global) 조건
━━━━━━━━━━━━━━━━━━

자동 노출 대상:

- audience_scope = global
- status = published
- 다시보지않기 되지 않은 공지
- 사용자 역할:
  - 관리자
  - 선생님
  - 학부모

이 전체 공지는 로그인 역할과 상관없이 위 3역할 모두에게 보인다.

━━━━━━━━━━━━━━━━━━
5. 수영장 공지(pool) 조건
━━━━━━━━━━━━━━━━━━

전체 공지 다음에 수영장 공지를 노출한다.

자동 노출 대상:

- audience_scope = pool
- status = published
- 로그인 사용자와 같은 수영장(swimming_pool_id) 공지
- 다시보지않기 되지 않은 공지

수영장 관리자 공지도 같은 규칙 적용:
- 수영장 관리자가 작성한 공지라도
- 동일 수영장 사용자에게만
- 전체 공지 다음 순서로 자동 노출

━━━━━━━━━━━━━━━━━━
6. 다시보지않기 규칙
━━━━━━━━━━━━━━━━━━

현재 정책 유지:
- 다시보지않기 = 이 기기 기준 숨김

동작:
- 다시보지않기 누른 공지는 다음 앱 실행부터 자동팝업에 나오지 않음
- 전체 공지든 수영장 공지든 동일하게 적용
- 공지함에서는 다시 확인 가능

즉:
- 자동팝업만 숨김
- 공지 데이터 자체 삭제 아님

━━━━━━━━━━━━━━━━━━
7. 닫기 규칙
━━━━━━━━━━━━━━━━━━

닫기 버튼은 임시 닫기다.

동작:
- 지금 팝업만 닫음
- 다음 앱 완전 재실행 시 다시 노출 가능
- 다시보지않기와 절대 같은 동작이면 안 됨

즉:
- 닫기 = 이번만 닫기
- 다시보지않기 = 다음 앱 실행부터 이 공지 자동팝업 숨김

━━━━━━━━━━━━━━━━━━
8. 공지 표시 방식
━━━━━━━━━━━━━━━━━━

공지 상세 페이지는 만들지 않는다.

규칙:
- 팝업 안에서 제목/작성일/내용 표시
- 내용이 길면 팝업 내부 스크롤
- 공지 여러 개면 순서대로 노출 또는 다음 공지로 이어지게 처리
- 우선순위는 항상 global → pool

━━━━━━━━━━━━━━━━━━
9. 공지함 규칙
━━━━━━━━━━━━━━━━━━

공지함은 계속 유지한다.

위치:
- 관리자 / 선생님 / 학부모 설정(톱니바퀴/더보기) 안

공지함에서는:
- 전체 공지 확인 가능
- 수영장 공지 확인 가능
- 다시보지않기 한 공지도 다시 볼 수 있음

즉:
- 자동팝업 숨김과 공지함 열람은 별개

━━━━━━━━━━━━━━━━━━
10. 구현 방식 권장
━━━━━━━━━━━━━━━━━━

앱 실행 시 아래 흐름으로 처리:

1. 앱 콜드 런치 감지
2. 공지 조회
3. global 공지 필터
4. dismissed 제외
5. 최신순 또는 우선순위순 1개씩 노출
6. global 종료 후 pool 공지 조회/노출
7. 모든 공지 소진 후 종료

주의:
- 백그라운드 복귀마다 다시 돌리지 말 것
- 세션 살아있는 상태의 단순 resume에서는 중복 팝업 금지

━━━━━━━━━━━━━━━━━━
11. 테스트
━━━━━━━━━━━━━━━━━━

반드시 실제로 확인:

1. 자동로그인 상태 유지 확인
2. 앱 백그라운드 → 복귀 시 공지 자동팝업이 다시 뜨지 않는지
3. 앱 완전 종료 후 재실행 시 공지 자동팝업 뜨는지
4. 전체 공지가 수영장 공지보다 먼저 뜨는지
5. 다시보지않기 누른 전체 공지가 다음 앱 실행 시 안 뜨는지
6. 다시보지않기 누른 수영장 공지도 다음 앱 실행 시 안 뜨는지
7. 닫기만 누른 공지는 다음 앱 재실행 시 다시 뜨는지
8. 공지함에서는 숨긴 공지도 다시 볼 수 있는지
9. 다른 수영장 공지는 노출되지 않는지

━━━━━━━━━━━━━━━━━━
12. 보고 형식
━━━━━━━━━━━━━━━━━━

반드시 포함:

- 수정 파일 경로
- 앱 재실행 기준 구현 방식
- 백그라운드 복귀 vs 콜드 런치 구분 방식
- global → pool 노출 순서 테스트 결과
- 닫기 / 다시보지않기 차이 테스트 결과
- 관리자/선생님/학부모별 노출 결과

“공지 뜸” 같은 말만 쓰지 말고
어떤 상황에서 뜨고 어떤 상황에서 안 뜨는지 실제 조건별로 제출할 것
