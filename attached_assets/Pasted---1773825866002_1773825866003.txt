지금부터 승인 관리 / 선생님 관리 구조를 아래 방식으로 수정하라.

이번 작업은 조직도 운영 현실에 맞게
“선생님 기본 가입 + 관리자에 의한 부관리자 지정/해제 + 퇴사/인수인계 처리”
구조로 바꾸는 작업이다.

중요:
- 관리자 가입 방식은 그대로 유지
- 관리자 제외 나머지 직원 계정은 기본적으로 선생님 가입만 한다
- 부관리자 권한은 회원가입 단계에서 받지 않는다
- 부관리자는 관리자 화면에서 기존 승인된 선생님 중 지정/해제하는 방식으로 바꾼다
- 승인된 사람도 다시 눌러서 상세 팝업으로 진입 가능해야 한다
- 기능 연결을 실제로 전부 확인하면서 작업할 것

==================================================
[1. 최종 운영 원칙]

1. 관리자 가입
- 기존처럼 별도 관리자 가입 유지

2. 직원 가입
- 관리자 제외한 직원은 모두 “선생님 회원가입”으로만 가입
- 가입 승인 시 기본 역할은 teacher

3. 부관리자 지정
- 부관리자는 회원가입 단계에서 받지 않음
- 관리자 모드에서 이미 승인된 선생님을 선택해서
  “부관리자 지정 / 해제” 하는 방식으로 변경

4. 퇴사/입사 대응
- 기존 선생님이 퇴사할 수 있으므로
  승인된 선생님 상세 팝업 안에서
  “승인 해제”, “부관리자 지정/해제”, “수업 인수” 기능을 처리할 수 있어야 함

==================================================
[2. 승인 관리 구조 변경]

현재 승인 관리에서
승인 버튼만 있는 구조를 바꿔라.

대상:
- 학부모 승인 탭
- 선생님 승인 탭

변경 규칙:
각 카드 우측 액션 영역에
기존 승인 버튼 옆에 “보기” 버튼을 추가하라.

즉:
- 승인
- 보기

2개 버튼 구조로 만든다.

중요:
이미 승인된 상태의 사람도
보기 버튼을 통해 상세 팝업 진입 가능해야 한다.

==================================================
[3. 선생님 승인 규칙 변경]

현재 선생님 승인 시
부관리자 체크 방식이 들어가 있다면 제거하라.

새 규칙:
- 선생님 가입 신청 → 관리자 승인
- 승인 시 기본적으로 teacher 역할만 부여
- 가입 단계에서 부관리자 직접 지정 금지

즉:
선생님 승인 버튼은 그냥
“선생님 승인”만 처리하면 된다.

==================================================
[4. 보기 버튼 동작 - 공통]

보기 버튼을 누르면
해당 회원/선생님 상세 팝업이 떠야 한다.

팝업 공통 규칙:
- 기존 팝업 규칙 유지
- 닫기 버튼 있음
- 배경 터치/드래그 닫기 기존대로
- 내용이 많으면 내부 스크롤 가능
- 현재 계정 정보 / 신청 상태 / 소속 정보 등 표시

==================================================
[5. 학부모 보기 팝업 기능]

학부모 카드에서 보기 버튼 클릭 시 팝업에 아래 기능 제공:

표시 정보:
- 이름
- 연락처
- 아이디
- 자녀 정보
- 가입일/승인일
- 상태

액션:
- 승인 해제

승인 해제 의미:
- 학부모 승인 상태를 해제
- 필요 시 다시 승인 대기 또는 비활성 상태로 전환
- 로그인/접근 정책은 현재 앱 구조에 맞춰 일관되게 처리

중요:
이미 승인된 학부모도 나중에 다시 관리할 수 있어야 하므로
반드시 승인 후에도 보기 버튼으로 진입 가능해야 한다.

==================================================
[6. 선생님 보기 팝업 기능]

선생님 카드에서 보기 버튼 클릭 시 팝업에 아래 기능 제공:

표시 정보:
- 이름
- 연락처
- 아이디
- 현재 역할
  (예: 선생님 / 선생님+부관리자)
- 담당 반 수
- 담당 회원 수
- 상태
- 승인일

액션 버튼:
- 승인 해제
- 부관리자 지정 또는 부관리자 해제
- 수업 인수

버튼 노출 규칙:
1. 일반 선생님이면
   - 승인 해제
   - 부관리자 지정
   - 수업 인수

2. 이미 부관리자이면
   - 승인 해제
   - 부관리자 해제
   - 수업 인수

즉:
부관리자 버튼은 현재 상태에 따라
지정 / 해제 텍스트가 바뀌어야 한다.

==================================================
[7. 부관리자 지정/해제 방식]

선생님 보기 팝업에서
관리자가 버튼을 누르면 즉시 역할이 변경되게 하라.

규칙:
- teacher만 가진 계정 → 부관리자 지정 클릭 → roles = ["teacher","sub_admin"]
- teacher + sub_admin 계정 → 부관리자 해제 클릭 → roles = ["teacher"]

중요:
- 가짜 UI 금지
- 실제 roles 값이 저장되어야 함
- 더보기의 역할 전환 버튼 조건에도 즉시 반영되어야 함
- 다음 로그인/다음 화면 진입에도 유지되어야 함

==================================================
[8. 승인 해제 의미 정리]

선생님 또는 학부모 승인 해제는
중간 퇴사/퇴원/비활성 운영을 위한 기능이다.

처리 원칙:
- 데이터는 삭제하지 말 것
- 상태만 승인 해제/비활성으로 바꿀 것
- 이후 재승인 가능하도록 남겨둘 것

가능한 상태 예:
- approved
- pending
- inactive
- revoked

현재 구조에 맞게 쓰되
“승인 해제 후 기록은 남는다”가 핵심이다.

==================================================
[9. 수업 인수 기능 추가]

이 기능이 중요하다.
중간에 선생님이 퇴사하거나 반을 넘길 수 있으므로
기존 담당 회원/반을 다른 선생님에게 한 번에 넘길 수 있어야 한다.

선생님 보기 팝업에
“수업 인수” 버튼을 추가하라.

버튼 클릭 시:
- 인수 대상 선생님 선택 팝업 또는 단계형 팝업 오픈

기능 목표:
현재 선택한 선생님의 담당 수업/회원/반을
다른 선생님에게 한 번에 넘길 수 있게 한다.

==================================================
[10. 수업 인수 상세 규칙]

예:
기존 선생님 A 퇴사
→ 보기 팝업 > 수업 인수
→ 새 선생님 B 선택
→ 확인
→ A가 맡던 반/회원/담당 정보를 B로 일괄 이전

이전 대상:
- 담당 반
- 담당 회원
- 관련 수업 배정
- 필요한 선생님 참조 필드

중요:
- 회원 개별 수동 수정으로 끝내지 말 것
- 일괄 이전 기능이어야 함

가능하면 인수 옵션:
- 전체 반 인수
- 전체 회원 인수
현재 구조상 둘이 사실상 같이 묶이면 같이 처리해도 됨

최소 구현:
A 선생님이 담당하던 회원/반을
선택한 B 선생님에게 일괄 이전

==================================================
[11. 수업 인수 후 처리]

수업 인수 완료 후 아래 확인 필요:

- 기존 선생님 카드의 담당 반 수/회원 수가 감소 또는 0 반영
- 새 선생님 카드의 담당 반 수/회원 수 증가 반영
- 회원 상세/반 상세에서 담당 선생님이 새 선생님으로 변경
- 출결/일지/수업관리에서 새 선생님 기준으로 이어짐
- 데이터 꼬임 없도록 실제 연결 전부 확인

==================================================
[12. UI 변경 요약]

승인 목록 카드 액션:
- 승인
- 보기

보기 팝업:
학부모:
- 승인 해제

선생님:
- 승인 해제
- 부관리자 지정/해제
- 수업 인수

==================================================
[13. 기존 방식 제거]

반드시 아래를 제거 또는 변경하라.

1. 선생님 가입 승인 시 부관리자 체크해서 같이 승인하는 방식 제거
2. 승인된 사람을 다시 관리할 수 없는 구조 제거
3. 승인 후엔 카드 눌러도 아무것도 못하는 구조 제거

새 방식:
- 승인 후에도 보기 버튼으로 계속 관리 가능
- 부관리자 지정/해제는 승인 이후 관리 단계에서 처리

==================================================
[14. 기능 연결 체크 필수]

이번 작업은 반드시 실제 연결을 체크하면서 작업하라.

체크 항목:
1. 선생님 가입 신청 후 승인 버튼으로 기본 teacher 승인 되는지
2. 승인 완료 후 보기 버튼이 계속 보이는지
3. 보기 버튼 클릭 시 상세 팝업이 뜨는지
4. 선생님 팝업에서 부관리자 지정이 실제 roles에 반영되는지
5. 부관리자 해제가 실제 roles에서 제거되는지
6. 역할 전환 버튼 노출 조건이 즉시 바뀌는지
7. 학부모 팝업에서 승인 해제가 되는지
8. 선생님 팝업에서 승인 해제가 되는지
9. 수업 인수 시 담당 반/회원이 새 선생님에게 실제 이전되는지
10. 이전 후 수업관리/회원관리/반관리 화면에 새 선생님으로 반영되는지

==================================================
[15. 테스트 시나리오]

반드시 실제 테스트하라.

시나리오 A.
새 선생님 가입 신청
→ 관리자 승인
→ 기본 teacher만 부여되는지 확인

시나리오 B.
승인된 선생님 보기 팝업 진입
→ 부관리자 지정
→ roles 반영 + 역할 전환 버튼 노출 확인

시나리오 C.
부관리자 지정된 선생님 보기 팝업 진입
→ 부관리자 해제
→ roles 제거 + 역할 전환 버튼 숨김 또는 변경 확인

시나리오 D.
승인된 선생님 보기 팝업 진입
→ 수업 인수
→ 다른 선생님에게 회원/반 일괄 이전 확인

시나리오 E.
승인된 학부모 보기 팝업 진입
→ 승인 해제 확인

==================================================

수정 전에 먼저 아래를 파악하라.

2. 선생님 승인 처리 파일
3. 학부모 승인 처리 파일
4. roles 저장 파일
5. 선생님/회원/반 연결 데이터 파일
6. 담당 선생님 배정 관련 파일

이걸 먼저 확인한 뒤 작업 시작

[18. 절대 금지]

- 관리자 가입 구조 건드리지 말 것
- 승인된 사람 데이터를 삭제하지 말 것
- 부관리자 권한을 가입 단계에서 다시 받게 만들지 말 것
- 보기 버튼만 만들고 실제 수정/반영 안 되는 가짜 기능 금지
- 수업 인수를 UI만 만들고 실제 이전 안 되는 상태 금지

이번 작업은
“선생님은 기본 가입/승인, 부관리자는 관리자에 의해 사후 지정/해제, 승인된 사람도 보기 팝업으로 계속 관리, 퇴사 시 수업 인수 가능”
구조를 실제 동작하게 만드는 작업이다.
