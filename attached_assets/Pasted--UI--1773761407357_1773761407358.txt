지금부터 학부모 회원가입 화면과 가입 저장 구조를 수정하라.

중요:
- 기존 테스트 로그인 화면 유지
- 기존 각 모드 진입 유지
- 다른 기능/탭/UI 구조는 건드리지 말 것
- 이번 작업은 학부모 회원가입에 로그인용 계정 정보(아이디/비밀번호) 추가만 수행
- 기존 학부모 가입 방식(수영장 찾기 / 코드 입력)은 유지
- 수정 후 반드시 실제 테스트까지 수행하고 결과를 상세히 보고할 것
- 기능은 최대한 건드리지 않은 상태에서 수정하고, 연결은 반드시 확인하면서 진행할 것

==================================================
[1. 현재 문제]

현재 학부모 회원가입 화면에
- 이름
- 전화번호
- 자녀정보
만 있고,

로그인에 필요한
- 아이디
- 비밀번호
가 없다.

이 상태로는 계정 생성이 아니라 단순 정보 등록만 되는 구조이므로,
학부모도 실제 로그인 가능한 계정 구조로 수정해야 한다.

==================================================
[2. 이번 작업 목표]

1. 학부모 회원가입 화면에 아이디 입력칸 추가
2. 학부모 회원가입 화면에 비밀번호 입력칸 추가
3. 필요 시 비밀번호 확인칸도 추가
4. 학부모 가입 저장 시 아이디/비밀번호가 실제 계정 정보로 저장되게 수정
5. 이후 로그인 화면에서 학부모도 아이디/비밀번호로 로그인 가능하게 유지
6. 기존 학부모 가입 방식(수영장 찾기 / 코드 입력)은 그대로 유지

==================================================
[3. 학부모 회원가입 화면 수정]

적용 대상:
- 수영장 찾기 방식 학부모 회원가입
- 코드 입력 방식에서 직접 입력이 필요한 경우의 학부모 회원가입/확인 화면

--------------------------------------
[3-1. 수영장 찾기 방식]

현재 항목:
- 이름
- 전화번호
- 자녀정보

수정 후 항목:
- 이름
- 전화번호
- 사용할 아이디
- 비밀번호
- 비밀번호 확인
- 자녀정보

배치 권장:
이름
전화번호
사용할 아이디
비밀번호
비밀번호 확인
자녀정보

주의:
- 아이디는 실제 로그인 식별값
- 이메일이 아니라 아이디 기준
- 비밀번호는 실제 로그인에 사용
- 비밀번호 확인까지 넣어 입력 실수 방지

--------------------------------------
[3-2. 코드 입력 방식]

코드 입력 후 자동으로 학부모/자녀 정보가 채워지는 경우에도
로그인용 계정 정보는 반드시 받아야 한다.

즉 코드 방식에서도 아래 중 하나로 처리:
A. 코드 입력 후 확인 화면에서 아이디/비밀번호 입력 추가
또는
B. 코드 입력 후 계정 생성 단계 별도 추가

필수 입력:
- 사용할 아이디
- 비밀번호
- 비밀번호 확인

자동으로 채워지는 정보:
- 이름
- 자녀정보
- 수영장 정보
- 기타 사전 등록 정보

즉,
사전 등록 정보는 자동 채우되,
로그인 계정 정보는 학부모가 직접 입력하게 할 것

==================================================
[4. 데이터 구조 수정]

학부모 가입 데이터에 아래 필드를 반드시 포함하라.

필수:
- parentName
- parentPhone
- username 또는 loginId
- password
- childName 또는 childInfo
- poolId 또는 연결 수영장 정보

주의:
- 아이디/비밀번호 없이 가입 완료되지 않게 할 것
- 로그인은 이 username/loginId + password 기준으로 처리

==================================================
[5. 로그인 구조 반영]

학부모 로그인도 아래 기준으로 유지/반영할 것

- 로그인 화면에서 아이디 입력
- 비밀번호 입력
- 학부모 계정이면 학부모 홈 진입

즉,
학부모도 선생님/관리자와 동일하게
“아이디 + 비밀번호” 로그인 구조여야 한다

==================================================
[6. 기존 흐름 유지 조건]

아래는 그대로 유지:
- 학부모 가입 방식 2가지
  1) 수영장 찾기
  2) 수영장 코드 입력
- 수영장 찾기 시 검색 로직
- 코드 입력 시 24시간 유효시간
- 기존 테스트 로그인 화면
- 기존 각 모드 진입

이번 작업은
학부모 회원가입에 로그인용 계정 필드 추가
까지만 수행한다.

==================================================
[7. 입력 검증]

아래 검증 추가:
1. 아이디 미입력 시 가입 불가
2. 비밀번호 미입력 시 가입 불가
3. 비밀번호 확인 불일치 시 가입 불가
4. 중복 아이디면 가입 불가
5. 정상 입력 시에만 가입 요청 또는 즉시 승인 진행

==================================================
[8. 완료 후 상태 처리]

--------------------------------------
[수영장 찾기 방식]
- 아이디/비밀번호 포함해 가입 요청 저장
- 승인 로직은 현재 구조 유지
- 로그인 시 승인 여부에 맞게 처리

--------------------------------------
[코드 입력 방식]
- 아이디/비밀번호 포함해 계정 생성
- 즉시 승인 구조면 바로 로그인 가능 상태로 저장

==================================================
[9. 연결 안정성 체크 - 반드시 수행]

[공통 체크]
1. 기존 학부모 가입 흐름을 깨뜨리지 않았는지 확인
2. 아이디/비밀번호 입력 후 실제 계정이 생성되는지 확인
3. 로그인 화면에서 그 아이디/비밀번호로 로그인되는지 확인
4. 코드 가입 방식에서도 아이디/비밀번호가 정상 반영되는지 확인
5. 기존 테스트 로그인과 다른 모드 진입이 깨지지 않았는지 확인

기능은 최대한 건드리지 않은 상태에서 수정하고,
서로 연결된 부분은 반드시 확인하면서 진행할 것

==================================================
[10. 테스트 지시 - 반드시 실제 수행]

수정 후 아래 테스트를 전부 수행하고 결과를 상세히 보고하라.

테스트 1.
수영장 찾기 방식 학부모 회원가입 화면에
아이디 / 비밀번호 / 비밀번호 확인 입력칸이 추가되었는지 확인

테스트 2.
코드 입력 방식에서도
아이디 / 비밀번호 / 비밀번호 확인을 입력받는지 확인

테스트 3.
아이디 없이 가입 시 가입이 막히는지 확인

테스트 4.
비밀번호 확인 불일치 시 가입이 막히는지 확인

테스트 5.
정상 입력 시 학부모 계정이 생성되는지 확인

테스트 6.
생성된 학부모 계정으로 로그인 화면에서
아이디/비밀번호 로그인 가능한지 확인

테스트 7.
기존 테스트 로그인 화면과 각 모드 진입이 그대로 유지되는지 확인

테스트 8.
선생님/관리자 회원가입 흐름이 이번 수정으로 깨지지 않았는지 확인

==================================================
[11. 보고 형식]

작업 후 아래 순서로 보고하라.

1. 학부모 회원가입 화면에 추가한 항목
2. 수영장 찾기 방식 반영 내용
3. 코드 입력 방식 반영 내용
4. 학부모 계정 저장 구조 수정 내용
5. 로그인 반영 방식
6. 테스트 1~8 결과
7. 아직 남은 임시 처리 부분
8. 다른 기능/화면에 영향 준 부분이 있으면 숨기지 말고 적기

==================================================
[12. 절대 금지]

- 테스트 로그인 삭제 금지
- 기존 모드 진입 삭제 금지
- 학부모 가입 방식 2가지 구조 변경 금지
- 선생님/관리자 가입 흐름 깨뜨리기 금지
- 다른 주요 탭 화면 수정 금지

이번 작업은
“학부모 회원가입에 아이디/비밀번호 추가 및 실제 로그인 계정화”
까지만 수행한다.
다른 기능은 건드리지 말 것.
