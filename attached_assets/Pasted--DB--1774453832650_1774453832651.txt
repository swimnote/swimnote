[스윔노트 아키텍처 통합 재정비 — 최종 지시문]

이번 작업은 앱 전체를 안정화하기 위한 핵심 작업이다.  
목표는 DB 구조 단순화 + 백업 체계 구축 + 기존 리팩토링 유지이다.

절대 규칙:
- mock 데이터 사용 금지
- “완료” 한 줄 보고 금지
- 실제 DB 연결 / 실제 저장 / 실제 UI 표시까지 검증
- 기존 기능 절대 깨지면 안 됨

━━━━━━━━━━━━━━━━━━
0. 핵심 목표
━━━━━━━━━━━━━━━━━━

운영 구조를 아래로 고정한다:

1) superAdminDb (운영 원본)
- 모든 읽기/쓰기
- 학생 / 반 / 출결 / 보강 / 일지 / 공지 / 정산 / 결제 전부 저장

2) pool DB (백업)
- 수영장 단위 백업
- 운영 사용 금지

3) super 보호백업 DB (백업)
- 전체 백업
- 운영 사용 금지

👉 앱은 오직 superAdminDb만 본다

━━━━━━━━━━━━━━━━━━
1. 매우 중요 — 기존 리팩토링 구조 유지
━━━━━━━━━━━━━━━━━━

다음 구조는 절대 건드리지 않는다:

- AuthContext → SessionContext / RoleContext 분리 구조 유지
- admin / teacher / parent / super 역할 분리 유지
- my-schedule / today-schedule / diary 분해 구조 유지
- store 배럴 구조 유지
- 컴포넌트 분해 구조 유지

금지:
- 대형 파일로 다시 합치기
- 기존 구조 무시하고 직접 화면 수정
- 역할 경계 깨기

👉 이번 작업은 “구조 재작성”이 아니라  
👉 “DB 참조 재정렬 + 안정화 작업”이다

━━━━━━━━━━━━━━━━━━
2. 작업 순서 (반드시 순서대로)
━━━━━━━━━━━━━━━━━━

1. 선백업
2. DB 단일화 (super 기준)
3. 백업 구조 구축
4. 백업 상태 기록 시스템 구축
5. 슈퍼관리자 상태 UI 구축
6. 수동 백업 기능 구축
7. 실패/장애 검증

━━━━━━━━━━━━━━━━━━
3. 1단계 — 선백업
━━━━━━━━━━━━━━━━━━

수정 전에 반드시 실행:

- superAdminDb 전체 백업
- pool DB 현재 상태 백업
- backup_id / 경로 / 용량 기록

👉 이 백업은 롤백 기준이다

━━━━━━━━━━━━━━━━━━
4. 2단계 — DB 단일화 (핵심)
━━━━━━━━━━━━━━━━━━

현재 코드에서 poolDb를 사용하는 모든 경로를 찾아라.

작업:
- 모든 API / service / repository를 superAdminDb로 전환
- pool DB 접근 제거
- super 보호백업 DB 접근 제거

조건:
- 기존 API 응답 동일해야 함
- 기존 UI 동작 유지
- 크로스 DB 참조 0개로 만들 것

제출:
- 변경된 API 목록
- 제거된 poolDb 접근 목록

━━━━━━━━━━━━━━━━━━
5. 3단계 — 백업 구조 구축
━━━━━━━━━━━━━━━━━━

자동 백업 흐름:

superAdminDb → pool DB  
superAdminDb → super 보호백업 DB

백업 메타 테이블 생성:

backup_logs
- target (pool / super_protect)
- status (success / failed / running)
- started_at
- finished_at
- last_success_at
- error_message
- size / row_count

👉 핵심:
“백업 됐다”가 아니라  
“마지막 성공 시각 + 실패 여부”를 기록해야 한다

━━━━━━━━━━━━━━━━━━
6. 4단계 — 슈퍼관리자 상태 카드 UI
━━━━━━━━━━━━━━━━━━

슈퍼관리자 화면에 카드 4개 생성:

1. 운영 DB (super)
- 연결 상태
- 응답속도
- 오류 여부

2. pool 백업
- 마지막 백업 시각
- 상태 (정상/실패)

3. 보호백업
- 마지막 백업 시각
- 상태 (정상/실패)

4. 전체 요약
- 최근 백업 성공 여부
- 실패 횟수

표시 규칙:
- 정상 = 초록
- 지연 = 노랑
- 실패 = 빨강

👉 fake 상태 금지 (실제 데이터 기반 표시)

━━━━━━━━━━━━━━━━━━
7. 5단계 — 수동 백업 기능
━━━━━━━━━━━━━━━━━━

버튼 위치:
슈퍼관리자 상단

클릭 시:
- superAdminDb 기준 백업 실행
- pool DB + 보호백업 동시 갱신
- 결과 표시 + backup_logs 기록

옵션:
- 전체 백업
- pool만 백업

━━━━━━━━━━━━━━━━━━
8. 6단계 — 장애 감지 시스템
━━━━━━━━━━━━━━━━━━

다음 상황 감지:

- 백업 실패
- DB 연결 실패
- 응답 지연

카드에 표시:
- “백업 실패”
- “백업 중단”
- “주의 상태”

👉 Supabase 알림 없이도 앱에서 확인 가능해야 함

━━━━━━━━━━━━━━━━━━
9. 검증 (필수)
━━━━━━━━━━━━━━━━━━

7. 기존 기능 유지
