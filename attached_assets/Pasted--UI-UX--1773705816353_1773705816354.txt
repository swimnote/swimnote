학부모 모드 UI/UX를 아래 요구사항 기준으로 전면 정리하고, 실제 버튼/스크롤/팝업/탭 이동까지 모두 작동하게 구현해라.
목표는 "학부모가 처음 써도 헷갈리지 않게, 한 손 조작이 편하고, 팝업 닫기/스크롤/다운로드/탭 이동이 자연스럽게 이어지는 인터페이스"다.

[공통 원칙]
- 모바일 우선 UI로 구현
- iPhone/Android 모두 자연스럽게 보이게 여백, 터치영역, 스크롤 반응 최적화
- 모든 버튼은 최소 터치 영역 44px 이상
- 스크롤 끊김 없게 하고, 긴 리스트는 성능 고려해 렌더링
- 팝업 열릴 때 배경 스크롤 잠금
- 팝업 닫기는 X 버튼 + 바깥 영역 터치 + 아래로 스와이프 닫기(가능하면) 지원
- 뒤로가기 동작은 팝업 닫기 우선, 그 다음 이전 화면 이동
- 로딩/빈 상태/오류 상태를 각각 구현
- 저장/다운로드/좋아요/감사합니다 클릭 시 즉각적인 시각 반응 제공
- 학부모 화면에서는 모든 게시물이 "우리 아이 기준 개인화 화면"처럼 보여야 함
- 다른 학부모 존재를 암시하는 숫자/카운트/공개 반응 수는 노출하지 말 것

[학부모 첫 화면 구조]
상단:
1. 자녀 선택 영역
- 현재 선택된 아이 이름, 현재 레벨, 반 정보 표시
- 자녀가 여러 명이면 드롭다운 또는 선택 팝업으로 전환 가능
- 자녀 전환 시 화면 전체 데이터가 해당 아이 기준으로 즉시 갱신

2. 바로가기 메뉴
- 수업피드백
- 앨범
- 출결
- 레벨
- 공지
- 아이콘 + 텍스트 버튼으로 구현
- 2열 또는 3열 그리드 배치
- 눌렀을 때 해당 탭/페이지로 부드럽게 이동

중단:
3. 최근 업데이트 피드
- 시간순 피드
- 최신순 정렬
- 카드형 UI
- 카드 상단에 담당선생님 이름과 날짜만 표시
- 시간은 표시하지 않음
- 게시물 유형은 학부모가 들어온 메뉴 기준으로 자연스럽게 표시
  예: 수업피드백 메뉴에서는 "수업일지", 앨범 메뉴에서는 "앨범"

하단:
4. 광고 영역
- 화면 하단에 고정 또는 자연스럽게 이어지는 광고/배너 영역
- 첫 화면 절반 정도는 광고 공간으로 확보
- 광고 영역은 메인 피드와 시각적으로 분리

[수업피드백 화면]
- 하루 1개의 수업일지만 보이게 구성
- 선생님은 반 기준으로 작성하지만 학부모는 자기 아이에게 쓴 글처럼 보이게 표시
- 글만 있을 수도 있고, 글 + 사진이 있을 수도 있음
- 사진은 썸네일 표시 후 누르면 전체 보기
- 게시물 하단 반응 버튼:
  - 좋아요
  - 감사합니다
  - 쪽지달기
- 숫자 표시 없음
- 다른 사람 반응 여부 노출 금지

[게시물 쪽지 기능]
- 댓글 기능은 만들지 말 것
- "쪽지달기" 누르면 모달형 대화창 열기
- 화면 가운데 뜨는 모달 방식
- 배경 어둡게
- 팝업 상단에 닫기 버튼 명확히 표시
- 모달 안에서 대화 스레드 형태로 보이게 구현
- 학부모는 쪽지 시작 가능, 추가 질문 계속 가능
- 선생님은 답변만 가능, 여러 번 답변 가능
- 관리자는 열람만 가능
- 쪽지는 게시물 기반 비공개 대화
- 해당 게시물과 연결되어야 함
- 쪽지가 있는 게시물은 작은 쪽지 아이콘만 표시하고 숫자는 표시하지 않음
- 메시지는 텍스트만 가능
- 사진/영상/파일 첨부 없음

[쪽지 삭제 구조]
- 학부모는 본인 메시지만 삭제 가능
- 선생님은 본인 메시지만 삭제 가능
- 삭제 시 바로 영구 삭제하지 말고 삭제메세지함으로 이동
- 삭제메세지함은 자동 삭제 없음
- 복구 가능
- 영구삭제 가능
- 삭제되어도 상대방에게 알림 보내지 말 것

[선생님 모드 쪽지함 연결]
- 선생님 모드 > 학부모관리 > 안읽은 쪽지함 / 읽은 쪽지함 구현
- 쪽지 클릭 시 해당 게시물 페이지로 이동하면서 쪽지 모달이 열린 상태로 진입
- 관리자는 로그기록 > 선생님들 쪽지함에서 통합 열람
- 관리자는 열람만 가능, 작성/삭제 불가
- 관리자 쪽지 목록은 최신 업데이트 순으로 갱신

[앨범 구조]
1. 전체앨범
- 수영장 모든 선생님이 업로드 가능
- 반 구분 없음
- 모든 학부모가 열람 가능
- 최신순 정렬
- 다운로드 가능
- 사진/영상 모두 동일 정책
- 학부모는 보기 전용 느낌으로 보되 다운로드는 가능

2. 개인앨범
- 담당 선생님만 자기 담당 학생에게 업로드 가능
- 다른 반 학생은 업로드 대상 선택 목록에 노출 금지
- 해당 학생 학부모만 열람 가능
- 최신순 정렬
- 다운로드 가능
- 학생별 앨범에 계속 누적
- 날짜별 폴더 없음

3. 앨범 보기 UX
- 기본은 그리드 보기
- 사진 하나 누르면 슬라이드 보기
- 영상은 썸네일만 보이고, 클릭 시 다운로드
- 스트리밍 재생은 하지 말 것
- 영상은 1개씩만 업로드 가능
- 사진은 한 번에 최대 10장 업로드 가능
- 선택 버튼을 통해 여러 사진 선택 후 묶음 다운로드 가능
- zip 생성 말고 원본 파일 개별 다운로드 처리

[다운로드 UX]
- 다운로드 버튼은 학부모가 쉽게 찾을 수 있게 명확히 배치
- 사진은 클릭 후 바로 저장 가능하게 처리
- 여러 장 선택 다운로드 가능
- 다운로드 중에는 로딩 표시
- 다운로드 완료 시 간단한 토스트 메시지 제공

[출결 화면]
- 출석 / 결석 / 보강 예정이 명확히 구분되게
- 월별 리스트 또는 캘린더형 보기 중 모바일에 자연스러운 쪽 선택
- 너무 복잡한 캘린더보다 스크롤 리스트형 우선
- 최근 기록이 상단에 보이게

[레벨 화면]
- 현재 레벨
- 레벨 테스트 기록
- 승급 날짜
- 최근 변화 위주로 위에 배치
- 성장 흐름이 보이게 타임라인 느낌 허용

[공지 화면]
- 수영장 공지 / 반 공지 구분
- 중요 공지는 상단 고정 가능
- 읽기 쉬운 카드형 구성

[UX 디테일]
- 모든 리스트는 당겨서 새로고침 지원
- 빈 상태일 때 친절한 안내 문구 표시
- 스크롤 최상단 이동 버튼 필요하면 추가
- 이미지 보기 화면은 확대/축소 가능하면 좋음
- 모달 안 긴 대화도 스크롤 가능해야 함
- 키보드 올라올 때 입력창 안 가려지게 처리
- iOS safe area, Android 하단 제스처 영역 침범하지 않게 처리
- 긴 텍스트는 2~3줄까지만 미리 보여주고 더보기 가능하게
- 빠른 반응 속도가 중요하므로 낙관적 업데이트 허용
- 상태관리와 데이터 구조는 학부모 기준 selected_student_id 중심으로 설계

[데이터 연결 원칙]
- 학부모 화면의 모든 데이터는 selected_student_id 기준으로 필터링
- 선생님은 반 기준 입력해도 학부모에겐 개인화된 결과처럼 노출
- 다른 학부모 데이터, 반응, 수치, 인원수, 공개 카운트 절대 노출 금지

[테스트 필수]
- 자녀 2명 이상 계정 테스트
- 팝업 열고 닫기 반복 테스트
- 긴 피드 스크롤 테스트
- 사진 다중 선택 다운로드 테스트
- 영상 썸네일 -> 다운로드 테스트
- 쪽지 생성/추가질문/답변/삭제/복구 테스트
- 선생님 안읽은/읽은 쪽지함 이동 테스트
- 관리자 통합 열람 테스트
- iPhone/Android 반응형 UI 테스트

구현 후 아래 항목을 체크리스트로 보고해라.
1. 화면별 구현 완료 여부
2. 버튼/팝업/스크롤 실제 동작 여부
3. 자녀 전환 시 데이터 갱신 여부
4. 쪽지 권한/삭제/복구 정상 여부
5. 앨범 다운로드 정상 여부
6. 모바일 UX 불편 요소 수정 내역
