관리자 앱 화면을 전면 개편하고, 실제 동작까지 연결해줘.
목표는 단순히 예쁜 화면이 아니라, 운영자가 직관적으로 전체 데이터를 제어할 수 있는 관리자 허브를 만드는 것이다.

[핵심 목표]
1. 관리자 메뉴를 세련되고 직관적인 구조로 재정리
2. 회원 이름 클릭 시 회원 기준 통합 상세 허브로 이동
3. 관리자 상세 페이지에서 선생님이 할 수 있는 작업 + 관리자 전용 작업까지 전부 제어 가능
4. 빠른 메뉴와 하단 탭이 겹치지 않게 역할 분리
5. 검색, 최근 본 회원, 미완료 작업 배지, 고정 액션 버튼까지 포함
6. 기존 데이터와 연결해서 실제 작동하게 구현
7. 수정/삭제 이력은 관리자만 보이게 하고 자동 로그 기록

--------------------------------------------------
[1. 하단 탭 구조 재설계]
기존 하단 탭을 아래 5개로 통합해줘.

- 대시보드
- 회원
- 수업
- 커뮤니티
- 더보기

각 탭 역할은 아래처럼 정리한다.

[대시보드]
- 전체 요약 현황
- 승인대기
- 오늘 출결
- 미배정 회원
- 최근 수정/삭제 이력
- 오늘 해야 할 일
- 빠른 액션
- 최근 본 회원

[회원]
- 전체 회원
- 학부모
- 선생님
- 승인대기
- 미배정
- 삭제회원
- 검색/필터

[수업]
- 반 관리
- 반 배정
- 출결 관리
- 레벨 테스트
- 수업일지

[커뮤니티]
- 공지사항
- 사진/앨범
- 발송 이력
- 학부모 공유 내역

[더보기]
- 결제 관리
- 지점 관리
- 브랜드 설정
- 권한 관리
- 시스템 설정
- 모드 변경
- 로그 관리

기존에 하단에 너무 많은 메뉴가 있으면 삭제하거나 더보기 안으로 이동해줘.

--------------------------------------------------
[2. 관리자 대시보드 재구성]
대시보드는 아래 순서로 구성해줘.

[상단 헤더]
- 수영장 로고
- 수영장 이름
- Powered by 스윔노트 문구 유지
- 구독 상태 배지
- 알림 버튼
- 로그아웃 버튼

[통합 검색창]
헤더 아래에 고정 검색창 추가
검색 대상:
- 회원 이름
- 학부모 이름
- 선생님 이름
- 반 이름
- 공지 제목
- 수업일지 키워드

검색 결과는 그룹별로 보여줘.
예:
- 회원 3건
- 학부모 1건
- 반 2건
- 공지 1건

검색 결과에서 사람 이름 클릭 시 무조건 통합 회원 상세 허브로 이동
반 클릭 시 반 상세 페이지
공지 클릭 시 공지 상세 페이지

[요약 KPI 카드 4개]
- 전체 회원 수
- 오늘 출석 수
- 승인대기 수
- 미배정 회원 수

이 카드는 한 줄 2개씩 배치
숫자가 바로 보이게 크게 표시

[미완료 작업 배지 섹션]
숫자 배지 형태로 아래 항목 보여줘.
- 승인대기
- 결제필요
- 미배정
- 레벨테스트 예정
- 미작성 일지
각 항목 클릭 시 해당 목록으로 이동

[빠른 액션 6개]
빠른 메뉴는 아래 6개만 보여줘.
- 회원 등록
- 승인 처리
- 반 배정
- 출결 체크
- 공지 작성
- 통합 검색

지점관리, 결제관리, 브랜드설정, 설정은 빠른 메뉴에서 제거하고 더보기로 이동

[오늘 해야 할 일]
리스트형 카드로 표시
예:
- 오늘 결제 만료 예정 3명
- 오늘 레벨 테스트 예정 2명
- 오늘 미출결 반 1개
- 승인 대기 4건

각 행 클릭 시 해당 화면 이동

[최근 수정/삭제 이력]
관리자만 보이는 섹션
최근 수정 또는 삭제된 데이터 10개 보여줘.
각 항목 왼쪽 상단에 배지 표시:
- 수정
- 삭제
표시 예:
[수정] 김민준 회원 반배정 변경
[삭제] 3시 초급반 공지 삭제

누가 변경했는지, 언제 변경했는지도 보여줘.

[최근 등록 회원]
최근 등록된 회원 5명 표시
이름, 상태, 반, 등록일 표시
이름 클릭 시 상세 허브 이동

[최근 본 회원]
운영자가 최근 열람한 회원 5명 표시
이름, 반, 최근 출석일 표시
이름 클릭 시 상세 허브 이동
최근 본 회원은 관리자 기기별 또는 계정별 local storage + server 저장 가능한 구조로 설계해줘.

--------------------------------------------------
[3. 회원 이름 클릭 시 통합 회원 상세 허브로 이동]
아래 모든 화면에서 회원 이름 클릭 시 동일한 회원 상세 페이지로 이동시켜줘.

- 회원 목록
- 학부모 목록
- 선생님이 보는 반회원 목록
- 반 배정 목록
- 출결 화면
- 승인완료 목록
- 승인대기 처리 후 결과 화면
- 검색 결과
- 최근 등록 회원
- 최근 본 회원
- 미완료 작업 리스트

즉, 회원을 나타내는 이름 텍스트/행/카드는 모두 동일한 member detail route로 연결

예시 route:
- /members/:memberId

--------------------------------------------------
[4. 통합 회원 상세 허브 설계]
이 페이지가 관리자 앱의 핵심이다.
관리자는 여기서 회원에 대한 거의 모든 작업을 한 번에 처리할 수 있어야 한다.

[상단 고정 요약 카드]
- 회원명
- 상태: 재원 / 미배정 / 승인대기 / 휴회 / 삭제회원
- 현재 레벨
- 소속 반
- 보호자명
- 담당 선생님
- 최근 출석일
- 결제 상태
- 학부모 연결 여부

상단 우측 또는 하단에 상태 변경 버튼 제공
예:
- 재원 변경
- 휴회 처리
- 삭제회원 이동
- 복구

[고정 액션 버튼]
회원 상세 페이지 하단 또는 상단에 고정 액션 버튼을 항상 보이게 해줘.
- 출결
- 일지
- 반배정
- 레벨기록
- 수정

관리자는 모두 가능
선생님은 권한 범위만 가능
권한 없는 버튼은 숨기거나 비활성화

[상세 허브 내부 탭 또는 세그먼트]
- 기본정보
- 수업정보
- 레벨/평가
- 학부모 공유정보
- 결제/이용상태
- 활동 로그

--------------------------------------------------
[5. 회원 상세 - 기본정보 탭]
포함 항목:
- 회원명
- 출생년도
- 성별
- 연락처
- 보호자명
- 보호자 연락처
- 보호자 계정 연결 상태
- 특이사항/주의사항
- 가입일
- 현재 상태
- 내부 관리자 메모

관리자 기능:
- 수정 가능
- 상태 변경 가능
- 삭제회원 이동 가능
- 복구 가능

선생님 기능:
- 내부 메모 작성 가능
- 기본정보 일부 읽기만 가능
- 민감정보 수정은 불가

--------------------------------------------------
[6. 회원 상세 - 수업정보 탭]
포함 항목:
- 현재 소속 반
- 반 배정 이력
- 요일/시간표
- 주 1회/2회/3회 설정
- 담당 선생님
- 최근 출결 현황
- 보강 기록
- 결석 기록
- 최근 수업일지
- 개별일지 추가 내용

관리자 기능:
- 반 배정 변경
- 시간표 변경
- 담당 선생님 변경
- 보강 입력/수정
- 출결 수정
- 일지 수정/삭제

선생님 기능:
- 출결 입력
- 수업일지 작성
- 개별일지 추가
- 보강/결석 메모
- 제한적 수정/삭제

회원별 수업정보는 수업 tab 뿐 아니라 여기에서도 바로 조작 가능하게 연결

--------------------------------------------------
[7. 회원 상세 - 레벨/평가 탭]
포함 항목:
- 현재 레벨
- 레벨 변경 이력
- 레벨 테스트 날짜
- 테스트 종류
- 기록 입력
예:
자유형 50m 48초
배영 50m 55초

- 통과 여부
- 평가 피드백
- 학부모에게 보여줄 코멘트

관리자 기능:
- 레벨 강제 변경 가능
- 이력 수정 가능
- 삭제된 평가 기록 복구 가능

선생님 기능:
- 레벨 테스트 기록 등록
- 피드백 입력
- 통과 여부 입력

회원 현황판과 학부모 화면에서도 동일 데이터 참조하게 연결

--------------------------------------------------
[8. 회원 상세 - 학부모 공유정보 탭]
포함 항목:
- 학부모에게 노출되는 최근 일지
- 발송된 공지 내역
- 개인 피드백 내역
- 사진/앨범 공개 범위
- 학부모 앱에서 보이는 화면 미리보기 카드

여기서는 실제 학부모가 어떤 정보를 보게 되는지 미리보기 제공
관리자는 미리보기에서
- 일지 공개/비공개
- 사진 공개 범위
- 개별 피드백 노출 여부
를 조정 가능하게 해줘.

--------------------------------------------------
[9. 회원 상세 - 결제/이용상태 탭]
포함 항목:
- 결제 상태
- 이용권 시작일
- 이용권 종료일
- 남은 기간
- 휴회 상태
- 정지 상태
- 만료 예정 여부
- 최근 결제 이력

관리자 기능:
- 결제 상태 수정
- 시작일/종료일 수정
- 휴회/정지 설정
- 만료 예정 표시 관리

선생님 권한에서는 읽기 전용으로 둬도 됨
단, 설정 파일에서 선생님도 조회는 가능하게 옵션 제공

--------------------------------------------------
[10. 회원 상세 - 활동 로그 탭]
관리자만 열람 가능
자동 기록 대상:
- 회원 정보 수정
- 반 배정 변경
- 출결 수정
- 일지 수정/삭제
- 레벨 변경
- 결제 상태 변경
- 회원 삭제/복구
- 학부모 연결/해제

각 로그 항목 표시:
- 작업 유형 배지: 수정 / 삭제 / 생성 / 복구 / 연결 / 해제
- 대상명
- 변경 전 값
- 변경 후 값
- 작업자
- 작업 시간

삭제 로그는 반드시 남기고, 가능하면 soft delete 구조로 복구 가능하게 구현

--------------------------------------------------
[11. 회원 목록 화면 개선]
회원 목록은 단순 리스트가 아니라 운영 필터 중심으로 구성해줘.

[상단]
- 검색창
- 필터 버튼
- 정렬 버튼

[필터]
- 전체
- 재원중
- 미배정
- 휴회
- 승인대기
- 삭제회원
- 학부모 미연결
- 결제필요
- 레벨테스트 예정

[회원 목록 컬럼/행 정보]
- 이름
- 구분: 회원 / 학부모 / 선생님
- 상태
- 반
- 레벨
- 보호자
- 담당 선생님
- 최근 출결
- 결제 상태

[행 액션]
- 상세보기
- 반 배정
- 출결 입력
- 일지 작성
- 레벨 기록
- 학부모 연결
- 삭제/복구

모바일에서는 테이블형이 아니라 카드형/리스트형으로 최적화
이름과 상태는 크게, 부가정보는 2줄 이하로 정리

--------------------------------------------------
[12. 선생님 권한 기능을 관리자 상세에 포함]
회원관리 목록 및 회원 상세에서는 선생님이 수행 가능한 기능도 같이 포함해야 한다.

선생님 가능 기능:
- 출결 입력
- 수업일지 작성
- 개별일지 추가
- 레벨 테스트 기록
- 사진 업로드
- 회원 메모 작성
- 제한적 수정/삭제

관리자 가능 기능:
- 선생님 가능 기능 전체
- 회원 생성/수정/삭제
- 반 생성/수정/삭제
- 반 배정 강제 변경
- 학부모 연결/해제
- 결제 상태 수정
- 이용상태 변경
- 삭제회원 복구
- 활동 로그 확인
- 전체 데이터 제어

같은 상세 페이지에서 권한별 노출 범위만 다르게 구현해줘.

--------------------------------------------------
[13. 반 관리도 사람 중심으로 연결]
반 관리에서 반을 열면 반 정보뿐 아니라 소속 회원 리스트가 나와야 하고,
회원 이름 클릭 시 통합 회원 상세 허브로 이동

반 상세 페이지 포함 항목:
- 반명
- 레벨
- 요일/시간
- 담당 선생님
- 정원
- 현재 인원
- 소속 회원 리스트
- 최근 출결 상태
- 최근 수업일지

액션:
- 회원 추가
- 회원 배정 해제
- 반 정보 수정
- 일지 바로 작성
- 출결 바로 이동

--------------------------------------------------
[14. 승인 화면 개선]
승인 화면은 학부모 승인 / 선생님 승인 / 회원 등록 승인으로 구분해서 탭 제공

승인대기 항목 표시:
- 구분
- 이름
- 신청일
- 연결 대상 정보
예:
학부모 신청이면 자녀 이름 + 출생년도
선생님 신청이면 지점/반 정보

버튼:
- 승인
- 반려
- 상세보기

승인 후 연결되는 엔티티가 맞게 생성되도록 실제 동작 구현

--------------------------------------------------
[15. 공지/커뮤니티 구조]
커뮤니티 탭 구성:
- 공지사항
- 사진/앨범
- 발송 이력
- 학부모 공유 내역

공지 작성 시:
- 전체 발송
- 반별 발송
- 회원 개별 발송
지원

사진 업로드 시:
- 반 전체 공개
- 개별 회원 공개
선택 가능

회원 상세 > 학부모 공유정보와 연결해서 어떤 자료가 누구에게 공개됐는지 확인 가능하게 구현

--------------------------------------------------
[16. 디자인 방향]
디자인 목표:
- 현재 파스텔톤 유지
- 더 고급스럽고 정돈된 느낌
- 카드 그림자는 약하게
- 불필요한 큰 카드 남발 금지
- 기능 나열보다 섹션 구조 강조
- 정보 우선순위가 눈에 잘 들어오게 구성
- 한 손 조작 고려

디자인 규칙:
- KPI 카드와 빠른 액션 카드 스타일 구분
- 리스트형 정보 섹션 적극 사용
- 상단 검색창 고정
- 회원 이름은 항상 클릭 가능한 스타일
- 상태 배지는 색으로 구분
- 하단 탭은 5개만 유지
- “모드변경”은 더보기 안으로 이동
- 대시보드에서는 카드보다 실제 필요한 리스트 정보를 더 강조

--------------------------------------------------
[17. 라우팅 및 공통 구조]
공통 라우팅 예시:
- /dashboard
- /members
- /members/:memberId
- /classes
- /classes/:classId
- /attendance
- /notices
- /settings
- /more/payments
- /more/branches
- /more/brand
- /more/roles
- /approvals

공통 원칙:
- 회원을 나타내는 모든 UI는 /members/:memberId 로 연결
- 같은 회원 상세 컴포넌트를 재사용
- 권한에 따라 액션만 다르게 표시
- 활동 로그는 공통 서비스에서 자동 적재

--------------------------------------------------
[18. 데이터 연결 방식]
기존 데이터 구조를 최대한 유지하되 memberId를 중심 허브 키로 사용해줘.

연결 대상:
- member
- parent
- teacher
- class assignment
- attendance
- lesson notes
- individual notes
- level test
- payments
- notices
- albums
- activity logs

모든 변경은 memberId 또는 related entity id 기준으로 연결
가능하면 soft delete 사용
복구 가능해야 함

--------------------------------------------------
[19. 최근 본 회원 기능]
최근 본 회원 기능 추가
동작 조건:
- 관리자가 회원 상세 페이지를 열면 최근 본 회원에 저장
- 중복이면 최신 순으로 재정렬
- 최대 5명 유지
- 대시보드에 노출

가능하면 서버 저장 + local storage fallback 방식으로 구현

--------------------------------------------------
[20. 통합 검색 기능]
통합 검색은 관리자 생산성을 위해 반드시 구현
검색 범위:
- 회원명
- 보호자명
- 선생님명
- 반명
- 공지 제목
- 수업일지 키워드

검색 결과 그룹 표시
사람 관련 결과 클릭 시 상세 허브
반은 반 상세
공지는 공지 상세

검색 결과가 없으면 빈 상태 화면 제공

--------------------------------------------------
[21. 미완료 작업 배지]
대시보드에 배지형 위젯 추가
항목:
- 승인대기
- 결제필요
- 미배정
- 레벨테스트 예정
- 미작성 일지

각 항목은 실제 데이터 개수와 연결
클릭 시 필터된 목록으로 이동

--------------------------------------------------
[22. 고정 액션 버튼]
회원 상세 페이지에는 항상 보이는 액션 버튼 영역을 만들어줘.
우선순위:
- 출결
- 일지
- 반배정
- 레벨기록
- 수정

모바일에서 스크롤해도 접근 가능하게 sticky bottom 또는 sticky section 적용

--------------------------------------------------
[23. 수정/삭제 로그 자동화]
아래 동작이 발생하면 무조건 로그를 남겨줘.
- 수정
- 삭제
- 생성
- 복구
- 연결
- 해제

로그 데이터:
- actionType
- entityType
- entityId
- memberId(optional)
- beforeValue
- afterValue
- actorId
- actorRole
- timestamp

관리자 화면 최근 이력과 활동 로그 탭에서 사용

--------------------------------------------------
[24. 빈 상태/에러 상태까지 구현]
데이터가 없을 때 어색하지 않게 아래 상태 추가
- 회원 없음
- 반 없음
- 공지 없음
- 검색 결과 없음
- 승인 대기 없음
- 최근 본 회원 없음

문구는 간단하고 직관적으로
버튼도 제공
예:
“회원 등록하러 가기”
“반 만들기”
“공지 작성하기”

--------------------------------------------------
[25. 실제 적용 조건]
중요:
- 임시 화면만 만들지 말고 실제 라우팅 연결
- 실제 버튼 동작 연결
- 실제 필터 동작 연결
- 실제 상세 이동 연결
- 기존 데이터와 매핑
- 관리자/선생님 권한 분기 반영
- 모바일에서 깨지지 않게 반응형 처리

--------------------------------------------------
[26. 기술 작업 요청]
현재 프로젝트 구조에 맞춰 아래 작업 진행해줘.
1. 관리자 대시보드 UI 리팩터링
2. 하단 탭 5개 구조로 재정비
3. 회원 상세 허브 페이지 신규 또는 전면 개편
4. 통합 검색 컴포넌트 생성
5. 최근 본 회원 기능 추가
6. 미완료 작업 배지 섹션 추가
7. 활동 로그 서비스 및 UI 추가
8. 회원 목록 필터/액션 강화
9. 반/출결/일지/레벨/결제 데이터를 회원 상세와 연결
10. 권한 제어 정비

--------------------------------------------------
[27. 구현 후 반드시 자체 점검]
아래를 직접 확인해줘.
- 모든 회원 이름 클릭이 동일한 상세 페이지로 이동하는지
- 관리자 상세에서 선생님 작업까지 바로 가능한지
- 빠른 메뉴와 하단 탭 역할이 겹치지 않는지
- 최근 본 회원이 정상 저장되는지
- 통합 검색이 여러 엔티티를 찾는지
- 미완료 작업 배지 숫자가 실제와 맞는지
- 수정/삭제 이력이 자동으로 남는지
- 삭제 후 복구가 되는지
- 모바일에서 버튼 겹침이 없는지
- 하단 고정 액션 버튼이 실제 작동하는지

작업 완료 후에는 변경된 파일 목록, 추가된 컴포넌트, 라우트 구조, 데이터 연결 포인트를 정리해서 보여줘.
