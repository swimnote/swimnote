지금부터 아래 버그를 최우선으로 수정하고, 수정 후 반드시 실제 테스트까지 수행해서 결과를 상세하게 보고해라.

[버그 내용]
학부모 모드에서 우측 상단 나가기 버튼을 누르면 원래 로그인 화면으로 이동해야 하는데,
현재는 로그인 입력창이 있는 화면으로 정상 복귀하지 않고,
중간에 스윔노트 로고/소개만 보이는 화면에 멈추거나 잘못된 중간 화면으로 이동하는 버그가 발생한다.

즉 현재 문제는:
- 나가기 버튼 클릭
- 로그인 화면으로 완전히 복귀하지 못함
- 스윔노트 소개/중간 랜딩 비슷한 화면만 보임
- 로그인 폼이 바로 안 보이거나 라우팅 상태가 꼬임

이 버그를 반드시 수정해라.

--------------------------------------------------
[수정 목표]
학부모 모드에서 나가기 버튼 클릭 시 반드시 아래 순서로 동작하게 수정한다.

1. 현재 학부모 세션 종료
2. 선택된 학생 상태 초기화
3. 학부모 관련 캐시/피드/앨범/출결/공지 상태 초기화
4. 라우팅 스택 reset
5. "로그인 입력창이 포함된 실제 로그인 화면"으로 바로 이동

중요:
- 스윔노트 소개만 있는 중간 화면에서 멈추면 실패
- 로그인 폼이 없는 화면으로 가면 실패
- 이전 학부모 데이터가 남아 있으면 실패
- 단순 goBack 같은 방식으로 처리하지 말 것
- 반드시 reset/replace 방식으로 로그인 화면을 최종 진입점으로 만들어라

--------------------------------------------------
[필수 수정 사항]

1. 학부모 모드 나가기 버튼 수정
- 우측 상단 나가기 버튼 클릭 시 아래 동작을 순서대로 수행하게 수정
  a. selected_student_id 초기화
  b. parent profile / children / recent updates / feed / album / attendance / notice cache 초기화
  c. auth/session/token/role/current user 초기화
  d. 임시 UI 상태 초기화
  e. 로그인 화면 route로 stack reset
  f. 최종적으로 로그인 입력창이 있는 화면이 바로 보여야 함

2. 로그인 화면 라우팅 수정
- reset 이후 최종 목적지는 반드시 로그인 폼이 포함된 화면이어야 함
- 스윔노트 소개/랜딩/인트로 화면과 로그인 화면이 분리되어 있다면,
  나가기 버튼은 랜딩이 아니라 로그인 폼 화면으로 바로 가게 수정
- 만약 소개/랜딩 화면과 로그인 폼이 같은 화면이라면,
  reset 후 로그인 폼 위치까지 정상 표시되게 보장할 것

3. 상태 꼬임 방지
- 학부모 모드에서 나간 뒤 다시 로그인했을 때
  이전 학생명, 이전 피드, 이전 앨범 상태가 남지 않게 수정
- 다른 계정 로그인 시 이전 학부모 상태가 섞이면 안 됨

4. 기존 구조 유지
- 이번 작업은 나가기/로그아웃 라우팅 버그 수정이 목적
- 기존 학부모 홈 UI 구조는 유지
- 홈/수업피드백/앨범/출결/쇼핑/더보기 탭 구조 유지
- 학부모 홈 첫 진입은 홈으로 유지

--------------------------------------------------
[테스트 지시 - 반드시 실제 실행]

아래 테스트를 수정 후 직접 수행하고 결과를 상세 보고해라.

테스트 1. 학부모 로그인 후 홈 진입
- 학부모 계정으로 로그인
- 홈 화면 정상 진입 확인

테스트 2. 학부모 모드 우측 상단 나가기 버튼
- 홈 화면에서 나가기 버튼 클릭
- 로그인 입력창이 있는 로그인 화면으로 즉시 이동하는지 확인
- 스윔노트 소개만 나오는 중간 화면에 멈추지 않는지 확인

테스트 3. 상태 초기화 확인
- 나가기 후 다시 학부모 로그인
- 이전 selected_student_id, 이전 피드, 이전 공지, 이전 앨범 상태가 남지 않는지 확인

테스트 4. 다른 계정 전환 확인
- 학부모 나가기
- 다른 계정 또는 테스트 계정 진입
- 이전 학부모 데이터가 섞이지 않는지 확인

테스트 5. 반복 테스트
- 로그인 → 학부모 홈 → 나가기
- 이 흐름을 3회 이상 반복
- 매번 로그인 입력창 화면으로 정상 복귀하는지 확인

아래 중 하나라도 해당하면 수정 실패로 보고해라.
