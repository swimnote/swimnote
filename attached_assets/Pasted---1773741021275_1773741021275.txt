지금부터 아래 내용을 기준으로 선생님 모드/관리자 모드의
시간표 + 출결 + 미실시 + 보강 + 임시이동 + 미등록 회원 + 월 정산(매출계산기)
기능을 실제로 연결된 구조로 구현하라.

중요:
- 화면만 만들지 말고 실제 데이터가 저장/계산/이동/정산에 반영되게 구현
- 가짜 UI 금지
- 상태값만 바꾸는 척하는 기능 금지
- 시간표, 출결, 일지, 보강, 회원상태, 정산표가 모두 서로 연결되어야 함
- 수정 후 반드시 실제 테스트까지 수행하고 결과를 상세 보고할 것

==================================================
[0. 전체 기능 목적]
이 기능은 단순 계산기가 아니라 아래가 모두 연결된 자동 정산 시스템이다.

- 회원 등록
- 수업유형 단가
- 시간표 생성
- 출결
- 보강
- 미실시(수영장/선생님)
- 임시이동 수업
- 미등록 회원
- 기타 수업
- 월말 정산
- 다음 달 수업 시작

정산의 최종 원칙:
"그날 실제 수업을 진행한 선생님 기준으로 정산한다"

==================================================
[1. 관리자 설정 - 수업유형/단가표]

관리자 설정에 정산용 수업유형 단가표를 만든다.

기본 고정 유형:
- 주1회
- 주2회
- 주3회

추가 유형:
- 커스텀 수업 2개까지
- 이름과 금액을 관리자가 직접 입력

예:
- 주1회 180000원
- 주2회 280000원
- 주3회 금액
- 커스텀1 이름/금액
- 커스텀2 이름/금액

추가 예외 처리:
- 계산기 하단에 기타 수기 정산 항목 1개 추가
- 선생님이 직접 입력 가능
- 삭제는 못 하게 할 것

단가 기준:
- 월 수업료 기준
- 회당 단가를 내부 계산용으로 자동 환산해도 되지만
  화면에는 월 수업료 기준이 기본으로 보여야 함

==================================================
[2. 회원 수업유형 구조]

회원에게 아래 수업유형 중 하나를 붙인다.
- 주1회
- 주2회
- 주3회
- 커스텀1
- 커스텀2

정산 기준 기본 횟수:
- 주1회 = 4회
- 주2회 = 8회
- 주3회 = 12회

중요:
- 5주차는 기본 횟수 계산에 자동 포함하지 않는다

==================================================
[3. 회원 상태 구조]

회원 상태:
- 정상
- 연기
- 탈퇴

정산 대상:
- 정상회원 기준

연기회원:
- 연기 전 실제 진행 수업까지만 정산
- 연기 후 수업은 제외
- 연기 상태가 되면 해당 기간 스케줄에서 자동 제외

탈퇴회원:
- 탈퇴 전 실제 진행 수업까지만 정산
- 탈퇴 후 수업 없음

체험회원:
- 정산 포함
- 정산표에 체험 횟수 별도 표시

무료회원:
- 추후 관리자 설정 여지 남겨두되,
  현재는 구조만 열어두고 기본 흐름은 일반 회원 중심으로 구현

==================================================
[4. 다음 달 수업 생성 구조]

관리자 탭에 "다음 달 수업 일정표" 기능 추가

관리자가 다음 달 달력을 보고
- 수영장 전체 휴무일
- 임시 휴무일
을 먼저 체크할 수 있게 한다.

그 후 시스템이 다음 달 수업을 자동 생성한다.

원칙:
- 수영장 전체 휴무는 관리자만 설정 가능
- 반별 휴무는 현재 넣지 않음

휴무일 체크 시:
- 해당 날짜는 전체 선생님 스케줄에서 수업 없음
- 해당 날짜 수업은 생성되지 않음

==================================================
[5. 휴무일 때문에 빠진 수업 처리]

예:
주1회 회원인데 휴무 때문에 이번 달 3회만 생성된 경우

처리:
- 빠진 수업은 결석이 아니라 "미실시(수영장)"로 처리
- 자동으로 다음 달 보강 리스트로 넘김
- 소멸 불가

즉 관리자 휴무일로 인해 빠진 수업은
"미실시(수영장) -> 보강 이월" 구조로 간다.

==================================================
[6. 선생님 결근 처리]

선생님 수업 화면 또는 시간표에 "결근 처리" 기능 추가

결근 처리 시 바로 전원 미실시 하지 말고 팝업 표시:

"결근 처리합니다. 옆 반 이동 수업하는 친구가 있습니까?"

선택지:
- 없음
- 있음

--------------------------------------
[6-1. 없음 선택]
- 해당 수업 학생 전체를 "미실시(선생님)" 처리
- 전부 보강 리스트로 이동
- 소멸 불가

--------------------------------------
[6-2. 있음 선택]
- 같은 시간대 수업표 표시
- 옆 반/선생님 선택 가능
- 이동할 학생 선택 가능

선택된 학생:
- 옆 반 임시 수업으로 생성
- 받은 선생님이 정상 출결 처리
- 받은 선생님이 정상 일지 작성
- 받은 선생님에게 해당 수업 정산 귀속

남은 학생:
- 자동으로 "미실시(선생님)" 처리
- 보강 리스트로 이동
- 소멸 불가

중요:
임시 이동은 1회성
- 그날 수업 후 다음 수업부터는 자동으로 원래 반/원래 선생님으로 복귀

==================================================
[7. 임시 이동 수업 정산 규칙]

임시 이동 수업은
"그날 실제 수업을 진행한 선생님"에게 정산 귀속

즉
- 원래 담당 선생님은 결근했으면 그 회차 정산 없음
- 실제 받은 옆 반 선생님이 정산 받음

정산표에는 "임시이동" 표시를 별도로 붙이고
임시 이동 수업 횟수도 별도로 카운팅해서 보여줘야 함
(체험 수업처럼 별도 카운터 필요)

==================================================
[8. 미실시 수업 종류]

미실시는 2종류로 나눠야 한다.

1) 미실시(수영장)
- 관리자 휴무일 등 수영장 사유
- 보강 이월
- 소멸 불가

2) 미실시(선생님)
- 선생님 결근
- 보강 이월
- 소멸 불가

보강 리스트에서 반드시 구분 표시할 것.

예시 표시:
- 일반 보강
- 미실시(수영장)
- 미실시(선생님)

중요:
미실시 수업은 소멸 버튼 비활성화 또는 불가 처리
환불/별도 처리 논의는 관리자 영역에서만 가능하게 할 것

==================================================
[9. 출결 반영 기준]

출결 상태:
- 출석
- 결석
- 보강

정산 포함:
- 출석
- 실제 진행된 보강
- 실제 진행된 기타 수업
- 체험 수업
- 임시 이동 수업

정산 제외:
- 결석
- 미실시(수영장)
- 미실시(선생님)

==================================================
[10. 보강 구조]

보강 리스트에는 아래가 올라가야 함
- 일반 결석 보강
- 미실시(수영장) 보강
- 미실시(선생님) 보강

보강 항목에 표시할 정보:
- 학생 이름
- 발생 사유
- 발생 날짜
- 상태
- 남은 기간(구조 여지만 마련)
- 미실시 여부 아이콘/태그

==================================================
[11. 5주차 처리 방식]

5주차는 기본 수업횟수에 자동 반영하지 않는다.

5주차 또는 특수 추가 수업은
"기타 수업"으로 처리한다.

즉:
- 5주차도 기타 수업으로 생성 가능
- 기본 4/8/12 횟수 차감 없이 학생을 넣을 수 있음
- 정산은 추가 반영

==================================================
[12. 기타 수업]

기타 수업은 선생님이 직접 생성 가능
단, 삭제는 못 하게 할 것

중요:
기타 수업은 정산표에 수기로 바로 입력되는 게 아니라
"실제 시간표에 생성되고, 실제 진행된 수업만" 정산에 들어가야 한다.

필수 구조:
- 선생님이 시간표에서 기타 수업 생성
- 날짜/시간/수업명 입력
- 학생 명단 선택
- 실제 수업 슬롯이 생성
- 출결/진행 처리
- 그때만 정산 반영

즉 시간표에 없는 기타 수업은 정산에 들어가면 안 된다.

==================================================
[13. 미등록 회원]

미등록 회원은 일반 회원관리에서 보이면 안 된다.

노출 위치:
- 시간표에서 반 생성 / 학생 추가할 때만
- 검색창 최상단에 "미등록" 항목 노출

동작:
- 미등록을 추가할 때마다
  미등록 1
  미등록 2
  미등록 3
  미등록 4
  미등록 5
  형식으로 생성
- 최대 5명까지
- 번호는 해당 수업 단위에서만 의미 있음
- 전역 공통 번호 불필요

미등록도 임시 학생 객체로 생성해서
아래가 기록돼야 함:
- 출결
- 일지
- 기타 수업 참여
- 정산

학부모 화면에는 미등록 회원 절대 보이면 안 됨

--------------------------------------
[13-1. 미등록 회원 교체]
미등록 회원은 나중에 실제 회원 등록되면 삭제가 아니라 "교체"해야 한다.

기능 이름:
- 미등록 회원 교체

권한:
- 담당 선생님이 수행
- 관리자는 직접 교체하지 않음

교체 흐름:
1. 미등록 목록 표시
2. 교체할 미등록 선택
3. 실제 등록 회원 검색
4. 선택한 회원으로 교체

교체 결과:
- 기존 미등록의 출결/일지/수업/정산 기록이 실제 회원에게 그대로 연결
- 기록 삭제/초기화/재생성 금지

==================================================
[14. 기타 수기 정산]

기타 수기 정산은 선생님이 직접 입력 가능
하지만 삭제는 못 하게 한다.

단, 기본 흐름은 어디까지나 시간표 기반 수업 정산이다.
기타 수기 정산은 정말 예외적인 경우만 쓰는 보정용이다.

화면 하단에 별도 항목으로 두고
정산표에 반영되게 한다.

==================================================
[15. 회원별 카운터]

각 회원마다 매월 보여줘야 하는 정보:
- 진행 횟수
- 총 예정 횟수
- 남은 횟수

표시 방식 추천:
- 현재 진행 / 총 횟수

예:
- 3 / 4
- 7 / 8
- 10 / 12

체험 수업 횟수
임시 이동 수업 횟수
도 별도 카운팅하여 정산표에 표시

==================================================
[16. 매출계산기 화면 구조]

선생님 모드에 "내 매출계산기" 추가

상단 요약:
- 이번 달 총 매출
- 총 수업 횟수
- 보강 횟수
- 체험 수업 횟수
- 임시 이동 수업 횟수

회원별 리스트:
- 이름
- 수업 유형
- 상태
- 진행 횟수 / 총 횟수
- 월 수업료
- 현재 정산 금액
- 체험 표시
- 신규 표시(구조 여지)
- 임시 이동 여부
- 미실시 여부가 있었는지 관련 정보

하단:
- 기타 수기 정산 항목
- 이번 달 정산 저장 버튼
- 다음 달 정산 시작 버튼

==================================================
[17. 월말 정산 -> 다음 달 시작]

월말 정산이 끝나면 선생님이 직접 "다음 달 정산 시작" 버튼을 누른다.

자동으로 무조건 넘어가면 안 된다.

흐름:
1. 이번 달 정산 저장
2. 다음 달 정산 시작 클릭
3. 다음 달 시작할 회원 목록 표시
4. 제외할 회원 제외
5. 다음 달 반 생성/수업 생성

즉
- 이번 달 종료는 자동 계산
- 다음 달 시작은 선생님 확인 후 수동 확정

==================================================
[18. 정산 귀속 최종 원칙]

정산은 등록 기준이 아니라 실제 수업 진행 기준

최종 원칙:
- 정규 수업: 실제 출결 처리한 선생님 귀속
- 보강 수업: 실제 진행한 선생님 귀속
- 임시 이동 수업: 받은 선생님 귀속
- 기타 수업: 실제 생성/진행한 선생님 귀속
- 미실시(수영장/선생님): 정산 없음

==================================================
[19. 연동 원칙]

아래가 실제로 연결되어야 한다.

- 시간표 생성 → 출결 생성 가능
- 휴무일 설정 → 수업 생성 제외
- 휴무일 제외 수업 → 미실시(수영장) 보강 이월
- 결근 처리 → 일부 임시 이동 / 나머지 미실시(선생님)
- 임시 이동 → 받은 선생님 시간표에 임시 학생 표시
- 미등록 학생 → 나중에 실제 회원으로 교체 가능
- 출결 기록 → 정산 카운터 반영
- 보강 진행 → 정산 포함
- 기타 수업 진행 → 정산 포함
- 회원 상태 변경(연기/탈퇴) → 스케줄/정산 반영

==================================================
[20. 테스트 - 반드시 실제 수행]

수정 후 아래 테스트를 반드시 직접 수행하고 상세 보고할 것.

테스트 1. 기본 정산
- 주1/주2/주3 회원 각각 등록
- 출결 입력
- 진행 횟수 / 총 횟수 / 정산 금액 정상 계산 확인

테스트 2. 관리자 휴무일
- 다음 달 일정표에서 휴무일 지정
- 해당 날짜 수업 생성 안 되는지 확인
- 빠진 수업이 미실시(수영장) 보강으로 넘어가는지 확인

테스트 3. 선생님 결근 처리
- 결근 처리
- "옆 반 이동 학생 있습니까?" 팝업 표시 확인
- 없음 선택 시 전체 미실시(선생님) 보강 이동 확인
- 있음 선택 시 이동 학생만 임시 수업 생성, 남은 학생은 미실시 처리 확인

테스트 4. 임시 이동 정산
- 임시 이동 학생 수업 진행
- 받은 선생님 정산에 반영되는지 확인
- 정산표에 임시이동 표시가 뜨는지 확인

테스트 5. 기타 수업
- 선생님이 기타 수업 생성
- 시간표에 실제 슬롯 생성 확인
- 등록 회원 추가 후 진행
- 정산 반영 확인

테스트 6. 미등록 회원
- 반 생성 시 미등록 1~5 추가 가능한지 확인
- 출결/일지/정산 반영 확인
- 미등록 회원 교체 후 기존 기록이 실제 회원에게 이어지는지 확인
- 학부모 화면에는 노출되지 않는지 확인

테스트 7. 보강/미실시
- 일반 보강 / 미실시(수영장) / 미실시(선생님) 구분 표시 확인
- 미실시 수업 소멸 불가 확인

테스트 8. 연기/복귀
- 연기 처리 시 스케줄 자동 제외 확인
- 복귀 시 자동/변경 복귀 구조 확인

테스트 9. 정산 저장/다음 달 시작
- 이번 달 정산 저장
- 다음 달 정산 시작
- 시작할 회원 목록 확인
- 제외/생성 흐름 확인

[실패 기준]

아래 중 하나라도 해당하면 실패로 보고해라.

- 시간표에 없는 기타 수업이 정산에 들어감
- 미등록 기록이 교체 시 끊김
- 미실시 수업이 소멸 가능함
- 임시 이동 수업이 원래 선생님 정산으로 잡힘
- 휴무일인데 수업이 생성됨
- 결근 처리 후 남은 학생이 미실시로 안 넘어감
- 정산표에 체험/임시이동 구분이 안 보임
- 다음 달 시작 시 회원 제외/생성 흐름이 작동하지 않음

이 작업은 실제 운영용 기능이므로
UI만 만드는 것이 아니라 반드시 실제 동작과 데이터 연결까지 완료해라.
