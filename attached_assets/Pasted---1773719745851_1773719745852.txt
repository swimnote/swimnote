지금부터 아래 내용을 기준으로 학부모 모드의 진입/뒤로가기/상태 초기화 버그를 먼저 수정하고, 수정 후 반드시 실제 동작 테스트까지 진행해서 보고해라.

중요:
- 기존에 합의된 학부모/선생님 구조는 유지
- 기능 삭제하거나 임시 우회하지 말 것
- 이번 작업은 "버그 수정 + 기존 구조 유지 + 테스트 검증"까지 한 번에 진행
- 화면만 바뀐 척하지 말고 로그인 상태/선택 학생 상태/캐시 상태까지 실제 초기화되게 수정할 것

[현재 발생 중인 핵심 버그]
1. 학부모 홈 화면에서 뒤로가기 버튼을 누르면 로그인 화면으로 가지 않고, 스윔노트 기본 화면처럼 보이는 이상한 중간 화면으로 빠진다.
2. 뒤로가기 후 계정 상태가 꼬여서, 다른 학부모 계정처럼 보이거나 이전 데이터가 남아 있는 비정상 상태가 생긴다.
3. 학부모 로그인 직후 첫 진입 화면이 홈이 아니라 자녀 카드 화면으로 먼저 뜬다.
4. 학부모는 로그인 후 무조건 홈 화면으로 바로 들어가야 하는데 현재 그 흐름이 깨져 있다.

[이번 수정 작업 목표]
A. 학부모 홈에서 뒤로가기 누르면 반드시 원래 로그인 화면으로 이동
B. 뒤로가기 시 현재 학부모 세션, 선택 학생 상태, 학부모 피드/캐시 상태를 전부 초기화
C. 학부모 로그인 성공 시 첫 진입 화면은 무조건 홈 화면
D. 자녀 카드 목록 화면이 첫 진입 랜딩이 되면 안 됨
E. 수정 후 실제 테스트까지 진행해서 결과를 상세 보고

[수정 원칙]
- goBack 류의 단순 뒤로가기 방식으로 처리하지 말 것
- 로그인 화면으로 갈 때는 stack reset / route reset 방식으로 처리
- 상태값만 비워놓는 척하지 말고, 실제 학부모 관련 store / context / local cache / selected_student_id / 화면 데이터까지 초기화
- 로그인 후 parent role이면 홈 화면으로 직접 진입하도록 라우팅 고정
- 홈 화면 뒤로가기 버튼은 "이전 페이지"가 아니라 "로그인 화면으로 복귀" 역할로 동작해야 함
- 뒤로가기 후 이전 학부모 데이터가 남아 있으면 실패로 간주

[필수 수정 사항]

1. 학부모 홈 뒤로가기 동작 수정
- 현재 학부모 홈 상단 뒤로가기 버튼 클릭 시 아래가 순서대로 실행되게 수정
  1) selected_student_id 초기화
  2) 학부모 정보/자녀 목록/최근 피드/공지/앨범 캐시 초기화
  3) 현재 로그인 세션(auth) 초기화
  4) 라우팅 스택 reset
  5) 로그인 화면으로 이동

- 뒤로가기 후 보이면 안 되는 것
  - 스윔노트 기본 홈 비슷한 중간 화면
  - 이전 학부모 이름/학생 이름 잔상
  - 이전 피드/업데이트 카드
  - 다른 계정처럼 보이는 데이터 꼬임

2. 학부모 로그인 후 첫 진입 화면 수정
- 학부모 로그인 성공 시 무조건 ParentHome(학부모 홈)으로 직접 진입
- 자녀 카드 선택 화면이 첫 화면처럼 뜨지 않게 수정
- 자녀가 여러 명이어도 홈 상단에서 자녀 전환하게 하고, 첫 랜딩은 홈으로 고정

3. 상태 초기화 범위
반드시 초기화할 것:
- auth 관련 로그인 상태
- parent profile
- children list
- selected_student_id
- home feed cache
- album cache
- attendance cache
- notice cache
- parent UI temporary state
- 학부모 role 전용 local state

4. 홈 화면 유지 조건
- 학부모 로그인 성공 → 홈 화면
- 홈 상단에 아이 카드, 레벨 아이콘, 바로가기, 최근 업데이트, 광고 일부 노출 유지
- 기존 홈 구조는 유지하고, 진입/뒤로가기 흐름만 정상화

5. 기존 합의 구조는 절대 깨지지 않게 유지
유지해야 할 것:
- 하단 탭: 홈 / 수업피드백 / 앨범 / 출결 / 쇼핑 / 더보기
- 수업피드백 명칭
- 앨범 구조
- 공지 전체/반 배지
- 홈 최근 업데이트 3개 제한
- 광고 영역 일부 노출
- 학부모는 모든 정보가 자기 아이 기준으로 보이는 구조

[테스트 지시 - 반드시 실제 확인]
수정 후 아래 테스트를 직접 수행하고 결과를 항목별로 보고해라.

테스트 1. 학부모 로그인 첫 진입
- 학부모 로그인
- 첫 화면이 홈인지 확인
- 자녀 카드 화면이 먼저 뜨지 않는지 확인

테스트 2. 학부모 홈 뒤로가기
- 홈 화면 진입 후 뒤로가기 클릭
- 로그인 화면으로 바로 이동하는지 확인
- 스윔노트 기본 홈 같은 중간 화면이 뜨지 않는지 확인

테스트 3. 뒤로가기 후 상태 초기화
- 로그인 화면으로 돌아간 뒤 다시 다른 계정 또는 같은 계정 로그인
- 이전 학부모 데이터가 남지 않는지 확인
- selected_student_id가 초기화되었는지 확인
- 피드/앨범/출결 데이터 잔상 없는지 확인

테스트 4. 자녀 여러 명 시나리오
- 자녀 여러 명 연결 계정으로 로그인
- 첫 화면은 홈인지 확인
- 홈 상단에서 자녀 전환이 정상 작동하는지 확인

테스트 5. 탭 이동 후 뒤로가기
- 홈 → 수업피드백/앨범/출결 등 이동
- 다시 홈 복귀
- 홈에서 뒤로가기 누를 때 로그인 화면으로 잘 나가는지 확인

[보고 형식 - 이렇게 보고해라]
1. 수정한 파일/구조
2. 뒤로가기 동작 수정 방식
3. 로그인 후 첫 진입 화면 수정 방식
4. 상태 초기화 범위
5. 테스트 1~5 결과
6. 아직 남은 버그/불안정한 부분
7. 임시 처리한 부분 있으면 숨기지 말고 명시

[중요]
- "수정 완료"라고만 말하지 말고 실제 테스트 결과까지 적어라.
- 중간 화면이 한 번이라도 보이면 실패다.
- 로그인 후 홈이 아니라 자녀 카드 화면이 먼저 뜨면 실패다.
- 이전 계정 데이터가 조금이라도 남아 있으면 실패다.

이 작업을 먼저 끝내고, 학부모 진입/로그아웃/뒤로가기 흐름이 완전히 정상화된 다음 다음 수정으로 넘어가라.
