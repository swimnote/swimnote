보안설정 > 외부 서비스 연결 상태 화면을 전면 개편해라.
목표는 단순 연결 상태 표시가 아니라, 슈퍼관리자가 앱을 켰을 때 플랫폼 자원의 연결 상태, 사용량, 경고 임계치, 최근 이상 상태를 전기 배전판처럼 한눈에 확인할 수 있게 만드는 것이다.

[최상단 구조]
상단 제목은 “플랫폼 인프라 상태”로 변경하고,
보조 문구는 “연결 상태 · 사용량 · 경고 임계치 · 최근 확인 결과”로 표시해라.
우측 상단에는 전체 새로고침 버튼과 전체 상태 요약 배지를 표시해라.
예: 정상 12개 / 주의 3개 / 오류 2개

[상단 핵심 4카드]
반드시 아래 4개를 최상단 요약 카드로 따로 분리해라.
1. 슈퍼관리자 DB
2. 수영장 운영 DB
3. 사진 저장소
4. 영상 저장소

각 카드에 반드시 표시할 정보:
- 상태 배지 (정상 / 임박 / 위험 / 가득 참 / 지연 / 오류 / 비활성)
- 설명 문구
- 현재 사용량
- 총 한도
- 사용률 %
- 응답속도(ms) 또는 성능 지표
- 핵심 운영 지표 또는 파일 수
- 최근 확인 시각
- 새로고침 버튼
- 상세보기 버튼

[카드별 상세 요구사항]

1) 슈퍼관리자 DB 카드
제목: 슈퍼관리자 DB
설명: 플랫폼 계정 · 구독 · 결제 · 감사로그 원본

표시:
- used_mb / limit_mb
- usage_percent

- student_count

3) 사진 저장소 카드
제목: 사진 저장소
설명: 기본 제공 사진 업로드 · 다운로드 압축본 기준
표시:
- used_mb / limit_mb
- usage_percent

4) 영상 저장소 카드

표시:

- recent_error_count
- last_checked_at

[경고 기준]
모든 저장소/DB 용량은 아래 임계치를 공통 적용해라.
- 0~79.9% = 정상
- 80~89.9% = 임박
- 90~99.9% = 위험
- 100% 이상 = 가득 참

응답속도 기준:
- 0~299ms = 정상
- 300~999ms = 지연
- 1000ms 이상 = 심각 지연

연결 실패 기준:
- 최근 점검 성공 = 정상
- 최근 1회 실패 후 복구 = 주의
- 최근 3회 연속 실패 = 오류

영상 기능 기준:
- video_storage_limit_mb = 0 이면 비활성
- video_storage_limit_mb > 0 이고 80% 미만이면 정상
- 80% 이상이면 임박
- 90% 이상이면 위험
- 100%이면 가득 참

[서비스 그룹 분리]

[프론트 표시 규칙]
- MB/GB 자동 변환
- 퍼센트 소수점 1자리
- 최근 확인은 “n분 전”
- 오류 수 0이면 “오류 없음”
- 비활성은 회색 계열
- 임박/위험/가득 참은 색과 문구를 둘 다 바꿔라
- super DB와 pool DB가 같은 카드로 합쳐져 보이면 실패
- 사진과 영상이 같은 저장소로 합쳐져 보이면 실패

[하단 추가 섹션]
반드시 아래 2개 섹션도 추가해라.

1. 최근 이상 감지
예:
- 사진 저장소 용량 82% 임박
- 이메일 인증 API 키 끊김
- Sentry 오류 이벤트 급증
- PortOne 토큰 만료 임박

2. 최근 24시간 상태 이력
예:
- 14:10 Cloudflare R2 재연결 성공
- 15:40 SendGrid 인증 실패
- 17:05 super DB 점검 완료

[중요]
- mock 데이터 금지
- 실제 API 기반으로 연결
- 현재 값이 없으면 null 또는 미측정으로 표시
- 확인하지 못한 값은 추정해서 채우지 마라
