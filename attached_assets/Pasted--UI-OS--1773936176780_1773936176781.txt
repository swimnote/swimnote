지금부터 관리자 모드 전체 UI를 “아이콘 기반 운영 OS 구조”로 재배치하라.

이번 작업 목표는
기존 기능은 하나도 깨뜨리지 않고,
관리자 모드의 진입 구조와 레이아웃만
아이콘 중심의 통일된 운영 시스템 형태로 바꾸는 것이다.

가장 중요:
- 기능 로직은 유지
- 버튼 연결 유지
- 데이터 구조 유지
- 반배정/출결/일지/보강/메신저/매출/설정 기능 전부 유지
- 이번 작업은 “진입 구조와 레이아웃 재배치”가 핵심
- 배치를 바꾼다고 기존 기능이 틀어지면 안 된다
- 통일감이 생명이다
- 심플하지만 전 화면이 같은 규칙으로 움직여야 한다

==================================================
[0. 공통 UI/UX 규칙 - 관리자/선생님 공통 규칙으로 별도 고정]

이번 작업을 시작하기 전에,
아래 규칙을 공통 규칙으로 먼저 정리하고
이후 관리자 모드 재배치에 적용하라.
이 규칙은 이후 선생님 모드에도 그대로 재사용할 것이다.

--------------------------------------
[공통 규칙 1. 전체 구조]

전체 구조는 아래 3단계까지만 허용한다.

1단계: 홈 화면
2단계: 아이콘 팝업
3단계: 실제 페이지

즉:
홈 → 팝업(아이콘) → 페이지

절대 4단계 이상 깊어지지 말 것

--------------------------------------
[공통 규칙 2. 홈 화면]

- 상단에는 배너 1개
- 그 아래 메인 아이콘 8개
- 홈은 항상 첫 화면
- 홈은 배너 + 아이콘만 깔끔하게 보이는 구조
- 홈에서 목록을 바로 길게 보여주지 말 것

--------------------------------------
[공통 규칙 3. 팝업]

- 홈에서 아이콘 클릭 시 반드시 팝업이 열린다
- 팝업은 “아이콘 창고”
- 팝업 안에서는 실제 데이터 리스트를 길게 보여주지 않는다
- 팝업 안에는 세부 아이콘만 보여준다
- 팝업 아이콘은 3열 그리드
- 팝업 디자인은 전부 통일
- 팝업 닫는 방식도 전부 통일
- 바깥 터치 닫기 / X 버튼 / 드래그 닫기 규칙 통일
- 팝업 내부 아이콘을 누르면 그때 페이지 이동

--------------------------------------
[공통 규칙 4. 페이지]

- 실제 작업은 페이지에서만 한다
- 페이지 상단에는 공통 헤더
- 헤더 왼쪽: 뒤로가기
- 헤더 오른쪽: 홈
- 수정 화면이면 저장 버튼도 항상 같이 존재
  예:
  왼쪽 ←
  오른쪽 홈 / 저장

규칙:
- 뒤로가기 = 바로 이전 단계
- 홈 = 앱 첫 홈 화면으로 즉시 이동
- 저장 가능한 화면은 저장 버튼이 항상 있어야 함

--------------------------------------
[공통 규칙 5. 페이지 내부 분류]

- 페이지 안에서 더 분류가 필요하면 최대 탭 1~2단계까지만 허용
- 탭 수는 과도하게 늘리지 말 것
- 예: 일간 / 주간 / 월간 정도는 허용
- 페이지 안에서 또다시 깊은 페이지 이동 남발 금지

--------------------------------------
[공통 규칙 6. 리스트]

- 리스트 화면에는 선택 기능과 삭제 기능 기본 지원
- 체크박스 또는 선택 모드 유지
- 선택 / 삭제는 기존 기능 유지
- 리스트 카드 크기는 한눈에 많이 볼 수 있게 컴팩트하게
- 검색창 위치는 상단 고정
- 상태칩은 2줄 컴팩트 구조 허용

--------------------------------------
[공통 규칙 7. 통일감]

반드시 통일할 것:
- 팝업 모양
- 팝업 아이콘 배치
- 페이지 헤더
- 뒤로가기/홈/저장 위치
- 카드 간격
- 검색창 위치
- 상태칩 높이
- 버튼 스타일

즉:
기능이 달라도 같은 앱처럼 보여야 한다

==================================================
[1. 관리자 홈 화면 재배치]

관리자 모드 첫 화면을 완전히 아이콘 기반 홈 화면으로 재배치한다.

구조:
1. 홈 배너
2. 메인 아이콘 8개

중요:
- 기존 목록형 첫 화면 제거
- 기존에 첫 화면에서 바로 길게 나오는 목록/칩 구조 제거
- 홈은 요약 + 아이콘 진입 허브만 남길 것

==================================================
[2. 홈 배너 구성]

홈 배너에는 아래 4개를 우선 배치한다.

1. 데이터 사용량
2. 이번 달 전체 매출
3. 이번 달 전체 회원
4. 미처리 보강

배너 규칙:
- 4줄 또는 2x2 요약 카드 형태 가능
- 클릭 가능 구조 준비
- 추후 항목 교체 가능하게 유연하게 구성
- 지금은 이 4개를 우선 배치

중요:
홈 배너는 관리자 첫 화면의 핵심 요약판이다

==================================================
[3. 관리자 홈 메인 아이콘 8개]

홈 배너 아래에 8개 메인 아이콘 배치

배치:
- 4 x 2 그리드
- 한 화면에 8개 보이게
- 나중에 확장 가능하도록 여백/크기 조절

메인 아이콘 8개:
1. 인원관리
2. 수업관리
3. 보강관리
4. 메신저
5. 매출관리
6. 데이터관리
7. 운영설정
8. 플랫폼설정

중요:
- 메신저는 단독 예외
- 메신저 아이콘 클릭 시 팝업 없이 바로 메신저 페이지로 이동
- 나머지 7개는 아이콘 팝업을 거친다

==================================================
[4. 메신저 예외 규칙]

메신저는 단일 기능 진입 구조이므로
팝업 없이 바로 메신저 페이지를 연다.

즉:
홈 → 메신저 → 바로 페이지

중요:
- 대화/공지 탭 구조 유지
- 기존 메신저 기능 깨지지 않게
- 이번 작업은 메신저 기능 수정이 아니라 진입 구조만 반영

==================================================
[5. 팝업 아이콘 구조 - 공통]

메신저를 제외한 7개 메인 아이콘은
누르면 공통 팝업이 떠야 한다.

공통 팝업 규칙:
- 3열 그리드 아이콘
- 제목 표시
- 닫기 버튼
- 바깥 터치 닫기 허용
- 드래그 닫기 허용
- 필요 시 스크롤 가능
- 디자인 전부 통일

중요:
팝업은 실제 작업 공간이 아니라
“중간 분류 아이콘 선택창”이다

==================================================
[6. 8개 아이콘의 세부 분류]

--------------------------------------
[6-1. 인원관리 팝업]

세부 아이콘:
- 회원관리
- 학부모관리
- 선생님관리
- 미배정관리
- 승인관리
- 초대관리

설명:
사람/연결/초대 관련 기능을 여기로 묶는다

--------------------------------------
[6-2. 수업관리 팝업]

세부 아이콘:
- 수업스케줄
- 반관리
- 출결관리
- 수업일지
- 수강생관리
- 반이동

설명:
수업 운영 핵심 기능 묶음
선생님 모드와 강하게 연동

--------------------------------------
[6-3. 보강관리 팝업]

세부 아이콘:
- 보강대기
- 보강배정
- 보강현황
- 보강정책

설명:
결석 → 보강대기 → 보강배정 흐름을 여기로 집중

--------------------------------------
[6-4. 매출관리 팝업]

세부 아이콘:
- 월별매출
- 결제현황
- 미납관리
- 정산확인
- 수업발생
- 휴무일관리

--------------------------------------
[6-5. 데이터관리 팝업]

세부 아이콘:
- 저장공간현황
- 계정별사용량
- 카테고리별사용량
- 원본데이터삭제
- 이벤트기록
- 백업/복구정책

--------------------------------------
[6-6. 운영설정 팝업]

세부 아이콘:
- 수업운영설정
- 보강정책설정
- 권한설정
- 알림설정
- 레벨/테스트설정
- 피드백기본설정

--------------------------------------
[6-7. 플랫폼설정 팝업]

세부 아이콘:
- 구독관리
- 화이트라벨
- 워터마크
- 로고/브랜드
- 앱기본설정
- 연결서비스

==================================================
[7. 각 세부 아이콘 클릭 시 페이지 이동]

팝업 안 세부 아이콘을 누르면
그때 실제 페이지가 열린다.

예:
홈 → 수업관리 → 팝업 → 수업스케줄 → 페이지

예:
홈 → 인원관리 → 팝업 → 회원관리 → 페이지

중요:
- 팝업 단계에서는 리스트/데이터 길게 안 보여줌
- 실제 데이터는 페이지에서만

==================================================
[8. 페이지 헤더 통일]

모든 페이지는 공통 헤더 규칙을 따른다.

공통 헤더:
- 왼쪽: 뒤로가기
- 오른쪽: 홈
- 수정/입력/설정 화면이면 저장 버튼 추가

예시:
왼쪽 ←
오른쪽 홈 / 저장

규칙:
- 뒤로가기 = 바로 이전 단계
- 홈 = 관리자 홈 첫 화면
- 저장 가능한 화면은 저장 버튼 항상 있음

중요:
저장 가능한데 저장 버튼 안 보이는 화면 금지

==================================================
[9. 리스트/카드/상태칩 규칙 유지]

기존 리스트형 페이지는 유지하되
아래 규칙으로 통일

- 카드 크기 컴팩트
- 선택 기능 유지
- 삭제 기능 유지
- 검색창 상단 고정
- 상태칩은 2줄 컴팩트 구조
- 한눈에 보기 쉽게 정리

즉:
페이지 안 리스트 기능은 유지하되
헤더/검색/칩/카드 스타일만 통일

==================================================
[10. 기존 기능 보호]

이번 작업은 “전체 재배치”지만
아래 기능은 하나도 깨지면 안 된다.

1. 회원관리 / 학부모관리 / 선생님관리
2. 승인 / 미배정 / 초대 발송
3. 수업스케줄 / 반관리 / 출결 / 일지 / 반배정 / 반이동
4. 보강대기 / 보강배정 / 보강현황
5. 메신저 대화 / 공지 / 자동 알림
6. 매출 / 정산 / 수업발생 / 휴무일
7. 저장공간 / 킬스위치 / 이벤트기록
8. 권한설정 / 구독 / 화이트라벨

기능 로직 유지
진입 구조만 바꿀 것

배너에 4개 항목(데이터 사용량, 이번 달 전체 매출, 이번 달 전체 회원, 미처리 보강)이 보이는지 확인

테스트 11.
홈 버튼을 누르면 언제나 관리자 홈 첫 화면으로 돌아오는지 확인

뒤로가기는 이전 단계로만 이동하는지 확인
