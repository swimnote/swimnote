[스윔노트 — 전화 기능 최종 적용 지시문]

이번 작업은 전화 기능을 실제 앱 전반에 최종 적용하는 작업이다.
출결/기타보강/다른 UX 작업은 건드리지 말고, 이번에는 전화 기능만 정확히 반영한다.

━━━━━━━━━━━━━━━━━━

1. 핵심 원칙
   ━━━━━━━━━━━━━━━━━━

* 전화번호가 보이는 곳이면 어디서든 즉시 전화 가능해야 한다
* 전화번호는 정보가 아니라 행동이다
* 아이콘만 사용하고, 항상 전화번호 왼쪽에 고정한다
* 전화번호 텍스트도 클릭 가능해야 한다
* 중간 확인창 없이 바로 기기 전화 앱으로 연결한다

━━━━━━━━━━━━━━━━━━
2. 공통 유틸 적용
━━━━━━━━━━━━━━━━━━

새 유틸 또는 기존 유틸 정리:

* `callPhone(phone)`

  * 숫자 외 문자 제거
  * 유효 번호일 때만 `tel:` 스킴 실행
  * 확인 단계 없음
  * 잘못된 번호면 실행 차단

* `formatPhone(phone)`

  * 예: `01012345678` → `010-1234-5678`

* `isValidPhone(phone)`

  * 숫자만 추출 후 10~11자리일 때만 true

* `CALL_COLOR`

  * `#F97316`

━━━━━━━━━━━━━━━━━━
3. UI 규칙
━━━━━━━━━━━━━━━━━━

[유효한 번호]

* 전화 아이콘 색상: `#F97316`
* 전화번호 텍스트도 같은 강조색 또는 명확한 클릭 가능 스타일
* 아이콘 클릭 가능
* 번호 클릭 가능

[유효하지 않은 번호]

* 회색 비활성 처리
* 클릭 불가
* 빈 값이면 숨기거나 `-` 처리

[배치]

* 전화 아이콘은 항상 번호 왼쪽
* 아이콘만 사용
* `전화하기` 같은 텍스트 버튼 금지

예:
📞 010-1234-5678

━━━━━━━━━━━━━━━━━━
4. 적용 범위
━━━━━━━━━━━━━━━━━━

전화번호가 노출되는 모든 화면에 동일 적용:

* `MemberCard`
* `MemberParentTab`
* 선생님 학생 프로필
* 관리자 학부모 관리 화면
* 회원목록에서 전화번호가 보이는 위치
* 앞으로 전화번호가 추가되는 모든 UI

중요:

* 화면마다 다르게 구현하지 말고 공통 규칙으로 맞출 것

━━━━━━━━━━━━━━━━━━
5. 학생 프로필 학부모 정보 노출
━━━━━━━━━━━━━━━━━━

학생 프로필에는 반드시 아래 정보 표시:

* 학부모 이름
* 학부모 전화번호
* 전화 아이콘

정책:

* 학생당 학부모 1명만 연결
* 부모 여러 명 구조 사용 안 함

데이터 연결:

* `student_id → parent_students → parent_accounts`

━━━━━━━━━━━━━━━━━━
6. 동작 규칙
━━━━━━━━━━━━━━━━━━

* 아이콘 클릭 → 전화 앱 실행
* 번호 클릭 → 동일하게 전화 앱 실행
* 복사 단계 없음
* 메뉴 안에 숨김 금지
* 별도 설명 문구 금지

━━━━━━━━━━━━━━━━━━
7. 테스트
━━━━━━━━━━━━━━━━━━

반드시 실제 확인:

1. 정상 번호 클릭 시 전화 앱 실행
2. 아이콘 클릭 / 번호 클릭 동일 동작
3. 하이픈 포함 번호도 정상 동작
4. 잘못된 번호는 실행 차단
5. 빈 번호는 UI 깨지지 않음
6. 학생 프로필에서 학부모 이름/번호 정상 표시
7. 관리자/선생님 화면 모두 동일 동작

━━━━━━━━━━━━━━━━━━
8. 보고 형식
━━━━━━━━━━━━━━━━━━

반드시 포함:

* 수정 파일 목록
* 전화 기능 적용 화면 목록
* 실제 클릭 테스트 결과
* 유효/무효 번호 처리 결과
* 학생 프로필 학부모 정보 노출 결과
* 실제 전화 앱 호출 여부

“적용 완료”만 쓰지 말고 실제 동작 결과 같이 제출
