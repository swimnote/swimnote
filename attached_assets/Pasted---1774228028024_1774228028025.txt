이번 작업은 기능 구현이 아니라
“학부모 다자녀 기능 검증을 위한 테스트 데이터 추가 및 연결 작업”이다.
반드시 실제 데이터에 반영하고, 화면에서 바로 확인 가능한 상태로 만들어라.
UI만 바꾸거나 더미 데이터로 보이게만 하지 말고,
실제 로그인 후 4/444444 학부모 계정에서 바로 확인 가능해야 한다.

중요:
- 기존 기능 절대 깨지지 않게 할 것
- 기존 연결 구조 유지할 것
- 기존 서태웅 데이터 보존할 것
- 새 자녀를 실제 연결된 학생으로 추가할 것
- 학부모 모드 탭 전환 검증이 가능하도록 즉시 반영할 것

- 학부모 모드에서 다자녀 탭 전환 테스트 가능하게 만들기

2. 추가할 학생 정보
==================================================

새로 추가할 자녀:
- 이름: 서순신
- 관계: 동일 학부모 계정의 자녀
- 나이 기준: 서태웅보다 2살 어리게 설정

중요:
- 생년월일/나이 계산은 현재 서태웅 데이터 기준으로 정확히 2살 어리게 생성
- 단순 텍스트 표기만 하지 말고 실제 학생 데이터로 생성
- 기존 서태웅 데이터와 형제관계처럼 같은 학부모 계정에 연결

==================================================
3. 연결 방식
==================================================

서순신 학생은 아래 조건으로 생성/연결한다.

- 학부모 계정 4/444444에 즉시 연결
- 별도 승인 대기 없이 바로 연결
- 학생 레코드 실제 생성
- 학부모 자녀 목록에 즉시 반영
- 학부모 모드 상단 자녀 탭에 “서태웅”, “서순신” 둘 다 보여야 함

즉:
현재 승인된 학부모 계정의 추가 자녀 등록 규칙에 맞게
자동 연결 구조로 반영하라.

==================================================
4. 학생 상태 설정
==================================================

서순신은 테스트 확인이 가능하도록 아래 상태로 넣는다.

- 학부모 계정에는 연결 완료
- 수업 배정 상태는 “미배정”
- 관리자/선생님 화면의 미배정 학생 목록에도 보여야 함

중요:
- 학부모 화면에서는 자녀로 바로 보이고
- 관리자/선생님 화면에서는 미배정 학생으로 바로 보여야 한다

==================================================
5. 학부모 모드 확인 가능 상태로 만들 것
==================================================

4/444444 계정으로 로그인했을 때 학부모 모드에서 아래가 보여야 한다.

- 상단 또는 해당 위치의 자녀 탭 목록에
  - 서태웅
  - 서순신
- 두 학생 이름 탭을 누르면 전환 가능
- 서순신은 미배정 상태 안내가 보여야 함
- 서태웅 데이터와 서순신 데이터가 섞이지 않아야 함

서순신은 새로 생성된 학생이므로
초기 데이터가 없다면 아래처럼 보여도 된다.
- 피드백 없음
- 출결 없음
- 사진 없음
- 수업 미배정 안내

하지만 학생 자체는 정상 연결되어 있어야 한다.
