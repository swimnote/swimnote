[스윔노트 UI 안정화 전용 절대 규칙 지시문 - 기능 보호 + 레이아웃 고정]

이 작업은 “UI 안정화 전용 작업”이다.
기능 추가, 로직 수정, 데이터 구조 변경 절대 금지.
현재 연결된 모든 기능, API, 상태 흐름은 절대 건드리지 마라.

이 지시문에서 허용되는 작업은 오직 아래 4개뿐이다:
- padding 조정
- margin 조정
- flex/grid 정렬 수정
- width/height 반응형 조정

그 외:
- 함수 수정 금지
- API 호출 수정 금지
- 상태(state) 수정 금지
- DB 구조 수정 금지
- 이벤트 핸들러 수정 금지
- 네비게이션 구조 수정 금지

이 규칙 위반 시 작업 실패로 간주한다.

==================================================
1. 전체 UI 안정화 목표
==================================================

목표:
- 모든 화면에서 카드/섹션 겹침 제거
- 화면 잘림 제거
- 스크롤 정상화
- safe area 침범 제거
- header/tab 가림 현상 제거

“예쁘게”가 아니라 “안 깨지게” 만드는 작업이다.

==================================================
2. 자동 UI 오류 탐지 기준 (반드시 이 기준으로 점검)
==================================================

모든 페이지를 순회하면서 아래 조건을 검사하고 수정한다:

1. 카드가 서로 겹치면 오류
2. 카드가 화면 밖으로 잘리면 오류
3. 텍스트가 잘리거나 줄이 깨지면 오류
4. 버튼이 가려지거나 클릭 안 되면 오류
5. 스크롤이 2중으로 생기면 오류
6. 상단 header에 가려지면 오류
7. 하단 safe area에 UI가 겹치면 오류
8. 카드 높이가 내용보다 작아서 넘치면 오류
9. flex shrink 때문에 찌그러지면 오류
10. overflow hidden 때문에 내용 잘리면 오류

각 화면:
- 문제 발견 → 원인 분석 → 최소 수정

==================================================
3. 수정 방법 강제 규칙
==================================================

허용:
- padding 증가/감소
- margin 정리
- flex 방향 수정
- align / justify 수정
- gap 적용
- min-height 적용
- overflow scroll/auto 정리

금지:
- position absolute로 억지 해결 ❌
- z-index로 덮기 ❌
- height 고정값으로 막기 ❌
- transform 이동 ❌

원칙:
👉 항상 “흐름(layout flow)”으로 해결

==================================================
4. 공통 레이아웃 고정 규칙
==================================================

모든 화면에 동일하게 적용:

[상단]
- header 높이 고정
- header 아래 content padding-top 확보

[본문]
- 모든 페이지는 단일 스크롤 구조만 허용
- 이중 스크롤 금지

[카드]
- 카드 간격 일정 (gap or margin 통일)
- 카드 min-height 확보
- 내부 padding 통일

[하단]
- safe area padding-bottom 확보
- 마지막 요소 잘림 방지

==================================================
5. 카드/섹션 충돌 방지 규칙
==================================================

- 카드끼리 겹치면 안 됨
- 다음 카드가 위로 올라오면 margin-bottom 추가
- grid 사용 시 row gap 반드시 적용
- flex column 사용 시 gap 적용

==================================================
6. 스크롤 구조 규칙
==================================================

- 전체 페이지는 1개의 scroll container만 허용
- 내부 카드에 scroll 금지 (특수 경우 제외)
- overflow hidden 남용 금지

==================================================
7. 텍스트 안정화 규칙
==================================================

- 모든 텍스트 줄바꿈 허용
- ellipsis는 필요한 경우만 사용
- line-height 일정하게 유지
- 버튼 텍스트 잘림 금지

==================================================
8. 팝업/모달 규칙
==================================================

- 팝업은 화면 위에 떠야 함 (레이어 분리)
- 배경 스크롤 차단
- 팝업이 화면 밖으로 나가면 안 됨

==================================================
9. “절대 수정 금지 영역” (가장 중요)
==================================================

다음은 절대 수정 금지:

- API 호출 코드
- 상태 관리 코드 (useState, store 등)
- 데이터 로딩 로직
- 이벤트 핸들러
- 네비게이션
- 결제 로직
- 복구 로직
- 승인 로직

👉 UI 수정 중 이 영역 건드리면 즉시 중단

==================================================
10. 수정 범위 제한
==================================================

- 공통 컴포넌트 수정 시 반드시 모든 화면 영향 확인
- 특정 화면 수정 시 다른 화면 절대 깨지지 않게 유지

==================================================
11. 작업 방식
==================================================

모든 화면을 하나씩 순회:

1. 슈퍼관리자 콘솔
2. 운영 관리
3. 매출/정산
4. 데이터 관리
5. 고객센터
6. 보안/통제
7. 시스템 상태
8. 수영장 관리자 화면
9. 선생님 화면
10. 학부모 화면

각 화면:
- 오류 찾기
- 최소 수정
- 다시 확인

==================================================
12. 검증 필수
==================================================

수정 후 반드시 확인:

- 카드 겹침 없음
- 잘림 없음
- 버튼 클릭 가능
- 스크롤 정상
- header 가림 없음
- safe area 정상
- 텍스트 깨짐 없음

==================================================
13. 작업 완료 보고 규칙
==================================================

반드시 아래 형식으로 보고:

1. 수정한 화면 목록
2. 수정 내용 요약
3. 해결된 UI 오류 목록
4. 건드리지 않은 기능 목록 확인
5. 아직 남은 UI 문제 (있으면)

거짓 보고 금지

==================================================
14. 최종 목표
==================================================

- 기능은 절대 그대로 유지
- UI만 안정화
- 이후 디자인 수정해도 기능 안 깨지게 기반 고정

👉 이 작업은 “기능 보호 상태에서 UI 틀 고정”이 목적이다
