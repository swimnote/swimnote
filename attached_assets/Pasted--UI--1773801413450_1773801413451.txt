지금부터 앱 전체 UI 구조 통일 작업을 시작한다.
목표는 네 가지 모드(슈퍼관리자 / 관리자 / 선생님 / 학부모)를
기능은 다르지만 화면 구조와 사용감은 동일하게 만드는 것이다.

중요:
- 기존 기능 로직, 데이터 구조, 저장 구조, 승인 구조, 회원가입 구조는 최대한 건드리지 말 것
- 이번 작업은 “화면 구조 통일”이 핵심이다
- 기능 추가 금지
- 기능 삭제 금지
- 내용 임의 변경 금지
- 버튼 연결 끊기 금지
- 공통 레이아웃/공통 UI 컴포넌트로 감싸는 방식으로 통일할 것
- 수정 후 반드시 실제 테스트까지 수행하고 결과를 상세히 보고할 것

==================================================
[1. 이번 작업 목표]

네 가지 모드 전부 아래 원칙으로 통일한다.

1) 상단 헤더 구조 통일
2) 홈 화면 구조 통일
3) 리스트 화면 구조 통일
4) 상세 화면 구조 통일
5) 팝업 구조 통일
6) 뒤로가기 / 홈 / 탭 이동 규칙 통일
7) safe area / iPhone 표시 위치 통일

핵심:
- 기능은 다를 수 있어도
- 화면 구조와 이동 방식은 같아야 한다

==================================================
[2. 공통 철학]

이번 작업의 기준은 아래와 같다.

- 슈퍼관리자도 같은 구조
- 관리자도 같은 구조
- 선생님도 같은 구조
- 학부모도 같은 구조

즉:
“모드마다 UI 느낌이 다르면 실패”
“기능만 다르고 구조는 같아야 성공”

==================================================
[3. 전 모드 공통 화면 계층]

모든 화면은 아래 4단계 구조로 통일한다.

1. 홈(Home)
2. 리스트(List)
3. 상세(Detail)
4. 실행(Action / Popup)

화면 흐름 예시:
홈 → 리스트 → 상세 → 팝업

이 구조를 전 모드에 적용한다.

==================================================
[4. 공통 상단 헤더 구조]

모든 일반 화면의 상단 헤더는 동일한 틀을 사용한다.

--------------------------------------
[4-1. 헤더 기본 구성]

좌측:
- 뒤로가기 버튼 (하위 화면에서만)
- 첫 화면에서는 뒤로가기 없음

중앙:
- 화면 제목

우측:
- 홈 버튼 또는 화면별 액션 버튼
- 아이콘 개수는 최소화

--------------------------------------
[4-2. 헤더 규칙]

1. 제목은 항상 중앙 정렬
2. 뒤로가기 위치는 모든 화면에서 동일
3. 홈 버튼 위치도 동일
4. safe area 고려해서 상단 시스템 UI와 겹치지 않게
5. 화면마다 헤더 높이/패딩 다르면 안 됨

==================================================
[5. 홈 화면 구조 통일]

네 가지 모드의 첫 화면은 모두 아래 구조를 따른다.

--------------------------------------
[5-1. 홈 공통 구조]

A. 상단 요약 카드
- 현재 상태
- 오늘 할 일
- 중요한 숫자

B. 주요 기능 버튼 영역
- 자주 쓰는 핵심 기능만 배치

C. 최근/빠른 이동 영역
- 최근 항목
- 빠른 진입 카드

--------------------------------------
[5-2. 홈 공통 원칙]

1. 첫 화면에서 길을 잃으면 안 된다
2. 가장 중요한 정보가 위
3. 자주 쓰는 기능이 중간
4. 부가 정보는 아래
5. 한 화면에 버튼이 너무 많아지면 안 된다

--------------------------------------
[5-3. 모드별 홈 내용은 다르되 구조는 같게]

슈퍼관리자 홈:
- 전체 플랫폼 현황
- 승인/구독/용량/수영장 요약

관리자 홈:
- 회원/출결/선생님/승인 현황

선생님 홈:
- 오늘 수업
- 출결 미확인
- 일지 미작성
- 보강 현황

학부모 홈:
- 자녀 현재 상태
- 최근 수업피드백
- 바로가기(앨범/출결/레벨/공지)

구조는 같고 내용만 다르게 배치할 것

==================================================
[6. 리스트 화면 구조 통일]

사람 목록, 회원 목록, 수영장 목록, 반 목록, 승인 목록 등
모든 리스트 화면은 아래 구조로 통일한다.

--------------------------------------
[6-1. 리스트 공통 구조]

1. 화면 제목
2. 검색창
3. 필터/탭 칩 영역
4. 카드형 리스트

--------------------------------------
[6-2. 리스트 공통 규칙]

1. 검색창은 항상 상단 고정
2. 필터칩은 검색창 바로 아래
3. 리스트 카드는 동일한 여백/둥근 모서리/정렬 사용
4. 카드 안 정보 순서는 통일:
   - 이름/제목
   - 상태
   - 보조 정보
   - 우측 이동 아이콘 또는 액션
5. 리스트 구조가 화면마다 달라지면 안 됨

--------------------------------------
[6-3. 적용 대상 예시]

- 슈퍼관리자 수영장 관리
- 관리자 사람/회원/선생님/승인
- 선생님 내 반/학생 목록
- 학부모 공지/피드백/앨범 리스트

==================================================
[7. 상세 화면 구조 통일]

모든 상세 화면은 아래 구조를 따른다.

--------------------------------------
[7-1. 상세 공통 구조]

1. 상단 기본 정보 카드
   - 이름 / 상태 / 핵심 정보

2. 주요 액션 버튼 영역
   - 수정 / 승인 / 이동 / 저장 등

3. 세부 정보 섹션
   - 섹션별 카드나 블록

--------------------------------------
[7-2. 상세 공통 규칙]

1. 정보 → 상태 → 행동 → 상세정보 순서 유지
2. 상단 카드 형태 통일
3. 섹션 제목 스타일 통일
4. 버튼 위치/간격 통일
5. 같은 앱인데 다른 페이지처럼 보이면 안 됨

==================================================
[8. 팝업 구조 통일]

앱 전체 모든 팝업은 같은 구조를 사용한다.

--------------------------------------
[8-1. 팝업 최종 규칙]

- 하단에서 올라오는 Bottom Sheet
- 높이: 화면의 75% 고정
- 모든 팝업 같은 위치 / 같은 크기 / 같은 패턴
- 제목 가운데 정렬
- 우측 상단 X 버튼
- 바깥 터치 닫기
- 아래로 스와이프 닫기

--------------------------------------
[8-2. 팝업 내부 구조]

[상단]
- 드래그 핸들
- 제목
- X 버튼

[중간]
- 내용 영역 (스크롤 가능)

[하단]
- 고정 버튼 영역

--------------------------------------
[8-3. 팝업 규칙]

1. 내용이 많으면 내부 스크롤
2. 버튼은 항상 하단 고정
3. 키보드가 팝업 내용을 가리면 안 됨
4. 팝업마다 위치/크기 다르면 안 됨

==================================================
[9. 네비게이션 규칙 통일]

전 모드 동일하게 아래 규칙 적용

--------------------------------------
[9-1. 뒤로가기]

- 현재 탭 내부의 이전 화면으로만 이동
- 다른 탭으로 넘어가면 안 됨
- 팝업 열려 있으면 팝업 먼저 닫기

--------------------------------------
[9-2. 홈 버튼]

- 앱 전체 홈이 아니라
- 현재 탭의 첫 화면으로 이동

예:
- 수업 하위 상세 → 홈 누르면 수업 탭 첫 화면
- 더보기 하위 설정 → 홈 누르면 더보기 첫 화면

--------------------------------------
[9-3. 탭 이동]

- 다른 탭으로 가려면 하단 탭을 눌러야 함
- 탭 이동 시 그 탭의 첫 화면으로 이동
- 탭별 하위 경로는 유지하지 않음

==================================================
[10. 하단 탭 구조 원칙]

이번 단계에서는 탭 배열 대공사는 하지 말고,
우선 각 모드 안에서 “같은 느낌”이 나도록 정리한다.

원칙:
1. 탭 수는 가능하면 현재 구조 유지
2. 첫 번째 탭은 항상 홈 역할
3. 자주 쓰는 기능은 앞쪽
4. 설정/더보기는 뒤쪽
5. 탭 아이콘/라벨 스타일 통일

즉:
대규모 탭 재배치는 보류 가능
하지만 탭 구조 감성은 통일할 것

==================================================
[11. safe area / iPhone 대응]

모든 화면에 아래 적용

1. 상단 헤더 safe area 확보
2. 하단 탭 safe area 확보
3. 뒤로가기/홈 버튼이 iPhone 상단 시스템 UI와 겹치지 않게
4. 팝업과 키보드가 home indicator 영역 침범하지 않게

==================================================
[12. 구현 방식]

반드시 아래 방식으로 작업할 것

1. 공통 ScreenLayout 컴포넌트 생성
   포함:
   - safe area
   - header
   - title
   - back button
   - home button
   - body container

2. 공통 ListPageLayout 생성
   포함:
   - search
   - filter chips
   - list body

3. 공통 DetailPageLayout 생성
   포함:
   - top summary card
   - action row
   - section blocks

4. 공통 ModalSheet 생성
   포함:
   - 75% height
   - centered title
   - close button
   - scroll body
   - bottom fixed action buttons

기존 화면은
새 공통 레이아웃으로 감싸는 방식으로 적용하고
기존 기능 코드는 최대한 유지할 것

==================================================
[13. 작업 우선순위]

이번 작업은 한 번에 전부 뜯어고치지 말고
아래 우선순위대로 적용한다.

1순위:
- 슈퍼관리자 홈/리스트/상세 공통화
- 관리자 홈/리스트/상세 공통화
- 선생님 홈/리스트/상세 공통화
- 학부모 홈/리스트/상세 공통화

2순위:
- 팝업 공통화

3순위:
- 세부 버튼 위치/간격 미세 조정

중요:
탭 배열 재배치는 이번 단계에서 최소화하거나 보류 가능

==================================================
[14. 절대 금지]

- 기능 로직 수정 금지
- 데이터 구조 수정 금지
- 회원가입/승인/출결/일지/정산 로직 수정 금지
- 임의로 문구 변경 금지
- 버튼 기능 끊기 금지
- 테스트 로그인 삭제 금지
- 모드 진입 삭제 금지

이번 작업은 UI 구조 통일이다.
기능 개발이 아니다.

==================================================
[15. 테스트 지시 - 반드시 실제 수행]

수정 후 아래 테스트를 전 모드에서 수행하라.

테스트 1.
슈퍼관리자 / 관리자 / 선생님 / 학부모 각각 정상 진입되는지 확인

테스트 2.
각 모드 첫 화면이 요약 + 주요기능 + 빠른이동 구조인지 확인

테스트 3.
각 리스트 화면이 검색 + 필터 + 카드리스트 구조로 통일됐는지 확인

테스트 4.
각 상세 화면이 상단 정보카드 + 액션 + 세부정보 구조인지 확인

테스트 5.
팝업이 75% 하단 시트 구조로 통일됐는지 확인

테스트 6.
뒤로가기 / 홈 / 탭 이동 규칙이 동일하게 적용되는지 확인

테스트 7.
iPhone safe area 겹침이 없는지 확인

테스트 8.
기존 기능 연결이 깨지지 않았는지 확인

==================================================

[17. 작업 범위 한정]

이번 작업은
“네 가지 모드 전부 헷갈리지 않게 같은 구조로 조정”
하는 UI 통일 작업이다.

기능은 건드리지 말고
화면 구조만 통일할 것.
