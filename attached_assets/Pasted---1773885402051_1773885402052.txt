지금부터 업무 메신저를 다시 정확하게 완성하라.

현재 상태 문제:
- 관리자와 선생님 모두 실제 채팅 입력창이 생성되지 않았다
- 대화 탭/공지 탭 구조는 보이지만 실제 카카오톡형 하단 채팅 입력 환경이 완성되지 않았다
- 사용자가 요구한 조건이 끝까지 반영되지 않았다

이번 작업은 부분 수정이 아니라
“대화 탭 + 공지 탭 + 실제 입력창 + 키보드 대응 + 공지 스타일 + 자동메시지 잠금”
을 한 번에 정확히 완성하는 작업이다.

중요:
- 관리자와 선생님 둘 다 대화 탭에서 실제 채팅 입력 가능해야 한다
- 공지 탭은 노란색 계열 강조 스타일로 보여야 한다
- 공지 탭의 자동 메시지는 모두 수정 불가여야 한다
- 키보드가 입력창/전송 버튼/마지막 메시지를 절대 가리면 안 된다
- 이번에는 가짜 UI 금지, 실제 채팅 입력/전송이 동작해야 한다

==================================================
[1. 메신저 최종 구조 다시 고정]

메신저 탭 내부 상단 탭 2개:
- 대화
- 공지

기본 진입:
- 항상 대화 탭

메신저 탭 진입 시:
- 항상 최신 상태 새로고침
- 항상 대화 탭 먼저 열기
- 마지막 메시지 보이게 아래쪽으로 위치

==================================================
[2. 대화 탭 - 실제 채팅 기능 완성]

대화 탭은 관리자와 선생님이 모두 실제로 쓰는 업무 채팅창이다.

반드시 구현할 것:
- 관리자 입력 가능
- 선생님 입력 가능
- 실제 입력창 생성
- 실제 전송 기능 동작
- 전송 후 본문에 즉시 반영
- 저장 후 재진입 시 유지

==================================================
[3. 대화 탭 UI - 카카오톡형으로 정확히]

하단 입력창 영역을 카카오톡과 유사하게 구성하라.

하단 입력창 구성:
좌측:
- + 버튼

중앙:
- 둥근 메시지 입력창
- placeholder: "메시지 입력"

우측:
- 이모지 버튼
- 전송 버튼
(또는 현재 구조상 우측 고정 액션 1개로 정리 가능하나, 카카오톡 느낌 유지)

중요:
- 하단 입력창은 항상 고정
- 메시지 리스트와 분리된 하단 고정 영역이어야 함
- 관리자/선생님 둘 다 동일 구조 사용

==================================================
[4. 키보드 대응 - 최우선]

이 작업에서 가장 중요하다.

반드시 아래 조건 모두 만족시켜라.

1. 입력창을 누르면 키보드가 반드시 정상적으로 올라온다
2. 키보드가 올라오면 입력창이 키보드 바로 위로 같이 올라간다
3. 입력창이 키보드에 가려지면 안 된다
4. 전송 버튼이 키보드에 가려지면 안 된다
5. 마지막 메시지가 키보드 뒤에 숨으면 안 된다
6. 입력 중 화면이 흔들리거나 깨지면 안 된다
7. 작은 아이폰 화면에서도 동일하게 동작해야 한다

즉:
카카오톡처럼 “키보드 위에 입력창이 딱 붙는 구조”로 구현할 것

==================================================
[5. 메시지 레이아웃 보정]

대화 탭 메시지 레이아웃도 정리하라.

반드시:
- 내 메시지: 오른쪽 정렬
- 상대/다른 사용자 메시지: 왼쪽 정렬
- 프로필/이름/시간 위치 자연스럽게 정리
- 말풍선 간격 균일
- 날짜 구분선 가운데 정렬

현재처럼 정렬이 어색하거나
메시지 단위가 보기 불편하면 카카오톡형 간격으로 다시 맞출 것

==================================================
[6. 공지 탭 규칙 다시 확정]

공지 탭은
- 관리자 공지
- 시스템 자동 메시지
를 보여주는 탭이다.

권한:
- 관리자만 작성 가능
- 선생님은 읽기 전용
- 선생님 입력창 노출 금지
- 선생님 댓글/채팅 금지

==================================================
[7. 공지 탭 스타일 - 노란색 적용]

공지 탭 메시지는 일반 대화 탭과 시각적으로 확실히 달라야 한다.

반드시:
- 공지 메시지/공지 카드/공지 말풍선은 노란색 계열 사용
- 카카오톡 알림/공지 느낌 나게 시각적 강조
- 대화 탭의 일반 메시지와 구분이 바로 되게 할 것

즉:
공지 탭은 노란색 기반으로 통일

==================================================
[8. 공지 탭 자동 메시지 규칙]

자동 메시지로 올라오는 건 딱 2개만:
- 이동
- 보강

올리지 않는 것:
- 퇴원
- 기타 과한 시스템 메시지

자동 메시지 예:
- 서태웅 회원이 월 15:00반에서 화 16:00반으로 이동되었습니다.
- 김민수 회원 보강 수업이 3월 20일 16:00에 배정되었습니다.

==================================================
[9. 자동 메시지 수정 불가 - 반드시 적용]

공지 탭의 자동 메시지는 모두 수정 불가여야 한다.

규칙:
- 자동 메시지(system_move / system_makeup)는 편집 금지
- 삭제도 일반 사용자에게 금지
- 관리자도 수정 불가로 두고, 시스템 기록처럼 유지
- 일반 공지 메시지와 시스템 메시지를 구분해서 처리

중요:
자동 생성된 공지/소식은 변경되지 않는 기록성 메시지다

==================================================
[10. 공지 탭 안읽음 표시]

공지 탭에 읽지 않은 항목이 있으면
공지 탭 라벨 옆에 작은 점 표시

규칙:
- 점 형태 배지
- 선생님/관리자 모두 적용 가능
- 공지 탭 들어가서 읽으면 제거

==================================================
[11. 대화 탭 입력 상태]

대화 탭에서 입력 중일 때
공지 탭으로 전환했다가 다시 돌아오면

우선순위:
1. 입력 중인 텍스트 보존
2. 최소한 입력창 위치/상태 정상 유지
3. 레이아웃 깨짐 금지

공지 탭 전환 시 키보드 내려가도 괜찮다
하지만 대화 탭으로 돌아오면 입력창이 정상적으로 다시 보여야 한다

==================================================
[12. 데이터 구조]

message_type 구분을 확실히 하라.

예:
- normal
- notice
- system_move
- system_makeup

노출 규칙:
대화 탭:
- normal

공지 탭:
- notice
- system_move
- system_makeup

자동 메시지는 수정 불가 속성 포함

==================================================
[13. 현재 문제 먼저 찾을 것]

수정 전에 아래를 먼저 확인하라.

1. 왜 관리자/선생님 둘 다 입력창이 렌더링되지 않았는지 원인 분석
2. 입력창 컴포넌트가 숨겨졌는지, 권한 조건이 잘못됐는지 확인
3. 키보드 대응 wrapper/KeyboardAvoidingView/safe area 구조 확인
4. 하단 고정 입력바가 탭/웹뷰 바에 가려지는지 확인
5. 공지 탭 style 분기가 적용됐는지 확인
6. 자동 메시지 편집 가능 상태가 남아 있는지 확인

이 원인을 먼저 찾고 그 다음 구현 시작

==================================================
[14. 테스트 - 반드시 실제 수행]

테스트 1.
관리자 계정으로 메신저 > 대화 탭 진입
→ 실제 입력창 보이는지 확인

테스트 2.
선생님 계정으로 메신저 > 대화 탭 진입
→ 실제 입력창 보이는지 확인

테스트 3.
관리자 메시지 입력/전송 정상 동작 확인

테스트 4.
선생님 메시지 입력/전송 정상 동작 확인

테스트 5.
아이폰 기준 키보드 올라와도 입력창/전송 버튼/마지막 메시지가 안 가려지는지 확인

테스트 6.
대화 탭 ↔ 공지 탭 전환 정상 확인

테스트 7.
공지 탭에서 관리자만 입력 가능하고 선생님은 입력창이 안 보이는지 확인

테스트 8.
공지 탭 메시지가 노란색 스타일로 보이는지 확인

테스트 9.
회원 이동 발생 시 공지 탭 자동 메시지 생성 확인

테스트 10.
보강 등록/배정 발생 시 공지 탭 자동 메시지 생성 확인

테스트 11.
자동 메시지가 편집/수정 불가인지 확인

테스트 12.
공지 탭 안읽음 점 배지 표시/해제 확인

==================================================
[15. 작업 후 보고 형식]

작업 후 아래 순서로 보고하라.

1. 수정 파일 목록
2. 입력창이 안 보이던 원인
3. 관리자/선생님 대화 입력창 구현 방식
4. 키보드 간섭 해결 방식
5. 공지 탭 노란색 스타일 적용 방식
6. 자동 메시지 수정 불가 처리 방식
7. 이동/보강 자동 메시지 연결 방식
8. 테스트 1~12 결과
9. 아직 남은 문제 있으면 명시

==================================================
[16. 절대 금지]

- 관리자/선생님 둘 중 하나라도 입력창 누락 금지
- 키보드가 입력창 가리는 상태로 완료 보고 금지
- 공지 탭을 일반 채팅색으로 두는 것 금지
- 자동 메시지 편집 가능 상태로 남겨두는 것 금지
- 가짜 입력 UI만 만들고 실제 저장/전송 안 되는 상태 금지

이번 작업은
업무 메신저를 실사용 가능한 수준으로 완성하는 작업이다.
관리자와 선생님 모두 대화 탭에서 실제 채팅 입력/전송이 가능해야 하고,
공지 탭은 노란색 기반으로 관리자 공지와 이동/보강 자동 메시지만 보이며,
자동 메시지는 모두 수정 불가여야 한다.
