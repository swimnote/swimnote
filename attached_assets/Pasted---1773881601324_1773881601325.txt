지금부터 메신저 기능을 처음부터 구조적으로 다시 정리해서 구현하라.

이번 작업은 부분 수정이 아니라
메신저를 내부 업무용 협업 메신저로 확정하는 작업이다.

중요:
- 한 번에 구조를 제대로 잡아라
- 중간 임시 수정 금지
- 키보드가 채팅창/입력창에 간섭하지 않게 하는 것이 최우선이다
- UI는 카카오톡처럼 익숙하게 만든다
- 기능은 관리자/선생님 내부 업무용으로만 동작한다
- 학부모는 메신저 사용 대상이 아니다

==================================================
[1. 메신저 최종 구조]

메신저 탭 안에 상단 탭 2개를 둔다.

상단 탭:
1. 대화
2. 공지

기본 진입 탭:
- 항상 대화 탭

규칙:
- 메신저 탭 진입 시 항상 대화 탭이 먼저 열릴 것
- 메신저 탭 진입 시 항상 새로고침해서 최신 상태를 반영할 것
- 상단 탭 누르면 즉시 전환
- 별도 페이지 push 느낌이 아니라 같은 화면 안에서 탭 전환 느낌으로 처리할 것

==================================================
[2. 대화 탭]

대화 탭은
관리자와 선생님들이 사용하는 실시간 업무 채팅창이다.

용도:
- 관리자 지시사항
- 선생님 보고사항
- 실시간 회의 느낌의 대화
- 일반 업무 채팅

권한:
- 관리자 작성 가능
- 선생님 작성 가능

UI:
- 카카오톡 채팅방과 유사하게 구현
- 좌우 말풍선
- 시간 표시
- 날짜 구분선
- 하단 입력창 고정
- 키보드 올라오면 입력창 같이 위로 올라감
- 마지막 메시지 항상 보이게 자동 스크롤

==================================================
[3. 공지 탭]

공지 탭은
업무 공지와 시스템성 소식을 모아보는 탭이다.

권한:
- 관리자만 작성 가능
- 선생님은 읽기만 가능
- 선생님은 댓글/채팅 작성 불가
- 입력창은 관리자에게만 보이게 하거나, 선생님에게는 완전히 숨길 것

공지 탭에 올라오는 내용:
1. 관리자 공지 직접 작성
2. 시스템 자동 소식

==================================================
[4. 공지 탭 자동 메시지 범위]

공지 탭에 시스템 자동 메시지로 남길 것은 아래 2개만 허용한다.

- 이동
- 보강

즉:
회원 이동 발생 시 → 공지 탭 자동 메시지
보강 등록/배정 발생 시 → 공지 탭 자동 메시지

올리지 않을 것:
- 퇴원
- 기타 과도한 시스템 알림

목표:
공지 탭이 너무 시끄럽지 않고
실제로 필요한 공지와 운영 소식만 보이게 할 것

==================================================
[5. 공지 탭 안읽음 표시]

공지 탭에 사용자가 아직 읽지 않은 공지/소식이 있으면
상단 탭의 “공지” 옆 또는 위에 작은 점 배지를 표시하라.

규칙:
- 카카오톡처럼 과하지 않은 점 표시
- 숫자 대신 점으로 먼저 구현 가능
- 공지 탭 들어가서 읽으면 점이 사라지게 처리

==================================================
[6. 메신저 첫 화면 동작]

메신저 탭을 누를 때마다 아래 규칙 적용:

1. 항상 대화 탭이 먼저 열린다
2. 최신 상태로 새로고침
3. 마지막 메시지 기준 아래쪽이 보이게
4. 입력창 레이아웃 정상 유지

즉:
메신저 탭 첫 화면 = 대화 탭 최신 상태

==================================================
[7. 상단 헤더 구조]

채팅 UI는 카카오톡 느낌으로 맞춘다.

상단 헤더:
- 좌측: 뒤로가기
- 중앙: 제목 (대화 또는 공지)
- 우측: 검색 / 메뉴

대화/공지 전환 탭은
헤더 바로 아래 배치한다.

==================================================
[8. 대화 탭 입력창 구조]

카카오톡과 유사하게 구성:

좌측:
- 플러스 버튼

중앙:
- 둥근 텍스트 입력창
- placeholder 적절히 표시

우측:
- 이모지 버튼
- 전송 버튼

중요:
- 엔터는 줄바꿈
- 전송은 버튼으로
- 입력창은 한 줄에서 여러 줄 자연 확장 가능하면 좋음

==================================================
[9. 키보드 간섭 금지 - 최우선]

이 작업에서 가장 중요한 규칙이다.

반드시 아래를 지켜라:

1. 키보드가 올라오면 입력창이 키보드 바로 위로 이동
2. 입력창이 키보드에 가려지면 안 됨
3. 마지막 메시지가 키보드에 가려지면 안 됨
4. 전송 버튼이 키보드에 가려지면 안 됨
5. 입력 중일 때 화면이 튀거나 레이아웃 깨지면 안 됨

아이폰 기준으로 반드시 확인
작은 화면에서도 문제 없어야 함

==================================================
[10. 탭 전환 시 입력 상태 처리]

대화 탭에서 입력 중일 때
공지 탭으로 이동했다가 다시 대화 탭으로 돌아오면

권장 동작:
- 입력 중이던 텍스트를 유지
또는 최소한
- 레이아웃이 깨지지 않고 입력창이 정상 복원

공지 탭은 선생님 입력 불가 구조이므로
공지 탭 전환 시 키보드는 내려가도 괜찮다

중요:
다시 대화 탭으로 돌아왔을 때
입력창 위치/상태가 이상해지면 안 된다

==================================================
[11. 데이터 구조]

메신저는 1:1 채팅이 아니라
2개의 논리 채널로 구분한다.

channel_type:
- talk
- notice

messages:
- id
- channel_type
- sender_id
- message_type
- message
- created_at
- read_state

message_type:
- normal
- notice
- system_move
- system_makeup

공지 탭에는
notice + system_move + system_makeup 표시

대화 탭에는
normal 표시

==================================================
[12. 시스템 메시지 연결]

아래 기능과 공지 탭을 연결하라.

1. 회원 이동 발생
→ 공지 탭에 자동 메시지 생성

2. 보강 등록/배정 발생
→ 공지 탭에 자동 메시지 생성

자동 메시지는 수정 불가한 시스템 메시지 형태로 표시

예:
- 서태웅 회원이 월 15:00반에서 화 16:00반으로 이동되었습니다.
- 김민수 회원 보강 수업이 3월 20일 16:00에 배정되었습니다.

==================================================
[13. 기존 기능 유지]

- 메신저 탭 위치 유지
- 뒤로가기/탭 규칙 유지
- 관리자/선생님 권한 구조 유지
- 학부모는 메신저 제외
- 기존 사용자 구조 유지

[14. 테스트 - 반드시 실제 수행]

테스트 1.
메신저 탭 진입 시 항상 대화 탭이 기본으로 열리는지 확인

테스트 2.
메신저 탭 진입 시 최신 메시지 기준으로 새로고침되는지 확인

테스트 3.
대화 탭에서 관리자 메시지 전송 정상 동작 확인

테스트 4.
대화 탭에서 선생님 메시지 전송 정상 동작 확인

테스트 5.
대화 탭에서 키보드 올라와도 입력창/전송 버튼/마지막 메시지가 안 가려지는지 확인

테스트 6.
공지 탭으로 즉시 전환되는지 확인

테스트 7.
공지 탭에서 관리자만 입력 가능하고 선생님은 입력 불가인지 확인

테스트 8.
회원 이동 발생 시 공지 탭에 자동 메시지 생성되는지 확인

테스트 9.
보강 등록/배정 발생 시 공지 탭에 자동 메시지 생성되는지 확인

테스트 10.
퇴원 발생 시 공지 탭에 자동 메시지가 올라가지 않는지 확인

테스트 11.
공지 탭 안읽음 점 배지가 정상 표시/해제되는지 확인

테스트 12.
대화 탭 입력 중 공지 탭 갔다가 다시 돌아왔을 때 입력 상태/레이아웃이 깨지지 않는지 확인

[17. 절대 금지]

- 학부모 메신저 연결 금지
- 선생님이 공지 탭에 댓글/채팅 입력 가능하게 만들기 금지
- 공지 탭에 퇴원 자동 메시지 올리기 금지
- 메신저 탭 진입 시 공지 탭이 먼저 열리게 만들기 금지
- 키보드가 입력창/마지막 메시지를 가리게 두는 것 금지
- 임시로만 보이는 가짜 탭 구현 금지

이번 작업은
메신저를 관리자/선생님 내부 업무용 협업 구조로 확정하고,
상단 2탭(대화/공지)으로 나누어
대화는 실시간 업무 채팅, 공지는 관리자 공지 + 이동/보강 자동 소식만 보이게 만드는 작업이다.
