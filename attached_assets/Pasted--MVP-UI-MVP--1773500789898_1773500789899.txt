이 앱은 토이키즈 내부용이 아니라 전국 수영장 플랫폼 테스트용 MVP입니다.

따라서 지금부터는 단순 UI나 버튼 연결이 아니라,
실제로 안정적으로 작동하는 MVP인지 전체 구조를 점검해주세요.

아래 항목을 하나씩 분석해서
“현재 상태 / 문제점 / 수정 필요 여부 / 수정 방법”
형태로 자세히 보고해주세요.

[1. 전체 아키텍처 점검]
현재 앱 구조를 설명해주세요.
- 앱 프론트엔드 구조
- API 서버 구조
- DB 구조
- 파일 저장 구조
- 권한 처리 구조
- 데이터 흐름 구조

특히 이 앱이 전국 여러 수영장이 동시에 사용하는 플랫폼 MVP로 적합한 구조인지 평가해주세요.

[2. 데이터베이스 점검]
현재 PostgreSQL에 실제로 저장되는 테이블을 모두 정리해주세요.
각 테이블별로 아래를 설명해주세요.
- 테이블명
- 용도
- 주요 컬럼
- swimming_pool_id 포함 여부
- 다른 테이블과의 연결 관계
- 현재 실제 저장되고 있는지 여부

특히 다음 핵심 기능이 영구 저장되는지 확인해주세요.
- 학생
- 반
- 출결
- 수영일지
- 사진
- 영상
- 사용자 계정
- 수영장 정보

DB 스키마가 없는 기능이 있다면 무엇인지 정확히 말해주세요.

[3. 파일 저장 구조 점검]
사진과 영상 파일은 현재 어디에 저장되는지 정확히 알려주세요.
반드시 아래를 답해주세요.
- 파일이 PostgreSQL에 직접 저장되는지
- 서버 로컬 디스크에 저장되는지
- Replit 임시 저장소인지
- 외부 스토리지인지
- 앱 재시작/배포/서버 재실행 후에도 유지되는지
- 파일 URL이 DB에 저장되는지
- 파일 삭제 시 DB와 실제 파일이 같이 정리되는지

특히 사진/영상이 전국 플랫폼 테스트용으로 충분히 안정적인 저장 구조인지 평가해주세요.
임시 저장 또는 서버 재시작 시 사라질 위험이 있으면 정확히 알려주세요.

[4. 미완성 기능 점검]
현재 API는 있지만 DB 저장 구조가 불완전하거나 없는 기능을 모두 찾아주세요.
특히 다음 기능이 완전한지 확인해주세요.
- 수영일지
- 사진 업로드
- 영상 업로드

각 기능에 대해 아래를 답해주세요.
- 버튼은 있는지
- API는 있는지
- DB 테이블은 있는지
- 실제 저장되는지
- 조회 가능한지
- 앱 재실행 후에도 유지되는지
- 다른 화면과 연동되는지

[5. 관리자-선생님-학부모 연동 점검]
다음 흐름이 실제로 정상 작동하는지 확인해주세요.

1) 관리자가 학생 등록
2) 관리자가 반배정
3) 선생님 계정에서 내 반에 즉시 반영
4) 선생님이 출결 입력
5) 선생님이 수영일지 작성
6) 선생님이 사진/영상 업로드
7) 학부모 계정에서 자기 자녀 정보만 조회

각 단계별로 아래를 알려주세요.
- 실제 연결되어 있는지
- 저장되는지
- 즉시 반영되는지
- 새로고침이 필요한지
- 연동이 끊길 가능성이 있는지

[6. 권한 분리 점검]
다음 역할별 권한이 서버 기준으로 안전하게 제한되는지 확인해주세요.
- platform_admin
- pool_admin
- teacher
- parent

각 역할마다 아래를 확인해주세요.
- 조회 가능한 데이터
- 수정 가능한 데이터
- 접근 가능한 API
- 자기 수영장 외 데이터 접근 가능 여부
- 자기 반 외 데이터 접근 가능 여부
- 자기 자녀 외 데이터 접근 가능 여부

중요:
프론트 화면 숨김만 있는 것이 아니라
서버 API 레벨에서 실제 차단되는지 확인해주세요.

[7. 멀티 수영장 플랫폼 구조 점검]
이 앱이 전국 수영장 플랫폼으로 가기 위해
수영장별 데이터 분리가 제대로 되고 있는지 확인해주세요.

특히 아래를 점검해주세요.
- 모든 핵심 데이터에 swimming_pool_id가 일관되게 적용되는지
- 잘못된 쿼리로 다른 수영장 데이터가 섞일 가능성이 없는지
- pool_admin이 다른 수영장 학생을 조회할 수 없는지
- teacher가 다른 수영장 반을 조회할 수 없는지
- parent가 다른 학생 정보를 조회할 수 없는지

문제 가능성이 있으면 구체적으로 설명해주세요.

[8. 버튼 연결 점검]
현재 앱에서 사용자 화면에 보이는 탭, 버튼, 카드, 저장 버튼 중
실제로 작동하지 않거나 화면만 이동하는 요소가 있는지 전부 점검해주세요.

각 버튼별로 아래를 알려주세요.
- UI만 있는지
- API 연결됐는지
- DB 저장되는지
- 저장 후 조회되는지
- 연동 화면에 반영되는지

특히 다음 기능을 우선 점검해주세요.
- 회원등록
- 반배정
- 내반 조회
- 출결 입력
- 모두출석
- 완료 버튼
- 수영일지 저장
- 지난일지 조회
- 사진 업로드
- 영상 업로드

[9. 버그 위험 점검]
AI로 생성된 앱이라서 특히 버그가 생기기 쉬운 부분을 찾아주세요.
다음 위험 항목을 기준으로 점검해주세요.
- 저장은 되는데 조회가 안 되는 문제
- 조회는 되는데 다른 계정에 반영 안 되는 문제
- 새로고침해야 반영되는 문제
- 중복 저장 문제
- 반이동 후 이전 반에 남아있는 문제
- 삭제 후 목록에 남는 문제
- 파일 업로드 성공처럼 보이지만 실제 저장 안 되는 문제
- 권한 우회 문제
- 날짜/시간표 정렬 오류
- 출결/일지/사진/영상이 다른 반에 잘못 연결되는 문제

각 항목마다
- 현재 위험 여부
- 원인
- 수정 필요 여부
를 알려주세요.

평가 대상:
