[스윔노트 전모드 공통 뒤로가기 스택 복구 + 홈 강제 점프 버그 전면 수정 지시서]

지금부터 앱 전체의 뒤로가기/팝업 닫기/상세화면 복귀 규칙을 전면 재점검하고 수정한다.

현재 핵심 버그:
- 관리자 모드에서 홈 > 운영 관리 아이콘 화면 > 회원 명부 > 학생 프로필로 들어간 뒤
  뒤로가기를 누르면 직전 화면(회원 명부)이 아니라 홈 화면으로 튄다.
- 학부모 계정 > 부모님 상세 > 학생 연결 팝업에서도
  뒤로가기를 누르면 직전 화면이 아니라 홈 화면으로 튄다.

이건 개별 화면 버그가 아니라
“앱 전체 공통 네비게이션 스택이 잘못 처리되는 구조적 문제”다.

중요:
- 이번 작업은 특정 화면만 땜질하는 게 아니라
  전모드 공통 뒤로가기 규칙을 바로잡는 작업이다.
- 홈 화면으로 가는 경우는 오직 명시적 홈 이동일 때만 허용
- 그 외 모든 뒤로가기/팝업 닫기/상세 복귀는 반드시 직전 화면으로 한 단계씩만 이동
- 관리자/선생님/학부모/슈퍼관리자 전체에 공통 적용
- 기능 로직은 건드리지 말고 네비게이션/스택 처리만 정확히 수정
- 수정 후 반드시 실제 클릭 테스트를 여러 단계로 반복하고 보고할 것

==================================================
1. 전역 네비게이션 규칙 (절대 기준)
==================================================

앱 전체에서 아래 규칙을 강제한다.

1. 상세 화면에서 뒤로가기
- 무조건 직전 목록/직전 화면으로 복귀
- 홈 화면으로 점프 금지
- 아이콘 첫 화면으로 강제 점프 금지

2. 팝업/모달 닫기
- 팝업을 연 직전 화면으로 복귀
- 홈 이동 금지
- 아이콘 첫 화면 강제 이동 금지

3. 아이콘 내부 화면
- 아이콘 화면 안에서 들어간 하위 단계는 스택처럼 한 단계씩 빠져나와야 함
- 아이콘 첫 화면으로 한 번에 리셋 금지

4. 홈 이동
- 홈 버튼을 사용자가 명시적으로 눌렀을 때만 허용
- 저장/닫기/뒤로가기 시 홈 이동 금지

==================================================
2. 기준 예시 (반드시 이 흐름대로 수정)
==================================================

[예시 1: 관리자 회원 명부]
홈 화면
→ 운영 관리 아이콘 화면
→ 회원 명부
→ 서태윤 프로필

여기서 뒤로가기:
1번 뒤로 → 회원 명부
2번 뒤로 → 운영 관리 아이콘 화면
3번 뒤로 또는 홈 → 홈 화면

절대 금지:
서태윤 프로필에서 뒤로가기 1번에 바로 홈 화면으로 가는 것

--------------------------------------------------

[예시 2: 관리자 학부모 계정]
홈 화면
→ 운영 관리 아이콘 화면
→ 학부모 계정
→ 서태윤 부모님 상세
→ 학생 연결 팝업

여기서 닫기/뒤로가기:
1번 닫기/뒤로 → 학부모 계정 또는 부모님 상세의 직전 화면
2번 뒤로 → 운영 관리 아이콘 화면
3번 뒤로 또는 홈 → 홈 화면

절대 금지:
학생 연결 팝업 닫기 후 바로 홈 화면 점프

==================================================
3. 적용 범위
==================================================

이 규칙을 아래 전체 모드에 전부 적용한다.

- 슈퍼관리자
- 관리자
- 선생님
- 학부모

특히 아래 유형 전부 점검:
- 목록 → 상세
- 상세 → 수정
- 상세 → 팝업
- 아이콘 화면 → 하위 메뉴
- 탭 안의 상세 진입
- 모달/팝업 닫기
- 저장 후 복귀
- 헤더 뒤로가기
- 사용자 정의 뒤로가기 버튼
- router.back / replace / push 사용 위치

==================================================
4. 반드시 우선 점검할 화면들
==================================================

최우선 점검:
- 관리자 > 운영 관리 > 회원 명부 > 학생 프로필
- 관리자 > 운영 관리 > 학부모 계정 > 부모님 상세 > 학생 연결 팝업
- 관리자 > 운영 관리 전체 하위 메뉴
- 선생님 > 스케줄 > 상세/팝업 흐름
- 선생님 > 학생 상세/일지/출결 팝업
- 학부모 > 자녀 상세/수영정보/설정 팝업
- 슈퍼관리자 > 운영설정/보안/상세 팝업

==================================================
5. 원인 점검 항목
==================================================

아래를 전수 점검하라.

1. 잘못된 router.replace 사용
- 상세/팝업 닫기에서 replace로 홈 또는 아이콘 첫 화면 보내는 코드 제거

2. 뒤로가기 버튼이 실제 back이 아니라 hardcoded navigation인지 확인
- navigate('/home')
- router.push('/admin')
- router.replace('/...') 같은 강제 이동 코드 점검

3. 팝업 닫기 로직이 스택 복귀가 아니라 초기화 이동인지 확인

4. 저장 후 성공 처리에서 홈/아이콘 화면으로 강제 이동하는 코드 점검

5. 중간 화면이 route stack에 안 쌓이고 있는지 확인
- push 해야 할 곳에 replace 사용 여부 확인

==================================================
6. 수정 원칙
==================================================

- 상세 진입은 스택에 쌓이게 한다
- 팝업은 레이어로 처리하고 닫으면 직전 화면으로 복귀
- 홈으로 보내는 코드는 홈 버튼/로그아웃/수영장 전환 등 예외 상황만 남긴다
- 나머지 저장/닫기/뒤로가기는 모두 back 또는 직전 화면 복귀로 통일
- 한 화면만 고치지 말고 공통 패턴을 찾아 전면 수정한다

==================================================
7. 테스트 방법 (반드시 실제 클릭 테스트)
==================================================

각 모드별로 아래를 반복 수행:

1. 홈 화면 진입
2. 아이콘 화면 진입
3. 하위 목록 진입
4. 상세 화면 진입
5. 팝업 열기
6. 뒤로가기/닫기
7. 다시 뒤로가기
8. 홈 버튼 눌렀을 때만 홈 이동 확인

반드시 확인:
- 1번 뒤로 = 직전 화면
- 2번 뒤로 = 그 전 화면
- 홈 버튼 누르지 않았는데 홈으로 튀는지 여부
- 저장 후에도 직전 화면으로 복귀하는지
- 팝업 닫기 후 스택 유지되는지

==================================================
8. 회귀 테스트
==================================================

수정 후 아래도 같이 확인:
- 일지 저장 후 복귀 규칙
- 공지 작성 후 복귀 규칙
- 설정 저장 후 복귀 규칙
- 반배정 팝업 닫기 규칙
- 학생 연결 팝업 닫기 규칙
- 프로필 수정 후 복귀 규칙
- 스케줄 상세/팝업 복귀 규칙

==================================================
9. 수정 금지 영역
==================================================

이번 작업에서 절대 깨지면 안 되는 것:
- CRUD 저장 로직
- 권한 구조
- 수영장 전환 로직
- 로그아웃 로직
- 홈 버튼 자체 동작
- 실제 데이터 저장/조회 기능

이번 작업은 네비게이션 스택 복구 작업이다.
기능 자체를 갈아엎지 마라.

==================================================
10. 완료 보고 형식
==================================================

반드시 아래 형식으로 보고하라.

1. 홈으로 잘못 튀던 화면 목록
2. 직접 원인
   - replace 남용인지
   - hardcoded navigation인지
   - stack 미구성인지
3. 수정한 공통 규칙
4. 수정한 파일 목록
5. 관리자 모드 테스트 결과
6. 선생님 모드 테스트 결과
7. 학부모 모드 테스트 결과
8. 슈퍼관리자 모드 테스트 결과
9. 아직 남은 예외 화면 여부

중요:
- “수정 완료” 한 줄 보고 금지
- 반드시 실제 클릭 흐름 기준으로 적어라
- 각 모드별 최소 3단계 이상 들어갔다가 뒤로 나오는 테스트를 수행하고 적어라

==================================================
11. 최종 목적
==================================================

이번 작업의 최종 목적은
앱 전체에서 뒤로가기/팝업 닫기/상세 복귀 규칙을 완전히 통일하는 것이다.

핵심:
- 홈은 홈 버튼으로만 간다
- 나머지는 직전 화면으로 한 단계씩만 복귀
- 아이콘 내부 화면은 스택을 유지한다
- 팝업은 닫으면 직전 화면으로 돌아간다
- 홈으로 강제 점프하는 버그를 전모드에서 제거한다

이 기준으로 전면 점검하고 수정하라.
