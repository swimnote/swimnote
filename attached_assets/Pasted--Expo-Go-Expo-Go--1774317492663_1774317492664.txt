[긴급 수정 지시서 - Expo Go 선생님 모드 스케줄러 멈춤 문제 전면 디버깅 및 수정]

지금부터 선생님 모드 스케줄러에서 Expo Go 기준으로 앱이 멈추는 문제를 최우선 긴급 버그로 처리한다.

중요:
- 웹에서 정상이어도 완료로 간주하지 않는다
- 반드시 iPhone Expo Go 기준으로 재현 → 원인 추적 → 수정 → 재테스트까지 수행
- “조금 느림”이 아니라 “멈춤/터치 불가/프리징”은 치명 버그다
- 기능 추가 금지, 우선 스케줄러 안정화가 목표다
- 원인을 못 찾으면 어디까지 좁혔는지 상세 보고할 것
- 거짓 완료 보고 절대 금지

==================================================
1. 문제 정의
==================================================

현재 문제:
- 선생님 모드에서 스케줄러를 켜고 작업하려고 하면 Expo Go에서 앱이 멈춤
- 날짜 선택, 시간 선택, 반배정, 화면 전환 중 프리징/터치 불가/응답 없음 발생 가능
- 웹에서는 될 수 있어도 Expo Go iPhone 기준에서 멈추면 실패로 본다

==================================================
2. 최우선 재현 시나리오
==================================================

반드시 iPhone Expo Go에서 아래 순서로 실제 재현:

1. 선생님 로그인
2. 선생님 홈 진입
3. 스케줄러 진입
4. 월간 탭 진입
5. 날짜 선택
6. 시간 슬롯 선택
7. 반배정/상세 진입 시도
8. 뒤로가기 / 팝업 닫기

각 단계별로 어디서 멈추는지 정확히 기록할 것

==================================================
3. 가장 의심되는 원인 전수 점검
==================================================

아래 항목을 하나라도 의심되면 바로 확인하고 로그 찍어라.

A. 무한 렌더 / 상태 루프
- useEffect dependency loop
- selectedDate 변경 시 setState 반복
- selectedSlot 변경 시 연쇄 상태 업데이트
- useMemo/useCallback 누락으로 매 렌더 무거운 계산 반복
- 파생 데이터 계산이 렌더마다 다시 실행되는지

B. 월간 캘린더 렌더 과부하
- 날짜 셀 개수 많음
- 각 셀마다 수업/반/회원 수 계산
- 렌더 시 데이터 가공이 과도함
- FlatList/ScrollView 최적화 누락
- key 중복/잘못된 keyExtractor

C. 바텀시트/모달/오버레이 충돌
- bottom sheet가 열린 채 route 이동
- invisible overlay가 터치 막음
- modal backdrop가 남아 있음
- pointerEvents / zIndex / absolute View 문제
- 반배정 화면 뒤에 팝업 잔상 남는 구조

D. 스케줄 데이터 조회 문제
- 스케줄 API 응답 null/undefined 처리 안 됨
- teacher_id / pool_id / date filter 꼬임
- schedule_days / schedule_time 변환 오류
- 비어 있는 데이터에서 map/filter 에러
- 날짜 포맷 문제로 iOS만 멈춤

E. 네비게이션 충돌
- router.push / replace / back 혼용
- 스케줄러 내부 상태 유지하면서 새 화면 push하는 구조 충돌
- 팝업 닫기와 페이지 이동이 동시에 발생
- 이전 세션의 하단 팝업 상태가 남아있음

F. iOS/Expo Go 전용 문제
- nested ScrollView
- Gesture conflict
- Touchable/Pressable이 ScrollView에 먹힘
- 성능 저하가 디버그 모드에서만 과장되어 발생
- console log 과다/상태 추적 과다로 dev freeze

==================================================
4. 디버깅 방식 (반드시 단계 분리)
==================================================

한 번에 전체를 보지 말고 단계적으로 원인을 분리해라.

1단계:
- 스케줄러 화면 최소 렌더
- 캘린더 껍데기만 표시
- 멈추는지 확인

2단계:
- 날짜 선택만 연결
- 날짜 선택 시 멈추는지 확인

3단계:
- 시간 슬롯 목록만 연결
- 시간 클릭 전까지 정상인지 확인

4단계:
- 시간 클릭 onPress에 console/log만 연결
- 이 단계에서 멈추는지 확인

5단계:
- 반배정/상세 진입 연결
- 이 단계에서 팝업 잔상이나 멈춤 발생 여부 확인

6단계:
- 뒤로가기/닫기까지 연결
- 네비게이션 루프 있는지 확인

중요:
어느 단계에서 멈추는지 명확히 찾아라.
“전체적으로 이상함” 같은 보고 금지.

==================================================
5. 반드시 넣어야 할 진단 로그
==================================================

다음 지점에 로그 또는 디버그 마커 추가:

- 스케줄러 최초 렌더
- 월간 탭 렌더 시작/끝
- 날짜 선택 직후
- 시간 슬롯 계산 직후
- 시간 슬롯 onPress 진입
- 반배정 버튼 클릭 직후
- route 이동 직전
- bottom sheet close 호출 직후
- API 응답 수신 직후

중요:
멈추기 직전 마지막 로그가 어디인지 반드시 찾아라.

6. 수정 원칙
==================================================

- 기능 추가 금지
- 먼저 멈춤을 없애는 것에 집중
- 렌더 계산은 최대한 가볍게
- 월간 캘린더 셀 렌더 최적화
- 스케줄 데이터 계산은 memoize
- 바텀시트/모달 상태는 페이지 이동 전에 반드시 정리
- nested scroll/overlay 충돌 제거
- iPhone Expo Go에서 터치 우선 확인

==================================================
9. 성능/안정성 확인
==================================================

다음도 같이 점검:
- 캘린더 셀 key 중복 없는지
- 리스트 virtualization 가능한지
- 큰 map/filter가 render body 안에 없는지
- 날짜 포맷 계산이 render마다 안 도는지
- 불필요한 상태 setState 반복 없는지
- bottom sheet ref가 중복 생성되지 않는지

- 관리자 스케줄 관련 기능
