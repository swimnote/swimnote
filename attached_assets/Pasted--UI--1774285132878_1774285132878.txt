[선생님 홈 퀵버튼 UI/연결 수정 지시서 - 회원추가 버튼 위치/디자인/팝업 통일]

지금 선생님 모드의 퀵버튼 기능은 기본 동작이 잘 되었으니,
이번 작업에서는 선생님 홈 화면에 “회원추가” 퀵버튼을 더 자연스럽고 예쁘게 배치하고,
누르면 관리자 모드의 직접등록 팝업과 같은 구조의 입력창이 뜨되,
최종 버튼은 “등록하기”가 아니라 “등록요청”으로 바꿔서 관리자 승인 대기 흐름으로 연결되게 수정한다.

중요:
- 기능은 이미 있는 선생님 등록 요청 구조를 유지한다
- 이번 작업은 위치/디자인/팝업 연결/문구 통일이 핵심이다
- 관리자 직접등록과 동일한 입력 폼 UX를 재사용하되, 선생님용은 승인 요청 흐름으로 바꾼다
- 홈 화면 흐름을 깨지 말고 자연스럽게 넣어라
- 실제로 눌러서 팝업 뜨고 요청 저장까지 테스트하고 보고할 것

==================================================
1. 퀵버튼 위치 수정
==================================================

대상:
- 선생님 모드 홈 화면

위치 기준:
- “월간 스케줄러”와 “오늘 수업” 사이 또는 그 인접 구간에
- 공간을 적절히 비우고
- 눈에 잘 띄는 직사각형 퀵버튼으로 배치

중요:
- 기존 홈 카드/섹션을 밀어 올리거나 겹치게 하지 말 것
- 버튼이 너무 작거나 둥근 칩처럼 보이지 말고
- 눌렀을 때 바로 “빠른 회원추가” 기능으로 인식되게 만들 것

==================================================
2. 퀵버튼 디자인
==================================================

버튼 이름:
- 회원추가

버튼 스타일:
- 약간 가로형 직사각형 버튼
- + 아이콘이 잘 보이게
- “맛있게” 보이도록 메인 퀵액션 느낌
- 너무 크지 않지만 홈에서 바로 눌러야 하는 핵심 버튼처럼 보이게

디자인 방향:
- 선생님 홈에서 튀되 과하지 않게
- 오늘 수업/월간 스케줄러와 자연스럽게 어울리는 톤
- 기존 앱 톤앤매너 유지

중요:
- 단순 텍스트 링크처럼 보이면 안 됨
- 눌러야 할 주요 액션으로 보여야 함

==================================================
3. 버튼 클릭 시 동작
==================================================

회원추가 버튼 클릭
→ 팝업(모달/바텀시트) 열기

팝업 구조는
관리자 모드의 “어린이 직접 등록” 화면과 최대한 비슷하게 가져간다.

참고 기준:
- 내가 보낸 사진 속 관리자 모드 직접등록 팝업 UI
- 입력창 배치와 구성은 유사하게 재사용

하지만 선생님 모드에서는 목적이 다르므로
최종 액션은 “즉시 등록”이 아니라 “등록 요청”이어야 한다.

==================================================
4. 선생님용 팝업 제목 및 문구
==================================================

팝업 제목:
- 회원 등록 요청
또는
- 학생 등록 요청

추천:
- 회원 등록 요청

하단 메인 버튼:
- 등록하기 ❌
- 등록요청 ⭕

안내 문구:
- 관리자 승인 후 정식 회원으로 반영됩니다
- 또는
- 등록 요청 후 관리자 승인 대기 상태로 생성됩니다

중요:
- 관리자 직접등록처럼 보이되
- 선생님이 최종 승인하는 구조처럼 보이면 안 됨

==================================================
5. 팝업 입력 항목
==================================================

관리자 직접등록 팝업과 최대한 동일하게 구성

입력 항목:
- 학생 이름 또는 회원 이름 (필수)
- 출생년도 (중복 체크용)
- 보호자 이름
- 보호자 전화번호
- 주 수업 횟수 (주1회 / 주2회 / 주3회+)

중요:
- 선생님 요청 팝업도 관리자 팝업과 비슷한 UX로 맞춘다
- 다만 저장 대상 상태는 pending 이어야 한다

==================================================
6. 선생님용 저장 로직
==================================================

등록요청 버튼 클릭 시:

- 상태 = pending
- 생성자 = 현재 선생님
- 관리자 승인 대기 리스트로 들어감

절대 금지:
- active 즉시 생성 금지
- 회원수 즉시 카운트 금지
- 학부모 연결 즉시 완료 금지
- 결제 대상 즉시 포함 금지

기존에 만든 선생님 생성 전용 구조가 있다면 그대로 재사용하고,
이번에는 UI와 액션명만 정확히 연결해라.

==================================================
7. 관리자 화면 연동
==================================================

선생님이 등록요청한 데이터는
관리자 모드의 승인 대기 학생/회원 리스트로 들어가야 한다.

관리자는
- 승인
- 거절
처리 가능해야 한다.

중요:
이번 작업에서 관리자 승인 구조가 깨지면 안 된다.

==================================================
8. 홈 화면 UX 규칙
==================================================

회원추가 버튼을 눌렀을 때:
- 현재 선생님 홈 화면 위에 팝업으로 떠야 한다
- 새 페이지로 어색하게 넘어가지 말 것
- 닫기(X) 누르면 직전 홈 화면 그대로 유지
- 뒤로가기도 홈으로 튀지 말고 팝업만 닫히게

중요:
팝업 닫기 후 홈 상태 유지 필수

==================================================
9. 디자인/배치 주의사항
==================================================

반드시 확인:
- 월간 스케줄러 / 오늘 수업 / 퀵버튼 간 간격 자연스러운지
- 버튼이 카드/리스트를 침범하지 않는지
- iPhone Expo Go에서도 위치가 안 깨지는지
- 웹/앱 둘 다 확인하되 iPhone 기준 우선 확인

==================================================
10. 테스트 필수
==================================================

반드시 실제 테스트:

1. 선생님 홈에서 회원추가 버튼이 잘 보이는지
2. 버튼 눌렀을 때 팝업이 뜨는지
3. 팝업 UI가 관리자 직접등록 화면과 유사한지
4. 하단 버튼이 “등록요청”으로 표시되는지
5. 등록요청 시 pending 상태로 생성되는지
6. 관리자 승인 대기 리스트에 들어가는지
7. 팝업 닫기 후 홈으로 튀지 않고 제자리 유지되는지
8. iPhone Expo Go에서 UI 안 깨지는지

==================================================
11. 수정 금지
==================================================

절대 깨지면 안 되는 것:
- 선생님 등록요청 pending 구조
- 관리자 승인 구조
- 기존 홈 화면 카드 기능
- 뒤로가기/팝업 닫기 규칙
- 관리자 직접등록 기능 자체

이번 작업은
선생님 홈 퀵버튼의 위치/디자인/팝업 UX 정리 작업이다.
