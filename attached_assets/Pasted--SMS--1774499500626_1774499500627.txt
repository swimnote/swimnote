[스윔노트 — 지원센터 구조 정리 + SMS 로그 노출 제거 + 상단 칩/리스트 겹침 버그 수정 지시문]

현재 지원센터는 역할이 섞여 있고, 상단 상태칩/탭/카드 영역과 아래 리스트 레이아웃이 겹쳐서
보여야 할 정보가 가려지고 일부 숫자칩은 눌리지 않는 상태다.

이번 작업은 지원센터를 “처리 중심 화면”으로 다시 정리하고,
불필요한 SMS 로그 노출을 제거하며,
상단 칩/리스트 겹침과 터치 불가 문제를 전부 수정하는 작업이다.

━━━━━━━━━━━━━━━━━━
1. 핵심 방향
━━━━━━━━━━━━━━━━━━

지원센터는 “슈퍼관리자가 지금 처리해야 할 것”만 보여주는 화면으로 정리한다.

원칙:
- 처리 대상 우선
- 운영 로그는 뒤로
- 복잡한 숫자판보다 작업 목록 우선
- 상단 칩/탭/요약카드가 본문을 가리면 실패

핵심:
👉 지원센터는 로그 전시장 말고, 처리 대시보드여야 한다

━━━━━━━━━━━━━━━━━━
2. SMS 관련 판단 및 처리
━━━━━━━━━━━━━━━━━━

현재 판단:
- 수영장 운영자가 보낸 SMS 개별 로그를 슈퍼관리자가 지원센터 메인에서 매번 볼 필요는 없음
- 지금 단계에서는 저장만 하고, 지원센터 메인 노출은 제거하는 것이 맞음

즉시 변경:

제거 또는 숨김 대상:
- SMS 판매·정산 관련 화면
- SMS 템플릿 관리
- SMS 로그·템플릿 관리 메인 진입
- 인증번호 발송 기록 메인 진입
- 지원센터 상단 SMS 초과 / SMS 차단 같은 핵심 지표 노출

유지:
- SMS/인증 발송 로그는 DB 저장 유지 가능
- 필요 시 별도 숨은 관리 페이지 또는 검색형 로그 화면으로만 조회
- 메인 지원센터에서는 기본 비노출

원칙:
- “보기 위한 로그”보다 “처리할 티켓”이 먼저
- SMS 로그는 문제 발생 시 조회하는 보조 수단이지 메인 콘텐츠가 아님

━━━━━━━━━━━━━━━━━━
3. 지원센터 메인 구조 재정리
━━━━━━━━━━━━━━━━━━

지원센터 메인에는 아래만 남긴다.

우선 노출:
1. SLA 초과 / 긴급 이슈
2. 고객센터 티켓 목록
3. 결제/환불/계정잠금/업로드 차단 등 실제 조치 필요한 항목
4. 정책/컴플라이언스 확인 항목 (필요 최소만)

뒤로 보내거나 제거:
- SMS 판매 통계
- SMS 템플릿 현황
- 인증번호 발송 로그
- 보기만 하는 운영 로그 카드
- 더미 숫자 카드

상단 요약칩은 최소화:
- 전체
- 긴급
- SLA 초과
- 결제
- 보안
- 환불
정도로 줄이고, 의미 약한 칩은 제거

━━━━━━━━━━━━━━━━━━
4. 레이아웃 겹침/터치 불가 버그 수정
━━━━━━━━━━━━━━━━━━

현재 문제:
- 상단 숫자칩/탭이 아래 리스트를 가림
- 아래 카드가 위로 밀려 올라감
- 일부 칩(예: 17)이 안 눌림
- 화면별로 칸 높이/간격이 안 맞음

원인 후보:
- sticky 헤더 높이 계산 오류
- absolute/relative 혼용
- z-index 충돌
- ScrollView/FlatList 헤더 구조 꼬임
- SafeArea / status bar padding 누락
- 상단 요약 영역이 실제 높이보다 작게 계산됨
- 터치 레이어가 다른 뷰에 가려짐

수정 원칙:
- 상단 요약 영역과 본문 리스트를 명확히 분리
- sticky 영역 높이를 실제 렌더 높이 기준으로 계산
- 리스트 시작 위치에 충분한 top padding 또는 header spacer 적용
- 겹침을 opacity/zIndex 꼼수로 덮지 말 것
- 숫자칩/탭은 전부 실제 터치 가능해야 함

반드시 수정:
- 칩 눌러도 안 먹는 문제
- 상단 요약이 아래 카드 가리는 문제
- 카드가 위로 밀려 보이는 문제
- 화면마다 높이 제각각인 문제

━━━━━━━━━━━━━━━━━━
5. 구조 권장 방식
━━━━━━━━━━━━━━━━━━

지원센터 화면은 아래 구조로 정리:

- 고정 헤더
- 간단한 상태 요약 1줄
- 카테고리 탭/칩 1줄
- 본문 리스트

주의:
- 상단에 카드 2~3층 쌓지 말 것
- 탭 + 숫자박스 + 경고배너 + 필터칩을 한 화면 상단에 너무 많이 쌓지 말 것
- 상단 정보가 많아지면 아래 리스트를 잡아먹음

권장:
- 숫자 요약은 1줄 또는 가로 스크롤
- 본문 리스트가 첫 화면에서 바로 보여야 함

━━━━━━━━━━━━━━━━━━
6. 고객센터 리스트 가독성 정리
━━━━━━━━━━━━━━━━━━

리스트 1개 항목에 보일 정보는 최소화:

- 제목
- 운영자명/수영장명
- 생성일 또는 경과시간
- 상태칩
- 담당자
- 긴급/SLA 여부

줄여야 할 것:
- 중복 상태 문구
- 작은 텍스트 과다 표시
- 색상 과다 사용
- 의미 없는 아이콘 반복

목표:
- 첫눈에 “뭘 먼저 처리해야 하는지” 보여야 함

━━━━━━━━━━━━━━━━━━
7. 실제 작업 대상 점검
━━━━━━━━━━━━━━━━━━

다음 화면 전부 점검:

- 지원센터 메인
- 고객센터 목록
- SMS 로그·템플릿 관리
- 인증번호 발송 기록
- 정책·컴플라이언스 카드 영역
- 공통 SubScreenHeader / 홈 버튼
- 상단 요약칩 / 탭 / 상태카드

필수:
- 칸 높이 안 맞는 화면 전수 점검
- 눌리지 않는 칩/버튼 전수 확인
- 카드 겹침/스크롤 충돌 전수 확인

━━━━━━━━━━━━━━━━━━
8. 테스트
━━━━━━━━━━━━━━━━━━

반드시 실제로 확인:

1. 지원센터 첫 화면 진입 시 상단이 아래 리스트를 가리지 않는지
2. 숫자칩 전부 눌리는지
3. 17 같은 상단 숫자칩 터치 정상 동작하는지
4. 스크롤 시 sticky 동작이 자연스러운지
5. SMS 관련 메뉴가 메인에서 제거되었는지
6. 고객센터 리스트가 첫 화면에서 바로 읽히는지
7. 홈 버튼 눌렀을 때 슈퍼 홈으로 정상 이동하는지

━━━━━━━━━━━━━━━━━━
9. 보고 형식
━━━━━━━━━━━━━━━━━━

반드시 포함:

- 수정 파일 경로
- 제거한 SMS 관련 메뉴/화면 목록
- 저장만 유지한 SMS 로그 테이블/기능 목록
- 겹침 문제 원인
- 수정한 레이아웃 방식
- 눌리지 않던 칩 테스트 결과
- 전/후 비교 결과

“정리 완료”만 쓰지 말고
어떤 영역이 왜 겹쳤고, 지금 어떻게 분리됐는지 같이 제출할 것
