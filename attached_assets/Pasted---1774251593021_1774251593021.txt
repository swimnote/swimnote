[스윔노트 보안설정 > 외부 서비스 연결 상태 고도화 구현 지시서]

지금부터 보안설정 안의 “외부 서비스 연결 상태”를
단순 텍스트 표시가 아니라 실제 운영용 연결 상태 관리 화면으로 확장 구현한다.

이번 작업의 목적은
외부에서 연결되는 서비스들을 한 화면에서 전부 확인하고,
각 서비스별로
- 어떤 경로(URL/엔드포인트/프로젝트 식별자)로 연결되어 있는지
- 언제 연결되었는지
- 현재 상태가 정상인지/끊김인지/작동 안 됨인지
- 새로고침으로 현재 연결 상태를 재확인할 수 있는지
를 보여주는 것이다.

중요:
- 실제 연결이 아직 안 되어 있어도 항목은 미리 전부 표시되어야 한다
- 연결 안 된 서비스도 “미연결” 상태로 보여야 한다
- 수정 기능이 없어도 조회/표시는 가능해야 한다
- 각 항목 클릭 시 상세 정보 페이지 또는 상세 패널이 열려야 한다
- 새로고침 버튼은 각 항목마다 있어야 한다
- 기존 보안설정 화면 구조와 기능은 절대 깨지지 않게 할 것
- 실제 연결 가능한 서비스는 실제 연결 상태를 읽고, 아직 미연결 서비스는 placeholder 상태라도 구조상 미리 준비해 둘 것

1. 구현 목표
==================================================

보안설정 > 외부 서비스 연결 상태 화면을 아래 기준으로 다시 만든다.

핵심 목표:
1. 외부 서비스 목록을 운영용으로 확장
2. 각 서비스별 상태를 개별 카드/행으로 표시
3. 각 서비스 클릭 시 상세 정보 확인 가능
4. 각 서비스별 새로고침 버튼 제공
5. 실제 연결된 것은 실상태 표시
6. 아직 연결 안 된 것도 미리 항목 표시
7. 나중에 연결이 추가되어도 구조가 안 깨지게 확장형으로 구현

2. 외부 서비스 연결 상태 목록 구성
==================================================

기존 항목은 유지하되 아래 항목들을 빠짐없이 전부 미리 넣는다.

필수 항목 목록:

[A. 데이터/인프라]
- Supabase DB
- Supabase Auth
- Supabase Storage (사용 시)
- Cloudflare R2
- 백업 스토리지/백업 DB (있다면 별도)
- CDN 또는 파일 전달 관련 서비스(있다면)

[B. 결제/정산]
- PortOne
- 실제 연결된 PG사
  - 예: KG이니시스 / 토스페이먼츠 / 나이스페이 / 다날 등
- 정산 계좌/정산 연동 관련 상태(조회용 구조만이라도 미리 준비)

[C. 푸시/메시징/알림]
- Apple Push (APNs)
- Firebase Cloud Messaging (FCM) 또는 안드로이드 푸시 관련 서비스
- SMS Provider (실제 SMS 과금용이 아니라 인증/시스템 메시지용 연결 상태가 있으면)
- Email Provider
- 알림/메일 발송 서비스가 있다면 추가

[D. 앱스토어/플레이스토어/배포 관련]
- Apple App Store Connect 관련 연결 상태
- Google Play Console 관련 연결 상태
- 앱 배포/구독 상태 조회용 외부 연결이 있다면 그 항목도 포함

[E. 기타 외부 연동]
- 이미지 업로드/광고/분석/로그 수집 등 외부 서비스가 있다면 확장 가능 구조로 포함
- 현재 미사용이어도 “예정” 또는 “미연결” 항목으로 미리 둘 수 있게 구현

중요:
실제 연결 유무와 상관없이
외부 서비스 연결 상태 목록에는 항목이 미리 보이게 하라.

3. 상태값 체계
==================================================

각 서비스는 아래 상태값 중 하나로 표시되게 한다.

필수 상태:
- 정상
- 주의
- 경고
- 작동 안 됨
- 끊김
- 미연결
- 점검중
- 예정

설명:
- 정상: 현재 정상 동작 중
- 주의: 연결은 되어 있으나 점검 필요
- 경고: 토큰 만료 임박, 재인증 필요 등
- 작동 안 됨: 연결은 있으나 실제 사용 불가
- 끊김: 이전에는 연결됐으나 현재 연결 실패
- 미연결: 아직 연결 설정 자체가 안 됨
- 점검중: 상태 확인 중
- 예정: 추후 연결 예정인 항목

중요:
단순 초록색/빨간색만 쓰지 말고 상태 텍스트를 명확히 보여라.

==================================================
4. 서비스 카드/행 UI 구성
==================================================

각 외부 서비스는 카드 또는 리스트 행 형태로 구성한다.

반드시 표시할 정보:
- 서비스명
- 현재 상태
- 마지막 확인 시각
- 간단 상태 설명
- 새로고침 버튼

예:
Supabase DB / 정상 / 마지막 확인: 5분 전 / 연결 정상
PortOne / 경고 / 마지막 확인: 1시간 전 / 인증 토큰 만료 임박
Email Provider / 끊김 / 마지막 확인: 3시간 전 / SMTP 연결 실패
Google Play Console / 미연결 / 마지막 확인 없음 / 아직 연결되지 않음

중요:
현재 스크린샷처럼 텍스트만 나열하지 말고
운영자가 상태를 빠르게 읽을 수 있는 구조로 바꿔라.

==================================================
5. 상세 보기 기능 (항목 클릭 시)
==================================================

각 외부 서비스 항목을 누르면
상세 정보 화면 또는 상세 패널이 열려야 한다.

상세 정보에 반드시 포함:
- 서비스명
- 현재 상태
- 연결 여부
- 연결된 URL 또는 엔드포인트
- 프로젝트 식별자 / 앱 ID / 버킷명 / DB 식별자 등
- 연결 시작 시각 또는 등록 시각
- 마지막 정상 확인 시각
- 마지막 오류 시각(있다면)
- 최근 상태 메시지
- 사용 목적 설명
- 연결 유형
  - API
  - DB
  - Storage
  - Push
  - Payment
  - App Store
  - Play Store 등

예시:
Supabase DB 상세:
- 서비스명: Supabase DB
- 상태: 정상
- 연결 URL: https://xxxxx.supabase.co
- 프로젝트 식별자: xxxxx
- 연결일: 2026-03-20
- 마지막 확인: 2026-03-23 14:01
- 최근 메시지: DB 응답 정상

Cloudflare R2 상세:
- 버킷명
- endpoint
- 연결 상태
- 마지막 확인
- 최근 오류 여부

PortOne 상세:
- 연결 여부
- 상점 식별자
- 최근 인증 확인 시간
- 최근 경고 메시지
- 연결 상태

Apple/Google 스토어 관련 상세:
- 연결 여부
- 앱 식별 정보
- 마지막 확인 시간
- 구독/배포 상태 조회 가능 여부

중요:
실제 URL/엔드포인트/식별자 정보가 있다면 보여줘야 하고,
아직 연결 안 된 경우에는 “미연결”로 보이게 하되
구조는 동일하게 유지하라.

==================================================
6. 새로고침 버튼 구현
==================================================

각 외부 서비스 항목 우측에 “새로고침” 버튼을 둔다.

동작:
- 해당 서비스의 현재 연결 상태를 재확인
- 마지막 확인 시각 갱신
- 상태 업데이트
- 최근 메시지 갱신

예:
- 정상 → 그대로 정상
- 정상 → 경고
- 경고 → 정상
- 연결됨 → 끊김
- 미연결 → 미연결 유지

중요:
전체 새로고침이 아니라
각 항목별 개별 새로고침이 가능해야 한다.

추가로 가능하면:
- 상단에 “전체 새로고침” 버튼도 추가 가능
- 하지만 각 항목 개별 새로고침은 반드시 있어야 함

==================================================
7. 실제 연결이 아직 없어도 항목은 보이게 하기
==================================================

중요:
실제 연결이 안 되어도 항목은 미리 목록에 있어야 한다.

예:
- App Store Connect → 미연결
- Google Play Console → 미연결
- PG사 → 예정 또는 미연결
- 분석 서비스 → 예정
- 광고 연결 → 예정

즉,
운영자가 나중에 무엇을 연결해야 할지
보안설정 화면에서 전체 구조를 미리 볼 수 있어야 한다.

8. 데이터 구조 권장
==================================================

현재 구조를 해치지 않는 선에서
외부 서비스 연결 상태를 저장할 수 있는 테이블/설정 구조를 만든다.

권장 필드 예시:
- service_key
- service_name
- service_type
- status
- is_connected
- endpoint_url
- project_id_or_app_id
- bucket_name(optional)
- connected_at
- last_checked_at
- last_error_at
- status_message
- notes
- is_placeholder
- is_editable(optional)

예시 service_key:
- supabase_db
- supabase_auth
- supabase_storage
- cloudflare_r2
- portone
- pg_primary
- apns
- fcm
- sms_provider
- email_provider
- app_store_connect
- google_play_console

중요:
실제 연결 정보와 placeholder 항목을 같이 관리 가능해야 한다.

10. URL/연결 정보 표시 정책
==================================================

항목 클릭 시 URL 또는 연결 정보가 보이게 한다.

예:
- Supabase URL
- R2 endpoint
- PortOne 연동 식별값
- App Store/Play Store 관련 외부 연결 식별 정보

주의:
보안상 민감한 전체 키/비밀값(secret)은 그대로 노출하지 말고
- 일부 마스킹
- 또는 “등록됨/미등록”으로 표시

표시 가능한 것:
- 공개 URL
- endpoint
- project ID
- 앱 식별 정보
- 버킷명
- 연결 일시
- 마지막 확인 시각

표시 금지 또는 마스킹:
- secret key
- access token 전문
- private credential 전문

12. UI/UX 규칙
==================================================

- 현재 보안설정 화면 안에서 자연스럽게 이어지게 구성

- 실제 연결 안 된 서비스도 빈 공간 없이 자연스럽게 보이게 구성

14. 테스트 및 검증 필수
==================================================

작업 후 반드시 직접 테스트하고 항목별로 보고하라.

최소 테스트 항목:
1. 외부 서비스 목록에 모든 항목이 미리 보이는지
2. 실제 연결 안 된 항목도 미연결/예정 상태로 보이는지
3. Supabase DB 상세 클릭 시 URL/연결정보가 보이는지
4. Cloudflare R2 상세 클릭 시 endpoint/연결정보가 보이는지
5. PortOne 및 PG 항목이 목록에 포함되는지
6. App Store / Play Store 관련 항목이 목록에 포함되는지
7. 각 항목별 새로고침 버튼이 작동하는지
8. 새로고침 후 마지막 확인 시각이 갱신되는지
9. 정상/주의/경고/작동 안 됨/끊김/미연결 상태 표시가 되는지
10. secret 값이 그대로 노출되지 않는지
11. 모바일 화면에서 UI가 깨지지 않는지
12. 기존 보안설정 기능이 깨지지 않았는지

[작업 완료 보고 형식]
1. 추가된 외부 서비스 항목 목록
2. 서비스별 상태값 구조
3. 상세 정보에 표시되는 항목
4. 실제 연결 / placeholder 구분 방식
5. 새로고침 동작 방식
6. 수정한 파일 목록
7. 실제 테스트 결과
8. 남은 제한사항 또는 추후 연동 필요 항목

거짓 완료 보고 금지.
실제 확인 안 했으면 안 했다고 적어라.

==================================================
15. 최종 목적
==================================================

이번 작업의 목적은
보안설정 안의 외부 서비스 연결 상태를
실제 운영자가 쓸 수 있는 모니터링/확인 화면으로 완성하는 것이다.

핵심:
- 외부 서비스 전부 미리 표시
- 연결 안 된 것도 구조상 보이게 함
- 실제 연결되면 URL/식별자/상태 확인 가능
