지금부터 회원관리 / 반배정 / 상태변경 / 휴원 / 퇴원 / 아카이브 흐름을 아래 규칙으로 수정한다.
기존 기능 삭제 금지, 기존 DB 구조 최대한 유지, 관리자-선생님 데이터 연동 유지, 하단 탭바 사용 금지.

[최우선 목표]
1. 휴원/퇴원 처리 시 즉시이동 / 다음달이동 선택 가능하게 수정
2. 이동예약 회원은 카드에 예약 배지 표시
3. 회원카드는 어디서 보든 완전히 같은 디자인과 같은 정보 구조로 통일
4. 회원관리 상단 필터 및 리스트 반영 로직 재정리
5. 선생님 모드와 관리자 모드 모두 동일 규칙 적용

=====================================
[회원 상태 구조]
대표 상태는 아래 4개 중 하나만 가진다.
- 정상
- 미배정
- 휴원
- 퇴원

과금 포함 상태:
- 정상
- 미배정
- 연기
- 대기
- 퇴원

과금 제외 상태:
- 아카이브
- 삭제

보조 상태(예약 상태)는 별도로 둔다.
- 휴원예정
- 퇴원예정
- null

예시 필드:
memberStatus = 정상 | 미배정 | 휴원 | 퇴원
pendingStatusChange = 휴원 | 퇴원 | null
pendingEffectiveMode = immediate | next_month | null
pendingEffectiveMonth = YYYY-MM | null

=====================================
[상단 필터 순서 고정]
회원관리 상단 가로 스크롤 필터는 아래 순서로 고정한다.

- 전체
- 정상
- 미배정
- 주1회
- 주2회
- 주3회
- 미연결
- 휴원
- 퇴원

제거 항목:
- 배정불일치 제거
- 연결완료 제거

연결대기 완료 후 규칙:
- 연결이 완료되면 연결완료라는 별도 항목으로 두지 말고 바로 미배정으로 이동

=====================================
[주횟수 규칙]
주횟수는 대표 상태와 별도 계산 태그로 관리한다.
- 주1회
- 주2회
- 주3회

규칙:
- 배정 반 1개 = 주1회
- 배정 반 2개 = 주2회
- 배정 반 3개 이상 = 주3회
- 주3회 이상은 모두 주3회로 묶는다

리스트 반영:
- 정상 회원은 정상 리스트에 표시
- 동시에 배정 개수에 따라 주1/주2/주3 리스트에도 표시
- 휴원/퇴원/미배정 상태면 정상 및 주횟수 리스트에서는 제외

예:
정상 + 반 3개 = 정상 리스트에도 있고 주3회 리스트에도 있음
휴원 처리되면 정상/주3회 리스트에서 빠지고 휴원 리스트로 이동

=====================================
[회원카드 공통 컴포넌트 절대 통일]
회원카드는 모든 화면에서 반드시 동일한 공통 컴포넌트 1개만 사용한다.
절대 화면마다 다른 카드 만들지 말고 하나로 통일한다.

적용 화면:
- 회원관리
- 반관리
- 반배정
- 미배정 리스트
- 휴원 리스트
- 퇴원 리스트
- 검색 결과
- 학부모관리 내부 회원 리스트
- 선생님 모드 회원 리스트
- 관리자 모드 회원 리스트

검색 결과도 예외 없이 똑같은 카드 사용한다.
즉, 회원을 어디서 검색하든 카드 모양, 정보 위치, 배지 위치, 버튼 구조가 완전히 같아야 한다.

공통 카드명 예시:
UnifiedMemberCard

=====================================
[회원카드 표시 정보 고정]
UnifiedMemberCard에는 아래 정보를 동일 위치에 고정 표시한다.

좌측:
- 원형 또는 사각 아바타
- 이름 첫 글자 또는 이니셜

중앙 상단:
- 회원 이름
- 대표 상태 배지 1개
  정상 / 미배정 / 휴원 / 퇴원 중 하나만 표시
- 주횟수 배지 1개
  주1회 / 주2회 / 주3회
- 예약 배지 1개 (있을 때만)
  휴원예정 / 퇴원예정

중앙 본문:
- 배정 반 정보
- 선생님 정보
- 보호자 정보
- 필요 시 연결 상태

하단:
- 회원 상세 버튼
- 화면 문맥에 따라 상태변경/삭제/복귀 버튼 추가 가능
단, 카드 기본 레이아웃은 유지

=====================================
[예약 배지 표시]
휴원 또는 퇴원을 다음달이동으로 예약한 회원은
대표 상태는 현재 상태를 유지하고
예약 배지만 추가로 표시한다.

예:
- [정상] [주2회] [휴원예정]
- [정상] [주1회] [퇴원예정]

예약 배지는 카드 오른쪽 상단 또는 상태 배지 옆에 작게 붙인다.
모든 화면에서 같은 위치에 보여야 한다.
검색 결과에서도 동일하게 보여야 한다.

=====================================
[휴원/퇴원 처리 시 이동시점 선택 팝업]
선생님 모드와 관리자 모드 모두 동일하게 적용한다.

휴원 또는 퇴원 버튼을 누르면 바로 상태를 바꾸지 말고
아래 팝업을 띄운다.

팝업 제목:
이동 시점을 선택하세요

버튼:
- 즉시 이동
- 다음 달 이동
- 취소

=====================================
[즉시 이동 규칙]
1. 휴원 + 즉시 이동
- 즉시 휴원 리스트에 추가
- 정상/주1/주2/주3 리스트에서 즉시 제거
- 대표 상태를 휴원으로 변경
- 현재 반에서도 제외 처리
- 회원카드는 [휴원] 상태로 표시

2. 퇴원 + 즉시 이동
- 즉시 퇴원 리스트에 추가
- 정상/주1/주2/주3 리스트에서 즉시 제거
- 대표 상태를 퇴원으로 변경
- 현재 반에서도 제외 처리
- 회원카드는 [퇴원] 상태로 표시
- 관리자 최종 퇴원 처리 전까지 기록은 유지
- 관리자 최종 퇴원 처리 시 해당 수영장 접근 즉시 차단

=====================================
[다음 달 이동 규칙]
1. 휴원 + 다음 달 이동
- 대표 상태는 현재 상태 유지
- pendingStatusChange = 휴원 저장
- pendingEffectiveMode = next_month 저장
- pendingEffectiveMonth 저장
- 카드에 [휴원예정] 배지 표시
- 현재 달에는 정상/주횟수 리스트에 그대로 남음
- 다음 달 수업 생성 시 실제 휴원으로 전환
- 전환 시 휴원 리스트로 이동하고 정상/주횟수 리스트에서 제거

2. 퇴원 + 다음 달 이동
- 대표 상태는 현재 상태 유지
- pendingStatusChange = 퇴원 저장
- pendingEffectiveMode = next_month 저장
- pendingEffectiveMonth 저장
- 카드에 [퇴원예정] 배지 표시
- 현재 달에는 정상/주횟수 리스트에 그대로 남음
- 다음 달 수업 생성 시 실제 퇴원으로 전환
- 전환 시 퇴원 리스트로 이동하고 정상/주횟수 리스트에서 제거

=====================================
[다음 달 수업 생성 시 예약 반영]
다음 달 스케줄 생성 또는 다음 달 반편성 생성 시
pendingStatusChange가 있는 회원들을 확인해서 자동 반영한다.

반영 규칙:
- 휴원예정이면 휴원 상태로 변경하고 휴원 리스트로 이동
- 퇴원예정이면 퇴원 상태로 변경하고 퇴원 리스트로 이동
- 반영 완료 후 pendingStatusChange / pendingEffectiveMode / pendingEffectiveMonth 초기화

주의:
- 이 자동 반영은 “예약된 상태를 실행하는 것”이지
- 자동 휴원/자동 퇴원을 새로 만드는 기능이 아님
- 사용자가 직접 예약한 건만 실행

=====================================
[선생님 모드 반배정 화면 수정]
선생님 모드 회원관리 > 반배정에서
회원 제거 시 단순 제거하지 말고 상태 선택부터 하게 한다.

회원 제거 흐름:
1. 회원 제거 버튼 클릭
2. 상태 선택 팝업
   - 대기
   - 연기
   - 휴원
   - 퇴원

3. 휴원/퇴원 선택 시
   - 이동 시점 팝업 추가
   - 즉시 이동 / 다음 달 이동 / 취소

4. 대기/연기는 바로 적용

대기/연기/휴원/퇴원 처리 결과는
회원관리 리스트와 반배정 리스트에 즉시 반영한다.

=====================================
[대기자 명단 통합 유지]
대기/연기/퇴원 관련 회원은 통합 리스트에서 필터로 구분 가능하게 유지한다.
필터:
- 전체
- 대기
- 연기
- 퇴원

단, 휴원과 퇴원은 회원관리 메인 상단 필터에서도 따로 접근 가능해야 한다.

=====================================
[휴원 복귀 규칙]
휴원 회원 복귀 시
선생님이 복귀 처리하면서 주1/주2/주3 중 복귀 횟수를 선택한다.

복귀 처리 흐름:
1. 휴원 회원 선택
2. 복귀 버튼 클릭
3. 복귀 횟수 선택
   - 주1회
   - 주2회
   - 주3회

4. 원래 반 또는 가능한 반에 자리 있으면 즉시 배정
5. 필요한 배정 수가 다 채워지면 대표 상태를 정상으로 변경
6. 일부만 배정되면 대표 상태는 미배정으로 두고 주횟수 태그는 유지

예:
휴원 회원이 주3회 복귀 선택
- 반 3개 모두 확보되면 정상 + 주3회
- 반 2개만 확보되면 미배정 + 주3회
- 이후 1개 추가 배정 완료 시 정상 전환

=====================================
[퇴원 처리 이후 접근 차단 규칙]
퇴원 즉시 리스트 이동은 가능하지만
해당 수영장 접근 차단은 관리자 최종 퇴원 처리 시 실행한다.

관리자 최종 퇴원 처리 시:
- 해당 학생의 해당 수영장 접근 권한 즉시 차단
- 학부모가 그 수영장을 열면 아래 메시지 표시

메시지:
아쉽게도 [수영장명]의 정보를 이용할 수 없습니다.
수영장 등록 후 다시 이용해 주세요.

중요:
- 앱 전체 탈퇴 아님
- 계정 전체 삭제 아님
- 해당 수영장만 접근 차단
- 다른 수영장 연결은 유지

=====================================
[리스트 반영 규칙 최종]
전체:
- 모든 회원 표시

정상:
- memberStatus = 정상

미배정:
- memberStatus = 미배정

휴원:
- memberStatus = 휴원

퇴원:
- memberStatus = 퇴원

주1회:
- memberStatus = 정상 이고 assignedClassCount = 1

주2회:
- memberStatus = 정상 이고 assignedClassCount = 2

주3회:
- memberStatus = 정상 이고 assignedClassCount >= 3

미연결:
- parentConnectionStatus = 미연결

주의:
- 검색 결과도 동일한 계산 로직 사용
- 화면마다 따로 계산하지 말고 공통 selector / helper 함수 사용

=====================================
[공통 계산 함수]
화면별로 조건문 따로 만들지 말고 공통 계산 함수로 통일한다.

예시 함수:
- getMemberPrimaryStatus(member)
- getMemberFrequencyTag(member)
- getMemberPendingBadge(member)
- getMemberVisibleClassInfo(member)
- getMemberConnectionStatus(member)

모든 리스트와 모든 카드와 검색 결과는 이 함수들만 사용해서 렌더링한다.

=====================================
[삭제 / 아카이브 규칙]
아카이브:
- 데이터 유지
- 학부모 접근 불가
- 과금 제외
- 복귀 가능

삭제:
- 관리자만 가능
- 영구 삭제
- 2단계 확인 필수
- 복구 불가

=====================================
[렌더링 / 상태관리 규칙]
- 회원카드 컴포넌트 하나만 사용
- 검색 결과도 동일 카드 사용
- 화면 진입 시 최신 데이터 fetch
- 저장 후 즉시 재조회
- 낙관적 업데이트 금지
- 리스트 key 안정화
- 불필요한 re-render 방지
- 관리자/선생님 화면 모두 같은 데이터 기준 사용

=====================================
[수정 후 반드시 테스트]
1. 휴원 선택 시 즉시 이동 / 다음 달 이동 팝업이 뜨는지
2. 퇴원 선택 시 즉시 이동 / 다음 달 이동 팝업이 뜨는지
3. 즉시 이동 선택 시 해당 리스트로 바로 이동하는지
4. 다음 달 이동 선택 시 현재 리스트 유지 + 예약 배지가 보이는지
5. 다음 달 수업 생성 시 예약 상태가 실제 반영되는지
6. 검색 결과에서도 회원카드 디자인이 동일한지
7. 정상/미배정/휴원/퇴원 대표 상태가 하나만 보이는지
8. 주1/주2/주3 태그가 올바르게 계산되는지
9. 휴원 복귀 시 배정 수에 따라 정상/미배정 전환이 맞는지
10. 관리자 최종 퇴원 처리 시 해당 수영장 접근이 차단되는지
11. 다른 수영장 연결은 그대로 유지되는지
