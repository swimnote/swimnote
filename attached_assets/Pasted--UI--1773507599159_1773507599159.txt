지금부터는 관리자 화면의 삭제 기능을 전면 재구성해줘.
중요한 건 “작업 완료”라고 말하는 게 아니라 실제로 눌렀을 때 작동하는 상태여야 한다.
UI만 만들지 말고, 관리자/선생님/학부모가 같은 원본 데이터를 보도록 삭제 로직까지 연결해라.

[최우선 목표]
1. 관리자에서 삭제 기능이 실제 작동해야 한다.
2. 현재 탭 구조와 관리목록 레이아웃은 유지하되, 삭제 UX를 공통 인터페이스로 통일한다.
3. 반삭제 / 회원삭제 / 공지삭제 / 기타 목록 삭제가 모두 같은 방식으로 작동해야 한다.
4. 삭제 시 확인창이 반드시 뜨고, 확인 후에만 삭제된다.
5. 개별삭제 + 부분선택삭제 + 전체선택삭제를 지원한다.
6. 삭제 결과는 선생님 로그인 / 학부모 로그인에도 즉시 반영되어야 한다.
7. 수정 후 실제 동작 테스트까지 완료하고, 어떤 화면에서 어떻게 확인했는지 보고해라.

==================================================
1. 공통 삭제 UI/UX로 관리자 화면 통일
==================================================

현재 관리자 탭과 목록 구조는 유지한다.
단, 삭제 관련 UI는 모든 목록에서 아래 공통 규칙을 적용한다.

[공통 규칙]
- 각 목록 우측 상단에 “선택모드” 또는 체크박스 아이콘 추가
- 선택모드 진입 시 각 카드/행 왼쪽에 체크박스 노출
- 상단에 전체선택 체크박스 표시
- 하단 또는 상단 액션바에 아래 버튼 표시
  - 선택해제
  - 부분삭제
  - 전체삭제
- 선택된 개수 표시
  - 예: 선택됨 3개
- 선택된 항목이 없으면 부분삭제 버튼 disabled
- 전체선택 시 현재 필터 기준으로 보이는 목록 전체 선택
- 현재 탭/필터/검색 결과 기준으로만 전체선택 적용

[삭제 확인 UX]
- 개별 삭제든 부분삭제든 전체삭제든 무조건 확인 모달 표시
- 모달 문구는 대상에 따라 다르게 표시
- 삭제 중에는 버튼 비활성화 + 로딩 표시
- 삭제 완료 후 토스트 표시
- 삭제 실패 시 실패 토스트 + 에러 로그 표시

==================================================
2. 관리자 내 모든 목록에 선택삭제 공통 적용
==================================================

관리자 앱 내 “목록형 화면”에는 전부 동일 삭제 시스템을 넣어라.

반드시 적용할 대상:
- 반 관리 목록
- 회원 관리 목록
- 승인 대기 목록 (삭제 정책이 있으면 적용)
- 공지사항 목록
- 기타 관리자 내 카드형/리스트형 데이터 목록

중요:
- 현재 탭과 관리목록은 그대로 두고 삭제 인터페이스만 공통화
- 화면마다 제각각 구현하지 말고 공통 컴포넌트/훅으로 만들 것

권장 구조:
- components/admin/SelectableListHeader.tsx
- components/admin/SelectionActionBar.tsx
- hooks/useSelectionMode.ts
- services/deleteService.ts

3. 반 삭제 정책
==================================================

반 삭제는 실제 작동해야 한다.
지금처럼 버튼이 있어도 동작 안 하는 상태는 절대 안 된다.

[반 삭제 동작]
- 관리자 반관리에서 개별삭제 가능
- 여러 개 체크 후 부분삭제 가능
- 전체선택 후 전체삭제 가능
- 삭제 전 확인 모달 표시

[확인 문구]
“선택한 반을 삭제하면 소속 회원은 삭제되지 않고 미배정 상태로 변경됩니다. 선생님/출결/스케줄/학부모 화면에도 즉시 반영됩니다. 삭제하시겠습니까?”

[반 삭제 실제 처리]
- classes는 hard delete 말고 soft delete 권장
  - isDeleted = true
  - deletedAt 저장
- 해당 반 소속 active 회원은 삭제하지 말고 미배정 처리
  - classId = null
  - className = null
  - assignmentStatus = 'unassigned'
- 선생님 반목록에서 제거
- 오늘 스케줄/출결 대상에서 제거
- 학부모 자녀 반정보에서도 제거 또는 “미배정” 표시
- 관리자 반 카드 인원수 재계산

==================================================
4. 회원 삭제 정책
==================================================

회원 삭제도 실제 작동해야 한다.

[회원 삭제 동작]
- 개별삭제 가능
- 여러 개 선택 후 부분삭제 가능
- 전체선택 후 전체삭제 가능
- 삭제 전 확인 모달 표시

[확인 문구]
“선택한 회원은 운영 목록에서 제거되고 삭제회원으로 이동됩니다. 학부모 계정은 유지되며 기존 수업정보는 보존됩니다. 삭제하시겠습니까?”

[회원 삭제 실제 처리]
- student hard delete 금지
- status = 'deleted'
- active = false
- deletedAt 저장
- archivedReason = 'member_deleted'
- 현재 반배정 해제
  - classId = null
  - className = null
- 출결 대상에서 제외
- 선생님 학생목록에서 제외
- 관리자 기본 회원목록에서 제외
- 삭제회원 탭/필터에서 조회 가능
- 학부모 계정은 유지
- 학부모는 추후 자녀 수업정보/기록 확인 가능

==================================================
5. 공지사항 삭제 정책
==================================================

공지사항도 삭제가 필요하면 동일 인터페이스로 맞춰라.

[공지 삭제]
- 개별삭제 / 부분삭제 / 전체삭제 지원
- 확인 모달 표시
- 관리자 공지 목록에서 즉시 제거
- 학부모 공지 목록에도 즉시 반영
- soft delete 또는 hidden 처리 가능하나 조회 필터 일관성 유지

==================================================
6. 가장 중요한 구조: 모든 권한 동기화
==================================================

삭제 후 관리자만 바뀌고 선생님/학부모 화면에 남아 있으면 실패다.
반드시 같은 원본 데이터 기준으로 즉시 반영되게 만들어라.

[반드시 지킬 것]
- 관리자 / 선생님 / 학부모가 같은 데이터 소스를 본다
- 화면별 mock 배열 금지
- 각 탭 useState 임시 삭제 금지
- service/repository 계층에서 실제 삭제 처리 후 전역 상태 갱신
- 삭제 후 refetch 또는 query invalidate
- soft delete 된 데이터는 각 역할별 화면에서 필터링

[즉시 반영 대상]
반 삭제 후:
- 관리자 반목록
- 관리자 회원목록 반배정 상태
- 선생님 내 반
- 선생님 오늘스케줄
- 선생님 출결
- 학부모 자녀 반정보

회원 삭제 후:
- 관리자 회원목록
- 관리자 반 인원수
- 선생님 학생목록
- 선생님 출결대상
- 학부모 자녀목록/상세

공지 삭제 후:
- 관리자 공지목록
- 학부모 공지목록

==================================================
7. 공통 선택모드 구현 방식
==================================================

아래 방식으로 공통 선택 시스템을 구현해라.

[공통 상태]
- selectionMode: boolean
- selectedIds: string[]
- isAllSelected: boolean

[공통 기능]
- toggleSelectionMode()
- toggleItemSelection(id)
- selectAll(visibleIds)
- clearSelection()
- deleteSelected()
- deleteOne(id)

[전체선택 규칙]
- 검색/필터 후 현재 보이는 항목만 전체선택
- 숨겨진 항목까지 삭제하면 안 됨

[UI 규칙]
- 기본모드: 기존 화면 유지 + 개별 삭제 버튼 가능
- 선택모드: 카드 왼쪽 체크박스 표시, 우측 개별삭제 버튼 숨기거나 비활성화 가능
- 상단바: 전체선택 체크박스 + 선택 개수 표시
- 액션바: 부분삭제 / 전체삭제 / 선택해제

8. 실제 동작 보장용 디버깅 추가
==================================================

지금 가장 큰 문제는 “됐다”고 해놓고 실제로는 안 되는 것이다.
그래서 아래 로그를 반드시 추가해라.

[반 삭제 로그]
- [admin][deleteClass] click classId=...
- [admin][deleteClass] selectedCount=...
- [admin][deleteClass] unassign students count=...
- [admin][deleteClass] class soft deleted success
- [admin][deleteClass] UI refresh complete

[회원 삭제 로그]
- [admin][deleteStudent] click studentId=...
- [admin][deleteStudent] selectedCount=...
- [admin][deleteStudent] archive success
- [admin][deleteStudent] parent link preserved
- [admin][deleteStudent] UI refresh complete

[공지 삭제 로그]
- [admin][deleteAnnouncement] click id=...
- [admin][deleteAnnouncement] success
- [admin][deleteAnnouncement] parent view updated

그리고 console만 찍지 말고 실제 DB/store 값이 바뀌는지 확인해라.

==================================================
9. 반드시 해결해야 할 현재 문제 점검
==================================================

지금 삭제 기능이 안 되는 원인을 전부 확인해라.

반드시 점검:
- 삭제 버튼 onPress/onClick 연결 여부
- id 전달 누락 여부
- 함수 실행 후 return만 하고 저장 안 하는 문제
- 화면에서만 제거하고 DB/store는 안 바꾸는 문제
- 관리자/선생님/학부모가 서로 다른 mock 데이터 쓰는 문제
- soft delete 후 필터가 잘못되어 계속 보이는 문제
- 캐시가 남아서 삭제 후 다시 보이는 문제

이 중 하나라도 남아 있으면 수정 완료로 보고하지 마라.

- 실패 시 사용자에게 이유 표시
- 모바일 화면에서도 체크박스/액션바 UX 깨지지 않게 처리

12. 절대 하면 안 되는 것
==================================================

- UI만 만들고 삭제 함수 미연결
- 목록에서만 사라지는 fake delete
- 반 삭제 시 회원까지 삭제
- 회원 삭제 시 학부모 계정까지 삭제
- 화면별 임시배열로 따로 관리
- 테스트 안 하고 완료했다고 보고
- 전체삭제가 숨겨진 항목까지 삭제하는 것

이 작업은 MVP 운영 구조 보완 작업이다.
실제 눌렀을 때 작동하는 수준으로 수정해라.
