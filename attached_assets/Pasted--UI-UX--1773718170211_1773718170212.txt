# 스윔노트 학부모/선생님 모드 구현 명령서

## 작업 목표
지금까지 합의된 학부모/선생님 모드 구조를 실제로 작동하는 UI/UX와 데이터 연동으로 구현한다.  
화면만 보이게 만드는 더미 UI는 금지하며, 저장/이동/삭제/읽음/다운로드/연동 상태가 실제 동작해야 한다.

---

## 0. 공통 원칙

- 모바일 우선 UI로 구현
- iOS/Android 모두 자연스럽게 보이게 safe area 처리
- 모든 버튼은 실제 클릭 동작 연결
- 팝업은 닫기 버튼 + 바깥 터치 닫기 지원
- 긴 리스트는 스크롤 가능해야 함
- 저장 후 재접속해도 데이터 유지
- 학부모와 선생님 화면은 서로 연결된 데이터로 실시간 반영
- 선생님은 반 단위로 작성해도 학부모는 자기 아이 정보만 보이게
- 다른 학부모 존재를 암시하는 숫자/좋아요 수/공개 댓글 수 노출 금지

---

## 1. 학부모 모드 하단 탭

하단 탭 순서:
1. 홈
2. 수업피드백
3. 앨범
4. 출결
5. 쇼핑
6. 더보기

### 쇼핑 탭
- 현재는 기능 없이 준비중 화면만 구현
- 문구 예시:
  - 쇼핑 준비중
  - 곧 다양한 수영용품과 혜택이 제공될 예정입니다.

---

## 2. 학부모 홈 화면

### 2-1. 상단 아이 카드
- 현재 선택된 아이 이름 표시
- 아이 이름 옆에 레벨 아이콘만 표시
- 레벨 설명/텍스트는 홈에서 노출하지 않음
- 형제자매 여러 명이면 아이 전환 가능
- 아이 전환 시 홈, 수업피드백, 앨범, 출결, 공지가 해당 아이 기준으로 즉시 갱신

### 2-2. 레벨 아이콘
- 수영장마다 다른 레벨 체계를 지원할 수 있게 관리자 설정 구조 반영
- 예시:
  - 숫자형 레벨
  - 모자색형 레벨
- 홈에는 아이콘만 표시
- 자세한 설명은 레벨 관련 화면에서 확인

### 2-3. 바로가기 메뉴
아이콘 + 텍스트 버튼으로 배치:
- 수업피드백
- 앨범
- 출결
- 레벨
- 공지

각 버튼은 실제 해당 탭/상세 화면으로 이동

### 2-4. 최근 업데이트
- 최대 3개까지만 노출
- 각 카드 설명은 3줄까지만 보이고 말줄임 처리
- 유형:
  - 수업피드백
  - 사진 업로드
  - 영상 업로드
  - 공지
  - 레벨 관련 업데이트
- 카드 누르면 해당 탭/상세 화면으로 이동

### 2-5. 광고 영역
- 홈에서 광고 영역이 일부 보이게 배치
- 최근 업데이트 아래에서 광고 상단이 살짝 보여야 함

## 3. 학부모 수업피드백

### 명칭
사용자 노출 명칭은 전부 `수업피드백` 으로 통일

### 표시 원칙
- 선생님은 반 단위로 작성
- 학부모는 우리 아이에게만 온 것처럼 개인화해서 표시
- 다른 학부모 존재를 느끼게 하는 요소 노출 금지

### 카드 구성
각 카드에 표시:
- 담당 선생님 이름
- 날짜
- 수업피드백 본문
- 개별 피드백
- 첨부 사진 썸네일 (있을 때)

### 반응 버튼
- 좋아요
- 감사합니다
- 쪽지달기

숫자 표시 없음

---

## 4. 학부모 쪽지 기능

### 기본 원칙
- 공개 댓글 없음
- 게시물 기반 비공개 대화만 허용

### 구조
- 학부모가 `쪽지달기`로 시작
- 해당 수업피드백 게시물과 연결된 비공개 쪽지 스레드 생성
- 학부모는 질문/추가 질문 가능
- 선생님은 답변만 가능
- 관리자 열람만 가능

### 팝업
- 가운데 모달형 대화창
- 닫기 버튼
- 바깥 터치 닫기 가능
- 텍스트만 허용
- 첨부파일/이미지/영상 금지

### 삭제
- 학부모: 본인 메시지만 삭제 가능
- 선생님: 본인 메시지만 삭제 가능
- 삭제 시 바로 영구삭제하지 않고 삭제메세지함 이동
- 자동 삭제 없음
- 복구 가능
- 영구삭제 가능
- 삭제해도 상대방에게 알림 보내지 않음

## 5. 학부모 앨범

### 5-1. 앨범 탭 상단 구조
순서:
1. 개인 앨범
2. 반 전체 앨범

### 5-2. 명칭
- `서태웅 앨범` 형식
- 사진첩 대신 앨범 사용

### 5-3. 개인 앨범 내부
2개 메뉴:
- 사진 앨범
- 영상 앨범

영상 앨범은 영상 서비스 옵션이 꺼져 있으면 잠금 표시  
옵션 활성화 시 잠금 해제

### 5-4. 반 전체 앨범
- 수영장 모든 선생님 업로드 가능
- 반 구분 없음
- 모든 학부모 열람 가능
- 최신순 정렬
- 다운로드 가능
- 사진/영상 동일 구조

### 5-5. 개인 앨범
- 담당 선생님만 자기 담당 학생에게 업로드 가능
- 다른 반 아이는 업로드 대상 목록에 보이면 안 됨
- 해당 학생 학부모만 열람 가능
- 최신순 정렬
- 학생별 앨범에 계속 누적
- 날짜별 폴더 없음
- 다운로드 가능

### 5-6. 보기 UX
- 기본: 그리드
- 사진 1장 누르면 슬라이드 보기
- 영상: 썸네일만 표시, 클릭 시 다운로드
- 영상 스트리밍 없음

### 5-7. 업로드 제한
- 사진: 한 번에 최대 10장
- 영상: 한 번에 1개

### 5-8. 다운로드
- ZIP 금지
- 선택 버튼으로 다중 선택 다운로드
- 개별 원본 파일 다운로드 방식
- 다운로드 중 로딩 표시
- 완료 시 토스트 표시

## 10. 선생님 홈 - 오늘 할 일

항목:
- 출결 체크
- 수업피드백 미작성
- 보강 관리
- 읽지 않은 쪽지 n건

### 10-1. 수업피드백 미작성 표시
큰 항목 아래 2줄 표시:
- 어제까지 미작성 n
- 오늘 미작성 n

#### 계산 규칙
- 오늘 미작성은 당일 마지막 수업 종료 후부터 계산
- 마지막 수업 종료 전에는 오늘 미작성 0 또는 미표시
- 어제까지 미작성은 오늘 이전 날짜 중 미작성된 수업피드백 총합

#### 표시
- 어제까지 미작성은 경고 느낌으로 눈에 띄게

#### 클릭 시
- 모달이 아니라 수업피드백 목록/작성 첫 페이지로 이동

### 10-2. 읽지 않은 쪽지
- 오늘 할 일에 `읽지 않은 쪽지 n건` 표시
- n은 안 읽은 쪽지 수 기준

---

## 11. 선생님 쪽지 흐름

### 11-1. 홈에서 읽지 않은 쪽지 클릭
- 화면 절반 높이 모달 오픈
- 제목: `읽지 않은 쪽지`

### 11-2. 목록 표시
각 항목 표시:
- 서태웅 부모님
- 0317

표시만 사용:
- 게시물 제목 없음
- 학생 이름 없음

안 읽은 항목은 앞에 점 표시

### 11-3. 항목 클릭
- 해당 수업피드백 게시물로 이동
- 연결된 쪽지 대화창 자동 오픈

## 12. 선생님 수업피드백 작성

- 자유 입력 메모창
- 사진 첨부 가능
- 템플릿 모달 열기 버튼(화살표)

- 하루 1개 작성 기준

## 13. 수업피드백 템플릿 모달

### 탭 순서
1. 선생님 커스텀
2. 초급
3. 중급
4. 상급
기본 시작은 위와 같지만, 관리자에서 추가 가능

### 문장 삽입 규칙
- 문장 하나 선택 시 즉시 본문에 삽입
- 현재 커서 위치 기준 삽입
- 문장 하나 선택할 때마다 줄바꿈해서 한 줄씩 추가
- 선택 순서대로 들어감
- 같은 문장 중복 선택 방지

### 모달 상단
- 선택한 문장 미리보기 목록 표시

### 모달 기능
- 바로 전 문장 지우기
- 전체 초기화
- 전체 초기화는 이번 모달에서 추가한 템플릿 문장만 삭제
- 직접 작성한 텍스트는 유지

### 닫기
- 반대 화살표 버튼
- 바깥 터치 닫기

### 14-1. 선생님 커스텀 문장

- 최대 30개

- 더보기
- 설정

기능:

- 탭 추가

## 17. 실시간 연동 원칙

이 부분은 전부 실제 데이터 저장과 연결해야 함.

### 학부모 ↔ 선생님
- 선생님이 수업피드백 저장하면 학부모 최근 업데이트/수업피드백에 즉시 반영
- 선생님이 수업피드백에 사진 첨부하면 학부모 카드에 즉시 표시
- 학부모가 쪽지 작성하면 선생님 홈 `읽지 않은 쪽지 n건` 즉시 증가
- 선생님이 쪽지 열면 읽음 처리 반영
- 선생님 답변은 학부모 게시물 쪽지창에 즉시 반영

### 앨범
- 전체앨범 업로드 → 모든 학부모 전체앨범 반영
- 개인앨범 업로드 → 해당 학생 학부모에게만 반영
- 영상옵션 OFF → 영상 앨범 잠금 유지
- 영상옵션 ON → 잠금 해제

### 미작성 계산
- 선생님 실제 수업 일정 + 작성 여부 비교해서
  - 오늘 미작성
  - 어제까지 미작성
자동 계산

### 템플릿
- 선생님 커스텀 수정 → 본인 작성 모달 반영
- 관리자 템플릿 수정 → 전체 선생님 작성 모달 반영

## 19. 구현 후 반드시 검수할 것

### 학부모
- 아이 전환 시 데이터 즉시 변경
- 최근 업데이트 3개 노출
- 광고 일부 노출
- 수업피드백 카드에 사진 보임
- 공지 전체/반 구분
- 개인 앨범이 먼저
- 영상 잠금 정상
- 출결 3개 상태만 표시
- 쇼핑 준비중 화면

### 선생님
- 오늘 할 일 4개 항목 표시
- 미작성 계산 정확
- 마지막 수업 종료 후 오늘 미작성 반영
- 읽지 않은 쪽지 n건 표시
- 절반 모달 목록 표시
- 목록 클릭 시 게시물 이동 + 쪽지 모달 열림
- 저장 후 완료 팝업 노출
- 팝업에서 다음 미작성 작성 이동
- 팝업 닫으면 일지 목록 이동
- 작성 중 이탈 시 임시저장
- 일지 목록 임시저장 배지 표시

### 템플릿
- 커서 위치 삽입
- 줄바꿈 삽입
- 중복 선택 방지
- 선택 문장 상단 미리보기
- 되돌리기
- 전체 초기화
- 커스텀 문장 저장/수정/삭제
- 관리자 탭/문장 수정 후 반영

### 연동
- 학부모 쪽지 ↔ 선생님 오늘 할 일 실시간 연동
- 수업피드백 ↔ 학부모 피드 실시간 연동
- 앨범 업로드 ↔ 학부모 앨범 반영
- 삭제메세지함 동작 확인

## 20. 보고 방식
작업 후 `구현 완료`라고만 말하지 말고, 아래 항목별로 실제 작동 결과를 보고할 것.

1. 화면별 구현 완료 여부
2. 버튼/탭/팝업 실제 이동 여부
3. 저장/재접속 유지 여부
4. 실시간 연동 여부
5. 임시저장/삭제/복구 여부
6. 미구현/오류/임시처리 항목
