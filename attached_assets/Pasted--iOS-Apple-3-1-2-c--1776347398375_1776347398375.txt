[스윔노트 iOS 리젝 최종 해결 지시문 — Apple 로그인 + 구독 표기 3.1.2(c)]

지금부터는 임시 수정 금지.
목표는 “다음 심사에서 같은 사유가 절대 재발하지 않게” Apple 로그인과 구독 표기 문제를 동시에 끝내는 것이다.

현재 리젝 2건:
1) Guideline 2.1(a) - Sign in with Apple 실패
2) Guideline 3.1.2(c) - 자동 갱신 구독 안내 정보/EULA/Privacy Policy 링크 부족

━━━━━━━━━━━━━━━━━━
1. Apple 로그인 — 실패 위치 확정
━━━━━━━━━━━━━━━━━━

추측 금지. 아래 단계 로그를 반드시 남겨라.

STEP1: signInAsync 호출 시작
STEP2: Apple credential 수신 성공/실패
STEP3: 서버 요청 시작 여부
STEP4: 서버 응답/DB 처리 성공/실패

반드시 실기기에서 2가지 모두 테스트:
- 기존 앱 삭제 후 신규 설치
- 이전 버전에서 업데이트 설치

기기:
- iPhone
- iPad 가능하면 같이

보고서에는 반드시 포함:
- STEP1~STEP4 실제 로그
- 실패가 STEP2인지 STEP3인지 STEP4인지 확정
- 신규 설치 성공/실패 스크린샷
- 업데이트 설치 성공/실패 스크린샷

중요:
- “Portal 설정 문제 같다” 같은 추측 금지
- “서버 문제 같다” 같은 추측 금지
- 실패 위치를 단계로 확정해서 제출

━━━━━━━━━━━━━━━━━━
2. 구독 화면 — 3.1.2(c) 근본 해결
━━━━━━━━━━━━━━━━━━

현재 구독 화면을 Apple 요구사항에 맞게 수정하라.

앱 안 결제/구독 화면에 반드시 보여야 하는 항목:
- 구독 서비스/플랜 이름
- 구독 기간
- 가격
- Terms of Use(EULA) 링크
- Privacy Policy 링크

필수 조건:
- 링크는 실제 탭 가능한 버튼/텍스트여야 함
- 리뷰어가 결제 화면에서 바로 볼 수 있어야 함
- 숨겨진 화면이나 설정 깊숙한 곳에 두지 말 것

가능하면 Apple 권장 방식인 SubscriptionStoreView 적용 검토.
단, 기존 구조 유지가 더 빠르고 안정적이면 현재 paywall에 요구사항을 명시적으로 추가해도 됨.

보고서에는 반드시 포함:
- 수정 전/후 화면 비교
- 가격, 기간, 서비스명, EULA, Privacy 링크가 한 화면에서 보이는 스크린샷
- iPhone 기준 결제 진입 플로우 스크린샷

━━━━━━━━━━━━━━━━━━
3. App Store Connect 메타데이터 수정
━━━━━━━━━━━━━━━━━━

앱 수정과 별개로 App Store Connect도 같이 수정하라.

반드시 확인:
- Privacy Policy 필드에 실제 링크 입력
- Terms of Use (EULA) 링크 처리

분기:
- Apple standard EULA 사용 시 → App Description에 Terms of Use 링크 명시
- Custom EULA 사용 시 → App Information / License Agreement 쪽에 커스텀 EULA 입력

보고서에 포함:
- Privacy Policy 필드 캡처
- EULA 링크/설정 캡처
- App Description 반영 캡처

━━━━━━━━━━━━━━━━━━
4. 최종 검증
━━━━━━━━━━━━━━━━━━

완료 조건:
- 신규 설치 후 Apple 로그인 성공
- 업데이트 설치 후 Apple 로그인 성공
- 구독 화면에서 서비스명/기간/가격/EULA/Privacy 링크 전부 노출
- 메타데이터에 Privacy Policy/EULA 반영 완료

━━━━━━━━━━━━━━━━━━
5. 최종 제출물 형식
━━━━━━━━━━━━━━━━━━

1) Apple 로그인 실패 위치 확정
2) 수정한 코드/화면 목록
3) 구독 화면 수정 내용
4) App Store Connect 수정 내용
5) 테스트 결과
6) 스크린샷 증빙
7) 새 빌드 제출 가능 여부 판정

중요:
