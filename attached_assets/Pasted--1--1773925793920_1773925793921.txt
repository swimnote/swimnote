이번 작업은 절대 범위를 넓히지 말고
“관리자 모드 > 사람 탭 화면 1개”만 수정하라.

현재 문제:
사람 탭 상단의 작은 필터/분류 칩(회원, 학부모, 선생님, 승인, 미등록 등)이
아래 리스트 영역 때문에 밀리거나 잘려서
정상적으로 보이지 않고 클릭도 불편하다.

이건 기능 문제가 아니라 레이아웃 깨짐 문제다.
이번에는 다른 화면, 다른 기능, 다른 탭은 절대 건드리지 말고
관리자 모드 > 사람 탭 상단 칩 영역만 원래처럼 정상 복구하라.

==================================================
[작업 범위 제한]

오직 아래만 수정:
- 관리자 모드
- 사람 탭
- 상단 칩/탭 영역 레이아웃

절대 수정 금지:
- 다른 탭
- 다른 모드
- 데이터 구조
- 사람 목록 데이터
- 검색 기능 로직
- 카드 내용
- 탭 이동 규칙
- 뒤로가기 규칙

==================================================
[목표]

사람 탭 상단의 칩/탭이
1. 위아래 잘리지 않고
2. 한 줄에서 정상 표시되고
3. 터치 영역이 충분하며
4. 다시 정상 클릭 가능하도록
예전 정상 상태처럼 복구할 것

==================================================
[반드시 만족해야 하는 조건]

1. 상단 칩 영역 높이를 충분히 확보할 것
2. 칩 텍스트가 세로로 잘리거나 위쪽이 가려지지 않을 것
3. 칩의 padding / lineHeight / minHeight / alignItems 를 정상화할 것
4. 칩 컨테이너와 아래 리스트 사이 간격을 확보할 것
5. 아래 카드 리스트가 상단 칩 영역을 침범하지 않게 할 것
6. 칩 영역 z-index / position 문제 있으면 바로잡을 것
7. 칩 터치 영역이 충분해서 정상적으로 다시 눌릴 것

==================================================
[가능성 높은 원인 - 우선 확인]

먼저 아래 원인을 점검하라.

1. 상단 칩 row/container의 height 또는 minHeight 부족
2. overflow: hidden 때문에 칩 윗부분이 잘리는 문제
3. paddingVertical / lineHeight / fontSize 불일치
4. 칩 영역과 검색창/리스트 사이 margin collapse 또는 음수 margin
5. FlatList/ScrollView contentContainerStyle이 상단 영역을 덮는 문제
6. position:absolute 또는 zIndex 충돌
7. safe area / header spacing 계산 오류

==================================================
[수정 방식]

수정은 “작게” 하라.
전체 레이아웃을 갈아엎지 말고
사람 탭 상단 칩 영역의 컨테이너 구조를 복구하는 방식으로 처리하라.

우선순위:
1. 상단 칩 컨테이너 높이/정렬 복구
2. 검색창과의 간격 복구
3. 리스트 시작 위치 복구
4. 터치 가능 영역 복구

==================================================
[원하는 최종 상태]

- 상단 칩이 모두 또렷하게 한 줄에 보임
- 각 칩의 상하 여백이 정상
- 칩 텍스트가 안 잘림
- 회원/학부모/선생님/승인/미등록 칩이 정상 눌림
- 검색창은 그 아래 자연스럽게 위치
- 리스트 카드가 칩 영역을 밀어 올리거나 덮지 않음

==================================================
[테스트 - 반드시 실제 수행]

아래를 실제로 확인하라.

테스트 1.
관리자 모드 > 사람 탭 진입 시 상단 칩이 잘리지 않는지 확인

테스트 2.
회원 / 학부모 / 선생님 / 승인 / 미등록 칩이 모두 한 줄에서 정상 표시되는지 확인

테스트 3.
각 칩을 눌렀을 때 정상적으로 탭 전환/필터 전환이 되는지 확인

테스트 4.
검색창이 칩 아래에서 겹치지 않고 정상 위치에 있는지 확인

테스트 5.
아래 카드 리스트가 칩 영역을 덮거나 침범하지 않는지 확인

테스트 6.
작은 아이폰 화면에서도 칩 텍스트가 잘리지 않는지 확인

==================================================
[작업 후 보고 형식]

1. 수정한 파일명
2. 원인(왜 잘렸는지)
3. 어떤 레이아웃 값을 고쳤는지
4. 테스트 1~6 결과
5. 다른 화면은 건드리지 않았는지 명시

==================================================
[절대 금지]

- 사람 탭 전체를 새로 만드는 것 금지
- 다른 탭/다른 모드 건드리기 금지
- 데이터 구조 수정 금지
- 기능 추가 금지
- 임시 땜질로만 보이게 하고 터치 안 되게 두는 것 금지
