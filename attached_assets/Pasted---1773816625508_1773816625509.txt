지금부터 앱 전체 네비게이션 규칙을 전 모드 공통으로 수정하라.

이번 작업은
슈퍼관리자 / 관리자 / 선생님 / 학부모
모든 모드에 동일하게 적용되는
“뒤로가기 / 하단 탭 이동 / 같은 탭 재탭 초기화” 규칙 수정 작업이다.

가장 중요:
- 이번 작업은 팝업 동작 수정이 아니다
- 팝업은 기존 방식 그대로 유지한다
- 수정 대상은 “탭 기반 화면 이동 규칙”만이다
- 기능 로직, 데이터 구조, 승인/출결/정산/회원가입 로직은 건드리지 말 것
- 화면 내용도 가급적 건드리지 말고 네비게이션 동작만 수정할 것

==================================================
[1. 목표]

현재 문제:
- 상단 뒤로가기 버튼을 누르면 탭 첫 화면(모드 홈)으로 바로 가고 있음
- 상세 화면에 들어가면 하단 탭 버튼이 비활성화되거나 정상 동작하지 않음
- 다른 탭 갔다가 원래 탭으로 돌아오면 마지막 상태 유지가 안 되거나 이상함
- 탭을 눌렀을 때 항상 첫 화면으로 가버리거나, 반대로 초기화가 안 됨

이 문제를 아래 최종 규칙으로 통일하라.

==================================================
[2. 최종 규칙 - 전 모드 공통]

--------------------------------------
[규칙 1. 상단 뒤로가기]

상단 좌측 화살표 뒤로가기는
절대 탭의 첫 화면이나 모드 홈으로 바로 가면 안 된다.

반드시:
- 현재 탭 내부에서
- 직전 화면으로만
- 한 단계씩 이동해야 한다

예:
수업 탭 첫 화면 → 반 상세 → 학생 상세

여기서 뒤로가기 동작:
학생 상세 → 반 상세
반 상세 → 수업 탭 첫 화면

즉:
뒤로가기는 “현재 탭 내부 스택에서 한 단계 뒤로가기”만 수행한다

--------------------------------------
[규칙 2. 하단 탭 버튼 항상 활성]

모든 하단 탭 버튼은
상세 화면 / 하위 화면 안에서도
항상 눌릴 수 있어야 한다.

즉:
- 상세 들어갔다고 탭 버튼 비활성화 금지
- 어느 화면이든 하단 탭은 항상 동작

--------------------------------------
[규칙 3. 다른 탭으로 이동할 때 상태 유지]

A 탭 내부 상세 화면에 있다가
B 탭으로 이동하면

A 탭의 현재 위치/상태는 유지되어야 한다.

예:
수업 탭 > 반 상세
→ 사람 탭 이동
→ 다시 수업 탭 누름

결과:
수업 탭 첫 화면이 아니라
아까 보던 “반 상세” 상태 그대로 복귀해야 한다

즉:
각 탭은 마지막 보던 화면 상태를 유지하는
독립 스택 구조로 동작해야 한다

--------------------------------------
[규칙 4. 같은 탭 재탭 시 첫 화면 복귀]

이미 들어와 있는 탭에서
그 탭 버튼을 한 번 더 누르면
그 탭의 첫 화면으로 복귀해야 한다.

예:
수업 탭 > 반 상세 > 학생 상세 상태에서
하단 수업 탭 버튼 다시 누름
→ 수업 탭 첫 화면으로 복귀

즉:
- 탭 한 번 누르기 = 그 탭으로 이동 (마지막 상태 유지)
- 같은 탭 다시 누르기 = 그 탭 첫 화면으로 초기화

--------------------------------------
[규칙 5. 재탭 시 스크롤도 초기화]

같은 탭을 다시 눌러서
그 탭의 첫 화면으로 복귀할 때는

- 화면 경로 초기화
- 스크롤 위치도 맨 위로 초기화

즉:
단순히 화면만 돌아가는 게 아니라
탭 첫 화면의 초기 상태처럼 보여야 한다

==================================================
[3. 팝업은 이번 작업에서 제외]

중요:
이 규칙은 팝업에는 적용하지 않는다.

팝업은 기존 규칙 유지:
- 닫기 버튼
- 배경 터치 닫기
- 드래그 닫기
등 기존 방식 그대로 둔다

이번 작업은 팝업이 아니라
“하단 탭과 뒤로가기” 규칙만 수정하는 작업이다.

==================================================
[4. 적용 대상]

아래 모든 모드에 동일 규칙 적용:

- 슈퍼관리자 모드
- 관리자 모드
- 선생님 모드
- 학부모 모드

그리고 각 모드의 모든 하단 탭에 동일 적용:

예시:
- 홈
- 사람
- 수업
- 메신저
- 매출관리/정산/구독
- 더보기
등

모드별 탭 이름이 달라도
동작 규칙은 완전히 같아야 한다.

==================================================
[5. 구현 방식]

가능하면 각 탭별로 독립 navigation stack 또는 state memory 구조를 사용하라.

핵심 구현 조건:
1. 탭마다 마지막 진입 화면 상태 기억
2. 다른 탭으로 이동해도 해당 탭 상태 유지
3. 같은 탭 재탭 시 해당 탭 스택 초기화
4. 재탭 시 스크롤 위치도 top으로 이동
5. 상단 뒤로가기는 현재 탭 stack에서만 pop 동작

즉:
전역적으로 무조건 홈으로 reset 하는 방식 금지
탭 눌렀을 때 무조건 첫 화면 이동하는 방식 금지
상세 화면에서 탭 비활성화하는 방식 금지

==================================================
[6. 현재 잘못된 동작 제거]

반드시 아래 잘못된 동작을 없애라.

1. 뒤로가기 누르면 모드 홈/탭 홈으로 한 번에 가는 동작 제거
2. 상세 화면 진입 시 하단 탭 비활성화 제거
3. 다른 탭 갔다 오면 상태 사라지는 문제 제거
4. 탭 눌렀을 때 항상 첫 화면으로 리셋되는 문제 제거
5. 같은 탭 재탭 시 아무 일도 안 일어나는 문제 제거

==================================================
[7. 테스트 지시 - 반드시 실제 수행]

수정 후 반드시 실제 테스트하라.

--------------------------------------
테스트 1. 같은 탭 내부 뒤로가기

각 모드에서 아무 탭이나 선택 후
첫 화면 → 상세 → 더 상세까지 들어간 뒤
상단 뒤로가기 버튼을 눌러서
한 단계씩만 뒤로 가는지 확인

--------------------------------------
테스트 2. 다른 탭 이동 후 상태 유지

예:
수업 탭에서 상세 화면 진입
→ 사람 탭 이동
→ 다시 수업 탭 1회 클릭

결과:
수업 탭 첫 화면이 아니라
마지막 보던 상세 화면 상태로 복귀하는지 확인

이 테스트를 각 모드 주요 탭에서 수행할 것

--------------------------------------
테스트 3. 같은 탭 재탭 초기화

예:
수업 탭 상세 상태에서
하단 수업 탭 버튼 다시 한 번 누름

결과:
수업 탭 첫 화면으로 복귀하는지 확인

--------------------------------------
테스트 4. 재탭 시 스크롤 초기화

탭 첫 화면에서 아래로 스크롤한 뒤
다른 화면으로 들어갔다가
같은 탭 재탭

결과:
첫 화면으로 복귀하면서 스크롤도 맨 위로 올라가는지 확인

--------------------------------------
테스트 5. 하단 탭 활성 여부

각 모드의 상세/하위 화면에서
모든 하단 탭 버튼이 여전히 눌리는지 확인

--------------------------------------
테스트 6. 팝업 영향 없음

팝업 열기/닫기 동작은
이번 수정 후에도 기존과 동일한지 확인

==================================================
[8. 작업 전 필수 확인]

수정 전에 먼저 아래를 출력하라.

1. 현재 탭 네비게이션을 처리하는 파일 목록
2. 각 모드별 하단 탭 구성 파일
3. 현재 뒤로가기 처리를 담당하는 파일/컴포넌트
4. 현재 탭 비활성화 처리 코드가 있는지 여부
5. 현재 탭 클릭 시 홈으로 리셋시키는 코드가 있는지 여부

이걸 먼저 출력한 뒤 작업 시작

==================================================
[9. 작업 후 보고 형식]

작업 후 아래 순서로 보고하라.

1. 수정한 파일 목록
2. 적용한 네비게이션 구조 방식 설명
3. 탭 상태 유지 방식 설명
4. 같은 탭 재탭 초기화 방식 설명
5. 스크롤 초기화 방식 설명
6. 테스트 1~6 결과
7. 아직 남은 네비게이션 문제 있으면 명시

==================================================
[10. 절대 금지]

- 팝업 구조 수정 금지
- 기존 기능 내용 수정 금지
- 화면 내용 대규모 수정 금지
- 데이터 로직 수정 금지
- 회원가입/승인/출결/정산 로직 수정 금지
- “일단 홈으로 보내는 방식”으로 임시 처리 금지

이번 작업은
탭 구조 기반 네비게이션 규칙을
전 모드에 동일하게 바로잡는 작업이다.
