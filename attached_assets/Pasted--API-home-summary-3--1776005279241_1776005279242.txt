학부모 홈 최종 완성 작업입니다.
기존 API/상태 로직은 유지하고, home-summary 데이터를 기준으로 홈 화면 정보 구조를 재배치하세요.

목표:
- 들어오자마자 3초 안에 핵심 정보 파악
- 데이터는 충분하므로 “무엇을 먼저 보여줄지” 우선순위 중심으로 구성
- 카드형 구조 유지
- 새 소식(new/unread) 강조
- 빈 상태도 완성도 있게 처리

절대 원칙:
- API 변경 금지
- home-summary 구조 그대로 사용
- 카드 수는 4개로 고정
- 화면을 복잡하게 늘리지 말 것

━━━━━━━━━━━━━━
1. 상단 고정 요약 영역
━━━━━━━━━━━━━━

홈 최상단에 학생 핵심 상태를 먼저 보여주세요.

표시 항목:
1. 자녀 이름
2. 반 이름
3. 반 시간
4. 이번 달 출석: attended / total
5. 오늘 수업 여부

데이터 매핑:
- students.name
- students.class_group.name
- students.class_group.schedule_time
- summary.attendance.attended
- summary.attendance.total
- summary.today_schedule

표시 규칙:
- 오늘 수업이 있으면: "오늘 10:00 수업"
- 오늘 수업이 없으면: "오늘은 예정된 수업이 없습니다"
- class_group이 없으면: "반 정보가 아직 없습니다"

━━━━━━━━━━━━━━
2. 새 소식 스트립
━━━━━━━━━━━━━━

상단 요약 바로 아래에 얇은 새 소식 스트립을 추가하세요.

데이터:
summary.unread_counts

표시 조건:
- notices, diaries, photos, messages 중 1개라도 1 이상이면만 표시
- 전부 0이면 숨김

예시:
- "새 일지 1개 · 새 사진 3장"
- "새 공지 2개"

너무 길지 않게 2~3개까지만 묶어서 표시

━━━━━━━━━━━━━━
3. 메인 카드 4개 구조
━━━━━━━━━━━━━━

순서 고정:

① 최근 수업일지 카드
데이터:
summary.latest_diaries[0]

표시:
- lesson_date
- teacher_name
- student_note 우선
- student_note 없으면 common_content 일부(최대 2줄)
- is_new=true면 NEW 배지

빈 상태:
- "아직 수업 기록이 없습니다"

클릭:
- 일지 화면으로 이동

---

② 최근 사진 카드
데이터:
summary.latest_photos 최대 2장만 노출

표시:
- 썸네일 1~2장
- 사진 수 2장 이상이면 "+N" 형태 가능
- is_new가 하나라도 있으면 "새 사진" 표시

빈 상태:
- "아직 등록된 사진이 없습니다"

클릭:
- 사진 화면으로 이동

---

③ 공지사항 카드
데이터:
summary.latest_notices[0]

표시:
- title
- is_pinned면 "중요" 표시
- unread_counts.notices > 0이면 "새 공지 N개" 표시
- content는 1줄 또는 생략 가능

빈 상태:
- "새로운 공지가 없습니다"

클릭:
- 공지 화면으로 이동

---

④ 레벨 상태 카드
카드명:
"현재 레벨"

데이터:
summary.growth

표시 규칙:
- growth가 있으면:
  - current_level
  - achieved_date
  - prev_level이 있으면 함께 표시
  - note가 있으면 1줄 표시
- teacher_name이 있으면 작게 표시 가능

빈 상태:
- "아직 레벨 기록이 없습니다"

중요:
- 퍼센트, 점수, 임의 계산 성장률 절대 표시 금지
- 실제 데이터만 사용

클릭:
- 레벨 화면으로 이동

━━━━━━━━━━━━━━
4. 카드 공통 UX 규칙
━━━━━━━━━━━━━━

모든 카드 공통:
- 제목 명확히
- 핵심 텍스트 2줄 이내
- 데이터 없을 때도 카드 자체는 유지하고 내부에 빈 상태 문구 표시
- NEW / 중요 / 새 공지 등 배지 적극 활용
- 카드 간 간격/높이 시각적으로 통일

━━━━━━━━━━━━━━
5. 상태별 처리 규칙
━━━━━━━━━━━━━━

1. students.length === 0
- 기존 자녀 없음 화면 유지

2. selectedStudent.access_blocked === true
- 홈 주요 카드 대신 제한 안내 카드 표시
- 예: "현재 일부 정보 열람이 제한되어 있습니다"

3. class_group 없음
- 반 정보 없음 문구
- 일지/사진/오늘 수업 카드 빈 상태 처리

4. growth === null
- 레벨 카드 빈 상태 처리

━━━━━━━━━━━━━━
6. 데이터 우선순위 원칙
━━━━━━━━━━━━━━

홈에서 가장 먼저 보여야 하는 순서:
1. 오늘 수업 / 출석
2. 최근 수업일지
3. 최근 사진
4. 공지
5. 현재 레벨

불필요한 데이터 추가 금지:
- 전체 출결 이력
- 전체 알림 목록
- 성장 리포트
- 긴 공지 본문 전체

이건 상세 화면에서만 보여주세요.
