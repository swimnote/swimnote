[수영장 관리자 홈 퀵버튼 추가 지시서 - 선생님과 동일 위치/동일 스타일 + 즉시 등록 연결]

지금부터 수영장 관리자 홈화면에도
선생님 모드의 회원추가 퀵버튼과 같은 위치, 같은 스타일의 퀵버튼을 추가한다.

중요:
- 위치와 디자인은 선생님 모드 퀵버튼과 최대한 동일해야 한다
- 하지만 기능은 다르다
- 선생님은 “등록요청”
- 관리자는 “즉시 등록”
- 관리자 버튼을 누르면 회원 관리의 “어린이 직접 등록” 팝업이 바로 열려야 한다
- 새 페이지로 어색하게 이동하지 말고, 홈 위에서 자연스럽게 팝업이 뜨는 흐름으로 만들 것
- 실제 클릭 테스트까지 하고 보고할 것

==================================================
1. 버튼 추가 위치
==================================================

대상:
- 수영장 관리자 홈화면

위치:
- 선생님 모드 홈에서 만든 회원추가 퀵버튼과 같은 위치
- 월간 스케줄러/오늘 수업과 같은 핵심 홈 섹션 사이의 동일한 자리
- 관리자 홈 전체 레이아웃을 깨지 않게 자연스럽게 삽입

중요:
- 선생님 홈과 시각적으로 같은 패턴으로 보여야 함
- 홈 카드/운영 현황/관리 메뉴를 밀거나 겹치지 말 것

==================================================
2. 버튼 디자인
==================================================

버튼 이름:
- 회원추가

버튼 스타일:
- 선생님 모드 퀵버튼과 동일
- 직사각형 퀵액션 버튼
- + 아이콘이 잘 보이게
- 홈에서 바로 눌러야 하는 주요 액션처럼 보이게

중요:
- 관리자 버튼과 선생님 버튼은 기능만 다르고 UI는 통일감 있게 맞춘다

==================================================
3. 버튼 클릭 시 동작
==================================================

수영장 관리자 홈에서 회원추가 버튼 클릭 시:

→ 회원 관리 화면으로 굳이 먼저 이동하지 말고
→ 바로 “어린이 직접 등록” 팝업을 열어라

즉,
관리자 홈에서 빠른 직접 등록 액션이어야 한다.

팝업은 내가 보낸 관리자 모드의 기존 “어린이 직접 등록” 팝업을 재사용한다.

==================================================
4. 팝업 구조
==================================================

팝업 제목:
- 어린이 직접 등록

입력 항목:
- 학생 이름
- 출생년도
- 보호자 이름
- 보호자 전화번호
- 주 수업 횟수

하단 버튼:
- 등록하기

중요:
- 선생님 모드처럼 “등록요청”이 아님
- 관리자는 즉시 등록 권한이 있으므로
  기존 관리자 팝업의 “등록하기” 버튼 구조를 그대로 사용

==================================================
5. 저장 로직
==================================================

관리자 홈 퀵버튼으로 등록한 경우:

- 즉시 active 상태로 생성
- 정식 회원으로 반영
- 회원 수 카운트 포함
- 보호자 정보가 있으면 기존 직접 등록 흐름과 동일하게 처리
- 회원 관리 리스트에 바로 반영

중요:
- 선생님 pending 생성 로직과 혼동 금지
- 관리자 홈 퀵버튼은 기존 관리자 직접 등록과 완전히 같은 저장 로직을 사용해야 함

==================================================
6. 네비게이션/팝업 규칙
==================================================

버튼 클릭
→ 관리자 홈 위에 팝업 열림

닫기(X) 또는 뒤로가기
→ 팝업만 닫힘
→ 관리자 홈 그대로 유지

중요:
- 회원 관리 첫 화면으로 튀지 말 것
- 홈 화면 밖으로 나가지 말 것
- 팝업 닫기 후 직전 홈 상태 유지

==================================================
7. 기존 회원 관리와의 관계
==================================================

기존 회원 관리 화면의
“어린이 직접 등록” 버튼과 기능이 달라도 되는 게 아니라,
기존 그 팝업을 홈 퀵버튼에서 바로 호출하는 방식으로 연결하라.

즉:
- 회원 관리 안의 직접 등록 기능은 그대로 유지
- 홈 퀵버튼은 그 기능의 빠른 진입점 역할

==================================================
8. 관리자/선생님 차이 명확히 유지
==================================================

선생님 홈 퀵버튼:
- 회원추가
- 팝업 제목: 회원 등록 요청 또는 학생 등록 요청
- 하단 버튼: 등록요청
- 저장 상태: pending

관리자 홈 퀵버튼:
- 회원추가
- 팝업 제목: 어린이 직접 등록
- 하단 버튼: 등록하기
- 저장 상태: active

중요:
둘을 섞지 말고 명확히 분리하라.

==================================================
9. 테스트 필수
==================================================

반드시 실제 테스트:

1. 관리자 홈에 회원추가 버튼이 선생님과 같은 위치/비슷한 디자인으로 잘 보이는지
2. 버튼 클릭 시 바로 어린이 직접 등록 팝업이 뜨는지
3. 팝업 내용이 기존 관리자 직접 등록 화면과 동일한지
4. 등록하기 클릭 시 active 회원으로 즉시 생성되는지
5. 회원 수 카운트가 바로 증가하는지
6. 회원 관리 리스트에 즉시 반영되는지
7. 팝업 닫기 후 홈으로 튀지 않고 관리자 홈 유지되는지
8. 기존 회원 관리 직접 등록 기능이 안 깨졌는지
9. 선생님 모드 회원추가 퀵버튼과 충돌이 없는지
10. iPhone Expo Go 기준으로 UI가 안 깨지는지

==================================================
10. 디자인/배치 주의사항
==================================================

반드시 확인:
- 관리자 홈에서 버튼이 카드와 겹치지 않는지
- 버튼이 너무 튀거나 따로 놀지 않는지
- 선생님 모드와 통일감이 있는지
- 아이폰에서 눌리기 쉬운 크기인지
- safe area / 상단 헤더와 간섭 없는지

==================================================
11. 수정 금지
==================================================

절대 깨지면 안 되는 것:
- 기존 관리자 회원 관리 > 어린이 직접 등록 기능
- 선생님 pending 등록요청 기능
- 홈 네비게이션 규칙
- 뒤로가기/팝업 닫기 규칙
- 회원 수 집계 로직
- 승인 대기 학생 구조

이번 작업은
관리자 홈에 빠른 직접 등록 퀵버튼을 추가하는 작업이다.
