지금부터 선생님모드 핵심 기능 복구를 최우선으로 진행해라.
다른 기능 추가는 중단하고, 아래 3개 문제를 먼저 고쳐라.

[현재 핵심 문제]
1. 관리자가 학생을 등록해도 선생님모드 미배정 목록에 즉시 안 뜬다.
2. 반배정 팝업에서 검색 입력창이 키패드에 가려지고, 팝업 위치가 낮아서 입력/선택이 불편하거나 불가능하다.
3. 선생님모드 월간 스케줄에서 일부 버튼을 누르면 프리징된다.

[이번 작업 목표]
1. 관리자 학생등록 즉시, 해당 수영장의 모든 선생님 미배정 목록에 바로 반영
2. 반배정/학생검색 팝업을 키보드 간섭 없이 상단 높은 위치로 올리고, 입력창/결과 리스트가 항상 보이게 수정
3. 월간 스케줄 프리징 원인 추적 및 제거
4. 선생님모드 전체 점검 후 저장/조회/배정 흐름 재검증

━━━━━━━━━━━━━━━━━━
1. 관리자 등록 학생 → 선생님 미배정 즉시 반영
━━━━━━━━━━━━━━━━━━

정책:
- 관리자가 학생을 직접 등록하면 그 학생은 즉시 active 생성 가능
- 반 미배정 상태면 해당 수영장 소속 모든 선생님에게 “미배정 목록”에 보여야 함
- 학부모 연결 여부와 무관하게 미배정 목록에는 떠야 함
- 미배정과 학부모미연결은 절대 섞지 말 것

반드시 확인:
- 학생 등록 API 성공 후 어떤 테이블/컬럼이 저장되는지
- 선생님모드 미배정 목록 조회 API가 어떤 조건으로 학생을 불러오는지
- pool_id 기준으로 해당 수영장 학생을 읽는지
- assigned_class_ids / class_group_id / weekly_count 기준 미배정 판단이 맞는지
- 캐시나 stale query 때문에 즉시 반영이 안 되는지
- 등록 후 목록 invalidate/refetch가 되는지

수정 목표:
- 관리자 학생등록 후 선생님모드 미배정 목록에 즉시 반영
- 앱 재실행 없이도 보이게
- 같은 수영장 선생님 전원에게 동일하게 보이게

검증:
- 관리자 학생등록
- 선생님모드 미배정 목록 진입
- 즉시 노출
- 반배정 후 목록에서 제거
- 학부모미연결 배지는 별도 유지

━━━━━━━━━━━━━━━━━━
2. 반배정 팝업 UI/키보드 간섭 수정
━━━━━━━━━━━━━━━━━━

현재 문제:
- 검색창이 키패드에 가려짐
- 팝업 위치가 낮음
- 입력 중 결과가 안 보이거나 선택이 어려움
- 채팅창/키보드 간섭 있음

수정 기준:
- 반배정 팝업은 화면 상단 쪽 높은 위치에 띄워라
- 검색 입력창은 항상 키보드 위에 보이게
- KeyboardAvoidingView 또는 동등한 방식 적용
- 팝업 내부 스크롤 가능
- 검색 결과 리스트는 입력창 바로 아래에 유지
- 키보드가 떠도 검색창/결과/선택 버튼이 가려지지 않게
- 팝업 높이는 고정 하단시트가 아니라 상단 중심 모달 또는 충분히 위로 올라온 바텀시트로 조정
- 입력 포커스 시 자동 스크롤
- 결과 선택 후 키보드 닫힘/유지 여부도 자연스럽게 처리
- 채팅창 간섭이나 하단 입력창 간섭 없게 할 것

반드시 확인:
- iPhone 화면 높이 기준
- 작은 화면에서도 검색창이 안 가려지는지
- 학생 10명 이상일 때 스크롤 되는지
- 검색어 입력 중 리스트가 안 사라지는지
- 선택 후 저장 버튼이 보이는지

팝업 문구/구성:
- 제목: 학생 반배정
- 상단에 검색창 고정
- 중간에 검색 결과 리스트
- 하단에 선택 완료/닫기 버튼

검증:
- 학생 이름 검색
- 키보드 떠 있는 상태에서 결과 확인
- 학생 선택
- 저장
- 저장 후 목록 반영

━━━━━━━━━━━━━━━━━━
3. 선생님모드 월간 스케줄 프리징 수정
━━━━━━━━━━━━━━━━━━

현재 문제:
- 월간 스케줄에서 안 눌리는 버튼 누르면 프리징
- 스케줄이 핵심인데 제기능 못함

반드시 추적:
- 어떤 버튼에서 프리징되는지 로그 추가
- onPress가 실제 타는지
- modal 중첩 여부
- useFocusEffect/useEffect 루프 여부
- selectedDate 상태 변경 루프 여부
- MonthlyCalendar 데이터 매핑 계산 비용 과다 여부
- overlay/pointerEvents 충돌 여부
- pressable이 막혀 있는지
- 잘못된 navigation 또는 setState loop가 있는지

우선 의심:
- 월간 셀 클릭 시 하단 액션 팝업/모달 중첩
- disabled 아닌데 눌렀을 때 silent return
- 날짜 map 생성이 매 렌더마다 비효율적으로 다시 계산됨
- 프리징 시 loading state가 풀리지 않음

수정 기준:
- dateClassMap/useMemo 적용
- 셀 클릭 시 최소 동작만 수행
- 중첩 modal 제거
- confirm/modal은 상위로 이동
- onPress 로그 추가 후 원인 제거
- freeze 원인 버튼은 임시 비활성화가 아니라 근본 수정
- 월간/주간/일간 전환 시 상태 초기화 충돌 제거

검증:
- 월간 진입
- 날짜 셀 클릭
- 반 있는 날/없는 날 각각 클릭
- 액션 진입
- 되돌아오기
- 5회 이상 반복해도 프리징 없어야 함

━━━━━━━━━━━━━━━━━━
4. 선생님모드 전체 점검 범위

- 프리징/무반응 여부
를 적어라

5. 제출 형식
━━━━━━━━━━━━━━━━━━

답변 마지막에는 반드시 아래 형식으로 제출해라.

1. 실제 원인
- 미배정 즉시 반영 안 된 원인
- 반배정 팝업 입력창 간섭 원인
- 월간 스케줄 프리징 원인

2. 수정한 파일
- 파일 경로
- 수정한 컴포넌트/함수명
- 무엇을 어떻게 바꿨는지

3. 실제 검증 결과
- 관리자 등록 → 선생님 미배정 반영 PASS/FAIL
- 반배정 팝업 검색/선택 PASS/FAIL
- 월간 스케줄 클릭 프리징 PASS/FAIL
- 주간/일간/보강/일지 진입 PASS/FAIL

4. 내가 직접 확인할 위치
- 선생님모드 어느 화면에서 확인하는지
- 어떤 학생명으로 테스트했는지
- 어떤 반으로 배정 테스트했는지

중요:
- “수정 완료” 한 줄 금지
- 반드시 실제 반영/실제 선택/실제 저장/실제 프리징 제거 결과를 제출해라
