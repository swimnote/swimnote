이번 작업은 비용 절감을 위해 한 번에 정리해서 진행해줘.
중복 수정 없이 전체 출결관리 구조를 먼저 분석한 뒤 필요한 변경만 한 번에 반영해줘.

이번 작업 범위:
"출결관리 화면에 일자별 / 주간 / 월간 보기 추가 + 날짜 탐색 인덱스 + 이름 검색 기반 출결 이력 조회"까지만 진행해줘.
그 외 UI 전체 리디자인, 불필요한 리팩토링, 성능 최적화, 다른 메뉴 수정은 하지 마.

목표:
현재 출결관리는 일자별 보기만 있는데,
관리자 화면에서 출결을
- 일자별
- 주간별
- 월간별
로 나눠서 볼 수 있게 하고,
특정 날짜/요일을 빠르게 찾아갈 수 있는 기능과
특정 이름 검색 시 해당 회원의 출석/결석 이력을 목록으로 볼 수 있게 해줘.

필수 기능:

1. 출결 보기 모드 추가
출결관리 화면 상단에 보기 모드를 추가해줘:
- 일자별
- 주간
- 월간

기본값은 현재처럼 일자별로 두고,
탭 또는 segmented control 방식으로 전환 가능하게 해줘.

2. 일자별 보기 유지
- 기존 일자별 출결 UI는 유지
- 현재 선택된 날짜 기준으로 반별/회원별 출결 체크 및 조회 가능하게 유지
- 기존 저장 구조와 충돌 없게 해줘

3. 주간 보기 추가
- 선택한 날짜가 포함된 주간 단위로 출결 현황을 볼 수 있게 해줘
- 예: 월~일 또는 운영 기준 주간
- 각 날짜별 출석/결석/지각 상태를 한눈에 볼 수 있게 해줘
- 회원별 주간 출결 현황을 표 또는 리스트로 표시
- 주간 이동 버튼:
  - 이전 주
  - 다음 주
- 주간 범위 표시 예:
  2026-03-09 ~ 2026-03-15

4. 월간 보기 추가
- 선택한 월 기준으로 월간 출결 현황을 볼 수 있게 해줘
- 회원별 월 출결 요약이 보이게 해줘
- 예:
  출석 n회 / 결석 n회 / 지각 n회
- 월 이동 버튼:
  - 이전 달
  - 다음 달
- 월 표시 예:
  2026년 3월

5. 날짜 탐색 기능
- 특정 날짜로 바로 이동할 수 있는 날짜 선택 기능 추가
- 일자별 / 주간 / 월간 보기 모두 현재 기준 날짜를 중심으로 동작하게 해줘
- 날짜를 바꾸면
  - 일자별: 해당 날짜
  - 주간: 해당 날짜가 포함된 주
  - 월간: 해당 날짜가 포함된 달
  로 자동 반영되게 해줘

6. 특정 요일 인덱스 기능
- 주간/월간 보기에서 특정 요일을 빠르게 찾을 수 있는 기능 추가
- 예:
  월 / 화 / 수 / 목 / 금 / 토 / 일
- 특정 요일 버튼을 누르면 해당 주 또는 월에서 그 요일 날짜들 중심으로 이동/하이라이트되게 해줘
- 너무 복잡한 캘린더보다는 단순하고 빠르게 찾을 수 있는 구조로 해줘

7. 이름 검색 기능
- 출결관리 화면 상단 또는 별도 영역에 이름 검색창 추가
- 특정 회원 이름 검색 시,
  해당 회원의 출결 이력을 목록 형태로 볼 수 있게 해줘
- 목록에는 최소 아래 정보 표시:
  - 날짜
  - 반 이름
  - 출결 상태 (출석/결석/지각)
- 검색 결과는 현재 로그인한 관리자 poolId 데이터에서만 조회되게 해줘
- 다른 수영장 데이터는 절대 섞이면 안 됨

8. 출결 이력 목록 기능
- 검색 결과는 최신순 정렬
- 필요하면 날짜 범위 필터도 간단하게 추가 가능
- 예:
  최근 7일
  최근 30일
  전체
- 기본은 최근 30일 또는 전체 중 현재 구조에 맞는 간단한 방식으로 해줘

9. 데이터 구조 유지
- 기존 출결 저장 구조를 최대한 유지하면서 조회 로직만 확장
- 현재 poolId 멀티테넌트 구조와 충돌 없게 해줘
- 관리자 계정은 자신의 poolId 출결 데이터만 조회 가능하게 유지

10. 읽기/수정 구분 유지
- 읽기 전용 관리자면 출결 현황 조회와 검색만 가능
- 수정 권한 있는 관리자만 일자별 출결 체크/저장 가능
- 주간/월간/검색 기능은 조회 중심으로 동작하게 해줘

11. JSON 응답 유지
- 출결 조회/검색 관련 API 응답은 모두 JSON 형식 유지
- 에러 발생 시에도 JSON 반환
- 예:
  { "success": true, "data": ... }
  { "success": false, "message": "error" }

12. UI 방향
- 현재 출결관리 화면 스타일을 크게 해치지 않는 선에서 확장
- 새 기능 때문에 화면이 너무 복잡해지지 않게
- 상단에
  보기모드 / 날짜이동 / 검색
  구조로 깔끔하게 정리해줘

13. 테스트 데이터
- 현재 테스트 회원 기준으로
  여러 날짜에 출석/결석/지각 데이터가 보이도록 테스트 가능하게 해줘
- 주간 보기, 월간 보기, 이름 검색이 실제로 결과를 보여줄 수 있게 최소 샘플 데이터 확인 가능하게 해줘

작업 방식:
1. 먼저 현재 출결 관련 화면, 컴포넌트, API, 데이터 모델을 분석
2. 같은 파일은 가능하면 한 번만 수정
3. UI 전체 변경, 스타일 정리, 성능 최적화는 하지 마
4. 이번 턴에서는 오직 출결관리 조회 확장 기능만 진행

마지막 결과 보고 형식:
1. 수정한 파일 목록
2. 추가된 보기 모드 목록
3. 추가된 날짜 탐색 기능 설명
4. 추가된 이름 검색 기능 설명
5. 읽기 전용 / 수정 가능 권한 분기 설명
6. 테스트 방법
7. 남은 후속 작업 목록
