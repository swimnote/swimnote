[스윔노트 — 스케줄 보드 파스텔 색상 선택 기능 구현 지시문]

이번 작업은 스케줄 보드에서 반/수업을 색상으로 쉽게 구분할 수 있게 만드는 기능 구현이다.  
핵심은 “예쁘게”가 아니라 “익숙하고 직관적으로” 만드는 것이다.

기능 목표:
- 기본 색상은 흰색
- 필요할 때만 색상 선택
- 검정 글씨가 잘 보이는 파스텔 색상만 제공
- 사용자가 한눈에 “색 고르는 버튼”임을 알아볼 수 있는 익숙한 UI 사용

중요:
- 자유 색상 입력 금지
- 진한 색 금지
- 텍스트 색상 변경 금지
- 색상은 반 기준으로 저장하고, 수업 카드에 반영

━━━━━━━━━━━━━━━━━━
1. 저장 기준
━━━━━━━━━━━━━━━━━━

색상은 “개별 수업”이 아니라 “반(class_group)” 기준으로 저장한다.

이유:
- 같은 반은 항상 같은 색으로 보여야 사용자가 빠르게 인식 가능
- 매번 수업마다 색이 바뀌면 실사용성이 떨어짐

DB:
- class_groups 테이블에 color 컬럼 추가 (TEXT)
- 기본값: "#FFFFFF"

예:
- color = "#DCFCE7"

━━━━━━━━━━━━━━━━━━
2. 기본 동작
━━━━━━━━━━━━━━━━━━

- 기본 색상은 흰색 (#FFFFFF)
- 사용자가 색상을 따로 선택하지 않으면 흰색 유지
- 반 상세 / 반 수정 / 스케줄 하단 시트에서 색상 변경 가능
- 저장 즉시 해당 반의 보드 카드 배경색에 반영

적용 위치:
- 주간 스케줄 보드
- 일간 스케줄 카드
- 월간에서 선택 후 뜨는 하단 시트의 반 정보 표시 영역
- 반 상세/반 수정 화면

텍스트:
- 무조건 #111827 고정

━━━━━━━━━━━━━━━━━━
3. 색상 선택 UI 방식
━━━━━━━━━━━━━━━━━━

“익숙한 디자인”으로 만들어야 하므로  
색상 선택은 아래 방식으로 구현한다.

기본 UI:
- “색상” 또는 작은 팔레트 아이콘 버튼 1개
- 현재 선택 색상이 작은 원형 프리뷰로 같이 보이게 한다
- 기본 흰색이면 흰색 원 + 회색 테두리로 표시

예:
[🎨 색상]   ○
또는
[색상]   (현재 색상 원형 표시)

동작:
- 색상 버튼 누르면 오른쪽에서 세로 색상 패널이 슬라이드 인
- 또는 하단 시트 내부 오른쪽에 세로 컬러 리스트가 나타남

중요:
- 사용자가 “아 여길 누르면 색상 고르는구나” 바로 이해할 수 있어야 함
- 숨겨진 제스처 금지
- 복잡한 팝업 금지

추천 구현:
- 하단 시트 안에 “색상” 행 추가
- 행 오른쪽에 현재 색상 원형 표시 + 화살표
- 누르면 우측 슬라이드 컬러 패널 오픈

━━━━━━━━━━━━━━━━━━
4. 색상 패널 구조
━━━━━━━━━━━━━━━━━━

패널 형태:
- 세로 리스트
- 스크롤 가능
- 원형 컬러칩 20개 내외
- 각 칩 크기: 3236px
- 칩 간 간격: 1012px
- 현재 선택된 색상은 굵은 테두리 + 체크 표시

위치:
- 오른쪽에서 슬라이드 인
- 폭은 너무 넓지 않게 (약 72~96px 수준)
- 기존 내용 가리지 않게 자연스럽게 겹치거나 시트 안에 붙이기

사용성:
- 색상칩만 봐도 익숙한 “컬러 피커”처럼 느껴져야 함
- 현재 선택 상태가 분명히 보여야 함

━━━━━━━━━━━━━━━━━━

- 형광색 금지

━━━━━━━━━━━━━━━━━━
6. 표시 방식
━━━━━━━━━━━━━━━━━━

반 카드/수업 카드:
- backgroundColor = class_group.color || "#FFFFFF"
- borderColor는 기존 스타일 유지 가능하나 너무 진하지 않게
- 텍스트는 검정 고정

기본 흰색인 경우:
- 완전 흰 배경이라 카드 경계가 안 보일 수 있으므로
- 얇은 회색 보더 추가
예:
- borderColor: #E5E7EB

━━━━━━━━━━━━━━━━━━
7. 사용자 흐름
━━━━━━━━━━━━━━━━━━

반 등록/수정 시:
1. 반 정보 입력
2. 색상 버튼 누름
3. 세로 색상표 열림
4. 색 선택
5. 선택 상태 즉시 반영
6. 저장
7. 스케줄 보드에서 해당 반 색 반영 확인

기존 스케줄 보드 하단 시트에서:
1. 날짜/반 선택
2. 하단 시트 열림
3. 색상 버튼 또는 색상 행 선택
4. 색상표에서 변경
5. 저장 시 반 전체 색상 변경

━━━━━━━━━━━━━━━━━━
8. 익숙한 디자인 규칙
━━━━━━━━━━━━━━━━━━

반드시 지킬 것:

- 색상칩은 원형
- 선택된 색상은 체크 표시 또는 진한 테두리
- 기본 흰색도 칩으로 포함
- 흰색 칩은 회색 테두리로 눈에 보이게
- 색상 버튼은 팔레트 아이콘 또는 원형 색상 프리뷰와 함께 표시
- 사용자가 버튼을 누르기 전에도 “이건 색상 설정이다” 알아볼 수 있어야 함

금지:
- 색상 선택을 텍스트 링크처럼 만들기
- 숨겨진 메뉴 안에 넣기
- 너무 작은 색상칩
- 색상표가 한 번에 안 보여서 의미를 못 알아보는 구조

━━━━━━━━━━━━━━━━━━
9. 기술 구현 권장
━━━━━━━━━━━━━━━━━━

권장 상태값:
- selectedColor
- isColorPickerOpen

권장 컴포넌트 분리:
- ColorPreviewButton
- PastelColorPickerPanel
- ScheduleColorChip

권장 props:
- value
- onChange
- colors
- defaultColor

권장 애니메이션:
- 오른쪽 슬라이드 인/아웃
- 너무 과한 애니메이션 금지

━━━━━━━━━━━━━━━━━━
10. 테스트
━━━━━━━━━━━━━━━━━━

반드시 실제로 확인:

1. 기본 상태에서 흰색으로 보이는지
2. 흰색일 때 카드 경계가 보이는지
3. 색상 버튼이 직관적으로 보이는지
4. 색상표 패널이 정상적으로 열리는지
5. 스크롤로 전체 색상 선택 가능한지
6. 색 선택 후 즉시 프리뷰 반영되는지
7. 저장 후 재진입해도 색 유지되는지
8. 주간/일간 보드에 동일 색 반영되는지
9. 검정 텍스트 가독성 문제 없는지
10. 여러 반을 서로 다른 색으로 구분 가능해지는지

━━━━━━━━━━━━━━━━━━
