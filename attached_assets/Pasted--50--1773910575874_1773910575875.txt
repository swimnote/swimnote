지금부터 토이키즈 관리자 계정을 “테스트용 초기 상태”로 재구성하라.

이번 작업 목표는
토이키즈 계정 안에 섞여 있는 기존 테스트 데이터와 반 구성을 모두 비우고,
회원 50명만 있는 깨끗한 상태를 만들어
관리자가 직접 반 개설, 반배정, 보강, 전반, 반이동을 테스트할 수 있게 만드는 것이다.

가장 중요:
- 토이키즈 관리자 계정 기준으로 작업
- 기존 반/배정/보강/출결/일지 관련 테스트 데이터는 모두 제거
- 반은 하나도 없는 상태로 만들 것
- 대신 테스트용 회원 50명만 생성
- 학부모 전화번호와 임시 학부모 정보도 같이 생성
- 이 작업은 테스트를 위한 데이터 재구성이지 기능 개편이 아니다
- 작업 전 반드시 체크포인트 저장할 것

==================================================
[1. 작업 전 안전조치]

반드시 먼저 할 것:
1. 현재 상태 체크포인트 저장
2. 토이키즈 관리자 계정에 연결된 데이터 범위 확인
3. 삭제 대상과 유지 대상을 구분

이번 작업은 토이키즈 계정 데이터만 대상으로 한다.
다른 계정/다른 수영장 데이터는 건드리지 말 것.

==================================================
[2. 최종 목표 상태]

토이키즈 관리자 계정은 아래 상태가 되어야 한다.

최종 상태:
- 반 개설 데이터 없음
- 현재 배정된 반 없음
- 보강 데이터 없음
- 임시 이동/전반 테스트 데이터 없음
- 출결 테스트 데이터 없음
- 일지 테스트 데이터 없음
- 회원 50명만 존재
- 각 회원은 아직 반배정 안 된 상태
- 학부모 임시 정보 존재
- 전화번호 임시값 존재

즉:
“토이키즈 계정 안은 깨끗한 상태 + 회원 50명만 준비된 상태”

==================================================
[3. 삭제/초기화 대상]

토이키즈 계정에서 아래 데이터는 모두 제거 또는 초기화하라.

1. 개설된 반
2. 반별 배정 데이터
3. 반 스케줄 연결 데이터
4. 보강 명단/보강 배정 데이터
5. 전반/이동 관련 테스트 데이터
6. 출결 기록
7. 수업 일지 기록
8. 반과 연결된 회원 배정 상태
9. 반 관련 임시 테스트 로그

중요:
회원 자체는 최종적으로 새 테스트 회원 50명만 남기면 된다.
기존 테스트 회원은 삭제하거나 전부 교체해도 됨.

==================================================
[4. 유지/생성 대상]

최종적으로 남길 것은 아래뿐이다.

1. 토이키즈 관리자 계정
2. 테스트 가능한 기본 선생님 계정들(기존 로그인 필요하면 유지)
3. 테스트용 회원 50명
4. 각 회원에 연결된 임시 학부모 이름/전화번호

반은 만들지 말 것.
반배정도 하지 말 것.
모든 회원은 “미배정/미등록 테스트 가능 상태”로 둘 것.

==================================================
[5. 테스트용 회원 50명 생성 규칙]

토이키즈 계정 안에 테스트용 회원 50명을 생성하라.

회원 데이터 예시 구조:
- 학생 이름
- 학부모 이름
- 학부모 전화번호
- 상태: 미배정 또는 미등록 테스트 가능 상태
- 반배정 없음

중요:
- 이름은 서로 겹치지 않게
- 전화번호도 서로 겹치지 않게
- 임시 더미 데이터라도 실제 테스트에 불편하지 않게 만들 것

예시:
학생 001 ~ 학생 050
또는 자연스러운 한국식 이름 조합 50개

추천:
- 이름은 자연스러운 한글 이름으로
- 전화번호는 010-1000-0001 ~ 010-1000-0050 같은 규칙형으로 생성
- 학부모 이름도 임시 한글 이름 생성

==================================================
[6. 회원 상태 규칙]

생성된 50명 회원은 아래 상태여야 한다.

- 아직 반 없음
- 아직 출결 없음
- 아직 일지 없음
- 아직 보강 없음
- 테스트를 위해 선생님이 직접 반배정 가능
- 전반/이동 테스트 가능
- 미배정 목록에서 바로 보이게 할 것

중요:
이미 어떤 반에 들어간 상태로 만들지 말 것

==================================================
[7. 선생님 테스트 가능 조건]

테스트용 회원 50명은
선생님이 아래를 직접 해볼 수 있는 상태여야 한다.

1. 반 개설
2. 학생 반배정
3. 미배정 학생 데려오기
4. 보강 배정
5. 전반/반이동 테스트
6. 출결 처리
7. 일지 작성

즉:
회원만 준비하고
수업 관련 데이터는 전부 비워둔 상태

==================================================
[8. 관리자 화면 확인 조건]

토이키즈 관리자 계정으로 로그인했을 때
아래처럼 보여야 한다.

- 사람 탭에 회원 50명 확인 가능
- 미배정/미등록 목록에서 테스트 가능
- 반 목록은 비어 있음
- 수업 관련 반 데이터는 없음
- 정산/보강/일지 기록 없음

==================================================
[9. 선생님 화면 확인 조건]

선생님 계정으로 로그인했을 때
아래처럼 보여야 한다.

- 내 반 없음
- 개설된 반 없음
- 테스트용 학생 50명은 미배정/미등록 쪽에서 가져올 수 있는 상태
- 기존 꼬인 반배정 데이터 없음

==================================================
[10. 더미 데이터 생성 방식]

회원 50명 생성 시 아래를 같이 준비하라.

필수 필드:
- 학생 이름
- 학부모 이름
- 학부모 전화번호
- 학생 식별 ID
- 기본 상태값

선택 가능:
- 출생연도 더미값
- 성별 더미값

중요:
추후 테스트에 방해되지 않도록 최소 구조만 넣고 복잡한 추가 데이터는 넣지 말 것

==================================================
[11. 작업 순서]

반드시 아래 순서로 진행하라.

1. 체크포인트 저장
2. 토이키즈 계정 데이터 분석
3. 기존 반/배정/보강/출결/일지 데이터 제거
4. 토이키즈 계정 반/수업 관련 상태 초기화
5. 테스트용 회원 50명 생성
6. 학부모 임시 정보 생성
7. 미배정/미등록 상태 확인
8. 관리자/선생님에서 실제 보이는지 확인

==================================================
[12. 테스트 - 반드시 실제 수행]

테스트 1.
토이키즈 관리자 로그인 후 반이 하나도 없는지 확인

테스트 2.
기존 배정/보강/출결/일지 데이터가 남아 있지 않은지 확인

테스트 3.
사람 탭에서 회원 50명이 보이는지 확인

테스트 4.
회원 50명이 모두 미배정/미등록 테스트 가능 상태인지 확인

테스트 5.
전화번호와 학부모 이름이 각 회원마다 존재하는지 확인

테스트 6.
선생님 로그인 후 기존 내 반 데이터가 비어 있는지 확인

테스트 7.
선생님이 테스트용 회원을 가져와 반배정할 수 있는 상태인지 확인

==================================================
[13. 작업 후 보고 형식]

작업 후 아래 순서로 보고하라.

1. 삭제/초기화한 데이터 목록
2. 생성한 테스트 회원 수
3. 생성한 학부모 더미 정보 규칙
4. 관리자 화면 확인 결과
5. 선생님 화면 확인 결과
6. 테스트 1~7 결과
7. 아직 남은 꼬인 데이터가 있으면 명시

==================================================
[14. 절대 금지]

- 토이키즈 외 다른 계정 데이터 삭제 금지
- 테스트 회원 50명 외에 기존 꼬인 반 데이터 남기기 금지
- 반이 남아 있는 상태로 완료 보고 금지
- 회원 수가 50명보다 적거나 많은 상태로 완료 보고 금지
- 더미 전화번호 중복 금지
- 테스트 가능 상태가 아닌 회원 생성 금지

이번 작업은
토이키즈 관리자 계정을 테스트용 초기 상태로 리셋하고,
반/보강/출결/일지 관련 기존 데이터를 전부 비운 뒤,
반배정과 전반 테스트를 할 수 있도록 회원 50명만 남긴 깨끗한 샌드박스 상태를 만드는 작업이다.
