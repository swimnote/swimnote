[스윔노트 — 전화 UX 통합 + 학생 프로필 학부모 정보 + 간결 UI 적용 지시문]

이번 작업은 전화 기능을 “즉시 실행 UX”로 통합하고,
학생 프로필에서 학부모 연락처를 기본 정보로 노출하며,
설명 없이도 이해되는 버튼 구조로 UI를 정리하는 작업이다.

━━━━━━━━━━━━━━━━━━

1. 핵심 원칙
   ━━━━━━━━━━━━━━━━━━

* 전화번호가 보이면 즉시 전화 가능해야 한다
* 버튼은 설명이 아니라 “행동”이어야 한다
* 기능 진입 경로는 하나로 통일한다

핵심 기준:
👉 전화번호 = 버튼
👉 버튼 = 설명 없이 이해 가능

━━━━━━━━━━━━━━━━━━
2. 학부모 계정 구조 확정
━━━━━━━━━━━━━━━━━━

* 학부모 계정은 수영장별 1명 고정
* 부모 여러 명 연결 구조 제거 (복잡도 증가 방지)

[DB 구조]

parent_accounts

* id
* pool_id
* phone_number
* name
* status

parent_students

* student_id
* parent_id
* pool_id

[조회]

student_id → parent_students → parent_accounts

━━━━━━━━━━━━━━━━━━
3. 학생 프로필 화면 수정 (중요)
━━━━━━━━━━━━━━━━━━

[추가 정보]

학생 프로필에 반드시 표시:

* 학부모 이름
* 학부모 전화번호
* 전화 아이콘 버튼

[UI 예시]

학부모
김철수
📞 010-1234-5678

━━━━━━━━━━━━━━━━━━
4. 전화 버튼 UX (핵심 기능)
━━━━━━━━━━━━━━━━━━

[적용 위치]

* 학생 프로필
* 회원관리 목록 (전화번호 표시 시)
* 학부모 관련 모든 화면

[UI 규칙]

* 전화 아이콘은 항상 전화번호 왼쪽에 고정
* 아이콘만 사용 (텍스트 없음)
* 전화번호도 클릭 가능

[동작]

* 아이콘 클릭 → 전화 실행
* 전화번호 클릭 → 전화 실행

동일 동작:
→ tel: 링크 호출
→ 기기 전화 앱 실행
→ 번호 자동 입력

[중요]

* 중간 확인창 없음
* 복사 단계 없음
* 즉시 실행

━━━━━━━━━━━━━━━━━━
5. 전화 버튼 색상 규칙
━━━━━━━━━━━━━━━━━━

[기본 배경]

* 카드 색상: 초록

[전화 버튼 색상]

* HEX: #F97316 (오렌지)

[이유]

* 초록과 대비 명확
* 행동 버튼으로 인식됨
* 위험/삭제 느낌 없음

[금지]

* 초록 계열 사용 금지
* 회색/검정 사용 금지
* 텍스트 버튼 금지

━━━━━━━━━━━━━━━━━━
6. 버튼/텍스트 간결화 규칙
━━━━━━━━━━━━━━━━━━

[원칙]

* 버튼 텍스트는 1~2단어
* 가능한 경우 아이콘 사용
* 설명 문장 제거

[예]

* "출석 처리하기" → "출석"
* "결석으로 변경" → "결석"
* "전화하기" → 아이콘만 사용

[금지]

* 긴 문장 버튼
* 설명 텍스트 추가

7. 기능 경로 단일화
━━━━━━━━━━━━━━━━━━

[원칙]

* 기능은 한 곳에서만 실행
* 다른 화면에서는 조회만

[예]

출결:

* 실행: 출결 화면
* 다른 곳: 조회만

보강:

* 실행: 보강/출결 흐름
* 다른 곳: 링크만

회원관리:

* 실행: 회원관리 탭
* 다른 곳: 상세 이동
