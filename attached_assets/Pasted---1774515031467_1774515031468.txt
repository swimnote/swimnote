[스윔노트 — 학부모 회원가입 수영장 검색/연결/승인 실시간 연동 전수 점검 지시문]

현재 심각한 문제:
수영장 관리자로 "토이키즈스윔클럽" 가입을 완료했는데,
학부모 회원가입 화면의 수영장 선택에서 해당 수영장이 목록에 나오지 않고 검색도 되지 않는다.

이 문제는 단순 검색 UI 문제가 아니라,
학부모 가입 → 수영장 선택 → 승인 요청 → 관리자 승인 → 학부모 연결까지의 핵심 흐름이 끊긴 상태다.

이번 작업은 학부모 가입 흐름과 서버 연결을 처음부터 끝까지 전수 점검하고,
수영장 검색 및 승인 연동이 실시간으로 정상 동작하게 만드는 작업이다.

━━━━━━━━━━━━━━━━━━
1. 핵심 목표
━━━━━━━━━━━━━━━━━━

반드시 아래 흐름이 실제로 연결되어야 한다.

1. 수영장 관리자 가입
2. 수영장 생성 및 approved 상태 저장
3. 학부모 회원가입 화면에서 수영장 목록 조회
4. 검색어 입력 시 등록 수영장 검색 가능
5. 학부모가 수영장 선택 후 가입/요청
6. 관리자 승인 목록에 즉시 반영
7. 승인 시 학부모-학생-수영장 연결 반영
8. 앱 화면에 실시간 또는 즉시 재조회로 반영

핵심:
👉 수영장이 검색되지 않으면 이후 흐름은 전부 실패로 본다

━━━━━━━━━━━━━━━━━━
2. 우선 의심 원인 전수 점검
━━━━━━━━━━━━━━━━━━

다음 원인들을 전부 점검:

A. 수영장 검색 API 문제
- 학부모 가입 화면이 어떤 API를 호출하는지 확인
- approved 수영장만 조회하는지 확인
- hidden/deleted/status 조건이 잘못 걸려 있는지 확인
- pool_name vs name vs display_name 컬럼 매칭 오류 확인

B. DB 저장 문제
- 관리자 가입 시 swimming_pools에 실제 저장됐는지 확인
- approval_status = approved 로 저장됐는지 확인
- search 대상 컬럼에 "토이키즈스윔클럽" 값이 들어갔는지 확인

C. 검색 쿼리 문제
- 부분검색이 정확히 동작하는지 확인
- LIKE / ILIKE / contains 조건 확인
- 공백 제거/정규화 문제 확인
- "토이키즈"로 검색해도 안 뜨면 쿼리 문제로 본다

D. 클라이언트 상태 문제
- 검색 API 응답은 오는데 화면 렌더링이 안 되는지 확인
- 빈 결과 처리 로직이 잘못된지 확인
- debounce/search state가 꼬였는지 확인

E. 승인/연결 후속 문제
- 학부모 가입 요청이 관리자 승인 목록에 올라오는지
- 승인 후 연결 데이터가 즉시 반영되는지
- 재조회/캐시 무효화 안 돼서 안 보이는지 확인

━━━━━━━━━━━━━━━━━━
3. 학부모 수영장 검색 정상화
━━━━━━━━━━━━━━━━━━

학부모 회원가입의 수영장 선택 화면은 아래 규칙으로 고정:

- 기본 진입 시 approved 수영장 목록 노출
- 검색어 입력 시 부분검색 가능
- 예: "토이", "키즈", "스윔", "클럽" 모두 검색 가능
- 결과가 없을 때만 빈 상태 노출
- 등록된 수영장이 있는데 빈 결과 뜨면 실패

검색 대상:
- superAdminDb의 swimming_pools
- status/approval_status 기준 유효 수영장만

필수 확인:
- "토이키즈스윔클럽"이 DB에 저장된 정확한 값
- 검색 UI가 그 컬럼을 제대로 조회하는지

━━━━━━━━━━━━━━━━━━
4. 관리자 가입 시 수영장 상태 검증
━━━━━━━━━━━━━━━━━━

이미 구현된 관리자 즉시가입 구조를 다시 검증:

관리자 가입 직후 수영장은 반드시 아래여야 한다.

- swimming_pools 레코드 생성 완료
- approval_status = approved
- subscription_status = trial 또는 활성 상태
- hidden/deleted 아님
- 학부모 검색 대상에 포함되는 상태

즉,
관리자 입장에서 가입 완료됐는데
학부모 검색에 안 나오면 저장 상태 또는 조회 조건 문제다

━━━━━━━━━━━━━━━━━━
5. 학부모 가입 요청 → 관리자 승인 연결
━━━━━━━━━━━━━━━━━━

수영장 검색이 고쳐진 후 아래 흐름도 같이 점검:

1. 학부모가 수영장 선택
2. 가입 요청 저장
3. 관리자 승인 관리 목록에 즉시 표시
4. 승인 시 연결 반영
5. 재조회 없이도 보이거나, 최소한 즉시 재요청 시 반영

필수:
- 학부모 요청은 실제 DB 저장
- 관리자 승인 목록은 mock 금지
- 승인/거절 후 상태 즉시 반영

━━━━━━━━━━━━━━━━━━
6. 실시간 반영 또는 즉시 반영
━━━━━━━━━━━━━━━━━━

“실시간”은 꼭 웹소켓이 아니어도 된다.
하지만 사용자는 즉시 반영되는 것처럼 느껴야 한다.

허용 방식:
- mutation 후 query invalidate
- 승인 직후 목록 재조회
- 화면 재진입 시 최신값 보장

금지:
- 승인했는데 새로고침 전까지 안 보이는 상태 방치
- 캐시 때문에 예전 값이 계속 보이는 상태
- 요청 저장은 됐는데 승인 목록에 안 뜨는 상태

━━━━━━━━━━━━━━━━━━
7. 점검 대상 파일 전수 확인
━━━━━━━━━━━━━━━━━━

반드시 확인:

- 학부모 회원가입 화면
- 수영장 검색 컴포넌트
- 수영장 검색 API
- auth/register 관련 학부모 분기
- swimming_pools 조회 로직
- 승인 관리 화면
- 승인 요청 저장 로직
- 승인 처리 API
- 관련 store / query cache / invalidate 로직

중요:
- UI만 보지 말고 API 응답, DB 저장값, 화면 반영까지 같이 확인

━━━━━━━━━━━━━━━━━━
8. 실제 테스트 시나리오
━━━━━━━━━━━━━━━━━━

반드시 아래 순서로 실제 검증:

1. 수영장 관리자 가입
- 수영장명: 토이키즈스윔클럽
- 가입 완료 확인

2. DB 확인
- swimming_pools에 토이키즈스윔클럽 저장 확인
- approval_status = approved 확인

3. 학부모 회원가입 진입
- 수영장 목록 기본 노출 확인
- "토이키즈" 검색 시 결과 노출 확인
- "토이키즈스윔클럽" 선택 가능 확인

4. 학부모 가입 요청 저장 확인
- 관련 요청 테이블/연결 테이블 저장 확인

5. 관리자 승인 관리 확인
- 방금 요청한 학부모가 목록에 즉시 표시되는지 확인

6. 승인 처리
- 승인 후 학부모 연결 반영 확인
- 학부모 계정에서 연결 상태 확인

7. 재접속 테스트
- 학부모 앱 재진입 시 수영장 연결 유지 확인
- 관리자 화면에서도 승인 상태 유지 확인

━━━━━━━━━━━━━━━━━━
9. 실패 기준
━━━━━━━━━━━━━━━━━━

아래 중 하나라도 해당하면 실패:

- approved 수영장이 학부모 검색에 안 나옴
- 검색어 입력해도 부분검색 안 됨
- 학부모 가입 요청 저장 안 됨
- 관리자 승인 목록에 즉시 안 뜸
- 승인 후 연결 반영 안 됨
- 캐시 때문에 오래된 상태가 계속 보임

━━━━━━━━━━━━━━━━━━
