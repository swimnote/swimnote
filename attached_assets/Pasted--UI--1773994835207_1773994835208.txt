학부모 모드를 아이콘 중심 + 최신소식 피드형 구조로 전면 개편한다.
UI만 만드는 것이 아니라 관리자/선생님 데이터와 실제 연결되어 작동하도록 구현한다.

=====================================
[최종 목표]

1. 깔끔하고 직관적인 화면
2. 엄마가 바로 이해하고 바로 누를 수 있는 구조
3. 모든 기능이 현재 선택된 자녀 기준으로 정확히 동작
4. 공지/일지/앨범/출결/쪽지/프로그램이 실제 데이터와 연결

=====================================
[전역 데이터 기준 - 매우 중요]

모든 데이터는 “현재 선택된 학생 연결 기준”으로 필터링한다.

필터 기준:
- studentId
- facilityId (수영장)
- 또는 studentConnectionId

적용 대상:
- 공지사항
- 수업일지
- 앨범
- 출결
- 수업일정표
- 쪽지
- 최신소식 피드

각 화면마다 따로 필터링하지 말고 공통 필터 함수로 통일한다.

=====================================
[1. 학부모 홈 화면]

구조는 3영역으로 고정한다.

-------------------------------------
A. 상단 헤더
-------------------------------------

구성:
- 왼쪽: 현재 선택된 자녀 이름
- 클릭 시 자녀 전환 드롭다운
- 오른쪽: 알림 아이콘
- 오른쪽: 쪽지 아이콘

규칙:
- 자녀 + 수영장 연결 기준으로 전환
- 모든 데이터 즉시 갱신
- 쪽지 아이콘: 안읽은 메시지 수 배지
- 알림 아이콘: 향후 확장용 (기본 구조만 유지)

-------------------------------------
B. 아이콘 영역 (3x2 고정)
-------------------------------------

순서 고정:

1. 공지사항
2. 수업일지
3. 앨범
4. 수업일정표
5. 교육 프로그램
6. 설정

규칙:
- 한 화면에 6개 모두 표시 (스크롤 없음)
- 클릭 영역 크게
- 텍스트 + 아이콘 같이 표시

배지 규칙:
- 공지사항: 안읽은 공지 숫자
- 쪽지: 안읽은 메시지 숫자
- 앨범: 새 사진 있으면 “N”
- 수업일지: 배지 없음

-------------------------------------
C. 최신소식 피드
-------------------------------------

제목:
- 최신소식

구성:
- 공지 + 수업일지 통합 피드
- 최신순 정렬 (createdAt 기준)

포함 데이터:
- 공지사항
- 수업일지

제외 데이터:
- 앨범
- 쪽지
- 출결
- 교육 프로그램

노출:
- 최근 10개만 표시
- “더보기” 클릭 시 전체 리스트 화면 이동

=====================================
[2. 최신소식 카드]

카드는 공지/일지 동일한 디자인 계열 사용

-------------------------------------
공지 카드
-------------------------------------
- 제목
- 본문 일부
- 태그: 전체공지 / 우리반공지
- 미열람 표시

동작:
- 클릭 시 상세 열기
- 열면 즉시 읽음 처리

-------------------------------------
수업일지 카드
-------------------------------------
- 날짜
- 선생님 이름
- 본문 일부
- 사진 썸네일
- 개별일지 존재 시 강조 박스

동작:
- 클릭 시 상세 이동
- 사진 저장 가능

=====================================
[3. 공지사항 화면]

규칙:
- 관리자 + 선생님 공지 모두 포함
- 태그는 2개만 사용
  → 전체공지 / 우리반공지

읽음 규칙:
- 학생 기준으로 읽음 상태 저장
- 상세 진입 시에만 읽음 처리
- 리스트 미리보기로는 읽음 처리 안함

배지:
- 현재 선택 자녀 기준 안읽은 개수 표시

정렬:
- 최신순

=====================================
[4. 수업일지 화면]

핵심 화면

노출 조건:
- 현재 선택 자녀 포함 수업
- publishedToParent = true
- 삭제 아님
- 숨김 아님

카드:
- 날짜
- 선생님
- 내용 일부
- 사진 썸네일
- 개별일지 강조

상세:
- 전체일지 + 개별일지 분리 표시
- 사진 전체 표시
- 사진 저장 가능

권한:
- 읽기만 가능
- 댓글/업로드 불가

=====================================
[5. 앨범]

구성:
- 최신순 정렬
- 반 전체 / 개인 사진 구분

권한:
- 보기
- 저장
- 업로드/삭제 불가

아이콘 규칙:
- 새 사진 있으면 “N”
- 앨범 진입 시 N 제거

=====================================
[6. 수업일정표 / 출결]

아이콘: 수업일정표

내부 구성:
- 출결 중심 UI

출결 방식:
- 도장형 UI

기준:
- 주1 → 4개
- 주2 → 8개
- 주3 → 12개

표시:
- 출석 → 채워진 도장
- 결석 → X
- 보강 → 별도 표시

목표:
- 이번 달 수업 횟수 한눈에 확인

=====================================
[7. 쪽지 (팝업 구조)]

쪽지는 페이지가 아니라 팝업

-------------------------------------
상단 안내문 (고정)
-------------------------------------
- 간단 문의만 보내주세요
- 상세 상담은 전화 이용
- 일지/출결 확인용

-------------------------------------
A. 쪽지함 팝업
-------------------------------------
- 카드형 리스트
- 스레드 목록

UI:
- 학부모 → 오른쪽
- 선생님 → 왼쪽
- 시간 표시 없음
- 읽음 표시만

-------------------------------------
B. 답장 팝업
-------------------------------------
- 기존 대화 표시
- 입력창 + 보내기

제한:
- 300~500자
- 첨부 없음

규칙:
- 담당 선생님 1명과만 1:1
- 관리자 쪽지 불가
- 스레드 기준:
  studentId + teacherId + facilityId

=====================================
[8. 교육 프로그램]

구조:
- 수영장당 1개 문서

권한:
- 관리자만 작성
- 학부모 읽기만

표시:
- 작성됨 → 제목 + 본문
- 없음 → “프로그램 공개전”

=====================================
[9. 설정]

메뉴:
- 학부모 정보 수정
- 자녀 연결/추가
- 연결된 자녀
- 알림 설정
- 약관
- 개인정보
- 고객센터
- 로그아웃
- 탈퇴

자녀 연결 흐름:
- 요청 → 승인 대기 → 승인 → 연결 완료

=====================================
[10. 공통 UX 규칙]

- 뒤로가기 위치 통일
- 홈 버튼 위치 통일
- 팝업 스타일 통일
- 키보드 가림 방지
- 빈 화면 안내문 표시
- 클릭 즉시 반응
- 죽은 버튼 금지

디자인:
- 밝고 부드럽게
- 여백 넉넉
- 큰 터치 영역

=====================================
[11. 데이터 연결]

-------------------------------------
선생님 → 학부모
-------------------------------------
- 일지 작성 → 자동 반영
- 사진 업로드 → 자동 반영
- 쪽지 답장 → 배지 반영

-------------------------------------
관리자 → 학부모
-------------------------------------
- 공지 반영
- 교육 프로그램 반영
- 읽음 상태 저장

=====================================
[12. 필수 테스트]

- 아이콘 6개 전부 작동
- 공지 배지 정상 작동
- 쪽지 배지 정상 작동
- 앨범 N 정상 작동
- 최신소식 공지+일지 섞임 확인
- 일지 실시간 반영 확인
- 사진 저장 가능 확인
- 자녀 전환 시 데이터 변경 확인
- 쪽지 팝업 정상 작동
- 키보드 가림 없음
- 빈 화면 없음

=====================================
[최종 명령]

학부모 모드를 아이콘 6개 중심의 심플한 홈 구조로 전면 개편하고,
공지/수업일지/앨범/출결/쪽지/교육프로그램이 현재 선택된 학생 연결 기준으로 정확히 필터링되어 실제 데이터와 연결되도록 구현해라.
UI는 깔끔하고 직관적으로, 엄마 사용자가 바로 이해하고 사용할 수 있게 완성한다.
