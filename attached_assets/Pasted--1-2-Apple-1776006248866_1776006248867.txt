목표는 두 가지입니다.

1. 슈퍼관리자 모드에서 전체 운영 상태가 실시간으로 정상 반영되는지 확인
2. 새 빌드가 Apple 심사에서 다시 걸릴 리스크가 없는지 실제 동작 기준으로 검증

이번 작업은 “수정”보다 “검증 + 필요한 최소 수정 + 재빌드 준비”가 핵심입니다.
추정 금지, 실제 코드/실행 결과 기준으로 보고하세요.

1. 슈퍼관리자 모드 실시간 운영 검증
━━━━━━━━━━━━━━

슈퍼관리자에서 아래 항목이 전부 현재 서버 상태와 실시간으로 맞는지 점검하세요.

검증 대상:
- 수영장 목록 / 운영 상태
- 회원 수 / 한도 / 플랜 표시
- 결제 상태
- 스케줄러 상태
- 락/heartbeat 반영 상태
- 부모 연결 상태
- 업로드 결과 반영
- 최근 변경사항 반영 여부

필수 확인:
1. 수영장별 회원 수가 실제 DB와 일치하는지
2. 최근 학생 등록/수정/엑셀 업로드가 슈퍼관리자 화면에 즉시 반영되는지
3. parent-link / auto-attendance / push / backup 관련 상태가 관리자에서 오해 없이 보이는지
4. scheduler heartbeat 또는 health가 있다면 최신 실행 시간이 보이는지
5. 실시간 반영이 안 되는 화면이 있으면 캐시/쿼리/리프레시 문제인지 원인까지 제시

특히 아래를 실제로 검증하세요.
- 학생 1명 수정 → 슈퍼관리자 화면 재조회 시 즉시 반영
- 엑셀 업로드 1회 성공 → 회원 수 / 학생 목록 반영
- 부모 연결 성공/대기 상태 변경 → 슈퍼관리자에서 반영
- 스케줄러 실행 후 관련 상태값 변화 확인

결과 보고 형식:
- 화면명
- 확인한 데이터
- 실시간 반영 여부 (정상/지연/오류)
- 문제 있으면 원인과 수정 파일

━━━━━━━━━━━━━━
2. Apple 심사 리스크 최종 점검
━━━━━━━━━━━━━━

이번 빌드는 Apple 심사 재제출 전 최종 점검입니다.
특히 아래 2개 기준을 중점 확인하세요.

A. App Completeness / Stability
- 앱 실행 시 크래시 없음
- 로그인/회원가입/Apple Sign In/학부모 연결/슈퍼관리자 주요 흐름 정상
- 빈 화면, 막힌 버튼, 작동 안 하는 탭, 잘못된 링크, 의미 없는 버튼 없음
- iPhone 실제 빌드에서 UI 깨짐 없음
- 오류 시 사용자에게 이해 가능한 메시지 노출
- 서버 변경 이후에도 앱 기능이 정상적으로 이어지는지 확인

B. Login Services / Apple Sign In
- Apple Sign In 신규 사용자 생성 정상
- phone/swimming_pool_id nullable 처리 후도 심사 흐름상 문제 없는지 확인
- Apple 계정 첫 로그인 → 계정 생성 → 정상 화면 진입 재검증
- 다른 로그인 방식과 Apple 로그인 흐름이 서로 다르게 막히는 부분 없는지 점검

공식 기준 참고 포인트:
- App Completeness (앱이 실제로 정상 동작해야 함)
- Login Services 4.8 (서드파티/소셜 로그인 시 동등한 로그인 옵션 요건) 기준에 다시 걸릴 요소가 없는지 검토

반드시 실제 점검할 항목:
1. 앱 첫 실행
2. 로그인 화면
3. 회원가입
4. Apple Sign In
5. 학부모 홈
6. 자녀 연결
7. 슈퍼관리자 핵심 화면
8. 결제/구독 진입
9. 메신저 / 공지 / 사진 / 일지
10. 빈 상태 화면 / 토스트 / 오류 메시지

서버 구조 변경이 새 빌드에서 문제를 만들지 점검하세요.

확인 대상:
- WORKER_MODE 분리 후 앱 기능 영향 없음
- scheduler_locks / heartbeat 추가 후 API 응답 영향 없음
- DB_POOL_MAX 변경 후 앱 응답 문제 없음
- bulk upload 전체 거부 구조가 앱 내 실제 UX와 충돌 없는지
- 이름 정규화(name_korean / normalizeKoreanName) 이후 학부모 표시/연결 이상 없음
- validation.ts 통일 이후 회원가입/프로필/설정 수정 흐름 이상 없음

특히 아래는 실제 기기 빌드 기준으로 검증:
- 앱 삭제 후 재설치
- 첫 실행
- 로그인
- 자녀 연결
- 홈 표시
- 로그아웃 후 재실행
- Apple 로그인
- 관리자/슈퍼관리자 접근
