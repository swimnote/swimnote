이번 작업은 비용 절감을 위해 한 번에 정리해서 진행해줘.
중복 수정 없이 전체 구조를 먼저 분석한 뒤 필요한 변경만 한 번에 반영해줘.

이번 작업 범위:
"플랫폼 관리자 권한 구조 세분화 + 수영장 상세 진입 시 읽기/수정 권한 분리 + 플랫폼 계정관리 화면 정리"까지만 진행해줘.
그 외 UI 리디자인, 불필요한 리팩토링, 성능 최적화는 하지 마.

목표:
플랫폼 운영자는 여러 명일 수 있지만,
플랫폼 최종 관리자 1명만 전체 최고 권한을 가지게 하고,
일반 플랫폼 관리자들은 권한에 따라 구독 중인 수영장을 읽기 전용 또는 수정 가능으로 접근할 수 있게 해줘.

필수 권한 구조:

1. 플랫폼 관리자 역할 구분
- superPlatformAdmin : 플랫폼 최종 관리자, 1명만 존재
- platformAdmin : 일반 플랫폼 관리자, 여러 명 가능

2. superPlatformAdmin 권한
- 플랫폼 내부 관리자 계정 생성/수정/삭제 가능
- 모든 플랫폼 관리자 권한 부여/회수 가능
- 모든 수영장 상세 페이지 자유롭게 진입 가능
- 모든 수영장 데이터 읽기/수정 가능
- 수영장 승인/반려/정지 가능
- 구독 상태 관리 가능
- 플랫폼 전체 설정 관리 가능

3. platformAdmin 권한
- 플랫폼 최종 관리자가 부여한 권한 범위 내에서만 동작
- 기본적으로는 수영장 상세 페이지 읽기 전용 접근 가능
- 수정 가능 여부는 별도 권한으로 부여받은 경우에만 허용
- 플랫폼 관리자 계정 권한 변경은 불가

4. platformAdmin 권한 옵션 항목 추가
각 플랫폼 관리자 계정에 아래 권한 옵션을 둘 수 있게 해줘:
- canViewPools : 수영장 상세 조회 가능
- canEditPools : 수영장 상세 수정 가능
- canApprovePools : 수영장 가입 승인/반려 가능
- canManageSubscriptions : 구독 상태 관리 가능
- canManagePlatformAdmins : 플랫폼 관리자 계정 관리 가능

5. 권한 기본값
- superPlatformAdmin 은 모든 권한 true
- 일반 platformAdmin 은 기본값:
  canViewPools = true
  canEditPools = false
  canApprovePools = false
  canManageSubscriptions = false
  canManagePlatformAdmins = false

6. 플랫폼 계정관리 화면 수정
- 플랫폼 계정관리 화면에는 플랫폼 내부 관리자 계정만 표시
- 전국 수영장의 선생님, 학부모, 회원 계정은 여기서 보이지 않게 유지
- 각 플랫폼 관리자 계정 카드나 상세 화면에서 권한 옵션을 확인/변경 가능하게 해줘
- 단, 권한 변경은 superPlatformAdmin 만 가능하게 해줘

7. 수영장 상세 진입 시 권한 분기
- canViewPools=true 인 계정은 수영장 상세 진입 가능
- canEditPools=false 인 경우:
  - 모든 화면을 읽기 전용으로 표시
  - 수정/저장/삭제 버튼 숨김 또는 비활성화
  - 회원/반/출결/수영일지/사진/공지 화면도 읽기만 가능
- canEditPools=true 인 경우:
  - 해당 수영장 상세에서 수정 가능
  - 회원/반/출결/수영일지/사진/공지 수정 가능

8. superPlatformAdmin 전용 기능
- 플랫폼 최종 관리자는 모든 구독 중인 수영장에 자유롭게 들어가서 수정 가능
- 일반 플랫폼 관리자는 부여된 권한 범위 내에서만 접근 가능

9. 보안 처리
- 프론트엔드에서 버튼만 숨기지 말고
  백엔드/API/DB 접근 단계에서도 권한 체크를 적용해줘
- 읽기 전용 계정이 직접 API 호출로 수정하지 못하게 해줘

10. JSON 응답 유지
- 모든 API 응답은 JSON 형식 유지
- 에러 발생 시에도 JSON 반환
- 예:
  { "success": true, "data": ... }
  { "success": false, "message": "forbidden" }

11. 테스트 조건
- superPlatformAdmin 1명 생성 또는 유지
- 일반 platformAdmin 1명은 읽기 전용 권한으로 테스트
- 일반 platformAdmin 1명은 수정 가능 권한으로 테스트
- 같은 수영장에 대해
  - 읽기 전용 관리자는 조회만 가능하고 저장/수정 불가
  - 수정 가능 관리자는 실제 수정 가능
  - superPlatformAdmin 은 전체 자유 접근 가능
- 기존 멀티테넌트 poolId 구조와 충돌 없이 동작하게 해줘

작업 방식:
1. 관련 auth, role, permissions, account management, pool detail 접근 로직을 먼저 분석
2. 같은 파일은 가능하면 한 번만 수정
3. UI 전체 변경, 스타일 정리, 성능 최적화는 하지 마
4. 이번 턴에서는 오직 플랫폼 관리자 권한 구조와 읽기/수정 분리만 진행

마지막 결과 보고 형식:
1. 수정한 파일 목록
2. 추가한 역할(role) 목록
3. 추가한 권한(permission) 목록
4. 읽기 전용으로 제한된 화면 목록
5. 수정 가능으로 열린 화면 목록
6. 테스트 계정과 테스트 방법
7. 남은 후속 작업 목록
