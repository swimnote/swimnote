[반 제외/반 이동/반 변경 이력 표시 강화 + 변경 메모 주차 한정 노출 + 주간 스케줄러 재구성 포함 최종 지시서]

지금부터 아래 기능을 기존 반 제외 시점 로직과 주간 스케줄러 수정 작업에 추가한다.

이번 작업 목표는 단순 반 제외가 아니라,
“변경이 있었던 주차에는 변경 흔적을 남기고,
그 이후 주차에는 깔끔한 최종 상태만 보이게 하는 운영형 스케줄러”를 완성하는 것이다.

중요:
- 과거 기록은 유지
- 변경이 발생한 기준 주차에만 점 표시와 변경 메모 노출
- 다음 주/다다음 주 이후에는 점 표시와 변경 메모가 보이면 안 됨
- 미래 스케줄은 변경 반영된 결과만 깔끔하게 보여야 함
- 반 제외/반 이동/반 변경 모두 같은 방식으로 처리
- 기존에 지시한 반 제외 시점 로직 + 주간 스케줄러 재구성 작업에 이번 요구사항까지 합쳐서 다시 정리하고 구현
- 거짓 완료 보고 절대 금지

==================================================
1. 최종 반 변경 이력 표시 정책
==================================================

반 제외 / 반 이동 / 반 변경이 발생한 경우
“변경이 실제로 발생한 기준 주차의 해당 반”에만
작은 점 표시(dot marker)를 남긴다.

예:
- 이번 주 수요일 반에서 회원 제외
→ 이번 주 수요일 해당 반 카드/블록에 점 1개 표시
→ 그 반을 누르면 변경 메모 확인 가능
→ 다음 주 수요일 스케줄에는 점 표시 없음

- 다음 주부터 반 이동 예약
→ 변경이 적용되는 다음 주의 해당 반에만 점 표시
→ 그 주차 반을 누르면 변경 메모 확인 가능
→ 그 다음 주차부터는 점 없이 변경된 상태만 표시

중요:
- 점 표시는 “이 주차에 변경 이력이 있다”는 뜻
- 영구 배지처럼 계속 보이면 안 됨

==================================================
2. 변경 메모 노출 정책
==================================================

반을 눌렀을 때
변경이 발생한 해당 주차/해당 날짜/해당 반에만
변경 메모가 보여야 한다.

변경 메모 예시:
- 회원 1명 반 제외 → 미배정 이동
- 회원 반 이동 예정
- 반 삭제 예정
- 반 시간 변경
- 담당 선생님 변경

중요:
- 변경 메모는 “변경이 있었던 그 주차”에만 보여야 한다
- 다음 주 스케줄에는 변경 메모가 보이면 안 된다
- 과거 해당 주차를 다시 보면 메모는 보여야 한다
- 미래 최종 반영 상태 화면은 깔끔해야 한다

==================================================
3. 변경 메모 저장 방식
==================================================

변경 이력은 단순 UI 상태가 아니라 기록으로 남겨야 한다.

권장 구조:
change_logs / schedule_change_logs / class_change_logs 등
별도 변경이력 저장 구조 사용

필수 저장 필드 예시:
- id
- pool_id
- class_group_id
- target_student_id (있으면)
- change_type (remove_from_class / move_class / delete_class / modify_class 등)
- effective_date
- display_week_start
- display_date
- note
- created_by
- created_at

핵심:
- effective_date = 실제 반영 시작일
- display_week_start 또는 display_date = 점 표시/메모 노출 기준 주차
- 이력은 저장하되, 노출은 해당 주차로 제한

==================================================
4. 점 표시 규칙
==================================================

스케줄러에서 반 블록에 작은 점(marker)을 표시하는 조건:

- 현재 보고 있는 날짜/주차에
  해당 반에 대한 change_log가 존재할 때만 표시

표시 위치:
- 반 카드 우측 상단 또는 제목 근처
- 너무 크지 않게
- 다른 배지/텍스트와 겹치지 않게

중요:
- 항상 보이면 안 됨
- 과거/기준 주차에만 보임
- 다음 주 이후에는 점 제거

==================================================
5. 반 눌렀을 때 변경 메모 노출 규칙
==================================================

반 카드/시간 블록 클릭 시:

기본 정보:
- 반 이름
- 시간
- 현재 소속 회원
- 반배정 / 출결 / 수업일지 등 기존 기능

추가:
- 현재 보고 있는 주차에 해당하는 change_log가 있으면
  “변경 메모” 영역 노출

예:
[변경 메모]
- 3월 24일: 회원 1명 반 제외 → 미배정 이동

중요:
- 다른 주차에서는 이 영역 숨김
- 미래 최종 상태 주차에서는 메모가 없어야 함

==================================================
6. 반 제외 시점 로직 + 변경 메모 결합
==================================================

기존 지시 유지:
- 오늘부터 제외
- 다음 주부터 제외
- 다다음 주부터 제외

추가 규칙:
- 사용자가 제외 시점을 선택하면
  그 적용 시작 주차에 change_log 생성
- 해당 주차의 해당 반에 점 표시
- 해당 주차에서만 변경 메모 노출
- 그 이후 주차는 변경 반영 결과만 깔끔하게 표시

예:
다음 주부터 제외 선택
→ 이번 주는 기존 반 유지, 점 없음
→ 다음 주 해당 반에 점 표시 + 변경 메모
→ 다다음 주부터는 점 없이 최종 상태 유지

==================================================
7. 반 삭제/이동도 동일하게 적용
==================================================

반 제외뿐 아니라 아래도 같은 구조로 통일:

- 반 삭제
- 반 이동
- 반 시간 변경
- 담당 선생님 변경

즉:
“변경이 일어난 주차에만 점 + 메모”
“그 이후는 최종 반영 상태만 노출”

중요:
- 기능마다 따로 놀지 말고
- 하나의 변경이력 시스템으로 묶어라

==================================================
8. 주간 스케줄러 재구성 (기존 지시 유지)
==================================================

반드시 같이 적용:

- 월 화 수 목 금 토 일
- 오전 6시 ~ 오후 9시
- 모든 시간 칸 생성
- 연한 그리드 라인
- 토요일 잘림 없음
- 일요일 추가
- 마지막 칸(21시) 깨짐 없음

그리고 변경 이력 점 표시도
이 새 주간 스케줄러 구조에서 정확하게 보이게 한다.

==================================================
9. 월간/일간/주간 공통 규칙
==================================================

변경 이력 표시가 가능하면
월간/주간/일간 기준도 최대한 통일:

- 주간: 점 표시 + 메모 노출 (핵심)
- 일간: 해당 날짜에 변경 이력 있으면 메모 노출 가능
- 월간: 점 표시까지만 검토 가능 (너무 복잡하면 최소 점만)

하지만 최소한:
- 주간 스케줄러에서는 완전하게 작동해야 함

==================================================
10. 과거 기록 유지 원칙 재강조
==================================================

절대 금지:
- 변경 때문에 과거 출결/일지/소속이 수정되는 것
- change_log 때문에 실제 수업기록이 덮어쓰기 되는 것

반드시 유지:
- 과거 주차는 당시 상태 그대로 조회
- 변경 주차는 점/메모 표시
- 이후 주차는 최종 반영 상태만 표시

==================================================
11. 테스트 필수 시나리오
==================================================

시나리오 A
- 이번 주 반 제외 실행
기대 결과:
- 이번 주 해당 반에 점 표시
- 반 클릭 시 변경 메모 표시
- 다음 주에는 점/메모 없음
- 다음 주부터 회원 제외 상태 반영

시나리오 B
- 다음 주부터 반 제외 실행
기대 결과:
- 이번 주 점/메모 없음
- 다음 주 해당 반에 점 표시 + 변경 메모
- 다다음 주부터는 깔끔한 최종 상태만 보임

시나리오 C
- 다다음 주부터 반 이동 실행
기대 결과:
- 다다음 주 해당 반에 점 표시 + 메모
- 이후 주차는 점 없음
- 이동된 결과만 반영

시나리오 D
- 반 삭제 실행
기대 결과:
- 삭제 적용 주차의 해당 반에 점 표시 + 메모
- 이후 주차는 반 미노출
- 소속 회원은 적용 시점부터 미배정 이동
- 과거 기록 유지

시나리오 E
- 지난주/이번주/다음주 비교 조회
기대 결과:
- 과거는 과거대로 유지
- 변경 주차에만 메모 보임
- 미래는 최종 상태만 보임

==================================================
12. 의심 항목 전수 점검
==================================================

조금이라도 의심되면 바로 확인:
- change_log가 계속 모든 주차에 따라다니는지
- dot marker가 다음 주에도 남아 있는지
- 메모가 미래 주차에도 보이는지
- 반 제외/이동/삭제별로 처리 방식이 다른지
- 스케줄러 성능 저하가 생기는지
- 점 표시 때문에 블록 레이아웃이 깨지는지
- 토요일/일요일 컬럼에서 점이 잘리는지
- Expo Go에서 점 렌더 후 터치 이상 없는지

==================================================
13. 완료 기준
==================================================

다음 조건을 모두 만족해야 완료다.

- 변경 주차에만 점 표시
- 변경 주차에만 메모 노출
- 다음 주 이후 스케줄에는 메모 비노출
- 미래 스케줄은 깔끔한 최종 상태만 표시
- 반 제외/이동/삭제 공통 적용
- 과거 기록 유지
- 주간 스케줄러 7일 / 06시~21시 구조 정상
- iPhone Expo Go에서 정상 작동

14. 완료 보고 형식
==================================================

반드시 아래 형식으로 보고하라.

1. change_log 저장 구조를 어떻게 구현했는지
2. 점 표시 조건
3. 메모 노출 조건
4. 반 제외/이동/삭제 각각 테스트 결과
5. 이번주/다음주/다다음주 비교 조회 결과
6. 주간 스케줄러 점 표시 UI 결과
7. iPhone Expo Go 기준 결과
8. 남은 예외/의심 항목 여부

중요:
- “메모 추가 완료” 한 줄 보고 금지
- 날짜별 조회 결과를 적어라
- 미확인 항목은 미확인이라고 적어라
