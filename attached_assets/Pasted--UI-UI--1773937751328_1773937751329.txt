지금부터 관리자 모드 UI를 “기존 하단 탭 앱 구조”가 아니라
“홈 중심 아이콘 운영 시스템” 구조로 정리하라.

이번 작업 목표는
현재 관리자 홈에 적용된 아이콘 UI를
진짜 운영 시스템 구조로 완성하는 것이다.

가장 중요:
- 기존 기능 로직은 깨뜨리지 말 것
- 지금 있는 아이콘 홈 구조는 살리되
  기존 하단 탭 기반 앱 구조 흔적을 제거할 것
- 홈 → 아이콘 → 팝업 → 페이지 구조를 강제로 통일할 것
- 결석 → 보강대기 자동 연결을 반드시 복구할 것
- 이번 작업은 관리자 모드만 우선 적용
- 이후 같은 규칙을 선생님 모드에 확장할 예정이므로
  공통 규칙을 먼저 정리하고 그 규칙대로 관리자 모드를 수정할 것

==================================================
[0. 공통 규칙 - 관리자/선생님 공통으로 재사용할 규칙 먼저 고정]

이번 작업 전에 아래 UI/UX 규칙을 공통 규칙으로 먼저 정리하고 적용하라.

--------------------------------------
[공통 규칙 1. 전체 구조]

전체 구조는 아래 3단계까지만 허용

1단계: 홈
2단계: 아이콘 팝업
3단계: 실제 페이지

즉:
홈 → 팝업 → 페이지

절대 4단계 이상 금지

--------------------------------------
[공통 규칙 2. 팝업 규칙]

- 홈에서 메인 아이콘 클릭 시 반드시 팝업 오픈
- 팝업은 “아이콘 창고”
- 팝업 안에서 긴 리스트/실데이터 직접 노출 금지
- 팝업 내부 아이콘은 3열 그리드
- 팝업 디자인 전부 통일
- 팝업 바깥 터치 닫기 통일
- 팝업 X 버튼 닫기 통일
- 드래그 닫기 통일
- 팝업 내부 아이콘 클릭 시 실제 페이지 이동

--------------------------------------
[공통 규칙 3. 페이지 규칙]

모든 페이지는 공통 헤더 사용

헤더 구성:
- 왼쪽: 뒤로가기
- 오른쪽: 홈
- 수정 가능한 화면은 저장 버튼 추가

규칙:
- 뒤로가기 = 바로 이전 단계
- 홈 = 관리자 홈 첫 화면
- 저장 가능한 화면은 반드시 저장 버튼 존재

--------------------------------------
[공통 규칙 4. 리스트 규칙]

- 검색창 상단 고정
- 상태칩은 2줄 컴팩트 구조 허용
- 카드 크기는 컴팩트
- 선택/삭제 기능 유지
- 카드/버튼/간격 스타일 통일

--------------------------------------
[공통 규칙 5. 통일감]

반드시 통일할 것:
- 팝업 모양
- 팝업 아이콘 배치
- 페이지 헤더
- 뒤로가기/홈/저장 위치
- 검색창 스타일
- 상태칩 스타일
- 카드 간격
- 버튼 색/크기/라운드

==================================================
[1. 하단 탭바 제거]

관리자 모드에서는 기존 하단 탭바를 제거하라.

제거 이유:
- 현재 구조는 홈 중심 운영 시스템이므로
  하단 탭바가 기존 앱 구조 흔적으로 남아 충돌한다
- 홈의 메인 아이콘이 네비게이션 역할을 하므로
  하단 탭바는 필요 없다

중요:
- 관리자 모드에 한해 하단 탭바 제거
- 선생님/학부모 모드는 이번 단계에서 건드리지 않음
- 하단 탭바 제거로 기존 기능 페이지 진입이 끊기지 않게
  홈 아이콘과 팝업 경로로 연결 유지

==================================================
[2. 관리자 홈 화면 정리]

현재 관리자 홈은 방향은 좋지만
정보가 일부 중복되고 기존 앱 흔적이 남아 있다.

최종 홈 구조는 아래로 고정

1. 상단 헤더
2. 요약 배너 (2x2)
3. 경고 배너
4. 메인 아이콘 8개
5. 불필요한 중복 요약 제거

==================================================
[3. 상단 헤더 구성]

상단 헤더 구성:
- 좌측: 수영장 이름(토이키즈)
- 우측: 검색
- 우측: 로그아웃 또는 나가기 버튼

중요:
- 관리자 첫 홈 화면에서는 홈 버튼 필요 없음
- 첫 홈 자체가 홈이기 때문
- 검색은 유지 가능
- 알림 버튼이 필요하면 추후 추가 가능 구조로 준비

==================================================
[4. 홈 요약 배너 구성]

요약 배너는 2x2 카드형으로 유지

배치할 4개 항목:
1. 데이터 사용량
2. 이번 달 전체 매출
3. 이번 달 전체 회원
4. 미처리 보강

규칙:
- 수치가 가장 크게 보이게
- 제목 + 보조 설명 작게
- 각 카드 클릭 시 해당 페이지 또는 해당 기능으로 이동 가능 구조 준비
- 지금은 우선 표시 우선, 클릭 연결도 가능하면 연결

예:
데이터 사용량 → 데이터관리
이번 달 전체 매출 → 매출관리
전체 회원 → 인원관리 > 회원관리
미처리 보강 → 보강관리 > 보강대기

==================================================
[5. 경고 배너 유지]

홈 요약 배너 아래 경고 배너 유지

예:
- 보강 미처리 3건 — 탭하여 처리

규칙:
- 클릭 시 보강관리 > 보강대기 페이지로 이동
- 미처리 건수가 0이면 숨기거나 비활성 표시 가능
- 경고/주의 시각 강조 유지

==================================================
[6. 메인 아이콘 8개 유지]

홈 배너 아래 8개 메인 아이콘 유지

배치:
- 4 x 2 그리드
- 한 화면에 8개 다 보이게

메인 아이콘 8개:
1. 인원관리
2. 수업관리
3. 보강관리
4. 메신저
5. 매출관리
6. 데이터관리
7. 운영설정
8. 플랫폼설정

규칙:
- 아이콘 카드 크기 통일
- 라벨 한 줄 또는 두 줄 허용 가능하되 전체 균형 유지
- 아이콘 뱃지(빨간 숫자) 지원
- 터치 영역 충분히 확보
- 눌림 피드백(pressed state) 추가

==================================================
[7. 메신저 예외 규칙 유지]

메신저는 단독 예외로 유지한다.

동작:
- 홈에서 메신저 아이콘 클릭
- 팝업 없이 바로 메신저 페이지 이동

이유:
- 메신저는 이미 대화/공지 탭 구조가 있음
- 단일 진입이 더 자연스럽다

중요:
- 메신저 기능은 수정하지 말 것
- 진입 방식만 유지

==================================================
[8. 나머지 7개는 무조건 팝업]

메신저를 제외한 7개 메인 아이콘은
클릭 시 반드시 공통 팝업이 떠야 한다.

즉:
인원관리 → 팝업
수업관리 → 팝업
보강관리 → 팝업
매출관리 → 팝업
데이터관리 → 팝업
운영설정 → 팝업
플랫폼설정 → 팝업

팝업 안에서는 세부 아이콘 3열 그리드로 표시

==================================================
[9. 공통 팝업 컴포넌트 생성]

공통 팝업 컴포넌트를 새로 만들거나 통일하여 사용하라.

공통 팝업 구성:
- 제목
- 닫기 버튼(X)
- 필요 시 설명 한 줄
- 세부 아이콘 3열 그리드
- 바깥 영역 터치 시 닫힘
- 아래로 드래그 시 닫힘
- 배경 dim 처리 통일

팝업 디자인 통일:
- 라운드 값
- 내부 padding
- 아이콘 크기
- 아이콘 라벨
- 컬러 톤
- 닫기 위치

중요:
기능마다 다른 팝업 모양 금지

==================================================
[10. 7개 팝업의 세부 아이콘 연결]

--------------------------------------
[10-1. 인원관리 팝업]

아이콘:
- 회원관리
- 학부모관리
- 선생님관리
- 미배정관리
- 승인관리
- 초대관리

--------------------------------------
[10-2. 수업관리 팝업]

아이콘:
- 수업스케줄
- 반관리
- 출결관리
- 수업일지
- 수강생관리
- 반이동

--------------------------------------
[10-3. 보강관리 팝업]

아이콘:
- 보강대기
- 보강배정
- 보강현황
- 보강정책

--------------------------------------
[10-4. 매출관리 팝업]

아이콘:
- 월별매출
- 결제현황
- 미납관리
- 정산확인
- 수업발생
- 휴무일관리

--------------------------------------
[10-5. 데이터관리 팝업]

아이콘:
- 저장공간현황
- 계정별사용량
- 카테고리별사용량
- 원본데이터삭제
- 이벤트기록
- 백업/복구정책

--------------------------------------
[10-6. 운영설정 팝업]

아이콘:
- 수업운영설정
- 보강정책설정
- 권한설정
- 알림설정
- 레벨/테스트설정
- 피드백기본설정

--------------------------------------
[10-7. 플랫폼설정 팝업]

아이콘:
- 구독관리
- 화이트라벨
- 워터마크
- 로고/브랜드
- 앱기본설정
- 연결서비스

==================================================
[11. 세부 아이콘 클릭 → 기존 페이지 연결]

각 팝업 아이콘 클릭 시
그때 실제 페이지가 열리게 하라.

중요:
- 기존 기능 페이지/라우팅 유지
- 기존 데이터 로직 유지
- 세부 아이콘은 기존 페이지의 새 진입점 역할만 해야 함

예:
홈 → 수업관리 → 팝업 → 출결관리 → 기존 출결 페이지
홈 → 인원관리 → 팝업 → 회원관리 → 기존 회원관리 페이지

==================================================
[12. 중복 정보 제거]

현재 홈 하단에 “오늘 수업 현황” 같은 추가 요약이 있다면
우선 제거하거나 아래로 밀어내라.

원칙:
- 홈은 “요약 + 진입” 중심
- 상세 통계는 해당 페이지에서 본다

즉:
홈에서 정보가 중복되지 않게 정리

우선 홈에는
- 2x2 요약 배너
- 경고 배너
- 메인 아이콘 8개
까지만 핵심으로 두고 깔끔하게 유지

==================================================
[13. 결석 → 보강대기 자동 연결 복구]

현재 핵심 버그:
결석 처리해도 보강대기리스트로 자동 연결이 안 되는 상태

이걸 반드시 수정하라.

정상 흐름:
1. 출결에서 결석 처리
2. 즉시 보강대기 생성
3. 보강대기 리스트에 즉시 반영
4. 홈 배너 “미처리 보강” 수치 갱신
5. 경고 배너 수치 갱신
6. 보강관리 아이콘에 뱃지 표시 가능하면 표시

즉:
결석 이벤트가 보강대기 생성 이벤트와 연결되어야 한다

==================================================
[14. 보강대기리스트에 보강배정 버튼 유지]

보강대기리스트 각 항목에는 반드시
- 보강배정 버튼
이 있어야 한다

규칙:
- 관리자가 직접 보강 배정 가능
- 추후 선생님도 가능하게 확장 가능
- 이번 단계에서는 버튼과 연결 구조 유지

==================================================

[18. 테스트 - 반드시 실제 수행]

테스트 1.
관리자 홈에서 하단 탭바가 제거되었는지 확인

테스트 2.
홈에 2x2 요약 배너가 보이는지 확인

테스트 3.
배너에 데이터 사용량 / 이번 달 전체 매출 / 이번 달 전체 회원 / 미처리 보강이 보이는지 확인

테스트 4.
경고 배너가 정상 표시되는지 확인

테스트 5.
8개 메인 아이콘이 한 화면에 잘 보이는지 확인

테스트 6.
메신저는 팝업 없이 바로 페이지로 이동하는지 확인

테스트 7.

[20. 절대 금지]

- 하단 탭바를 남겨두는 것 금지
- 메신저를 억지로 팝업 구조에 넣는 것 금지
- 팝업 디자인을 기능마다 다르게 만드는 것 금지
- 저장 화면에 저장 버튼 빠뜨리기 금지
- 결석 → 보강대기 연결 안 하고 넘어가기 금지
- 기존 기능 페이지/데이터 로직 끊어먹기 금지

이번 작업은
관리자 모드를 진짜 아이콘 기반 운영 시스템으로 재배치하고,
하단 탭을 제거한 뒤 배너 + 8개 아이콘 + 공통 팝업 + 통일된 페이지 헤더 구조로 정리하며,
결석 처리와 보강대기 연결까지 정상 복구하는 작업이다.
