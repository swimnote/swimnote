[스윔노트 — 수영장 관리자 자동 선생님 등록 + 전환 구조 단순화 + 홈 라우팅 버그 수정 지시문]

이번 작업은 관리자/선생님 권한 구조를 단순화하고,
사라진 전환 버튼 및 슈퍼관리자 홈 이동 버그를 수정하는 작업이다.

━━━━━━━━━━━━━━━━━━

1. 핵심 구조 변경
   ━━━━━━━━━━━━━━━━━━

[목표]
수영장 관리자가 선생님 모드도 사용할 수 있게 하되,
권한 전환/추가 승인 로직을 복잡하게 만들지 않는다.

[정책]

* 수영장 관리자로 가입하면 선생님 계정도 자동 생성한다
* 별도 “관리자에게 선생님 권한 부여” 절차 없음
* 별도 전환 권한 설정 절차 없음
* 관리자 계정 1개 + 자동 생성된 선생님 계정 1개 구조로 단순화

핵심:
👉 관리자는 가입과 동시에 선생님도 된다
👉 권한 부여 UI를 따로 만들지 않는다

━━━━━━━━━━━━━━━━━━
2. 자동 생성 규칙
━━━━━━━━━━━━━━━━━━

[관리자 가입 시 자동 처리]

수영장 관리자 계정 생성 완료 후 즉시:

1. 관리자 계정 생성
2. 같은 수영장 기준 선생님 계정 자동 생성
3. 두 계정 연결 관계 저장
4. 관리자 화면/선생님 화면 모두 진입 가능 상태로 처리

[중요]

* 이 자동 생성은 관리자 가입 시 1회만 수행
* 중복 생성 금지
* 이미 연결된 선생님 계정이 있으면 재생성 금지

━━━━━━━━━━━━━━━━━━
3. 표시 이름 규칙
━━━━━━━━━━━━━━━━━━

[관리자 표시명]

* 형식: "수영장이름+관리자"

예:

* 토이키즈수영장관리자
* 블루웨이브관리자

[자동 생성된 선생님 표시명]

* 형식: "실명+선생님"

예:

* 김민수선생님
* 박지영선생님

[원칙]

* 관리자 계정 이름과 선생님 계정 이름은 다르게 보이게 처리
* 역할 구분이 화면에서 바로 되게 할 것

━━━━━━━━━━━━━━━━━━
4. 이후 일반 선생님 가입 구조
━━━━━━━━━━━━━━━━━━

[정책]

* 일반 사용자가 선생님으로 가입 신청
* 관리자 승인 시 선생님 계정 활성화
* 일반 선생님은 선생님 모드만 사용
* 관리자 전환 버튼 필요 없음
* 관리자 권한 없음

즉,

* 관리자는 자동으로 관리자+선생님 구조
* 일반 선생님은 선생님만

이 구조로 고정한다.

━━━━━━━━━━━━━━━━━━
5. 전환 버튼 규칙 수정
━━━━━━━━━━━━━━━━━━

[현재 문제]

* 선생님 모드에서 관리자로 전환하는 버튼이 사라짐

[수정]

* 자동 생성된 관리자-선생님 연결 계정에만
  선생님 모드에서 관리자 모드로 전환하는 버튼 표시
* 일반 선생님 계정에는 전환 버튼 비노출

[검증]

* 관리자 가입 계정 → 선생님 모드 진입 → 관리자 전환 버튼 보여야 함
* 일반 선생님 계정 → 전환 버튼 없어야 함

━━━━━━━━━━━━━━━━━━
6. 슈퍼관리자 홈 라우팅 버그 수정
━━━━━━━━━━━━━━━━━━

[현재 문제]

* 슈퍼관리자 모드에서 홈 버튼 누르면 수영장 관리자로 연결됨

[수정]

* 슈퍼관리자 홈 버튼은 반드시 슈퍼관리자 홈/대시보드로 이동
* admin 라우트로 절대 연결되지 않게 수정
* role/session 기반 라우팅 재점검

[검증]

* 슈퍼관리자 로그인
* 홈 버튼 클릭
* super 홈 유지 확인
* admin 화면 이동 금지

━━━━━━━━━━━━━━━━━━
7. 데이터/권한 구조 점검
━━━━━━━━━━━━━━━━━━

[필수 점검]

* 관리자 계정과 자동 생성 선생님 계정 연결 키 존재 여부 확인
* 로그인 세션에 role 정확히 저장되는지 확인
* 관리자→선생님, 선생님→관리자 전환 시 pool 기준 유지되는지 확인
* 일반 선생님은 관리자 기능 접근 불가 확인

[금지]

* 관리자 1계정에 role 여러 개를 덕지덕지 붙여서 UI만 바꾸는 방식
* 자동 생성 없이 억지 전환
* 일반 선생님에게 관리자 전환 버튼 노출

━━━━━━━━━━━━━━━━━━
8. 테스트
━━━━━━━━━━━━━━━━━━

반드시 실제로 확인:

1. 새 수영장 관리자 가입
2. 관리자 계정 생성 확인
3. 자동 생성된 선생님 계정 생성 확인
4. 관리자 모드 진입 가능 확인
5. 선생님 모드 진입 가능 확인
6. 선생님 모드에서 관리자 전환 버튼 표시 확인
7. 일반 선생님 승인 계정은 선생님 모드만 보이는지 확인
8. 슈퍼관리자 홈 버튼 버그 수정 확인

━━━━━━━━━━━━━━━━━━
9. 보고 형식
━━━━━━━━━━━━━━━━━━

반드시 포함:

* 수정 파일 경로
* 관리자 가입 후 생성된 계정 구조
* 자동 생성 선생님 계정 표시명 결과
* 전환 버튼 표시/비표시 테스트 결과
* 슈퍼관리자 홈 버튼 테스트 결과
* 실제 DB 저장 결과

“수정 완료”만 쓰지 말고 실제 생성/전환/라우팅 결과 같이 제출
