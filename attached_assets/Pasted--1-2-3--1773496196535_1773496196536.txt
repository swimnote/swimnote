회원관리 기능을 전면 구현해줘.

목표:
1. 현재 회원관리 화면에 실제 기능을 추가
2. 관리자에서 어린이 회원을 직접 등록하면 승인절차 없이 즉시 회원 생성되게 해줘
3. 직접 등록한 회원은 보호자 전화번호를 입력받고 문자 초대 메시지를 보낼 수 있는 구조로 만들어줘
4. 문자에는 학원 초대코드, 앱 다운로드 안내, 접속 URL이 포함되게 해줘
5. 학부모가 먼저 앱 가입 후 수영장을 찾아 아이 정보를 입력하고 승인 요청하는 경로도 유지해줘
6. 두 경로(admin_created / parent_requested)로 등록된 회원은 최종적으로 같은 학생 회원 데이터 구조로 합쳐지게 해줘
7. 회원 등록 후 회원 목록에서 이름을 누르면 바로 반 배정 화면으로 이동하게 해줘
8. 반 배정 권한은 관리자만 가능하게 해줘
9. 회원별로 주1 / 주2 / 주3회를 선택할 수 있게 해줘
10. 회원 목록에서 이름 옆 또는 아래에 주 몇 회인지 아이콘/배지로 표시해줘
11. 회원 목록 아래에 월4, 목7 같은 수업 진행 날짜/시간 라벨을 작게 표시해줘
12. 수업 변경 시 저장 즉시 목록, 반 배정, 선생님 화면까지 바로 반영되게 해줘
13. 선생님이 헷갈리지 않게 회원 목록과 배정 구조를 명확하게 유지해줘

구현 상세:
- 회원 등록 경로 2개
  A. 관리자 직접 등록
  B. 학부모 선가입 후 승인 요청
- 관리자 직접 등록 시 입력값:
  학생 이름, 출생년도, 보호자 전화번호, 보호자 이름(선택)
- 등록 즉시 StudentMember 생성
- 상태는 pending_parent_link 또는 active로 관리
- 문자 초대용 inviteCode 생성
- MVP에서는 문자 API 대신 문자 내용 자동 생성 및 복사 가능한 구조로 우선 구현 가능

데이터 구조:
- StudentMember
- ParentJoinRequest
- ClassAssignment
- SwimClass

StudentMember 필드:
- id
- studentName
- birthYear
- phoneNumber
- parentName
- parentUserId
- swimCenterId
- registrationPath
- status
- weeklyCount
- assignedClassIds
- scheduleLabels
- createdAt
- updatedAt

반 배정 규칙:
- 관리자만 가능
- 회원 이름 클릭 시 상세/배정 화면 진입
- 주1/주2/주3 선택
- 선택한 횟수만큼 반 선택 가능
- 저장하면 classAssignments와 students에 동시에 반영
- scheduleLabels에는 월4, 목7 같은 형식으로 저장
- 저장 즉시 회원 목록에도 반영
- 저장 즉시 선생님 내 반/스케줄/출결에도 반영될 수 있게 연결 구조 유지

UI 요구:
- 회원관리 상단에 버튼 2개
  1. 어린이 직접 등록
  2. 학부모 요청 승인
- 회원 목록 카드에 다음 표시:
  학생 이름
  주1/주2/주3 배지
  수업 라벨(월4·목7)
  학부모 연결상태
  반 배정 상태
- 이름 클릭 시 회원 상세/반배정 화면 이동
- 수업 미배정이면 명확하게 표시
- 필터/검색/목록 구조는 기존 공통 레이아웃 시스템과 통일

구현 방식:
- Expo React Native + TypeScript
- 관리자/학부모/선생님 흐름이 연결되도록 상태 구조 정리
- 현재 mock 카드만 있는 회원관리 화면을 실제 등록/배정 가능한 화면으로 교체

추가로 반드시 같이 구현해줘:

1. 중복 회원 체크
- 같은 수영장 안에서 학생 이름 + 출생년도 + 보호자 전화번호가 같으면 중복 가능성 경고 표시
- 완전 동일한 경우에는 바로 저장하지 말고 관리자에게 확인 모달을 띄워줘
- 모달에는 "기존 회원 연결", "새 회원으로 계속 등록", "취소" 선택지를 넣어줘
- 중복 검사 함수는 회원 등록 시 공통으로 사용하게 해줘

2. 미배정 회원 필터
- 회원관리 필터에 "미배정" 항목을 추가해줘
- assignedClassIds가 비어 있거나 weeklyCount는 있는데 실제 classAssignments가 부족한 회원도 미배정으로 분류해줘
- 관리자 입장에서 아직 반 편성이 안 된 회원만 바로 볼 수 있게 해줘

3. 연결대기 필터
- 회원관리 필터에 "연결대기" 항목을 추가해줘
- status가 pending_parent_link 인 회원만 따로 볼 수 있게 해줘
- 보호자 앱 연결이 아직 안 된 회원을 빠르게 관리할 수 있게 해줘

4. 배정불일치 감지
- 주2인데 반이 1개만 배정된 경우
- 주3인데 반이 2개 이하만 배정된 경우
- weeklyCount와 assignedClassIds 길이가 다를 경우
- 목록에 "배정불일치" 경고 배지를 표시해줘
- 필터에도 "배정불일치" 항목을 추가해줘

5. 문자 초대 관리
- 관리자 직접 등록 후 문자 내용이 자동 생성되게 해줘
- MVP에서는 실제 문자 발송 대신
  "초대문구 보기"
  "복사하기"
  "공유하기"
  버튼으로 먼저 구현해줘
- 추후 문자 API 붙이기 쉽게 inviteCode, inviteMessage를 상태에 저장해줘

6. 회원 상세 화면 보강
- 이름 클릭 시 들어가는 회원 상세 화면에서 아래 항목을 모두 확인 가능하게 해줘
  학생 이름
  출생년도
  보호자 이름
  보호자 전화번호
  등록 경로(admin_created / parent_requested)
  학부모 연결 상태
  주 몇 회
  현재 반 배정 목록
  수업 라벨(월4·목7)
- 이 화면에서 바로 반배정 수정 가능하게 해줘

7. 반배정 수정 즉시 반영
- 반배정 변경 후 저장하면
  students
  classAssignments
  teacherClasses
  teacherSchedule
  attendance 대상 목록
  에 모두 즉시 반영되게 해줘
- 페이지를 나갔다 다시 들어오지 않아도 즉시 보이게 상태 업데이트 해줘

8. 회원 목록 아이콘/배지 규칙
- 주1 = 파랑 배지
- 주2 = 초록 배지
- 주3 = 보라 배지
- 연결대기 = 회색 또는 주황 배지
- 미배정 = 빨강 또는 주황 경고 배지
- 배정불일치 = 빨강 경고 배지
- 학부모 연결완료 = 초록 작은 배지
- 모든 배지 위치와 크기는 통일해줘

9. 회원 목록 정렬/검색 보강
- 기본 정렬은 최근 등록순
- 검색은 학생 이름, 보호자 이름, 전화번호로 가능하게 해줘
- 필터는 아래 항목을 모두 지원해줘:
  전체
  미배정
  연결대기
  배정불일치
  주1
  주2
  주3
  학부모 연결완료

10. 안정성 처리
- 회원 등록 시 필수값 누락되면 저장 막기
- 잘못된 출생년도 형식 막기
- 전화번호 형식이 비정상이면 경고
- weeklyCount보다 많은 반을 선택하지 못하게 막기
- weeklyCount보다 적게 선택한 상태에서 저장 시 경고 띄우기
- 이미 같은 시간대에 같은 회원이 중복 배정되지 않게 막기

11. 공통 상태 계산 함수로 분리
- 아래 함수들을 유틸로 분리해줘
  checkDuplicateStudent
  getStudentAssignmentStatus
  getStudentConnectionStatus
  toShortScheduleLabel
  buildInviteMessage
- 화면마다 중복 계산하지 말고 공통 함수 사용

12. 실제 운영용으로 보기 좋게 정리
- 회원 목록 카드 높이와 간격 통일
- 상단 검색/필터 고정
- 하단 목록만 스크롤
- 카드 눌렀을 때 상세 이동
- 배정 관련 경고는 한눈에 보이게
- 선생님이 봐도 헷갈리지 않도록 수업 라벨을 명확하게 표시

최종 결과:
- 관리자 직접 등록과 학부모 선가입 승인 요청 두 경로가 모두 동작
- 회원 등록 후 바로 목록 반영
- 회원 이름 클릭 시 바로 반배정 가능
- 주1/주2/주3와 월4/목7 표시가 한눈에 보이게
- 연결대기, 미배정, 배정불일치까지 필터와 배지로 관리 가능
- 반배정 변경 시 선생님 화면까지 즉시 반영
