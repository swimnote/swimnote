지금부터 관리자 모드 아이콘 OS 구조에서 공통 페이지 헤더와 홈 버튼 동작을 전면 점검하고 수정하라.

현재 상태:
- 전체 인터페이스 방향은 마음에 든다
- 하지만 아이콘 안에서 열리는 기존 페이지들은 원래 페이지를 옮겨붙인 상태라
  일부 페이지에 뒤로가기/홈 버튼이 없거나
  홈 버튼이 눌려도 동작하지 않는 곳이 있다
- 홈 배너는 조금 크고
  아이콘 배치는 조금 더 여유 있게 보이도록 조정할 필요가 있다
- 아이콘 위 새소식/알림 숫자 배지는 좋은 방향이므로 유지한다

이번 작업 목표는
1. 모든 아이콘 안 페이지에 공통 헤더(뒤로가기/홈)를 강제 적용하고
2. 홈 버튼이 전체 페이지에서 실제 작동하는지 전수 테스트 후 수정하며
3. 홈 배너 높이를 줄이고 아이콘 배치를 더 여유 있게 다듬는 것이다.

가장 중요:
- 현재 인터페이스 컨셉은 유지
- 기능 로직은 건드리지 말 것
- 페이지 이동 구조는 유지
- 공통 헤더와 홈 버튼 동작만 완전히 통일할 것
- 홈 버튼은 “항상 관리자 홈 첫 화면으로 즉시 이동”해야 한다
- 테스트 없이 완료 보고 금지

==================================================
[0. 공통 규칙 재확인]

이번 작업에서도 아래 공통 규칙을 유지한다.

1. 홈 → 팝업 → 페이지 구조
2. 모든 실제 페이지는 공통 헤더 사용
3. 공통 헤더 왼쪽: 뒤로가기
4. 공통 헤더 오른쪽: 홈
5. 수정 가능한 페이지는 저장 버튼 추가 가능
6. 홈 버튼은 언제나 관리자 홈 첫 화면으로 이동
7. 뒤로가기는 바로 이전 단계로 이동

==================================================
[1. 모든 아이콘 안 페이지에 공통 헤더 강제 적용]

현재 아이콘 팝업에서 들어가는 페이지 중
기존 페이지를 그대로 재사용한 곳들은
뒤로가기/홈 버튼이 빠져 있는 경우가 있다.

이걸 전부 수정하라.

대상:
- 인원관리 안의 모든 페이지
- 수업관리 안의 모든 페이지
- 보강관리 안의 모든 페이지
- 매출관리 안의 모든 페이지
- 데이터관리 안의 모든 페이지
- 운영설정 안의 모든 페이지
- 플랫폼설정 안의 모든 페이지
- 더보기에서 이동하는 페이지 포함
- 메신저 페이지도 공통 헤더 규칙 확인

규칙:
- 모든 페이지 상단에 공통 헤더를 강제로 붙인다
- 기존 페이지가 헤더를 가지고 있더라도 공통 규칙과 다르면 통일
- 페이지마다 제각각인 헤더 금지

==================================================
[2. 공통 헤더 구성 통일]

공통 헤더 구성은 아래로 통일

왼쪽:
- 뒤로가기 버튼

오른쪽:
- 홈 버튼

추가:
- 수정/설정/입력 페이지는 저장 버튼 필요 시 홈 옆 또는 헤더 우측에 같이 배치 가능

예:
[←]               [홈]
또는
[←]         [홈] [저장]

중요:
- 관리자 홈 첫 화면에서는 홈 버튼 불필요
- 하위 페이지에서만 공통 헤더 적용
- 버튼 크기/아이콘/간격/위치 전부 통일

==================================================
[3. 뒤로가기 버튼 규칙 점검]

뒤로가기 버튼은 현재 대부분 잘 동작하는 것으로 보이지만
공통 헤더 적용 후 아래를 다시 점검하라.

규칙:
- 페이지 → 직전 팝업 또는 직전 페이지로 이동
- 홈으로 가지 말 것
- 관리자 홈 첫 화면으로 바로 튀면 안 됨
- 페이지 스택이 꼬이지 않게 할 것

==================================================
[4. 홈 버튼 규칙 전면 수정]

현재 핵심 문제:
- 어떤 페이지에서는 홈 버튼이 없고
- 어떤 페이지에서는 홈 버튼이 눌려도 안 먹는다

이걸 전면 수정하라.

홈 버튼 동작 규칙:
- 어느 하위 페이지에서 누르든 무조건 관리자 홈 첫 화면으로 즉시 이동
- 중간 팝업/중간 페이지 스택 정리
- 홈 화면 상태로 복귀
- 홈 스크롤도 첫 위치
- 홈 아이콘/배너 정상 표시

즉:
“홈 버튼 = 관리자 OS 첫 화면 초기화 복귀”

중요:
- 일부 페이지만 작동하면 안 됨
- 전체 페이지에서 전부 동일하게 작동해야 함

==================================================
[5. 홈 버튼 전수 테스트 대상]

반드시 아래 페이지군 전부 테스트할 것

1. 인원관리 계열
   - 회원관리
   - 학부모관리
   - 선생님관리
   - 승인관리
   - 미배정관리
   - 초대관리

2. 수업관리 계열
   - 수업스케줄
   - 반관리
   - 출결관리
   - 수업일지
   - 수강생관리
   - 반이동

3. 보강관리 계열
   - 보강대기
   - 보강배정
   - 보강현황
   - 보강정책

4. 매출관리 계열
   - 월별매출
   - 결제현황
   - 미납관리
   - 정산확인
   - 수업발생
   - 휴무일관리

5. 데이터관리 계열
   - 저장공간현황
   - 계정별사용량
   - 카테고리별사용량
   - 원본데이터삭제
   - 이벤트기록
   - 백업/복구정책

6. 운영설정 계열
7. 플랫폼설정 계열
8. 메신저 페이지
9. 더보기에서 재사용되는 설정 페이지들

==================================================
[6. 홈 버튼 불량 페이지 전부 수정]

전수 테스트 후
홈 버튼이 없는 페이지
홈 버튼이 눌리지 않는 페이지
홈 버튼이 잘못된 위치로 가는 페이지
전부 수정하라.

가능한 원인 점검:
- 기존 페이지 라우팅을 그대로 써서 home action 누락
- 공통 헤더 props 누락
- 화면별 네비게이션 컨텍스트 차이
- 중첩 navigator 문제
- overlay/modal/page 혼용 시 라우팅 누락

중요:
원인을 찾아서 개별 땜질이 아니라
공통 헤더/공통 home action 구조로 정리할 것

==================================================
[7. 홈 배너 크기 축소]

현재 홈 상단 요약 배너는 방향은 좋지만
조금 커서 메인 아이콘 영역이 답답해 보인다.

수정 목표:
- 배너 높이를 소폭 줄인다
- 여백을 다듬는다
- 수치/텍스트 가독성은 유지
- 2x2 카드 구조는 유지

조정 규칙:
- 카드 세로 높이 줄이기
- 내부 패딩 소폭 축소
- 제목/보조문구 간격 조정
- 숫자는 충분히 강조 유지

중요:
작게만, 너무 납작하게 만들지 말 것

==================================================
[8. 메인 아이콘 배치 여유 조정]

현재 메인 아이콘 8개는 방향이 좋다.
다만 더 “여유 있게” 배치되도록 조정하라.

수정 목표:
- 아이콘 카드 사이 간격 약간 확보
- 숨 쉬는 느낌
- 너무 붙어 보이지 않게
- 하지만 한 화면 8개 유지

조정 규칙:
- 4x2 그리드 유지
- 카드 margin/padding 재조정
- 아이콘과 텍스트 간격 통일
- 눌림 영역 충분히 유지

중요:
아이콘 크기 자체는 과도하게 키우지 말고
간격/배치 균형으로 해결할 것

==================================================
[9. 아이콘 뱃지 유지]

현재 아이콘 위 새소식/알림 숫자 뱃지는 좋은 구조이므로 유지하라.

규칙:
- 뱃지 위치 통일
- 숫자 표시 스타일 통일
- 메신저/보강관리/승인관리 등 필요한 곳에 사용 가능
- 뱃지가 아이콘 또는 라벨을 가리지 않게 조정

==================================================
[10. 더보기 내부 페이지도 헤더 점검]

현재 스크린샷 기준 더보기 내부 페이지(설정 메뉴 등)는
기존 페이지 재사용 흔적이 있으므로
공통 헤더/홈 버튼 규칙을 반드시 점검하라.

즉:
- 더보기 > 설정 메뉴 > 세부 페이지
이 라인도 전부 홈/뒤로가기 통일

[16. 절대 금지]

- 페이지마다 제각각 헤더 유지 금지
- 홈 버튼 없는 페이지 남겨두기 금지
- 홈 버튼 눌렀는데 중간 페이지로 가는 것 금지
- 배너를 과도하게 줄여 가독성 망치기 금지
- 아이콘 수 줄이기 금지
- 기능 로직 건드리기 금지

이번 작업은
현재 마음에 드는 관리자 홈 인터페이스를 유지하면서,
모든 아이콘 안 페이지에 공통 뒤로가기/홈 헤더를 붙이고,
홈 버튼이 전체 페이지에서 실제로 작동하도록 전수 수정한 뒤,
배너를 조금 줄이고 아이콘을 더 여유 있게 배치하는 작업이다.
