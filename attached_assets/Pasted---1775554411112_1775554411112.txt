[스윔노트 — 출시 직전 최종 결제/구독 통합 검증 지시문]

지금부터는 기능 추가나 부분 수정이 아니라,
"실제 결제 + 실시간 반영 + 관리자 반영"이 끝까지 완벽하게 동작하는지 검증하는 최종 작업만 진행한다.

이번 작업의 완료 기준은 단 하나다.

“수영장관리자가 실제로 결제하면
선택한 플랜/가격이 App Store 또는 Play Store에서 정확히 결제되고,
결제 직후 구독 상태/인원 제한/용량 제한/영상 허용/화이트라벨 여부가 즉시 반영되며,
수영장관리자 화면과 슈퍼관리자 화면, API, DB가 모두 같은 값을 보여야 한다.”

설명으로 완료 처리하지 말고,
반드시 실제 증거(DB / API / 앱 화면 / 결제 로그)로 검증하라.

━━━━━━━━━━━━━━━━━━
0. 최종 검증 범위
━━━━━━━━━━━━━━━━━━

이번 검증은 아래 4가지를 모두 포함한다.

1. iOS 결제 흐름 검증
2. Android 결제 흐름 검증
3. 슈퍼관리자 수동 변경 흐름 검증
4. RevenueCat 결제와 슈퍼관리자 수동 변경 충돌 검증

이 4개 중 하나라도 빠지면 완료로 보지 않는다.

━━━━━━━━━━━━━━━━━━
1. 결제 전 사전 점검
━━━━━━━━━━━━━━━━━━

반드시 먼저 아래를 표로 제출하라.

[플랜 매핑표]
각 플랜별로 아래 값이 정확히 매칭되는지 정리:
- 앱 표시 플랜명
- 내부 tier 코드
- RevenueCat package id
- RevenueCat entitlement
- iOS product id
- Android product id
- App Store 표시 가격
- Play Store 표시 가격
- subscription_plans.price_per_month
- member_limit
- storage_mb
- display_storage
- video enabled 여부
- white label enabled 여부

필수 플랜:
- starter
- basic
- standard
- center_200
- advance
- pro
- max

중요:
앱 표시명, DB 플랜명, RevenueCat 상품, 스토어 가격이 모두 일치해야 한다.
하나라도 다르면 실제 결제 테스트 시작 금지.

━━━━━━━━━━━━━━━━━━
2. App Store / Play Store 결제 진입 검증
━━━━━━━━━━━━━━━━━━

반드시 iOS와 Android를 분리해서 각각 테스트하라.

[iOS]
1. 수영장관리자 계정 로그인
2. 구독 화면 진입
3. 각 플랜 선택
4. App Store 인앱결제 팝업이 뜨는지 확인
5. 이상한 웹페이지/죽은 화면/임시 페이지로 이동하지 않는지 확인
6. 선택한 플랜명과 가격이 App Store 팝업에 정확히 보이는지 확인

[Android]
1. 수영장관리자 계정 로그인
2. 구독 화면 진입
3. 각 플랜 선택
4. Play Store 인앱결제 팝업이 뜨는지 확인
5. 이상한 웹페이지/죽은 화면/임시 페이지로 이동하지 않는지 확인
6. 선택한 플랜명과 가격이 Play Store 팝업에 정확히 보이는지 확인

반드시 각 OS에서 최소 3개 플랜 테스트:
- basic 또는 standard
- pro
- max

제출:
- iOS 각 플랜 결제 진입 캡처
- Android 각 플랜 결제 진입 캡처
- 플랜명/가격 일치 여부 표

━━━━━━━━━━━━━━━━━━
3. 실제 결제 성공 후 즉시 반영 검증
━━━━━━━━━━━━━━━━━━

목표:
결제 완료 즉시 아래 흐름이 한 번에 이어져야 한다.

결제 성공
→ RevenueCat entitlement 반영
→ 서버 동기화
→ applySubscriptionState 실행
→ swimming_pools 현재 상태 반영
→ resolver 계산
→ /billing/status 반영
→ /pools/my 반영
→ /super/operators/:id 반영
→ 수영장관리자 앱 화면 반영
→ 슈퍼관리자 화면 반영

각 플랜 결제 후 반드시 아래를 제출:
1. RevenueCat customer info 또는 entitlement 결과
2. swimming_pools 실제 DB 값
3. /billing/status 응답 JSON
4. /pools/my 응답 JSON
5. /super/operators/:id 응답 JSON
6. 수영장관리자 구독 화면 캡처
7. 슈퍼관리자 수영장 상세 화면 캡처

비교 필드:
- subscription_tier
- subscription_status
- subscription_source
- plan_name
- member_limit
- storage_mb
- display_storage
- video_storage_limit_mb
- white_label_enabled
- next_billing_at

정상 기준:
- subscription_source = revenuecat
- plan_name 정확
- member_limit 정확
- display_storage 정확
- 슈퍼관리자/수영장관리자/API/DB 모두 동일 값

━━━━━━━━━━━━━━━━━━
4. 인원 제한 / 용량 제한 실시간 반영 검증
━━━━━━━━━━━━━━━━━━

결제 성공 직후 단순 화면 표시만 확인하지 말고,
실제 운영 제한이 바뀌었는지 반드시 검증하라.

[인원 제한]
예:
- free에서 pro 결제
- 즉시 회원 한도 10 → 500으로 변경
- 실제 회원 추가 가능 여부 확인

[용량 제한]
예:
- free에서 pro 결제
- 500MB → 130GB로 즉시 반영
- 저장 용량 표시 변경 확인
- 업로드 제한 기준 변경 확인

[영상 기능]
- 비허용 플랜 → 허용 플랜 전환 시
- video_storage_limit_mb 변경 확인
- 실제 영상 업로드 가능 여부 확인

[화이트라벨]
- 허용 플랜 여부에 따라 즉시 반영 확인

제출:
- DB 값
- API 응답
- 실제 동작 결과
- 앱 화면 캡처

중요:
“표시만 변경”은 실패다.
실제 제한값이 같이 바뀌어야 통과다.

━━━━━━━━━━━━━━━━━━
5. 슈퍼관리자 수동 변경 검증
━━━━━━━━━━━━━━━━━━

결제와 별도로, 슈퍼관리자가 결제 없이 수영장 플랜을 직접 바꿀 수 있어야 한다.

반드시 아래 시나리오를 실제로 검증:
- free → pro
- pro → max
- max → free

각 단계마다 즉시 확인:
- swimming_pools DB 값
- /billing/status
- /pools/my
- /super/operators/:id
- 수영장관리자 앱 화면
- 슈퍼관리자 화면

비교 필드:
- subscription_tier
- subscription_status
- subscription_source = manual
- plan_name
- member_limit
- display_storage
- video_storage_limit_mb
- white_label_enabled

정상 기준:
- PATCH 응답 전에 DB 반영 완료
- 3개 API 동일
- 수영장관리자 화면 즉시 반영
- 슈퍼관리자 화면도 즉시 반영

━━━━━━━━━━━━━━━━━━
6. RevenueCat vs 수동 변경 충돌 검증
━━━━━━━━━━━━━━━━━━

반드시 아래 순서로 테스트:
1. 수영장관리자가 유료 결제하여 revenuecat 활성 상태 만들기
2. 슈퍼관리자가 수동으로 free 또는 다른 플랜 지정
3. 최종 적용값이 manual 우선인지 확인
4. manual 해제 기능이 있다면 해제 후 revenuecat 상태로 복귀하는지 확인
5. manual 해제 기능이 없으면 현재 제한사항으로 명확히 보고

제출:
- 충돌 시 최종 우선순위 표
- DB 값
- API 값
- 앱 화면 캡처

━━━━━━━━━━━━━━━━━━
7. 복원 / 재실행 / 재로그인 검증
━━━━━━━━━━━━━━━━━━

반드시 아래도 확인:
1. 결제 후 앱 완전 종료 → 재실행
2. 로그아웃 → 재로그인
3. Restore Purchases 실행
4. 기존 구독 상태가 유지/복구되는지 확인

정상 기준:
- plan_name 유지
- member_limit 유지
- display_storage 유지
- source/status 정상 유지

━━━━━━━━━━━━━━━━━━

- DB

- /pools/my
- /super/operators/:id
- 수영장관리자 화면
